MySQL 的 `WHERE` 子句用于在查询数据时筛选符合特定条件的记录，是数据查询中最常用的过滤手段。以下是关于 `WHERE` 子句的核心知识：


### 1. `WHERE` 子句的基本用法
`WHERE` 子句通常跟在 `FROM` 子句之后，用于指定筛选条件，语法如下：
```sql
SELECT 字段名 
FROM 表名 
WHERE 条件表达式;
```

**示例**：
```sql
-- 查询年龄大于18的用户
SELECT username, age FROM users 
WHERE age > 18;
```


### 2. 常用条件运算符
`WHERE` 子句支持多种运算符，用于构建条件表达式：

| 运算符         | 说明                                  | 示例                                  |
|----------------|---------------------------------------|---------------------------------------|
| `=`            | 等于                                  | `WHERE status = 1`                    |
| `!=` / `<>`    | 不等于                                | `WHERE gender != 'male'`              |
| `>` / `<`      | 大于 / 小于                           | `WHERE price > 100`                   |
| `>=` / `<=`    | 大于等于 / 小于等于                   | `WHERE created_at <= '2023-12-31'`    |
| `BETWEEN...AND...` | 在某个范围之间（包含边界）          | `WHERE age BETWEEN 18 AND 30`         |
| `IN(值1,值2...)` | 在指定集合中                          | `WHERE category IN ('手机','电脑')`   |
| `NOT IN(...)`  | 不在指定集合中                        | `WHERE status NOT IN (0, 2)`          |
| `LIKE`         | 模糊匹配（配合 `%` 或 `_`）           | `WHERE username LIKE '张%'`           |
| `IS NULL`      | 为空值                                | `WHERE email IS NULL`                 |
| `IS NOT NULL`  | 不为空值                              | `WHERE phone IS NOT NULL`             |


### 3. 逻辑运算符（组合多个条件）
可以使用逻辑运算符将多个条件组合起来：

| 逻辑运算符 | 说明                 | 示例                                     |
|------------|----------------------|------------------------------------------|
| `AND`      | 同时满足所有条件     | `WHERE age > 18 AND status = 1`          |
| `OR`       | 满足任一条件         | `WHERE category = '图书' OR price < 50`  |
| `NOT`      | 取反条件             | `WHERE NOT (age < 18)`                   |

**示例**：
```sql
-- 查询年龄18-30岁之间，且状态为正常（1）的女性用户
SELECT * FROM users 
WHERE age BETWEEN 18 AND 30 
  AND status = 1 
  AND gender = 'female';
```


### 4. 特殊场景用法

#### （1）模糊查询（`LIKE`）
- `%`：匹配任意长度的字符（包括0个字符）
- `_`：匹配单个字符

```sql
-- 用户名以"李"开头
SELECT * FROM users WHERE username LIKE '李%';

-- 用户名第二个字是"小"
SELECT * FROM users WHERE username LIKE '_小%';

-- 邮箱包含"example"
SELECT * FROM users WHERE email LIKE '%example%';
```

#### （2）空值判断（`IS NULL`）
注意：`NULL` 表示“未知”，不能用 `=` 或 `!=` 判断，必须用 `IS NULL` 或 `IS NOT NULL`：
```sql
-- 查询未填写邮箱的用户
SELECT * FROM users WHERE email IS NULL;

-- 查询已填写手机号的用户
SELECT * FROM users WHERE phone IS NOT NULL;
```

#### （3）范围查询（`BETWEEN` 与 `IN`）
```sql
-- 查询价格在100到500之间的商品
SELECT * FROM products WHERE price BETWEEN 100 AND 500;

-- 查询分类为"家电"、"数码"或"服装"的商品
SELECT * FROM products WHERE category IN ('家电', '数码', '服装');
```


### 5. `WHERE` 子句的注意事项
- **数据类型匹配**：条件中的值必须与字段数据类型一致，例如字符串需用单引号包裹
- **优先级问题**：`AND` 优先级高于 `OR`，复杂条件建议用括号 `()` 明确优先级：
  ```sql
  -- 正确：查询（年龄>18且状态=1）或（VIP用户）
  SELECT * FROM users WHERE (age > 18 AND status = 1) OR is_vip = 1;
  ```
- **索引使用**：`WHERE` 子句中的字段若建立索引，可大幅提高查询效率
- **不支持聚合函数**：`WHERE` 不能直接使用聚合函数（如 `COUNT()`、`AVG()`），需用 `HAVING` 子句
- **对 NULL 的处理**：`NULL` 与任何值比较（包括 `NULL` 本身）结果都为 `NULL`，需用 `IS NULL` 判断


`WHERE` 子句是筛选数据的核心工具，灵活运用各种条件运算符和逻辑组合，能精确获取所需数据。在实际查询中，合理设计 `WHERE` 条件也是优化查询性能的关键。
