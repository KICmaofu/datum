MySQL 的 `ALTER` 命令用于修改已存在的数据库表结构，包括添加/删除列、修改列类型、添加约束、重命名表等。它是数据库维护和演进的重要工具。以下是关于 `ALTER` 命令的核心知识：


### 1. 基本语法
```sql
ALTER TABLE 表名
操作类型 [参数];
```

**说明**：
- `ALTER TABLE` 是核心关键字，后面跟要修改的表名
- 支持多种操作类型，可组合使用（用逗号分隔）


### 2. 常用操作类型及示例
假设有初始表 `users`：
```sql
CREATE TABLE users (
  id INT,
  name VARCHAR(50),
  age INT
);
```


#### ① 添加列（`ADD`）
```sql
-- 添加单列
ALTER TABLE users
ADD email VARCHAR(100);

-- 添加多列
ALTER TABLE users
ADD phone VARCHAR(20),
ADD address TEXT;

-- 在指定位置添加列（FIRST 或 AFTER）
ALTER TABLE users
ADD created_at DATETIME AFTER name; -- 添加在 name 列之后
```


#### ② 修改列（`MODIFY` 或 `CHANGE`）
- `MODIFY`：只能修改列的数据类型或属性
- `CHANGE`：可以修改列名和数据类型

```sql
-- 修改列类型和约束（MODIFY）
ALTER TABLE users
MODIFY age TINYINT UNSIGNED; -- 将 age 改为无符号 tinyint

-- 修改列名和类型（CHANGE）
ALTER TABLE users
CHANGE COLUMN email user_email VARCHAR(150); -- 列名从 email 改为 user_email
```


#### ③ 删除列（`DROP`）
```sql
-- 删除单列
ALTER TABLE users
DROP COLUMN address;

-- 删除多列
ALTER TABLE users
DROP phone,
DROP created_at;
```


#### ④ 添加约束
```sql
-- 添加主键
ALTER TABLE users
ADD PRIMARY KEY (id);

-- 添加唯一约束
ALTER TABLE users
ADD UNIQUE (user_email);

-- 添加外键（假设已有 orders 表）
ALTER TABLE orders
ADD CONSTRAINT fk_orders_users
FOREIGN KEY (user_id) REFERENCES users(id);

-- 添加非空约束
ALTER TABLE users
MODIFY name VARCHAR(50) NOT NULL;
```


#### ⑤ 删除约束
```sql
-- 删除主键（无需指定名称）
ALTER TABLE users
DROP PRIMARY KEY;

-- 删除唯一约束（需知道约束名）
ALTER TABLE users
DROP INDEX user_email; -- 唯一约束名通常与列名相同

-- 删除外键（需知道约束名）
ALTER TABLE orders
DROP FOREIGN KEY fk_orders_users;
```


#### ⑥ 重命名表（`RENAME TO`）
```sql
ALTER TABLE users
RENAME TO system_users;
```


#### ⑦ 修改表的其他属性
```sql
-- 添加默认值
ALTER TABLE users
ALTER COLUMN age SET DEFAULT 18;

-- 删除默认值
ALTER TABLE users
ALTER COLUMN age DROP DEFAULT;

-- 修改表的存储引擎
ALTER TABLE users
ENGINE = InnoDB;
```


### 3. 注意事项
1. **数据影响**：
   - 修改列类型可能导致数据截断（如 `INT` 改为 `TINYINT` 时大数值会丢失）
   - 删除列会永久丢失该列数据，操作前务必备份
   - 添加非空约束时，需确保现有数据无 `NULL` 值

2. **性能影响**：
   - 对大表执行 `ALTER` 操作可能锁表，导致业务中断
   - MySQL 5.6+ 支持 `Online DDL`（部分操作可在线执行不锁表）
   - 复杂修改建议在低峰期执行

3. **约束依赖**：
   - 有外键依赖的表不能直接删除或修改关联列
   - 需先删除外键约束，再修改关联列

4. **权限要求**：
   - 需要表的 `ALTER` 权限


### 4. 适用场景
- 业务需求变更导致表结构调整（如新增字段）
- 优化表设计（如修改列类型节省空间）
- 添加约束保证数据完整性
- 数据库迁移或升级时的结构调整


`ALTER` 命令是数据库演进的重要工具，但由于其可能对数据和性能产生影响，使用时必须谨慎。操作前建议：备份数据、在测试环境验证、选择合适的执行时间。
