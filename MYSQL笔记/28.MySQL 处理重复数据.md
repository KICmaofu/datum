在 MySQL 中，重复数据指表中存在一行或多行完全相同（或关键列相同）的记录，可能导致查询结果错误、统计偏差等问题。处理重复数据的核心是**预防重复**和**清除已有重复**，以下是具体方法和注意事项：


### 一、重复数据的类型
1. **完全重复**：所有列的值都相同（极少出现，通常是误操作导致）。  
2. **部分重复**：关键列（如 `username`、`email` 等业务唯一标识）的值相同，其他列可能不同（更常见，如用户重复注册）。  


### 二、预防重复数据（最佳实践）
通过数据库约束主动避免重复数据插入，是最高效的方式：

#### 1. 主键约束（PRIMARY KEY）
主键列的值必须唯一且非空，适合作为唯一标识（如 `id`），天然防止完全重复。  
```sql
CREATE TABLE users (
  id INT PRIMARY KEY AUTO_INCREMENT,  -- 主键自动唯一
  username VARCHAR(50) NOT NULL,
  email VARCHAR(100) NOT NULL
);
```


#### 2. 唯一约束（UNIQUE）
对非主键的关键列添加唯一约束，强制其值不重复（如 `email`、`username`）：  
```sql
-- 创建表时添加
CREATE TABLE users (
  id INT PRIMARY KEY AUTO_INCREMENT,
  username VARCHAR(50) NOT NULL UNIQUE,  -- 用户名唯一
  email VARCHAR(100) NOT NULL UNIQUE     -- 邮箱唯一
);

-- 已有表添加唯一约束
ALTER TABLE users ADD UNIQUE (email);
```
- 插入重复值时，MySQL 会报 `Duplicate entry` 错误，阻止插入。  
- 可对多列组合添加唯一约束（如 `(username, phone)` 联合唯一）：  
  ```sql
  ALTER TABLE users ADD UNIQUE (username, phone);
  ```


#### 3. 插入时忽略重复（INSERT IGNORE）
若希望插入重复数据时不报错，而是直接忽略，可用 `INSERT IGNORE`：  
```sql
INSERT IGNORE INTO users (username, email) 
VALUES ('alice', 'alice@example.com');  -- 若 username 重复，此语句无操作
```


#### 4. 重复时更新（INSERT ... ON DUPLICATE KEY UPDATE）
若希望插入重复数据时，自动更新已有记录的其他字段（需依赖唯一约束或主键）：  
```sql
INSERT INTO users (username, email) 
VALUES ('alice', 'alice@new.com')
ON DUPLICATE KEY UPDATE email = VALUES(email);  -- 若 username 重复，更新 email
```


### 三、清除已有重复数据
若表中已存在重复数据，需先查询并删除，步骤如下：

#### 1. 查找重复数据
根据“部分重复”的关键列（如 `email`）查询重复记录：  
```sql
-- 查找 email 重复的记录
SELECT email, COUNT(*) AS count 
FROM users 
GROUP BY email 
HAVING count > 1;

-- 查看重复的具体记录
SELECT * FROM users 
WHERE email IN (
  SELECT email FROM users GROUP BY email HAVING COUNT(*) > 1
) 
ORDER BY email;
```


#### 2. 删除重复数据（保留一条）
需保留一条重复记录（通常保留最早或最新的），方法如下：

##### 方法1：通过主键删除（适合有主键的表）
假设 `id` 是主键，保留最小 `id` 的记录，删除其他重复项：  
```sql
DELETE FROM users 
WHERE id NOT IN (
  SELECT min_id FROM (
    SELECT MIN(id) AS min_id FROM users GROUP BY email  -- 按 email 分组，取最小 id
  ) AS temp  -- 子查询需别名，否则报错
);
```
- 外层 `NOT IN` 排除需保留的 `id`，删除其余重复记录。  


##### 方法2：通过临时表重建（适合无主键的表）
若表无主键，可先将去重后的数据存入临时表，再清空原表并导回：  
```sql
-- 1. 创建临时表，存储去重后的数据（保留每条重复记录的第一行）
CREATE TEMPORARY TABLE temp_users AS
SELECT DISTINCT * FROM users;  -- DISTINCT 去重所有列（完全重复）

-- 若仅去重部分列（如 email），保留最新的一条：
CREATE TEMPORARY TABLE temp_users AS
SELECT * FROM users 
WHERE id IN (
  SELECT MAX(id) FROM users GROUP BY email  -- 按 email 分组，取最大 id（最新）
);

-- 2. 清空原表
TRUNCATE TABLE users;

-- 3. 将临时表数据导回原表
INSERT INTO users SELECT * FROM temp_users;

-- 4. 删除临时表
DROP TEMPORARY TABLE temp_users;
```


##### 方法3：使用窗口函数（MySQL 8.0+）
通过 `ROW_NUMBER()` 标记重复记录，删除序号大于 1 的行：  
```sql
DELETE FROM users 
WHERE id IN (
  SELECT id FROM (
    SELECT 
      id,
      ROW_NUMBER() OVER (PARTITION BY email ORDER BY id) AS rn  -- 按 email 分组，按 id 排序
    FROM users
  ) AS temp 
  WHERE rn > 1  -- 序号 >1 的为重复记录
);
```


### 四、注意事项
1. **操作前备份**：删除重复数据前，务必备份表（如 `CREATE TABLE users_backup AS SELECT * FROM users;`），防止误删。  
2. **性能影响**：大表删除重复数据可能耗时较长，建议分批次操作或在低峰期执行。  
3. **唯一约束与索引**：清除重复后，应立即添加唯一约束（如 `UNIQUE`），防止再次产生重复。  
4. **自增主键的处理**：删除重复数据后，自增主键（`AUTO_INCREMENT`）不会自动重置，若需重置可执行：  
   ```sql
   ALTER TABLE users AUTO_INCREMENT = 1;  -- 下一个自增值会从当前最大 id +1 开始
   ```


### 五、总结
处理重复数据的核心原则是“**预防为主，清除为辅**”：  
- 优先通过 `PRIMARY KEY` 或 `UNIQUE` 约束从源头避免重复。  
- 已有重复数据时，先查询确认重复范围，再通过保留唯一记录的方式删除冗余数据。  
- 操作后需通过约束或业务逻辑（如前端校验）防止重复再次产生。
