MySQL 的 `GROUP BY` 语句用于将查询结果按照一个或多个列进行分组，通常与聚合函数（如 `COUNT()`、`SUM()`、`AVG()` 等）配合使用，以实现对每组数据的汇总分析。以下是关于 `GROUP BY` 的核心知识：


### 1. 基本语法
```sql
SELECT 列1, 列2, 聚合函数(列)
FROM 表名
[WHERE 条件]
GROUP BY 列1, 列2, ...
[HAVING 条件]; -- 用于过滤分组后的结果
```

**说明**：
- `GROUP BY` 后面的列称为"分组列"，查询结果会按这些列的值进行分组
- `SELECT` 子句中只能包含分组列和聚合函数（除非启用了特殊模式）
- `HAVING` 用于过滤分组后的结果（类似 `WHERE`，但 `WHERE` 过滤的是原始数据）


### 2. 基本示例
假设有 `orders` 表：

| id  | customer_id | product  | amount | order_date  |
|-----|-------------|----------|--------|-------------|
| 1   | 101         | 手机     | 3999   | 2023-01-15  |
| 2   | 102         | 电脑     | 5999   | 2023-01-16  |
| 3   | 101         | 平板     | 2999   | 2023-01-17  |
| 4   | 103         | 手机     | 3999   | 2023-01-18  |
| 5   | 102         | 耳机     | 799    | 2023-01-19  |

#### 示例 1：单列分组
```sql
-- 按客户分组，统计每个客户的订单数量
SELECT customer_id, COUNT(*) AS order_count
FROM orders
GROUP BY customer_id;
```
结果：
| customer_id | order_count |
|-------------|-------------|
| 101         | 2           |
| 102         | 2           |
| 103         | 1           |

#### 示例 2：多列分组
```sql
-- 按客户和产品分组，统计每种产品的购买次数和总金额
SELECT customer_id, product, 
       COUNT(*) AS buy_count, 
       SUM(amount) AS total_amount
FROM orders
GROUP BY customer_id, product;
```

#### 示例 3：结合 `HAVING` 过滤分组
```sql
-- 筛选订单数量 >= 2 的客户
SELECT customer_id, COUNT(*) AS order_count
FROM orders
GROUP BY customer_id
HAVING order_count >= 2; -- 注意：这里不能用 WHERE，因为 WHERE 不支持聚合函数
```


### 3. 与聚合函数的配合
常用聚合函数：
- `COUNT()`：统计行数
- `SUM()`：求和
- `AVG()`：求平均值
- `MAX()`：求最大值
- `MIN()`：求最小值

示例：
```sql
-- 按产品分组，计算每种产品的销售总量、平均价格和最高订单金额
SELECT product,
       COUNT(*) AS sales_count,
       AVG(amount) AS avg_price,
       MAX(amount) AS max_amount
FROM orders
GROUP BY product;
```


### 4. 关键规则
1. **`SELECT` 与 `GROUP BY` 的关系**：
   - `SELECT` 中出现的非聚合列必须全部出现在 `GROUP BY` 中
   ```sql
   -- 正确示例
   SELECT customer_id, product, COUNT(*) 
   FROM orders 
   GROUP BY customer_id, product; -- 包含了所有非聚合列
   
   -- 错误示例（非聚合列未在 GROUP BY 中）
   SELECT customer_id, product, COUNT(*) 
   FROM orders 
   GROUP BY customer_id; -- 缺少 product 列
   ```

2. **`WHERE` 与 `HAVING` 的区别**：
   | 特性       | `WHERE`                  | `HAVING`                  |
   |------------|--------------------------|---------------------------|
   | 作用时机   | 分组前过滤原始数据       | 分组后过滤分组结果        |
   | 支持内容   | 不能使用聚合函数         | 可以使用聚合函数和别名    |
   | 执行顺序   | 在 `GROUP BY` 之前执行   | 在 `GROUP BY` 之后执行    |

3. **与 `ORDER BY` 配合**：
   ```sql
   -- 按产品分组后，再按销售总量降序排列
   SELECT product, COUNT(*) AS sales_count
   FROM orders
   GROUP BY product
   ORDER BY sales_count DESC;
   ```


### 5. 高级用法
#### ① 分组后取每组的第一条记录
```sql
-- 按客户分组，获取每个客户的第一笔订单
SELECT * FROM (
  SELECT * FROM orders 
  ORDER BY order_date ASC
) AS ordered_orders
GROUP BY customer_id;
```

#### ② 与 `WITH ROLLUP` 配合（小计和总计）
```sql
-- 按产品分组统计，并添加总计行
SELECT product, SUM(amount) AS total_amount
FROM orders
GROUP BY product WITH ROLLUP;
```


### 6. 注意事项
1. **性能优化**：
   - 对分组列建立索引可显著提高 `GROUP BY` 性能
   - 避免对大量数据进行不必要的分组

2. **`ONLY_FULL_GROUP_BY` 模式**：
   - MySQL 5.7+ 默认启用该模式，严格检查 `SELECT` 与 `GROUP BY` 的列对应关系
   - 禁用该模式可能导致非分组列的取值不可预测

3. **空值处理**：
   - `GROUP BY` 会将 `NULL` 值视为相同的组进行分组


### 7. 适用场景
- 统计分析（如各部门人数、各产品销量）
- 数据汇总（如每日/每月销售额汇总）
- 去重统计（如不同类别下的记录数）
- 分组过滤（如筛选订单量超过 10 的客户）


`GROUP BY` 是数据聚合分析的核心工具，通过与聚合函数和 `HAVING` 子句的配合，可以实现复杂的统计分析需求。掌握其使用规则和性能优化技巧，对于数据分析类查询至关重要。
