在 MySQL 中，插入数据是向数据表中添加新记录的操作，是数据库交互的基本操作之一。以下是关于 MySQL 插入数据的核心知识：


### 1. 插入数据的基本语法
#### （1）完整插入（指定所有字段）
```sql
INSERT INTO 表名 (字段1, 字段2, ..., 字段n)
VALUES (值1, 值2, ..., 值n);
```

#### （2）简化插入（不指定字段，需按表结构顺序插入所有字段值）
```sql
INSERT INTO 表名
VALUES (值1, 值2, ..., 值n);
```
> 注意：不推荐此方式，若表结构变更（如新增字段），会导致插入失败。


### 2. 插入多条数据
可以一次性插入多条记录，比单条插入更高效：
```sql
INSERT INTO 表名 (字段1, 字段2, 字段3)
VALUES 
  (值1, 值2, 值3),
  (值4, 值5, 值6),
  (值7, 值8, 值9);
```


### 3. 插入指定字段（忽略部分字段）
如果某些字段有默认值（`DEFAULT`）或自增属性（`AUTO_INCREMENT`），可以省略这些字段：
```sql
-- 假设 users 表有 id(自增)、username、password、created_at(默认当前时间) 字段
INSERT INTO users (username, password)
VALUES ('zhangsan', '123456');
```
此时 `id` 会自动增长，`created_at` 会使用默认值。


### 4. 插入查询结果（从其他表复制数据）
可以将一个查询的结果插入到目标表中（需保证字段匹配）：
```sql
-- 从 user_backup 表复制数据到 users 表
INSERT INTO users (username, email, status)
SELECT username, email, 1  -- 1 是固定值，表示状态正常
FROM user_backup
WHERE create_time > '2023-01-01';
```


### 5. 插入数据的注意事项
#### （1）数据类型匹配
插入的值必须与字段的数据类型一致：
- 字符串类型需用单引号 `''` 包裹（如 `'张三'`）
- 日期时间类型需用单引号包裹，格式需符合 MySQL 要求（如 `'2023-10-01 12:30:00'`）
- 数值类型直接写值（如 `25`、`3.14`）

#### （2）约束遵守
- 非空约束（`NOT NULL`）：必须为字段提供值
- 唯一约束（`UNIQUE`）：插入的值不能与表中已有值重复
- 外键约束（`FOREIGN KEY`）：插入的外键值必须在关联表的主键中存在

#### （3）自增字段处理
- 自增字段（`AUTO_INCREMENT`）通常是主键，插入时可省略，MySQL 会自动生成唯一值
- 若手动指定自增字段值，需确保不重复且大于当前最大值

#### （4）特殊字符处理
字符串中包含单引号时，需用转义符 `\` 处理或使用双单引号：
```sql
-- 插入包含单引号的字符串
INSERT INTO users (username) VALUES ('O\'Neil');  -- 使用转义符
INSERT INTO users (username) VALUES ('O''Neil');  -- 使用双单引号
```


### 6. 查看插入结果
```sql
-- 查看刚插入的记录（假设 id 是自增主键）
SELECT * FROM 表名 WHERE id = LAST_INSERT_ID();
```
`LAST_INSERT_ID()` 函数返回当前会话最后一次插入的自增ID。


### 7. 插入数据的优化
- 批量插入：优先使用多条数据一次性插入（`VALUES` 后跟多个值列表），减少网络交互
- 关闭自动提交：在大量插入时，可先执行 `SET autocommit = 0`，插入完成后 `COMMIT`，提高效率
- 禁用索引：临时禁用非主键索引（插入后重建），适合超大批量数据插入


插入数据是数据库操作的基础，合理使用批量插入和遵循数据约束，能保证数据准确性和操作效率。在生产环境中，对于重要数据的插入，建议通过事务（`BEGIN`/`COMMIT`）确保原子性。
