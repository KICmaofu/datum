MySQL 的 `ORDER BY` 语句用于对查询结果集按照一个或多个列进行排序，是数据查询中常用的排序工具。以下是关于 `ORDER BY` 的核心知识：


### 1. 基本语法
```sql
SELECT 列1, 列2, ...
FROM 表名
[WHERE 条件]
ORDER BY 列名1 [ASC|DESC], 列名2 [ASC|DESC], ...;
```

**说明**：
- `ORDER BY` 通常放在查询语句的最后（`LIMIT` 除外）
- `ASC` 表示升序（默认，可省略）
- `DESC` 表示降序


### 2. 基本示例
假设有 `students` 表：

| id  | name   | age  | score |
|-----|--------|------|-------|
| 1   | Alice  | 20   | 90    |
| 2   | Bob    | 22   | 85    |
| 3   | Cindy  | 19   | 95    |
| 4   | David  | 20   | 88    |

#### 示例 1：单列排序（升序）
```sql
-- 按年龄升序排列（默认 ASC）
SELECT * FROM students
ORDER BY age;
```

#### 示例 2：单列排序（降序）
```sql
-- 按分数降序排列
SELECT * FROM students
ORDER BY score DESC;
```

#### 示例 3：多列排序
```sql
-- 先按年龄升序，年龄相同则按分数降序
SELECT * FROM students
ORDER BY age ASC, score DESC;
```


### 3. 高级用法
#### ① 按列位置排序（不推荐）
可以使用列在 `SELECT` 中的位置索引（从 1 开始）排序：
```sql
-- 按第 3 列（age）升序，第 4 列（score）降序
SELECT id, name, age, score FROM students
ORDER BY 3, 4 DESC;
```
**注意**：不推荐使用，当 `SELECT` 列顺序改变时会导致排序逻辑错误。

#### ② 按表达式排序
```sql
-- 按分数的 1.2 倍计算后排序
SELECT name, score, score * 1.2 AS adjusted_score
FROM students
ORDER BY adjusted_score DESC;
```

#### ③ 按别名排序
```sql
-- 按别名排序（与上面示例等效）
SELECT name, score, score * 1.2 AS adjusted_score
FROM students
ORDER BY adjusted_score DESC;
```

#### ④ 按 NULL 值排序
MySQL 中 `NULL` 被视为最小值，升序时排在最前，降序时排在最后：
```sql
-- 处理包含 NULL 的列排序
SELECT * FROM products
ORDER BY price DESC; -- NULL 会排在最后
```


### 4. 与其他语句的配合
#### ① 与 `WHERE` 配合
```sql
-- 先筛选年龄 > 19 的学生，再按分数排序
SELECT * FROM students
WHERE age > 19
ORDER BY score DESC;
```

#### ② 与 `LIMIT` 配合（常用）
```sql
-- 获取分数最高的 2 名学生
SELECT * FROM students
ORDER BY score DESC
LIMIT 2;
```

#### ③ 与 `GROUP BY` 配合
```sql
-- 按班级分组后，每组按平均分降序
SELECT class_id, AVG(score) AS avg_score
FROM students
GROUP BY class_id
ORDER BY avg_score DESC;
```


### 5. 注意事项
1. **性能影响**：
   - 对未建立索引的列排序会触发全表扫描和文件排序（`Using filesort`），效率较低
   - 对索引列排序可利用索引顺序，提高效率
   - 大数据量排序可能消耗较多内存和 CPU

2. **排序规则**：
   - 字符串排序取决于数据库的 collation（排序规则）
   - 例如 `utf8_general_ci` 不区分大小写，`utf8_bin` 区分大小写

3. **避免过度排序**：
   - 只对必要的列排序
   - 分页查询时合理使用 `LIMIT` 减少排序数据量


### 6. 常见使用场景
- 排行榜功能（如成绩排名、销量排名）
- 按时间顺序展示数据（最新/最早记录）
- 字母序排列（如用户名、产品名称）
- 分页查询（配合 `LIMIT` 实现分页）


`ORDER BY` 是实现数据有序展示的核心工具，合理使用不仅能提升用户体验，还能通过索引优化提高查询性能。在实际应用中，应根据业务需求选择合适的排序字段和方式。
