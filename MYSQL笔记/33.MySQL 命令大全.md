MySQL 命令涵盖数据库管理、表操作、数据查询、用户权限等多个方面，按使用场景可分为**客户端命令**（在 `mysql` 交互模式中使用）和**SQL 语句命令**（对数据库执行操作）。以下是常用命令的分类汇总：


### 一、客户端连接与退出命令
用于连接 MySQL 服务器或退出客户端。

| 命令                  | 作用                                  | 示例                                  |
|-----------------------|---------------------------------------|---------------------------------------|
| `mysql -u 用户名 -p`   | 连接本地 MySQL 服务器（需输入密码）   | `mysql -u root -p`                    |
| `mysql -h 主机 -P 端口 -u 用户名 -p` | 连接远程服务器                        | `mysql -h 192.168.1.1 -P 3306 -u root -p` |
| `exit` 或 `quit` 或 `\q` | 退出 MySQL 客户端                     | `exit`                                |


### 二、数据库管理命令
用于创建、查看、删除数据库，切换数据库等。

| 命令                                  | 作用                                  | 示例                                  |
|---------------------------------------|---------------------------------------|---------------------------------------|
| `SHOW DATABASES;`                     | 查看所有数据库                        | `SHOW DATABASES;`                     |
| `CREATE DATABASE 库名 [CHARACTER SET 编码];` | 创建数据库（指定编码，如 utf8mb4） | `CREATE DATABASE test CHARACTER SET utf8mb4;` |
| `USE 库名;`                           | 切换到指定数据库（后续操作默认在此库） | `USE test;`                           |
| `DROP DATABASE [IF EXISTS] 库名;`      | 删除数据库（谨慎使用）                | `DROP DATABASE IF EXISTS test;`       |
| `SHOW CREATE DATABASE 库名;`          | 查看数据库的创建语句（含编码等信息）  | `SHOW CREATE DATABASE test;`          |


### 三、表结构操作命令
用于创建、修改、查看、删除表结构。

| 命令                                  | 作用                                  | 示例                                  |
|---------------------------------------|---------------------------------------|---------------------------------------|
| `SHOW TABLES;`                        | 查看当前数据库中的所有表              | `SHOW TABLES;`                        |
| `CREATE TABLE 表名 (列定义...);`       | 创建表（列定义含字段名、类型、约束）  | ```CREATE TABLE users (id INT PRIMARY KEY AUTO_INCREMENT, username VARCHAR(50) NOT NULL);``` |
| `DESCRIBE 表名;` 或 `DESC 表名;`      | 查看表结构（字段名、类型、约束等）    | `DESC users;`                         |
| `SHOW CREATE TABLE 表名;`             | 查看表的完整创建语句（含索引、引擎等） | `SHOW CREATE TABLE users;`            |
| `ALTER TABLE 表名 操作;`              | 修改表结构（添加/删除列、修改类型等） | - 添加列：`ALTER TABLE users ADD email VARCHAR(100);` <br> - 修改列类型：`ALTER TABLE users MODIFY email VARCHAR(200);` <br> - 删除列：`ALTER TABLE users DROP COLUMN email;` |
| `DROP TABLE [IF EXISTS] 表名;`        | 删除表（谨慎使用）                    | `DROP TABLE IF EXISTS users;`         |
| `TRUNCATE TABLE 表名;`                | 清空表数据（保留结构，速度比 DELETE 快） | `TRUNCATE TABLE users;`               |


### 四、数据操作命令（DML）
用于插入、查询、更新、删除表中的数据。

| 命令                                  | 作用                                  | 示例                                  |
|---------------------------------------|---------------------------------------|---------------------------------------|
| `INSERT INTO 表名 (列1, 列2...) VALUES (值1, 值2...);` | 插入数据 | `INSERT INTO users (username) VALUES ('alice');` |
| `INSERT INTO 表名 VALUES (值1, 值2...);` | 按表结构顺序插入所有列 | `INSERT INTO users VALUES (1, 'bob');` |
| `SELECT 列1, 列2... FROM 表名 [WHERE 条件];` | 查询数据 | `SELECT * FROM users WHERE id > 1;` |
| `UPDATE 表名 SET 列1=值1, 列2=值2... [WHERE 条件];` | 更新数据（务必加 WHERE，否则全表更新） | `UPDATE users SET username='alice2' WHERE id=1;` |
| `DELETE FROM 表名 [WHERE 条件];`      | 删除数据（务必加 WHERE，否则全表删除） | `DELETE FROM users WHERE id=1;`       |


### 五、索引操作命令
用于创建、查看、删除索引（提升查询效率）。

| 命令                                  | 作用                                  | 示例                                  |
|---------------------------------------|---------------------------------------|---------------------------------------|
| `CREATE INDEX 索引名 ON 表名(列名);`  | 创建普通索引                          | `CREATE INDEX idx_username ON users(username);` |
| `CREATE UNIQUE INDEX 索引名 ON 表名(列名);` | 创建唯一索引（列值不重复） | `CREATE UNIQUE INDEX idx_email ON users(email);` |
| `ALTER TABLE 表名 ADD PRIMARY KEY(列名);` | 添加主键索引（非空且唯一） | `ALTER TABLE users ADD PRIMARY KEY(id);` |
| `SHOW INDEX FROM 表名;`               | 查看表的所有索引                      | `SHOW INDEX FROM users;`              |
| `DROP INDEX 索引名 ON 表名;`          | 删除索引                              | `DROP INDEX idx_username ON users;`   |


### 六、用户与权限管理命令
用于创建用户、分配权限、修改密码等。

| 命令                                  | 作用                                  | 示例                                  |
|---------------------------------------|---------------------------------------|---------------------------------------|
| `CREATE USER '用户名'@'主机' IDENTIFIED BY '密码';` | 创建用户（主机为 `%` 表示允许所有IP连接） | `CREATE USER 'app_user'@'%' IDENTIFIED BY '123456';` |
| `GRANT 权限 ON 库.表 TO '用户名'@'主机';` | 授予权限（权限如 SELECT, INSERT, ALL） | `GRANT SELECT, INSERT ON test.* TO 'app_user'@'%';` |
| `REVOKE 权限 ON 库.表 FROM '用户名'@'主机';` | 撤销权限 | `REVOKE INSERT ON test.* FROM 'app_user'@'%';` |
| `FLUSH PRIVILEGES;`                   | 刷新权限（修改权限后生效）            | `FLUSH PRIVILEGES;`                   |
| `SET PASSWORD FOR '用户名'@'主机' = PASSWORD('新密码');` | 修改用户密码（MySQL 5.7及之前） | `SET PASSWORD FOR 'root'@'localhost' = PASSWORD('newpass');` |
| `ALTER USER '用户名'@'主机' IDENTIFIED BY '新密码';` | 修改用户密码（MySQL 8.0及之后） | `ALTER USER 'root'@'localhost' IDENTIFIED BY 'newpass';` |
| `DROP USER '用户名'@'主机';`          | 删除用户                              | `DROP USER 'app_user'@'%';`           |
| `SELECT User, Host FROM mysql.user;`  | 查看所有用户                          | `SELECT User, Host FROM mysql.user;`  |


### 七、事务与锁命令
用于控制事务和锁表操作。

| 命令                                  | 作用                                  | 示例                                  |
|---------------------------------------|---------------------------------------|---------------------------------------|
| `START TRANSACTION;` 或 `BEGIN;`      | 开始事务                              | `BEGIN;`                              |
| `COMMIT;`                             | 提交事务（永久保存修改）              | `COMMIT;`                             |
| `ROLLBACK;`                           | 回滚事务（撤销未提交的修改）          | `ROLLBACK;`                           |
| `SET AUTOCOMMIT = 0;`                 | 关闭自动提交（需手动 COMMIT）         | `SET AUTOCOMMIT = 0;`                 |
| `LOCK TABLES 表名 [READ/WRITE];`      | 锁表（READ 只读，WRITE 读写）         | `LOCK TABLES users WRITE;`            |
| `UNLOCK TABLES;`                      | 解锁所有表                            | `UNLOCK TABLES;`                      |


### 八、其他常用命令
| 命令                                  | 作用                                  | 示例                                  |
|---------------------------------------|---------------------------------------|---------------------------------------|
| `SHOW VARIABLES [LIKE '模式'];`       | 查看系统变量（如编码、端口）          | `SHOW VARIABLES LIKE 'port';`         |
| `SHOW STATUS [LIKE '模式'];`          | 查看数据库状态（如连接数、查询数）    | `SHOW STATUS LIKE 'Threads_connected';` |
| `SHOW PROCESSLIST;`                   | 查看当前所有连接进程（kill 慢查询）   | `SHOW PROCESSLIST;`                   |
| `KILL 进程ID;`                        | 终止指定进程                          | `KILL 123;`（123 为进程ID）           |
| `SOURCE 脚本路径;`                    | 在客户端执行 SQL 脚本（如导入数据）   | `SOURCE /backup/test.sql;`            |
| `DELIMITER 分隔符;`                   | 修改语句分隔符（用于创建存储过程/函数） | `DELIMITER //`                        |


### 九、命令使用注意事项
1. **大小写**：MySQL 命令不区分大小写（如 `SELECT` 和 `select` 等效），但表名、列名在 Linux 系统中区分大小写（Windows 不区分）。  
2. **分号结尾**：所有 SQL 语句需以 `;` 结尾（客户端命令如 `exit` 可不带）。  
3. **权限限制**：部分命令（如 `DROP DATABASE`、`GRANT`）需管理员权限（如 `root`）。  
4. **谨慎操作**：`DROP`、`TRUNCATE`、`DELETE` 等命令会删除数据或结构，执行前务必确认，建议先备份。  


以上命令覆盖了 MySQL 日常操作的核心场景，熟练掌握可高效管理数据库、表和数据，结合具体业务场景灵活运用即可。
