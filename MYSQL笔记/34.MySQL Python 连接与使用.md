在 Python 中操作 MySQL 数据库，需通过 MySQL 连接器（如 `mysql-connector-python` 或 `PyMySQL`）建立连接，执行执行 SQL 语句实现数据交互。以下是 Python 连接和使用 MySQL 的核心知识：


### 一、安装 MySQL 连接器
Python 需通过第三方库连接 MySQL，常用的有两个：

| 库名                | 特点                                  | 安装命令                          |
|---------------------|---------------------------------------|-----------------------------------|
| `mysql-connector-python` | Oracle 官方提供，兼容性好            | `pip install mysql-connector-python` |
| `PyMySQL`            | 纯 Python 实现，轻量，使用广泛        | `pip install pymysql`             |

两者用法类似，以下以 **`PyMySQL`** 为例（更常用）。


### 二、连接 MySQL 数据库
连接数据库需提供 **主机地址、端口、用户名、密码、数据库名** 等信息，步骤如下：

#### 基本连接代码：
```python
import pymysql

# 1. 建立连接
conn = pymysql.connect(
    host='localhost',    # 数据库主机地址（默认 localhost）
    port=3306,           # 端口（默认 3306）
    user='root',         # 用户名
    password='your_password',  # 密码
    database='test',     # 要连接的数据库名
    charset='utf8mb4'    # 字符集（支持中文和 emoji）
)

# 2. 创建游标（用于执行 SQL 语句）
cursor = conn.cursor()

# 3. 执行操作（后续步骤）...

# 4. 关闭游标和连接（必须执行，释放资源）
cursor.close()
conn.close()
```

- **连接参数说明**：  
  - `host`：远程数据库需填 IP 地址（如 `192.168.1.100`）。  
  - `charset='utf8mb4'`：务必指定，否则中文可能乱码。  
  - 若连接失败，会抛出 `pymysql.err.OperationalError`（检查账号密码、端口是否开放）。


### 三、执行 SQL 语句（核心操作）
通过游标 `cursor` 执行 SQL 语句，分**查询操作**（`SELECT`）和**写操作**（`INSERT`/`UPDATE`/`DELETE`）。


#### 1. 查询数据（`SELECT`）
使用 `cursor.execute(sql)` 执行查询，通过 `cursor.fetch*()` 获取结果：

```python
try:
    # 执行查询 SQL
    sql = "SELECT id, username FROM users WHERE age > %s"  # %s 是参数占位符（防止注入）
    cursor.execute(sql, (18,))  # 第二个参数是元组，传递查询参数

    # 获取结果
    # 方法1：获取所有结果（返回列表，每个元素是一行数据的元组）
    all_rows = cursor.fetchall()
    print("所有结果：", all_rows)  # 如：[(1, 'alice'), (2, 'bob')]

    # 方法2：获取一行结果
    # cursor.execute(sql, (18,))  # 需重新执行查询（游标会移动）
    # one_row = cursor.fetchone()
    # print("一行结果：", one_row)

    # 方法3：获取指定行数（如前2行）
    # cursor.execute(sql, (18,))
    # many_rows = cursor.fetchmany(2)
    # print("前2行：", many_rows)

except pymysql.err.MySQLError as e:
    print("查询失败：", e)
```

- **参数占位符**：必须用 `%s`（即使参数是数字），且参数需放在元组中（如 `(18,)`），避免 SQL 注入。  
- **结果格式**：默认返回元组（如 `(1, 'alice')`），若想返回字典（键为列名），可在创建游标时指定：  
  ```python
  cursor = conn.cursor(cursor=pymysql.cursors.DictCursor)  # 游标返回字典
  # 结果：[{'id': 1, 'username': 'alice'}, ...]
  ```


#### 2. 写操作（`INSERT`/`UPDATE`/`DELETE`）
写操作需通过 `conn.commit()` 提交事务，否则修改不会生效；失败时用 `conn.rollback()` 回滚。

```python
try:
    # 示例1：插入数据
    insert_sql = "INSERT INTO users (username, age) VALUES (%s, %s)"
    # 单条插入
    cursor.execute(insert_sql, ('charlie', 25))
    # 批量插入（效率更高）
    # users = [('david', 30), ('eve', 22)]
    # cursor.executemany(insert_sql, users)  # 批量执行

    # 示例2：更新数据
    update_sql = "UPDATE users SET age = %s WHERE username = %s"
    cursor.execute(update_sql, (26, 'charlie'))

    # 示例3：删除数据
    delete_sql = "DELETE FROM users WHERE age < %s"
    cursor.execute(delete_sql, (18,))

    # 提交事务（必须执行，否则所有修改无效）
    conn.commit()
    print("操作成功，影响行数：", cursor.rowcount)  # rowcount 是受影响的行数

except pymysql.err.MySQLError as e:
    # 失败时回滚
    conn.rollback()
    print("操作失败，已回滚：", e)
```

- **事务控制**：MySQL 默认开启自动提交（`autocommit=True`），但手动控制事务更安全（尤其是多步操作），可关闭自动提交：  
  ```python
  conn.autocommit(False)  # 关闭自动提交，需手动 commit
  ```


### 四、获取插入数据的自增 ID
插入含 `AUTO_INCREMENT` 列的数据时，可通过 `cursor.lastrowid` 获取自动生成的 ID：

```python
try:
    sql = "INSERT INTO users (username) VALUES (%s)"
    cursor.execute(sql, ('frank',))
    conn.commit()
    print("插入的自增 ID 是：", cursor.lastrowid)  # 输出新插入行的 id
except Exception as e:
    conn.rollback()
```


### 五、连接池（提升性能）
频繁创建和关闭连接会消耗资源，生产环境中建议使用**连接池**（如 `DBUtils`）管理连接。

#### 安装连接池库：
```bash
pip install DBUtils
```

#### 连接池使用示例：
```python
from DBUtils.PooledDB import PooledDB
import pymysql

# 创建连接池
pool = PooledDB(
    creator=pymysql,  # 使用的数据库连接器
    maxconnections=10,  # 最大连接数
    mincached=2,       # 初始化时连接池中的连接数
    host='localhost',
    user='root',
    password='your_password',
    database='test',
    charset='utf8mb4'
)

# 从连接池获取连接
conn = pool.connection()
cursor = conn.cursor()

# 执行 SQL（与之前相同）
cursor.execute("SELECT * FROM users")
print(cursor.fetchall())

# 关闭连接（实际是放回连接池，而非真正关闭）
cursor.close()
conn.close()
```


### 六、注意事项
1. **SQL 注入防护**：  
   必须使用参数化查询（`%s` 占位符 + 元组传参），禁止直接拼接字符串（如 `f"SELECT * FROM users WHERE name = '{name}'"`）。

2. **编码问题**：  
   - 连接时指定 `charset='utf8mb4'`。  
   - 确保数据库、表、字段的编码为 `utf8mb4`（避免中文乱码）。

3. **资源释放**：  
   - 无论操作成功与否，都需关闭游标和连接（可使用 `with` 语句自动释放）：  
     ```python
     with conn.cursor() as cursor:  # 自动关闭游标
         cursor.execute(sql)
     ```

4. **异常处理**：  
   对所有数据库操作添加 `try-except` 捕获异常（如网络中断、SQL 语法错误），并在失败时回滚事务。

5. **版本兼容性**：  
   - `PyMySQL` 支持 Python 3.6+ 和 MySQL 5.5+。  
   - 若连接 MySQL 8.0 报密码错误，可能是认证插件问题，需修改用户认证方式：  
     ```sql
     ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY 'your_password';
     FLUSH PRIVILEGES;
     ```


### 总结
Python 操作 MySQL 的核心流程是：  
**安装连接器 → 建立连接 → 创建游标 → 执行 SQL（参数化） → 处理结果 → 提交/回滚事务 → 释放资源**。  

合理使用连接池和事务控制，可提升性能和数据安全性，参数化查询是防止 SQL 注入的关键。
