MySQL 支持使用正则表达式进行复杂的模式匹配查询，通过 `REGEXP` 或 `RLIKE` 操作符实现（两者功能相同）。正则表达式比 `LIKE` 子句更灵活，能处理更复杂的匹配规则。以下是核心知识：


### 1. 基本语法
```sql
SELECT 列名 FROM 表名
WHERE 列名 REGEXP '模式';
```

**说明**：
- `REGEXP` 用于指定正则表达式模式
- 默认不区分大小写（可使用 `BINARY` 关键字开启区分大小写）
- `NOT REGEXP` 用于排除匹配模式的记录


### 2. 常用正则表达式元字符
| 元字符 | 描述 | 示例 |
|--------|------|------|
| `^` | 匹配字符串开头 | `^a` 匹配以 "a" 开头的字符串 |
| `$` | 匹配字符串结尾 | `b$` 匹配以 "b" 结尾的字符串 |
| `.` | 匹配任意单个字符 | `a.c` 匹配 "abc"、"a1c" 等 |
| `*` | 匹配前一个元素 0 次或多次 | `ab*` 匹配 "a"、"ab"、"abb" 等 |
| `+` | 匹配前一个元素 1 次或多次 | `ab+` 匹配 "ab"、"abb" 等（不匹配 "a"） |
| `?` | 匹配前一个元素 0 次或 1 次 | `ab?` 匹配 "a" 或 "ab" |
| `|` | 匹配两个或多个模式之一 | `a\|b` 匹配 "a" 或 "b" |
| `[ ]` | 匹配括号内的任意一个字符 | `[abc]` 匹配 "a"、"b" 或 "c" |
| `[^ ]` | 匹配不在括号内的任意字符 | `[^abc]` 匹配非 "a"、"b"、"c" 的字符 |
| `{n}` | 匹配前一个元素恰好 n 次 | `a{2}` 匹配 "aa" |
| `{n,}` | 匹配前一个元素至少 n 次 | `a{2,}` 匹配 "aa"、"aaa" 等 |
| `{n,m}` | 匹配前一个元素 n 到 m 次 | `a{2,3}` 匹配 "aa" 或 "aaa" |
| `[0-9]` | 匹配任意数字 | 等价于 `\d` |
| `[a-z]` | 匹配任意小写字母 |  |
| `[A-Z]` | 匹配任意大写字母 |  |
| `[[:alpha:]]` | 匹配任意字母 |  |
| `[[:digit:]]` | 匹配任意数字 |  |
| `[[:alnum:]]` | 匹配任意字母或数字 |  |


### 3. 基本示例
假设有 `users` 表的 `email` 列数据：
`alice@test.com`、`bob123@gmail.com`、`cindy-123@example.org`、`david@company.cn`

#### 示例 1：基础匹配
```sql
-- 查找邮箱包含 "test" 或 "example" 的用户
SELECT email FROM users 
WHERE email REGEXP 'test|example';

-- 查找邮箱以 "a" 开头的用户（不区分大小写）
SELECT email FROM users 
WHERE email REGEXP '^a';
```

#### 示例 2：字符集匹配
```sql
-- 查找邮箱包含数字的用户
SELECT email FROM users 
WHERE email REGEXP '[0-9]';

-- 查找邮箱以字母开头的用户
SELECT email FROM users 
WHERE email REGEXP '^[a-zA-Z]';
```

#### 示例 3：重复次数匹配
```sql
-- 查找邮箱中包含连续两个相同字母的用户
SELECT email FROM users 
WHERE email REGEXP '([a-zA-Z])\\1'; -- \\1 表示引用第一个分组的内容

-- 查找域名部分（@后）长度为 2-3 个字母的邮箱（如 .com、.cn）
SELECT email FROM users 
WHERE email REGEXP '@.+\.[a-zA-Z]{2,3}$';
```

#### 示例 4：区分大小写
```sql
-- 区分大小写，只匹配以大写字母开头的邮箱
SELECT email FROM users 
WHERE email REGEXP BINARY '^[A-Z]';
```

#### 示例 5：否定匹配
```sql
-- 查找邮箱不包含数字的用户
SELECT email FROM users 
WHERE email NOT REGEXP '[0-9]';
```


### 4. 注意事项
1. **性能问题**：正则表达式匹配比 `LIKE` 更消耗资源，大数据量下可能影响性能
2. **转义字符**：在 MySQL 中，需要使用双反斜杠 `\\` 表示转义（如 `\\.` 匹配点号本身）
3. **默认行为**：`REGEXP` 只要字符串中存在匹配的子串就会返回 true，而不是整个字符串匹配（如需整个字符串匹配，需用 `^` 和 `$` 限制）
   ```sql
   -- 确保整个字符串是纯数字（从头到尾都是数字）
   SELECT * FROM table WHERE column REGEXP '^[0-9]+$';
   ```
4. **版本差异**：某些高级正则特性（如lookahead）在 MySQL 中不支持


### 5. 适用场景
- 复杂格式验证（如邮箱、手机号、身份证号格式校验）
- 模糊搜索中更精确的模式匹配
- 提取或过滤符合特定规则的字符串
- 处理不规则的文本数据清洗


正则表达式为 MySQL 查询提供了强大的模式匹配能力，适合处理 `LIKE` 子句无法满足的复杂匹配需求。但在使用时需权衡性能，避免在大数据集上过度使用复杂正则表达式。
