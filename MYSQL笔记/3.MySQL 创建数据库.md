在 MySQL 中，创建数据库是管理数据的基础操作之一。以下是关于 MySQL 创建数据库的核心知识：

### 1. 创建数据库的基本语法
```sql
CREATE DATABASE [IF NOT EXISTS] 数据库名
[CHARACTER SET 字符集] 
[COLLATE 排序规则];
```

- `IF NOT EXISTS`：可选，防止创建已存在的数据库时出现错误
- `CHARACTER SET`：指定数据库默认字符集（如 utf8mb4）
- `COLLATE`：指定数据库默认排序规则（如 utf8mb4_general_ci）

### 2. 常用创建示例
```sql
-- 创建基本数据库
CREATE DATABASE mydb;

-- 创建数据库前检查是否存在
CREATE DATABASE IF NOT EXISTS mydb;

-- 指定字符集和排序规则（推荐）
CREATE DATABASE IF NOT EXISTS mydb
CHARACTER SET utf8mb4
COLLATE utf8mb4_general_ci;
```

### 3. 字符集选择
- **utf8mb4**：推荐使用，支持所有 Unicode 字符（包括表情符号）
- **utf8**：MySQL 中的 utf8 实际是 utf8mb3，不支持某些 Unicode 字符
- 其他字符集：如 latin1、gbk 等，根据实际需求选择

### 4. 查看数据库
```sql
-- 查看所有数据库
SHOW DATABASES;

-- 查看数据库创建语句（包含字符集等信息）
SHOW CREATE DATABASE mydb;
```

### 5. 选择使用数据库
创建数据库后，需要选择它才能进行后续操作：
```sql
USE mydb;
```

### 6. 删除数据库
```sql
DROP DATABASE [IF EXISTS] 数据库名;
```
> ⚠️ 注意：删除数据库会删除其中所有表和数据，操作需谨慎！

### 7. 数据库命名规则
- 名称长度不超过 64 个字符
- 可包含字母、数字、下划线
- 不能使用 MySQL 关键字（如 `SELECT`、`TABLE` 等）
- 区分大小写（取决于操作系统）

### 8. 权限控制
创建数据库通常需要管理员权限，若要授权其他用户使用：
```sql
GRANT ALL PRIVILEGES ON mydb.* TO 'username'@'host';
FLUSH PRIVILEGES;
```

合理规划数据库创建是良好数据库设计的开始，尤其是字符集的选择，会影响到数据存储和查询的正确性，建议在创建时明确指定。
