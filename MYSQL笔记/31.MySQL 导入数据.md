MySQL 导入数据是将外部数据（如 SQL 脚本、CSV 文件等）加载到数据库中的操作，常用于数据恢复、迁移或批量录入。以下是关于 MySQL 数据导入的核心知识，包括常用方法、工具及注意事项：


### 一、常用导入工具与方法
根据数据格式的不同，MySQL 提供多种导入方式，核心工具包括 `mysql` 命令行客户端（导入 SQL 脚本）、`LOAD DATA INFILE` 语句（导入文本文件），以及图形化工具。


### 二、使用 `mysql` 命令导入 SQL 脚本
`mysql` 是 MySQL 的交互式命令行工具，可读取 SQL 脚本文件（含 `CREATE TABLE`、`INSERT` 等语句）并执行，适用于导入通过 `mysqldump` 导出的备份文件。

#### 基本语法：
```bash
mysql [选项] 数据库名 < 导入文件路径.sql
```

#### 常用选项：
- `-u 用户名`：指定数据库用户名  
- `-p`：提示输入密码（安全起见，不建议直接写 `-p密码`）  
- `-h 主机名`：数据库主机（默认 `localhost`）  
- `-P 端口号`：数据库端口（默认 3306）  


#### 典型示例：
1. **导入整个数据库**：  
   假设 `test_backup.sql` 是 `test` 数据库的完整备份（含表结构和数据），导入命令：  
   ```bash
   # 若数据库已存在
   mysql -u root -p test < /backup/test_backup.sql
   
   # 若数据库不存在（脚本中含 CREATE DATABASE）
   mysql -u root -p < /backup/test_backup.sql  # 无需指定数据库名
   ```
   执行后输入密码，脚本中的 SQL 语句会被逐条执行。

2. **只导入数据（忽略表结构）**：  
   若脚本仅含 `INSERT` 语句（如通过 `mysqldump -t` 导出），需确保目标表已存在，否则会报错：  
   ```bash
   mysql -u root -p test < /backup/orders_data.sql
   ```


### 三、使用 `LOAD DATA INFILE` 导入文本文件
`LOAD DATA INFILE` 是 MySQL 内置语句，用于高效导入 CSV、TXT 等文本文件（比 `INSERT` 语句批量插入快得多），适合大数据量导入。

#### 基本语法：
```sql
LOAD DATA [LOCAL] INFILE '文件路径'
INTO TABLE 表名
[OPTIONS];  -- 指定分隔符、换行符等，需与导出格式匹配
```

#### 关键参数：
- `LOCAL`：可选，若加 `LOCAL`，文件路径是**客户端本地路径**；否则是**数据库服务器端路径**（需符合 `secure_file_priv` 配置）。  
- `OPTIONS`：与导出时的 `SELECT ... INTO OUTFILE` 对应，确保格式一致：  
  - `FIELDS TERMINATED BY '分隔符'`：字段分隔符（如 `','` 对应 CSV）  
  - `OPTIONALLY ENCLOSED BY '字符'`：字段包裹符（如 `"`）  
  - `LINES TERMINATED BY '换行符'`：行分隔符（如 `\r\n` 对应 Windows）  
  - `IGNORE n LINES`：跳过前 `n` 行（如跳过 CSV 表头行，`IGNORE 1 LINES`）  


#### 示例：
1. **导入 CSV 文件（带表头）**：  
   假设 `users.csv` 是包含表头的 CSV 文件（格式：`id,username,email`），导入到 `users` 表：  
   ```sql
   -- 客户端本地文件（加 LOCAL）
   LOAD DATA LOCAL INFILE '/data/users.csv'
   INTO TABLE test.users
   FIELDS TERMINATED BY ',' 
   OPTIONALLY ENCLOSED BY '"'
   LINES TERMINATED BY '\r\n'
   IGNORE 1 LINES;  -- 跳过第一行表头
   ```

2. **导入服务器端 TXT 文件（制表符分隔）**：  
   若文件在数据库服务器的 `/var/lib/mysql-files/orders.txt`（需在 `secure_file_priv` 目录）：  
   ```sql
   LOAD DATA INFILE '/var/lib/mysql-files/orders.txt'
   INTO TABLE test.orders
   FIELDS TERMINATED BY '\t'  -- 制表符分隔
   LINES TERMINATED BY '\n'
   (order_id, amount, order_time);  -- 显式指定列对应关系（可选，默认按表结构顺序）
   ```


### 四、其他导入方式
1. **图形化工具**：  
   Navicat、DBeaver、MySQL Workbench 等工具提供可视化导入向导：  
   - 右键目标表 → 导入向导 → 选择文件（SQL、CSV、Excel 等）→ 配置字段映射、分隔符等 → 执行。  
   适合非命令行用户，操作简单但大数据量时效率较低。

2. **`source` 命令（交互式）**：  
   在 `mysql` 客户端交互模式下，使用 `source` 命令导入 SQL 脚本：  
   ```sql
   mysql> use test;  -- 切换到目标数据库
   mysql> source /backup/test_backup.sql;  -- 执行脚本
   ```


### 五、导入注意事项
1. **表结构一致性**：  
   - 导入前需确保目标表已存在（`LOAD DATA` 和数据脚本导入时），且字段数量、类型与源数据匹配，否则会报“列不匹配”或“数据类型错误”。  
   - 若表结构不同，可先通过 `ALTER TABLE` 调整，或在导入时指定字段映射（如 `LOAD DATA` 中的列列表）。

2. **字符集问题**：  
   - 导入 SQL 脚本时，若包含中文，需确保脚本字符集与数据库/表的字符集一致（如 `utf8mb4`），可在脚本开头添加：  
     ```sql
     SET NAMES utf8mb4;  -- 声明字符集
     ```
   - 文本文件（如 CSV）需保存为与数据库一致的编码（如 UTF-8 无 BOM），避免中文乱码。

3. **索引与性能**：  
   - 大数据量导入时，建议先删除目标表的索引（尤其是主键、唯一索引），导入完成后再重建，可大幅提升速度（索引会导致插入时频繁更新 B+ 树）。  
   - 示例：  
     ```sql
     -- 导入前删除索引
     ALTER TABLE users DROP INDEX idx_username;
     -- 执行导入...
     -- 导入后重建索引
     ALTER TABLE users ADD INDEX idx_username (username);
     ```

4. **权限与路径**：  
   - `LOAD DATA INFILE` 无 `LOCAL` 时，文件需在数据库服务器的 `secure_file_priv` 目录（通过 `SHOW VARIABLES LIKE 'secure_file_priv';` 查看），且数据库用户需有 `FILE` 权限。  
   - `LOCAL` 选项依赖客户端配置（`local_infile` 需开启），部分环境可能禁用（出于安全考虑）。

5. **数据冲突处理**：  
   - 若导入数据与现有数据存在主键/唯一键冲突，默认会报错并终止导入。可通过以下方式处理：  
     - 导入前清理目标表（`TRUNCATE TABLE 表名;`，谨慎使用）。  
     - 在 `LOAD DATA` 中添加 `IGNORE` 关键字，跳过冲突行：  
       ```sql
       LOAD DATA LOCAL INFILE 'users.csv' INTO TABLE users IGNORE ...;
       ```


### 六、导入性能优化（针对大数据量）
- **禁用自动提交**：导入前执行 `SET autocommit = 0;`，导入完成后 `COMMIT;`，减少事务提交开销。  
- **使用 `--max_allowed_packet`**：若 SQL 脚本中有大字段（如 `TEXT`），需增大客户端数据包限制（如 `mysql -u root -p --max_allowed_packet=64M test < big_data.sql`）。  
- **分批次导入**：超大型文件（如 10GB 以上）可拆分为多个小文件，分批次导入，避免内存溢出。  


### 总结
- **`mysql` 命令**：适合导入 SQL 脚本（备份恢复、结构+数据迁移），支持全库或单表导入。  
- **`LOAD DATA INFILE`**：适合导入文本文件（CSV/TXT），效率极高，是大数据量导入的首选。  
- 图形化工具：适合简单场景，操作直观但性能有限。  

导入前需验证表结构、字符集和权限，大数据量时注意性能优化，确保数据准确高效地加载到数据库中。
