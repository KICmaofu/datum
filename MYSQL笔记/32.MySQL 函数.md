MySQL 函数是用于处理数据的预定义或自定义的代码块，可接收参数并返回结果，广泛用于查询、数据转换、计算等场景。按功能可分为**内置函数**（MySQL 自带）和**自定义函数**（用户创建），以下是核心知识：


### 一、内置函数（常用分类）
MySQL 提供了数百个内置函数，覆盖字符串、数值、日期、聚合等场景，以下是高频使用的类别及示例：


#### 1. 字符串函数
用于字符串处理（拼接、截取、替换等）：

| 函数               | 作用                                  | 示例                                  | 结果                  |
|--------------------|---------------------------------------|---------------------------------------|-----------------------|
| `CONCAT(s1, s2...)` | 拼接字符串                            | `CONCAT('Hello', ' ', 'MySQL')`       | `Hello MySQL`         |
| `SUBSTRING(s, pos, len)` | 截取子串（pos 起始位置，len 长度） | `SUBSTRING('abcdef', 2, 3)`           | `bcd`                 |
| `LENGTH(s)`        | 字符串长度（字节数，受编码影响）      | `LENGTH('中国')`（utf8mb4 下）        | `6`（每个汉字 3 字节） |
| `CHAR_LENGTH(s)`   | 字符数（与编码无关）                  | `CHAR_LENGTH('中国')`                 | `2`                   |
| `TRIM(s)`          | 去除首尾空格                          | `TRIM('  test  ')`                    | `test`                |
| `REPLACE(s, old, new)` | 替换字符串                          | `REPLACE('abcabc', 'a', 'x')`         | `xbcxbc`              |


#### 2. 数值函数
用于数值计算（加减乘除、取整、随机数等）：

| 函数               | 作用                                  | 示例                  | 结果          |
|--------------------|---------------------------------------|-----------------------|---------------|
| `ABS(n)`           | 绝对值                                | `ABS(-3.14)`          | `3.14`        |
| `ROUND(n, d)`      | 四舍五入（d 为小数位数，默认 0）      | `ROUND(3.1415, 2)`    | `3.14`        |
| `FLOOR(n)`         | 向下取整                              | `FLOOR(3.99)`         | `3`           |
| `CEIL(n)`          | 向上取整                              | `CEIL(3.01)`          | `4`           |
| `RAND()`           | 生成 0-1 随机数                       | `RAND()`              | 如 `0.7234`   |
| `MOD(a, b)`        | 取余（a % b）                         | `MOD(7, 3)`           | `1`           |


#### 3. 日期与时间函数
用于日期时间的获取、转换、计算：

| 函数               | 作用                                  | 示例                  | 结果（示例）       |
|--------------------|---------------------------------------|-----------------------|--------------------|
| `NOW()`            | 当前日期时间（`YYYY-MM-DD HH:MM:SS`） | `NOW()`               | `2023-10-05 14:30:00` |
| `CURDATE()`        | 当前日期                              | `CURDATE()`           | `2023-10-05`       |
| `CURTIME()`        | 当前时间                              | `CURTIME()`           | `14:30:00`         |
| `DATE_FORMAT(d, fmt)` | 格式化日期                            | `DATE_FORMAT(NOW(), '%Y年%m月%d日')` | `2023年10月05日` |
| `DATEDIFF(d1, d2)` | 计算日期差（d1 - d2，单位：天）       | `DATEDIFF('2023-10-05', '2023-10-01')` | `4` |
| `DATE_ADD(d, INTERVAL expr unit)` | 日期加间隔 | `DATE_ADD(NOW(), INTERVAL 1 HOUR)` | 1小时后的时间 |


#### 4. 聚合函数（GROUP BY 中使用）
用于对一组数据进行统计计算：

| 函数               | 作用                                  | 示例                                  |
|--------------------|---------------------------------------|---------------------------------------|
| `COUNT(expr)`      | 计数（`COUNT(*)` 含 NULL，`COUNT(col)` 不含 NULL） | `SELECT COUNT(*) FROM users`          |
| `SUM(col)`         | 求和                                  | `SELECT SUM(amount) FROM orders`      |
| `AVG(col)`         | 平均值                                | `SELECT AVG(score) FROM students`     |
| `MAX(col)`         | 最大值                                | `SELECT MAX(age) FROM users`          |
| `MIN(col)`         | 最小值                                | `SELECT MIN(price) FROM products`     |


#### 5. 条件函数
用于逻辑判断（类似编程语言的 `if-else`）：

| 函数               | 作用                                  | 示例                                  | 结果          |
|--------------------|---------------------------------------|---------------------------------------|---------------|
| `IF(expr, v1, v2)` | 若 expr 为真，返回 v1，否则 v2        | `IF(1>0, 'yes', 'no')`                | `yes`         |
| `IFNULL(v1, v2)`   | 若 v1 不为 NULL，返回 v1，否则 v2     | `IFNULL(NULL, '空')`                  | `空`          |
| `CASE`             | 多条件判断                            | `CASE WHEN score >=90 THEN 'A' WHEN score >=60 THEN 'B' ELSE 'C' END` | 按分数返回等级 |


### 二、自定义函数（UDF）
当内置函数无法满足需求时，可创建自定义函数（User-Defined Function），封装特定逻辑，供查询或存储过程调用。

#### 1. 创建自定义函数
语法：
```sql
CREATE FUNCTION 函数名(参数名 数据类型 [, 参数名 数据类型...])
RETURNS 返回数据类型
DETERMINISTIC  -- 可选：表示输入相同则输出相同（优化用）
BEGIN
  -- 函数体（SQL 语句，如计算、查询等）
  RETURN 返回值;
END;
```

#### 示例：创建计算两数之和的函数
```sql
-- 创建函数
DELIMITER //  -- 临时修改分隔符（避免 ; 提前结束函数定义）
CREATE FUNCTION add_two_num(a INT, b INT)
RETURNS INT
DETERMINISTIC
BEGIN
  DECLARE sum INT;  -- 声明变量
  SET sum = a + b;
  RETURN sum;
END //
DELIMITER ;  -- 恢复分隔符

-- 调用函数
SELECT add_two_num(3, 5);  -- 结果：8
```


#### 2. 查看与删除自定义函数
```sql
-- 查看所有自定义函数
SHOW FUNCTION STATUS WHERE Db = '数据库名';

-- 查看函数定义
SHOW CREATE FUNCTION 函数名;

-- 删除函数
DROP FUNCTION IF EXISTS 函数名;
```


### 三、使用函数的注意事项
1. **性能影响**：  
   - 聚合函数（如 `COUNT(*)`）在大表上可能耗时，需合理使用索引。  
   - 自定义函数若逻辑复杂（如包含多表查询），可能降低查询效率，建议谨慎使用。

2. **函数与存储过程的区别**：  
   - 函数**必须返回一个值**，且只能用于 `SELECT` 语句或表达式中。  
   - 存储过程**可以不返回值**，可通过 `CALL` 调用，功能更灵活（如包含 `INSERT`、`UPDATE` 等操作）。

3. **权限问题**：  
   - 创建自定义函数需 `CREATE ROUTINE` 权限，调用需 `EXECUTE` 权限。  
   - MySQL 可能因 `log_bin_trust_function_creators` 配置限制函数创建（需管理员开启）。

4. **避免在 WHERE 子句滥用函数**：  
   - 如 `WHERE SUBSTRING(username, 1, 1) = 'a'` 会导致索引失效，降低查询速度，尽量通过其他方式优化。


### 总结
MySQL 函数是数据处理的重要工具：  
- **内置函数**覆盖日常开发需求，需熟练掌握字符串、日期、聚合等常用函数。  
- **自定义函数**用于封装特定逻辑，提升代码复用性，但需注意性能和权限。  

合理使用函数可简化 SQL 语句，提高数据处理效率，但需避免过度使用导致性能问题。
