MySQL 中的 `NULL` 表示"无值"或"未知值"，与空字符串（`""`）或数字 `0` 不同，它代表数据的缺失或未定义状态。处理 `NULL` 值是数据库操作中的常见需求，以下是核心知识：


### 1. `NULL` 的特性
- `NULL` 不等于任何值，包括它自己（`NULL = NULL` 结果为 `NULL`）
- `NULL` 与任何值运算结果仍为 `NULL`（如 `1 + NULL` 结果为 `NULL`）
- 空字符串（`""`）和 `0` 不是 `NULL`，它们是有效的值
- 未赋值的列默认可能为 `NULL`（除非定义了 `NOT NULL` 约束）


### 2. 检测 `NULL` 值：`IS NULL` 和 `IS NOT NULL`
由于 `NULL` 不与任何值相等，不能用 `=` 或 `!=` 检测，必须使用专用运算符：

#### 示例：
假设有 `users` 表：

| id  | name   | email          | age  |
|-----|--------|----------------|------|
| 1   | Alice  | alice@test.com | 25   |
| 2   | Bob    | NULL           | 30   |
| 3   | Cindy  | cindy@test.com | NULL |

```sql
-- 查找邮箱为 NULL 的用户
SELECT * FROM users WHERE email IS NULL;

-- 查找年龄不为 NULL 的用户
SELECT * FROM users WHERE age IS NOT NULL;
```


### 3. 处理 `NULL` 值的函数
MySQL 提供多个函数处理 `NULL` 值，常用的有：

#### ① `IFNULL(expr1, expr2)`
- 如果 `expr1` 是 `NULL`，返回 `expr2`；否则返回 `expr1`
```sql
-- 用 "未提供" 替换 NULL 的邮箱
SELECT name, IFNULL(email, '未提供') AS email FROM users;
```

#### ② `COALESCE(expr1, expr2, ...)`
- 返回第一个非 `NULL` 的表达式（支持多个参数）
```sql
-- 优先使用 email，其次用备用邮箱，都为 NULL 则显示 "无"
SELECT name, COALESCE(email, backup_email, '无') AS contact FROM users;
```

#### ③ `NULLIF(expr1, expr2)`
- 如果 `expr1` 等于 `expr2`，返回 `NULL`；否则返回 `expr1`
```sql
-- 当 score 为 0 时返回 NULL，否则返回实际分数
SELECT name, NULLIF(score, 0) AS adjusted_score FROM students;
```


### 4. 聚合函数与 `NULL`
聚合函数会自动忽略 `NULL` 值（不参与计算）：
- `COUNT(*)` 包含 `NULL` 行，`COUNT(列名)` 忽略 `NULL`
- `SUM()`、`AVG()`、`MAX()`、`MIN()` 均忽略 `NULL`

```sql
-- 计算平均年龄（自动忽略 age 为 NULL 的行）
SELECT AVG(age) FROM users;

-- 统计有邮箱的用户数量（忽略 NULL）
SELECT COUNT(email) FROM users;
```


### 5. `NULL` 与索引
- `NULL` 值可以被索引（MySQL 索引支持 `NULL`）
- `IS NULL` 查询可以利用索引，效率较高
- 复合索引中包含可空列时，索引仍有效


### 6. 创建表时处理 `NULL`
定义表结构时，可以指定列是否允许 `NULL`：
```sql
CREATE TABLE products (
  id INT NOT NULL, -- 不允许 NULL（必须有值）
  name VARCHAR(50) NOT NULL, -- 不允许 NULL
  description TEXT NULL, -- 允许 NULL（默认）
  price DECIMAL(10,2) NOT NULL DEFAULT 0 -- 不允许 NULL，默认值为 0
);
```
- `NOT NULL`：强制列必须有值，插入/更新时不能为 `NULL`
- `DEFAULT`：当未提供值时，自动使用默认值（避免 `NULL`）


### 7. 注意事项
1. **避免滥用 `NULL`**：能使用默认值（如 `0`、`""`）时尽量不使用 `NULL`
2. **查询逻辑陷阱**：
   - `WHERE 列名 != 'value'` 不会返回 `NULL` 的行（需额外加 `OR 列名 IS NULL`）
   - `SELECT * FROM 表名 WHERE NULL` 永远返回空结果
3. **外键与 `NULL`**：外键列允许 `NULL`，表示该记录可以不关联其他表
4. **排序中的 `NULL`**：`ORDER BY` 中 `NULL` 被视为最小值（升序排最前，降序排最后）


### 8. 常见场景示例
```sql
-- 查找未填写年龄或年龄大于 30 的用户
SELECT * FROM users 
WHERE age IS NULL OR age > 30;

-- 计算用户总数（包含 NULL）和有邮箱的用户数
SELECT 
  COUNT(*) AS total_users,
  COUNT(email) AS users_with_email 
FROM users;

-- 用 0 替换 NULL 的分数，再计算总和
SELECT SUM(IFNULL(score, 0)) AS total_score FROM students;
```


正确处理 `NULL` 值是保证数据准确性和查询结果正确性的关键。理解 `NULL` 的特殊性质，并合理使用相关函数和约束，可以有效避免因 `NULL` 导致的逻辑错误。
