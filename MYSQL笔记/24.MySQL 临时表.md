MySQL 临时表是一种特殊的表，用于在当前会话中临时存储数据，会话结束后会自动销毁，适用于临时数据处理场景（如复杂查询、中间结果存储等）。以下是关于 MySQL 临时表的核心知识：


### 一、临时表的特点
1. **会话隔离性**：  
   - 临时表仅对当前会话可见，不同会话可以创建同名临时表，互不干扰。
   - 若临时表与普通表同名，当前会话中临时表会优先被访问（隐藏普通表）。

2. **自动销毁**：  
   - 会话结束（正常断开连接）或手动执行 `DROP TEMPORARY TABLE` 时被删除。
   - 即使会话异常终止（如崩溃），MySQL 也会自动清理临时表。

3. **存储位置**：  
   - 数据默认存储在内存中（`MEMORY` 引擎），若数据量超过内存限制，会自动转存到磁盘（`InnoDB` 或 `MyISAM` 引擎，取决于配置）。


### 二、创建临时表
语法与普通表类似，只需在 `CREATE TABLE` 后加 `TEMPORARY` 关键字：

```sql
-- 基本语法
CREATE TEMPORARY TABLE 临时表名 (
  列名1 数据类型 [约束],
  列名2 数据类型 [约束],
  ...
);

-- 示例：从查询结果创建临时表
CREATE TEMPORARY TABLE temp_orders AS
SELECT order_id, user_id, order_time 
FROM orders 
WHERE status = 'paid';
```


### 三、操作临时表
临时表支持普通表的大部分操作：

```sql
-- 插入数据
INSERT INTO temp_orders (order_id, user_id, order_time)
VALUES (1001, 1, '2023-10-01 10:00:00');

-- 查询数据
SELECT * FROM temp_orders WHERE user_id = 1;

-- 更新数据
UPDATE temp_orders SET order_time = NOW() WHERE order_id = 1001;

-- 删除数据
DELETE FROM temp_orders WHERE user_id = 1;
```


### 四、删除临时表
- 手动删除需使用 `DROP TEMPORARY TABLE`（不可省略 `TEMPORARY`，否则可能误删普通表）：
  ```sql
  DROP TEMPORARY TABLE IF EXISTS temp_orders; -- 推荐加 IF EXISTS 避免报错
  ```
- 若直接用 `DROP TABLE` 删除临时表，会报错（需指定 `TEMPORARY`）。


### 五、使用场景
1. **存储中间结果**：复杂查询中，将中间结果存入临时表，简化后续操作。
2. **会话内临时数据处理**：如多步骤数据清洗、计算，避免污染普通表。
3. **测试环境**：临时创建表验证逻辑，无需手动清理。


### 六、注意事项
1. **事务影响**：  
   - `InnoDB` 临时表的操作受事务影响（可回滚），但表结构在事务结束后仍存在（直到会话结束）。
2. **不支持的功能**：  
   - 无法创建外键约束。
   - 不能被其他会话引用（如通过视图或存储过程跨会话访问）。
3. **内存限制**：  
   - 若使用 `MEMORY` 引擎，需注意 `max_heap_table_size` 配置，避免数据溢出到磁盘影响性能。
4. **命名冲突**：  
   - 避免与普通表同名，防止误操作（可加前缀如 `tmp_` 区分）。


通过合理使用临时表，可以提升临时数据处理的效率，同时避免对数据库永久表的干扰。
