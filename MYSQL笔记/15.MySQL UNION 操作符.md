MySQL 的 `UNION` 操作符用于合并两个或多个 `SELECT` 语句的结果集，将它们组合成一个单一的结果集。以下是关于 `UNION` 操作符的核心知识：


### 1. 基本语法
```sql
SELECT 列1, 列2, ... FROM 表1
[WHERE 条件]
UNION [ALL]
SELECT 列1, 列2, ... FROM 表2
[WHERE 条件];
```

**说明**：
- 每个 `SELECT` 语句必须返回相同数量的列
- 对应列的数据类型必须兼容
- `UNION` 会自动去除结果集中的重复行
- `UNION ALL` 会保留所有结果，包括重复行（性能优于 `UNION`）


### 2. 基本示例
假设有两个表：
- `customers`（客户表）：`id`, `name`, `city`
- `suppliers`（供应商表）：`id`, `name`, `city`

#### 示例 1：使用 `UNION`（去重）
```sql
-- 合并客户和供应商的名称与城市，去除重复项
SELECT name, city FROM customers
UNION
SELECT name, city FROM suppliers;
```

#### 示例 2：使用 `UNION ALL`（保留重复）
```sql
-- 合并所有记录，包括重复项
SELECT name, city FROM customers
UNION ALL
SELECT name, city FROM suppliers;
```

#### 示例 3：带条件的 `UNION`
```sql
-- 合并北京的客户和上海的供应商
SELECT name, city, 'customer' AS type FROM customers
WHERE city = 'Beijing'
UNION
SELECT name, city, 'supplier' AS type FROM suppliers
WHERE city = 'Shanghai';
```


### 3. 关键规则
1. **列数匹配**：所有 `SELECT` 语句必须返回相同数量的列
   ```sql
   -- 错误示例：列数不同
   SELECT name FROM customers
   UNION
   SELECT name, city FROM suppliers; -- 会报错
   ```

2. **数据类型兼容**：对应列的数据类型需要兼容（无需完全相同，但需可隐式转换）
   ```sql
   -- 允许：INT 与 DECIMAL 兼容
   SELECT id, amount FROM orders
   UNION
   SELECT id, total FROM payments;
   ```

3. **排序结果**：可在最后添加 `ORDER BY` 对合并结果排序
   ```sql
   SELECT name, city FROM customers
   UNION
   SELECT name, city FROM suppliers
   ORDER BY city, name; -- 对合并后的结果排序
   ```


### 4. `UNION` 与 `JOIN` 的区别
| 操作       | 作用                          | 结果形式                |
|------------|-------------------------------|-------------------------|
| `UNION`    | 横向合并多个结果集（行合并）  | 结果集列数与原表相同    |
| `JOIN`     | 纵向关联多个表（列合并）      | 结果集列数是各表列数之和|


### 5. 注意事项
1. **性能考虑**：`UNION ALL` 比 `UNION` 快，因为无需去重操作，应优先使用（如果不需要去重）
2. **括号分组**：复杂查询可使用括号分组
   ```sql
   (SELECT name FROM customers WHERE city = 'Beijing')
   UNION ALL
   (SELECT name FROM suppliers WHERE city = 'Shanghai')
   ORDER BY name;
   ```
3. **权限要求**：需要对所有涉及的表有 `SELECT` 权限
4. **子查询支持**：`UNION` 也可用于子查询中
   ```sql
   SELECT * FROM (
     SELECT id FROM table1
     UNION
     SELECT id FROM table2
   ) AS combined_ids;
   ```


### 6. 适用场景
- 合并多个相似结构表的数据（如分年度的销售表）
- 从不同表中获取同类信息（如客户和供应商的联系信息）
- 组合不同条件的查询结果（如"活跃用户"和"高消费用户"的合并列表）


`UNION` 操作符是处理分散在多个表中同类数据的有效工具，合理使用可以简化复杂的查询逻辑，但需注意保持各 `SELECT` 语句的结构一致性。
