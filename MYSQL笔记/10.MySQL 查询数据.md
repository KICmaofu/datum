MySQL 查询数据是数据库操作中最常用的功能，用于从数据表中检索所需信息。以下是关于 MySQL 查询数据的核心知识：


### 1. 基本查询语法
最基础的查询语句结构如下：
```sql
SELECT 字段1, 字段2, ... 
FROM 表名 
[WHERE 条件] 
[GROUP BY 分组字段] 
[HAVING 分组条件] 
[ORDER BY 排序字段 [ASC|DESC]] 
[LIMIT 限制结果数量];
```


### 2. 常用查询类型及示例

#### （1）查询所有字段
```sql
-- 查询表中所有字段的所有记录（不推荐在生产环境使用，效率低）
SELECT * FROM users;
```

#### （2）查询指定字段
```sql
-- 只查询需要的字段，提高效率
SELECT username, email, created_at FROM users;
```

#### （3）带条件查询（WHERE 子句）
```sql
-- 条件查询：年龄大于18且状态为1的用户
SELECT username, age FROM users 
WHERE age > 18 AND status = 1;

-- 常用条件运算符：=、!=、>、<、>=、<=、IN、NOT IN、LIKE、BETWEEN...AND...
SELECT * FROM products 
WHERE price BETWEEN 100 AND 500  -- 价格在100-500之间
  AND category IN ('电子', '家居');  -- 分类是电子或家居
```

#### （4）模糊查询（LIKE）
```sql
-- % 匹配任意字符（包括0个），_ 匹配单个字符
SELECT * FROM users 
WHERE username LIKE '张%';  -- 匹配姓张的用户

SELECT * FROM articles 
WHERE title LIKE '_品%';  -- 匹配第二个字是"品"的标题
```

#### （5）排序查询（ORDER BY）
```sql
-- 按年龄升序（默认ASC），相同年龄按注册时间降序
SELECT username, age, created_at FROM users 
ORDER BY age ASC, created_at DESC;
```

#### （6）限制结果数量（LIMIT）
```sql
-- 查询前10条记录
SELECT * FROM products LIMIT 10;

-- 分页查询：从第20条开始，查询10条（常用于分页，第3页数据）
SELECT * FROM articles LIMIT 20, 10;  -- 偏移量20，取10条
```

#### （7）分组查询（GROUP BY）
```sql
-- 按性别分组，统计每组人数
SELECT gender, COUNT(*) AS user_count FROM users 
GROUP BY gender;

-- 配合HAVING筛选分组结果（HAVING用于分组后，WHERE用于分组前）
SELECT category, AVG(price) AS avg_price FROM products 
GROUP BY category 
HAVING avg_price > 200;  -- 筛选平均价格大于200的分类
```


### 3. 聚合函数
聚合函数用于对一组数据进行计算并返回单一结果：

| 函数         | 说明                  |
|--------------|-----------------------|
| `COUNT()`    | 统计记录数            |
| `SUM()`      | 计算总和              |
| `AVG()`      | 计算平均值            |
| `MAX()`      | 取最大值              |
| `MIN()`      | 取最小值              |

示例：
```sql
SELECT 
  COUNT(*) AS total_users,  -- 总用户数
  SUM(amount) AS total_sales,  -- 总销售额
  AVG(score) AS avg_score,  -- 平均评分
  MAX(create_time) AS last_time  -- 最后创建时间
FROM orders;
```


### 4. 多表连接查询
当需要从多个相关表中获取数据时，使用连接查询：

#### （1）内连接（INNER JOIN）：只返回匹配的记录
```sql
-- 查询用户及其订单信息（只显示有订单的用户）
SELECT u.username, o.order_no, o.total_amount 
FROM users u
INNER JOIN orders o ON u.id = o.user_id;
```

#### （2）左连接（LEFT JOIN）：返回左表所有记录，右表匹配不到则为NULL
```sql
-- 查询所有用户及其订单（包括没有订单的用户）
SELECT u.username, o.order_no 
FROM users u
LEFT JOIN orders o ON u.id = o.user_id;
```

#### （3）右连接（RIGHT JOIN）：返回右表所有记录，左表匹配不到则为NULL
```sql
-- 查询所有订单及对应的用户（包括用户信息不存在的订单）
SELECT o.order_no, u.username 
FROM users u
RIGHT JOIN orders o ON u.id = o.user_id;
```


### 5. 子查询
子查询是嵌套在其他查询中的查询，用于复杂条件判断：
```sql
-- 子查询：查询订单金额大于平均值的订单
SELECT * FROM orders 
WHERE total_amount > (SELECT AVG(total_amount) FROM orders);

-- 子查询作为条件列表
SELECT * FROM users 
WHERE id IN (SELECT user_id FROM orders WHERE status = 1);
```


### 6. 查询优化建议
- 避免使用 `SELECT *`，只查询需要的字段
- 为查询条件中的字段建立索引（如 `WHERE`、`JOIN`、`ORDER BY` 后的字段）
- 合理使用 `LIMIT` 减少返回数据量
- 复杂查询优先考虑 `JOIN` 而非子查询（多数情况下性能更好）
- 避免在 `WHERE` 子句中对字段进行函数操作（会导致索引失效）


查询是 MySQL 最核心的功能，掌握各种查询技巧能高效获取所需数据。实际应用中，需根据数据量和业务需求选择合适的查询方式，并关注查询性能。
