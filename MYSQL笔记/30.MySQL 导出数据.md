MySQL 导出数据是数据库备份、迁移、数据共享的常用操作，可通过命令行工具或客户端工具实现，支持导出为 SQL 脚本、CSV 等格式。以下是关于 MySQL 数据导出的核心知识：


### 一、常用导出工具
MySQL 提供多种工具导出数据，最常用的是 **`mysqldump` 命令行工具**（适合 SQL 脚本导出）和 **`SELECT ... INTO OUTFILE` 语句**（适合文本文件导出）。


### 二、使用 `mysqldump` 导出数据
`mysqldump` 是 MySQL 官方提供的命令行工具，可导出数据库结构、数据，或两者的组合，生成 SQL 脚本（包含 `CREATE TABLE`、`INSERT` 等语句），适用于备份和迁移。

#### 基本语法：
```bash
mysqldump [选项] 数据库名 [表名1 表名2 ...] > 导出文件路径.sql
```

#### 常用选项：
| 选项                | 作用                                  |
|---------------------|---------------------------------------|
| `-u 用户名`         | 指定数据库用户名                      |
| `-p`                | 提示输入密码（不建议直接写密码 `-p123`，有安全风险） |
| `-h 主机名`         | 指定数据库主机（默认本地 `localhost`） |
| `-P 端口号`         | 指定数据库端口（默认 3306）           |
| `--databases 库1 库2` | 导出多个数据库（需显式指定，否则默认导出单个库的表） |
| `--all-databases`   | 导出所有数据库（包括系统库）          |
| `-d` 或 `--no-data` | 只导出表结构，不导出数据              |
| `-t` 或 `--no-create-info` | 只导出数据，不导出表结构           |
| `--where="条件"`    | 导出符合条件的数据（如 `--where="id < 100"`） |
| `--ignore-table=库.表` | 排除指定表（如需排除多个，重复该选项） |


#### 典型示例：
1. **导出单个数据库（结构+数据）**：  
   导出 `test` 数据库到 `test_backup.sql`：  
   ```bash
   mysqldump -u root -p test > /backup/test_backup.sql
   ```
   执行后输入密码，生成的 SQL 脚本可直接用于恢复（通过 `mysql` 命令导入）。

2. **只导出表结构**：  
   导出 `test` 库中 `users` 表的结构：  
   ```bash
   mysqldump -u root -p -d test users > /backup/users_struct.sql
   ```

3. **只导出数据**：  
   导出 `test` 库中 `orders` 表的数据（不含 `CREATE TABLE`）：  
   ```bash
   mysqldump -u root -p -t test orders > /backup/orders_data.sql
   ```

4. **导出符合条件的数据**：  
   导出 `users` 表中 `status=1` 的数据：  
   ```bash
   mysqldump -u root -p test users --where="status=1" > /backup/users_active.sql
   ```

5. **导出多个数据库**：  
   导出 `test` 和 `blog` 两个库：  
   ```bash
   mysqldump -u root -p --databases test blog > /backup/multi_db.sql
   ```


### 三、使用 `SELECT ... INTO OUTFILE` 导出文本文件
该语句可将查询结果导出为 CSV、TXT 等文本文件，适合数据交换（如导入到 Excel、其他系统）。

#### 基本语法：
```sql
SELECT 列1, 列2, ... 
FROM 表名 
[WHERE 条件]
INTO OUTFILE '文件路径'
[OPTIONS];  -- 可选：指定分隔符、换行符等
```

#### 常用 OPTIONS：
- `FIELDS TERMINATED BY '分隔符'`：字段间分隔符（如 `','` 用于 CSV）。  
- `OPTIONALLY ENCLOSED BY '字符'`：字段用指定字符包裹（如 `"` 处理含空格的字段）。  
- `LINES TERMINATED BY '换行符'`：行分隔符（默认 `\n`，Windows 常用 `\r\n`）。  


#### 示例：
1. **导出为 CSV 文件**（逗号分隔，字段用双引号包裹）：  
   ```sql
   SELECT id, username, email 
   FROM test.users 
   INTO OUTFILE '/var/lib/mysql-files/users.csv'  -- 路径需在 MySQL 允许的目录（由 secure_file_priv 配置）
   FIELDS TERMINATED BY ',' 
   OPTIONALLY ENCLOSED BY '"'
   LINES TERMINATED BY '\r\n';  -- Windows 换行符
   ```

2. **导出符合条件的数据到 TXT**（制表符分隔）：  
   ```sql
   SELECT order_id, amount, order_time 
   FROM test.orders 
   WHERE order_time >= '2023-01-01'
   INTO OUTFILE '/var/lib/mysql-files/2023_orders.txt'
   FIELDS TERMINATED BY '\t'  -- 制表符分隔
   LINES TERMINATED BY '\n';  -- Linux 换行符
   ```


#### 注意事项：
- **文件路径限制**：导出路径需在 MySQL 配置 `secure_file_priv` 指定的目录中（可通过 `SHOW VARIABLES LIKE 'secure_file_priv';` 查看）。若为 `NULL`，则禁止导出；若为空，允许任意目录（不推荐，有安全风险）。  
- **权限要求**：执行用户需有 `FILE` 权限，且操作系统用户（如 `mysql`）需有写入目标路径的权限。  
- **文件不存在**：目标文件必须不存在，否则报错（避免覆盖现有文件）。  


### 四、其他导出方式
1. **图形化工具**：  
   如 Navicat、DBeaver、MySQL Workbench 等，通过可视化界面操作：  
   - 右键数据库/表 → 导出向导 → 选择格式（SQL、CSV、Excel 等）→ 配置选项 → 执行。  
   适合非命令行用户，操作简单但效率较低（大表可能卡顿）。

2. **导出为 Excel 兼容格式**：  
   先通过 `SELECT ... INTO OUTFILE` 导出为 CSV，再用 Excel 打开（CSV 是 Excel 兼容的文本格式）。


### 五、导出注意事项
1. **大表导出性能**：  
   - 大表（千万级以上）用 `mysqldump` 时可添加 `--quick` 选项（逐行读取，不缓存到内存），避免内存溢出。  
   - 或分批次导出（结合 `--where` 按 ID 范围拆分）。

2. **数据一致性**：  
   - 导出时若表有写入操作，可能导致数据不一致。可先锁表（`FLUSH TABLES WITH READ LOCK;`），导出后解锁（`UNLOCK TABLES;`），但会阻塞写入，建议在低峰期执行。  
   - InnoDB 表可使用 `--single-transaction` 选项（生成一致性快照，不锁表）：  
     ```bash
     mysqldump -u root -p --single-transaction test > test_consistent.sql
     ```

3. **字符集问题**：  
   导出时指定字符集（如 `--default-character-set=utf8mb4`），避免中文乱码：  
   ```bash
   mysqldump -u root -p --default-character-set=utf8mb4 test > test_utf8.sql
   ```

4. **安全问题**：  
   - `mysqldump` 脚本包含数据（可能含密码等敏感信息），需限制文件权限（如 `chmod 600 backup.sql`）。  
   - 避免在命令行直接写密码（`-p123`），防止被系统日志记录。


### 总结
- **`mysqldump`**：适合导出 SQL 脚本（结构+数据），用于备份和迁移，支持灵活筛选。  
- **`SELECT ... INTO OUTFILE`**：适合导出文本文件（CSV/TXT），用于数据交换，需注意路径和权限。  
- 图形化工具：适合简单场景，操作便捷但效率较低。  

根据需求选择合适的方式，并注意性能、一致性和安全问题，确保导出结果可靠可用。
