在 MySQL 中，`AUTO_INCREMENT` 是实现“序列”功能的主要方式，用于自动生成唯一的递增数值（通常作为主键）。它能简化插入操作，避免手动管理序号，确保唯一性。以下是关于 `AUTO_INCREMENT` 的核心知识：


### 一、基本概念
`AUTO_INCREMENT` 是列的属性，仅适用于 **整数类型**（如 `INT`、`BIGINT` 等），且该列通常需要设置为 **主键或唯一索引**（否则可能报错或出现重复值）。

- 特点：  
  - 插入数据时，若不指定该列的值，会自动生成比当前最大值大 1 的数值。  
  - 若手动指定值，需确保不重复（否则报主键冲突错误）。  
  - 序列值仅在当前表内唯一，不同表的 `AUTO_INCREMENT` 相互独立。


### 二、使用方法
#### 1. 创建表时定义 `AUTO_INCREMENT`
```sql
CREATE TABLE users (
  id INT NOT NULL AUTO_INCREMENT,  -- 自增列
  username VARCHAR(50) NOT NULL,
  PRIMARY KEY (id)  -- 自增列通常作为主键
);
```
- 必须指定 `NOT NULL`（自增列默认非空）。  
- 若未指定主键，需为该列创建唯一索引，否则无法使用 `AUTO_INCREMENT`。


#### 2. 插入数据时使用
```sql
-- 不指定自增列的值，自动生成
INSERT INTO users (username) VALUES ('alice');  -- id 自动为 1
INSERT INTO users (username) VALUES ('bob');    -- id 自动为 2

-- 手动指定自增列的值（需确保不重复）
INSERT INTO users (id, username) VALUES (5, 'charlie');  -- id=5
INSERT INTO users (username) VALUES ('david');           -- 下一个自动值为 6（基于当前最大值 5）
```


#### 3. 查看当前自增值
通过 `SHOW TABLE STATUS` 查看表的当前自增序列值：
```sql
SHOW TABLE STATUS LIKE 'users';  -- 结果中 Auto_increment 字段即为下一个将生成的值
```


#### 4. 修改自增起始值或步长
- **修改起始值**：创建表后，可通过 `ALTER TABLE` 调整下一个自增值（需大于当前最大值，否则无效）：
  ```sql
  ALTER TABLE users AUTO_INCREMENT = 10;  -- 下一个值从 10 开始
  ```

- **修改步长**：`AUTO_INCREMENT` 的步长由系统变量 `auto_increment_increment` 控制（默认值为 1），可全局或会话级修改：
  ```sql
  -- 全局生效（需权限，重启后失效，永久生效需改配置文件）
  SET GLOBAL auto_increment_increment = 2;  -- 步长改为 2（1,3,5...）
  
  -- 当前会话生效
  SET SESSION auto_increment_increment = 2;
  ```


### 三、注意事项
#### 1. 自增值的“不连续性”
- **删除数据后不回滚**：若删除表中最大 ID 的记录，下一个自增值仍会基于删除前的最大值递增（例如删除 id=5 的记录，下一个值仍为 6）。  
- **插入失败后仍占用**：若插入时指定了自增值但失败（如主键冲突），该值可能被“占用”，导致后续自增值跳号（例如尝试插入 id=5 失败，下一个自动值可能为 6）。


#### 2. 存储引擎的差异
- **InnoDB**：自增值默认存储在内存中，重启数据库后，会重新计算（取表中最大 id + 1），可能导致跳号。  
  - 若需持久化自增值（避免重启跳号），MySQL 8.0 可通过 `innodb_autoinc_lock_mode = 0` 实现（但会影响并发性能）。  
- **MyISAM**：自增值存储在磁盘中，重启后不会丢失，无跳号问题。


#### 3. 批量插入的自增行为
- 对于批量插入（如 `INSERT ... SELECT`），InnoDB 会预分配一批自增值（数量取决于插入行数），若实际插入行数少于预分配数量，未使用的值会被浪费，导致跳号。  
- 可通过 `innodb_autoinc_lock_mode` 调整锁定模式（0/1/2），平衡并发与连续性（默认值 1，兼顾性能和大部分场景）。


#### 4. 避免重复的最佳实践
- 始终将 `AUTO_INCREMENT` 列设为 **主键**（主键自带唯一约束）。  
- 避免手动插入自增列的值，除非明确需要（如数据迁移）。  
- 若需“连续序号”（如订单号），不建议依赖 `AUTO_INCREMENT`（因可能跳号），可通过事务 + 单独的序列表实现。


### 四、与其他数据库“序列”的区别
- MySQL 没有 Oracle 等数据库中的独立 `SEQUENCE` 对象，`AUTO_INCREMENT` 是表级属性，绑定到具体列。  
- 若需跨表共享序列，需手动实现（如创建一个专门的序列表，通过 `SELECT ... FOR UPDATE` 锁定并获取下一个值）。


### 五、适用场景
- 作为表的主键（最常见），确保每行数据唯一标识。  
- 生成无业务含义的序号（如日志 ID、操作记录 ID）。  

通过合理使用 `AUTO_INCREMENT`，可简化主键管理，提升插入效率，但需注意其“不连续”特性，避免在强依赖连续序号的场景中使用。
