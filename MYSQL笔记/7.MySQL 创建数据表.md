创建数据表是 MySQL 中组织数据的核心操作，数据表由行（记录）和列（字段）组成，每个字段需指定数据类型和约束条件。以下是关于 MySQL 创建数据表的核心知识：


### 1. 创建数据表的基本语法
```sql
CREATE TABLE [IF NOT EXISTS] 表名 (
  字段1 数据类型 [约束条件],
  字段2 数据类型 [约束条件],
  ...
  [表级约束条件]
) [表选项];
```

- `IF NOT EXISTS`：可选，防止创建已存在的表时出错
- 字段定义：每个字段需指定名称、数据类型和约束（如 `id INT PRIMARY KEY`）
- 表选项：可指定存储引擎、字符集等（如 `ENGINE=InnoDB DEFAULT CHARSET=utf8mb4`）


### 2. 常用字段约束
约束用于保证数据完整性，常见约束如下：

| 约束          | 说明                                  |
|---------------|---------------------------------------|
| `PRIMARY KEY` | 主键，唯一标识记录，不允许重复和 NULL |
| `NOT NULL`    | 字段值不允许为 NULL                   |
| `UNIQUE`      | 字段值唯一（允许 NULL，但 NULL 只出现一次） |
| `DEFAULT`     | 字段默认值                            |
| `AUTO_INCREMENT` | 自增，常用于主键（仅整数类型可用）    |
| `FOREIGN KEY` | 外键，关联其他表的主键，保证参照完整性 |


### 3. 创建表的示例
```sql
-- 创建用户表（users）
CREATE TABLE IF NOT EXISTS users (
  id INT PRIMARY KEY AUTO_INCREMENT,  -- 自增主键
  username VARCHAR(50) NOT NULL UNIQUE,  -- 用户名（非空且唯一）
  password VARCHAR(100) NOT NULL,     -- 密码（非空）
  age TINYINT UNSIGNED,               -- 年龄（无符号整数）
  gender ENUM('male', 'female', 'other'),  -- 性别（枚举类型）
  email VARCHAR(100) UNIQUE,          -- 邮箱（唯一）
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,  -- 创建时间（默认当前时间）
  status TINYINT DEFAULT 1            -- 状态（默认1：正常）
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;  -- 表选项：存储引擎和字符集
```


### 4. 表级约束示例（多字段联合约束）
```sql
-- 创建订单表（orders），包含外键和联合唯一约束
CREATE TABLE IF NOT EXISTS orders (
  order_id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL,
  order_no VARCHAR(50) NOT NULL,
  total_amount DECIMAL(10,2) NOT NULL,
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  
  -- 表级约束：联合唯一（同一用户不能有相同订单号）
  UNIQUE KEY uk_user_order (user_id, order_no),
  
  -- 外键约束：关联 users 表的 id
  FOREIGN KEY (user_id) REFERENCES users(id) 
    ON DELETE CASCADE  -- 当 users 表中记录删除时，关联的订单也删除
    ON UPDATE CASCADE  -- 当 users 表中 id 更新时，订单表的 user_id 也更新
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```


### 5. 常用表选项
- **存储引擎**：`ENGINE=InnoDB`（默认，支持事务、外键）；`ENGINE=MyISAM`（不支持事务，查询快）
- **字符集**：`DEFAULT CHARSET=utf8mb4`（推荐，支持所有字符包括表情）
- **自增起始值**：`AUTO_INCREMENT=1000`（设置自增字段的起始值）


### 6. 查看表结构
```sql
-- 查看表基本结构
DESCRIBE 表名;
-- 或简写
DESC 表名;

-- 查看表创建语句（包含完整信息）
SHOW CREATE TABLE 表名;
```


### 7. 创建表的注意事项
- **命名规范**：表名建议使用小写字母，用下划线分隔（如 `user_orders`）
- **数据类型选择**：根据实际需求选择最合适的类型（如手机号用 `CHAR(11)`）
- **主键设计**：每张表建议设置主键，通常用自增 `INT` 或 `BIGINT`
- **外键使用**：外键会降低写入性能，高并发场景可考虑逻辑关联而非物理外键
- **字符集**：统一使用 `utf8mb4` 避免中文乱码问题


创建数据表时，合理的结构设计和约束设置是保证数据完整性和查询性能的基础，需结合业务场景仔细规划。
