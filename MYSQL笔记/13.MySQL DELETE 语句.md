MySQL 的 `DELETE` 语句用于从表中删除记录，是数据库操作中常用的命令之一。以下是关于 `DELETE` 语句的核心知识：


### 1. 基本语法
```sql
DELETE FROM 表名
[WHERE 条件];  -- 可选，指定要删除的行
```

**说明**：
- `FROM` 后指定要删除数据的表名
- `WHERE` 子句用于筛选需要删除的行（**如果省略 `WHERE`，将删除表中所有行**）


### 2. 基本示例
假设有 `products` 表：

| id  | name      | price | status   |
|-----|-----------|-------|----------|
| 1   | 手机      | 3999  | in_stock |
| 2   | 电脑      | 5999  | out_stock|
| 3   | 平板      | 2999  | in_stock |

#### 示例 1：删除单行
```sql
-- 删除 id=2 的产品
DELETE FROM products
WHERE id = 2;
```

#### 示例 2：删除符合条件的多行
```sql
-- 删除所有库存不足的产品
DELETE FROM products
WHERE status = 'out_stock';
```

#### 示例 3：删除所有行（清空表）
```sql
-- 谨慎使用！会删除表中所有数据
DELETE FROM products;
```


### 3. 高级用法
#### ① 多表关联删除
可以通过 `JOIN` 关联多个表，删除符合条件的记录：
```sql
-- 删除没有订单的用户（关联 orders 表）
DELETE u
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
WHERE o.id IS NULL;
```

#### ② 使用 `LIMIT` 限制删除行数（仅 MySQL 支持）
```sql
-- 最多删除 5 条状态为过期的记录
DELETE FROM logs
WHERE status = 'expired'
LIMIT 5;
```

#### ③ 按顺序删除
结合 `ORDER BY` 和 `LIMIT` 按特定顺序删除：
```sql
-- 删除最早的 10 条日志
DELETE FROM system_logs
ORDER BY create_time ASC
LIMIT 10;
```


### 4. 与 `TRUNCATE` 的区别
| 特性         | `DELETE`                  | `TRUNCATE`                |
|--------------|---------------------------|---------------------------|
| 操作类型     | DML（数据操纵语言）       | DDL（数据定义语言）       |
| 回滚         | 支持事务回滚              | 不支持回滚（隐式提交）    |
| 自增列       | 不重置自增计数器          | 重置自增计数器（通常）    |
| 速度         | 逐行删除，速度较慢        | 整表清空，速度更快        |
| 触发器       | 会触发删除触发器          | 不会触发触发器            |
| 权限         | 需要 `DELETE` 权限        | 需要 `DROP` 权限          |


### 5. 注意事项
1. **备份数据**：删除前务必备份重要数据，防止误操作
2. **事务保护**：关键删除操作建议使用事务（`BEGIN`/`COMMIT`）：
   ```sql
   BEGIN;
   DELETE FROM users WHERE id = 100;
   -- 确认无误后提交
   COMMIT;
   -- 出错时回滚
   -- ROLLBACK;
   ```
3. **索引利用**：`WHERE` 条件中的列建议建立索引，提高删除效率
4. **避免全表删除**：除非明确需要，否则不要省略 `WHERE` 子句
5. **外键约束**：删除被外键引用的记录可能失败，需先处理关联数据
6. **性能影响**：删除大量数据可能锁表，建议分批删除（配合 `LIMIT`）


### 6. 常见错误
- 忘记写 `WHERE` 子句，导致全表数据被误删
- 没有处理外键关联，导致删除失败
- 对大型表执行全表删除，导致长时间锁表影响业务
- 权限不足（需要表的 `DELETE` 权限）


合理使用 `DELETE` 语句可以有效管理数据库中的无效数据，但由于其操作的不可逆性（除非有备份或事务保护），使用时必须格外谨慎。
