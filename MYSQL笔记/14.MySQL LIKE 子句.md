MySQL 的 `LIKE` 子句用于在 `WHERE` 子句中搜索列中的指定模式，通常与通配符配合使用，实现模糊查询功能。以下是关于 `LIKE` 子句的核心知识：


### 1. 基本语法
```sql
SELECT 列名 FROM 表名
WHERE 列名 LIKE 模式;
```

**说明**：
- `LIKE` 用于指定搜索模式
- 通常需要配合通配符使用，否则效果等同于 `=` 运算符


### 2. 常用通配符
`LIKE` 子句支持两种通配符：

| 通配符 | 描述 |
|--------|------|
| `%` | 匹配任意长度的字符串（包括 0 个字符） |
| `_` | 匹配单个字符 |


### 3. 基本示例
假设有 `employees` 表：

| id  | name      | job         | email               |
|-----|-----------|-------------|---------------------|
| 1   | Zhang San | Developer   | zhang@example.com   |
| 2   | Li Si     | Designer    | lisi@company.cn     |
| 3   | Wang Wu   | Developer   | wangwu@test.org     |
| 4   | Zhao Liu  | Manager     | zhaoliu@example.com |

#### 示例 1：使用 `%` 通配符
```sql
-- 查找名字以 "Zhang" 开头的员工
SELECT * FROM employees WHERE name LIKE 'Zhang%';

-- 查找邮箱包含 "example" 的员工
SELECT * FROM employees WHERE email LIKE '%example%';

-- 查找职位以 "er" 结尾的员工
SELECT * FROM employees WHERE job LIKE '%er';
```

#### 示例 2：使用 `_` 通配符
```sql
-- 查找名字是 3 个字符的员工（匹配 "Li Si" 中的 "Li" 不适用，这里仅举例单个字符匹配）
SELECT * FROM employees WHERE name LIKE '___';  -- 三个下划线匹配三个字符

-- 查找邮箱第二个字符是 "h" 的员工
SELECT * FROM employees WHERE email LIKE '_h%';
```

#### 示例 3：组合使用通配符
```sql
-- 查找邮箱格式为 "xxx@example.com" 的员工
SELECT * FROM employees WHERE email LIKE '%@example.com';

-- 查找名字第二个字符是 "i" 且长度为 5 的员工
SELECT * FROM employees WHERE name LIKE '_i___';
```


### 4. 否定形式：`NOT LIKE`
用于排除匹配指定模式的记录：
```sql
-- 查找职位不是以 "Developer" 开头的员工
SELECT * FROM employees WHERE job NOT LIKE 'Developer%';
```


### 5. 转义特殊字符
如果需要匹配包含 `%` 或 `_` 的数据，需要使用 `ESCAPE` 定义转义字符：
```sql
-- 查找密码包含 "50%" 的用户（假设转义字符为 #）
SELECT * FROM users WHERE password LIKE '%50#%%' ESCAPE '#';

-- 查找名称包含 "a_b" 的产品
SELECT * FROM products WHERE name LIKE '%a#_b%' ESCAPE '#';
```


### 6. 注意事项
1. **性能影响**：`LIKE` 以 `%` 开头（如 `%abc`）会导致索引失效，全表扫描，大数据量下效率较低
2. **区分大小写**：
   - 取决于数据库 collation（排序规则），如 `utf8_general_ci` 不区分大小写，`utf8_bin` 区分大小写
   - 可使用 `BINARY` 强制区分大小写：`WHERE BINARY name LIKE 'zhang%'`
3. **替代方案**：
   - 复杂模式匹配可使用 `REGEXP` 正则表达式
   - 全文搜索可使用 `FULLTEXT` 索引


### 7. 常见使用场景
- 搜索包含特定关键词的记录（如搜索文章内容）
- 按前缀/后缀筛选数据（如按邮箱域名筛选）
- 模糊匹配用户输入（如搜索框自动提示）


`LIKE` 子句是实现模糊查询的基础工具，但在处理大量数据时需注意优化查询性能，避免滥用通配符导致的效率问题。
