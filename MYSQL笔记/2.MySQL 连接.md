MySQL 连接是数据库操作的基础，以下是关于 MySQL 连接的核心知识：

### 1. 连接方式
- **本地连接**：通过 `localhost` 或 `127.0.0.1` 连接本地 MySQL 服务器
- **远程连接**：连接网络中其他服务器上的 MySQL 服务

### 2. 连接基本要素
- **主机地址**：服务器 IP 或域名（本地常用 `localhost`）
- **端口号**：默认 3306（可在配置文件中修改）
- **用户名**：用于认证的账号（如 root）
- **密码**：用户对应的密码
- **数据库名**：可选，指定连接后默认使用的数据库

### 3. 常用连接工具
- **命令行客户端**：MySQL 自带的 `mysql` 命令
- **图形化工具**：Navicat、DBeaver、MySQL Workbench 等
- **编程语言连接**：通过各种语言的数据库驱动

### 4. 命令行连接示例
```bash
# 基本连接（默认端口）
mysql -h 主机地址 -u 用户名 -p

# 连接并指定端口和数据库
mysql -h 192.168.1.100 -P 3307 -u root -p mydatabase
```
输入命令后按提示输入密码即可连接。

### 5. 编程语言连接示例（Python）
使用 `pymysql` 库连接 MySQL：
```python
import pymysql

# 建立连接
connection = pymysql.connect(
    host='localhost',
    port=3306,
    user='root',
    password='your_password',
    database='your_database',
    charset='utf8mb4'
)

# 使用连接（示例）
try:
    with connection.cursor() as cursor:
        sql = "SELECT * FROM users"
        cursor.execute(sql)
        result = cursor.fetchall()
        print(result)
finally:
    connection.close()  # 关闭连接
```

### 6. 连接管理
- **连接池**：在高并发场景下使用，避免频繁创建和关闭连接
- **超时设置**：通过 `wait_timeout` 等参数设置连接超时时间
- **连接状态**：`SHOW PROCESSLIST` 命令查看当前所有连接

### 7. 常见连接问题
- **拒绝连接**：检查主机、端口是否正确，MySQL 服务是否启动
- **权限问题**：确保用户有远程连接权限（可通过 `GRANT` 命令授权）
- **防火墙**：检查服务器防火墙是否开放了 MySQL 端口

正确管理 MySQL 连接对于保证数据库性能和安全性至关重要，实际应用中需根据场景选择合适的连接策略。
