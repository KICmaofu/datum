Kali Linux 是渗透测试与网络安全领域的专用系统，内置大量安全工具，其命令可分为 **系统基础、网络操作、信息收集、漏洞扫描、权限提升、密码破解、流量分析** 等核心场景。以下是最常用命令的分类整理，附 **功能说明+实战示例**，适合渗透测试入门和日常使用：


## 一、系统基础命令（必备）
### 1. 系统更新与软件管理（Kali 工具依赖更新）
```bash
# 1. 更新软件源缓存（必做，避免安装失败）
sudo apt update

# 2. 更新已安装软件（包括渗透工具版本）
sudo apt upgrade -y  # -y 自动确认

# 3. 安装工具（示例：安装hydra密码破解工具）
sudo apt install hydra -y

# 4. 卸载工具
sudo apt remove hydra -y

# 5. 搜索工具（不确定工具名时使用）
apt search 关键词  # 例：apt search port scan
```

### 2. 用户与权限管理
```bash
# 1. 切换用户（root为最高权限，Kali默认禁用root登录，需手动启用）
sudo -i  # 切换到root用户（输入当前用户密码）
su 用户名  # 切换到普通用户

# 2. 启用root用户（首次使用需设置密码）
sudo passwd root  # 输入新root密码，之后可su root登录

# 3. 文件/目录权限修改（渗透中常用SUID提权）
chmod +x 脚本.sh  # 给脚本执行权限
chmod 777 目录  # 所有用户可读可写可执行（慎用）
chown 用户名:用户组 文件  # 修改文件所有者（例：chown kali:kali test.txt）

# 4. 查找SUID文件（提权常用，SUID文件可临时获得root权限）
find / -perm -4000 2>/dev/null  # 2>/dev/null 忽略错误输出
```

### 3. 进程与服务管理
```bash
# 1. 查看进程（查找恶意进程或占用端口的进程）
ps aux  # 查看所有进程
ps aux | grep 进程名  # 过滤特定进程（例：ps aux | grep mysql）

# 2. 终止进程（强制结束占用端口的服务）
kill -9 进程PID  # 例：kill -9 1234（PID通过ps命令获取）

# 3. 启动/停止服务（例：Apache、MySQL）
sudo systemctl start apache2  # 启动Apache服务
sudo systemctl stop apache2   # 停止服务
sudo systemctl status apache2 # 查看服务状态
```

### 4. 文件与目录操作
```bash
# 1. 基础导航与查看
ls -la  # 查看当前目录所有文件（包括隐藏文件，显示权限、大小）
cd 目录路径  # 切换目录（例：cd /root/Desktop）
pwd  # 显示当前目录路径

# 2. 文件创建/编辑
touch 文件名  # 创建空文件（例：touch payload.txt）
nano 文件名   # 简单文本编辑（按Ctrl+O保存，Ctrl+X退出）
vim 文件名    # 高级编辑（按i插入，Esc+:wq保存退出）

# 3. 文件复制/移动/删除
cp 源文件 目标路径  # 复制（例：cp payload.txt /tmp/）
mv 源文件 目标路径  # 移动/重命名（例：mv payload.txt shell.txt）
rm -rf 文件/目录  # 强制删除（慎用！-r递归删目录，-f强制）

# 4. 压缩/解压（渗透中常用传输文件）
tar -zcvf 压缩包.tar.gz 目录  # 压缩（例：tar -zcvf tools.tar.gz /usr/share/tools）
tar -zxvf 压缩包.tar.gz      # 解压（-x提取，-v显示过程）
unzip 压缩包.zip             # 解压zip文件
```


## 二、网络操作命令（渗透测试核心）
### 1. 网络配置与状态
```bash
# 1. 查看网卡信息（IP、MAC地址）
ip addr  # 替代老旧的ifconfig（推荐）
ifconfig  # 兼容旧系统

# 2. 测试网络连通性
ping 目标IP/域名  # 例：ping 8.8.8.8（测试外网）、ping google.com
ping -c 4 目标IP  # 发送4个数据包后停止（避免无限循环）

# 3. 查看路由表
route -n  # 显示网关、路由规则

# 4. 查看端口监听状态（类似Windows的netstat）
netstat -tuln  # 查看所有监听的TCP/UDP端口
netstat -ano | grep 端口号  # 查找特定端口（例：netstat -ano | grep 80）
ss -tuln  # 更高效的端口查看工具（替代netstat）
```

### 2. 网络探测与扫描（Kali核心功能）
#### （1）Nmap（端口扫描之王，必备！）
```bash
# 1. 快速扫描目标所有端口（默认扫描1000个常用端口）
nmap 目标IP  # 例：nmap 192.168.1.100

# 2. 全端口扫描（1-65535端口，耗时较长）
nmap -p- 目标IP  # 例：nmap -p- 192.168.1.100

# 3. 扫描端口+服务版本+操作系统指纹
nmap -sV -O 目标IP  # -sV：探测服务版本；-O：识别操作系统

# 4. 隐蔽扫描（SYN扫描，不易被防火墙检测）
nmap -sS 目标IP  # 需root权限（sudo nmap -sS 目标IP）

# 5. 脚本扫描（利用Nmap脚本库检测漏洞，如弱密码、端口漏洞）
nmap --script=vuln 目标IP  # 扫描常见漏洞
nmap --script=http-title 目标IP  # 获取网站标题
```

#### （2）Netcat（瑞士军刀，端口测试/反弹shell）
```bash
# 1. 测试目标端口是否开放
nc -zv 目标IP 端口  # 例：nc -zv 192.168.1.100 80（测试80端口）

# 2. 监听本地端口（接收反弹shell）
nc -lvp 端口  # -l：监听；-v：详细模式；-p：指定端口（例：nc -lvp 4444）

# 3. 连接目标端口（发送数据）
nc 目标IP 端口  # 例：nc 192.168.1.100 80（连接目标80端口，可发送HTTP请求）
```

#### （3）Ping扫描（批量检测存活主机）
```bash
# 快速扫描网段内存活主机（192.168.1.1-255）
fping -g 192.168.1.0/24 2>/dev/null  # 只显示存活主机
```


## 三、信息收集命令（渗透前期必备）
### 1. DNS查询（域名信息收集）
```bash
# 1. 基础DNS解析（获取IP）
nslookup 域名  # 例：nslookup google.com
dig 域名       # 更详细的DNS信息（A记录、MX记录等）

# 2. 反向DNS查询（IP反查域名）
dig -x 目标IP  # 例：dig -x 8.8.8.8

# 3. 批量查询子域名（Kali内置工具）
crunch 3 3 0-9 | xargs -I {} nslookup {}.target.com  # 生成3位数字子域名并查询
```

### 2. 域名/IP信息查询
```bash
# 1. 查看域名注册信息（WHOIS）
whois 域名  # 例：whois github.com（获取注册商、联系人、过期时间）

# 2. 查看IP归属地（Kali内置）
geoiplookup 目标IP  # 例：geoiplookup 8.8.8.8（显示美国谷歌服务器）
```

### 3. 网站信息收集
```bash
# 1. 网站指纹识别（服务器类型、框架、CMS）
whatweb 网址  # 例：whatweb https://www.baidu.com（识别Apache、PHP、MySQL等）

# 2. 目录扫描（查找隐藏页面/后台）
gobuster dir -u 网址 -w /usr/share/wordlists/dirb/common.txt  # 常用字典扫描
# 解释：-dir：目录扫描；-u：目标URL；-w：字典路径（Kali自带字典）

# 3. 爬取网站链接（递归获取所有页面）
wget -r -p 网址  # -r：递归爬取；-p：下载所有资源（图片、CSS）
```


## 四、漏洞扫描与利用命令
### 1. 漏洞库查询（Searchsploit）
```bash
# 1. 搜索漏洞（基于Exploit-DB数据库）
searchsploit 关键词  # 例：searchsploit Apache 2.4  # 查找Apache 2.4相关漏洞

# 2. 搜索并复制漏洞利用脚本到当前目录
searchsploit -m 漏洞ID  # 例：searchsploit -m 48978（复制ID为48978的脚本）

# 3. 更新漏洞数据库
searchsploit -u
```

### 2. Metasploit框架（msfconsole，渗透核心工具）
```bash
# 1. 启动Metasploit控制台

# 2. 搜索漏洞模块（在msfconsole内执行）
search 漏洞名称  # 例：search EternalBlue（永恒之蓝漏洞）

# 3. 使用漏洞模块（示例：永恒之蓝利用）
use exploit/windows/smb/ms17_010_eternalblue  # 加载模块

```

## 五、密码破解命令（暴力/字典攻击）
### 1. Hydra（多协议密码暴力破解）
```bash
# 1. SSH密码破解（常用）
hydra -l 用户名 -P 字典路径 目标IP ssh  # -l：单个用户名；-P：密码字典
# 例：hydra -l root -P /usr/share/wordlists/rockyou.txt 192.168.1.100 ssh

# 2. HTTP登录破解（如网站后台）
hydra -l admin -P 字典路径 目标IP http-post-form "/login.php:username=^USER^&password=^PASS^:登录失败提示"
# 解释：^USER^和^PASS^是占位符，替换为字典中的用户名和密码

# 3. RDP密码破解（Windows远程桌面）
hydra -l administrator -P 字典路径 目标IP rdp
```

### 2. John the Ripper（密码哈希破解）
```bash
# 1. 破解Linux影子文件（需获取/etc/shadow）
john /etc/shadow  # 自动识别哈希类型，使用默认字典

# 2. 破解指定哈希文件（例：破解MD5哈希）
# 先创建哈希文件（格式：用户名:哈希值）
echo "test:$1$abc$xyz123456789" > hash.txt  # MD5哈希示例
john --format=md5crypt hash.txt  # 指定哈希格式破解

# 3. 使用自定义字典破解
john --wordlist=自定义字典.txt hash.txt
```

### 3. Crunch（密码字典生成）
```bash
# 1. 生成3-5位数字密码字典
crunch 3 5 0-9 -o num_dict.txt  # -o：输出到文件

# 2. 生成字母+数字混合字典（长度4）
crunch 4 4 abc123 -o mix_dict.txt

# 3. 生成指定规则字典（例：首字母大写+3位数字）
crunch 4 4 -t @### -o rule_dict.txt  # @代表字母，#代表数字
```


## 六、流量分析命令（抓包与分析）
### 1. Tcpdump（命令行抓包工具）
```bash
# 1. 抓取所有网卡流量（保存到文件）
tcpdump -i any -w 抓包文件.pcap  # -i any：所有网卡；-w：保存到文件

# 2. 抓取特定网卡+端口流量（例：eth0网卡80端口）
tcpdump -i eth0 port 80 -w http.pcap

# 3. 读取抓包文件（分析已保存的流量）
tcpdump -r 抓包文件.pcap  # 显示包内容
tcpdump -r 抓包文件.pcap host 目标IP  # 过滤特定主机的流量
```

### 2. Wireshark（图形化抓包工具，命令行启动）
```bash
# 启动Wireshark图形界面（需root权限）
sudo wireshark

# 命令行模式过滤流量（保存到文件）
wireshark -i eth0 -f "tcp port 443" -w https.pcap  # 抓取HTTPS流量
```

## 八、重要注意事项
1. **法律风险**：所有渗透测试必须获得目标系统的 **书面授权**，未经授权的扫描/破解属于违法行为！
2. **工具更新**：Kali工具库更新频繁，建议定期执行 `sudo apt update && sudo apt upgrade` 保持工具最新。
3. **字典路径**：Kali自带常用密码字典（如 `rockyou.txt`），路径为 `/usr/share/wordlists/`，部分字典需解压（`gunzip rockyou.txt.gz`）。
4. **权限问题**：多数渗透工具（如nmap隐蔽扫描、hydra SSH破解）需要root权限，执行时加 `sudo`。

如果需要某类工具的深度使用教程（如Metasploit实战、Hydra针对性破解），可以进一步说明场景！
