Django 项目的结构遵循清晰的模块化设计，便于开发者分工协作和后期维护。一个典型的 Django 项目结构可分为**项目（Project）核心配置**和**应用（App）功能模块**两部分。以下是详细解析：


### 一、项目（Project）基础结构  
使用 `django-admin startproject myproject` 创建项目后，初始目录结构如下（以项目名 `myproject` 为例）：  

```
myproject/                  # 项目根目录（可自定义名称，如"blog_project"）
├── manage.py               # 项目管理脚本（核心命令入口）
└── myproject/              # 项目核心配置目录（与项目名同名，下称"配置目录"）
    ├── __init__.py         # 标识该目录为 Python 包（空文件即可）
    ├── settings.py         # 项目全局配置文件（核心）
    ├── urls.py             # 项目主 URL 路由配置
    ├── asgi.py             # 异步 Web 服务器入口（生产环境用）
    └── wsgi.py             # 同步 Web 服务器入口（生产环境用）
```  


#### 1. 根目录（`myproject/`）  
根目录是项目的容器，名称可自定义（建议与项目功能相关，如 `ecommerce` 电商项目）。核心文件是 `manage.py`：  

- **`manage.py`**：  
  项目的命令行管理工具，封装了 `django-admin` 命令，并自动关联当前项目的配置（无需手动指定 `settings.py`）。  
  常用命令：启动服务器（`runserver`）、数据库迁移（`migrate`）、创建超级用户（`createsuperuser`）等（详见 `django-admin` 命令详解）。  


#### 2. 配置目录（`myproject/myproject/`）  
存放项目的核心配置文件，是 Django 识别项目的关键，名称与项目名一致（不可随意修改）。  

- **`__init__.py`**：  
  空文件，告诉 Python 该目录是一个可导入的包（无需编写内容）。  

- **`settings.py`**：  
  项目的全局配置中心，所有核心配置均在此定义，例如：  
  - `INSTALLED_APPS`：注册项目中包含的应用（如 `myapp`）；  
  - `DATABASES`：配置数据库（默认 SQLite，可改为 MySQL、PostgreSQL 等）；  
  - `DEBUG`：是否开启调试模式（开发环境 `True`，生产环境 `False`）；  
  - `ALLOWED_HOSTS`：允许访问项目的域名/IP（生产环境需指定，如 `['example.com']`）；  
  - `STATIC_URL`/`MEDIA_URL`：静态文件（CSS/JS）和媒体文件（用户上传）的访问路径；  
  - `MIDDLEWARE`：配置中间件（如 CSRF 防护、会话管理等）。  

- **`urls.py`**：  
  项目的主 URL 路由配置，负责将用户请求的 URL 映射到对应的视图（View）。  
  例如，将 `/blog/` 路径转发到 `blog` 应用的路由配置：  
  ```python
  from django.urls import path, include

  urlpatterns = [
      path('admin/', admin.site.urls),  # Django 自带的 Admin 后台路由
      path('blog/', include('blog.urls')),  # 转发到 blog 应用的 urls.py
  ]
  ```  

- **`asgi.py` 与 `wsgi.py`**：  
  用于生产环境的 Web 服务器入口文件，分别对应异步（ASGI）和同步（WSGI）协议。  
  - WSGI：传统同步服务器（如 Gunicorn）使用；  
  - ASGI：支持异步处理的服务器（如 Uvicorn）使用，兼容 WSGI。  
  开发阶段无需修改，部署时需配合服务器配置使用。  


### 二、应用（App）结构  
Django 项目由多个**应用（App）** 组成（如博客项目可分为 `article` 文章模块、`user` 用户模块）。使用 `python manage.py startapp myapp` 创建应用后，目录结构如下：  

```
myapp/                      # 应用目录（名称自定义，如"blog"、"shop"）
├── __init__.py             # 标识为 Python 包
├── admin.py                # 配置 Admin 后台（注册模型，实现数据管理）
├── apps.py                 # 应用配置（可自定义应用名称、信号等）
├── migrations/             # 数据库迁移文件目录（自动生成，记录模型变更）
│   └── __init__.py
├── models.py               # 数据模型（定义数据库表结构，核心文件）
├── tests.py                # 单元测试代码（测试应用功能）
├── urls.py                 # 应用内部 URL 路由（需手动创建）
└── views.py                # 视图（处理业务逻辑，接收请求并返回响应）
```  


#### 应用核心文件解析  
- **`models.py`**：  
  定义数据模型（对应数据库表），通过 Django ORM 实现数据操作（无需写 SQL）。  
  例如，定义一个 `Article` 文章模型：  
  ```python
  from django.db import models

  class Article(models.Model):
      title = models.CharField(max_length=100)  # 标题（字符串类型）
      content = models.TextField()              # 内容（长文本类型）
      create_time = models.DateTimeField(auto_now_add=True)  # 创建时间（自动添加）
  ```  

- **`views.py`**：  
  处理业务逻辑的核心，接收用户请求（如 `GET`/`POST`），调用模型处理数据，最终返回响应（如渲染模板、JSON 数据）。  
  例如，一个简单的视图函数：  
  ```python
  from django.shortcuts import render
  from .models import Article

  def article_list(request):
      articles = Article.objects.all()  # 从数据库获取所有文章
      return render(request, 'article/list.html', {'articles': articles})  # 渲染模板
  ```  

- **`admin.py`**：  
  配置 Admin 后台，注册模型后可通过 `/admin/` 路径管理数据（无需手动开发后台界面）。  
  例如，注册 `Article` 模型：  
  ```python
  from django.contrib import admin
  from .models import Article

  admin.site.register(Article)  # 注册后 Admin 后台可管理 Article 数据
  ```  

- **`migrations/`**：  
  存放数据库迁移文件（自动生成），记录 `models.py` 的每一次修改（如新增字段、删除模型）。通过 `makemigrations` 生成，`migrate` 应用到数据库。  

- **`urls.py`**（需手动创建）：  
  应用内部的 URL 路由配置，定义当前应用的 URL 规则，再通过项目的 `urls.py` 转发。  
  例如，`myapp/urls.py`：  
  ```python
  from django.urls import path
  from . import views

  urlpatterns = [
      path('list/', views.article_list, name='article_list'),  # 文章列表页
  ]
  ```  


### 三、其他常用目录（需手动创建）  
在实际开发中，还需手动创建以下目录（需在 `settings.py` 中配置路径）：  

- **`static/`**：存放静态文件（CSS、JS、图片等）。  
  配置：`settings.py` 中 `STATIC_URL = '/static/'`，可通过 `{% static 'css/style.css' %}` 在模板中引用。  

- **`templates/`**：存放 HTML 模板文件（可在项目根目录或应用目录下创建）。  
  配置：`settings.py` 的 `TEMPLATES` 列表中 `DIRS` 指定项目级模板目录（如 `[BASE_DIR / 'templates']`）。  

- **`media/`**：存放用户上传的媒体文件（如头像、图片）。  
  配置：需在 `settings.py` 中定义 `MEDIA_ROOT`（本地存储路径）和 `MEDIA_URL`（访问路径）。  


### 四、项目与应用的关系  
- **项目（Project）**：是整个 Web 应用的容器，包含全局配置（如数据库、域名、注册的应用等），一个项目可包含多个应用。  
- **应用（App）**：是功能独立的模块（如用户管理、商品展示），可被多个项目复用（如将一个博客应用集成到不同项目）。  


### 总结  
Django 项目结构通过“项目配置 + 应用模块”的设计，实现了**关注点分离**：  
- 项目配置（`settings.py`、`urls.py`）负责全局统筹；  
- 应用模块（`models.py`、`views.py`）负责具体功能实现。  

这种结构使得项目易于扩展和维护，尤其适合大型 Web 应用开发。
