Django ORM 对多表关系的支持非常强大，通过模型间的关联（一对一、一对多、多对多），可轻松实现复杂数据结构的操作。以下通过一个**博客系统**的场景（包含 `User`、`Article`、`Comment`、`Tag` 模型），详细讲解多表关系的定义与核心操作。


### 一、定义多表关系模型  
先设计四个关联模型，覆盖三种核心关系：  

| 模型       | 作用                | 关联关系                          |  
|------------|---------------------|-----------------------------------|  
| `User`     | 用户（复用Django内置） | 与 `Profile` 是**一对一**关系      |  
| `Profile`  | 用户资料            | 扩展 `User` 的信息                |  
| `Article`  | 文章                | 与 `User` 是**一对多**（一个用户多篇文章）；与 `Tag` 是**多对多**（一篇文章多个标签，一个标签多篇文章） |  
| `Comment`  | 评论                | 与 `Article` 是**一对多**（一篇文章多条评论）；与 `User` 是**一对多**（一个用户多条评论） |  


#### 模型定义代码  
```python
# myapp/models.py
from django.db import models
from django.contrib.auth.models import User  # 复用Django内置用户模型
from django.utils import timezone

# 1. 一对一：User（内置）与 Profile
class Profile(models.Model):
    user = models.OneToOneField(
        User, 
        on_delete=models.CASCADE,  # 级联删除：User删除，Profile也删除
        related_name="profile"  # 反向查询别名：user.profile 可获取Profile
    )
    bio = models.TextField(blank=True, verbose_name="个人简介")
    avatar = models.ImageField(upload_to="avatars/", null=True, verbose_name="头像")

    def __str__(self):
        return f"{self.user.username}的资料"


# 2. 多对多：Article 与 Tag
class Tag(models.Model):
    name = models.CharField(max_length=50, unique=True, verbose_name="标签名")

    def __str__(self):
        return self.name


# 3. 一对多：User 与 Article；多对多：Article 与 Tag
class Article(models.Model):
    title = models.CharField(max_length=200, verbose_name="标题")
    content = models.TextField(verbose_name="内容")
    create_time = models.DateTimeField(auto_now_add=True, verbose_name="创建时间")
    # 外键：一对多（User -> Article）
    author = models.ForeignKey(
        User, 
        on_delete=models.CASCADE,  # User删除，其文章也删除
        related_name="articles"  # 反向查询别名：user.articles 可获取所有文章
    )
    # 多对多：Article <-> Tag
    tags = models.ManyToManyField(
        Tag, 
        related_name="articles",  # 反向查询别名：tag.articles 可获取所有关联文章
        blank=True  # 允许文章无标签
    )

    def __str__(self):
        return self.title


# 4. 一对多：Article 与 Comment；一对多：User 与 Comment
class Comment(models.Model):
    content = models.TextField(verbose_name="评论内容")
    create_time = models.DateTimeField(auto_now_add=True, verbose_name="评论时间")
    # 外键1：一对多（Article -> Comment）
    article = models.ForeignKey(
        Article, 
        on_delete=models.CASCADE,  # 文章删除，评论也删除
        related_name="comments"  # 反向查询别名：article.comments 可获取所有评论
    )
    # 外键2：一对多（User -> Comment）
    user = models.ForeignKey(
        User, 
        on_delete=models.CASCADE,  # 用户删除，其评论也删除
        related_name="comments"  # 反向查询别名：user.comments 可获取所有评论
    )

    def __str__(self):
        return f"{self.user.username}对《{self.article.title}》的评论"
```  


#### 执行迁移  
定义模型后，生成并执行迁移文件，创建关联表：  
```bash
python manage.py makemigrations
python manage.py migrate
```  


### 二、多表数据的创建（Create）  
多表数据创建的核心是**维护关联关系**（如为文章指定作者、为评论关联文章和用户）。  


#### 1. 一对一关系（`User` 与 `Profile`）  
```python
# 进入Django Shell：python manage.py shell
from django.contrib.auth.models import User
from myapp.models import Profile, Article, Comment, Tag

# 创建用户（内置User模型）
user1 = User.objects.create_user(username="alice", password="123456")  # 普通用户
user2 = User.objects.create_user(username="bob", password="123456")

# 创建用户资料（一对一关联User）
profile1 = Profile.objects.create(
    user=user1,  # 关联user1
    bio="我是Alice，喜欢写博客"
)
# 或通过User的反向关联创建（更直观）
profile2 = user2.profile.create(bio="我是Bob，喜欢评论")  # 利用related_name="profile"
```  


#### 2. 一对多关系（`User` 与 `Article`）  
```python
# 创建文章（关联作者user1）
article1 = Article.objects.create(
    title="Django ORM 入门",
    content="Django ORM 是个好东西...",
    author=user1  # 关联作者（一对多的“一”方）
)

# 再创建一篇user1的文章
article2 = Article.objects.create(
    title="Python 基础语法",
    content="变量、函数、类...",
    author=user1
)

# 通过User的反向关联创建文章（利用related_name="articles"）
article3 = user2.articles.create(
    title="Java 与 Python 对比",
    content="各有优劣..."
)  # 无需指定author，自动关联user2
```  


#### 3. 多对多关系（`Article` 与 `Tag`）  
多对多关系需先创建关联双方，再通过 `add()` 方法建立关联。  

```python
# 创建标签
tag1 = Tag.objects.create(name="Django")
tag2 = Tag.objects.create(name="Python")
tag3 = Tag.objects.create(name="编程")

# 为文章添加标签（方式1：通过Article的tags属性）
article1.tags.add(tag1, tag2)  # 给article1添加 tag1（Django）和 tag2（Python）

# 为文章添加标签（方式2：通过Tag的反向关联）
tag3.articles.add(article1, article2)  # 给tag3（编程）关联 article1 和 article2
```  


#### 4. 一对多关系（`Article` 与 `Comment`）  
```python
# 创建评论（关联文章和用户）
comment1 = Comment.objects.create(
    content="写得真好！",
    article=article1,  # 关联文章article1
    user=user2  # 关联用户user2（Bob）
)

# 通过Article的反向关联创建评论（利用related_name="comments"）
comment2 = article1.comments.create(
    content="请问ORM如何联表查询？",
    user=user1  # 关联user1（Alice）
)  # 无需指定article，自动关联article1
```  


### 三、多表数据的查询（Read）  
多表查询的核心是**正向查询**（从“关联方”查“被关联方”）和**反向查询**（从“被关联方”查“关联方”，依赖 `related_name`）。  


#### 1. 一对一查询（`User` ↔ `Profile`）  
- **正向查询**：从 `Profile` 查 `User`（直接访问外键字段）。  
- **反向查询**：从 `User` 查 `Profile`（通过 `related_name="profile"`）。  

```python
# 正向查询：Profile -> User
profile = Profile.objects.get(user=user1)
print(profile.user.username)  # 输出：alice（通过profile的user字段查用户）

# 反向查询：User -> Profile
user = User.objects.get(username="alice")
print(user.profile.bio)  # 输出：我是Alice，喜欢写博客（通过related_name="profile"）
```  


#### 2. 一对多查询（`User` ↔ `Article`）  
- **正向查询**：从 `Article` 查 `User`（访问 `author` 字段）。  
- **反向查询**：从 `User` 查所有 `Article`（通过 `related_name="articles"`）。  

```python
# 正向查询：Article -> User（查文章的作者）
article = Article.objects.get(title="Django ORM 入门")
print(article.author.username)  # 输出：alice（文章的作者是alice）

# 反向查询：User -> 所有Article（查用户的所有文章）
user = User.objects.get(username="alice")
alice_articles = user.articles.all()  # 利用related_name="articles"
for art in alice_articles:
    print(art.title)  # 输出：Django ORM 入门、Python 基础语法
```  


#### 3. 多对多查询（`Article` ↔ `Tag`）  
- **正向查询**：从 `Article` 查所有 `Tag`（访问 `tags` 字段）。  
- **反向查询**：从 `Tag` 查所有 `Article`（通过 `related_name="articles"`）。  

```python
# 正向查询：Article -> 所有Tag（查文章的所有标签）
article = Article.objects.get(title="Django ORM 入门")
article_tags = article.tags.all()
for tag in article_tags:
    print(tag.name)  # 输出：Django、Python、编程（前面添加的标签）

# 反向查询：Tag -> 所有Article（查标签关联的所有文章）
tag = Tag.objects.get(name="Python")
python_articles = tag.articles.all()  # 利用related_name="articles"
for art in python_articles:
    print(art.title)  # 输出：Django ORM 入门、Python 基础语法
```  


#### 4. 多层关联查询（跨表查询）  
通过“双下划线 `__`”实现跨表条件查询（类似 SQL 的 `JOIN`）。  

```python
# 示例1：查询所有标签包含“Django”的文章（Article -> Tag）
django_articles = Article.objects.filter(tags__name="Django")
for art in django_articles:
    print(art.title)  # 输出：Django ORM 入门

# 示例2：查询作者是“alice”的所有评论（Comment -> User）
alice_comments = Comment.objects.filter(user__username="alice")
for com in alice_comments:
    print(com.content)  # 输出：请问ORM如何联表查询？

# 示例3：查询发表过“Python”标签文章的用户（User -> Article -> Tag）
python_authors = User.objects.filter(articles__tags__name="Python").distinct()
# distinct() 去重（避免一个用户被多次查询）
for user in python_authors:
    print(user.username)  # 输出：alice（她的文章有Python标签）
```  


#### 5. 聚合与统计（多表）  
结合 `annotate()` 对关联表进行分组统计（如统计每个用户的文章数、每个文章的评论数）。  

```python
from django.db.models import Count, Avg

# 示例1：统计每个用户的文章数量（User + Article）
users_with_article_count = User.objects.annotate(
    article_count=Count("articles")  # 用related_name="articles"统计
).values("username", "article_count")
# 结果：[{'username': 'alice', 'article_count': 2}, {'username': 'bob', 'article_count': 1}]

# 示例2：统计每篇文章的评论数量（Article + Comment）
articles_with_comment_count = Article.objects.annotate(
    comment_count=Count("comments")  # 用related_name="comments"统计
).values("title", "comment_count")
# 结果：[{'title': 'Django ORM 入门', 'comment_count': 2}, ...]
```  


### 四、多表数据的更新（Update）  
多表更新主要涉及**关联关系的修改**（如更换文章作者、移除标签、修改评论的关联文章）。  


#### 1. 更新外键关联（一对多）  
```python
# 示例：将article3的作者从bob改为alice
article3 = Article.objects.get(title="Java 与 Python 对比")
article3.author = user1  # 更换作者（外键字段直接赋值新的User实例）
article3.save()

# 验证：查询alice的文章是否包含article3
print(article3 in user1.articles.all())  # 输出：True
```  


#### 2. 更新多对多关联（添加/移除标签）  
多对多关系通过 `add()`（添加）、`remove()`（移除）、`clear()`（清空）修改关联。  

```python
# 示例：给article1移除“Python”标签，添加“后端”标签
article1 = Article.objects.get(title="Django ORM 入门")
tag_python = Tag.objects.get(name="Python")
tag_backend = Tag.objects.create(name="后端")

article1.tags.remove(tag_python)  # 移除标签
article1.tags.add(tag_backend)    # 添加新标签

# 验证
print([tag.name for tag in article1.tags.all()])  # 输出：['Django', '编程', '后端']
```  


### 五、多表数据的删除（Delete）  
多表删除的核心是**级联删除策略**（通过 `on_delete` 参数控制），默认 `CASCADE` 会删除所有关联数据。  


#### 1. 删除“一”方，关联的“多”方自动删除（级联删除）  
```python
# 示例：删除用户alice，其关联的文章、评论、资料会被级联删除（因为on_delete=CASCADE）
user1 = User.objects.get(username="alice")
user1.delete()  # 删除用户

# 验证：alice的文章是否还存在
print(Article.objects.filter(author=user1).exists())  # 输出：False（已被删除）
print(Profile.objects.filter(user=user1).exists())    # 输出：False（已被删除）
```  


#### 2. 删除多对多中的一方，关联关系自动解除（不删除另一方）  
多对多关系中，删除 `Article` 或 `Tag` 时，仅会删除中间表的关联记录，不会删除另一方。  

```python
# 示例：删除标签“Django”，关联的文章仍存在，仅中间表记录删除
tag_django = Tag.objects.get(name="Django")
tag_django.delete()

# 验证：article1是否还存在（存在），但标签中已无“Django”
article1 = Article.objects.get(title="Django ORM 入门")  # 假设未被级联删除
print(tag_django in article1.tags.all())  # 输出：False（关联已解除）
```  


### 六、核心概念总结  
1. **关系类型**：  
   - 一对一（`OneToOneField`）：如用户与资料，适合扩展主表信息。  
   - 一对多（`ForeignKey`）：如用户与文章、文章与评论，是最常用的关系。  
   - 多对多（`ManyToManyField`）：如文章与标签，适合双向多关联场景。  

2. **关联查询**：  
   - 正向查询：通过模型的外键字段直接访问（如 `article.author`）。  
   - 反向查询：通过 `related_name` 访问（如 `user.articles`）。  
   - 跨表条件：用双下划线 `__` 实现（如 `filter(articles__tags__name="Python")`）。  

3. **级联删除**：`on_delete` 参数控制删除“一”方时“多”方的行为（`CASCADE` 级联删除、`SET_NULL` 设为NULL等）。  


通过以上实例，可掌握多表关系的定义与核心操作。实际开发中，多表查询是业务逻辑的核心，需灵活运用正向/反向查询和跨表条件筛选。
