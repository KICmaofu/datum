Django ORM（对象关系映射）的核心价值是通过 Python 代码操作数据库，无需编写 SQL。单表操作是 ORM 的基础，涵盖**创建（Create）、查询（Read）、更新（Update）、删除（Delete）** 四大核心操作。以下通过一个具体的 `Book` 模型实例，详细讲解单表操作的常用方法。


### 一、定义单表模型  
首先，在应用的 `models.py` 中定义一个简单的 `Book` 模型（对应数据库中的 `book` 表）：  

```python
# myapp/models.py
from django.db import models
from django.utils import timezone

class Book(models.Model):
    """图书模型（单表）"""
    title = models.CharField(max_length=200, verbose_name="书名")  # 书名（字符串）
    author = models.CharField(max_length=100, verbose_name="作者")  # 作者
    publish_date = models.DateField(default=timezone.now, verbose_name="出版日期")  # 出版日期
    price = models.DecimalField(max_digits=10, decimal_places=2, verbose_name="价格")  # 价格（精确到分）
    stock = models.IntegerField(default=0, verbose_name="库存")  # 库存数量
    is_active = models.BooleanField(default=True, verbose_name="是否在售")  # 是否在售

    class Meta:
        verbose_name = "图书"
        verbose_name_plural = "图书"
        ordering = ["-publish_date"]  # 默认按出版日期降序排序（最新的在前）

    def __str__(self):
        return f"{self.title}（{self.author}）"  # 实例的字符串表示
```  

定义后，执行数据库迁移，生成对应的表：  
```bash
python manage.py makemigrations  # 生成迁移文件
python manage.py migrate         # 执行迁移，创建表
```  


### 二、操作前准备：进入 Django Shell  
为了方便测试 ORM 操作，推荐使用 Django 自带的交互式 Shell（可直接执行 Python 代码操作数据库）：  
```bash
python manage.py shell  # 启动 Shell
```  

进入后，先导入模型：  
```python
from myapp.models import Book  # 导入 Book 模型
```  


### 三、创建数据（Create）  
通过 ORM 创建单表记录，有两种常用方式：  


#### 1. 实例化模型 + `save()` 方法  
先创建模型实例，设置字段值，再调用 `save()` 保存到数据库（类似 SQL 的 `INSERT`）。  

```python
# 方式 1：实例化后手动赋值
book1 = Book()
book1.title = "Django 实战"
book1.author = "张三"
book1.price = 89.00
book1.stock = 100
book1.save()  # 保存到数据库（此时才会生成 ID）

# 方式 2：实例化时直接传参（更简洁）
book2 = Book(
    title="Python 编程：从入门到实践",
    author="李四",
    publish_date="2020-01-15",  # 字符串自动转换为 date 类型
    price=79.80,
    stock=50
)
book2.save()  # 保存
```  


#### 2. `create()` 方法（一步到位）  
通过模型的 `objects.create()` 方法，直接创建并保存实例（内部自动调用 `save()`）。  

```python
book3 = Book.objects.create(
    title="算法导论",
    author="Thomas H. Cormen",
    publish_date="2006-08-01",
    price=128.00,
    stock=30,
    is_active=True
)
# 无需手动调用 save()，create() 会直接保存到数据库
```  


### 四、查询数据（Read）  
查询是 ORM 中最常用的操作，Django 提供了丰富的 API 用于单表查询。核心是通过模型的 `objects` 管理器（默认提供）调用查询方法，返回 `QuerySet`（查询集，类似列表的可迭代对象）或单个实例。  


#### 1. 基础查询  
| 方法                | 作用                                  | 返回值类型               | 示例                                  |  
|---------------------|---------------------------------------|--------------------------|---------------------------------------|  
| `all()`             | 查询所有记录                          | `QuerySet`（所有实例）   | `all_books = Book.objects.all()`      |  
| `get(**kwargs)`     | 查询符合条件的**单个**记录            | 模型实例（如 `Book`）    | `book = Book.objects.get(id=1)`       |  
| `filter(** kwargs)`  | 查询符合条件的**多个**记录            | `QuerySet`（匹配的实例） | `cheap_books = Book.objects.filter(price__lt=50)` |  


**示例解析**：  
```python
# 1. 查询所有图书（返回 QuerySet，可迭代）
all_books = Book.objects.all()
for book in all_books:
    print(book.title, book.price)  # 输出每本书的标题和价格

# 2. 查询 ID=1 的图书（get() 只能返回一条记录，否则抛异常）
try:
    book = Book.objects.get(id=1)  # 等价于 Book.objects.get(pk=1)（pk 是主键的别名）
    print(book.title, book.author)
except Book.DoesNotExist:  # 没有匹配记录时的异常
    print("图书不存在")
except Book.MultipleObjectsReturned:  # 匹配多条记录时的异常
    print("存在多条匹配的图书")

# 3. 查询价格 < 100 的图书（filter() 支持多条件）
cheap_books = Book.objects.filter(price__lt=100, is_active=True)  # 价格<100 且 在售
print(cheap_books.count())  # 统计数量（等价于 SQL 的 COUNT()）
```  


#### 2. 条件查询（字段查找）  
`filter()`、`get()` 等方法的参数支持**字段查找**（Field lookups），通过 `字段名__查找类型` 表示，用于实现复杂条件（如模糊匹配、范围查询）。  

常用字段查找类型：  

| 查找类型       | 作用                                  | 示例（查询条件）                          |  
|----------------|---------------------------------------|-------------------------------------------|  
| `exact`        | 精确匹配（默认，可省略）              | `title__exact="Django 实战"` → 标题完全等于 |  
| `icontains`    | 包含指定字符串（不区分大小写）        | `author__icontains="张"` → 作者名含“张”   |  
| `gt`/`lt`      | 大于/小于                             | `price__gt=50` → 价格>50                  |  
| `gte`/`lte`    | 大于等于/小于等于                     | `stock__lte=10` → 库存≤10                 |  
| `in`           | 在指定列表中                          | `id__in=[1,3,5]` → ID 是 1、3、5 中的一个 |  
| `range`        | 在指定范围内（如日期、数字）          | `publish_date__range=("2020-01-01", "2023-12-31")` → 出版日期在 2020-2023 年 |  
| `year`         | 日期字段的年份                        | `publish_date__year=2020` → 2020 年出版   |  


**示例**：  
```python
# 1. 查找作者名包含“李”的图书（不区分大小写）
li_books = Book.objects.filter(author__icontains="李")

# 2. 查找价格在 50~100 之间的在售图书
mid_price_books = Book.objects.filter(
    price__gte=50, 
    price__lte=100, 
    is_active=True
)

# 3. 查找 2020 年之后（含 2020）出版的图书
new_books = Book.objects.filter(publish_date__year__gte=2020)

# 4. 查找库存为空（0）或已下架（is_active=False）的图书（用 Q 对象实现 OR 条件）
from django.db.models import Q
invalid_books = Book.objects.filter(Q(stock=0) | Q(is_active=False))  # Q 表示“或”关系
```  


#### 3. 排序与限制结果  
- **排序**：通过 `order_by()` 方法指定排序字段（默认升序，字段前加 `-` 表示降序）。  
- **限制结果**：通过 Python 切片语法限制返回的记录数量（如 `[:5]` 取前 5 条）。  


**示例**：  
```python
# 1. 按价格升序排序（从低到高）
sorted_by_price = Book.objects.all().order_by("price")

# 2. 按出版日期降序（最新的在前），价格升序（同日期的低价在前）
sorted_complex = Book.objects.all().order_by("-publish_date", "price")

# 3. 取价格最高的 3 本在售图书
top3_expensive = Book.objects.filter(is_active=True).order_by("-price")[:3]
```  


#### 4. 聚合与统计  
通过 `django.db.models` 提供的聚合函数（如 `Count`、`Sum`、`Avg`），可对查询结果进行统计计算。  


**示例**：  
```python
from django.db.models import Count, Sum, Avg, Max

# 1. 统计所有图书的平均价格
avg_price = Book.objects.aggregate(avg=Avg("price"))  # 返回字典：{'avg': 89.2}
print(f"平均价格：{avg_price['avg']}")

# 2. 统计在售图书的总库存
total_stock = Book.objects.filter(is_active=True).aggregate(total=Sum("stock"))
print(f"在售总库存：{total_stock['total']}")

# 3. 统计 2020 年出版的图书数量
count_2020 = Book.objects.filter(publish_date__year=2020).aggregate(count=Count("id"))
print(f"2020 年出版的图书数量：{count_2020['count']}")

# 4. 查找最高价格
max_price = Book.objects.aggregate(max=Max("price"))["max"]
```  


### 五、更新数据（Update）  
更新单表记录有两种方式：修改实例属性后 `save()`，或通过 `update()` 批量更新。  


#### 1. 单条记录更新（`save()`）  
先查询实例，修改属性后调用 `save()`（会更新所有字段，适合单条记录）。  

```python
# 1. 查询要更新的图书
book = Book.objects.get(id=1)

# 2. 修改属性
book.price = 99.00  # 涨价
book.stock -= 10    # 库存减少 10

# 3. 保存更新（类似 SQL 的 UPDATE）
book.save()
```  


#### 2. 批量更新（`update()`）  
通过 `filter()` 筛选记录后，调用 `update()` 批量修改字段（更高效，不触发 `save()` 方法的信号）。  

```python
# 1. 批量将所有“张三”的图书价格提高 5 元
Book.objects.filter(author="张三").update(price=F("price") + 5)  # F() 表示字段自身的值

# 2. 批量将库存为 0 的图书标记为“下架”
Book.objects.filter(stock=0).update(is_active=False)
```  


### 六、删除数据（Delete）  
删除记录也支持单条删除和批量删除，通过 `delete()` 方法实现（类似 SQL 的 `DELETE`）。  


#### 1. 单条记录删除  
```python
# 1. 查询要删除的图书
book = Book.objects.get(id=3)

# 2. 删除（会立即执行）
book.delete()
```  


#### 2. 批量删除  
```python
# 1. 批量删除已下架（is_active=False）的图书
Book.objects.filter(is_active=False).delete()
```  


### 七、核心概念总结  
1. **`QuerySet`**：查询方法（如 `all()`、`filter()`）返回的可迭代对象，支持链式调用（如 `filter().order_by()`），**惰性执行**（仅在需要结果时才执行 SQL）。  
2. **`objects`**：模型的默认管理器，所有查询都通过它发起（可自定义管理器扩展功能）。  
3. **字段查找**：通过 `字段名__查找类型` 实现复杂条件，是单表查询的核心。  
4. **F/Q 对象**：`F()` 用于字段间比较（如 `price__gt=F("cost")`），`Q()` 用于组合条件（`OR` 关系）。  


通过以上实例，可掌握 Django ORM 单表的完整操作流程。实际开发中，单表操作是基础，复杂业务通常基于单表操作扩展（如关联查询、事务等）。
