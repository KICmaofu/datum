在多表关联场景中，聚合（Aggregation）和分组查询（Group by）是数据分析的核心需求，例如“统计每个作者的文章数”“计算每篇文章的平均评论数”等。Django ORM 通过 `aggregate()`（聚合）和 `annotate()`（分组）方法，结合 `Count`、`Sum` 等聚合函数，轻松实现多表统计，无需编写复杂 SQL。  


### 一、核心概念  
- **聚合（`aggregate()`）**：对**整个查询集**进行统计，返回一个包含统计结果的字典（如“所有文章的总评论数”）。  
- **分组（`annotate()`）**：对查询集中的**每个对象**添加一个统计属性（如“每篇文章各自的评论数”），相当于 SQL 的 `GROUP BY`。  
- **聚合函数**：来自 `django.db.models`，常用的有 `Count`（计数）、`Sum`（求和）、`Avg`（平均值）、`Max`（最大值）、`Min`（最小值）。  


### 二、模型回顾  
基于之前的博客系统模型（多表关联），本文示例将围绕以下模型展开：  
- `User`（内置用户模型）  
- `Article`（文章，外键关联 `User`，多对多关联 `Tag`）  
- `Comment`（评论，外键关联 `User` 和 `Article`）  
- `Tag`（标签，多对多关联 `Article`）  


### 三、聚合查询（`aggregate()`）  
聚合查询用于对**整个数据集**进行统计，结果是一个单一值（或多个统计值的字典）。  


#### 1. 单表聚合（复习）  
对单个模型的字段进行统计（如所有文章的平均价格），但多表场景中更常用跨表聚合。  


#### 2. 多表聚合（跨表统计）  
通过关联关系（外键、多对多），对关联表的字段进行统计。  


##### 示例 1：统计所有文章的总评论数  
`Article` 与 `Comment` 是一对多关系（`Article.comments`），统计所有评论的总数：  

```python
from django.db.models import Count
from myapp.models import Article

# 统计所有文章的评论总数（跨 Article -> Comment）
total_comments = Article.objects.aggregate(
    total=Count('comments')  # 'comments' 是 Article 中 Comment 的 related_name
)

print(total_comments)  # 输出：{'total': 5}（假设共有5条评论）
```  

**说明**：`Count('comments')` 表示统计所有 `Article` 关联的 `Comment` 记录总数，等价于 SQL：  
```sql
SELECT COUNT(comment.id) AS total FROM article
LEFT JOIN comment ON article.id = comment.article_id;
```  


##### 示例 2：统计所有用户的平均文章数  
`User` 与 `Article` 是一对多关系（`User.articles`），计算每个用户平均发表的文章数：  

```python
from django.db.models import Avg
from django.contrib.auth.models import User

# 统计所有用户的平均文章数（跨 User -> Article）
avg_articles_per_user = User.objects.aggregate(
    avg=Avg('articles')  # 'articles' 是 User 中 Article 的 related_name
)

print(avg_articles_per_user)  # 输出：{'avg': 2.5}（假设有2个用户，共5篇文章）
```  


##### 示例 3：统计所有标签的总文章数（去重）  
`Tag` 与 `Article` 是多对多关系，一个文章可能有多个标签，需去重统计（避免同一文章被多次计数）：  

```python
from django.db.models import Count
from myapp.models import Tag

# 统计所有标签关联的文章总数（去重，同一文章被多个标签关联时只算1次）
total_articles_across_tags = Tag.objects.aggregate(
    total=Count('articles', distinct=True)  # distinct=True 去重
)

print(total_articles_across_tags)  # 输出：{'total': 3}（假设有3篇不同的文章）
```  


### 四、分组查询（`annotate()`）  
分组查询为查询集中的**每个对象**添加一个统计属性（基于关联表），相当于“按当前模型分组，统计每组的关联数据”。  


#### 1. 按主模型分组（统计每个对象的关联数据）  


##### 示例 1：统计每篇文章的评论数  
为每篇 `Article` 添加 `comment_count` 属性，记录该文章的评论数：  

```python
from django.db.models import Count
from myapp.models import Article

# 为每篇文章添加评论数字段（Article -> Comment 一对多）
articles_with_comment_count = Article.objects.annotate(
    comment_count=Count('comments')  # 'comments' 是 Article 关联 Comment 的 related_name
).values('title', 'comment_count')  # 只保留标题和评论数

# 结果示例：
# [
#   {'title': 'Django ORM 入门', 'comment_count': 3},
#   {'title': 'Python 基础', 'comment_count': 1}
# ]
```  

**说明**：`annotate()` 返回的是 `Article` 查询集，每个 `Article` 实例多了 `comment_count` 属性，等价于 SQL：  
```sql
SELECT article.title, COUNT(comment.id) AS comment_count
FROM article
LEFT JOIN comment ON article.id = comment.article_id
GROUP BY article.id;
```  


##### 示例 2：统计每个用户的文章数和评论数  
为每个 `User` 添加 `article_count`（文章数）和 `comment_count`（评论数）：  

```python
from django.db.models import Count
from django.contrib.auth.models import User

# 统计每个用户的文章数和评论数（User -> Article 和 User -> Comment）
users_stats = User.objects.annotate(
    article_count=Count('articles'),  # 文章数（一对多）
    comment_count=Count('comments')   # 评论数（一对多）
).values('username', 'article_count', 'comment_count')

# 结果示例：
# [
#   {'username': 'alice', 'article_count': 2, 'comment_count': 3},
#   {'username': 'bob', 'article_count': 1, 'comment_count': 2}
# ]
```  


##### 示例 3：统计每个标签的关联文章数  
为每个 `Tag` 添加 `article_count`，记录该标签关联的文章数（多对多关系）：  

```python
from django.db.models import Count
from myapp.models import Tag

tags_with_article_count = Tag.objects.annotate(
    article_count=Count('articles')  # 'articles' 是 Tag 关联 Article 的 related_name
).values('name', 'article_count')

# 结果示例：
# [
#   {'name': 'Django', 'article_count': 2},
#   {'name': 'Python', 'article_count': 3}
# ]
```  


#### 2. 按关联模型分组（跨表分组）  
有时需要按关联表的字段分组，例如“按文章的作者分组，统计每个作者的平均评论数”。  


##### 示例：按作者分组，统计每个作者的平均文章评论数  
```python
from django.db.models import Avg, Count
from myapp.models import Article

# 步骤1：先为每篇文章添加评论数（article_comment_count）
# 步骤2：按作者分组，计算平均评论数
author_avg_comments = Article.objects.annotate(
    article_comment_count=Count('comments')  # 每篇文章的评论数
).values('author__username').annotate(  # 按作者（author__username）分组
    avg_comment=Avg('article_comment_count')  # 计算每个作者的平均评论数
)

# 结果示例：
# [
#   {'author__username': 'alice', 'avg_comment': 2.5},  # 她的2篇文章共5条评论，平均2.5
#   {'author__username': 'bob', 'avg_comment': 1.0}     # 他的1篇文章1条评论
# ]
```  


#### 3. 带条件的分组统计  
通过 `filter()` 在分组前筛选数据，或在聚合函数中添加条件，实现“符合条件的统计”。  


##### 示例：统计每个作者的“已发布”文章数（假设 `Article` 有 `is_published` 字段）  
```python
from django.db.models import Count
from myapp.models import Article

# 方式1：先筛选已发布的文章，再按作者分组统计
published_articles_by_author = Article.objects.filter(
    is_published=True  # 只统计已发布的文章
).annotate(
    article_count=Count('id')  # 统计每篇文章（但这里按作者分组，实际是作者的已发布文章数）
).values('author__username', 'article_count')

# 方式2：在聚合函数中添加条件（更灵活）
author_published_stats = User.objects.annotate(
    published_count=Count(
        'articles', 
        filter=models.Q(articles__is_published=True)  # 只统计满足条件的文章
    )
).values('username', 'published_count')
```  


### 五、关键注意事项  
1. **`distinct=True` 去重**：多对多关系或关联表有重复数据时，需用 `Count(..., distinct=True)` 避免重复计数（如一个文章被多个标签关联，统计标签的总文章数时去重）。  
2. **`values()` 控制输出字段**：`annotate()` 后用 `values()` 可只保留需要的字段，简化结果。  
3. **链式分组**：支持多次 `annotate()` 实现多级分组（如先按文章统计评论数，再按作者分组计算平均值）。  
4. **与 `F()`/`Q()` 结合**：`F()` 可引用字段值（如 `Sum(F('price') * F('stock'))`），`Q()` 可在聚合函数中添加复杂条件。  


### 六、总结  
聚合（`aggregate()`）和分组（`annotate()`）是多表统计的核心工具：  
- 聚合用于整体统计（如“所有文章的总评论数”），返回单一结果；  
- 分组用于为每个对象添加统计属性（如“每篇文章的评论数”），返回带统计字段的查询集；  
- 结合多表关联（外键、多对多）和 `related_name`，可轻松实现跨表统计，无需编写 SQL 联表查询。  

这些功能在后台数据分析、报表生成、 Dashboard 开发中尤为重要，是 Django ORM 强大之处的集中体现。
