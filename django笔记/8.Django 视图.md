Django 视图（View）是处理用户请求、执行业务逻辑并返回响应的核心组件，相当于 MVC 架构中的“控制器（Controller）”。它连接模型（数据）、模板（展示）和表单（用户输入），是整个应用的“业务逻辑层”。


### 一、视图的核心作用  
1. **接收请求**：获取用户发送的 HTTP 请求（如 `GET`/`POST`）及相关数据（URL 参数、表单数据等）。  
2. **处理逻辑**：调用模型操作数据（查询、新增、修改）、验证表单输入、实现业务规则（如登录验证、权限判断）。  
3. **返回响应**：生成并返回 HTTP 响应（如渲染模板页面、返回 JSON 数据、重定向到其他页面）。  


### 二、视图的类型  
Django 视图主要分为两种类型：**函数视图（Function-Based View, FBV）** 和 **类视图（Class-Based View, CBV）**。  


#### 1. 函数视图（FBV）  
以 Python 函数形式定义，接收 `request` 对象作为参数，根据请求方法（`GET`/`POST` 等）处理逻辑。适合简单场景，代码直观。  


##### 基本结构  
```python
# myapp/views.py
from django.http import HttpResponse, JsonResponse
from django.shortcuts import render, redirect

def simple_view(request):
    # request 是 HttpRequest 对象，包含请求的所有信息
    # 1. 处理逻辑（如获取请求方法、数据）
    method = request.method  # 获取请求方法（GET/POST 等）
    get_data = request.GET  # 获取 GET 参数（类似字典）
    post_data = request.POST  # 获取 POST 数据（表单提交）

    # 2. 返回响应
    if method == "GET":
        return HttpResponse("这是一个 GET 请求的响应")  # 返回纯文本
    elif method == "POST":
        return JsonResponse({"status": "success"})  # 返回 JSON 数据
```  


##### 常用响应方式  
- **`HttpResponse(content)`**：返回自定义内容（文本、HTML 等）。  
  ```python
  return HttpResponse("<h1>Hello Django</h1>")  # 返回 HTML
  ```  

- **`render(request, template_name, context)`**：渲染模板并返回（最常用）。  
  ```python
  def article_list(request):
      articles = Article.objects.all()  # 从模型获取数据
      context = {"articles": articles}  # 传递给模板的数据
      return render(request, "myapp/list.html", context)  # 渲染模板
  ```  

- **`redirect(to)`**：重定向到其他 URL（如成功提交后跳转）。  
  ```python
  return redirect("home")  # 跳转至名为 "home" 的路由
  return redirect("/article/1/")  # 跳转至指定 URL
  ```  

- **`JsonResponse(data)`**：返回 JSON 数据（适合前后端分离场景）。  
  ```python
  return JsonResponse({"name": "Django", "version": "4.2"})
  ```  


##### 处理表单与请求方法  
函数视图中需通过 `request.method` 判断请求类型（`GET` 显示表单，`POST` 处理提交）：  
```python
from .forms import LoginForm

def login(request):
    if request.method == "POST":
        # POST 请求：处理表单提交
        form = LoginForm(request.POST)
        if form.is_valid():
            # 验证成功：执行登录逻辑
            return redirect("home")
    else:
        # GET 请求：显示空表单
        form = LoginForm()
    
    # 渲染模板（无论 GET 还是验证失败的 POST 都走这里）
    return render(request, "myapp/login.html", {"form": form})
```  


#### 2. 类视图（CBV）  
以 Python 类形式定义，继承自 Django 内置的视图基类（如 `View`），通过类方法（`get()`/`post()` 等）处理不同请求。适合复杂场景，代码复用性强（如继承、混入）。  


##### 基本结构  
```python
# myapp/views.py
from django.views import View
from django.http import HttpResponse

class SimpleClassView(View):
    # 处理 GET 请求
    def get(self, request):
        return HttpResponse("这是类视图的 GET 响应")
    
    # 处理 POST 请求
    def post(self, request):
        return HttpResponse("这是类视图的 POST 响应")
```  


##### 与 URL 关联  
类视图需通过 `as_view()` 方法转换为可调用的视图函数，才能在 `urls.py` 中配置：  
```python
# myapp/urls.py
from django.urls import path
from .views import SimpleClassView

urlpatterns = [
    path('simple-class/', SimpleClassView.as_view(), name='simple_class'),
]
```  


##### 常用内置类视图  
Django 提供了多个封装好的通用类视图（`django.views.generic`），简化常见操作（如列表查询、详情展示、表单处理）：  

| 类视图          | 作用                          | 核心属性/方法                          |  
|-----------------|-------------------------------|---------------------------------------|  
| `TemplateView`  | 渲染指定模板（无复杂逻辑）    | `template_name`（指定模板路径）       |  
| `ListView`      | 展示模型列表（如文章列表）    | `model`（指定模型）、`context_object_name`（模板中变量名）、`queryset`（自定义查询集） |  
| `DetailView`    | 展示模型详情（如文章详情）    | `model`、`pk_url_kwarg`（URL 中的主键参数名，默认 `pk`） |  
| `FormView`      | 处理表单（如登录、注册）      | `form_class`（指定表单类）、`success_url`（验证成功后跳转地址） |  
| `CreateView`    | 创建模型实例（如发布文章）    | `model`、`fields`（需提交的字段）、`success_url` |  


##### 通用类视图示例  
- **`ListView` 展示文章列表**：  
  ```python
  from django.views.generic import ListView
  from .models import Article

  class ArticleListView(ListView):
      model = Article  # 关联的模型
      template_name = "myapp/article_list.html"  # 模板路径
      context_object_name = "articles"  # 模板中使用的变量名（默认是 object_list）
      paginate_by = 10  # 分页：每页 10 条数据

      # 自定义查询集（如只显示已发布的文章）
      def get_queryset(self):
          return Article.objects.filter(is_published=True).order_by("-publish_time")
  ```  

  模板中直接使用 `{{ articles }}` 遍历数据：  
  ```html
  {% for article in articles %}
      <h2>{{ article.title }}</h2>
  {% endfor %}
  ```  


- **`DetailView` 展示文章详情**：  
  ```python
  from django.views.generic import DetailView
  from .models import Article

  class ArticleDetailView(DetailView):
      model = Article
      template_name = "myapp/article_detail.html"
      context_object_name = "article"  # 模板中变量名（默认是 object）
  ```  

  配置 URL 时需传递主键（`pk`）：  
  ```python
  # urls.py
  path('article/<int:pk>/', ArticleDetailView.as_view(), name='article_detail'),
  ```  


- **`FormView` 处理登录表单**：  
  ```python
  from django.views.generic import FormView
  from django.urls import reverse_lazy
  from .forms import LoginForm

  class LoginView(FormView):
      form_class = LoginForm  # 关联的表单类
      template_name = "myapp/login.html"  # 表单模板
      success_url = reverse_lazy("home")  # 验证成功后跳转的 URL

      # 表单验证成功后执行的逻辑（如登录）
      def form_valid(self, form):
          username = form.cleaned_data["username"]
          password = form.cleaned_data["password"]
          # 这里添加登录验证逻辑（如调用 auth.authenticate）
          return super().form_valid(form)  # 调用父类方法，跳转到 success_url
  ```  


### 三、请求对象（`request`）  
视图函数/方法的第一个参数是 `HttpRequest` 对象，包含请求的所有信息，常用属性：  

| 属性/方法          | 说明                                  |  
|---------------------|---------------------------------------|  
| `request.method`    | 请求方法（`GET`/`POST`/`PUT` 等，大写字符串） |  
| `request.GET`       | GET 参数（类字典对象，如 `request.GET.get("page")`） |  
| `request.POST`      | POST 数据（表单提交，如 `request.POST.get("username")`） |  
| `request.FILES`     | 上传的文件（如 `request.FILES.get("avatar")`） |  
| `request.user`      | 当前登录用户（未登录为 `AnonymousUser`） |  
| `request.path`      | 请求的路径（如 `/article/1/`）         |  
| `request.META`      | 包含 HTTP 头信息的字典（如 `REMOTE_ADDR` 客户端 IP） |  


### 四、视图装饰器  
装饰器可为视图添加额外功能（如登录验证、权限控制），常用内置装饰器：  

- **`@login_required`**：限制未登录用户访问（未登录自动跳转登录页）。  
  ```python
  from django.contrib.auth.decorators import login_required

  @login_required(login_url="/login/")  # 未登录跳转至 /login/
  def profile(request):
      return render(request, "myapp/profile.html")
  ```  

- **`@permission_required`**：限制只有特定权限的用户访问。  
  ```python
  from django.contrib.auth.decorators import permission_required

  @permission_required("myapp.change_article")  # 需有修改文章的权限
  def edit_article(request, pk):
      # 编辑文章逻辑
  ```  

- **类视图装饰器**：通过 `@method_decorator` 应用到类视图的方法。  
  ```python
  from django.utils.decorators import method_decorator
  from django.contrib.auth.decorators import login_required

  @method_decorator(login_required, name="dispatch")  # 装饰整个类（所有方法）
  class ProfileView(View):
      def get(self, request):
          return render(request, "myapp/profile.html")
  ```  


### 五、FBV 与 CBV 的对比  
| 类型       | 优点                                  | 缺点                                  | 适用场景                          |  
|------------|---------------------------------------|---------------------------------------|-----------------------------------|  
| 函数视图   | 代码直观、学习成本低                  | 复杂逻辑下代码冗余，复用性差          | 简单页面（如静态页、简单表单）    |  
| 类视图     | 代码复用性强（继承、混入），结构清晰  | 学习成本高，逻辑较抽象                | 复杂功能（如列表分页、CRUD 操作） |  


### 总结  
Django 视图是处理用户请求的核心，通过函数或类的形式实现业务逻辑：  
- 函数视图（FBV）适合简单场景，直接通过 `request.method` 处理不同请求；  
- 类视图（CBV）适合复杂场景，通过继承通用视图（如 `ListView`、`FormView`）减少重复代码；  
- 视图需结合 URL 配置（`urls.py`）对外暴露访问路径，并通过请求对象（`request`）和响应方法（`render`、`redirect` 等）完成交互。  

掌握视图是实现 Django 应用功能的关键，需结合模型、表单、模板灵活使用。
