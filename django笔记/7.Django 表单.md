Django 表单（Form）是处理用户输入数据的核心工具，主要用于**数据验证**、**生成表单 HTML** 和**处理表单提交**。它简化了前端表单的创建和后端数据的校验流程，同时内置了安全机制（如 CSRF 防护），是 Django 中连接用户输入与后端逻辑的重要桥梁。


### 一、表单的核心作用  
1. **数据验证**：自动校验用户输入（如邮箱格式、密码长度等），减少手动校验代码。  
2. **生成 HTML**：自动生成表单控件（输入框、下拉框等）的 HTML 代码，无需手动编写。  
3. **处理提交**：简化表单提交后的数据流处理（获取有效数据、返回错误信息）。  
4. **安全防护**：内置 CSRF 保护、防 XSS 等安全机制，避免常见攻击。  


### 二、表单的定义  
Django 表单通常定义在应用的 `forms.py` 文件中，主要有两种类型：  


#### 1. 基础表单（`Form`）  
适用于不直接关联数据库模型的场景（如搜索框、登录表单），需手动定义字段和验证规则。  

**示例**：登录表单（用户名 + 密码）  
```python
# myapp/forms.py
from django import forms

class LoginForm(forms.Form):
    # 用户名：字符串类型，最大长度 100，必填
    username = forms.CharField(
        label="用户名",  # 表单标签（前端显示）
        max_length=100,
        widget=forms.TextInput(attrs={"class": "form-control"})  # 指定控件类型及 HTML 属性（如 CSS 类）
    )
    # 密码：密码类型（输入时隐藏），最小长度 6
    password = forms.CharField(
        label="密码",
        min_length=6,
        widget=forms.PasswordInput(attrs={"class": "form-control"})  # PasswordInput 渲染为密码框
    )
```  


#### 2. 模型表单（`ModelForm`）  
与数据库模型（Model）直接关联，可**自动生成表单字段**（基于模型字段），并提供 `save()` 方法直接保存数据到数据库，适合用于数据的增删改查（如文章发布、用户注册）。  

**示例**：基于 `Article` 模型的表单（用于发布文章）  
```python
# myapp/forms.py
from django import forms
from .models import Article  # 导入模型

class ArticleForm(forms.ModelForm):
    # 模型表单通过 Meta 类关联模型
    class Meta:
        model = Article  # 关联的模型
        fields = ["title", "content", "publish_time"]  # 需包含的模型字段（或用 "__all__" 表示所有字段）
        exclude = ["read_count"]  # 排除的字段（与 fields 二选一）
        widgets = {  # 自定义控件属性
            "title": forms.TextInput(attrs={"class": "form-control"}),
            "content": forms.Textarea(attrs={"class": "form-control", "rows": 5}),
            "publish_time": forms.DateTimeInput(attrs={"type": "datetime-local"})  # 日期时间选择器
        }
        labels = {  # 自定义字段标签
            "title": "文章标题",
            "content": "文章内容"
        }
```  


### 三、表单字段与验证  
表单字段类似模型字段，但更侧重于**输入验证**。每个字段可通过参数定义验证规则，也支持自定义验证方法。  


#### 1. 常用字段类型  
| 字段类型           | 说明                          | 常用验证参数                          |  
|--------------------|-------------------------------|---------------------------------------|  
| `CharField`        | 文本输入（单行）              | `max_length`（最大长度）、`min_length`（最小长度）、`required`（是否必填，默认 True） |  
| `EmailField`       | 邮箱输入（自动验证格式）      | -                                     |  
| `IntegerField`     | 整数输入                      | `max_value`（最大值）、`min_value`（最小值） |  
| `DateField`/`DateTimeField` | 日期/日期时间输入      | `input_formats`（自定义输入格式，如 `["%Y-%m-%d"]`） |  
| `BooleanField`     | 复选框（是否勾选）            | `required`（默认 False，勾选为 True） |  


#### 2. 自定义验证  
除了字段自带的验证，还可通过以下方式自定义验证逻辑：  

- **字段级验证**：定义 `clean_<字段名>()` 方法，验证单个字段。  
- **表单级验证**：定义 `clean()` 方法，验证多个字段间的关系（如两次密码输入一致）。  


**示例**：带自定义验证的注册表单  
```python
class RegisterForm(forms.Form):
    username = forms.CharField(label="用户名", max_length=50)
    password1 = forms.CharField(label="密码", widget=forms.PasswordInput)
    password2 = forms.CharField(label="确认密码", widget=forms.PasswordInput)

    # 字段级验证：检查用户名是否包含空格
    def clean_username(self):
        username = self.cleaned_data.get("username")  # 获取已验证的字段值
        if " " in username:
            raise forms.ValidationError("用户名不能包含空格")  # 抛出验证错误
        return username  # 必须返回字段值

    # 表单级验证：检查两次密码是否一致
    def clean(self):
        cleaned_data = super().clean()  # 调用父类方法，获取所有已验证字段
        p1 = cleaned_data.get("password1")
        p2 = cleaned_data.get("password2")
        if p1 and p2 and p1 != p2:
            raise forms.ValidationError("两次密码输入不一致")  # 错误信息会显示在表单顶部
        return cleaned_data
```  


### 四、表单的使用（视图 + 模板）  
表单的完整使用流程包括：在视图中处理请求（显示表单/验证提交）→ 在模板中渲染表单 → 反馈结果。  


#### 1. 视图中处理表单  
```python
# myapp/views.py
from django.shortcuts import render, redirect
from .forms import LoginForm, ArticleForm

# 处理基础表单（登录示例）
def login(request):
    if request.method == "POST":
        # 用 POST 数据实例化表单（绑定数据）
        form = LoginForm(request.POST)
        if form.is_valid():  # 验证数据是否有效
            # 获取验证后的干净数据（字典形式）
            username = form.cleaned_data["username"]
            password = form.cleaned_data["password"]
            # 这里添加登录逻辑（如验证用户名密码）
            return redirect("home")  # 验证成功，跳转首页
    else:
        # GET 请求：显示空表单（未绑定数据）
        form = LoginForm()
    
    # 渲染模板（传递表单对象）
    return render(request, "myapp/login.html", {"form": form})

# 处理模型表单（发布文章示例）
def publish_article(request):
    if request.method == "POST":
        form = ArticleForm(request.POST)  # 绑定 POST 数据
        if form.is_valid():
            # 保存数据到数据库（ModelForm 特有的 save() 方法）
            article = form.save()  # 等价于创建模型实例并调用 save()
            return redirect(article.get_absolute_url())  # 跳转文章详情页
    else:
        form = ArticleForm()  # 空表单
    
    return render(request, "myapp/publish.html", {"form": form})
```  


#### 2. 模板中渲染表单  
模板中可通过表单对象渲染字段、错误信息等，常用方式有两种：  


##### （1）快捷渲染（适合快速开发）  
Django 提供内置方法自动生成表单 HTML：  
```html
<!-- myapp/templates/myapp/login.html -->
<form method="post">
    {% csrf_token %}  <!-- 必须添加，防止 CSRF 攻击 -->
    
    <!-- 快捷渲染：p 标签包裹每个字段（含标签和输入框） -->
    {{ form.as_p }}
    
    <button type="submit">登录</button>
</form>
```  

其他快捷方式：  
- `{{ form.as_table }}`：用表格 `<table>` 渲染  
- `{{ form.as_ul }}`：用列表 `<ul>` 渲染  


##### （2）手动渲染（适合自定义样式）  
手动控制每个字段的 HTML 结构，更灵活：  
```html
<form method="post">
    {% csrf_token %}
    
    <!-- 用户名字段 -->
    <div class="form-group">
        <label for="{{ form.username.id_for_label }}">
            {{ form.username.label }}  <!-- 字段标签 -->
        </label>
        {{ form.username }}  <!-- 输入框 -->
        {% if form.username.errors %}  <!-- 字段级错误 -->
            <div class="error">{{ form.username.errors.0 }}</div>
        {% endif %}
    </div>
    
    <!-- 密码字段 -->
    <div class="form-group">
        <label for="{{ form.password.id_for_label }}">
            {{ form.password.label }}
        </label>
        {{ form.password }}
        {% if form.password.errors %}
            <div class="error">{{ form.password.errors.0 }}</div>
        {% endif %}
    </div>
    
    <!-- 表单级错误（如两次密码不一致） -->
    {% if form.non_field_errors %}
        <div class="error">{{ form.non_field_errors.0 }}</div>
    {% endif %}
    
    <button type="submit">登录</button>
</form>
```  


### 五、表单的高级用法  
- **初始数据**：通过 `initial` 参数为表单设置默认值。  
  ```python
  # 视图中：给登录表单设置默认用户名
  form = LoginForm(initial={"username": "guest"})
  ```  

- **动态修改字段**：在视图中根据条件修改表单字段（如隐藏、禁用）。  
  ```python
  form = ArticleForm()
  if not request.user.is_staff:  # 非管理员隐藏发布时间字段
      form.fields["publish_time"].widget = forms.HiddenInput()
  ```  

- **表单集（FormSet）**：处理多个同类型表单（如批量上传多条数据），需导入 `formset_factory`。  
  ```python
  from django.forms import formset_factory

  ArticleFormSet = formset_factory(ArticleForm, extra=3)  # 生成包含 3 个空白表单的表单集
  ```  


### 总结  
Django 表单通过**定义字段**、**自动验证**、**简化渲染**和**集成模型**，极大降低了处理用户输入的复杂度。核心要点：  
- 基础表单（`Form`）适用于非模型场景，需手动定义字段；  
- 模型表单（`ModelForm`）与模型绑定，自动生成字段并支持 `save()`；  
- 自定义验证通过 `clean_<字段名>()` 或 `clean()` 实现；  
- 模板中可通过快捷方式或手动渲染表单，需注意 `{% csrf_token %}` 安全防护。  

掌握表单是实现用户交互功能（如注册、提交、编辑）的关键。
