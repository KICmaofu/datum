Django 中间件（Middleware）是一个轻量级的钩子框架，用于在 **请求/响应周期** 的特定阶段插入自定义逻辑。它介于客户端请求和服务器响应之间，能够全局拦截请求、处理视图、修改响应，甚至捕获异常，是实现“横切关注点”（如身份验证、日志记录、跨域处理）的核心工具。


### 一、中间件的核心作用  
中间件的本质是在 Django 处理请求的各个阶段执行自定义代码，常见用途包括：  
- 身份验证与权限校验（如拦截未登录用户）；  
- 日志记录（如记录请求时间、路径、IP）；  
- 跨域资源共享（CORS）处理；  
- 全局异常捕获（统一返回错误格式）；  
- 请求/响应数据修改（如添加响应头、过滤敏感参数）。  


### 二、中间件的工作流程  
Django 的请求/响应周期会按固定顺序触发中间件的方法，流程如下：  

1. **请求阶段**（客户端 → 视图）：  
   - 请求到达服务器后，依次经过所有中间件的 **`process_request`** 方法（按 `settings.py` 中注册的顺序）。  
   - 然后经过所有中间件的 **`process_view`** 方法。  
   - 执行视图函数/类（`view`）。  

2. **响应阶段**（视图 → 客户端）：  
   - 视图返回响应后，依次经过所有中间件的 **`process_response`** 方法（按注册顺序的 **反向** 执行）。  
   - 若视图抛出异常，会触发中间件的 **`process_exception`** 方法（按注册顺序的反向）。  
   - 若视图返回的是模板响应，会触发 **`process_template_response`** 方法。  


### 三、内置中间件（`settings.py`）  
Django 默认内置了多个中间件，在 `settings.py` 的 `MIDDLEWARE` 列表中配置，例如：  

```python
MIDDLEWARE = [
    'django.middleware.security.SecurityMiddleware',  # 安全相关（如HTTPS重定向）
    'django.contrib.sessions.middleware.SessionMiddleware',  # 会话管理
    'django.middleware.common.CommonMiddleware',  # 通用处理（如URL重定向）
    'django.middleware.csrf.CsrfViewMiddleware',  # CSRF防护
    'django.contrib.auth.middleware.AuthenticationMiddleware',  # 用户认证
    'django.contrib.messages.middleware.MessageMiddleware',  # 消息框架
    'django.middleware.clickjacking.XFrameOptionsMiddleware',  # 防点击劫持
]
```  

这些中间件按顺序执行，且顺序**非常重要**（例如 `AuthenticationMiddleware` 依赖 `SessionMiddleware`，必须在其后）。  


### 四、自定义中间件  
自定义中间件需创建一个包含特定方法的类，并在 `settings.py` 中注册。Django 1.10+ 推荐使用**类式中间件**，核心方法如下：  


#### 1. 中间件核心方法  
| 方法名                  | 触发时机                                  | 返回值                                                                 |  
|-------------------------|-------------------------------------------|------------------------------------------------------------------------|  
| `__init__(self, get_response)` | 中间件初始化（服务器启动时调用一次）      | 无                                                                     |  
| `__call__(self, request)`     | 每个请求都会调用（封装整个请求/响应周期） | 必须返回 `response`（`get_response(request)` 的结果）                  |  
| `process_view(self, request, view_func, view_args, view_kwargs)` | 视图调用前触发 | `None`（继续流程）或 `HttpResponse`（跳过视图，直接返回响应）          |  
| `process_exception(self, request, exception)` | 视图抛出异常时触发 | `None`（继续抛出异常）或 `HttpResponse`（捕获异常，返回自定义响应）    |  
| `process_template_response(self, request, response)` | 视图返回 `TemplateResponse` 时触发 | 必须返回 `response`（可修改响应）                                      |  


#### 2. 自定义中间件步骤  


##### 步骤 1：创建中间件类  
在应用中新建 `middleware.py` 文件，定义中间件类：  

```python
# myapp/middleware.py
import time
from django.http import HttpResponse
from django.utils.deprecation import MiddlewareMixin  # 兼容旧版本（可选）

class SimpleMiddleware:
    """简单中间件示例：记录请求耗时"""
    
    def __init__(self, get_response):
        # 初始化：接收 get_response 函数（处理下一个中间件或视图）
        self.get_response = get_response
        # 服务器启动时执行一次（如初始化资源）
        print("SimpleMiddleware 初始化")

    def __call__(self, request):
        # 1. 请求到达视图前的逻辑（相当于 process_request）
        start_time = time.time()  # 记录开始时间
        
        # 调用下一个中间件或视图（核心：不调用则请求会被拦截）
        response = self.get_response(request)
        
        # 2. 视图返回响应后的逻辑（相当于 process_response）
        end_time = time.time()
       耗时 = end_time - start_time
        print(f"请求 {request.path} 耗时：{耗时:.2f}秒")
        
        return response  # 必须返回响应

    def process_view(self, request, view_func, view_args, view_kwargs):
        """视图调用前执行（可选）"""
        print(f"即将调用视图：{view_func.__name__}")
        # 若返回 HttpResponse，则跳过视图，直接返回该响应
        # 例如：拦截未登录用户访问特定路径
        if request.path.startswith("/admin/") and not request.user.is_authenticated:
            return HttpResponse("请先登录管理员账号")
        return None  # 继续执行视图

    def process_exception(self, request, exception):
        """视图抛出异常时执行（可选）"""
        print(f"视图抛出异常：{str(exception)}")
        # 返回自定义错误响应（避免暴露原始异常）
        return HttpResponse(f"服务器内部错误：{str(exception)}", status=500)
```  


##### 步骤 2：注册中间件  
在 `settings.py` 的 `MIDDLEWARE` 列表中添加自定义中间件的路径（注意顺序）：  

```python
MIDDLEWARE = [
    # ... 其他内置中间件 ...
    'myapp.middleware.SimpleMiddleware',  # 注册自定义中间件
]
```  


#### 3. 常用自定义中间件示例  


##### 示例 1：IP 黑名单拦截  
```python
# myapp/middleware.py
class IPBlacklistMiddleware:
    def __init__(self, get_response):
        self.get_response = get_response
        self.blacklist = ["192.168.1.100", "10.0.0.5"]  # 黑名单IP

    def __call__(self, request):
        # 获取客户端IP（简单方式，生产环境需考虑代理）
        client_ip = request.META.get("REMOTE_ADDR")
        if client_ip in self.blacklist:
            return HttpResponse("你的IP已被禁止访问", status=403)
        return self.get_response(request)
```  


##### 示例 2：全局添加响应头（跨域支持）  
```python
class CORSMiddleware:
    def __init__(self, get_response):
        self.get_response = get_response

    def __call__(self, request):
        response = self.get_response(request)
        # 添加跨域响应头
        response["Access-Control-Allow-Origin"] = "*"  # 允许所有域名（生产环境需限制）
        response["Access-Control-Allow-Methods"] = "GET, POST, PUT, DELETE"
        response["Access-Control-Allow-Headers"] = "Content-Type"
        return response
```  


### 五、中间件的执行顺序  
`MIDDLEWARE` 列表的顺序决定了中间件的执行顺序：  
- **请求阶段**（`__call__` 前半部分、`process_view`）：按列表**正序**执行（从上到下）。  
- **响应阶段**（`process_response`、`process_exception`）：按列表**倒序**执行（从下到上）。  

**示例**：若 `MIDDLEWARE = [A, B, C]`，则：  
- 请求阶段：`A.process_request` → `B.process_request` → `C.process_request` → `A.process_view` → `B.process_view` → `C.process_view` → 视图。  
- 响应阶段：视图 → `C.process_response` → `B.process_response` → `A.process_response`。  


### 六、注意事项  
1. **性能影响**：每个请求都会经过所有中间件，过多的中间件会增加请求耗时，需按需添加。  
2. **依赖关系**：中间件可能存在依赖（如 `AuthenticationMiddleware` 依赖 `SessionMiddleware`），需保证正确顺序。  
3. **避免重复初始化**：`__init__` 方法在服务器启动时调用一次，不要在其中处理请求相关逻辑。  
4. **兼容问题**：Django 1.10+ 废弃了旧的函数式中间件，推荐使用类式中间件（带 `__init__` 和 `__call__`）。  


### 总结  
Django 中间件是处理全局请求/响应逻辑的强大工具，通过拦截请求、处理视图、修改响应等方式，实现身份验证、日志记录、跨域等功能。核心要点：  
- 中间件按 `MIDDLEWARE` 列表顺序执行，请求阶段正序、响应阶段倒序。  
- 自定义中间件需实现 `__init__` 和 `__call__` 方法，可选实现 `process_view` 等钩子。  
- 合理使用中间件可简化全局逻辑，但需注意性能和依赖顺序。  

掌握中间件是构建灵活、健壮 Django 应用的重要技能。
