在 Web 开发中，**Cookie** 和 **Session** 是用于跟踪用户状态的两种核心机制。由于 HTTP 协议是“无状态”的（每次请求都是独立的，服务器无法记住用户身份），Cookie 和 Session 解决了这一问题，使得用户在多次请求之间能够保持登录状态、保存偏好设置等。Django 对两者提供了简洁的封装，简化了状态管理的开发。


### 一、Cookie 与 Session 的基本概念  
| 特性         | Cookie                                  | Session                                  |  
|--------------|-----------------------------------------|------------------------------------------|  
| **存储位置** | 客户端浏览器（明文存储，用户可修改）    | 服务器端（如数据库、缓存，客户端仅存 Session ID） |  
| **安全性**   | 较低（易被篡改、窃取）                  | 较高（敏感数据在服务器，客户端仅存 ID）  |  
| **大小限制** | 通常限制在 4KB 以内                     | 理论上无限制（取决于服务器存储能力）      |  
| **生命周期** | 可设置过期时间（持久化）或随浏览器关闭失效 | 可设置过期时间，默认随浏览器关闭失效      |  
| **适用场景** | 存储非敏感信息（如用户名、主题偏好）    | 存储敏感信息（如登录状态、用户权限）      |  


### 二、Django 中的 Cookie 操作  
Cookie 是通过 HTTP 响应头（`Set-Cookie`）发送到客户端，客户端后续请求会通过请求头（`Cookie`）携带 Cookie 数据。Django 中通过 `HttpResponse` 对象操作 Cookie。  


#### 1. 设置 Cookie  
使用 `response.set_cookie()` 方法在响应中添加 Cookie，常用参数：  
- `key`：Cookie 名称（字符串）。  
- `value`：Cookie 值（字符串，需手动序列化复杂类型）。  
- `max_age`：过期时间（秒，`None` 表示随浏览器关闭失效）。  
- `expires`：过期时间（datetime 对象，与 `max_age` 二选一）。  
- `path`：Cookie 生效的路径（默认 `/`，即全站有效）。  
- `domain`：Cookie 生效的域名。  
- `secure`：是否仅通过 HTTPS 传输（`True` 表示仅 HTTPS）。  
- `httponly`：是否禁止 JavaScript 访问（`True` 更安全，防 XSS 窃取）。  


**示例**：登录成功后设置“记住用户名”Cookie  
```python
# myapp/views.py
from django.http import HttpResponse
from django.shortcuts import render

def login(request):
    if request.method == "POST":
        username = request.POST.get("username")
        password = request.POST.get("password")
        remember = request.POST.get("remember")  # 复选框：是否记住用户名
        
        # 假设验证成功
        response = HttpResponse("登录成功")
        
        if remember:
            # 设置 Cookie，有效期 7 天（7*24*3600 秒）
            response.set_cookie(
                key="username",
                value=username,
                max_age=7*24*3600,
                httponly=True  # 禁止 JS 访问，增强安全
            )
        return response
    
    # GET 请求：从 Cookie 中获取用户名（回显到表单）
    username = request.COOKIES.get("username", "")  # 无则返回空字符串
    return render(request, "login.html", {"username": username})
```  


#### 2. 获取 Cookie  
通过 `request.COOKIES`（类字典对象）获取客户端发送的 Cookie：  
```python
def profile(request):
    # 获取名为 "username" 的 Cookie
    username = request.COOKIES.get("username")
    if username:
        return HttpResponse(f"欢迎回来，{username}")
    else:
        return HttpResponse("请先登录")
```  


#### 3. 删除 Cookie  
使用 `response.delete_cookie(key)` 方法删除 Cookie（本质是设置过期时间为过去）：  
```python
def logout(request):
    response = HttpResponse("已注销")
    # 删除 "username" Cookie
    response.delete_cookie("username")
    return response
```  


### 三、Django 中的 Session 操作  
Session 是服务器端存储用户状态的机制，客户端仅保存一个随机生成的 `sessionid`（通过 Cookie 传递），服务器通过 `sessionid` 找到对应的用户数据。Django 内置 Session 支持，默认使用数据库存储 Session 数据。  


#### 1. 启用 Session（默认已启用）  
Django 项目默认已配置 Session 支持，需确保：  
- `settings.py` 的 `INSTALLED_APPS` 包含 `django.contrib.sessions`。  
- `settings.py` 的 `MIDDLEWARE` 包含 `django.contrib.sessions.middleware.SessionMiddleware`。  

首次使用前需执行迁移（创建 Session 数据表）：  
```bash
python manage.py migrate  # 生成 django_session 表
```  


#### 2. 设置 Session  
通过 `request.session`（类字典对象）设置 Session 数据，支持任何可序列化的 Python 类型（字符串、列表、字典等）：  
```python
def login(request):
    if request.method == "POST":
        username = request.POST.get("username")
        # 验证成功后，将用户信息存入 Session
        request.session["username"] = username  # 字符串
        request.session["is_login"] = True      # 布尔值
        request.session["permissions"] = ["read", "write"]  # 列表
        return HttpResponse("登录成功")
    return render(request, "login.html")
```  


#### 3. 获取 Session  
通过 `request.session.get(key, default)` 获取 Session 数据：  
```python
def dashboard(request):
    # 检查是否登录（从 Session 获取）
    if not request.session.get("is_login", False):
        return HttpResponse("请先登录")
    
    # 获取用户信息
    username = request.session.get("username")
    permissions = request.session.get("permissions", [])
    return HttpResponse(f"{username} 的权限：{permissions}")
```  


#### 4. 删除 Session  
- **删除单个键**：用 `del` 关键字。  
- **清空所有数据**：用 `request.session.flush()`（推荐，同时删除客户端的 `sessionid` Cookie）。  
- **设置过期时间**：修改 `request.session.set_expiry(value)`。  


**示例**：  
```python
def logout(request):
    # 方式1：删除单个键
    # del request.session["username"]
    # del request.session["is_login"]
    
    # 方式2：清空所有 Session 数据（推荐）
    request.session.flush()  # 同时删除客户端的 sessionid Cookie
    
    return HttpResponse("已注销")
```  


#### 5. Session 配置（`settings.py`）  
Django 支持多种 Session 存储方式，可在 `settings.py` 中配置：  

| 配置项                | 说明                                  |  
|-----------------------|---------------------------------------|  
| `SESSION_ENGINE`      | 存储引擎：<br>- `django.contrib.sessions.backends.db`（默认，数据库）<br>- `django.contrib.sessions.backends.cache`（缓存）<br>- `django.contrib.sessions.backends.file`（文件） |  
| `SESSION_COOKIE_AGE`  | Session 过期时间（秒，默认 2 周）      |  
| `SESSION_EXPIRE_AT_BROWSER_CLOSE` | 关闭浏览器后是否过期（`True` 表示是，默认 `False`） |  
| `SESSION_COOKIE_SECURE` | 是否仅通过 HTTPS 传输（`True` 更安全） |  
| `SESSION_COOKIE_HTTPONLY` | 是否禁止 JavaScript 访问（默认 `True`） |  


**示例**：设置 Session 2 小时过期，关闭浏览器失效  
```python
# settings.py
SESSION_COOKIE_AGE = 2 * 3600  # 2小时（秒）
SESSION_EXPIRE_AT_BROWSER_CLOSE = True  # 关闭浏览器后过期
```  


### 四、Cookie 与 Session 的协同工作  
Django 中 Session 依赖 Cookie 传递 `sessionid`（默认），流程如下：  
1. 用户首次请求时，服务器生成唯一 `sessionid`，并创建对应的 Session 数据。  
2. 服务器通过 `Set-Cookie` 头将 `sessionid=xxx` 发送到客户端，客户端存储该 Cookie。  
3. 后续请求中，客户端通过 `Cookie` 头携带 `sessionid`，服务器据此找到对应的 Session 数据，识别用户状态。  


### 五、最佳实践  
1. **敏感数据用 Session**：如用户 ID、权限、登录状态等，避免存在 Cookie 中（易被篡改）。  
2. **非敏感数据用 Cookie**：如用户名回显、主题偏好、广告跟踪等，减轻服务器存储压力。  
3. **安全配置**：  
   - Cookie 设 `httponly=True`（防 XSS 窃取）、`secure=True`（生产环境用 HTTPS）。  
   - Session 设 `SESSION_COOKIE_SECURE=True` 和 `SESSION_COOKIE_HTTPONLY=True`。  
4. **及时清理**：用户注销时，用 `session.flush()` 清空 Session 并删除 `sessionid` Cookie。  


### 总结  
- **Cookie** 是客户端存储，适合非敏感、小数据，需手动处理过期和安全。  
- **Session** 是服务器存储，适合敏感数据，安全性高，Django 自动管理 `sessionid` 和存储。  
- 两者通常协同工作：Session 通过 Cookie 传递 `sessionid`，实现用户状态跟踪。  

掌握 Cookie 和 Session 的操作是实现用户登录、权限控制、个性化服务等功能的基础。
