Django 路由（URLconf）是用于将用户请求的 URL 地址映射到对应视图（View）的配置系统。它的核心作用是**解析 URL 路径，找到对应的视图函数/类，从而处理请求并返回响应**。合理的路由设计能让 URL 结构清晰、可维护，同时支持动态参数传递。


### 一、路由的基本配置  
Django 路由配置主要通过 `urls.py` 文件实现，分为**项目级路由**和**应用级路由**，通过“路由分发”机制实现模块化管理。  


#### 1. 项目级路由（主路由）  
项目创建后，默认在配置目录（如 `myproject/myproject/`）下生成 `urls.py`，用于定义项目的主路由规则。它通常负责将不同前缀的 URL 分发给对应的应用（App）处理。  

**示例**：项目级 `urls.py`  
```python
# myproject/myproject/urls.py
from django.contrib import admin
from django.urls import path, include  # include 用于路由分发

urlpatterns = [
    path('admin/', admin.site.urls),  # Django 自带 Admin 后台路由
    path('blog/', include('blog.urls')),  # 将以 /blog/ 开头的 URL 分发给 blog 应用的 urls.py
    path('users/', include('users.urls')),  # 将 /users/ 开头的 URL 分发给 users 应用
]
```  


#### 2. 应用级路由（子路由）  
每个应用可创建自己的 `urls.py`，管理该应用内的 URL 规则，使路由配置更清晰（遵循“单一职责”原则）。  

**示例**：`blog` 应用的 `urls.py`  
```python
# myproject/blog/urls.py
from django.urls import path
from . import views  # 导入当前应用的视图

urlpatterns = [
    path('', views.blog_index, name='blog_index'),  # 匹配 /blog/（空路径，即应用根路径）
    path('articles/', views.article_list, name='article_list'),  # 匹配 /blog/articles/
    path('articles/<int:pk>/', views.article_detail, name='article_detail'),  # 匹配 /blog/articles/1/（带参数）
]
```  


### 二、`path()` 函数详解  
路由规则通过 `path(route, view, kwargs=None, name=None)` 函数定义，参数说明：  

| 参数      | 作用                                                                 |  
|-----------|----------------------------------------------------------------------|  
| `route`   | URL 模式字符串，用于匹配请求路径（不包含域名和查询参数）。           |  
| `view`    | 匹配成功后调用的视图（函数视图或类视图的 `as_view()` 方法）。         |  
| `kwargs`  | 传递给视图的额外关键字参数（字典形式）。                             |  
| `name`    | 路由的别名，用于模板或视图中“反向解析”生成 URL（避免硬编码）。       |  


#### 1. 静态 URL 匹配  
`route` 为固定字符串，直接匹配 URL 路径。  

```python
# 匹配 /about/
path('about/', views.about, name='about'),

# 匹配 /contact/
path('contact/', views.contact, name='contact'),
```  


#### 2. 动态 URL 匹配（路径转换器）  
`route` 中可包含**路径转换器**，用于捕获 URL 中的动态参数（如 ID、用户名），并将参数传递给视图。  

Django 内置的路径转换器：  

| 转换器       | 作用                                  | 示例 URL                | 捕获的值       |  
|--------------|---------------------------------------|-------------------------|----------------|  
| `str`        | 匹配非空字符串（不含 `/`），默认类型  | `/user/alice/`          | `alice`        |  
| `int`        | 匹配正整数                            | `/article/123/`         | `123`          |  
| `slug`       | 匹配字母、数字、下划线、连字符组成的字符串（适合 URL 友好的名称） | `/post/my-first-post/`  | `my-first-post` |  
| `uuid`       | 匹配 UUID 格式字符串（含连字符，如 `a1b2c3...`） | `/file/1b9d6bcd-bbfd-4b2d-9b5d-ab8dfbbd4bed/` | 对应的 UUID 值 |  
| `path`       | 匹配包含 `/` 的非空字符串（如完整路径） | `/docs/path/to/file/`   | `path/to/file` |  


**示例**：带参数的路由  
```python
# blog/urls.py
urlpatterns = [
    # 1. 捕获文章 ID（int 类型），传递给视图的 pk 参数
    path('articles/<int:pk>/', views.article_detail, name='article_detail'),
    
    # 2. 捕获用户名（str 类型），传递给视图的 username 参数
    path('user/<str:username>/', views.user_profile, name='user_profile'),
    
    # 3. 捕获文章别名（slug 类型）
    path('post/<slug:slug>/', views.post_detail, name='post_detail'),
]
```  

对应的视图需接收参数：  
```python
# blog/views.py
def article_detail(request, pk):  # pk 对应 URL 中的 <int:pk>
    article = Article.objects.get(pk=pk)
    return render(request, 'blog/detail.html', {'article': article})

def user_profile(request, username):  # username 对应 <str:username>
    user = User.objects.get(username=username)
    return render(request, 'blog/user.html', {'user': user})
```  


#### 3. 自定义路径转换器（高级）  
若内置转换器不满足需求，可自定义转换器（如匹配手机号、日期），步骤：  

1. 创建转换器类（实现 `regex`、`to_python`、`to_url` 方法）；  
2. 在 `urls.py` 中注册转换器；  
3. 使用自定义转换器。  

**示例**：匹配手机号的转换器  
```python
# blog/converters.py
class PhoneConverter:
    regex = r'1[3-9]\d{9}'  # 手机号正则（1开头，第二位3-9，后9位数字）

    def to_python(self, value):
        # 将匹配的字符串转换为 Python 对象（此处直接返回字符串）
        return value

    def to_url(self, value):
        # 将 Python 对象转换为 URL 字符串（用于反向解析）
        return value
```  

在 `blog/urls.py` 中注册并使用：  
```python
from django.urls import path, register_converter
from .converters import PhoneConverter
from . import views

# 注册转换器，指定别名 phone
register_converter(PhoneConverter, 'phone')

urlpatterns = [
    # 匹配 /user/phone/13800138000/
    path('user/phone/<phone:phone_num>/', views.phone_user, name='phone_user'),
]
```  


### 三、正则表达式路由（`re_path()`）  
对于复杂的 URL 匹配（如多格式参数、模糊匹配），可使用 `re_path(route, view, kwargs=None, name=None)`，其 `route` 参数支持正则表达式。  

**示例**：  
```python
from django.urls import re_path
from . import views

urlpatterns = [
    # 匹配 /articles/2023/10/（年份4位，月份2位）
    re_path(r'^articles/(?P<year>\d{4})/(?P<month>\d{2})/$', views.article_archive, name='article_archive'),
]
```  
- `(?P<year>\d{4})`：用正则分组捕获参数，`year` 是参数名，`\d{4}` 匹配4位数字；  
- 视图需接收 `year` 和 `month` 参数：  
  ```python
  def article_archive(request, year, month):
      # 处理逻辑...
  ```  


### 四、路由分发（`include()`）  
`include(module, namespace=None)` 用于将子路由模块（如应用的 `urls.py`）集成到主路由，实现路由的模块化管理。  

#### 1. 基本用法（分发到应用路由）  
```python
# 主 urls.py
from django.urls import path, include

urlpatterns = [
    path('blog/', include('blog.urls')),  # 分发 /blog/ 到 blog/urls.py
]
```  

`blog/urls.py` 中的路由会自动拼接前缀 `/blog/`，例如 `path('articles/', ...)` 实际匹配 `/blog/articles/`。  


#### 2. 分发到路由列表（无应用）  
也可直接分发到一个路由列表（适合简单场景，无需单独创建 `urls.py`）：  
```python
# 主 urls.py
from django.urls import path, include
from blog import views

# 定义子路由列表
blog_urls = [
    path('', views.blog_index),
    path('articles/', views.article_list),
]

urlpatterns = [
    path('blog/', include(blog_urls)),  # 分发到路由列表
]
```  


### 五、路由命名与反向解析  
为路由设置 `name` 参数后，可通过“反向解析”动态生成 URL（无需硬编码），当 URL 模式改变时，只需修改 `urls.py`，无需修改模板或视图中的引用。  


#### 1. 模板中反向解析（`{% url %}` 标签）  
```html
<!-- 生成 /blog/articles/1/（对应 name='article_detail'，参数 pk=1） -->
<a href="{% url 'article_detail' pk=1 %}">文章详情</a>

<!-- 生成 /user/alice/（对应 name='user_profile'，参数 username='alice'） -->
<a href="{% url 'user_profile' username='alice' %}">Alice 的主页</a>
```  


#### 2. 视图中反向解析（`reverse()` 函数）  
```python
from django.urls import reverse
from django.shortcuts import redirect

def some_view(request):
    # 生成 /blog/articles/1/ 的 URL 字符串
    detail_url = reverse('article_detail', args=[1])  # args 传递位置参数
    # 或用 kwargs 传递关键字参数
    detail_url = reverse('article_detail', kwargs={'pk': 1})
    
    # 重定向到该 URL
    return redirect(detail_url)
```  


### 六、路由命名空间（Namespace）  
当多个应用有同名路由（`name` 相同）时，会导致反向解析冲突。通过“命名空间”可区分不同应用的路由。  


#### 1. 应用级命名空间  
在应用的 `urls.py` 中定义 `app_name` 变量，指定应用的命名空间：  
```python
# blog/urls.py
app_name = 'blog'  # 应用级命名空间

urlpatterns = [
    path('articles/<int:pk>/', views.article_detail, name='article_detail'),
]
```  


#### 2. 实例级命名空间（多实例应用）  
若同一应用被多次包含（如多版本 API），可通过 `namespace` 参数指定实例命名空间：  
```python
# 主 urls.py
urlpatterns = [
    path('blog/v1/', include(('blog.urls', 'blog'), namespace='blog_v1')),  # 实例命名空间 blog_v1
    path('blog/v2/', include(('blog.urls', 'blog'), namespace='blog_v2')),  # 实例命名空间 blog_v2
]
```  


#### 3. 带命名空间的反向解析  
模板中需指定命名空间：  
```html
<!-- 格式：{% url '命名空间:路由名' 参数 %} -->
<a href="{% url 'blog:article_detail' pk=1 %}">博客文章详情</a>
```  

视图中：  
```python
reverse('blog:article_detail', kwargs={'pk': 1})  # 应用级命名空间
reverse('blog_v1:article_detail', kwargs={'pk': 1})  # 实例级命名空间
```  


### 七、路由的匹配顺序  
Django 按 `urlpatterns` 列表中路由的**定义顺序**匹配 URL，一旦找到第一个匹配的路由，就会调用对应的视图，不再继续匹配。因此，**更具体的路由应放在前面，通用路由放在后面**。  

**错误示例**（通用路由在前，导致具体路由无法匹配）：  
```python
urlpatterns = [
    path('articles/<int:pk>/', views.article_detail),  # 具体路由
    path('articles/latest/', views.latest_article),     # 会被上面的路由拦截（因为 <int:pk> 会匹配 'latest' 吗？不，int 不匹配字符串，这里实际没问题，但如果是 str 转换器则有问题）
]

# 若使用 str 转换器，下面的写法会有问题：
urlpatterns = [
    path('articles/<str:slug>/', views.article_by_slug),  # 通用路由在前
    path('articles/latest/', views.latest_article),        # 永远不会被匹配（因为 'latest' 会被 <str:slug> 捕获）
]
```  

**正确示例**（具体路由在前）：  
```python
urlpatterns = [
    path('articles/latest/', views.latest_article),        # 具体路由在前
    path('articles/<str:slug>/', views.article_by_slug),  # 通用路由在后
]
```  


### 总结  
Django 路由是 URL 与视图的“桥梁”，核心知识点：  
- 路由配置采用“项目级 + 应用级”的分发机制，通过 `include()` 实现模块化；  
- `path()` 用于简单匹配，支持路径转换器捕获动态参数；`re_path()` 支持正则表达式，适合复杂匹配；  
- 路由命名（`name`）和反向解析（`{% url %}`/`reverse()`）避免 URL 硬编码，提高可维护性；  
- 命名空间解决多应用路由同名冲突问题。  

合理设计路由结构是构建清晰、可扩展 Django 应用的基础。
