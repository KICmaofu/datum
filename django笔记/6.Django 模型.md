Django 模型（Model）是 Django ORM（对象关系映射）系统的核心，用于定义数据库表结构，并通过 Python 类的方式操作数据库（无需编写 SQL 语句）。模型是“数据的单一、明确的信息来源”，既包含数据的字段定义，也包含操作数据的方法。


### 一、模型的核心作用  
1. **定义数据结构**：通过 Python 类映射数据库表，类的属性对应表的字段。  
2. **封装数据库操作**：通过模型提供的 API（如 `save()`、`objects.all()`）实现数据的增删改查，无需直接编写 SQL。  
3. **关联数据库**：支持多种数据库（SQLite、MySQL、PostgreSQL 等），切换数据库只需修改配置，无需修改模型代码。  


### 二、模型的定义  
模型定义在应用的 `models.py` 文件中，每个模型类继承自 `django.db.models.Model`，类的属性对应数据库表的字段。  


#### 1. 基本示例  
以“博客文章”模型为例，定义包含标题、内容、发布时间等字段的模型：  
```python
# myapp/models.py
from django.db import models
from django.utils import timezone

class Article(models.Model):
    # 标题：字符串类型，最大长度 200，不允许为空
    title = models.CharField(max_length=200)
    # 内容：长文本类型，允许为空（blank=True 表示表单提交时可空）
    content = models.TextField(blank=True)
    # 发布时间：日期时间类型，默认值为当前时间（timezone.now 是函数，不加括号）
    publish_time = models.DateTimeField(default=timezone.now)
    # 阅读量：整数类型，默认值 0，允许数据库中为 null（null=True）
    read_count = models.IntegerField(default=0, null=True)

    # 定义模型实例的字符串表示（在 Admin 后台或打印时显示）
    def __str__(self):
        return self.title  # 显示文章标题

    # 自定义方法：获取文章的绝对 URL（用于模板中生成详情页链接）
    def get_absolute_url(self):
        from django.urls import reverse
        return reverse('article_detail', args=[str(self.id)])  # 假设路由别名为 article_detail
```  


#### 2. 常用字段类型  
Django 提供了多种字段类型，对应数据库中的不同数据类型，常用如下：  

| 字段类型               | 说明                                  | 常用参数                          |  
|------------------------|---------------------------------------|-----------------------------------|  
| `CharField`            | 短字符串（如标题、用户名）            | `max_length`（必传，最大长度）    |  
| `TextField`            | 长文本（如文章内容、评论）            | `blank=True`（允许表单为空）      |  
| `IntegerField`         | 整数（如阅读量、数量）                | `default`（默认值）、`null=True`  |  
| `FloatField`/`DecimalField` | 浮点数/高精度小数（如价格）       | `max_digits`（总位数）、`decimal_places`（小数位） |  
| `DateTimeField`        | 日期时间（如发布时间、注册时间）      | `auto_now_add`（创建时自动设为当前时间）、`auto_now`（更新时自动刷新） |  
| `DateField`            | 日期（如生日）                        | 同 `DateTimeField`（无时间部分）  |  
| `BooleanField`         | 布尔值（如是否置顶、是否激活）        | `default=False`                   |  
| `FileField`/`ImageField` | 文件/图片上传（如头像、附件）      | `upload_to`（上传路径，如 `'avatars/%Y/%m/'`） |  
| `ForeignKey`           | 外键（一对多关系，如“文章-分类”）    | `to`（关联的模型）、`on_delete`（级联删除策略） |  
| `ManyToManyField`      | 多对多关系（如“文章-标签”）          | `to`（关联的模型）                |  


#### 3. 字段参数（通用）  
- `null=True`：允许数据库字段为 `NULL`（默认 `False`）。  
- `blank=True`：允许表单提交时为空（与 `null` 不同，`blank` 是表单验证层面，`null` 是数据库层面）。  
- `default`：默认值（可传值或函数，如 `default=0`、`default=timezone.now`）。  
- `unique=True`：字段值在表中唯一（如用户名）。  
- `db_index=True`：为字段创建数据库索引（加速查询）。  
- `verbose_name`：字段在 Admin 后台的显示名称（如 `verbose_name="文章标题"`）。  


### 三、模型元数据（Meta 类）  
通过在模型中定义 `Meta` 子类，可配置模型的元信息（如表名、排序方式等）：  
```python
class Article(models.Model):
    # ... 字段定义 ...

    class Meta:
        # 数据库表名（默认是“应用名_模型类名小写”，如 myapp_article）
        db_table = "blog_article"
        # 默认排序（按发布时间降序，- 表示降序）
        ordering = ['-publish_time']
        # Admin 后台显示的模型名称（单数）
        verbose_name = "文章"
        # Admin 后台显示的模型名称（复数）
        verbose_name_plural = "文章"
```  


### 四、数据库迁移（Migrations）  
模型定义或修改后，需通过迁移命令将变更同步到数据库（生成/修改表结构）。  

#### 1. 生成迁移文件  
根据模型的变更（新增字段、修改字段等），生成记录变更的迁移文件（存于应用的 `migrations/` 目录）：  
```bash
python manage.py makemigrations  # 为所有应用生成迁移文件
python manage.py makemigrations myapp  # 仅为 myapp 应用生成
```  


#### 2. 执行迁移（同步到数据库）  
将迁移文件中的变更应用到数据库，实际创建/修改表结构：  
```bash
python manage.py migrate  # 执行所有未应用的迁移
python manage.py migrate myapp 0001  # 仅执行 myapp 应用的 0001 号迁移
```  


#### 3. 查看迁移状态  
查看所有迁移的执行情况（哪些已应用，哪些未应用）：  
```bash
python manage.py showmigrations
```  


### 五、模型查询（ORM API）  
Django 模型通过 `objects` 管理器（默认提供）提供丰富的查询方法，实现数据查询（无需写 SQL）。  


#### 1. 基本查询  
```python
from myapp.models import Article

# 1. 获取所有记录（返回 QuerySet，类似列表）
all_articles = Article.objects.all()

# 2. 过滤记录（条件查询）
# 发布时间在 2023 年之后的文章
filtered = Article.objects.filter(publish_time__year__gt=2023)
# 标题包含“Django”的文章（不区分大小写）
filtered = Article.objects.filter(title__icontains="Django")

# 3. 获取单条记录（若不存在会抛 DoesNotExist 异常）
article = Article.objects.get(id=1)  # 按 ID 查询

# 4. 排序（按阅读量升序，-read_count 表示降序）
sorted_articles = Article.objects.order_by('read_count')

# 5. 限制查询结果（取前 5 条）
limited = Article.objects.all()[:5]
```  


#### 2. 复杂查询（Q 对象和 F 对象）  
- **`Q` 对象**：用于组合条件（`AND`/`OR`）。  
  ```python
  from django.db.models import Q

  # 标题含“Django” 或 阅读量 > 100 的文章（OR 关系）
  articles = Article.objects.filter(Q(title__icontains="Django") | Q(read_count__gt=100))
  ```  

- **`F` 对象**：用于字段间的比较（如“阅读量 > 评论量”）。  
  ```python
  from django.db.models import F

  # 阅读量大于评论量的文章（假设模型有 comment_count 字段）
  articles = Article.objects.filter(read_count__gt=F('comment_count'))
  ```  


#### 3. 新增与修改数据  
```python
# 新增数据
new_article = Article(
    title="Django 模型教程",
    content="这是一篇关于 Django 模型的文章...",
    read_count=0
)
new_article.save()  # 保存到数据库

# 修改数据（先查询，再修改属性，最后保存）
article = Article.objects.get(id=1)
article.read_count += 1  # 阅读量 +1
article.save()  # 同步到数据库
```  


#### 4. 删除数据  
```python
# 删除单条记录
article = Article.objects.get(id=1)
article.delete()

# 批量删除（删除所有阅读量为 0 的文章）
Article.objects.filter(read_count=0).delete()
```  


### 六、模型关系  
实际应用中，模型间通常存在关联关系（如“文章属于某个分类”“用户有多篇文章”），Django 提供三种核心关系类型：  


#### 1. 一对多（ForeignKey）  
一个模型的实例对应另一个模型的多个实例（如“分类-文章”：一个分类下有多篇文章）。  
```python
class Category(models.Model):
    name = models.CharField(max_length=100)  # 分类名称

    def __str__(self):
        return self.name

class Article(models.Model):
    # 外键关联 Category，on_delete=models.CASCADE 表示：若分类删除，关联的文章也删除
    category = models.ForeignKey(Category, on_delete=models.CASCADE, related_name="articles")
    # ... 其他字段 ...

# 使用：获取分类下的所有文章
category = Category.objects.get(id=1)
articles_in_category = category.articles.all()  # related_name 定义了反向查询的属性名
```  


#### 2. 多对多（ManyToManyField）  
两个模型的实例可相互关联多个（如“文章-标签”：一篇文章可多个标签，一个标签可多个文章）。  
```python
class Tag(models.Model):
    name = models.CharField(max_length=50)  # 标签名称

class Article(models.Model):
    # 多对多关联 Tag
    tags = models.ManyToManyField(Tag, related_name="articles")
    # ... 其他字段 ...

# 使用：给文章添加标签
article = Article.objects.get(id=1)
tag1 = Tag.objects.get(id=1)
tag2 = Tag.objects.get(id=2)
article.tags.add(tag1, tag2)  # 添加标签

# 获取文章的所有标签
article_tags = article.tags.all()
```  


#### 3. 一对一（OneToOneField）  
两个模型的实例一一对应（如“用户-个人资料”：一个用户只有一个资料）。  
```python
from django.contrib.auth.models import User  # Django 内置用户模型

class Profile(models.Model):
    # 一对一关联 User，on_delete=models.CASCADE 表示：用户删除，资料也删除
    user = models.OneToOneField(User, on_delete=models.CASCADE, related_name="profile")
    bio = models.TextField(blank=True)  # 个人简介
    avatar = models.ImageField(upload_to='avatars/', null=True)  # 头像

# 使用：获取用户的资料
user = User.objects.get(username="Alice")
profile = user.profile  # 通过 related_name 反向查询
```  


### 总结  
Django 模型是连接 Python 代码与数据库的桥梁，通过 ORM 机制屏蔽了底层 SQL 细节，让开发者能以面向对象的方式操作数据。核心知识点包括：  
- 模型定义（字段类型、参数、元数据）；  
- 数据库迁移（`makemigrations`、`migrate`）；  
- 查询 API（基础查询、复杂条件、关系查询）；  
- 模型关系（一对多、多对多、一对一）。  

掌握模型是开发 Django 应用的基础，后续的视图、模板都依赖模型提供的数据支持。
