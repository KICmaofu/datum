`django-admin` 是 Django 提供的命令行工具，用于执行各种项目管理任务（如创建项目、生成配置、数据库操作等）。它是 Django 开发的核心工具之一，与项目内的 `manage.py` 功能类似（`manage.py` 本质是对 `django-admin` 的封装，自动绑定当前项目配置）。


### **基本用法**  
语法：  
```bash
django-admin <命令> [选项]
```  

- 若未安装 Django 到全局环境，可能需要通过 `python -m django` 调用（等效于 `django-admin`）：  
  ```bash
  python -m django <命令> [选项]
  ```  


### **与 manage.py 的区别**  
- `django-admin`：全局命令，适用于**未进入特定项目**时的操作（如创建新项目）。  
- `manage.py`：每个 Django 项目根目录下的脚本，自动关联当前项目的配置（`settings.py`），适用于**项目内操作**（如启动服务器、数据库迁移）。  

简单说：创建项目用 `django-admin`，项目内开发用 `manage.py`。


### **常用命令详解**  

#### 1. `startproject`：创建新 Django 项目  
作用：生成一个新的 Django 项目目录及核心配置文件。  
语法：  
```bash
django-admin startproject <项目名> [可选目录]
```  
示例：  
```bash
# 在当前目录创建名为 myproject 的项目
django-admin startproject myproject

# 在指定目录（如 ./projects/）创建项目
django-admin startproject myproject ./projects/
```  


#### 2. `startapp`：创建新应用（App）  
作用：生成一个新的应用目录（包含模型、视图等基础文件）。  
语法：  
```bash
django-admin startapp <应用名> [可选目录]
```  
示例：  
```bash
# 在当前目录创建名为 blog 的应用
django-admin startapp blog

# 注意：创建应用后需在项目的 settings.py 中注册（INSTALLED_APPS 添加应用名）
```  


#### 3. `makemigrations`：生成数据库迁移文件  
作用：根据模型（`models.py`）的变更，生成记录数据结构修改的迁移文件（存于 `migrations/` 目录）。  
语法：  
```bash
# 通常在项目内用 manage.py 执行（依赖项目配置）
python manage.py makemigrations [应用名]  # 可选指定应用，默认所有应用
```  
示例：  
```bash
# 为所有应用生成迁移文件
python manage.py makemigrations

# 仅为 blog 应用生成迁移文件
python manage.py makemigrations blog
```  


#### 4. `migrate`：执行数据库迁移  
作用：将 `makemigrations` 生成的迁移文件应用到数据库，实际修改数据库表结构。  
语法：  
```bash
python manage.py migrate [应用名] [迁移文件名]  # 可选指定应用或迁移文件
```  
示例：  
```bash
# 应用所有未执行的迁移
python manage.py migrate

# 仅应用 blog 应用的 0001_initial 迁移
python manage.py migrate blog 0001_initial
```  


#### 5. `runserver`：启动开发服务器  
作用：启动 Django 内置的开发用 Web 服务器（默认地址 `127.0.0.1:8000`）。  
语法：  
```bash
python manage.py runserver [IP:端口]
```  
示例：  
```bash
# 默认端口 8000
python manage.py runserver

# 自定义端口 8080
python manage.py runserver 8080

# 允许局域网其他设备访问（需替换为自己的 IP）
python manage.py runserver 192.168.1.100:8000
```  


#### 6. `createsuperuser`：创建管理员账号  
作用：创建可登录 Django Admin 后台的超级用户（管理员）。  
语法：  
```bash
python manage.py createsuperuser
```  
执行后会提示输入用户名、邮箱、密码（密码输入时不显示，按回车确认）。  


#### 7. `admin`：启动 Admin 后台相关操作（较少直接使用）  
作用：主要用于生成 Admin 后台的配置代码（通常通过 `admin.py` 手动配置，无需直接调用此命令）。  


#### 8. `collectstatic`：收集静态文件  
作用：将项目中所有应用的静态文件（`static/` 目录）收集到 `settings.py` 中 `STATIC_ROOT` 指定的目录（用于生产环境部署）。  
语法：  
```bash
python manage.py collectstatic
```  
执行时会提示确认覆盖，输入 `yes` 即可。  


#### 9. `dumpdata` / `loaddata`：导出/导入数据  
- `dumpdata`：将数据库数据导出为 JSON/XML 等格式的文件。  
  语法：  
  ```bash
  python manage.py dumpdata [应用名.模型名] > 文件名.json
  ```  
  示例：导出 `blog` 应用的所有数据到 `blog_data.json`：  
  ```bash
  python manage.py dumpdata blog > blog_data.json
  ```  

- `loaddata`：将导出的数据文件导入到数据库。  
  语法：  
  ```bash
  python manage.py loaddata 文件名.json
  ```  


#### 10. `test`：运行测试用例  
作用：执行项目中 `tests.py` 或 `tests/` 目录下的测试代码。  
语法：  
```bash
python manage.py test [应用名]  # 可选指定应用，默认所有应用
```  


### **全局选项（适用于所有命令）**  
- `--version`：显示 Django 版本。  
  ```bash
  django-admin --version
  ```  

- `--help`：显示命令帮助信息（可查看所有命令或单个命令的用法）。  
  ```bash
  django-admin --help  # 所有命令帮助
  django-admin startproject --help  # 单个命令帮助
  ```  

- `--settings=<模块路径>`：指定项目配置文件（默认使用 `settings.py`）。  
  ```bash
  django-admin runserver --settings=myproject.settings_prod
  ```  

- `--pythonpath=<路径>`：添加 Python 模块搜索路径（用于指定项目所在目录）。  
  ```bash
  django-admin runserver --pythonpath=/path/to/project
  ```  

- `--verbosity=<级别>`：设置输出详细程度（0=最少，1=默认，2=详细，3=调试）。  
  ```bash
  python manage.py migrate --verbosity=2  # 显示详细迁移过程
  ```  


### **总结**  
`django-admin` 是 Django 开发的“瑞士军刀”，涵盖了从项目创建到部署的全流程操作。熟练掌握常用命令（如 `startproject`、`makemigrations`、`runserver` 等）能极大提升开发效率。实际开发中，项目内操作更常用 `manage.py`，而项目初始化则依赖 `django-admin`。
