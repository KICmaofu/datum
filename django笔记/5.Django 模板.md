Django 模板（Template）是用于生成动态 HTML 页面的文本文件，它将静态 HTML 代码与动态数据（由视图传递）结合，实现前端页面的动态渲染。模板系统的核心目标是**分离业务逻辑（视图）和页面展示（模板）**，同时通过语法简化动态内容的处理。


### 一、模板的基本概念  
- **作用**：接收视图传递的数据，生成最终的 HTML 响应。  
- **文件位置**：通常放在应用目录下的 `templates/应用名/` 文件夹（如 `myapp/templates/myapp/index.html`），或项目根目录的 `templates/` 文件夹（需在 `settings.py` 中配置）。  
- **语法**：混合静态 HTML 和 Django 模板语法（变量、标签、过滤器）。  


### 二、模板语法核心要素  

#### 1. 变量（Variables）  
用于显示视图传递的数据，格式：`{{ 变量名 }}`。  

- **数据类型**：支持字符串、数字、列表、字典、对象等。  
- **访问属性/方法**：  
  - 字典：`{{ user.name }}`（等价于 `user['name']`）  
  - 对象：`{{ article.title }}`（调用对象的 `title` 属性）  
  - 列表：`{{ list.0 }}`（访问列表第一个元素）  
  - 方法：`{{ article.get_summary }}`（调用对象的方法，无需括号）  

**示例**：  
视图传递数据：  
```python
# views.py
from django.shortcuts import render

def index(request):
    context = {
        "name": "Django",
        "user": {"username": "Alice", "age": 25},
        "articles": ["第一篇", "第二篇"]
    }
    return render(request, "myapp/index.html", context)  # 第三个参数是传递给模板的字典
```  

模板中使用：  
```html
<!-- myapp/templates/myapp/index.html -->
<h1>Hello, {{ name }}!</h1>
<p>用户名：{{ user.username }}, 年龄：{{ user.age }}</p>
<p>第一篇文章：{{ articles.0 }}</p>
```  


#### 2. 标签（Tags）  
用于实现逻辑控制（如条件判断、循环）、模板复用等，格式：`{% 标签 %}`（需闭合，如 `{% endif %}`）。  

常用标签：  

| 标签 | 作用 | 示例 |  
|------|------|------|  
| `{% if 条件 %}`<br>`{% elif 条件 %}`<br>`{% else %}`<br>`{% endif %}` | 条件判断 | ```html{% if user.age > 18 %}<p>成年</p>{% else %}<p>未成年</p>{% endif %}``` |  
| `{% for 变量 in 列表 %}`<br>`{% empty %}`<br>`{% endfor %}` | 循环遍历 | ```html<ul>{% for art in articles %}<li>{{ art }}</li>{% empty %}<li>无数据</li>{% endfor %}</ul>``` |  
| `{% url '路由别名' 参数 %}` | 生成 URL（避免硬编码） | ```html<a href="{% url 'article_detail' 1 %}">文章1</a>```（对应 `urls.py` 中 `path('detail/<int:id>/', ..., name='article_detail')`） |  
| `{% csrf_token %}` | 生成 CSRF 令牌（用于表单提交，防止跨站请求伪造） | ```html<form method="post">{% csrf_token %}<input type="submit"></form>``` |  
| `{% extends '父模板' %}` | 继承父模板（模板复用核心） | 见下文“模板继承” |  
| `{% block 块名 %}`<br>`{% endblock %}` | 定义可被重写的块（配合继承） | 见下文“模板继承” |  
| `{% include '模板片段' %}` | 包含其他模板（复用小片段） | ```html{% include 'myapp/sidebar.html' %}``` |  


#### 3. 过滤器（Filters）  
用于修改变量的显示格式，格式：`{{ 变量|过滤器:参数 }}`（可链式调用：`{{ 变量|过滤器1|过滤器2 }}`）。  

常用过滤器：  

| 过滤器 | 作用 | 示例 |  
|--------|------|------|  
| `lower`/`upper` | 转小写/大写 | `{{ name|upper }}` → "DJANGO" |  
| `truncatechars:N` | 截断为 N 个字符（含省略号） | `{{ article.content|truncatechars:20 }}` |  
| `date:"格式"` | 格式化日期（默认 `DATE_FORMAT` 配置） | `{{ article.create_time|date:"Y-m-d H:i" }}` → "2023-10-01 15:30" |  
| `default:"默认值"` | 变量为 `None`/空时显示默认值 | `{{ user.nickname|default:"匿名用户" }}` |  
| `length` | 获取长度 | `{{ articles|length }}` → 2（列表有2个元素） |  


### 三、模板继承（Template Inheritance）  
模板继承是 Django 模板最强大的功能之一，用于复用页面中不变的部分（如导航栏、页脚），仅修改动态内容。  

#### 步骤：  
1. **定义父模板（base.html）**：用 `{% block 块名 %}` 标记可被重写的区域。  
   ```html
   <!-- templates/base.html -->
   <!DOCTYPE html>
   <html>
   <head>
       <title>{% block title %}默认标题{% endblock %}</title>  <!-- 标题块 -->
   </head>
   <body>
       <nav>导航栏</nav>  <!-- 不变的部分 -->
       <main>
           {% block content %}{% endblock %}  <!-- 内容块（子模板重写） -->
       </main>
       <footer>页脚</footer>  <!-- 不变的部分 -->
   </body>
   </html>
   ```  

2. **子模板继承父模板**：用 `{% extends 'base.html' %}` 继承，并通过 `{% block %}` 重写指定块。  
   ```html
   <!-- myapp/templates/myapp/index.html -->
   {% extends 'base.html' %}  <!-- 继承父模板 -->

   {% block title %}首页{% endblock %}  <!-- 重写标题块 -->

   {% block content %}  <!-- 重写内容块 -->
       <h1>这是首页内容</h1>
       <p>{{ name }}</p>
   {% endblock %}
   ```  


### 四、模板加载机制  
Django 会按以下顺序查找模板文件：  
1. 项目级 `templates/` 目录（需在 `settings.py` 的 `TEMPLATES[0]['DIRS']` 中配置，如 `[BASE_DIR / 'templates']`）。  
2. 各应用下的 `templates/应用名/` 目录（依赖 `TEMPLATES[0]['APP_DIRS'] = True`，默认开启）。  

**示例**：若项目结构为：  
```
myproject/
├── templates/              # 项目级模板目录
│   └── base.html
└── myapp/
    └── templates/
        └── myapp/
            └── index.html
```  
则 `{% extends 'base.html' %}` 会加载项目级的 `base.html`，`render(request, 'myapp/index.html')` 会加载应用内的 `index.html`。  


### 五、静态文件在模板中的引用  
静态文件（CSS、JS、图片等）需通过 `static` 标签引用，步骤：  
1. 在模板顶部加载 `static` 标签：`{% load static %}`。  
2. 引用文件：`{% static '路径' %}`（路径基于 `STATIC_URL` 配置）。  

**示例**：  
```html
{% load static %}  <!-- 加载静态文件标签 -->
<link rel="stylesheet" href="{% static 'css/style.css' %}">  <!-- 引用 CSS -->
<img src="{% static 'images/logo.png' %}" alt="Logo">  <!-- 引用图片 -->
```  


### 总结  
Django 模板通过**变量**展示动态数据，**标签**实现逻辑控制与复用，**过滤器**格式化输出，配合**继承**和**包含**机制，极大减少了代码重复，实现了前后端分离。掌握模板语法是构建 Django 动态页面的核心技能。
