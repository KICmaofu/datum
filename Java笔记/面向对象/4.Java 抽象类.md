在Java中，**抽象类（Abstract Class）** 是一种特殊的类，它不能被实例化（无法创建对象），主要用于作为其他类的父类，抽取子类的公共特征（属性和方法），并通过**抽象方法**定义子类必须实现的行为规范。抽象类是实现抽象概念、代码复用和多态的重要工具。


### 一、抽象类的定义与语法  
抽象类通过`abstract`关键字修饰，语法如下：  

```java
// 抽象类的定义
abstract class 抽象类名 {
    // 成员变量（可以是普通变量）
    // 非抽象方法（有方法体）
    // 抽象方法（无方法体，用abstract修饰）
}
```  


#### 核心特征：  
1. **不能实例化**：抽象类无法通过`new`关键字创建对象（如`new 抽象类名()`会编译报错），只能作为父类被继承。  
2. **可包含抽象方法**：抽象类中可以定义**抽象方法**（用`abstract`修饰，没有方法体的方法），子类必须重写这些抽象方法（除非子类也是抽象类）。  
3. **可包含普通成员**：抽象类可以像普通类一样包含成员变量、非抽象方法（有方法体）、构造方法等。  


### 二、抽象方法  
**抽象方法**是抽象类中用`abstract`修饰的方法，它**没有方法体**（即没有`{}`和实现代码），仅声明方法的签名（返回值类型、方法名、参数列表），目的是强制子类必须实现该方法。  

语法：  
```java
abstract 返回值类型 方法名(参数列表); // 无方法体，以分号结尾
```  


### 三、抽象类的使用示例  
以“几何图形”为例：所有图形都有“面积”和“周长”，但不同图形（如圆形、矩形）的计算方式不同。此时可定义`Shape`抽象类，包含抽象方法`getArea()`（求面积）和`getPerimeter()`（求周长），由子类具体实现。  


```java
// 1. 抽象类：图形（无法实例化）
abstract class Shape {
    // 普通成员变量：颜色
    String color;

    // 非抽象方法：设置颜色（有方法体）
    public void setColor(String color) {
        this.color = color;
    }

    // 抽象方法：求面积（无方法体，强制子类实现）
    public abstract double getArea();

    // 抽象方法：求周长（无方法体，强制子类实现）
    public abstract double getPerimeter();
}

// 2. 子类：圆形（继承抽象类，必须实现所有抽象方法）
class Circle extends Shape {
    double radius; // 半径

    public Circle(double radius) {
        this.radius = radius;
    }

    // 实现抽象方法：求圆的面积（πr²）
    @Override
    public double getArea() {
        return Math.PI * radius * radius;
    }

    // 实现抽象方法：求圆的周长（2πr）
    @Override
    public double getPerimeter() {
        return 2 * Math.PI * radius;
    }
}

// 3. 子类：矩形（继承抽象类，必须实现所有抽象方法）
class Rectangle extends Shape {
    double length; // 长
    double width;  // 宽

    public Rectangle(double length, double width) {
        this.length = length;
        this.width = width;
    }

    // 实现抽象方法：求矩形的面积（长×宽）
    @Override
    public double getArea() {
        return length * width;
    }

    // 实现抽象方法：求矩形的周长（2×(长+宽)）
    @Override
    public double getPerimeter() {
        return 2 * (length + width);
    }
}

// 4. 测试类：使用抽象类实现多态
public class AbstractDemo {
    public static void main(String[] args) {
        // 父类（抽象类）引用指向子类对象（多态）
        Shape circle = new Circle(5); // 圆的半径为5
        Shape rectangle = new Rectangle(3, 4); // 矩形长3、宽4

        circle.setColor("红色");
        rectangle.setColor("蓝色");

        System.out.println("红色圆的面积：" + circle.getArea()); // 78.539...
        System.out.println("蓝色矩形的周长：" + rectangle.getPerimeter()); // 14.0
    }
}
```  


### 四、抽象类的核心规则  
1. **抽象类不能实例化**：  
   ```java
   Shape shape = new Shape(); // 编译报错：Shape是抽象类，无法实例化
   ```  

2. **子类必须实现所有抽象方法**：  
   若子类继承抽象类后，**没有实现父类的所有抽象方法**，则子类自身必须声明为`abstract`（抽象子类）。  
   ```java
   // 抽象子类：未实现所有抽象方法，必须用abstract修饰
   abstract class Triangle extends Shape {
       // 仅实现部分抽象方法，未实现getPerimeter()
       @Override
       public double getArea() {
           return 0; // 临时实现
       }
       // 未实现getPerimeter() → 子类必须是抽象类
   }
   ```  

3. **抽象类可以有构造方法**：  
   抽象类的构造方法用于子类初始化时调用（通过`super()`），初始化抽象类中的成员变量。  
   ```java
   abstract class Shape {
       String color;
       
       // 抽象类的构造方法
       public Shape(String color) {
           this.color = color;
       }
   }
   
   class Circle extends Shape {
       public Circle(String color, double radius) {
           super(color); // 调用父类（抽象类）的构造方法
           this.radius = radius;
       }
   }
   ```  

4. **抽象类中可以没有抽象方法**：  
   抽象类可以仅包含非抽象方法（此时仍不能实例化），但这种情况较少见（通常用于限制实例化，仅作为父类）。  


### 五、抽象类 vs 普通类 vs 接口  
| 特性                | 抽象类（Abstract Class）               | 普通类（Class）                       | 接口（Interface，Java 8+）           |
|---------------------|----------------------------------------|---------------------------------------|--------------------------------------|
| 实例化              | 不能实例化                              | 可以实例化                            | 不能实例化                            |
| 成员变量            | 可以有普通变量（可修改）                | 可以有普通变量（可修改）              | 只能是常量（`public static final`）  |
| 方法                | 可包含抽象方法和非抽象方法              | 只能包含非抽象方法（有方法体）        | 可包含抽象方法、默认方法（`default`）、静态方法（`static`） |
| 继承/实现           | 子类通过`extends`继承（单继承）         | 子类通过`extends`继承（单继承）       | 类通过`implements`实现（多实现）      |
| 核心作用            | 抽取子类公共特征，定义部分实现，强制子类实现抽象方法 | 封装具体功能，可直接实例化            | 定义行为规范（“能做什么”），不关心实现 |  


### 六、抽象类的适用场景  
1. **定义抽象概念**：当某个类代表的是“抽象概念”（如“图形”“动物”），而非具体事物，且无法实例化时（不存在“图形”对象，只有“圆形”“矩形”对象）。  
2. **抽取公共代码**：多个子类有共同的属性或方法实现时，将这些公共部分放在抽象类中，子类只需关注自身特有的实现（减少代码冗余）。  
3. **强制子类实现特定方法**：通过抽象方法，要求所有子类必须实现某些核心行为（如所有图形必须能计算面积和周长）。  


### 总结  
抽象类是Java中介于普通类和接口之间的一种结构，核心特点是：  
- 用`abstract`修饰，不能实例化。  
- 可包含抽象方法（强制子类实现）和非抽象方法（提供公共实现）。  
- 作为父类，用于抽取子类共性、定义规范，并支持多态。  

抽象类与接口的选择原则：若强调“is-a”关系（如“圆是图形”），且需要复用代码，用抽象类；若强调“has-a”能力（如“可飞翔”），用接口。
