在Java中，**继承（Inheritance）** 是面向对象编程（OOP）的三大核心特性之一（另外两个是封装和多态），它允许一个类（称为**子类/派生类**）继承另一个类（称为**父类/基类**）的属性和方法，从而实现代码复用、扩展现有类的功能，并建立类之间的层次关系。


### 一、继承的核心目的  
1. **代码复用**：子类无需重复编写父类已有的属性和方法，直接继承使用。  
2. **功能扩展**：子类可以在继承父类的基础上，添加新的属性和方法，或重写父类的方法以实现特定逻辑。  
3. **建立层次关系**：通过继承将类组织成树形结构（如“动物→哺乳动物→狗”），体现现实世界的分类逻辑。  


### 二、继承的基本语法  
Java中通过`extends`关键字实现继承，语法如下：  

```java
// 父类（基类）
class 父类名 {
    // 父类的属性和方法
}

// 子类（派生类）：通过extends继承父类
class 子类名 extends 父类名 {
    // 子类的属性和方法（可新增或重写父类方法）
}
```  


#### 示例：基础继承关系  
```java
// 父类：动物
class Animal {
    // 父类的属性
    String name;
    
    // 父类的方法
    public void eat() {
        System.out.println(name + "在吃东西");
    }
}

// 子类：狗（继承自动物）
class Dog extends Animal {
    // 子类新增属性
    String breed; // 品种
    
    // 子类新增方法
    public void bark() {
        System.out.println(name + "在汪汪叫");
    }
    
    // 子类重写父类的eat()方法（扩展功能）
    @Override // 注解：标记方法重写（可选，但推荐，编译时会校验正确性）
    public void eat() {
        System.out.println(name + "（" + breed + "）在吃狗粮");
    }
}
```  

在上述例子中：  
- `Dog`是子类，`Animal`是父类，`Dog`通过`extends Animal`继承了`Animal`的`name`属性和`eat()`方法。  
- `Dog`新增了`breed`属性和`bark()`方法，还重写了`eat()`方法以实现狗的特定进食逻辑。  


### 三、继承的核心规则  
#### 1. 成员的继承范围  
子类会继承父类中**非私有（non-private）** 的成员（成员变量和方法），但以下情况例外：  
- **私有成员（private）**：父类的`private`属性和方法被封装在父类内部，子类无法直接访问（需通过父类的`public`/`protected`方法间接访问）。  
- **构造方法**：父类的构造方法不能被继承（子类需通过`super()`调用父类构造方法）。  


#### 2. 方法重写（Override）  
子类可以重写父类的方法（方法名、参数列表、返回值类型必须与父类完全一致），以实现子类特有的逻辑。重写需遵循“两同两小一大”规则：  
- **两同**：方法名相同、参数列表相同（顺序、类型、个数一致）。  
- **两小**：  
  - 返回值类型：子类返回值类型必须是父类返回值类型的子类（或相同，称为“协变返回类型”）。  
  - 抛出的异常：子类方法抛出的检查型异常不能比父类更宽泛（可更少或不抛）。  
- **一大**：子类方法的访问修饰符不能比父类更严格（如父类是`public`，子类不能是`protected`）。  


**示例：正确的方法重写**  
```java
class Parent {
    public void print() {
        System.out.println("父类方法");
    }
    
    public Number calculate() { // 返回值是Number
        return 100;
    }
}

class Child extends Parent {
    // 重写print()：方法名、参数相同，访问修饰符相同（public）
    @Override
    public void print() {
        System.out.println("子类方法");
    }
    
    // 重写calculate()：返回值是Integer（Number的子类，符合协变规则）
    @Override
    public Integer calculate() {
        return 200;
    }
}
```  


#### 3. `super`关键字  
`super`用于在子类中访问父类的成员，主要场景：  
- **访问父类的成员变量**：当子类与父类有同名成员变量时，用`super.变量名`区分。  
- **调用父类的方法**：当子类重写了父类方法后，用`super.方法名()`调用父类的原始方法。  
- **调用父类的构造方法**：用`super(参数)`在子类构造方法中调用父类的构造方法（必须放在子类构造方法的第一行）。  


**示例：`super`的使用**  
```java
class Parent {
    String name = "父类名称";
    
    public Parent() {
        System.out.println("父类无参构造");
    }
    
    public Parent(String name) {
        this.name = name;
        System.out.println("父类有参构造：" + name);
    }
    
    public void show() {
        System.out.println("父类show()：" + name);
    }
}

class Child extends Parent {
    String name = "子类名称"; // 与父类同名
    
    public Child() {
        super("传递给父类的名称"); // 调用父类有参构造（必须在第一行）
        System.out.println("子类无参构造");
    }
    
    @Override
    public void show() {
        super.show(); // 调用父类的show()
        System.out.println("子类show()：" + name); // 子类自己的name
        System.out.println("父类的name：" + super.name); // 访问父类的name
    }
}

// 测试
public class Test {
    public static void main(String[] args) {
        Child child = new Child();
        child.show();
    }
}
```  

**输出结果**：  
```
父类有参构造：传递给父类的名称
子类无参构造
父类show()：传递给父类的名称
子类show()：子类名称
父类的name：传递给父类的名称
```  


#### 4. 构造方法的继承与调用  
子类**不能继承**父类的构造方法，但必须在自己的构造方法中**间接调用父类的构造方法**（确保父类成员被正确初始化）：  
- 若子类构造方法中未显式调用`super()`，编译器会**自动添加`super()`**（调用父类的无参构造）。  
- 若父类**没有无参构造**（仅定义了有参构造），子类必须**显式调用父类的有参构造**（`super(参数)`），否则编译报错。  


**示例：父类无无参构造时的处理**  
```java
class Parent {
    // 父类仅定义有参构造，无默认无参构造
    public Parent(String name) {
        System.out.println("父类有参构造：" + name);
    }
}

class Child extends Parent {
    // 子类必须显式调用父类的有参构造，否则编译报错
    public Child() {
        super("默认名称"); // 调用父类有参构造
    }
    
    public Child(String name) {
        super(name); // 调用父类有参构造
    }
}
```  


### 四、继承的特性  
1. **单继承**：Java中一个类**只能直接继承一个父类**（`class A extends B, C` 是错误的），避免多继承带来的歧义（如多个父类有同名方法时无法确定调用哪个）。  
2. **多层继承**：一个类可以继承另一个子类，形成“祖父→父→子”的多层继承关系（如`class A extends B`，`class B extends C`，则`A`间接继承`C`）。  
3. **所有类的根类：`Object`**：Java中所有类（包括自定义类）都**直接或间接继承`java.lang.Object`类**。`Object`类提供了所有对象的通用方法（如`toString()`、`equals()`、`hashCode()`等），因此任何对象都可以调用这些方法。  


**示例：`Object`类的间接继承**  
```java
// 自定义类未显式继承任何类，默认继承Object
class MyClass {
    // 隐含继承Object的所有方法
}

public class Test {
    public static void main(String[] args) {
        MyClass obj = new MyClass();
        // 调用Object类的toString()方法
        System.out.println(obj.toString()); // 输出：MyClass@xxxxxx（默认格式）
    }
}
```  


### 五、继承与组合的对比  
继承是“is-a”关系（如“狗是动物”），而**组合**是“has-a”关系（如“汽车有发动机”）。在设计类时，应优先考虑组合而非继承，避免过度继承导致的类层次复杂、耦合度高。  

- **继承**：适合“子类是父类的一种”的场景（如`Dog is a Animal`），优势是代码复用和多态。  
- **组合**：适合“类包含另一个类的实例”的场景（如`Car has a Engine`），优势是灵活性高、耦合度低。  


### 总结  
继承是Java实现代码复用和类层次化的核心机制，核心要点：  
- 通过`extends`关键字实现，子类继承父类的非私有成员。  
- 子类可通过**方法重写**扩展父类功能，需遵循“两同两小一大”规则。  
- `super`关键字用于访问父类成员和调用父类构造方法。  
- Java支持单继承和多层继承，所有类最终继承自`Object`类。  

合理使用继承能减少代码冗余，建立清晰的类关系，但需避免过度继承（优先考虑组合）。
