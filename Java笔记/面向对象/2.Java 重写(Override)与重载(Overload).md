在Java中，**重写（Override）** 和**重载（Overload）** 是实现多态的两种重要机制，但二者的适用场景、规则和目的截然不同。理解它们的区别是掌握Java面向对象编程的基础。


### 一、重写（Override）：子类对父类方法的“改造”  
**重写**指的是**子类继承父类后，对父类中已有的方法进行重新实现**（方法名、参数列表、返回值类型与父类保持一致），以满足子类的特定需求。重写是**运行时多态**的体现，核心是“子类覆盖父类的行为”。  


#### 1. 重写的核心规则（“两同两小一大”）  
- **两同**：  
  - 方法名**必须相同**。  
  - 参数列表**必须相同**（参数的类型、个数、顺序完全一致）。  
- **两小**：  
  - 返回值类型：子类方法的返回值类型**必须是父类方法返回值类型的子类（或相同）**（称为“协变返回类型”）。  
  - 抛出的异常：子类方法抛出的**检查型异常**范围**不能比父类更宽泛**（可更少或不抛）。  
- **一大**：  
  - 访问修饰符：子类方法的访问权限**不能比父类更严格**（如父类是`public`，子类不能是`protected`或`private`）。  


#### 2. 重写的示例  
```java
// 父类：动物
class Animal {
    // 父类的方法：吃
    public void eat() {
        System.out.println("动物在吃东西");
    }
    
    // 父类的方法：返回年龄
    public Number getAge() {
        return 0;
    }
}

// 子类：狗（继承自动物）
class Dog extends Animal {
    // 重写父类的eat()方法：方法名、参数相同，访问修饰符相同（public）
    @Override // 注解：标记重写（可选，但推荐，编译时校验正确性）
    public void eat() {
        System.out.println("狗在吃狗粮"); // 子类特有的实现
    }
    
    // 重写父类的getAge()：返回值是Integer（Number的子类，符合协变规则）
    @Override
    public Integer getAge() {
        return 3;
    }
}
```  

- 子类`Dog`重写了父类`Animal`的`eat()`和`getAge()`方法，保留了方法名和参数列表，但实现了狗的特定逻辑。  
- `@Override`注解的作用：若方法未正确重写（如方法名拼写错误、参数不一致），编译时会报错，避免逻辑错误。  


#### 3. 重写的注意事项  
- 父类的**私有方法（private）不能被重写**（子类无法继承私有方法）。  
- 父类的**静态方法（static）不能被重写**（静态方法属于类，而非实例，子类可定义同名静态方法，但这是“隐藏”而非“重写”）。  
- 父类的**最终方法（final）不能被重写**（`final`修饰的方法不可修改）。  


### 二、重载（Overload）：同一类中方法的“多态”  
**重载**指的是**在同一个类中，定义多个方法名相同但参数列表不同的方法**。重载是**编译时多态**的体现，核心是“同一功能的方法适应不同参数输入”。  


#### 1. 重载的核心规则  
- **方法名必须相同**。  
- **参数列表必须不同**（至少满足以下一点）：  
  - 参数**个数不同**（如`add(int a)`与`add(int a, int b)`）。  
  - 参数**类型不同**（如`add(int a)`与`add(double a)`）。  
  - 参数**顺序不同**（如`add(int a, double b)`与`add(double a, int b)`）。  
- **与返回值类型、访问修饰符无关**（仅通过返回值或修饰符不同无法构成重载）。  


#### 2. 重载的示例  
```java
// 计算器类：重载add方法，支持不同参数
class Calculator {
    // 1. 两个int相加
    public int add(int a, int b) {
        return a + b;
    }
    
    // 2. 三个int相加（参数个数不同 → 重载）
    public int add(int a, int b, int c) {
        return a + b + c;
    }
    
    // 3. 两个double相加（参数类型不同 → 重载）
    public double add(double a, double b) {
        return a + b;
    }
    
    // 4. int和double相加（参数顺序不同 → 重载）
    public double add(int a, double b) {
        return a + b;
    }
    
    // 以下不构成重载（仅返回值不同）
    // public double add(int a, int b) { ... } // 编译报错：与第一个方法冲突
}
```  

- 上述`Calculator`类中，`add`方法被重载了4次，方法名相同，但参数列表不同，分别处理不同的输入场景。  
- 调用时，编译器会根据**实参的类型和个数**自动匹配对应的方法（编译时确定）。  


#### 3. 重载的注意事项  
- 重载**仅发生在同一个类中**（或父类与子类的继承关系中，但子类重载父类方法本质上是在子类中新增方法，而非重写）。  
- 参数的“顺序不同”需基于不同类型的参数（如`add(int, double)`与`add(double, int)`），若参数类型相同仅顺序不同（如`add(int a, int b)`与`add(int b, int a)`），不构成有效重载（编译器无法区分）。  


### 三、重写与重载的核心区别  
| 对比维度         | 重写（Override）                          | 重载（Overload）                          |
|------------------|-------------------------------------------|-------------------------------------------|
| 发生范围         | 子类与父类之间（继承关系）                | 同一个类中（或子类与父类，但本质是子类新增） |
| 方法名           | 必须相同                                  | 必须相同                                  |
| 参数列表         | 必须相同（类型、个数、顺序）              | 必须不同（类型、个数、顺序至少一个不同）  |
| 返回值类型       | 子类返回值必须是父类返回值的子类（或相同） | 无要求（可相同或不同）                    |
| 访问修饰符       | 子类权限不能比父类更严格                  | 无要求（可任意修饰）                      |
| 异常处理         | 子类抛出的检查型异常不能比父类更宽泛      | 无要求                                    |
| 多态类型         | 运行时多态（调用哪个方法由对象类型决定）  | 编译时多态（调用哪个方法由实参类型决定）  |
| 核心目的         | 扩展或修改父类方法的功能                  | 提供多种参数组合的调用方式，方便使用      |  


### 四、总结  
- **重写**是“父子类间的方法覆盖”，要求方法签名（名称+参数）完全一致，用于在继承中定制子类行为，是运行时多态的基础。  
- **重载**是“同类中方法的多版本”，要求方法名相同但参数不同，用于适配不同的输入场景，是编译时多态的体现。  

两者都是Java多态机制的重要组成部分，理解其规则和区别，能帮助写出更灵活、可扩展的代码。
