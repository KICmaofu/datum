Java 中的 `java.lang.Object` 类是所有类的**根类（超类）**——任何类（包括数组）都直接或间接继承自 `Object` 类。这意味着 Java 中所有对象都拥有 `Object` 类中定义的方法，这些方法为对象提供了最基础的行为（如对象比较、字符串表示、哈希计算等）。理解 `Object` 类是掌握 Java 面向对象编程的基础。


### 一、Object 类的核心地位  
- 所有类默认继承 `Object`（即使未显式声明 `extends Object`）。  
- 数组也被视为 `Object` 的子类（可直接调用 `Object` 的方法）。  
- `Object` 类定义了 11 个方法，这些方法是所有 Java 对象的“通用行为”。  


### 二、Object 类的常用方法  
以下是 `Object` 类中最常用的方法，也是开发中最需要掌握的核心内容：  


#### 1. `toString()`：对象的字符串表示  
- **作用**：返回对象的字符串描述，便于打印或日志输出。  
- **默认实现**：返回 `类名@哈希码的十六进制`（如 `com.example.Student@1b6d3586`），其中哈希码是 `hashCode()` 方法的返回值。  
- **为什么需要重写**：默认实现几乎没有实际意义，重写后可返回对象的关键属性（如 `Student{id='001', name='张三'}`）。  


**示例：重写 `toString()`**  
```java
class Student {
    private String id;
    private String name;

    public Student(String id, String name) {
        this.id = id;
        this.name = name;
    }

    // 重写 toString()，返回有意义的信息
    @Override
    public String toString() {
        return "Student{id='" + id + "', name='" + name + "'}";
    }
}

public class Test {
    public static void main(String[] args) {
        Student s = new Student("001", "张三");
        System.out.println(s); // 输出：Student{id='001', name='张三'}（调用 s.toString()）
    }
}
```  


#### 2. `equals(Object obj)`：对象内容比较  
- **作用**：判断当前对象与参数 `obj` 是否“相等”（逻辑上的相等，如内容相同）。  
- **默认实现**：等价于 `==`，即比较两个对象的**引用地址**（只有当两个变量指向同一个对象时才返回 `true`）。  
- **为什么需要重写**：实际开发中，常需判断“内容是否相同”（如两个 `Student` 对象的 `id` 相同则认为相等），而非引用地址。  


**重写 `equals()` 的规范**：  
- 自反性：`x.equals(x)` 必须返回 `true`。  
- 对称性：若 `x.equals(y)` 为 `true`，则 `y.equals(x)` 也必须为 `true`。  
- 传递性：若 `x.equals(y)` 和 `y.equals(z)` 为 `true`，则 `x.equals(z)` 也必须为 `true`。  
- 一致性：多次调用 `x.equals(y)`，结果应保持一致（对象未被修改时）。  
- 非空性：`x.equals(null)` 必须返回 `false`。  


**示例：重写 `equals()`**  
```java
class Student {
    private String id;
    private String name;

    // 构造方法省略...

    @Override
    public boolean equals(Object obj) {
        // 1. 引用相同（指向同一个对象），直接返回 true
        if (this == obj) return true;
        // 2. 参数为 null 或类型不同，返回 false
        if (obj == null || getClass() != obj.getClass()) return false;
        // 3. 强转后比较关键属性（如 id）
        Student student = (Student) obj;
        return id.equals(student.id); // 假设 id 唯一标识学生
    }
}

public class Test {
    public static void main(String[] args) {
        Student s1 = new Student("001", "张三");
        Student s2 = new Student("001", "张三三"); // id 相同，内容相等
        System.out.println(s1.equals(s2)); // 输出：true
    }
}
```  


#### 3. `hashCode()`：获取对象的哈希码  
- **作用**：返回对象的哈希码（一个 int 整数），主要用于哈希表（如 `HashMap`、`HashSet`）中快速定位对象。  
- **默认实现**：返回对象的内存地址相关的整数（不同对象哈希码通常不同）。  
- **与 `equals()` 的关系**：**若两个对象 `equals()` 返回 `true`，则它们的 `hashCode()` 必须相等**；反之，若 `hashCode()` 不同，则 `equals()` 必为 `false`（哈希表的核心约定）。  


**为什么必须同时重写 `equals()` 和 `hashCode()`**？  
若只重写 `equals()` 而不重写 `hashCode()`，会导致两个“相等”的对象（`equals()` 为 `true`）拥有不同的哈希码，从而在 `HashMap` 等集合中被视为不同对象，破坏集合的去重逻辑。  


**示例：同时重写 `hashCode()`**  
```java
class Student {
    private String id;
    private String name;

    // 构造方法、equals() 省略...

    // 基于 equals() 中比较的属性（id）计算哈希码
    @Override
    public int hashCode() {
        return id.hashCode(); // 直接使用 id 的哈希码（id 是 String，已重写 hashCode()）
    }
}
```  


#### 4. `getClass()`：获取对象的运行时类  
- **作用**：返回对象的**运行时类**（`Class` 对象），用于反射（动态获取类信息、调用方法等）。  
- **特点**：方法被 `final` 修饰，**不能被重写**。  


**示例**：  
```java
class Student {}
class Teacher {}

public class Test {
    public static void main(String[] args) {
        Object s = new Student();
        Object t = new Teacher();

        System.out.println(s.getClass()); // 输出：class com.example.Student（运行时实际类型）
        System.out.println(t.getClass()); // 输出：class com.example.Teacher
    }
}
```  


#### 5. 线程通信方法：`wait()`、`notify()`、`notifyAll()`  
这些方法用于**多线程间的协作**（如线程等待、唤醒），依赖对象的“内置锁”（`synchronized` 锁），必须在同步代码块/方法中使用。  

| 方法                | 作用                                  |  
|---------------------|---------------------------------------|  
| `void wait()`       | 让当前线程释放对象锁并进入等待状态，直到被 `notify()` 或 `notifyAll()` 唤醒。 |  
| `void wait(long timeout)` | 同上，但最多等待 `timeout` 毫秒（超时后自动唤醒）。 |  
| `void notify()`     | 唤醒在此对象锁上等待的**一个**线程（随机选择）。 |  
| `void notifyAll()`  | 唤醒在此对象锁上等待的**所有**线程。 |  


**示例：简单的线程通信**  
```java
public class WaitNotifyDemo {
    public static void main(String[] args) {
        Object lock = new Object(); // 共享锁对象

        // 线程1：等待通知
        new Thread(() -> {
            synchronized (lock) {
                try {
                    System.out.println("线程1：等待通知...");
                    lock.wait(); // 释放锁并等待
                    System.out.println("线程1：收到通知，继续执行");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }).start();

        // 线程2：发送通知
        new Thread(() -> {
            synchronized (lock) {
                System.out.println("线程2：发送通知...");
                lock.notify(); // 唤醒一个等待的线程
                System.out.println("线程2：通知完成");
            }
        }).start();
    }
}
```  

输出：  
```
线程1：等待通知...
线程2：发送通知...
线程2：通知完成
线程1：收到通知，继续执行
```  


#### 6. `clone()`：对象克隆  
- **作用**：创建并返回当前对象的“副本”（克隆体），默认是**浅克隆**（基本类型复制值，引用类型复制地址）。  
- **使用条件**：  
  - 类必须实现 `Cloneable` 接口（标记接口，无实际方法，仅表示允许克隆）。  
  - 重写 `clone()` 方法（`Object` 中的 `clone()` 是 `protected`，需重写为 `public`）。  
- **注意**：若未实现 `Cloneable` 而调用 `clone()`，会抛出 `CloneNotSupportedException`。  


**示例：对象克隆**  
```java
class Student implements Cloneable { // 实现 Cloneable 接口
    private String id;
    private String name;

    public Student(String id, String name) {
        this.id = id;
        this.name = name;
    }

    // 重写 clone() 方法
    @Override
    public Student clone() throws CloneNotSupportedException {
        return (Student) super.clone(); // 调用父类的 clone()
    }
}

public class Test {
    public static void main(String[] args) throws CloneNotSupportedException {
        Student s1 = new Student("001", "张三");
        Student s2 = s1.clone(); // 克隆 s1

        System.out.println(s1 == s2); // 输出：false（不同对象）
        System.out.println(s1.equals(s2)); // 输出：false（未重写 equals()，比较引用）
        System.out.println(s2.id); // 输出：001（克隆了属性值）
    }
}
```  


#### 7. `finalize()`：对象回收前的清理（已过时）  
- **作用**：垃圾回收器回收对象前会调用此方法，用于释放资源（如关闭文件、网络连接）。  
- **现状**：Java 9 起被标记为 `@Deprecated`（过时），因为其执行时间不确定（无法保证一定会被调用），且可能导致性能问题。**不推荐使用**，资源释放应通过 `try-with-resources` 等显式方式。  


### 三、总结  
`Object` 类是 Java 类层次结构的根基，所有对象都继承其方法，核心要点：  
- `toString()`：返回对象的字符串描述，需重写以提供有意义信息。  
- `equals()`：判断对象内容是否相等，默认比较引用，重写时需遵循规范。  
- `hashCode()`：返回哈希码，与 `equals()` 强关联，必须同时重写。  
- `getClass()`：获取运行时类，用于反射。  
- `wait()`/`notify()`/`notifyAll()`：多线程通信的核心方法，依赖同步锁。  

理解 `Object` 类的方法及其设计意图，是写出规范、高效 Java 代码的基础，尤其在集合操作（如 `HashMap`）、多线程编程中至关重要。
