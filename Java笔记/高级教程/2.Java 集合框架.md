Java 集合框架（Java Collections Framework）是 Java 中用于存储、操作和管理对象的统一架构，提供了一套灵活、高效的容器类，解决了数组长度固定、操作繁琐等问题。其核心目标是简化数据存储和处理的逻辑，同时提供丰富的功能（如排序、查找、遍历等）。


### 一、集合框架的整体结构
Java 集合框架的核心接口分为两大分支：**`Collection`**（存储单值元素的集合）和**`Map`**（存储键值对的集合）。


#### 1. Collection 接口（单值集合）
`Collection` 是所有单值集合的根接口，定义了操作元素的通用方法（如 `add()`、`remove()`、`size()` 等）。其主要子接口包括：

- **`List`**：有序、可重复的集合（允许元素为 `null`）。
- **`Set`**：无序、不可重复的集合（多数实现类允许 `null`，但仅能有一个）。
- **`Queue`**：队列结构（通常遵循 FIFO 原则，即先进先出）。


#### 2. Map 接口（键值对集合）
`Map` 用于存储键值对（`key-value`），其中 **键（key）唯一**，**值（value）可重复**（允许 `null` 键和 `null` 值，具体取决于实现类）。其核心方法包括 `put(key, value)`、`get(key)`、`containsKey(key)` 等。


### 二、核心接口及实现类详解


#### 1. List 接口（有序可重复）
`List` 最大的特点是**元素有序（插入顺序）且可重复**，支持通过索引（类似数组）访问元素。

| 实现类       | 底层数据结构               | 特点                                                                 | 适用场景                     |
|--------------|----------------------------|----------------------------------------------------------------------|------------------------------|
| `ArrayList`  | 动态数组（默认初始容量 10） | 查询快（索引访问），增删慢（需移动元素）；线程不安全；扩容为原容量 1.5 倍。 | 查询频繁、增删少的场景       |
| `LinkedList` | 双向链表                   | 增删快（仅需修改指针），查询慢（需遍历）；线程不安全；实现了 `Queue` 接口。 | 增删频繁、查询少的场景       |
| `Vector`     | 动态数组（古老实现）       | 线程安全（方法加锁），但效率低；扩容为原容量 2 倍（默认）。              | 需线程安全但可接受低效率场景 |


#### 2. Set 接口（无序不可重复）
`Set` 不允许元素重复（通过 `equals()` 和 `hashCode()` 判断），多数实现类**无序**（插入顺序与存储顺序无关）。

| 实现类          | 底层依赖                     | 特点                                                                 | 适用场景                     |
|-----------------|------------------------------|----------------------------------------------------------------------|------------------------------|
| `HashSet`       | 基于 `HashMap`（键存储元素） | 无序；查询、增删效率高（哈希表）；允许 `null` 元素（仅一个）。           | 去重且无需顺序的场景         |
| `LinkedHashSet` | 基于 `LinkedHashMap`         | 有序（维护插入顺序）；效率略低于 `HashSet`。                           | 去重且需保留插入顺序的场景   |
| `TreeSet`       | 基于 `TreeMap`（红黑树）     | 有序（按元素自然排序或自定义排序）；查询、增删效率为 O(log n)。          | 去重且需排序的场景           |


#### 3. Queue 接口（队列）
`Queue` 主要用于模拟队列操作，通常遵循 FIFO（先进先出）原则，部分实现类支持优先级排序。

| 实现类           | 特点                                                                 | 适用场景                     |
|------------------|----------------------------------------------------------------------|------------------------------|
| `LinkedList`     | 实现 `Queue` 接口，可作为双向队列（FIFO）。                           | 普通队列或双端队列操作       |
| `PriorityQueue`  | 优先级队列（默认按自然排序，可自定义 `Comparator`）；元素不能为 `null`。 | 需要按优先级处理元素的场景   |


#### 4. Map 接口（键值对）
`Map` 存储键值对，键唯一（重复键会覆盖旧值），值可重复。

| 实现类             | 底层数据结构               | 特点                                                                 | 适用场景                     |
|--------------------|----------------------------|----------------------------------------------------------------------|------------------------------|
| `HashMap`          | 数组 + 链表/红黑树（JDK8+） | 无序；查询、增删效率高（哈希表）；线程不安全；允许 `null` 键和 `null` 值。 | 快速键值对查询，无需顺序     |
| `LinkedHashMap`    | 基于 `HashMap` + 双向链表  | 有序（维护插入顺序或访问顺序）；效率略低于 `HashMap`。                  | 需保留键值对顺序的场景       |
| `TreeMap`          | 红黑树                     | 有序（按键自然排序或自定义排序）；查询、增删效率为 O(log n)；不允许 `null` 键。 | 需按键排序的键值对场景       |
| `Hashtable`        | 哈希表（古老实现）         | 线程安全（方法加锁）；效率低；不允许 `null` 键和 `null` 值。              | 需线程安全但可接受低效率场景 |
| `ConcurrentHashMap`| 分段锁/CAS（JDK8+优化）    | 线程安全（高效并发）；支持 `null` 值（不允许 `null` 键）。               | 高并发场景下的键值对操作     |


### 三、集合框架的工具类
- **`Collections`**：操作集合的工具类，提供静态方法如：
  - 排序：`sort(list)`（对 `List` 排序）。
  - 查找：`binarySearch(list, key)`（二分查找）。
  - 同步化：`synchronizedList(list)`（将集合转为线程安全版本）。
  - 不可变集合：`unmodifiableList(list)`（返回不可修改的集合视图）。

- **`Arrays`**：操作数组的工具类，提供数组与集合的转换（如 `Arrays.asList(arr)`）。


### 四、重要概念
1. **迭代器（Iterator）**：用于遍历集合的接口，`Collection` 提供 `iterator()` 方法返回迭代器，通过 `hasNext()`（判断是否有下一个元素）和 `next()`（获取下一个元素）遍历，`remove()` 可删除当前元素。`List` 还提供 `ListIterator`，支持双向遍历和修改元素。

2. **泛型（Generics）**：JDK5 引入，用于约束集合元素类型（如 `List<String>` 仅存字符串），避免运行时类型转换错误，提高安全性。

3. **哈希表原理**：`HashMap`、`HashSet` 等基于哈希表，通过 `hashCode()` 计算元素存储位置，`equals()` 解决哈希冲突（同一位置的元素用链表/红黑树存储）。


### 五、总结
Java 集合框架通过接口标准化了各类数据结构的操作，开发者可根据需求选择合适的实现类：
- 需有序可重复 → `List`（`ArrayList`/`LinkedList`）。
- 需去重 → `Set`（`HashSet`/`TreeSet`）。
- 需键值对 → `Map`（`HashMap`/`TreeMap`）。
- 需队列操作 → `Queue`（`LinkedList`/`PriorityQueue`）。

合理选择集合可显著提升程序效率和可读性。
