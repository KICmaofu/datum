Java 序列化（Serialization）是指将**对象的状态**（即对象的成员变量值）转换为**字节流**的过程，以便将其存储到文件、数据库，或通过网络传输；反之，将字节流恢复为对象的过程称为**反序列化（Deserialization）**。  

序列化的核心价值是**跨平台、跨网络的数据传输**和**对象持久化**（如将对象状态保存到磁盘，下次程序启动时恢复）。


### 一、序列化的基本实现  
Java 通过 `java.io.Serializable` 接口（标记接口）和 `ObjectOutputStream`/`ObjectInputStream` 类实现序列化和反序列化。  


#### 1. 核心条件：实现 `Serializable` 接口  
一个类的对象要能被序列化，必须满足：  
- 类**直接或间接实现 `java.io.Serializable` 接口**（该接口无任何方法，仅作为“可序列化”的标记）。  
- 类的所有非 `transient` 成员变量的类型也必须可序列化（基本类型默认可序列化）。  


#### 2. 核心类与方法  
| 类                  | 作用                          | 核心方法                                  |  
|---------------------|-------------------------------|-------------------------------------------|  
| `ObjectOutputStream` | 将对象序列化为字节流          | `writeObject(Object obj)`：序列化对象     |  
| `ObjectInputStream`  | 将字节流反序列化为对象        | `readObject()`：反序列化对象，返回 `Object` 类型 |  


### 二、基本使用示例  
#### 步骤 1：定义可序列化的类  
```java
import java.io.Serializable;

// 实现 Serializable 接口，标记为可序列化
class User implements Serializable {
    // 显式声明序列化版本号（强烈推荐）
    private static final long serialVersionUID = 1L;

    private String username;
    private int age;
    // transient 修饰的变量不参与序列化（临时变量）
    private transient String password; 

    public User(String username, int age, String password) {
        this.username = username;
        this.age = age;
        this.password = password;
    }

    @Override
    public String toString() {
        return "User{username='" + username + "', age=" + age + ", password='" + password + "'}";
    }
}
```  


#### 步骤 2：序列化对象到文件  
```java
import java.io.FileOutputStream;
import java.io.ObjectOutputStream;
import java.io.IOException;

public class SerializeDemo {
    public static void main(String[] args) {
        User user = new User("zhangsan", 20, "123456");
        
        // 序列化：将对象写入文件
        try (ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream("user.ser"))) {
            oos.writeObject(user); // 序列化对象
            System.out.println("序列化完成：" + user);
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```  

输出：  
```
序列化完成：User{username='zhangsan', age=20, password='123456'}
```  


#### 步骤 3：从文件反序列化对象  
```java
import java.io.FileInputStream;
import java.io.ObjectInputStream;
import java.io.IOException;
import java.io.ClassNotFoundException;

public class DeserializeDemo {
    public static void main(String[] args) {
        // 反序列化：从文件读取对象
        try (ObjectInputStream ois = new ObjectInputStream(new FileInputStream("user.ser"))) {
            User user = (User) ois.readObject(); // 反序列化，需强转
            System.out.println("反序列化完成：" + user);
        } catch (IOException | ClassNotFoundException e) {
            e.printStackTrace();
        }
    }
}
```  

输出：  
```
反序列化完成：User{username='zhangsan', age=20, password='null'}
```  

**说明**：`password` 被 `transient` 修饰，未参与序列化，因此反序列化后为 `null`。  


### 三、关键概念：`serialVersionUID`  
`serialVersionUID` 是**序列化版本号**，是一个 `static final long` 类型的常量，用于标识类的序列化版本。  

- **作用**：确保反序列化时，字节流对应的类版本与当前类版本一致。若类结构发生变化（如增减字段），但 `serialVersionUID` 不变，反序列化仍能兼容旧版本对象；若 `serialVersionUID` 变化，会抛出 `InvalidClassException`。  
- **生成方式**：  
  - 显式声明：推荐手动指定（如 `1L`），避免编译器自动生成（编译器会根据类结构计算，类结构变化会导致值变化）。  
  - 隐式生成：未显式声明时，编译器会自动计算（不推荐，易导致版本不兼容）。  


### 四、不可序列化的情况  
以下场景的对象无法被序列化：  
1. 类未实现 `Serializable` 接口（或其父类均未实现）。  
2. 成员变量的类型不可序列化（且未用 `transient` 修饰）。  
3. 被 `transient` 关键字修饰的成员变量（主动排除序列化）。  
4. 静态变量（属于类而非对象，不参与对象状态的序列化）。  


### 五、自定义序列化/反序列化逻辑  
默认情况下，序列化会保存所有非 `transient` 成员变量。若需自定义逻辑（如加密敏感字段），可在类中定义以下**私有方法**（JVM 会自动调用，无需显式实现接口）：  

```java
private void writeObject(ObjectOutputStream out) throws IOException {
    // 自定义序列化逻辑（如先加密再写入）
    out.defaultWriteObject(); // 调用默认序列化（保存非transient字段）
    // 额外处理：如加密password后写入（即使被transient修饰）
    String encryptedPwd = encrypt(password); // 假设encrypt()是加密方法
    out.writeObject(encryptedPwd);
}

private void readObject(ObjectInputStream in) throws IOException, ClassNotFoundException {
    // 自定义反序列化逻辑（如解密后恢复）
    in.defaultReadObject(); // 调用默认反序列化
    // 额外处理：读取加密的password并解密
    String encryptedPwd = (String) in.readObject();
    this.password = decrypt(encryptedPwd); // 解密
}
```  


### 六、应用场景  
1. **对象持久化**：将对象状态保存到文件/数据库（如游戏存档）。  
2. **网络传输**：在分布式系统中通过网络传输对象（如 RMI 远程方法调用、Socket 通信）。  
3. **深拷贝**：通过序列化+反序列化实现对象的深复制（避免引用传递导致的副作用）。  


### 七、注意事项  
1. **安全性**：反序列化不可信数据可能导致安全漏洞（如远程代码执行），需谨慎处理（可使用 `java.io.Externalizable` 接口更严格控制，或避免反序列化未知数据）。  
2. **性能**：序列化会产生额外的字节流开销，频繁序列化可能影响性能。  
3. **版本兼容**：修改类结构后，需确保 `serialVersionUID` 兼容旧版本，或通过自定义逻辑处理版本差异。  
4. **特殊类**：部分系统类不可序列化（如 `Thread`、`InputStream`、`ClassLoader` 等），因其状态与底层资源绑定，无法通过字节流恢复。  


### 总结  
Java 序列化通过 `Serializable` 接口和 `ObjectOutputStream`/`ObjectInputStream` 实现对象与字节流的转换，核心用于对象持久化和网络传输。关键要点：  
- 类必须实现 `Serializable` 接口，推荐显式声明 `serialVersionUID`。  
- `transient` 修饰的变量不参与序列化，静态变量也不参与。  
- 可通过 `writeObject`/`readObject` 自定义序列化逻辑。  

合理使用序列化可简化跨平台数据交互，但需注意版本兼容和安全风险。
