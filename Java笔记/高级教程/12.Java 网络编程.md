Java 网络编程是指通过 Java 语言实现不同设备（如计算机、服务器）之间的网络通信，核心基于 **TCP/IP 协议栈**，主要涉及 **Socket（套接字）** 编程。Java 提供了丰富的类库（如 `java.net` 包）简化网络通信开发，支持 TCP（可靠传输）和 UDP（高效传输）两种主流协议。


### 一、网络编程基础概念  
在开始编程前，需理解几个核心概念：  

| 概念         | 说明                                  |  
|--------------|---------------------------------------|  
| **IP 地址**  | 标识网络中设备的唯一地址（如 `192.168.1.1` 或 `www.baidu.com` 域名解析后的地址）。 |  
| **端口（Port）** | 设备内不同应用的唯一标识（0-65535，其中 0-1023 为系统端口，如 HTTP 用 80，HTTPS 用 443）。 |  
| **TCP 协议** | 面向连接的可靠传输协议（三次握手建立连接，四次挥手断开连接），保证数据不丢失、不重复、按序到达，适合文件传输、登录等场景。 |  
| **UDP 协议** | 无连接的不可靠传输协议（直接发送数据，不保证到达），速度快，适合实时通信（如视频通话、游戏）。 |  
| **Socket**   | 网络通信的端点，封装了 IP 地址和端口，是 TCP/UDP 通信的核心对象。 |  


### 二、TCP 编程（可靠传输）  
TCP 通信需先建立连接（类似“打电话”），通信双方分为 **服务器端（Server）** 和 **客户端（Client）**，流程如下：  

1. 服务器端：创建 `ServerSocket` 绑定端口，监听客户端连接，接受连接后通过 `Socket` 与客户端通信。  
2. 客户端：创建 `Socket`，指定服务器 IP 和端口，发起连接，连接成功后通过 `Socket` 与服务器通信。  


#### 核心类  
- `ServerSocket`：服务器端套接字，用于监听客户端连接请求。  
- `Socket`：客户端/服务器端的通信套接字，通过输入流（`InputStream`）读取数据，输出流（`OutputStream`）发送数据。  


#### 示例：TCP 服务器与客户端通信  
**1. 服务器端（接收客户端消息并回复）**  
```java
import java.io.*;
import java.net.ServerSocket;
import java.net.Socket;

public class TcpServer {
    public static void main(String[] args) {
        int port = 8888; // 绑定端口
        try (ServerSocket serverSocket = new ServerSocket(port)) { // 自动关闭资源
            System.out.println("服务器启动，监听端口：" + port);

            // 监听客户端连接（阻塞方法，直到有客户端连接）
            Socket clientSocket = serverSocket.accept();
            System.out.println("客户端已连接：" + clientSocket.getInetAddress());

            // 获取输入流（读取客户端消息）和输出流（向客户端发送消息）
            try (BufferedReader in = new BufferedReader(
                    new InputStreamReader(clientSocket.getInputStream()));
                 PrintWriter out = new PrintWriter(
                         clientSocket.getOutputStream(), true)) {

                String clientMsg;
                // 循环读取客户端消息（阻塞，直到客户端发送数据）
                while ((clientMsg = in.readLine()) != null) {
                    System.out.println("客户端：" + clientMsg);
                    // 回复客户端
                    out.println("服务器已收到：" + clientMsg);

                    // 若客户端发送 "bye"，断开连接
                    if ("bye".equals(clientMsg)) {
                        break;
                    }
                }
            }
            System.out.println("客户端已断开连接");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```  


**2. 客户端（向服务器发送消息并接收回复）**  
```java
import java.io.*;
import java.net.Socket;

public class TcpClient {
    public static void main(String[] args) {
        String serverIp = "localhost"; // 服务器 IP（本地测试用 localhost）
        int serverPort = 8888; // 服务器端口

        try (Socket socket = new Socket(serverIp, serverPort)) { // 连接服务器
            System.out.println("已连接到服务器");

            // 获取输入流（读取服务器回复）和输出流（向服务器发送消息）
            try (BufferedReader in = new BufferedReader(
                    new InputStreamReader(socket.getInputStream()));
                 PrintWriter out = new PrintWriter(
                         socket.getOutputStream(), true);
                 BufferedReader userInput = new BufferedReader(
                         new InputStreamReader(System.in))) {

                String msg;
                // 从控制台读取用户输入，发送给服务器
                while ((msg = userInput.readLine()) != null) {
                    out.println(msg); // 发送消息
                    // 读取服务器回复
                    String serverReply = in.readLine();
                    System.out.println("服务器：" + serverReply);

                    // 若输入 "bye"，断开连接
                    if ("bye".equals(msg)) {
                        break;
                    }
                }
            }
            System.out.println("已断开与服务器的连接");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```  


**运行流程**：  
1. 先启动服务器（`TcpServer`），控制台显示“服务器启动，监听端口：8888”。  
2. 启动客户端（`TcpClient`），控制台显示“已连接到服务器”。  
3. 客户端控制台输入消息（如“Hello”），服务器收到后回复“服务器已收到：Hello”。  
4. 输入“bye”，双方断开连接。  


#### 多线程服务器（处理多个客户端）  
上述服务器是单线程的，只能处理一个客户端。实际应用中需用**多线程**或**线程池**同时处理多个客户端：  

```java
// 多线程服务器：为每个客户端连接创建一个线程
public class TcpMultiThreadServer {
    public static void main(String[] args) throws IOException {
        int port = 8888;
        try (ServerSocket serverSocket = new ServerSocket(port)) {
            System.out.println("多线程服务器启动，监听端口：" + port);
            while (true) { // 循环接受客户端连接
                Socket clientSocket = serverSocket.accept();
                // 启动新线程处理客户端
                new Thread(() -> handleClient(clientSocket)).start();
            }
        }
    }

    // 处理单个客户端的逻辑（复用之前的通信代码）
    private static void handleClient(Socket clientSocket) {
        try (BufferedReader in = new BufferedReader(
                new InputStreamReader(clientSocket.getInputStream()));
             PrintWriter out = new PrintWriter(
                     clientSocket.getOutputStream(), true)) {

            String clientMsg;
            while ((clientMsg = in.readLine()) != null) {
                System.out.println("客户端 " + clientSocket.getInetAddress() + "：" + clientMsg);
                out.println("服务器已收到：" + clientMsg);
                if ("bye".equals(clientMsg)) break;
            }
            System.out.println("客户端 " + clientSocket.getInetAddress() + " 已断开");
        } catch (IOException e) {
            e.printStackTrace();
        } finally {
            try {
                clientSocket.close(); // 关闭客户端连接
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
    }
}
```  


### 三、UDP 编程（高效传输）  
UDP 通信无需建立连接（类似“发快递”），直接发送数据报（`DatagramPacket`），适合实时性要求高的场景。  

流程：  
1. 发送方：创建 `DatagramSocket`，封装数据到 `DatagramPacket`（包含目标 IP 和端口），发送数据。  
2. 接收方：创建 `DatagramSocket` 绑定端口，通过 `DatagramPacket` 接收数据。  


#### 核心类  
- `DatagramSocket`：用于发送和接收数据报的套接字。  
- `DatagramPacket`：数据报载体，包含数据、数据长度、目标 IP 和端口（发送时）或源 IP 和端口（接收时）。  


#### 示例：UDP 发送方与接收方通信  
**1. 接收方（监听端口，接收数据报）**  
```java
import java.net.DatagramPacket;
import java.net.DatagramSocket;

public class UdpReceiver {
    public static void main(String[] args) {
        int port = 9999; // 监听端口
        byte[] buffer = new byte[1024]; // 缓冲区（存储接收的数据）

        try (DatagramSocket socket = new DatagramSocket(port)) { // 绑定端口
            System.out.println("UDP 接收方启动，监听端口：" + port);

            while (true) {
                // 创建数据报（用于接收数据）
                DatagramPacket packet = new DatagramPacket(buffer, buffer.length);
                // 接收数据（阻塞方法，直到收到数据报）
                socket.receive(packet);

                // 解析数据（将字节转为字符串）
                String msg = new String(packet.getData(), 0, packet.getLength());
                System.out.println("收到来自 " + packet.getAddress() + ":" + packet.getPort() + " 的消息：" + msg);

                // 若收到 "bye"，退出
                if ("bye".equals(msg)) {
                    break;
                }
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```  


**2. 发送方（发送数据报到指定地址和端口）**  
```java
import java.net.DatagramPacket;
import java.net.DatagramSocket;
import java.net.InetAddress;
import java.util.Scanner;

public class UdpSender {
    public static void main(String[] args) {
        String targetIp = "localhost"; // 目标 IP
        int targetPort = 9999; // 目标端口

        try (DatagramSocket socket = new DatagramSocket()) { // 不绑定固定端口（系统自动分配）
            Scanner scanner = new Scanner(System.in);
            while (true) {
                // 从控制台读取输入
                System.out.print("请输入消息：");
                String msg = scanner.nextLine();

                // 将消息转为字节数组
                byte[] data = msg.getBytes();
                // 创建数据报（包含数据、长度、目标 IP 和端口）
                DatagramPacket packet = new DatagramPacket(
                        data, data.length,
                        InetAddress.getByName(targetIp),
                        targetPort
                );

                // 发送数据报
                socket.send(packet);

                // 若输入 "bye"，退出
                if ("bye".equals(msg)) {
                    break;
                }
            }
            scanner.close();
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```  


**运行流程**：  
1. 启动接收方（`UdpReceiver`），控制台显示“UDP 接收方启动，监听端口：9999”。  
2. 启动发送方（`UdpSender`），输入消息（如“Hi UDP”），接收方会显示消息来源和内容。  
3. 输入“bye”，双方退出。  


### 四、Java NIO 网络编程（非阻塞 IO）  
传统 Socket 编程是**阻塞 IO**（`accept()`、`read()` 等方法会阻塞线程），在高并发场景下效率低。Java NIO（New IO）提供**非阻塞 IO** 机制，通过 `Selector`（选择器）、`SocketChannel`（通道）实现单线程处理多个连接，适合高性能网络应用（如服务器）。  


#### 核心组件  
- `Selector`：监控多个 `Channel` 的事件（如连接就绪、读就绪、写就绪），实现单线程处理多通道。  
- `SocketChannel`：TCP 通道，支持非阻塞读写。  
- `ServerSocketChannel`：TCP 服务器通道，支持非阻塞监听连接。  


#### 简单示例（NIO 服务器）  
```java
import java.io.IOException;
import java.net.InetSocketAddress;
import java.nio.ByteBuffer;
import java.nio.channels.*;
import java.util.Iterator;
import java.util.Set;

public class NioServer {
    public static void main(String[] args) throws IOException {
        // 1. 创建服务器通道并绑定端口
        ServerSocketChannel serverChannel = ServerSocketChannel.open();
        serverChannel.socket().bind(new InetSocketAddress(8888));
        serverChannel.configureBlocking(false); // 设置为非阻塞

        // 2. 创建选择器并注册服务器通道（关注“接受连接”事件）
        Selector selector = Selector.open();
        serverChannel.register(selector, SelectionKey.OP_ACCEPT);

        System.out.println("NIO 服务器启动，监听端口：8888");

        while (true) {
            // 3. 等待事件就绪（阻塞，可设置超时时间）
            selector.select();

            // 4. 获取就绪事件的集合
            Set<SelectionKey> keys = selector.selectedKeys();
            Iterator<SelectionKey> iterator = keys.iterator();

            while (iterator.hasNext()) {
                SelectionKey key = iterator.next();
                iterator.remove(); // 移除已处理的事件

                // 5. 处理事件
                if (key.isAcceptable()) { // 接受连接事件
                    ServerSocketChannel server = (ServerSocketChannel) key.channel();
                    SocketChannel clientChannel = server.accept(); // 接受客户端连接
                    clientChannel.configureBlocking(false); // 客户端通道设为非阻塞
                    // 注册客户端通道到选择器，关注“读”事件
                    clientChannel.register(selector, SelectionKey.OP_READ, ByteBuffer.allocate(1024));
                    System.out.println("客户端连接：" + clientChannel.getRemoteAddress());
                } else if (key.isReadable()) { // 读事件（客户端发送数据）
                    SocketChannel clientChannel = (SocketChannel) key.channel();
                    ByteBuffer buffer = (ByteBuffer) key.attachment(); // 获取缓冲区
                    int bytesRead = clientChannel.read(buffer); // 读取数据

                    if (bytesRead > 0) {
                        buffer.flip(); // 切换为读模式
                        String msg = new String(buffer.array(), 0, bytesRead);
                        System.out.println("收到消息：" + msg);

                        // 回复客户端
                        buffer.clear();
                        buffer.put(("服务器已收到：" + msg).getBytes());
                        buffer.flip();
                        clientChannel.write(buffer);
                    } else if (bytesRead == -1) { // 客户端断开连接
                        clientChannel.close();
                        System.out.println("客户端断开连接");
                    }
                }
            }
        }
    }
}
```  


### 五、TCP 与 UDP 的对比及适用场景  
| 特性         | TCP                                  | UDP                                  |  
|--------------|--------------------------------------|--------------------------------------|  
| 连接方式     | 面向连接（三次握手建立连接）          | 无连接（直接发送）                    |  
| 可靠性       | 可靠（重传丢失数据，保证顺序）        | 不可靠（不重传，可能丢失或乱序）      |  
| 速度         | 较慢（需确认和重传）                  | 较快（无额外开销）                    |  
| 数据大小     | 无限制（通过流传输）                  | 有限制（数据报最大 64KB）             |  
| 适用场景     | 文件传输、登录、HTTP/HTTPS 等        | 视频通话、语音聊天、广播、游戏等      |  


### 六、注意事项  
1. **端口占用**：确保使用的端口未被其他程序占用（可通过 `netstat -ano` 查看端口占用）。  
2. **异常处理**：网络操作可能抛出 `IOException`（如连接超时、端口被占用），需妥善处理。  
3. **资源关闭**：使用 `try-with-resources` 自动关闭 `Socket`、`ServerSocket` 等资源，避免资源泄漏。  
4. **多线程安全**：多线程服务器需注意共享资源的同步（如使用锁或线程安全集合）。  


### 总结  
Java 网络编程基于 TCP 和 UDP 协议，核心是 Socket 编程：  
- **TCP** 适合可靠传输场景，通过 `ServerSocket` 和 `Socket` 实现，需建立连接。  
- **UDP** 适合高效实时场景，通过 `DatagramSocket` 和 `DatagramPacket` 实现，无需连接。  
- 高并发场景可使用 Java NIO 的非阻塞机制，通过 `Selector` 单线程处理多连接。  

掌握网络编程是实现分布式系统、客户端-服务器通信的基础，实际开发中可结合框架（如 Netty）简化复杂场景的开发。
