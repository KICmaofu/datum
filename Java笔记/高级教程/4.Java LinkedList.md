Java 中的 `LinkedList` 是集合框架（`java.util`）中 `List` 接口的实现类，同时也实现了 `Deque` 接口（双端队列）。它底层基于**双向链表**数据结构，因此具有插入/删除效率高、随机访问效率低的特点，同时支持作为列表、队列、栈等多种数据结构使用。


### 一、LinkedList 基本特性  
- **底层结构**：双向链表（每个节点包含 `prev`（前驱节点引用）、`next`（后继节点引用）和存储的数据 `item`）。  
- **元素允许重复**：可以存储重复元素。  
- **允许 null 值**：支持添加 `null` 元素。  
- **非线程安全**：多线程环境下需手动同步（如使用 `Collections.synchronizedList()`）。  
- **有序集合**：元素的插入顺序与遍历顺序一致。  


### 二、双向链表结构解析  
`LinkedList` 的核心是双向链表节点（`Node`），每个节点包含三个部分：  
- `item`：存储的元素值；  
- `prev`：指向当前节点的前一个节点（前驱）；  
- `next`：指向当前节点的后一个节点（后继）。  

链表的头部（`first`）和尾部（`last`）节点分别作为入口，通过节点间的引用串联所有元素。这种结构使得：  
- **插入/删除中间元素**时，只需修改相邻节点的引用（无需移动其他元素），效率高（时间复杂度 `O(1)`，前提是已找到目标位置）；  
- **随机访问元素**（如通过索引 `get(index)`）时，需从头部或尾部遍历到目标位置，效率低（时间复杂度 `O(n)`）。  


### 三、核心方法与使用示例  
`LinkedList` 继承了 `List` 和 `Deque` 的方法，支持列表操作（增删改查）和双端队列操作（首尾元素操作）。


#### 1. 初始化  
```java
import java.util.LinkedList;
import java.util.List;

// 1. 创建空的 LinkedList
List<String> list = new LinkedList<>();

// 2. 从其他集合初始化
List<Integer> numbers = new LinkedList<>(List.of(1, 2, 3)); // JDK 9+ 支持
```  


#### 2. 基本增删改查（List 接口方法）  
| 方法                | 作用                                  | 示例                                  |  
|---------------------|---------------------------------------|---------------------------------------|  
| `add(E e)`          | 尾部添加元素                          | `list.add("a");`                      |  
| `add(int index, E e)` | 指定索引插入元素                      | `list.add(1, "b");`  // 在索引1处插入"b" |  
| `remove(int index)` | 删除指定索引元素并返回                | `String removed = list.remove(0);`    |  
| `remove(Object o)`  | 删除首个匹配元素（equals()判断）      | `list.remove("a");`                   |  
| `get(int index)`    | 获取指定索引元素                      | `String elem = list.get(2);`          |  
| `set(int index, E e)` | 修改指定索引元素并返回旧值            | `String old = list.set(0, "new");`    |  
| `size()`            | 返回元素个数                          | `int size = list.size();`             |  


#### 3. 双端队列操作（Deque 接口方法）  
由于实现了 `Deque`，`LinkedList` 可作为**队列**（FIFO）或**栈**（LIFO）使用：  

| 方法                | 作用                                  | 队列/栈场景                           |  
|---------------------|---------------------------------------|---------------------------------------|  
| `addFirst(E e)`     | 头部添加元素（栈的 `push` 操作）      | 栈：`push(e)` 等价于 `addFirst(e)`    |  
| `addLast(E e)`      | 尾部添加元素（队列的 `offer` 操作）   | 队列：`offer(e)` 等价于 `addLast(e)`  |  
| `removeFirst()`     | 删除并返回头部元素（栈的 `pop`）      | 栈：`pop()` 等价于 `removeFirst()`    |  
| `removeLast()`      | 删除并返回尾部元素                    | -                                     |  
| `getFirst()`        | 获取头部元素（不删除）                | -                                     |  
| `getLast()`         | 获取尾部元素（不删除）                | -                                     |  


#### 4. 遍历方式  
`LinkedList` 支持多种遍历方式，推荐使用迭代器或增强 for 循环（避免随机访问的性能损耗）：  

```java
LinkedList<String> list = new LinkedList<>(List.of("a", "b", "c"));

// 1. 增强 for 循环（推荐）
for (String s : list) {
    System.out.println(s);
}

// 2. 迭代器（支持删除操作）
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
    if (s.equals("b")) {
        it.remove(); // 安全删除当前元素
    }
}

// 3. 列表迭代器（支持双向遍历和添加/修改）
ListIterator<String> lit = list.listIterator();
while (lit.hasNext()) {
    System.out.println(lit.next()); // 正向遍历
}
while (lit.hasPrevious()) {
    System.out.println(lit.previous()); // 反向遍历
}
```  


### 四、LinkedList 与 ArrayList 的对比  
| 特性                | LinkedList                            | ArrayList                              |  
|---------------------|---------------------------------------|----------------------------------------|  
| 底层结构            | 双向链表                              | 动态数组（扩容时复制元素）             |  
| 随机访问（`get(index)`） | 效率低（`O(n)`，需遍历）              | 效率高（`O(1)`，直接索引访问）         |  
| 插入/删除中间元素    | 效率高（`O(1)`，仅改节点引用）        | 效率低（`O(n)`，需移动后续元素）       |  
| 插入/删除首尾元素    | 效率高（`O(1)`）                      | 尾部添加效率高（`O(1)`），头部插入效率低（`O(n)`） |  
| 内存占用            | 较高（每个节点需存储前后引用）        | 较低（仅存储元素，可能有冗余空间）     |  


### 五、适用场景  
- 频繁在**中间位置插入/删除元素**（如实现链表式数据结构）；  
- 需要作为**队列**（FIFO）或**栈**（LIFO）使用（利用 `Deque` 接口方法）；  
- 不频繁进行随机访问（如很少用 `get(index)`）。  


### 总结  
`LinkedList` 是基于双向链表的集合类，优势在于灵活的插入/删除操作和多数据结构支持（列表、队列、栈），但随机访问性能较差。实际开发中需根据业务场景（访问 vs 增删频率）选择 `LinkedList` 或 `ArrayList`。
