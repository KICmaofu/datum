Java 中的 **Iterator（迭代器）** 是集合框架（`java.util`）中用于**遍历集合元素**的接口，它提供了一种统一的方式访问集合中的元素，而无需暴露集合的底层数据结构（如数组、链表等）。迭代器模式是 Java 中“迭代器设计模式”的典型实现，旨在简化不同集合的遍历逻辑。


### 一、Iterator 基本概念  
- **作用**：遍历集合（如 `List`、`Set` 等）中的元素，支持在遍历过程中安全删除元素。  
- **统一接口**：无论集合底层是数组（`ArrayList`）、链表（`LinkedList`）还是哈希表（`HashSet`），都可通过相同的迭代器方法遍历，降低代码耦合度。  
- **所属包**：`java.util.Iterator`，所有集合类（`Collection` 的实现类）都通过 `iterator()` 方法提供迭代器实例。  


### 二、Iterator 核心方法  
`Iterator` 接口定义了 3 个核心方法，用于控制遍历流程：  

| 方法                | 作用                                                                 | 返回值/异常                          |  
|---------------------|----------------------------------------------------------------------|---------------------------------------|  
| `boolean hasNext()` | 判断集合中是否还有未遍历的元素（是否可以调用 `next()`）。             | `true`（有元素）/ `false`（已遍历完） |  
| `E next()`          | 获取下一个元素，并将迭代器指针向后移动一位。                         | 返回下一个元素 `E`；若已无元素，抛出 `NoSuchElementException`。 |  
| `void remove()`     | 删除**上一次 `next()` 方法返回的元素**（可选操作，并非所有迭代器都支持）。 | 无返回值；若未调用 `next()` 就调用 `remove()`，抛出 `IllegalStateException`。 |  


### 三、迭代器的工作原理  
迭代器的本质是一个“指针”，指向集合中的元素，遍历过程如下：  
1. 通过集合的 `iterator()` 方法获取迭代器实例（与当前集合绑定）。  
2. 调用 `hasNext()` 检查是否有未遍历的元素：  
   - 若有，调用 `next()` 获取元素，同时指针后移。  
   - 若无，遍历结束。  
3. （可选）调用 `remove()` 删除上一次 `next()` 获取的元素（仅能在 `next()` 之后调用，且每次 `next()` 后只能调用一次 `remove()`）。  


### 四、基本使用示例  
迭代器可遍历所有实现 `Collection` 接口的集合（如 `ArrayList`、`HashSet`、`LinkedList` 等），使用流程一致。  


#### 1. 遍历 List 集合（以 `ArrayList` 为例）  
```java
import java.util.ArrayList;
import java.util.Iterator;
import java.util.List;

public class IteratorDemo {
    public static void main(String[] args) {
        List<String> list = new ArrayList<>();
        list.add("A");
        list.add("B");
        list.add("C");

        // 1. 获取迭代器实例
        Iterator<String> iterator = list.iterator();

        // 2. 遍历元素
        while (iterator.hasNext()) { // 检查是否有下一个元素
            String element = iterator.next(); // 获取下一个元素
            System.out.println(element); // 输出：A、B、C
        }
    }
}
```  


#### 2. 遍历 Set 集合（以 `HashSet` 为例）  
```java
import java.util.HashSet;
import java.util.Iterator;
import java.util.Set;

public class IteratorSetDemo {
    public static void main(String[] args) {
        Set<Integer> set = new HashSet<>();
        set.add(1);
        set.add(2);
        set.add(3);

        Iterator<Integer> iterator = set.iterator();
        while (iterator.hasNext()) {
            Integer num = iterator.next();
            System.out.println(num); // 输出：1、2、3（顺序不确定，因 HashSet 无序）
        }
    }
}
```  


#### 3. 遍历中删除元素（安全删除）  
迭代器的 `remove()` 方法是遍历过程中删除元素的**唯一安全方式**（避免 `ConcurrentModificationException` 异常）。  

```java
import java.util.ArrayList;
import java.util.Iterator;
import java.util.List;

public class IteratorRemoveDemo {
    public static void main(String[] args) {
        List<String> list = new ArrayList<>(List.of("A", "B", "C", "D"));

        Iterator<String> iterator = list.iterator();
        while (iterator.hasNext()) {
            String element = iterator.next();
            if (element.equals("B")) {
                // 删除上一次 next() 返回的元素（即 "B"）
                iterator.remove(); 
            }
        }

        System.out.println(list); // 输出：[A, C, D]
    }
}
```  


### 五、迭代器的“快速失败”机制（Fail-Fast）  
迭代器在遍历过程中，若集合被**结构性修改**（如通过集合的 `add()`、`remove()` 方法添加/删除元素），会立即抛出 `ConcurrentModificationException` 异常，这就是“快速失败”机制。  

**原因**：迭代器内部维护一个“修改次数计数器”，集合被修改时计数器递增，迭代器每次操作会检查计数器是否与初始值一致，不一致则抛出异常。  

**示例：错误的删除方式（抛出异常）**  
```java
List<String> list = new ArrayList<>(List.of("A", "B", "C"));

// 错误：通过集合的 remove() 方法删除元素，触发 ConcurrentModificationException
for (String s : list) { // 增强 for 循环底层是迭代器
    if (s.equals("B")) {
        list.remove(s); // 直接修改集合，导致迭代器检测到不一致
    }
}
```  

**注意**：迭代器的 `remove()` 方法不会触发异常（因为它会同步更新计数器）。  


### 六、迭代器与其他遍历方式的对比  
| 遍历方式         | 特点                                  | 适用场景                          |  
|------------------|---------------------------------------|-----------------------------------|  
| 迭代器（Iterator） | 统一接口，支持安全删除，适配所有集合 | 需在遍历中删除元素；遍历 `Set` 等无索引集合 |  
| 增强 for 循环（foreach） | 语法简洁，底层是迭代器（本质是迭代器的语法糖） | 仅遍历元素，无需删除或修改集合顺序 |  
| 普通 for 循环（索引） | 通过索引访问，仅适用于 `List`（有索引） | 需按索引操作元素（如 `ArrayList` 随机访问高效） |  


### 七、ListIterator 扩展迭代器  
`ListIterator` 是 `Iterator` 的子接口，仅适用于 `List` 集合，新增了**双向遍历**和**添加/修改元素**的功能：  

| 方法                  | 作用                                  |  
|-----------------------|---------------------------------------|  
| `boolean hasPrevious()` | 判断是否有前一个元素（反向遍历）      |  
| `E previous()`        | 获取前一个元素，指针向前移动          |  
| `void add(E e)`       | 在当前指针位置插入元素                |  
| `void set(E e)`       | 修改上一次 `next()` 或 `previous()` 返回的元素 |  


**示例：双向遍历 `List`**  
```java
import java.util.ArrayList;
import java.util.List;
import java.util.ListIterator;

public class ListIteratorDemo {
    public static void main(String[] args) {
        List<String> list = new ArrayList<>(List.of("A", "B", "C"));

        ListIterator<String> lit = list.listIterator();

        // 正向遍历
        System.out.println("正向遍历：");
        while (lit.hasNext()) {
            System.out.println(lit.next()); // A、B、C
        }

        // 反向遍历（指针已在末尾，可向前移动）
        System.out.println("反向遍历：");
        while (lit.hasPrevious()) {
            System.out.println(lit.previous()); // C、B、A
        }
    }
}
```  


### 八、总结  
迭代器（`Iterator`）是 Java 集合遍历的核心工具，其核心优势：  
1. **统一性**：提供一致的遍历接口，无关集合底层结构（数组、链表等）。  
2. **安全性**：支持在遍历中通过 `remove()` 方法安全删除元素，避免并发修改异常。  
3. **灵活性**：`ListIterator` 扩展支持双向遍历和元素修改，适配 `List` 集合的复杂需求。  

使用迭代器时需注意：遍历过程中不可通过集合方法修改集合（否则触发 `ConcurrentModificationException`），删除元素必须使用迭代器的 `remove()` 方法。掌握迭代器是高效操作集合的基础。
