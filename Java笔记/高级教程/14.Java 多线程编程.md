Java 多线程编程是指通过创建多个线程并发执行任务，以提高程序效率（尤其是多核CPU环境下），适用于IO密集型（如网络请求、文件读写）或计算密集型场景。Java 提供了完善的多线程支持，核心围绕 `Thread` 类、`Runnable` 接口及并发工具类展开。


### 一、基本概念  
- **进程（Process）**：操作系统分配资源的基本单位（如一个运行中的Java程序）。  
- **线程（Thread）**：进程内的执行单元，共享进程资源，是CPU调度的基本单位。一个进程可包含多个线程。  
- **并发（Concurrency）**：多个线程交替执行（CPU快速切换），看似同时运行。  
- **并行（Parallelism）**：多个线程在多核CPU上同时运行。  


### 二、线程的创建方式  
Java 中创建线程有 3 种核心方式：**继承 `Thread` 类**、**实现 `Runnable` 接口**、**实现 `Callable` 接口（带返回值）**。  


#### 1. 继承 `Thread` 类  
`Thread` 类是线程的基类，通过重写其 `run()` 方法定义线程任务。  

**示例**：  
```java
// 1. 继承 Thread 类
class MyThread extends Thread {
    // 2. 重写 run() 方法：线程执行的任务
    @Override
    public void run() {
        for (int i = 0; i < 5; i++) {
            System.out.println("线程 " + Thread.currentThread().getName() + "：" + i);
            try {
                Thread.sleep(500); // 休眠 500ms（模拟任务耗时）
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
}

public class ThreadDemo {
    public static void main(String[] args) {
        // 3. 创建线程实例
        MyThread thread1 = new MyThread();
        MyThread thread2 = new MyThread();
        
        // 4. 启动线程（调用 start()，而非直接调用 run()）
        thread1.start(); // 启动线程1，JVM会调用 run()
        thread2.start(); // 启动线程2
    }
}
```  

**输出（顺序可能不同，因线程并发）**：  
```
线程 Thread-0：0
线程 Thread-1：0
线程 Thread-0：1
线程 Thread-1：1
...
```  

**注意**：  
- 必须调用 `start()` 启动线程（JVM会创建新线程并执行 `run()`），直接调用 `run()` 会以普通方法在当前线程执行。  
- 缺点：Java 单继承，继承 `Thread` 后无法再继承其他类。  


#### 2. 实现 `Runnable` 接口  
`Runnable` 是函数式接口（仅含 `run()` 方法），通过实现该接口定义任务，再传入 `Thread` 类启动线程。  

**示例**：  
```java
// 1. 实现 Runnable 接口
class MyRunnable implements Runnable {
    // 2. 实现 run() 方法：线程任务
    @Override
    public void run() {
        for (int i = 0; i < 5; i++) {
            System.out.println("线程 " + Thread.currentThread().getName() + "：" + i);
            try {
                Thread.sleep(500);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
}

public class RunnableDemo {
    public static void main(String[] args) {
        // 2. 创建任务实例
        MyRunnable task = new MyRunnable();
        
        // 3. 将任务传入 Thread 并启动
        Thread thread1 = new Thread(task, "线程A"); // 可指定线程名
        Thread thread2 = new Thread(task, "线程B");
        thread1.start();
        thread2.start();
    }
}
```  

**优点**：  
- 避免单继承限制（可同时实现其他接口）。  
- 适合多线程共享同一个任务实例（如共享资源）。  


#### 3. 实现 `Callable` 接口（带返回值）  
`Callable` 接口（JDK 5+）支持线程执行后返回结果，需配合 `Future` 或 `FutureTask` 获取结果。  

**示例**：  
```java
import java.util.concurrent.Callable;
import java.util.concurrent.ExecutionException;
import java.util.concurrent.FutureTask;

// 1. 实现 Callable 接口（泛型指定返回值类型）
class MyCallable implements Callable<Integer> {
    private int num;
    public MyCallable(int num) {
        this.num = num;
    }

    // 2. 实现 call() 方法（有返回值，可抛异常）
    @Override
    public Integer call() throws Exception {
        int sum = 0;
        for (int i = 1; i <= num; i++) {
            sum += i;
        }
        return sum; // 返回 1+2+...+num 的和
    }
}

public class CallableDemo {
    public static void main(String[] args) throws ExecutionException, InterruptedException {
        // 2. 创建 Callable 任务
        MyCallable task = new MyCallable(100);
        
        // 3. 用 FutureTask 包装任务（兼具 Runnable 和 Future 功能）
        FutureTask<Integer> futureTask = new FutureTask<>(task);
        
        // 4. 启动线程
        new Thread(futureTask).start();
        
        // 5. 获取结果（get() 会阻塞，直到任务完成）
        int result = futureTask.get();
        System.out.println("1+2+...+100 = " + result); // 输出：5050
    }
}
```  

**适用场景**：需要线程返回执行结果的场景（如计算任务）。  


### 三、线程的生命周期  
线程从创建到销毁经历 5 种状态，状态转换由 JVM 和线程方法控制：  

1. **新建（New）**：创建 `Thread` 实例后，未调用 `start()` 前的状态。  
2. **就绪（Runnable）**：调用 `start()` 后，线程进入就绪队列，等待 CPU 调度。  
3. **运行（Running）**：CPU 调度后，执行 `run()` 方法的状态。  
4. **阻塞（Blocked/Waiting/Timed Waiting）**：线程暂停执行，如：  
   - `sleep(long)`：进入**计时等待**（指定时间后自动就绪）。  
   - `wait()`：进入**等待**（需其他线程 `notify()` 唤醒）。  
   - 争夺同步锁失败：进入**阻塞**（获取锁后就绪）。  
5. **死亡（Terminated）**：`run()` 执行完毕，或调用 `stop()`（已过时，不推荐）、发生未捕获异常。  


### 四、线程同步（解决线程安全问题）  
当多个线程共享资源（如全局变量、对象属性）时，可能因并发操作导致**数据不一致**（线程安全问题）。需通过同步机制保证资源访问的原子性。  


#### 1. 线程安全问题示例  
```java
// 共享资源：计数器
class Counter {
    private int count = 0;

    // 非同步方法：多线程并发调用时可能出错
    public void increment() {
        count++; // 看似简单，实际包含 3 步：读值→加1→写回，可能被打断
    }

    public int getCount() {
        return count;
    }
}

public class ThreadUnsafeDemo {
    public static void main(String[] args) throws InterruptedException {
        Counter counter = new Counter();
        int threadNum = 1000; // 1000 个线程同时自增

        // 创建 1000 个线程，每个线程自增 1000 次
        Thread[] threads = new Thread[threadNum];
        for (int i = 0; i < threadNum; i++) {
            threads[i] = new Thread(() -> {
                for (int j = 0; j < 1000; j++) {
                    counter.increment();
                }
            });
            threads[i].start();
        }

        // 等待所有线程执行完毕
        for (Thread t : threads) {
            t.join(); // 主线程等待子线程完成
        }

        // 预期结果：1000*1000=1000000，实际可能小于 1000000（线程不安全）
        System.out.println("最终计数：" + counter.getCount());
    }
}
```  


#### 2. 同步解决方法  
##### （1）`synchronized` 关键字  
保证方法或代码块在同一时间只能被一个线程执行，底层依赖“对象锁”实现。  

- **同步方法**：锁是当前对象（`this`）或类对象（静态方法）。  
  ```java
  public synchronized void increment() { // 同步方法
      count++;
  }
  ```  

- **同步代码块**：锁是指定对象（更灵活，可缩小同步范围）。  
  ```java
  public void increment() {
      synchronized (this) { // 同步代码块，锁为当前对象
          count++;
      }
  }
  ```  


##### （2）`Lock` 接口（JDK 5+）  
`java.util.concurrent.locks.Lock` 是更灵活的同步机制，需手动获取和释放锁（推荐配合 `try-finally` 使用）。  

**示例（`ReentrantLock`，可重入锁）**：  
```java
import java.util.concurrent.locks.Lock;
import java.util.concurrent.locks.ReentrantLock;

class Counter {
    private int count = 0;
    private Lock lock = new ReentrantLock(); // 创建锁对象

    public void increment() {
        lock.lock(); // 获取锁
        try {
            count++; // 临界区（需同步的代码）
        } finally {
            lock.unlock(); // 释放锁（必须在 finally 中，避免死锁）
        }
    }

    public int getCount() {
        return count;
    }
}
```  

**`Lock` vs `synchronized`**：  
- `Lock` 可手动控制锁的获取/释放，支持超时获取锁、中断获取锁。  
- `synchronized` 由 JVM 自动管理锁（进入同步块时获取，退出时释放），更简洁。  


##### （3）原子类（`java.util.concurrent.atomic`）  
针对基本类型和引用类型的原子操作类（如 `AtomicInteger`、`AtomicLong`），通过 CAS（Compare-And-Swap）机制实现无锁同步，性能优于锁。  

**示例**：  
```java
import java.util.concurrent.atomic.AtomicInteger;

class Counter {
    private AtomicInteger count = new AtomicInteger(0); // 原子整数

    public void increment() {
        count.incrementAndGet(); // 原子自增（线程安全）
    }

    public int getCount() {
        return count.get();
    }
}
```  


### 五、线程通信  
多线程协作时（如生产者-消费者模型），需通过通信机制协调线程执行顺序，核心依赖 `Object` 类的 `wait()`、`notify()`、`notifyAll()` 方法（需在同步块/方法中使用）。  


#### 生产者-消费者模型示例  
```java
import java.util.LinkedList;
import java.util.Queue;

// 共享缓冲区（仓库）
class Warehouse {
    private Queue<Integer> queue = new LinkedList<>();
    private static final int MAX_SIZE = 5; // 仓库最大容量

    // 生产者：向仓库添加商品
    public synchronized void produce(int num) throws InterruptedException {
        // 若仓库满，等待（释放锁，让消费者消费）
        while (queue.size() == MAX_SIZE) {
            System.out.println("仓库已满，生产者等待...");
            wait(); // 释放锁，进入等待状态
        }
        queue.add(num);
        System.out.println("生产：" + num + "，当前库存：" + queue.size());
        notifyAll(); // 唤醒所有等待的线程（消费者）
    }

    // 消费者：从仓库取商品
    public synchronized int consume() throws InterruptedException {
        // 若仓库空，等待（释放锁，让生产者生产）
        while (queue.isEmpty()) {
            System.out.println("仓库为空，消费者等待...");
            wait(); // 释放锁，进入等待状态
        }
        int num = queue.poll();
        System.out.println("消费：" + num + "，当前库存：" + queue.size());
        notifyAll(); // 唤醒所有等待的线程（生产者）
        return num;
    }
}

// 生产者线程
class Producer extends Thread {
    private Warehouse warehouse;
    public Producer(Warehouse warehouse) {
        this.warehouse = warehouse;
    }
    @Override
    public void run() {
        for (int i = 1; i <= 10; i++) {
            try {
                warehouse.produce(i);
                Thread.sleep(500); // 模拟生产耗时
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
}

// 消费者线程
class Consumer extends Thread {
    private Warehouse warehouse;
    public Consumer(Warehouse warehouse) {
        this.warehouse = warehouse;
    }
    @Override
    public void run() {
        for (int i = 0; i < 10; i++) {
            try {
                warehouse.consume();
                Thread.sleep(1000); // 模拟消费耗时
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
}

public class ProducerConsumerDemo {
    public static void main(String[] args) {
        Warehouse warehouse = new Warehouse();
        new Producer(warehouse).start();
        new Consumer(warehouse).start();
    }
}
```  


### 六、线程池（`Executor` 框架）  
频繁创建/销毁线程会消耗资源，**线程池**通过复用线程提高效率，是实际开发的首选（而非直接创建 `Thread`）。  


#### 核心接口与类  
- `Executor`：线程池顶级接口，定义 `execute(Runnable)` 方法。  
- `ExecutorService`：扩展 `Executor`，支持关闭线程池、提交 `Callable` 任务等。  
- `ThreadPoolExecutor`：线程池核心实现类，可自定义参数。  
- `Executors`：工具类，提供常用线程池的快捷创建方法（但部分方法存在隐患，推荐手动创建 `ThreadPoolExecutor`）。  


#### 线程池示例（`ThreadPoolExecutor`）  
```java
import java.util.concurrent.*;

public class ThreadPoolDemo {
    public static void main(String[] args) {
        // 核心参数：
        int corePoolSize = 2; // 核心线程数（始终保持的线程数）
        int maximumPoolSize = 5; // 最大线程数
        long keepAliveTime = 60; // 非核心线程空闲超时时间
        TimeUnit unit = TimeUnit.SECONDS; // 超时时间单位
        BlockingQueue<Runnable> workQueue = new LinkedBlockingQueue<>(10); // 任务队列
        ThreadFactory threadFactory = Executors.defaultThreadFactory(); // 线程创建工厂
        RejectedExecutionHandler handler = new ThreadPoolExecutor.AbortPolicy(); // 任务拒绝策略

        // 创建线程池
        ExecutorService executor = new ThreadPoolExecutor(
                corePoolSize, maximumPoolSize, keepAliveTime, unit,
                workQueue, threadFactory, handler
        );

        // 提交任务（10个任务）
        for (int i = 0; i < 10; i++) {
            int taskNum = i;
            executor.execute(() -> { // 提交 Runnable 任务
                System.out.println("线程 " + Thread.currentThread().getName() + " 执行任务 " + taskNum);
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            });
        }

        // 关闭线程池（不再接受新任务，等待现有任务完成）
        executor.shutdown();
    }
}
```  


### 七、常见问题与最佳实践  
1. **死锁**：两个线程互相持有对方需要的锁，导致无限等待。避免方式：按固定顺序获取锁、设置锁超时。  
2. **线程泄露**：线程未正确终止（如阻塞在 `wait()` 未被唤醒），导致资源浪费。  
3. **减少同步范围**：只同步必要的代码（如用同步代码块替代同步方法），提高并发效率。  
4. **优先使用线程池**：避免手动创建线程，通过线程池管理线程生命周期。  
5. **慎用 `stop()`、`suspend()`**：已过时，可能导致资源未释放，推荐用 `interrupt()` 中断线程。  


### 总结  
Java 多线程编程的核心是**线程创建**、**同步机制**、**线程通信**和**线程池管理**。合理使用多线程可充分利用CPU资源，提升程序效率，但需注意线程安全问题（通过 `synchronized`、`Lock` 或原子类解决）。实际开发中，线程池是管理线程的首选方式，配合并发工具类（如 `CountDownLatch`、`Semaphore`）可实现复杂的多线程协作。
