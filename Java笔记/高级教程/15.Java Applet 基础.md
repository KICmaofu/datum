Java Applet 是一种**嵌入在网页中运行的 Java 小程序**，由 `java.applet.Applet` 类（或 `javax.swing.JApplet` 类，基于 Swing）实现，主要用于增强网页的交互性和动态效果。它是 Java 早期的重要技术，但随着浏览器安全限制收紧、HTML5 等技术的兴起，目前已基本被淘汰。不过了解其基础原理，有助于理解 Java 早期的跨平台思想。


### 一、Applet 的核心特点  
1. **跨平台性**：依赖 Java 虚拟机（JVM）运行，一次编写可在任何支持 Java 的浏览器中运行（“一次编写，到处运行”）。  
2. **嵌入网页**：通过 HTML 标签（`<applet>`）嵌入网页，由浏览器的 Java 插件加载执行。  
3. **安全沙箱**：受严格的安全限制（如不能访问本地文件系统、不能连接非来源服务器、不能执行系统命令等），防止恶意操作。  
4. **生命周期管理**：由浏览器或 Applet 容器（如 `appletviewer`）控制其创建、运行、暂停、销毁的整个过程。  


### 二、Applet 的生命周期（核心方法）  
Applet 的生命周期由 5 个核心方法控制，这些方法由浏览器自动调用，无需手动触发：  

| 方法           | 调用时机                                  | 作用                                  |  
|----------------|-------------------------------------------|---------------------------------------|  
| `init()`       | Applet 第一次被加载到浏览器时调用（仅一次） | 初始化资源（如加载图片、设置参数）    |  
| `start()`      | Applet 可见时调用（如页面打开、从后台切回） | 启动线程、开始动画或交互逻辑          |  
| `paint(Graphics g)` | 当 Applet 需要重绘时调用（如首次显示、窗口大小改变） | 绘制界面（文字、图形、图片等）        |  
| `stop()`       | Applet 不可见时调用（如页面切换、最小化）   | 暂停线程、释放临时资源                |  
| `destroy()`    | Applet 被永久卸载时调用（如关闭页面）     | 释放所有资源（如关闭连接、释放内存）  |  


### 三、简单 Applet 示例  
#### 1. 编写 Applet 类  
创建一个简单的 Applet，在页面上绘制文字和图形：  
```java
import java.applet.Applet;
import java.awt.Graphics;

// 继承 Applet 类
public class SimpleApplet extends Applet {

    // 初始化：仅调用一次
    @Override
    public void init() {
        System.out.println("Applet 初始化");
        // 设置背景色
        setBackground(Color.WHITE);
    }

    // 启动：可见时调用
    @Override
    public void start() {
        System.out.println("Applet 启动");
    }

    // 绘制界面：由系统自动调用
    @Override
    public void paint(Graphics g) {
        // 绘制文字
        g.drawString("Hello, Applet!", 50, 50); // (x=50, y=50) 位置绘制文字
        // 绘制矩形
        g.drawRect(50, 70, 100, 50); // 左上角 (50,70)，宽100，高50
    }

    // 暂停：不可见时调用
    @Override
    public void stop() {
        System.out.println("Applet 暂停");
    }

    // 销毁：卸载时调用
    @Override
    public void destroy() {
        System.out.println("Applet 销毁");
    }
}
```  


#### 2. 编写 HTML 嵌入 Applet  
创建 `applet_demo.html` 文件，通过 `<applet>` 标签嵌入上述 Applet：  
```html
<!DOCTYPE html>
<html>
<body>
    <h1>我的第一个 Applet</h1>
    <!-- 嵌入 Applet：code 指定类名，width/height 指定尺寸 -->
    <applet code="SimpleApplet.class" width="300" height="200">
        <!-- 浏览器不支持 Applet 时显示的替代文本 -->
        您的浏览器不支持 Java Applet。
    </applet>
</body>
</html>
```  


#### 3. 运行 Applet  
早期可通过以下方式运行，但目前多数环境已不支持：  
- **使用 `appletviewer` 工具**（JDK 自带，需先编译 `.java` 文件）：  
  1. 编译 `SimpleApplet.java`：`javac SimpleApplet.java` 生成 `SimpleApplet.class`。  
  2. 运行：`appletviewer applet_demo.html`，会弹出窗口显示 Applet 内容。  
- **通过支持 Java 的浏览器**：早期浏览器（如旧版 IE、Firefox）可通过 Java 插件运行，但现代浏览器（Chrome、Edge 等）已禁用 Java 插件，无法运行。  


### 四、Applet 的安全限制（沙箱机制）  
为防止恶意 Applet 破坏用户系统，Applet 运行在“安全沙箱”中，受以下限制：  
- 不能访问本地文件系统（读/写文件），除非获得数字签名授权。  
- 不能连接除“来源服务器”（即加载 Applet 的网页所在服务器）之外的网络地址。  
- 不能执行本地系统命令或访问系统硬件（如打印机、摄像头）。  
- 不能修改浏览器的配置或操作其他网页内容。  


### 五、Applet 的现状与替代技术  
- **过时原因**：  
  1. 安全风险：沙箱机制虽限制严格，但仍存在漏洞，浏览器厂商逐渐弃用 Java 插件。  
  2. 性能问题：Applet 加载依赖 JVM，启动慢，影响用户体验。  
  3. 技术替代：HTML5（Canvas、SVG）、JavaScript、Flash（虽也衰退）等技术可实现更轻量的网页交互，无需额外插件。  

- **替代方案**：  
  - 网页交互：HTML5 + JavaScript + CSS3。  
  - 复杂应用：WebAssembly、Java Web Start（已过时）、桌面应用（Swing/JavaFX）或基于 Spring Boot 的 Web 应用。  


### 总结
