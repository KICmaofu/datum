在 Java 中，**`ArrayList`** 是 `List` 接口最常用的实现类，基于**动态数组**实现，是处理“有序、可重复元素”集合的首选工具。它解决了普通数组长度固定的问题，能自动扩容，同时保留了数组通过索引快速访问元素的特性。


### 一、ArrayList 的核心特性
1. **底层结构**：基于**动态数组**（`Object[]`）实现，默认初始容量为 10。  
2. **有序性**：元素按**插入顺序**存储，支持通过索引（0 开始）访问（类似数组）。  
3. **可重复性**：允许存储重复元素，也允许 `null` 元素（可多个）。  
4. **线程不安全**：非同步实现，多线程并发修改时可能出现数据不一致（需手动加锁或使用同步工具类）。  
5. **自动扩容**：当元素数量超过当前容量时，会自动扩容（JDK8+ 扩容为原容量的 1.5 倍），避免数组溢出。  


### 二、扩容机制（关键原理）
`ArrayList` 的扩容是其核心特性之一，过程如下：  
1. 当调用 `add()` 方法时，先检查当前元素数量（`size`）是否等于数组容量（`capacity`）。  
2. 若已满，触发扩容：  
   - 计算新容量：`newCapacity = oldCapacity + (oldCapacity >> 1)`（即原容量的 1.5 倍）。  
   - 创建一个容量为 `newCapacity` 的新数组。  
   - 将旧数组中的元素**复制到新数组**（通过 `System.arraycopy()` 高效完成）。  
   - 用新数组替换旧数组，完成扩容。  

**注意**：若初始已知元素数量（如预计存储 1000 个元素），可通过构造方法 `new ArrayList(1000)` 指定初始容量，减少扩容次数，提升性能。


### 三、常用方法（核心操作）
`ArrayList` 实现了 `List` 接口，提供了丰富的操作方法，以下是最常用的 API：  

| 方法签名                          | 功能描述                                  |
|-----------------------------------|-------------------------------------------|
| `boolean add(E e)`                | 在末尾添加元素 `e`，返回是否成功          |
| `void add(int index, E e)`        | 在指定索引 `index` 处插入元素 `e`（后续元素后移） |
| `E get(int index)`                | 获取索引 `index` 处的元素                 |
| `E set(int index, E e)`           | 替换索引 `index` 处的元素为 `e`，返回旧元素 |
| `E remove(int index)`             | 删除索引 `index` 处的元素，返回被删除元素（后续元素前移） |
| `boolean remove(Object o)`        | 删除第一个匹配 `o` 的元素，返回是否成功    |
| `int size()`                      | 返回当前元素数量                          |
| `boolean isEmpty()`               | 判断集合是否为空                          |
| `boolean contains(Object o)`      | 判断是否包含元素 `o`（依赖 `equals()`）    |
| `void clear()`                    | 清空所有元素                              |
| `List<E> subList(int from, int to)`| 返回从 `from` 到 `to-1` 的子列表（视图，修改会影响原集合） |


### 四、与 LinkedList 的对比
`ArrayList` 和 `LinkedList` 都是 `List` 的实现类，但适用场景不同，核心区别如下：  

| 维度         | ArrayList                  | LinkedList                  |
|--------------|----------------------------|-----------------------------|
| 底层结构     | 动态数组                   | 双向链表                    |
| 查询效率     | 高（索引访问，O(1)）       | 低（需遍历，O(n)）          |
| 增删效率     | 低（中间位置需移动元素，O(n)） | 高（仅修改指针，O(1)）       |
| 内存占用     | 较少（连续空间，无额外指针） | 较多（每个元素需存储前后指针） |
| 适用场景     | 查询频繁、增删少           | 增删频繁（尤其首尾）、查询少 |


### 五、线程安全问题
`ArrayList` 是非线程安全的（方法未加锁），多线程并发修改时可能出现异常（如 `ConcurrentModificationException`）。若需线程安全，可使用以下方案：  
1. **`Collections.synchronizedList()`**：通过包装 `ArrayList` 生成同步集合，所有方法加锁（效率较低）。  
   ```java
   List<String> syncList = Collections.synchronizedList(new ArrayList<>());
   ```  
2. **`CopyOnWriteArrayList`**：JUC 包中的线程安全类，修改时复制新数组，适合**读多写少**场景（效率更高）。  


### 六、遍历方式
`ArrayList` 支持多种遍历方式，推荐根据场景选择：  
1. **普通 for 循环**（适合需要索引的场景）：  
   ```java
   for (int i = 0; i < list.size(); i++) {
       System.out.println(list.get(i));
   }
   ```  
2. **增强 for 循环**（简洁，无需索引）：  
   ```java
   for (String s : list) {
       System.out.println(s);
   }
   ```  
3. **迭代器（Iterator）**（支持遍历中删除元素）：  
   ```java
   Iterator<String> it = list.iterator();
   while (it.hasNext()) {
       String s = it.next();
       if (s.equals("remove")) {
           it.remove(); // 安全删除，不会触发并发修改异常
       }
   }
   ```  


### 总结
`ArrayList` 是 Java 中最常用的集合类之一，凭借**快速查询**和**动态扩容**的特性，适合处理“查询频繁、增删操作少”的场景。使用时需注意：  
- 初始化时指定容量以减少扩容开销；  
- 多线程环境下需保证线程安全；  
- 避免在遍历中使用 `list.remove()`（可能触发 `ConcurrentModificationException`，推荐用迭代器删除）。
