Java 中的 `HashMap` 是集合框架（`java.util`）中 `Map` 接口的实现类，用于存储**键值对（Key-Value）** 数据，底层基于**哈希表（Hash Table）** 实现。它提供了高效的元素插入、查询和删除操作，是日常开发中处理键值映射最常用的工具之一。


### 一、HashMap 基本特性  
- **存储结构**：键值对集合，每个键（Key）唯一，一个键对应一个值（Value），值可以重复。  
- **底层实现**：JDK 1.8 及之后为 **数组 + 链表 + 红黑树**（当链表长度超过阈值时转为红黑树，优化查询效率）。  
- **允许 null**：支持 `null` 作为键（最多一个 `null` 键）和值（多个 `null` 值）。  
- **无序性**：不保证键值对的存储顺序和遍历顺序（顺序可能随扩容发生变化）。  
- **非线程安全**：多线程环境下并发修改可能导致数据不一致（如死循环、数据丢失），需使用 `ConcurrentHashMap` 替代。  
- **高效性**：平均情况下，插入、查询、删除的时间复杂度为 `O(1)`（哈希表的优势）。  


### 二、底层数据结构解析  
`HashMap` 的核心是**哈希表**，其结构可理解为“数组（桶）+ 链表/红黑树”的组合：  

1. **数组（Bucket Array）**：  
   数组中的每个元素称为“桶（Bucket）”，桶的索引由键的哈希值计算得出。数组的初始长度（容量）为 `16`（默认值），且始终保持为 **2的幂**（方便通过位运算计算索引）。  

2. **链表（Linked List）**：  
   当多个键计算出相同的桶索引时（哈希冲突），这些键值对会以链表的形式存储在同一个桶中。  

3. **红黑树（Red-Black Tree）**：  
   当链表的长度超过 **8**（默认阈值），且数组长度 ≥ **64** 时，链表会转为红黑树（一种自平衡二叉查找树），将查询时间复杂度从 `O(n)` 优化为 `O(log n)`。  


### 三、核心机制：哈希计算与索引定位  
`HashMap` 通过键的哈希值确定其在数组中的存储位置，核心步骤如下：  


#### 1. 计算键的哈希值（Hash Code）  
调用键的 `hashCode()` 方法获取原始哈希值，再通过“扰动函数”优化哈希值（减少哈希冲突）：  
```java
static final int hash(Object key) {
    int h;
    // 扰动函数：将 key 的 hashCode 高 16 位与低 16 位异或，增强哈希值的随机性
    return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
}
```  


#### 2. 计算桶索引（Bucket Index）  
用优化后的哈希值与数组长度 `n` 减 1 进行**与运算**（`&`），得到桶索引：  
```java
int index = (n - 1) & hash;  // n 是数组长度（2的幂），保证索引在 [0, n-1] 范围内
```  

例如：若数组长度 `n=16`（`n-1=15`，二进制 `1111`），哈希值 `hash=20`（二进制 `10100`），则 `index=20 & 15 = 4`（二进制 `100`），键值对存入索引为 4 的桶。  


### 四、核心方法：`put()` 与 `get()` 流程  
`HashMap` 的核心操作是添加（`put()`）和查询（`get()`）键值对，理解其流程是掌握 `HashMap` 的关键。  


#### 1. `put(K key, V value)`：添加键值对  
步骤解析：  
1. **计算哈希值与索引**：通过 `hash(key)` 得到哈希值，再计算桶索引 `index`。  
2. **检查数组是否初始化**：若数组为空（首次添加），调用 `resize()` 初始化数组（默认容量 16）。  
3. **处理当前桶**：  
   - 若桶为空（`table[index] == null`）：直接创建新节点存入桶中。  
   - 若桶不为空：  
     - 检查桶中第一个节点的键是否与当前键相同（`hash` 相等且 `key.equals(节点.key)`）：若相同，覆盖其值。  
     - 若第一个节点不同，且桶是红黑树：调用树的方法添加节点（存在则覆盖，否则新增）。  
     - 若桶是链表：遍历链表，检查是否有相同键（有则覆盖）；无则添加新节点到链表尾部，之后判断链表长度是否 ≥ 8，若满足则转为红黑树。  
4. **检查是否需要扩容**：若元素总数（`size`）超过阈值（`capacity * loadFactor`，默认 16*0.75=12），调用 `resize()` 扩容。  


#### 2. `get(Object key)`：查询键对应的值  
步骤解析：  
1. **计算哈希值与索引**：同 `put()`，得到桶索引 `index`。  
2. **检查当前桶**：  
   - 若桶为空：返回 `null`（无此键）。  
   - 若桶不为空：  
     - 检查桶中第一个节点的键是否匹配：匹配则返回其值。  
     - 若不匹配，且桶是红黑树：调用树的方法查询匹配节点并返回值。  
     - 若桶是链表：遍历链表，找到匹配节点返回值；遍历结束无匹配则返回 `null`。  


### 五、扩容机制（`resize()`）  
当 `HashMap` 中的元素数量超过阈值（`capacity * loadFactor`）时，会触发扩容（`resize()`），流程如下：  
1. **计算新容量**：新容量 = 旧容量 × 2（保证始终是 2 的幂），新阈值 = 新容量 × 负载因子（默认 0.75）。  
2. **创建新数组**：初始化一个长度为新容量的数组。  
3. **迁移元素**：将旧数组中的所有键值对重新计算索引（基于新容量），迁移到新数组的对应桶中（链表可能拆分，红黑树可能退化为链表）。  

**注意**：扩容是耗时操作（需重新计算索引并迁移所有元素），实际开发中应尽量预设合适的初始容量（避免频繁扩容）。  


### 六、关键参数  
`HashMap` 的性能受以下参数影响，可在构造方法中自定义：  

| 参数           | 作用                                  | 默认值  |  
|----------------|---------------------------------------|---------|  
| `initialCapacity` | 初始容量（数组长度，必须是 2 的幂）    | 16      |  
| `loadFactor`    | 负载因子（阈值 = 容量 × 负载因子）    | 0.75    |  
| `TREEIFY_THRESHOLD` | 链表转红黑树的阈值（长度 ≥ 此值） | 8       |  
| `UNTREEIFY_THRESHOLD` | 红黑树转链表的阈值（长度 ≤ 此值） | 6       |  


### 七、自定义对象作为 Key 的注意事项  
当使用自定义对象作为 `HashMap` 的 Key 时，**必须同时重写 `hashCode()` 和 `equals()` 方法**，否则会导致：  
- 无法正确查询值（相同内容的对象被认为是不同 Key）；  
- 哈希冲突增多（`hashCode()` 实现不佳）；  
- 内存泄漏（删除元素时无法正确匹配）。  


**示例：自定义 Key 类**  
```java
import java.util.Objects;

class Student {
    private String id;
    private String name;

    public Student(String id, String name) {
        this.id = id;
        this.name = name;
    }

    // 重写 equals()：id 相同则认为是同一个对象
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        Student student = (Student) o;
        return Objects.equals(id, student.id);
    }

    // 重写 hashCode()：基于 id 计算哈希值（与 equals() 逻辑一致）
    @Override
    public int hashCode() {
        return Objects.hash(id);
    }
}
```  

使用自定义 Key：  
```java
import java.util.HashMap;
import java.util.Map;

public class Test {
    public static void main(String[] args) {
        Map<Student, Integer> scoreMap = new HashMap<>();
        Student s1 = new Student("001", "张三");
        Student s2 = new Student("001", "张三三"); // id 相同，视为同一个 Key

        scoreMap.put(s1, 90);
        System.out.println(scoreMap.get(s2)); // 输出 90（正确匹配）
    }
}
```  


### 八、HashMap 与其他 Map 实现类的对比  
| 实现类          | 底层结构         | 线程安全 | 有序性               | 允许 null 键值 | 适用场景                          |  
|-----------------|------------------|----------|----------------------|----------------|-----------------------------------|  
| `HashMap`       | 数组+链表+红黑树 | 否       | 无序                 | 是             | 单线程、无需顺序、高效查询场景    |  
| `Hashtable`     | 数组+链表        | 是（同步方法） | 无序                 | 否             | 旧版线程安全场景（已被 `ConcurrentHashMap` 替代） |  
| `LinkedHashMap` | 哈希表+双向链表  | 否       | 有序（插入/访问顺序） | 是             | 需要保留插入/访问顺序的场景       |  
| `TreeMap`       | 红黑树           | 否       | 有序（自然/定制排序） | 否（键）       | 需要排序或范围查询的场景          |  


### 九、总结  
`HashMap` 是基于哈希表的键值对集合，核心优势是**高效的插入、查询和删除操作**（平均 `O(1)`），适合处理大量键值映射且无需顺序的场景。其底层通过“数组+链表+红黑树”解决哈希冲突，并通过扩容机制保证性能。  

使用时需注意：  
- 自定义对象作为 Key 必须重写 `hashCode()` 和 `equals()`；  
- 多线程环境下需使用 `ConcurrentHashMap` 保证线程安全；  
- 预设合适的初始容量可减少扩容次数，提升性能。  

`HashMap` 是 Java 开发中最常用的 Map 实现，掌握其原理对写出高效代码至关重要。
