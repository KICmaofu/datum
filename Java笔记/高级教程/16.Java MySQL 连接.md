Java 连接 MySQL 数据库主要通过 **JDBC（Java Database Connectivity）** 技术实现。JDBC 是 Java 提供的一套用于操作关系型数据库的标准 API，通过加载数据库驱动、建立连接、执行 SQL 语句等步骤，实现 Java 程序与 MySQL 的交互。


### 一、准备工作  
#### 1. 下载 MySQL JDBC 驱动  
MySQL 官方提供的 JDBC 驱动为 **Connector/J**，需根据 MySQL 版本选择对应驱动版本：  
- MySQL 8.0+ → 推荐使用 8.x 版本驱动（如 `mysql-connector-java:8.0.33`）。  
- MySQL 5.7 及以下 → 推荐使用 5.x 版本驱动（如 `mysql-connector-java:5.1.49`）。  

驱动下载地址：[MySQL 官方驱动下载](https://dev.mysql.com/downloads/connector/j/)  


#### 2. 引入驱动依赖  
将驱动添加到项目的类路径中：  

- **Maven 项目**（推荐）：在 `pom.xml` 中添加依赖：  
  ```xml
  <!-- MySQL 8.x 驱动 -->
  <dependency>
      <groupId>com.mysql</groupId>
      <artifactId>mysql-connector-java</artifactId>
      <version>8.0.33</version>
  </dependency>

  <!-- 若使用 MySQL 5.x，替换为： -->
  <!-- <dependency>
      <groupId>mysql</groupId>
      <artifactId>mysql-connector-java</artifactId>
      <version>5.1.49</version>
  </dependency> -->
  ```  

- **非 Maven 项目**：手动下载 `mysql-connector-java-x.x.xx.jar`，将其复制到项目的 `lib` 目录，并添加到类路径（IDE 中右键 JAR 包 → `Add as Library`）。  


#### 3. 确认 MySQL 环境  
- 确保 MySQL 服务已启动（可通过 `net start mysql` 命令启动）。  
- 已创建目标数据库（如 `testdb`）和测试表（如 `user`），示例表结构：  
  ```sql
  CREATE DATABASE IF NOT EXISTS testdb;
  USE testdb;

  CREATE TABLE IF NOT EXISTS user (
      id INT PRIMARY KEY AUTO_INCREMENT,
      username VARCHAR(50) NOT NULL,
      age INT
  );
  ```  


### 二、JDBC 连接 MySQL 的核心步骤  
JDBC 操作数据库的通用流程：**加载驱动 → 建立连接 → 执行 SQL → 处理结果 → 关闭资源**。  


#### 1. 核心类与接口  
| 类/接口          | 作用                                  |  
|------------------|---------------------------------------|  
| `DriverManager`  | 管理数据库驱动，用于获取数据库连接（`Connection`）。 |  
| `Connection`     | 代表与数据库的连接，用于创建执行 SQL 的对象（`Statement`/`PreparedStatement`）。 |  
| `Statement`      | 执行静态 SQL 语句（存在 SQL 注入风险）。 |  
| `PreparedStatement` | 预编译 SQL 语句（参数化查询，避免 SQL 注入，推荐使用）。 |  
| `ResultSet`      | 存储 SQL 查询（`SELECT`）的结果集，提供遍历方法。 |  


#### 2. 连接数据库的基本代码  
```java
import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;

public class MySQLConnection {
    // 数据库连接信息（根据实际环境修改）
    private static final String URL = "jdbc:mysql://localhost:3306/testdb?useSSL=false&serverTimezone=UTC";
    private static final String USER = "root";       // MySQL 用户名
    private static final String PASSWORD = "123456"; // MySQL 密码

    public static Connection getConnection() {
        Connection conn = null;
        try {
            // 1. 加载驱动（MySQL 8.x 驱动类为 com.mysql.cj.jdbc.Driver；5.x 为 com.mysql.jdbc.Driver）
            // 注意：JDBC 4.0+ 后可省略此步骤，DriverManager 会自动扫描并加载驱动
            Class.forName("com.mysql.cj.jdbc.Driver");

            // 2. 建立连接（通过 DriverManager 获取 Connection）
            conn = DriverManager.getConnection(URL, USER, PASSWORD);
            System.out.println("数据库连接成功！");
        } catch (ClassNotFoundException e) {
            System.err.println("驱动加载失败：" + e.getMessage());
        } catch (SQLException e) {
            System.err.println("连接失败：" + e.getMessage());
        }
        return conn;
    }

    public static void main(String[] args) {
        // 测试连接
        Connection conn = getConnection();
        // 关闭连接（实际开发中需在使用后关闭）
        if (conn != null) {
            try {
                conn.close();
                System.out.println("连接已关闭");
            } catch (SQLException e) {
                e.printStackTrace();
            }
        }
    }
}
```  


#### 3. URL 参数说明  
连接 URL 格式：`jdbc:mysql://主机名:端口号/数据库名?参数1&参数2&...`  

- `localhost:3306`：MySQL 服务器地址和端口（默认端口 3306）。  
- `testdb`：目标数据库名。  
- 关键参数（MySQL 8.x 必需）：  
  - `useSSL=false`：禁用 SSL 连接（开发环境可关闭，生产环境根据需求开启）。  
  - `serverTimezone=UTC`：指定时区（MySQL 8.x 必须设置，否则会报时区错误，可选 `Asia/Shanghai` 表示北京时间）。  


### 三、执行 SQL 语句（CRUD 操作）  
通过 `PreparedStatement` 执行 SQL，支持增删改查（CRUD），且能防止 SQL 注入。  


#### 1. 插入数据（INSERT）  
```java
import java.sql.Connection;
import java.sql.PreparedStatement;
import java.sql.SQLException;

public class InsertDemo {
    public static void main(String[] args) {
        // SQL 插入语句（? 为参数占位符）
        String sql = "INSERT INTO user (username, age) VALUES (?, ?)";

        // 使用 try-with-resources 自动关闭资源（Connection、PreparedStatement 会自动关闭）
        try (Connection conn = MySQLConnection.getConnection();
             PreparedStatement pstmt = conn.prepareStatement(sql)) {

            // 设置参数（索引从 1 开始）
            pstmt.setString(1, "张三"); // 第一个 ? 赋值为字符串 "张三"
            pstmt.setInt(2, 20);       // 第二个 ? 赋值为整数 20

            // 执行更新（返回受影响的行数）
            int rows = pstmt.executeUpdate();
            System.out.println("插入成功，影响行数：" + rows);

        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}
```  


#### 2. 查询数据（SELECT）  
```java
import java.sql.Connection;
import java.sql.PreparedStatement;
import java.sql.ResultSet;
import java.sql.SQLException;

public class SelectDemo {
    public static void main(String[] args) {
        // SQL 查询语句
        String sql = "SELECT id, username, age FROM user WHERE age > ?";

        try (Connection conn = MySQLConnection.getConnection();
             PreparedStatement pstmt = conn.prepareStatement(sql)) {

            // 设置参数：查询年龄大于 18 的用户
            pstmt.setInt(1, 18);

            // 执行查询，返回结果集（ResultSet）
            try (ResultSet rs = pstmt.executeQuery()) {
                // 遍历结果集
                while (rs.next()) { // rs.next() 移动到下一行，若有数据返回 true
                    int id = rs.getInt("id");         // 通过列名获取 int
                    String username = rs.getString("username"); // 通过列名获取 String
                    int age = rs.getInt(3);           // 也可通过列索引（从 1 开始）获取

                    System.out.printf("id: %d, username: %s, age: %d%n", id, username, age);
                }
            }

        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}
```  


#### 3. 更新数据（UPDATE）  
```java
import java.sql.Connection;
import java.sql.PreparedStatement;
import java.sql.SQLException;

public class UpdateDemo {
    public static void main(String[] args) {
        // SQL 更新语句
        String sql = "UPDATE user SET age = ? WHERE username = ?";

        try (Connection conn = MySQLConnection.getConnection();
             PreparedStatement pstmt = conn.prepareStatement(sql)) {

            // 设置参数：将 "张三" 的年龄改为 21
            pstmt.setInt(1, 21);
            pstmt.setString(2, "张三");

            int rows = pstmt.executeUpdate();
            System.out.println("更新成功，影响行数：" + rows);

        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}
```

### 四、关键注意事项  
1. **资源关闭**：  
   - `Connection`、`PreparedStatement`、`ResultSet` 都是资源，必须关闭（否则会导致数据库连接泄漏）。  
   - 推荐使用 **try-with-resources** 语法（JDK 7+），会自动关闭实现 `AutoCloseable` 接口的资源。  


2. **SQL 注入防护**：  
   - 禁止使用 `Statement` 拼接 SQL 字符串（如 `String sql = "SELECT * FROM user WHERE username = '" + name + "'"`），恶意用户可通过输入 `' OR '1'='1` 等语句窃取数据。  
   - 必须使用 `PreparedStatement` 的参数化查询（`?` 占位符），驱动会自动转义参数，防止 SQL 注入。  


3. **驱动版本与 MySQL 版本匹配**：  
   - MySQL 8.x 驱动（`com.mysql.cj.jdbc.Driver`）需搭配 MySQL 8.0+ 服务器，且必须指定时区参数（`serverTimezone`）。  
   - MySQL 5.x 驱动（`com.mysql.jdbc.Driver`）适用于 MySQL 5.7 及以下，无需强制指定时区。  


4. **连接信息配置**：  
   - 实际开发中，数据库 URL、用户名、密码不应硬编码在代码中，应放在配置文件（如 `db.properties`）中，通过工具类读取。  

   示例 `db.properties`：  
   ```properties
   url=jdbc:mysql://localhost:3306/testdb?useSSL=false&serverTimezone=Asia/Shanghai
   user=root
   password=123456
   driver=com.mysql.cj.jdbc.Driver
   ```  

   读取配置文件的工具类：  
   ```java
   import java.io.IOException;
   import java.io.InputStream;
   import java.util.Properties;

   public class DBConfig {
       private static Properties props = new Properties();

       static {
           // 加载配置文件
           try (InputStream is = DBConfig.class.getClassLoader().getResourceAsStream("db.properties")) {
               props.load(is);
           } catch (IOException e) {
               e.printStackTrace();
           }
       }

       public static String getUrl() {
           return props.getProperty("url");
       }

       public static String getUser() {
           return props.getProperty("user");
       }

       public static String getPassword() {
           return props.getProperty("password");
       }

       public static String getDriver() {
           return props.getProperty("driver");
       }
   }
   ```  


### 五、常见错误与解决  
1. **ClassNotFoundException**：驱动类未找到 → 检查驱动 JAR 是否引入，或驱动类名是否正确（8.x 是 `com.mysql.cj.jdbc.Driver`，5.x 是 `com.mysql.jdbc.Driver`）。  
2. **SQLException: The server time zone value ...**：时区未设置 → URL 中添加 `serverTimezone=UTC` 或 `Asia/Shanghai`。  
3. **Access denied for user 'root'@'localhost'**：用户名/密码错误 → 检查 `USER` 和 `PASSWORD` 是否正确。  
4. **Communications link failure**：MySQL 服务未启动，或主机/端口错误 → 确认 MySQL 服务已启动，`URL` 中的主机和端口正确。  


### 总结  
Java 连接 MySQL 的核心是通过 JDBC  API，步骤为：引入驱动 → 建立连接 → 用 `PreparedStatement` 执行 SQL → 处理结果 → 关闭资源。关键是注意资源关闭、防止 SQL 注入，以及驱动与 MySQL 版本的匹配。实际开发中，可进一步使用数据库连接池（如 HikariCP、Druid）优化连接管理，提升性能。
