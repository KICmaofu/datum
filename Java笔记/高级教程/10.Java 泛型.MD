Java 泛型（Generics）是 JDK 5 引入的核心特性，它允许在**定义类、接口、方法**时使用**类型参数（Type Parameter）**，从而实现“编写一次，通用适配多种类型”的效果。泛型的核心价值是**编译时类型安全检查**和**消除显式类型转换**，避免运行时的 `ClassCastException`。


### 一、为什么需要泛型？  
在泛型出现之前，Java 集合（如 `List`、`Map`）只能存储 `Object` 类型，导致两个问题：  
1. **类型不安全**：可以向集合中添加任意类型的元素（如向 `List` 中同时添加 `String` 和 `Integer`）。  
2. **强制类型转换**：从集合中取元素时必须手动强转，若类型不匹配会抛出 `ClassCastException`。  


**示例：无泛型的问题**  
```java
import java.util.ArrayList;
import java.util.List;

public class NoGenericsDemo {
    public static void main(String[] args) {
        List list = new ArrayList();
        list.add("Java");    // 添加 String
        list.add(123);       // 添加 Integer（编译不报错，类型不安全）

        // 取出元素时必须强转，运行时抛出 ClassCastException
        String str = (String) list.get(1); // 123 是 Integer，强转失败
    }
}
```  


**泛型解决上述问题**：  
通过泛型指定集合存储的元素类型，编译时会检查类型匹配，且取出时无需强转：  
```java
List<String> list = new ArrayList<>();
list.add("Java");    // 正确：类型匹配
list.add(123);       // 编译报错：不允许添加 Integer
String str = list.get(0); // 无需强转，直接使用
```  


### 二、泛型的基本语法  
泛型通过**类型参数**（用 `<>` 包裹，如 `<T>`）定义，使用时指定具体类型（如 `String`、`Integer`）。  


#### 1. 泛型类（Generic Class）  
在类定义时声明类型参数，使类能适配多种数据类型。  

**语法**：  
```java
class 类名<类型参数1, 类型参数2, ...> {
    // 类型参数可作为成员变量、方法参数、返回值的类型
    private 类型参数1 变量名;
    
    public 类型参数2 方法名(类型参数1 参数) { ... }
}
```  


**示例：自定义泛型类 `Box<T>`**  
```java
// 泛型类：T 是类型参数（可理解为“占位符”）
class Box<T> {
    private T content; // 用 T 作为成员变量类型

    // 用 T 作为方法参数和返回值类型
    public void setContent(T content) {
        this.content = content;
    }

    public T getContent() {
        return content;
    }
}

// 使用泛型类：指定具体类型（如 String、Integer）
public class GenericClassDemo {
    public static void main(String[] args) {
        // 1. 存储 String 类型
        Box<String> stringBox = new Box<>();
        stringBox.setContent("Hello");
        String str = stringBox.getContent(); // 无需强转

        // 2. 存储 Integer 类型
        Box<Integer> intBox = new Box<>();
        intBox.setContent(123);
        Integer num = intBox.getContent(); // 无需强转
    }
}
```  


#### 2. 泛型接口（Generic Interface）  
接口定义时声明类型参数，实现类需指定具体类型或继续保留泛型。  

**语法**：  
```java
interface 接口名<类型参数> {
    类型参数 方法名();
}
```  


**示例：泛型接口 `Generator<T>`**  
```java
// 泛型接口：生成指定类型的对象
interface Generator<T> {
    T generate();
}

// 实现类 1：指定具体类型（String）
class StringGenerator implements Generator<String> {
    @Override
    public String generate() {
        return "Generated String";
    }
}

// 实现类 2：保留泛型（适用于通用实现）
class DefaultGenerator<T> implements Generator<T> {
    private T defaultValue;
    public DefaultGenerator(T defaultValue) {
        this.defaultValue = defaultValue;
    }
    @Override
    public T generate() {
        return defaultValue; // 返回默认值
    }
}

// 使用
public class GenericInterfaceDemo {
    public static void main(String[] args) {
        Generator<String> strGen = new StringGenerator();
        System.out.println(strGen.generate()); // 输出：Generated String

        Generator<Integer> intGen = new DefaultGenerator<>(100);
        System.out.println(intGen.generate()); // 输出：100
    }
}
```  


#### 3. 泛型方法（Generic Method）  
方法定义时声明类型参数，可独立于类的泛型存在（即使类不是泛型类，也可定义泛型方法）。  

**语法**：  
```java
修饰符 <类型参数> 返回值类型 方法名(参数列表) {
    // 方法体中使用类型参数
}
```  


**示例：泛型方法 `printArray(T[] array)`**  
```java
public class GenericMethodDemo {
    // 泛型方法：打印任意类型的数组
    public static <T> void printArray(T[] array) {
        for (T element : array) {
            System.out.print(element + " ");
        }
        System.out.println();
    }

    public static void main(String[] args) {
        Integer[] intArray = {1, 2, 3};
        String[] strArray = {"A", "B", "C"};

        printArray(intArray); // 输出：1 2 3 （自动推断 T 为 Integer）
        printArray(strArray); // 输出：A B C （自动推断 T 为 String）
    }
}
```  


### 三、类型参数命名规范  
类型参数通常用单个大写字母表示，遵循约定俗成的含义：  
- `T`（Type）：表示任意类型  
- `E`（Element）：表示集合中的元素类型  
- `K`（Key）：表示键（如 `Map` 中的键）  
- `V`（Value）：表示值（如 `Map` 中的值）  
- `N`（Number）：表示数值类型  


### 四、泛型通配符（Wildcards）  
当需要灵活处理泛型类型（如接受任意泛型类型作为参数）时，需使用**通配符 `?`**。通配符有三种形式：  


#### 1. 无界通配符（`?`）  
表示“任意类型”，适用于只读取数据、不关心具体类型的场景。  

**示例**：打印任意 `Box<T>` 的内容  
```java
class Box<T> { /* 省略实现 */ }

public class WildcardDemo {
    // 无界通配符：接受任意类型的 Box
    public static void printBox(Box<?> box) {
        System.out.println(box.getContent()); // 只能读取，不能修改（因类型未知）
    }

    public static void main(String[] args) {
        Box<String> strBox = new Box<>();
        strBox.setContent("Hello");
        Box<Integer> intBox = new Box<>();
        intBox.setContent(123);

        printBox(strBox); // 合法
        printBox(intBox); // 合法
    }
}
```  


#### 2. 上界通配符（`? extends T`）  
表示“`T` 及其子类”，限制类型的上限，适用于**读取数据**（生产者场景）。  

**示例**：计算数字集合的总和（只读取，不写入）  
```java
import java.util.List;

public class UpperBoundDemo {
    // 上界通配符：只接受 Number 及其子类（Integer、Double 等）
    public static double sum(List<? extends Number> numbers) {
        double total = 0;
        for (Number num : numbers) {
            total += num.doubleValue(); // 读取数据（安全）
        }
        return total;
    }

    public static void main(String[] args) {
        List<Integer> ints = List.of(1, 2, 3);
        List<Double> doubles = List.of(1.5, 2.5);

        System.out.println(sum(ints));    // 输出：6.0
        System.out.println(sum(doubles)); // 输出：4.0
    }
}
```  

**注意**：上界通配符的集合**不能添加元素**（除 `null`），因为无法确定具体类型（如 `List<? extends Number>` 可能是 `List<Integer>` 或 `List<Double>`，添加 `Integer` 到 `Double` 集合会出错）。  


#### 3. 下界通配符（`? super T`）  
表示“`T` 及其父类”，限制类型的下限，适用于**写入数据**（消费者场景）。  

**示例**：向集合中添加整数（只写入，读取时类型可能为 `Object`）  
```java
import java.util.ArrayList;
import java.util.List;

public class LowerBoundDemo {
    // 下界通配符：接受 Integer 及其父类（Number、Object）
    public static void addIntegers(List<? super Integer> list) {
        list.add(1); // 写入 Integer 及其子类（安全）
        list.add(2);
    }

    public static void main(String[] args) {
        List<Integer> ints = new ArrayList<>();
        List<Number> nums = new ArrayList<>();

        addIntegers(ints); // 合法（Integer 是 Integer 的本身）
        addIntegers(nums); // 合法（Number 是 Integer 的父类）

        System.out.println(ints); // 输出：[1, 2]
        System.out.println(nums); // 输出：[1, 2]
    }
}
```  

**注意**：下界通配符的集合**读取时只能视为 `Object` 类型**（因可能包含父类对象），适合写入为主的场景。  


#### 通配符使用原则：PECS  
- **Producer Extends**：如果集合是“生产者”（主要用于读取数据），用 `? extends T`。  
- **Consumer Super**：如果集合是“消费者”（主要用于写入数据），用 `? super T`。  


### 五、类型擦除（Type Erasure）  
Java 泛型是**编译时特性**，在运行时会被“擦除”——泛型类型信息（如 `T`、`String`）会被替换为**原始类型**（默认是 `Object`，有上界时替换为上界类型）。  

**示例：类型擦除的效果**  
```java
class Box<T> {
    private T content;
    public T getContent() { return content; }
}

// 编译后，类型擦除为：
class Box {
    private Object content;
    public Object getContent() { return content; }
}
```  


**类型擦除的影响**：  
1. 运行时无法获取泛型类型信息（如 `instanceof List<String>` 编译报错，因擦除后都是 `List`）。  
2. 不能创建泛型数组（如 `new T[10]` 报错，因擦除后无法确定数组类型）。  
3. 静态方法中不能使用类的泛型参数（因静态成员属于类，而泛型参数在实例化时确定）。  


### 六、泛型的限制  
1. **不能使用基本类型作为类型参数**：必须用包装类（如 `Box<int>` 报错，需用 `Box<Integer>`）。  
2. **不能实例化泛型类型的对象**：如 `new T()` 报错（类型擦除后 `T` 为 `Object`，失去泛型意义）。  
3. **不能捕获泛型类型的异常**：如 `catch (T e)` 报错（泛型类型不能是异常类型）。  
4. **泛型类的静态成员不能依赖泛型参数**：静态方法/变量属于类，而泛型参数绑定到实例。  


### 七、总结  
泛型是 Java 实现“通用编程”的核心机制，核心价值：  
- **类型安全**：编译时检查类型匹配，避免运行时 `ClassCastException`。  
- **代码复用**：一套逻辑适配多种类型（如 `List` 可存储 `String`、`Integer` 等）。  
- **消除强转**：从泛型集合中取元素时无需手动强转，简化代码。  

核心概念包括：泛型类、接口、方法的定义与使用，通配符（无界、上界、下界）的适用场景，以及类型擦除的特性。掌握泛型是编写灵活、安全的 Java 代码的基础，尤其在集合框架、工具类开发中不可或缺。
