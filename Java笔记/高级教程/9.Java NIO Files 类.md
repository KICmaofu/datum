Java NIO 的 `java.nio.file.Files` 类是 JDK 7 引入的**文件操作工具类**，封装了大量静态方法，用于简化文件/目录的读写、复制、移动、删除、属性查询等操作。它基于 NIO.2 的 `Path` 接口（替代传统 `java.io.File` 类），API 更简洁、功能更全面，且支持异常处理和高级文件操作（如文件树遍历、文件属性批量获取）。


### 一、核心定位与依赖
- **所属包**：`java.nio.file.Files`（需导入此类及 `java.nio.file.Path` 接口）。  
- **核心依赖**：所有方法均围绕 `Path` 接口（文件/目录路径的抽象），需先通过 `Paths.get()`（JDK 7+）或 `Path.of()`（JDK 11+）创建 `Path` 对象。  
- **异常处理**：大部分方法抛出 `java.io.IOException`（受检异常），需手动捕获或声明抛出。  
- **核心优势**：静态方法直接调用，无需手动管理流资源（部分方法自动关闭流），支持批量操作和高级功能（如文件树遍历）。


### 二、常用方法分类与示例
`Files` 类的方法按功能可分为**文件读写、文件操作、路径操作、属性查询**四大类，以下是高频使用场景：

#### 1. 路径创建（基础前提）
所有 `Files` 操作都依赖 `Path` 对象，先创建路径：
```java
import java.nio.file.Path;
import java.nio.file.Paths; // JDK 7+
// import java.nio.file.Path; // JDK 11+ 可直接用 Path.of()

// 1. 创建文件路径（绝对路径/相对路径）
Path file = Paths.get("D:/test.txt"); // JDK 7+
// Path file = Path.of("D:/test.txt"); // JDK 11+ 更简洁

// 2. 创建目录路径
Path dir = Paths.get("D:/test_dir");
```


#### 2. 文件读写（快速操作）
无需手动创建流，`Files` 提供一键读写方法，适合小文件（大文件建议用流操作）。

| 方法                          | 作用                                  | 示例                                  |
|-------------------------------|---------------------------------------|---------------------------------------|
| `readAllBytes(Path path)`     | 读取文件所有字节（返回 `byte[]`）      | 读取文本文件、二进制文件              |
| `readAllLines(Path path)`     | 读取文件所有行（返回 `List<String>`）  | 读取文本文件（自动按行分割）          |
| `write(Path path, byte[] b)`  | 写入字节数组到文件（覆盖原有内容）    | 写入二进制文件、字符串（需转字节）    |
| `write(Path path, List<String> lines)` | 写入字符串列表到文件（每行自动换行） | 写入文本文件                          |
| `newBufferedReader(Path path)`| 创建字符缓冲读取流（`BufferedReader`） | 大文本文件逐行读取                    |
| `newBufferedWriter(Path path)`| 创建字符缓冲写入流（`BufferedWriter`） | 大文本文件逐行写入                    |

**示例 1：快速读取文本文件**
```java
import java.nio.file.Files;
import java.nio.file.Path;
import java.nio.file.Paths;
import java.util.List;
import java.io.IOException;

public class FilesDemo {
    public static void main(String[] args) {
        Path file = Paths.get("D:/test.txt");
        try {
            // 读取所有行（自动按行分割，UTF-8 编码）
            List<String> lines = Files.readAllLines(file);
            for (String line : lines) {
                System.out.println(line);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

**示例 2：快速写入文本文件**
```java
import java.util.Arrays;
import java.util.List;

public class FilesWriteDemo {
    public static void main(String[] args) {
        Path file = Paths.get("D:/output.txt");
        List<String> content = Arrays.asList("Hello", "Java NIO Files", "2024");
        
        try {
            // 写入列表（每行自动换行，覆盖原有内容）
            Files.write(file, content);
            System.out.println("写入成功");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

**示例 3：大文件逐行读取（缓冲流）**
```java
import java.io.BufferedReader;
import java.io.IOException;

public class LargeFileReadDemo {
    public static void main(String[] args) {
        Path file = Paths.get("D:/large_file.txt");
        
        try (BufferedReader br = Files.newBufferedReader(file)) {
            String line;
            // 逐行读取（流会自动关闭，try-with-resources 语法）
            while ((line = br.readLine()) != null) {
                System.out.println(line);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```


#### 3. 文件/目录操作（复制、移动、删除、创建）
涵盖文件/目录的核心操作，支持覆盖、原子操作等选项。

| 方法                          | 作用                                  | 关键参数/选项                          |
|-------------------------------|---------------------------------------|---------------------------------------|
| `createFile(Path path)`       | 创建空文件（文件已存在抛异常）        | -                                     |
| `createDirectory(Path dir)`   | 创建单级目录（父目录不存在抛异常）    | -                                     |
| `createDirectories(Path dir)` | 递归创建多级目录（父目录不存在则创建） | -                                     |
| `copy(Path source, Path target)` | 复制文件/目录                        | 可选 `StandardCopyOption`（如 `REPLACE_EXISTING` 覆盖） |
| `move(Path source, Path target)` | 移动/重命名文件/目录                | 可选 `StandardCopyOption.ATOMIC_MOVE`（原子操作，避免文件丢失） |
| `delete(Path path)`           | 删除文件/空目录（非空目录抛异常）    | -                                     |
| `deleteIfExists(Path path)`   | 存在则删除（无异常）                  | -                                     |

**示例 1：创建多级目录**
```java
Path dir = Paths.get("D:/a/b/c");
try {
    Files.createDirectories(dir); // 递归创建 a/b/c 目录
} catch (IOException e) {
    e.printStackTrace();
}
```

**示例 2：复制文件（覆盖已存在文件）**
```java
import java.nio.file.StandardCopyOption;

Path source = Paths.get("D:/test.txt");
Path target = Paths.get("D:/backup/test.txt");

try {
    // 复制文件，覆盖目标文件（需先确保目标目录存在）
    Files.createDirectories(target.getParent()); // 创建目标目录
    Files.copy(source, target, StandardCopyOption.REPLACE_EXISTING);
} catch (IOException e) {
    e.printStackTrace();
}
```

**示例 3：移动文件（原子操作）**
```java
Path source = Paths.get("D:/test.txt");
Path target = Paths.get("D:/archive/test.txt");

try {
    Files.createDirectories(target.getParent());
    // 原子移动（避免移动过程中文件损坏/丢失）
    Files.move(source, target, StandardCopyOption.ATOMIC_MOVE);
} catch (IOException e) {
    e.printStackTrace();
}
```


#### 4. 路径查询与属性判断
快速判断文件/目录的存在性、类型、属性等。

| 方法                          | 作用                                  | 示例返回值                          |
|-------------------------------|---------------------------------------|---------------------------------------|
| `exists(Path path)`           | 判断路径是否存在                      | `true`/`false`                        |
| `isFile(Path path)`           | 判断是否为文件                        | `true`/`false`                        |
| `isDirectory(Path path)`      | 判断是否为目录                        | `true`/`false`                        |
| `size(Path path)`             | 获取文件大小（字节数）                | `long` 类型（如 1024 表示 1KB）       |
| `getLastModifiedTime(Path path)` | 获取最后修改时间                      | `FileTime` 类型（可转 `LocalDateTime`） |
| `isReadable(Path path)`       | 判断是否可读                          | `true`/`false`                        |
| `isWritable(Path path)`       | 判断是否可写                          | `true`/`false`                        |

**示例：查询文件属性**
```java
Path file = Paths.get("D:/test.txt");
try {
    if (Files.exists(file) && Files.isFile(file)) {
        long size = Files.size(file);
        FileTime lastModified = Files.getLastModifiedTime(file);
        System.out.printf("文件大小：%d 字节%n", size);
        System.out.printf("最后修改时间：%s%n", lastModified);
    }
} catch (IOException e) {
    e.printStackTrace();
}
```


#### 5. 高级功能：文件树遍历
通过 `Files.walk()` 递归遍历目录下所有文件/目录，支持深度限制。

**示例：遍历目录下所有 `.txt` 文件**
```java
import java.nio.file.FileVisitOption;
import java.util.EnumSet;

Path dir = Paths.get("D:/test_dir");
try {
    // 遍历目录（深度限制为 5，支持符号链接）
    Files.walk(dir, 5, EnumSet.of(FileVisitOption.FOLLOW_LINKS))
         .filter(Files::isFile) // 只保留文件
         .filter(path -> path.getFileName().toString().endsWith(".txt")) // 筛选 .txt 文件
         .forEach(path -> System.out.println("找到 TXT 文件：" + path));
} catch (IOException e) {
    e.printStackTrace();
}
```


### 三、关键注意事项
1. **异常处理**：所有 IO 操作均需捕获 `IOException`，避免程序崩溃。  
2. **路径兼容性**：`Path` 接口自动适配操作系统（Windows 用 `\`，Linux/Mac 用 `/`），无需手动处理分隔符。  
3. **大文件操作**：`readAllBytes()`/`readAllLines()` 会将文件全部加载到内存，适合小文件（<100MB）；大文件需用 `newBufferedReader()`/`newBufferedWriter()` 或 `Files.lines()`（流式读取）。  
4. **权限问题**：操作系统可能限制文件读写权限（如 Windows 系统盘文件），需确保程序有对应权限。


### 四、总结
`Files` 类是 Java 处理文件/目录的“瑞士军刀”，核心优势是**API 简洁、功能全面、无需手动管理流**。它替代了传统 `File` 类的繁琐操作，支持路径适配、高级遍历、属性查询等功能，是现代 Java 开发中文件操作的首选工具。

掌握其核心方法（读写、复制、移动、遍历），可大幅提升文件操作的开发效率，同时避免传统 IO 的冗余代码。
