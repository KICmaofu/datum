Java 中的 `HashSet` 是集合框架（`java.util`）中 `Set` 接口的实现类，底层基于 **HashMap** 实现，其核心特性是**不允许存储重复元素**且**元素无序**（存储顺序与插入顺序不一致）。`HashSet` 提供了高效的元素添加、删除和查找操作，是处理“去重”需求的常用工具。


### 一、HashSet 基本特性  
- **底层结构**：基于 `HashMap` 实现（`HashSet` 内部维护一个 `HashMap` 实例，元素作为 `HashMap` 的 `key` 存储，`value` 是一个固定的空对象）。  
- **元素唯一性**：不允许存储重复元素（通过 `hashCode()` 和 `equals()` 方法判断元素是否重复）。  
- **无序性**：元素的存储顺序与插入顺序无关（底层哈希表的存储位置由元素的哈希值决定）。  
- **允许 null 值**：最多只能存储一个 `null` 元素（因不允许重复）。  
- **非线程安全**：多线程环境下并发修改可能导致数据不一致，需手动同步（如使用 `Collections.synchronizedSet()`）。  


### 二、底层实现原理  
`HashSet` 的核心是对 `HashMap` 的封装，其源码简化理解如下：  

```java
public class HashSet<E> extends AbstractSet<E> implements Set<E> {
    // 内部维护的 HashMap，元素存储在 key 中
    private transient HashMap<E, Object> map;
    // 所有 key 对应的固定 value（占位符，无实际意义）
    private static final Object PRESENT = new Object();

    // 构造方法：初始化内部 HashMap
    public HashSet() {
        map = new HashMap<>();
    }

    // 添加元素：本质是向 HashMap 中 put 元素（key 为元素，value 为 PRESENT）
    public boolean add(E e) {
        return map.put(e, PRESENT) == null; 
        // 若 key 不存在，put 返回 null → add 返回 true；若 key 已存在，put 返回旧 value → add 返回 false
    }

    // 删除元素：本质是删除 HashMap 中的 key
    public boolean remove(Object o) {
        return map.remove(o) == PRESENT;
    }

    // 判断是否包含元素：本质是判断 HashMap 中是否包含 key
    public boolean contains(Object o) {
        return map.containsKey(o);
    }
}
```  

简言之：`HashSet` 的所有操作（添加、删除、查询）都是通过操作内部 `HashMap` 的 `key` 实现的，而 `HashMap` 的 `value` 固定为一个无意义的占位对象（`PRESENT`）。  


### 三、元素去重的核心逻辑  
`HashSet` 能去重的关键是通过 **`hashCode()`** 和 **`equals()`** 方法判断元素是否相同，具体流程：  

1. 当添加元素 `e` 时，先调用 `e.hashCode()` 计算哈希值，确定元素在哈希表中的存储位置。  
2. 若该位置**没有元素**，直接存入（认为是新元素）。  
3. 若该位置**已有元素**（哈希冲突），则调用 `e.equals(已有元素)` 比较：  
   - 若 `equals()` 返回 `true`：认为是重复元素，不添加。  
   - 若 `equals()` 返回 `false`：认为是不同元素（哈希值相同但内容不同），以“链表”或“红黑树”形式存储在该位置（解决哈希冲突）。  


#### 重要结论：  
- 自定义对象作为 `HashSet` 元素时，**必须同时重写 `hashCode()` 和 `equals()` 方法**，否则可能导致去重失效（默认的 `hashCode()` 基于对象地址，`equals()` 比较地址，即使内容相同也会被认为是不同元素）。  


**示例：自定义对象重写 `hashCode()` 和 `equals()`**  
```java
import java.util.Objects;

class Student {
    private String id;
    private String name;

    public Student(String id, String name) {
        this.id = id;
        this.name = name;
    }

    // 重写 equals()：当 id 相同时，认为是同一个学生
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        Student student = (Student) o;
        return Objects.equals(id, student.id);
    }

    // 重写 hashCode()：基于 id 计算哈希值（与 equals() 逻辑一致）
    @Override
    public int hashCode() {
        return Objects.hash(id);
    }

    @Override
    public String toString() {
        return "Student{id='" + id + "', name='" + name + "'}";
    }
}
```  

使用 `HashSet` 存储 `Student` 对象：  
```java
import java.util.HashSet;
import java.util.Set;

public class Test {
    public static void main(String[] args) {
        Set<Student> students = new HashSet<>();
        students.add(new Student("001", "张三"));
        students.add(new Student("001", "张三三")); // id 相同，视为重复元素，不添加
        students.add(new Student("002", "李四"));

        System.out.println(students); 
        // 输出：[Student{id='001', name='张三'}, Student{id='002', name='李四'}]
    }
}
```  


### 四、核心方法与使用示例  
`HashSet` 继承了 `Set` 接口的方法，核心操作如下：  


#### 1. 初始化  
```java
import java.util.HashSet;
import java.util.Set;

// 1. 创建空的 HashSet
Set<String> set = new HashSet<>();

// 2. 从其他集合初始化（JDK 9+）
Set<Integer> numbers = new HashSet<>(Set.of(1, 2, 3));
```  


#### 2. 基本操作  
| 方法                | 作用                                  | 示例                                  |  
|---------------------|---------------------------------------|---------------------------------------|  
| `add(E e)`          | 添加元素（成功返回 `true`，重复返回 `false`） | `set.add("apple");`                   |  
| `remove(Object o)`  | 删除元素（存在则删除并返回 `true`）    | `set.remove("banana");`               |  
| `contains(Object o)` | 判断是否包含元素（返回 `true`/`false`） | `boolean hasApple = set.contains("apple");` |  
| `size()`            | 返回元素个数                          | `int size = set.size();`              |  
| `isEmpty()`         | 判断是否为空                          | `boolean empty = set.isEmpty();`      |  
| `clear()`           | 清空所有元素                          | `set.clear();`                        |  


#### 3. 遍历方式  
`HashSet` 支持迭代器、增强 for 循环等遍历方式（无索引，不能用普通 for 循环）：  

```java
Set<String> fruits = new HashSet<>(Set.of("apple", "banana", "orange"));

// 1. 增强 for 循环（推荐）
for (String fruit : fruits) {
    System.out.println(fruit); // 输出顺序不确定（无序）
}

// 2. 迭代器（支持删除操作）
Iterator<String> it = fruits.iterator();
while (it.hasNext()) {
    String fruit = it.next();
    if (fruit.startsWith("a")) {
        it.remove(); // 安全删除当前元素
    }
}
```  


### 五、HashSet 与其他 Set 实现类的对比  
| 实现类          | 底层结构         | 有序性               | 去重依据               | 特点与适用场景                          |  
|-----------------|------------------|----------------------|------------------------|-----------------------------------------|  
| `HashSet`       | 哈希表（HashMap） | 无序                 | `hashCode()` + `equals()` | 查找/添加/删除效率高（`O(1)`），适合无需顺序的去重场景 |  
| `LinkedHashSet` | 哈希表 + 链表    | 有序（插入顺序）     | 同 `HashSet`           | 保留插入顺序，性能略低于 `HashSet`，适合需要顺序的去重场景 |  
| `TreeSet`       | 红黑树           | 有序（自然排序/定制排序） | `compareTo()`（或 `compare()`） | 支持排序和范围查询，效率 `O(log n)`，适合需要排序的场景 |  


### 六、适用场景  
- 需要**去重**且**不关心元素顺序**的场景（如存储用户 ID、标签等唯一标识）。  
- 频繁进行**添加、删除、查找**操作的场景（哈希表的平均时间复杂度为 `O(1)`）。  


### 七、注意事项  
1. **线程安全**：多线程环境下，可通过 `Collections.synchronizedSet()` 包装为线程安全集合：  
   ```java
   Set<String> safeSet = Collections.synchronizedSet(new HashSet<>());
   ```  
2. **哈希冲突**：若元素的 `hashCode()` 实现不佳（如大量元素哈希值相同），会导致哈希冲突频繁，性能下降（退化为链表，时间复杂度接近 `O(n)`）。  
3. **自定义对象**：务必同时重写 `hashCode()` 和 `equals()`，且逻辑保持一致（如 `equals()` 基于 `id`，`hashCode()` 也需基于 `id`）。  


### 总结  
`HashSet` 是基于 `HashMap` 的 `Set` 实现，核心优势是**高效去重**和**快速操作**（添加、删除、查找），但元素无序。其去重逻辑依赖 `hashCode()` 和 `equals()` 方法，自定义对象使用时必须重写这两个方法。适合处理无需顺序的唯一元素场景，是日常开发中去重需求的首选工具。
