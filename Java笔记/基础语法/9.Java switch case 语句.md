Java的`switch case`语句是一种多分支条件控制结构，用于根据**表达式的离散值**匹配不同的分支（`case`），执行对应的代码块。相比`if...else if...else`，它在处理**多个固定常量值判断**时（如菜单选择、状态码匹配）更简洁、易读。


### 一、基本语法结构
```java
switch (表达式) {
    case 常量值1:
        // 表达式的值等于常量值1时执行的代码
        [break;] // 可选，用于跳出switch
    case 常量值2:
        // 表达式的值等于常量值2时执行的代码
        [break;]
    ... // 多个case分支
    default:
        // 所有case都不匹配时执行的代码（可选）
        [break;]
}
```


### 二、核心要素说明
1. **表达式（`switch (表达式)`）**  
   表达式的结果必须是**可比较的离散值**，支持的类型包括：  
   - 整数类型：`byte`、`short`、`int`（`long`在JDK7+支持）；  
   - 字符型：`char`；  
   - 字符串：`String`（JDK7+新增）；  
   - 枚举类型（`enum`）；  
   - 密封类的常量（JDK16+）。  

   注意：表达式结果不能是`boolean`、`float`、`double`等类型。


2. **case标签（`case 常量值:`）**  
   - `常量值`必须是**编译期已知的常量**（如字面量、`final`变量），不能是变量或动态计算的值；  
   - 多个`case`的常量值不能重复（否则编译报错）；  
   - 执行逻辑：当表达式的值与某个`case`的常量值相等时，从该`case`开始执行代码。


3. **break语句**  
   - 作用：执行完当前`case`的代码后，立即跳出`switch`语句，避免继续执行后续`case`（防止“穿透”）；  
   - 若省略`break`，会发生**case穿透**：程序会继续执行下一个`case`的代码，直到遇到`break`或`switch`结束。


4. **default分支**  
   - 可选，当所有`case`的常量值都与表达式的值不匹配时执行；  
   - 通常放在所有`case`之后（无需`break`，但加上更规范），也可放在中间（需加`break`，否则会穿透）。  


### 三、执行流程
```
计算表达式的值 → 依次匹配case的常量值 → 匹配成功 → 从该case开始执行代码 → 遇到break则跳出switch；若未遇到break，继续执行后续case... → 若所有case都不匹配，执行default（若有）
```


### 四、示例解析
#### 1. 基本用法（匹配整数）
```java
int day = 3;
switch (day) {
    case 1:
        System.out.println("星期一");
        break; // 跳出switch
    case 2:
        System.out.println("星期二");
        break;
    case 3:
        System.out.println("星期三"); // 匹配成功，执行此句
        break; // 跳出
    case 4:
        System.out.println("星期四");
        break;
    default:
        System.out.println("其他星期");
}
// 输出：星期三
```


#### 2. case穿透（省略break的效果）
```java
int num = 2;
switch (num) {
    case 1:
        System.out.println("1");
    case 2:
        System.out.println("2"); // 匹配成功，执行
    case 3:
        System.out.println("3"); // 因case 2无break，继续执行
    default:
        System.out.println("默认"); // 继续执行
}
// 输出：
// 2
// 3
// 默认
```
**注意**：穿透是有意为之的特性（如多个case执行相同逻辑），但需谨慎使用，避免逻辑错误。


#### 3. 匹配字符串（JDK7+）
```java
String operation = "add";
switch (operation) {
    case "add":
        System.out.println("执行加法");
        break;
    case "sub":
        System.out.println("执行减法");
        break;
    default:
        System.out.println("未知操作");
}
// 输出：执行加法
```


#### 4. 多个case共享逻辑（利用穿透）
```java
char grade = 'B';
switch (grade) {
    case 'A':
    case 'B': // A和B共享同一逻辑
        System.out.println("优秀");
        break;
    case 'C':
        System.out.println("及格");
        break;
    default:
        System.out.println("不及格");
}
// 输出：优秀（grade为B，匹配case B，执行与A相同的逻辑）
```


### 五、JDK12+ 增强：switch表达式（简化语法）
JDK12引入了**switch表达式**，支持更简洁的语法（箭头`->`），无需`break`，还可直接返回值（用`yield`），进一步减少代码冗余。

#### 语法示例：
```java
int day = 5;
String result = switch (day) {
    case 1 -> "星期一"; // 箭头后直接跟结果，自动跳出
    case 2 -> "星期二";
    case 3 -> "星期三";
    case 4 -> "星期四";
    case 5 -> "星期五";
    default -> "周末"; // 所有不匹配的情况
};
System.out.println(result); // 输出：星期五
```

#### 复杂逻辑（代码块+yield返回值）：
```java
int num = 3;
int square = switch (num) {
    case 1 -> 1;
    case 2 -> 4;
    case 3 -> { // 代码块用{}包裹，yield返回值
        System.out.println("计算3的平方");
        yield 9; // 返回结果
    }
    default -> 0;
};
System.out.println(square); // 输出：9
```


### 六、switch vs if...else if...else
| 场景                     | 推荐使用               | 原因                                                                 |
|--------------------------|------------------------|----------------------------------------------------------------------|
| 判断条件是**离散的常量值**（如1/2/3、"add"/"sub"） | `switch`               | 代码更简洁，分支清晰，执行效率更高（编译时优化）                     |
| 判断条件是**范围或复杂逻辑**（如`x>10`、`a&&b`） | `if...else if...else`  | 支持任意布尔表达式，更灵活                                           |


### 七、注意事项
1. `case`标签必须是**常量**，不能用变量（如`case x:`，`x`是变量则报错）；  
2. 省略`break`会导致穿透，需确保逻辑正确（如共享代码）；  
3. `default`不是必须的，但建议添加（处理未预期的情况，增强健壮性）；  
4. JDK7+才支持`String`类型，低版本需避免使用。  


### 总结
`switch case`是处理多分支常量判断的高效结构，核心通过表达式匹配`case`标签执行代码，`break`用于避免穿透，`default`处理默认情况。JDK12+的增强语法（箭头和`yield`）进一步简化了使用。在合适的场景下（离散常量判断），它比`if...else`更易读、高效。
