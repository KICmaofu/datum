在Java中，**异常（Exception）** 是程序运行时出现的非预期错误（如除以零、数组越界、文件不存在等），会导致程序中断。异常处理机制的核心是**捕获并处理这些错误**，使程序能够优雅地恢复或退出，而不是直接崩溃。


### 一、异常的本质与分类  
Java将异常封装为**对象**，所有异常类都直接或间接继承自`java.lang.Throwable`类。根据性质和处理方式，异常可分为三大类：  


#### 1. `Throwable`的继承体系  
```  
Throwable  
├─ Error（错误）：JVM级别的严重错误，程序无法处理（如内存溢出）  
└─ Exception（异常）：程序可处理的异常  
   ├─ 检查型异常（Checked Exception）：编译时必须处理的异常（如IO异常）  
   └─ 非检查型异常（Unchecked Exception）：运行时异常，编译时无需强制处理（如空指针异常）  
```  


#### 2. 核心分类说明  
- **Error（错误）**：由JVM抛出的严重问题（如`OutOfMemoryError`内存溢出、`StackOverflowError`栈溢出），程序无法恢复，通常无需处理（也处理不了）。  

- **Exception（异常）**：程序运行中可预测的错误，分为：  
  - **检查型异常**：继承自`Exception`（非`RuntimeException`），编译时必须显式处理（捕获或声明抛出），否则编译失败。例如：`IOException`（文件读写错误）、`ClassNotFoundException`（类未找到）。  
  - **非检查型异常**：继承自`RuntimeException`，编译时无需处理，通常是代码逻辑错误导致。例如：`NullPointerException`（空指针）、`ArrayIndexOutOfBoundsException`（数组越界）、`ArithmeticException`（算术错误，如除以零）。  


### 二、异常处理的核心机制：`try-catch-finally`  
Java通过`try`（监控可能出错的代码）、`catch`（捕获并处理异常）、`finally`（释放资源，无论是否异常都会执行）三个块实现异常处理。  


#### 基本语法  
```java
try {
    // 可能抛出异常的代码（监控区域）
} catch (异常类型1 变量名1) {
    // 处理异常类型1的逻辑
} catch (异常类型2 变量名2) {
    // 处理异常类型2的逻辑（可多个catch，顺序从子类到父类）
} finally {
    // 必须执行的代码（如关闭流、释放资源）
}
```  


#### 1. `try`块  
- 用于包裹**可能抛出异常的代码**（如文件读写、类型转换等）。  
- 若代码正常执行，跳过`catch`块，直接执行`finally`。  
- 若代码抛出异常，立即终止`try`块执行，跳转到匹配的`catch`块。  


#### 2. `catch`块  
- 用于**捕获并处理特定类型的异常**，每个`catch`块处理一种或多种异常（用`|`分隔）。  
- 异常类型必须是`Throwable`的子类，且**子类异常必须放在父类异常之前**（否则子类异常的`catch`块会被父类覆盖，编译报错）。  

**示例：捕获多种异常**  
```java
try {
    int[] arr = {1, 2};
    System.out.println(arr[3]); // 数组越界，抛出ArrayIndexOutOfBoundsException
    int num = 10 / 0; // 算术错误，抛出ArithmeticException（若上一行未抛异常才执行）
} catch (ArrayIndexOutOfBoundsException e) { // 子类异常在前
    System.out.println("数组越界：" + e.getMessage()); // 获取异常信息
} catch (ArithmeticException e) {
    System.out.println("算术错误：" + e.getMessage());
} catch (Exception e) { // 父类异常在后，捕获其他未处理的Exception
    System.out.println("其他异常：" + e);
}
```  


#### 3. `finally`块  
- **无论是否发生异常，都会执行**（即使`try`或`catch`中用了`return`）。  
- 主要用于释放资源（如关闭文件流、数据库连接等），避免资源泄漏。  

**示例：`finally`的执行时机**  
```java
public static int test() {
    try {
        int a = 10 / 0; // 抛异常
        return 1;
    } catch (Exception e) {
        return 2; // 执行到return前，先执行finally
    } finally {
        System.out.println("finally执行"); // 必然执行
    }
}

// 调用test()，输出："finally执行"，返回2
```  


### 三、主动抛出异常：`throw`与`throws`  
除了系统自动抛出异常，开发者也可**主动抛出异常**，或**声明方法可能抛出的异常**。  


#### 1. `throw`：手动抛出异常对象  
用于在方法内部主动抛出一个具体的异常对象（通常在满足特定条件时）。  

**示例：手动抛出自定义异常**  
```java
public static void checkAge(int age) {
    if (age < 0 || age > 150) {
        // 主动抛出IllegalArgumentException（非检查型异常）
        throw new IllegalArgumentException("年龄必须在0-150之间：" + age);
    }
    System.out.println("年龄合法：" + age);
}

// 调用
checkAge(200); // 抛出异常：IllegalArgumentException: 年龄必须在0-150之间：200
```  


#### 2. `throws`：声明方法可能抛出的异常  
用于在方法声明处标注该方法可能抛出的异常类型，告知调用者“此方法可能出错，需处理”。  
- 若方法抛出**检查型异常**，必须用`throws`声明（或在方法内用`try-catch`处理），否则编译报错。  
- 若抛出**非检查型异常**（`RuntimeException`及其子类），`throws`可省略（但建议声明，提高可读性）。  

**示例：声明方法抛出检查型异常**  
```java
import java.io.FileNotFoundException;
import java.io.FileReader;

// 方法声明抛出FileNotFoundException（检查型异常）
public static void readFile(String path) throws FileNotFoundException {
    // FileReader构造可能抛出FileNotFoundException（检查型）
    FileReader fr = new FileReader(path); 
}

// 调用者必须处理该异常（try-catch或继续throws）
public static void main(String[] args) {
    try {
        readFile("nonexistent.txt"); // 可能抛异常
    } catch (FileNotFoundException e) {
        System.out.println("文件未找到：" + e);
    }
}
```  


### 四、自定义异常  
Java内置异常可能无法满足业务需求（如“用户不存在”“余额不足”等），此时可自定义异常类，通常继承`Exception`（检查型）或`RuntimeException`（非检查型）。  


#### 自定义异常的步骤  
1. 继承`Exception`或`RuntimeException`。  
2. 提供构造方法（通常复写父类的构造方法，用于传递异常信息）。  


**示例：自定义“余额不足”异常**  
```java
// 自定义检查型异常（继承Exception）
class InsufficientFundsException extends Exception {
    // 无参构造
    public InsufficientFundsException() {
        super();
    }
    // 带异常信息的构造
    public InsufficientFundsException(String message) {
        super(message); // 调用父类构造，传递信息
    }
}

// 使用自定义异常
public class Bank {
    private double balance;

    public void withdraw(double amount) throws InsufficientFundsException {
        if (amount > balance) {
            // 抛出自定义异常
            throw new InsufficientFundsException("余额不足：当前余额" + balance + "，需取出" + amount);
        }
        balance -= amount;
    }

    public static void main(String[] args) {
        Bank bank = new Bank();
        bank.balance = 100;
        try {
            bank.withdraw(200); // 触发异常
        } catch (InsufficientFundsException e) {
            System.out.println("错误：" + e.getMessage()); // 输出异常信息
        }
    }
}
```  


### 五、异常处理的最佳实践  
1. **避免捕获所有异常**：不要用`catch (Exception e)`捕获所有异常（可能掩盖严重错误），应捕获具体异常类型。  
2. **不要忽略异常**：避免空的`catch`块（`catch (Exception e) {}`），至少记录异常信息（如打印日志）。  
3. **`finally`中不使用`return`**：`finally`的`return`会覆盖`try`或`catch`中的`return`值，导致逻辑错误。  
4. **优先处理异常，而非声明抛出**：能在方法内处理的异常（如参数校验失败），尽量用`try-catch`处理，而非`throws`抛给上层。  
5. **自定义异常细化业务错误**：针对业务场景定义异常（如`UserNotFoundException`），使错误处理更清晰。  


### 总结  
Java异常处理的核心是通过`try-catch-finally`捕获并处理运行时错误，通过`throw`主动抛出异常，通过`throws`声明方法可能的异常。关键要点：  
- 异常分为`Error`（无法处理）和`Exception`（可处理），`Exception`又分检查型和非检查型。  
- `try`监控异常，`catch`处理异常，`finally`释放资源。  
- 自定义异常可增强业务错误的可读性和处理效率。  

合理的异常处理能提高程序的健壮性和可维护性，是Java开发的必备技能。
