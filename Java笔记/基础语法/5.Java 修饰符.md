Java修饰符（Modifiers）是用于限定类、方法、变量或构造方法的声明，以控制其**访问权限**、**行为特性**或**生命周期**。修饰符分为两大类：**访问修饰符**（控制访问范围）和**非访问修饰符**（赋予特殊功能）。


### 一、访问修饰符（Access Modifiers）
访问修饰符控制类、成员变量、方法或构造方法对其他类的可见性（访问范围）。Java提供4种访问修饰符：`private`、`default`（默认，无关键字）、`protected`、`public`。


#### 1. 访问范围对比
| 修饰符   | 同一个类中 | 同一个包中（非子类） | 不同包的子类中 | 不同包的非子类中 |
|----------|------------|----------------------|----------------|------------------|
| `private` | ✅ 可访问   | ❌ 不可访问          | ❌ 不可访问     | ❌ 不可访问       |
| `default` | ✅ 可访问   | ✅ 可访问            | ❌ 不可访问     | ❌ 不可访问       |
| `protected` | ✅ 可访问 | ✅ 可访问            | ✅ 可访问       | ❌ 不可访问       |
| `public`  | ✅ 可访问   | ✅ 可访问            | ✅ 可访问       | ✅ 可访问         |  


#### 2. 详细说明及示例
##### （1）`private`（私有）
- **适用对象**：成员变量、方法、构造方法（**不能修饰类和接口**）。  
- **作用**：仅在当前类内部可访问，完全隐藏实现细节，是封装的核心。  
- **示例**：  
  ```java
  public class Person {
      // 私有成员变量（仅类内可直接访问）
      private String name;
      
      // 私有方法（仅类内可调用）
      private void checkName() {
          if (name == null) {
              name = "默认姓名";
          }
      }
      
      // 公共方法（对外暴露接口，间接访问私有成员）
      public String getName() {
          checkName(); // 类内可调用私有方法
          return name;
      }
  }
  ```  


##### （2）`default`（默认，无关键字）
- **适用对象**：类、接口、成员变量、方法、构造方法（不写任何访问修饰符时默认生效）。  
- **作用**：仅在**同一个包**内可访问，跨包不可见。  
- **示例**：  
  ```java
  // 包：com.example.pkg1
  class DefaultClass { // 默认类，仅同包可见
      int age; // 默认变量，同包可见
      
      void print() { // 默认方法，同包可见
          System.out.println("默认方法");
      }
  }
  ```  


##### （3）`protected`（受保护）
- **适用对象**：成员变量、方法、构造方法（**不能修饰类和接口**，除非是内部类）。  
- **作用**：同包内可访问；不同包中，仅子类可访问（通过子类对象或继承的方法）。  
- **示例**：  
  ```java
  // 包：com.example.parent
  public class Parent {
      protected String info = "受保护信息";
      
      protected void protectedMethod() {
          System.out.println("父类受保护方法");
      }
  }
  
  // 包：com.example.child（不同包的子类）
  import com.example.parent.Parent;
  
  public class Child extends Parent {
      public void useProtected() {
          // 子类可访问父类的protected成员
          System.out.println(info); // 输出：受保护信息
          protectedMethod(); // 输出：父类受保护方法
      }
  }
  ```  


##### （4）`public`（公共）
- **适用对象**：类、接口、成员变量、方法、构造方法。  
- **作用**：全局可见，任何包中的任何类都可访问。  
- **示例**：  
  ```java
  // 公共类（可被其他包访问）
  public class PublicClass {
      public int publicVar = 100; // 公共变量
      
      public void publicMethod() { // 公共方法
          System.out.println("公共方法");
      }
  }
  ```  


### 二、非访问修饰符（Non-Access Modifiers）
非访问修饰符不控制访问范围，而是赋予类、方法、变量特殊特性（如静态、不可修改、抽象等）。


#### 1. `static`（静态）
- **适用对象**：变量（静态变量）、方法（静态方法）、代码块（静态代码块）、内部类（静态内部类）。  
- **作用**：被`static`修饰的成员属于**类本身**，而非对象（所有对象共享同一份），可直接通过“类名.成员”访问（无需创建对象）。  
- **示例**：  
  ```java
  public class StaticDemo {
      // 静态变量（类变量，所有对象共享）
      public static int count = 0;
      
      // 静态方法（类方法，可直接通过类名调用）
      public static void increment() {
          count++; // 静态方法可直接访问静态变量
      }
      
      // 静态代码块（类加载时执行，仅执行一次）
      static {
          System.out.println("类加载时执行静态代码块");
      }
  }
  
  // 调用静态成员
  class Test {
      public static void main(String[] args) {
          StaticDemo.increment(); // 直接通过类名调用静态方法
          System.out.println(StaticDemo.count); // 输出：1
      }
  }
  ```  


#### 2. `final`（最终）
- **适用对象**：类、方法、变量。  
- **作用**：  
  - 修饰**类**：类不能被继承（如`String`类是`final`，不可继承）；  
  - 修饰**方法**：方法不能被重写（子类无法修改该方法逻辑）；  
  - 修饰**变量**：变量成为常量，值初始化后不可修改（基本类型值不可变，引用类型地址不可变）。  
- **示例**：  
  ```java
  // final类：不可被继承
  final class FinalClass {}
  // class Sub extends FinalClass {} // 错误：FinalClass是final，不能继承
  
  public class FinalDemo {
      // final变量：值不可修改
      final int MAX = 100;
      
      // final方法：不可被重写
      public final void finalMethod() {
          System.out.println("最终方法");
      }
  }
  
  class SubDemo extends FinalDemo {
      // public void finalMethod() {} // 错误：不能重写final方法
  }
  ```  


#### 3. `abstract`（抽象）
- **适用对象**：类（抽象类）、方法（抽象方法）。  
- **作用**：  
  - 修饰**类**：类不能被实例化（不能`new`），必须被继承，子类需重写所有抽象方法（否则子类也需为抽象类）；  
  - 修饰**方法**：方法只有声明（无实现体），必须在抽象类中，子类必须重写。  
- **示例**：  
  ```java
  // 抽象类：不能实例化
  abstract class AbstractClass {
      // 抽象方法：无实现体，必须被重写
      public abstract void abstractMethod();
      
      // 抽象类可以有非抽象方法
      public void normalMethod() {
          System.out.println("普通方法");
      }
  }
  
  // 子类必须重写抽象方法
  class ConcreteClass extends AbstractClass {
      @Override
      public void abstractMethod() {
          System.out.println("实现抽象方法");
      }
  }
  ```  


#### 4. `synchronized`（同步）
- **适用对象**：方法、代码块。  
- **作用**：保证多线程环境下，同一时刻只有一个线程能执行该方法/代码块（解决线程安全问题）。  
- **示例**：  
  ```java
  public class SyncDemo {
      // 同步方法
      public synchronized void syncMethod() {
          // 多线程下，同一时刻只有一个线程进入
      }
      
      public void someMethod() {
          // 同步代码块（锁定当前对象）
          synchronized (this) {
              // 多线程下，同一时刻只有一个线程进入
          }
      }
  }
  ```  


#### 5. `volatile`（易变）
- **适用对象**：成员变量（基本类型或引用类型）。  
- **作用**：保证变量的**可见性**（一个线程修改后，其他线程能立即看到最新值），禁止指令重排序（用于多线程通信）。  
- **示例**：  
  ```java
  public class VolatileDemo {
      // volatile变量：保证可见性
      private volatile boolean flag = false;
      
      public void setFlag() {
          flag = true; // 线程A修改
      }
      
      public void checkFlag() {
          while (!flag) { 
              // 线程B会立即看到flag的最新值（true），退出循环
          }
      }
  }
  ```  


#### 6. `transient`（瞬态）
- **适用对象**：成员变量。  
- **作用**：当对象被序列化（如写入文件）时，`transient`修饰的变量会被忽略（不参与序列化）。  
- **示例**：  
  ```java
  import java.io.Serializable;
  
  public class TransientDemo implements Serializable {
      String name; // 会被序列化
      transient int password; // 瞬态变量，不会被序列化
  }
  ```  


### 三、修饰符的组合规则
部分修饰符不能同时使用，核心规则：  
- `abstract`与`final`：不能同时修饰类（抽象类需被继承，final类禁止继承）或方法（抽象方法需被重写，final方法禁止重写）；  
- `static`与`abstract`：不能同时修饰方法（静态方法属于类，抽象方法需实例化后重写，矛盾）；  
- 访问修饰符之间：同一元素只能用一个访问修饰符（如不能同时`private`和`public`）；  
- `static`与`this`/`super`：静态方法中不能使用`this`（当前对象）或`super`（父类对象），因为静态成员属于类，与对象无关。  


### 总结
Java修饰符是控制程序结构和行为的关键：  
- **访问修饰符**（`private`/`default`/`protected`/`public`）控制访问范围，核心用于封装和权限管理；  
- **非访问修饰符**赋予特殊功能：`static`（类级成员）、`final`（不可修改）、`abstract`（抽象结构）、`synchronized`（线程安全）等。  

合理使用修饰符能提高代码的安全性、可读性和可维护性，是Java编程的基础技能。
