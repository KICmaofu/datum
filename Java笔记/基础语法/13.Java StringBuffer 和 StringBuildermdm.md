在Java中，`StringBuffer`和`StringBuilder`是用于处理**可变字符串**的类。由于`String`类具有不可变性（修改时会创建新对象），在需要频繁拼接、修改字符串的场景（如循环拼接）中，`StringBuffer`和`StringBuilder`能显著提升效率。它们的核心特点是**内部维护可动态扩容的字符数组**，修改操作直接在原数组上进行，避免频繁创建新对象。


### 一、StringBuffer与StringBuilder的共性  
1. **继承关系**：两者均继承自`AbstractStringBuilder`类，底层通过字符数组（`char[] value`）存储字符串，支持动态扩容。  
2. **核心功能**：提供几乎相同的方法（如`append`、`insert`、`delete`、`reverse`等），用于字符串的拼接、插入、删除、反转等操作。  
3. **与String的转换**：通过`toString()`方法可将其转换为`String`对象（会创建新的`String`，复制有效字符）。  


### 二、核心区别：线程安全性  
两者的唯一核心区别是**线程安全性**：  
- `StringBuffer`：**线程安全**。其所有公开方法都被` synchronized`修饰（同步锁），可在多线程环境中安全使用，但会带来额外的性能开销。  
- `StringBuilder`：**线程不安全**。方法没有同步锁，单线程环境下效率更高（推荐优先使用）。  


### 三、构造方法  
两者的构造方法完全一致，用于初始化对象并指定初始容量（默认容量为16，可减少扩容次数）：  

| 构造方法                          | 说明                                  |
|-----------------------------------|---------------------------------------|
| `StringBuffer()` / `StringBuilder()` | 默认构造，初始容量为16个字符          |
| `StringBuffer(int capacity)` / `StringBuilder(int capacity)` | 指定初始容量（推荐：已知大致长度时使用，减少扩容） |
| `StringBuffer(String str)` / `StringBuilder(String str)` | 以指定字符串初始化，初始容量为`str.length() + 16` |  


**示例**：  
```java
// 初始容量16
StringBuilder sb1 = new StringBuilder();
// 初始容量100（适合长字符串拼接）
StringBuilder sb2 = new StringBuilder(100);
// 以"hello"初始化，容量=5+16=21
StringBuilder sb3 = new StringBuilder("hello");
```  


### 四、常用方法（以StringBuilder为例，StringBuffer用法相同）  
#### 1. 拼接字符串（核心方法）  
`append(...)`：在末尾追加任意类型数据（`int`、`String`、`char`等），返回自身对象（支持链式调用）。  

```java
StringBuilder sb = new StringBuilder("hello");
sb.append(" ").append("world").append(123); // 链式调用
System.out.println(sb.toString()); // 输出：hello world123
```  


#### 2. 插入字符/字符串  
`insert(int offset, ...)`：在指定索引`offset`处插入数据（索引从0开始），返回自身。  

```java
StringBuilder sb = new StringBuilder("abc");
sb.insert(1, "123"); // 在索引1处插入"123"
System.out.println(sb); // 输出：a123bc
```  


#### 3. 删除字符/字符串  
- `delete(int start, int end)`：删除`[start, end)`范围内的字符（包含start，不包含end）。  
- `deleteCharAt(int index)`：删除指定索引处的单个字符。  

```java
StringBuilder sb = new StringBuilder("hello world");
sb.delete(5, 6); // 删除索引5的空格（范围[5,6)）
System.out.println(sb); // 输出：helloworld

sb.deleteCharAt(5); // 删除索引5的'w'
System.out.println(sb); // 输出：helloorld
```  


#### 4. 替换字符/字符串  
`replace(int start, int end, String str)`：将`[start, end)`范围的字符替换为`str`。  

```java
StringBuilder sb = new StringBuilder("hello");
sb.replace(2, 4, "xx"); // 替换索引2-3的"ll"为"xx"
System.out.println(sb); // 输出：hexxo
```  


#### 5. 反转字符串  
`reverse()`：反转整个字符串。  

```java
StringBuilder sb = new StringBuilder("hello");
sb.reverse();
System.out.println(sb); // 输出：olleh
```  


#### 6. 其他常用方法  
- `int length()`：返回当前字符串长度（有效字符数）。  
- `int capacity()`：返回当前字符数组的容量（总可容纳字符数，可能大于长度）。  
- `char charAt(int index)`：返回指定索引的字符。  
- `setCharAt(int index, char ch)`：修改指定索引的字符。  

```java
StringBuilder sb = new StringBuilder("hello");
System.out.println(sb.length()); // 5（有效字符数）
System.out.println(sb.capacity()); // 21（初始容量=5+16）
sb.setCharAt(1, 'E'); 
System.out.println(sb); // 输出：hEllo
```  


### 五、扩容机制  
当字符数量超过当前容量时，`StringBuffer`和`StringBuilder`会自动扩容：  
1. 新容量 = 旧容量 × 2 + 2（JDK 8及以上）。  
2. 若新容量仍不足，则直接扩容到所需的最小容量（如追加的字符串很长）。  
3. 扩容时会复制原字符数组到新数组，因此**初始化时指定合适容量可减少扩容次数，提升性能**。  


### 六、适用场景对比  
| 类          | 线程安全 | 效率   | 适用场景                          |
|-------------|----------|--------|-----------------------------------|
| `String`    | 安全（不可变） | 低（修改创建新对象） | 字符串不频繁修改、直接赋值或少量操作 |
| `StringBuilder` | 不安全   | 高     | 单线程环境，频繁修改字符串（推荐） |
| `StringBuffer`  | 安全     | 中（同步开销） | 多线程环境，频繁修改字符串         |  


### 总结  
- `StringBuffer`和`StringBuilder`是处理可变字符串的工具类，解决`String`修改效率低的问题。  
- 核心区别：`StringBuffer`线程安全（效率低），`StringBuilder`线程不安全（效率高）。  
- 单线程优先使用`StringBuilder`，多线程且需同步时使用`StringBuffer`；初始化时指定容量可优化性能。  

掌握这两个类的用法，能在字符串频繁操作场景中显著提升程序效率。
