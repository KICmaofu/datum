在Java中，**方法（Method）** 是一段封装了特定功能的可重用代码块，用于完成某个独立的操作（如计算、打印、数据处理等）。通过方法，可以实现代码的复用、模块化和可读性提升，是Java程序的基本组成单元之一。


### 一、方法的基本概念  
方法本质是“函数”，但在Java中（面向对象语言），方法必须定义在类中，且分为**实例方法**（需通过对象调用）和**静态方法**（通过类名调用）。  


#### 1. 方法的作用  
- **代码复用**：相同功能的代码无需重复编写，调用方法即可。  
- **模块化**：将复杂逻辑拆分为多个方法，每个方法负责单一功能，便于维护。  
- **可读性**：通过方法名直观了解功能（如`calculateSum()`、`printInfo()`）。  


### 二、方法的定义格式  
方法的完整定义包括**修饰符**、**返回值类型**、**方法名**、**参数列表**和**方法体**，语法如下：  

```java
修饰符 返回值类型 方法名(参数列表) {
    // 方法体（实现功能的代码）
    [return 返回值;] // 若返回值类型非void，必须有return语句
}
```  


#### 各部分说明：  
1. **修饰符**：控制方法的访问权限或特性（可选，可多个组合）。  
   - 访问修饰符：`public`（公开）、`private`（私有）、`protected`（受保护）、默认（无修饰符，同包可见）。  
   - 其他修饰符：`static`（静态方法）、`final`（不可重写）、`abstract`（抽象方法，无方法体）等。  

2. **返回值类型**：方法执行后返回的结果类型。  
   - 若方法无返回值，用`void`表示（此时可省略`return`语句，或用`return;`提前结束方法）。  
   - 若有返回值，需指定具体类型（如`int`、`String`、自定义类等），且方法体中必须有`return 结果;`语句，结果类型需与声明一致。  

3. **方法名**：遵循驼峰命名法（首字母小写，后续单词首字母大写，如`getUserInfo`），需见名知意。  

4. **参数列表**：方法接收的输入数据（可选，无参数则为空括号`()`）。  
   - 格式：`参数类型 参数名1, 参数类型 参数名2, ...`（如`int a, String b`）。  
   - 参数是“形式参数”（简称“形参”），仅在方法内部有效，调用时需传入“实际参数”（简称“实参”）。  

5. **方法体**：用`{}`包裹的代码块，实现方法的具体功能。  


### 三、方法的分类  
按不同维度可分为：  


#### 1. 按返回值类型  
- **有返回值方法**：返回值类型为具体类型（如`int`、`String`），必须有`return`语句。  
  ```java
  // 计算两个数的和（返回int类型）
  public int add(int a, int b) {
      return a + b; // 返回结果
  }
  ```  

- **无返回值方法**：返回值类型为`void`，可省略`return`，或用`return;`提前结束。  
  ```java
  // 打印信息（无返回值）
  public void printMessage(String msg) {
      System.out.println(msg);
      // 无需return，或可写 return;
  }
  ```  


#### 2. 按参数列表  
- **有参方法**：参数列表非空，调用时需传入对应类型的实参。  
  ```java
  // 有两个int参数
  public void showSum(int x, int y) {
      System.out.println(x + y);
  }
  ```  

- **无参方法**：参数列表为空（`()`中无内容），调用时无需传参。  
  ```java
  // 无参数，返回当前时间
  public String getCurrentTime() {
      return LocalDateTime.now().toString();
  }
  ```  


#### 3. 按是否为静态  
- **静态方法（static修饰）**：属于类，通过“类名.方法名()”调用，不能直接访问类的非静态成员（实例变量、实例方法）。  
  ```java
  public class MathUtil {
      // 静态方法：计算平方
      public static int square(int num) {
          return num * num;
      }
  }
  // 调用：类名.方法名
  int result = MathUtil.square(5); // 25
  ```  

- **实例方法（无static修饰）**：属于对象，必须通过“对象.方法名()”调用，可访问类的所有成员（静态/非静态）。  
  ```java
  public class Person {
      private String name;
      
      // 实例方法：设置姓名
      public void setName(String n) {
          name = n; // 访问实例变量
      }
  }
  // 调用：先创建对象，再通过对象调用
  Person p = new Person();
  p.setName("Alice");
  ```  


### 四、方法的调用  
调用方法即执行方法体中的代码，需根据方法类型（静态/实例）使用不同语法，且实参与形参需满足“类型兼容、数量一致、顺序对应”。  


#### 1. 调用步骤  
```java
// 1. 定义方法（以实例方法为例）
public class Calculator {
    // 实例方法：计算乘积
    public int multiply(int a, int b) {
        return a * b;
    }
    
    // 静态方法：计算差
    public static int subtract(int x, int y) {
        return x - y;
    }
}

// 2. 调用方法
public class Test {
    public static void main(String[] args) {
        // 调用实例方法：先创建对象
        Calculator calc = new Calculator();
        int product = calc.multiply(3, 4); // 实参3、4传给形参a、b，返回12
        
        // 调用静态方法：直接用类名
        int difference = Calculator.subtract(10, 5); // 返回5
        
        System.out.println(product); // 12
        System.out.println(difference); // 5
    }
}
```  


#### 2. 实参与形参的关系  
- 形参是方法定义时的变量（如`multiply(int a, int b)`中的`a`、`b`），仅在方法内部有效。  
- 实参是调用时传入的值（如`calc.multiply(3, 4)`中的`3`、`4`），需与形参的类型、数量、顺序一致。  
- **Java中参数传递是“值传递”**：  
  - 若参数是基本类型（`int`、`char`等），传递的是值的副本，方法内修改不会影响外部实参。  
  - 若参数是引用类型（对象、数组等），传递的是对象地址的副本，方法内修改对象的属性会影响外部对象（但修改引用本身不会影响外部引用）。  

  ```java
  // 基本类型参数：修改不影响外部
  public static void changeInt(int num) {
      num = 100; // 仅修改方法内的副本
  }
  int x = 10;
  changeInt(x);
  System.out.println(x); // 仍为10（外部未变）

  // 引用类型参数：修改对象属性会影响外部
  class Person {
      int age;
  }
  public static void changeAge(Person p) {
      p.age = 20; // 修改对象的属性（外部对象也会变）
      p = new Person(); // 修改引用本身（外部引用不受影响）
  }
  Person p = new Person();
  p.age = 10;
  changeAge(p);
  System.out.println(p.age); // 20（属性被修改）
  ```  


### 五、方法的重载（Overload）  
**重载**是指在同一个类中，定义多个**方法名相同、参数列表不同**的方法（与返回值类型、修饰符无关）。目的是让同一功能的方法能适应不同的参数输入。  


#### 重载的规则  
1. 方法名必须相同。  
2. 参数列表必须不同（至少满足以下一点）：  
   - 参数个数不同（如`add(int a)`与`add(int a, int b)`）。  
   - 参数类型不同（如`add(int a)`与`add(double a)`）。  
   - 参数顺序不同（如`add(int a, double b)`与`add(double a, int b)`）。  
3. 返回值类型、修饰符可以不同，但不能仅通过返回值或修饰符区分重载（编译器无法识别）。  


#### 示例：重载的加法方法  
```java
public class Calculator {
    // 1. 两个int相加
    public int add(int a, int b) {
        return a + b;
    }
    
    // 2. 三个int相加（参数个数不同 → 重载）
    public int add(int a, int b, int c) {
        return a + b + c;
    }
    
    // 3. 两个double相加（参数类型不同 → 重载）
    public double add(double a, double b) {
        return a + b;
    }
    
    // 4. int和double相加（参数顺序不同 → 重载）
    public double add(int a, double b) {
        return a + b;
    }
}

// 调用时，编译器根据实参类型自动匹配对应方法
Calculator calc = new Calculator();
System.out.println(calc.add(1, 2)); // 调用add(int,int) → 3
System.out.println(calc.add(1.5, 2.5)); // 调用add(double,double) → 4.0
```  


### 六、方法的递归（Recursion）  
**递归**是指方法自身调用自身的现象，用于解决可拆分为“重复子问题”的场景（如阶乘、斐波那契数列、树的遍历等）。  


#### 递归的条件  
1. **终止条件**：存在一个或多个无需递归就能返回结果的情况（避免无限递归导致栈溢出）。  
2. **递归步骤**：方法能将问题拆解为规模更小的子问题，且子问题解法与原问题一致。  


#### 示例：计算n的阶乘（n! = n × (n-1) × ... × 1）  
```java
public class RecursionDemo {
    // 递归计算阶乘
    public static int factorial(int n) {
        // 终止条件：n=0或n=1时，阶乘为1
        if (n <= 1) {
            return 1;
        }
        // 递归步骤：n! = n × (n-1)!
        return n * factorial(n - 1);
    }

    public static void main(String[] args) {
        System.out.println(factorial(5)); // 5! = 5×4×3×2×1 = 120
    }
}
```  


### 七、方法的作用域  
方法体中定义的变量（包括形参）称为**局部变量**，其作用域仅限于当前方法内部（从定义处到方法结束），在方法外不可访问。  

```java
public void test() {
    int a = 10; // 局部变量，仅在test()中有效
    System.out.println(a); // 正确
}

public void another() {
    System.out.println(a); // 错误：a未定义（超出作用域）
}
```  


### 总结  
方法是Java中封装功能的核心机制，核心要点：  
- 定义格式：修饰符 + 返回值类型 + 方法名 + 参数列表 + 方法体。  
- 调用方式：静态方法用“类名.方法名”，实例方法用“对象.方法名”。  
- 重载：同一类中方法名相同、参数列表不同，提升代码灵活性。  
- 递归：方法自调用，需注意终止条件，适用于子问题拆分场景。  

合理使用方法能显著提升代码的复用性和可维护性，是Java编程的基础技能。
