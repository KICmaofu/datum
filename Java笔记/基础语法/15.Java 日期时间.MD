Java中处理日期和时间的API经历了重大更新：早期的`java.util.Date`、`Calendar`等类因设计缺陷（如线程不安全、API混乱）被诟病；Java 8引入了**`java.time`包**（JSR 310），提供了一套全新的、线程安全的日期时间API，成为目前推荐的使用方式。


### 一、旧日期时间API（了解即可，不推荐使用）  
旧API主要包括`java.util.Date`、`java.util.Calendar`和`java.text.SimpleDateFormat`，存在较多问题，仅作简单介绍。  


#### 1. `java.util.Date`  
- 表示**特定瞬间**（精确到毫秒），但大部分方法（如`getYear()`、`getMonth()`）已过时（被`Calendar`取代）。  
- 设计缺陷：年份从1900开始计算，月份从0开始（0=1月，11=12月），容易出错。  

```java
Date date = new Date(); // 当前时间
System.out.println(date); // 输出：Tue Nov 04 15:30:00 CST 2025（默认格式）
```  


#### 2. `java.util.Calendar`  
- 抽象类，主要实现类为`GregorianCalendar`，用于日期时间的计算（如加减年/月/日）。  
- 问题：线程不安全，API繁琐，月份仍从0开始。  

```java
Calendar cal = Calendar.getInstance(); // 获取当前时间的Calendar实例
cal.set(2025, Calendar.NOVEMBER, 4); // 设置日期（月份用常量，NOVEMBER=10，对应11月）
int year = cal.get(Calendar.YEAR); // 获取年份：2025
int month = cal.get(Calendar.MONTH) + 1; // 月份需+1（10+1=11）
```  


#### 3. `SimpleDateFormat`  
- 用于日期时间的**格式化**（`Date`→`String`）和**解析**（`String`→`Date`）。  
- 问题：**线程不安全**（多线程环境下可能导致数据错乱），因内部有可变的日期格式化状态。  

```java
SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");
// 格式化：Date → String
String dateStr = sdf.format(new Date()); // 如："2025-11-04 15:30:00"
// 解析：String → Date
try {
    Date date = sdf.parse("2025-11-04 15:30:00");
} catch (ParseException e) {
    e.printStackTrace();
}
```  


### 二、新日期时间API（Java 8+，推荐使用）  
Java 8引入的`java.time`包（基于Joda-Time思想）解决了旧API的问题，核心特点：  
- **不可变性**：所有类都是`final`，对象创建后不可修改（线程安全）。  
- **清晰的职责**：日期、时间、带时区的日期时间等拆分为不同类，API直观。  
- **方法命名规范**：如`plusXxx()`（加）、`minusXxx()`（减）、`withXxx()`（修改）等。  


#### 核心类及功能  
| 类名               | 功能描述                                  | 示例                  |
|--------------------|-------------------------------------------|-----------------------|
| `LocalDate`        | 仅包含**日期**（年/月/日），无时间和时区   | 2025-11-04            |
| `LocalTime`        | 仅包含**时间**（时/分/秒/纳秒），无日期    | 15:30:25.123          |
| `LocalDateTime`    | 包含**日期+时间**，无时区                 | 2025-11-04T15:30:25   |
| `ZonedDateTime`    | 包含**日期+时间+时区**（如UTC、CST）      | 2025-11-04T15:30:25+08:00[Asia/Shanghai] |
| `Instant`          | 表示**时间戳**（UTC时间，从1970-01-01 00:00:00开始的毫秒数） | 2025-11-04T07:30:25Z |
| `DateTimeFormatter`| 用于日期时间的格式化和解析（线程安全）    | 格式化模板：yyyy-MM-dd |  


#### 1. 实例创建（获取当前/指定日期时间）  
##### （1）获取当前日期时间  
```java
// LocalDate：当前日期（无时间）
LocalDate today = LocalDate.now(); // 2025-11-04

// LocalTime：当前时间（无日期）
LocalTime nowTime = LocalTime.now(); // 15:35:42.123

// LocalDateTime：当前日期+时间
LocalDateTime now = LocalDateTime.now(); // 2025-11-04T15:35:42.123

// ZonedDateTime：当前带时区的日期时间（默认系统时区）
ZonedDateTime zonedNow = ZonedDateTime.now(); // 包含Asia/Shanghai时区

// Instant：当前时间戳（UTC时间，比北京时间晚8小时）
Instant instant = Instant.now(); // 2025-11-04T07:35:42.123Z（UTC）
```  


##### （2）创建指定日期时间（`of()`方法）  
```java
// 指定日期（年,月,日）：月份从1开始（1=1月，12=12月，无偏移）
LocalDate date = LocalDate.of(2025, 11, 4); // 2025-11-04

// 指定时间（时,分,秒）
LocalTime time = LocalTime.of(15, 30, 25); // 15:30:25

// 指定日期+时间
LocalDateTime dt = LocalDateTime.of(2025, 11, 4, 15, 30, 25); // 2025-11-04T15:30:25
```  


#### 2. 日期时间的格式化与解析（`DateTimeFormatter`）  
`DateTimeFormatter`是线程安全的，取代了`SimpleDateFormat`，支持预定义模板和自定义模板。  


##### （1）格式化（`日期对象 → String`）  
```java
LocalDateTime dt = LocalDateTime.of(2025, 11, 4, 15, 30);

// 预定义模板（如ISO_LOCAL_DATE_TIME）
DateTimeFormatter isoFormatter = DateTimeFormatter.ISO_LOCAL_DATE_TIME;
String isoStr = dt.format(isoFormatter); // "2025-11-04T15:30"

// 自定义模板（常用：yyyy-MM-dd HH:mm:ss）
DateTimeFormatter customFormatter = DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss");
String customStr = dt.format(customFormatter); // "2025-11-04 15:30:00"
```  


##### （2）解析（`String → 日期对象`）  
```java
String dateStr = "2025-11-04 15:30:00";
DateTimeFormatter formatter = DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss");

// 解析为LocalDateTime
LocalDateTime dt = LocalDateTime.parse(dateStr, formatter);

// 解析为LocalDate（仅日期部分，需对应模板）
String str = "2025-11-04";
LocalDate date = LocalDate.parse(str, DateTimeFormatter.ISO_LOCAL_DATE);
```  


#### 3. 日期时间的计算（加减/修改）  
新API的日期时间对象是不可变的，计算后会返回**新对象**，原对象不变。  


##### （1）加减操作（`plusXxx()` / `minusXxx()`）  
```java
LocalDate date = LocalDate.of(2025, 11, 4);

// 加1年、减2个月、加3天
LocalDate newDate = date.plusYears(1)
                       .minusMonths(2)
                       .plusDays(3); 
// 结果：2026-09-07（2025+1=2026；11-2=9；4+3=7）

// LocalDateTime的加减（支持小时、分钟等）
LocalDateTime dt = LocalDateTime.of(2025, 11, 4, 15, 30);
LocalDateTime newDt = dt.plusHours(2).minusMinutes(10); // 17:20
```  


##### （2）修改指定字段（`withXxx()`）  
```java
LocalDate date = LocalDate.of(2025, 11, 4);

// 修改年份为2026，月份为12月
LocalDate modified = date.withYear(2026).withMonth(12); // 2026-12-04
```  


#### 4. 日期时间的比较  
使用`isBefore()`、`isAfter()`、`isEqual()`方法：  
```java
LocalDate date1 = LocalDate.of(2025, 11, 4);
LocalDate date2 = LocalDate.of(2025, 12, 1);

System.out.println(date1.isBefore(date2)); // true（11月4日在12月1日之前）
System.out.println(date1.isAfter(date2));  // false
System.out.println(date1.isEqual(date1));  // true
```  


#### 5. 时区处理（`ZonedDateTime`）  
全球有多个时区（如北京时间是`Asia/Shanghai`，UTC是世界协调时间），`ZonedDateTime`可处理跨时区转换。  

```java
// 获取纽约时区的当前时间（America/New_York）
ZoneId newYorkZone = ZoneId.of("America/New_York");
ZonedDateTime nyTime = ZonedDateTime.now(newYorkZone);

// 北京时区的当前时间（Asia/Shanghai）
ZonedDateTime bjTime = ZonedDateTime.now(ZoneId.of("Asia/Shanghai"));

// 转换：纽约时间 → 北京时间
ZonedDateTime nyToBj = nyTime.withZoneSameInstant(ZoneId.of("Asia/Shanghai"));
```  


#### 6. 与旧API的转换  
若需兼容旧代码，可在新API与旧API间转换：  

```java
// 旧Date → 新Instant（Instant是新API的时间戳）
Date oldDate = new Date();
Instant instant = oldDate.toInstant();

// 新Instant → 旧Date
Instant instant = Instant.now();
Date newDate = Date.from(instant);

// 旧Calendar → 新ZonedDateTime
Calendar cal = Calendar.getInstance();
ZonedDateTime zdt = ZonedDateTime.ofInstant(cal.toInstant(), cal.getTimeZone().toZoneId());
```  


### 三、新旧API对比与推荐  
| 特性         | 旧API（Date/Calendar）       | 新API（java.time）           |
|--------------|------------------------------|------------------------------|
| 线程安全     | 不安全（如SimpleDateFormat） | 安全（不可变对象）           |
| API设计      | 混乱（方法过时、月份0开始）  | 清晰（职责分离、方法直观）   |
| 功能完整性   | 弱（时区处理繁琐）           | 强（原生支持时区、纳秒精度） |
| 推荐程度     | 不推荐（仅兼容旧代码）       | 强烈推荐（新代码首选）       |  


### 总结  
Java 8的`java.time`包是处理日期时间的最佳选择，核心类包括：  
- `LocalDate`（日期）、`LocalTime`（时间）、`LocalDateTime`（日期+时间）：适用于无时区场景。  
- `ZonedDateTime`：处理带时区的日期时间。  
- `DateTimeFormatter`：安全的格式化与解析工具。  

避免使用旧的`Date`、`Calendar`和`SimpleDateFormat`，除非需要兼容历史代码。掌握新API能更简洁、安全地处理各类日期时间需求。
