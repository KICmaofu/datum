Java中的**流（Stream）**、**文件（File）** 和**IO（Input/Output，输入/输出）** 是处理数据传输的核心机制，用于实现程序与外部设备（如文件、网络、键盘、显示器等）之间的数据交互。其中，`File`类用于操作文件和目录的元数据（如创建、删除、查询），而**流（Stream）** 则是数据传输的载体，负责实际的读写操作。


### 一、核心概念  
- **IO**：指数据在程序与外部设备之间的输入（从外部读入程序）和输出（从程序写入外部）过程。  
- **流（Stream）**：数据以“流”的形式传输（类似水流），是字节或字符的有序序列。流具有方向性：  
  - **输入流（Input Stream）**：数据从外部流入程序（读操作）。  
  - **输出流（Output Stream）**：数据从程序流向外部（写操作）。  
- **File类**：封装了文件或目录的路径信息，用于操作文件/目录的属性（如名称、大小、是否存在），但不负责数据的读写。  


### 二、File类：操作文件和目录  
`java.io.File`类用于表示文件或目录的路径，提供了创建、删除、查询文件/目录信息的方法（但不处理文件内容）。  


#### 1. File类的构造方法  
通过路径字符串创建`File`对象（路径可以是绝对路径或相对路径）：  
```java
// 绝对路径（完整路径，如Windows的"C:/test.txt"，Linux的"/home/test.txt"）
File file1 = new File("C:/data/test.txt");

// 相对路径（相对于当前程序的工作目录，如"src/test.txt"）
File file2 = new File("data/logs"); // 表示目录

// 父路径+子路径的组合
File parent = new File("C:/data");
File file3 = new File(parent, "test.txt"); // 等价于"C:/data/test.txt"
```  


#### 2. File类的常用方法  
| 方法                  | 说明                                  |
|-----------------------|---------------------------------------|
| `exists()`            | 判断文件/目录是否存在                  |
| `isFile()`            | 判断是否为文件                        |
| `isDirectory()`       | 判断是否为目录                        |
| `createNewFile()`     | 创建新文件（若不存在，返回true）      |
| `mkdir()`             | 创建单级目录（父目录不存在则失败）    |
| `mkdirs()`            | 创建多级目录（父目录不存在则自动创建）|
| `delete()`            | 删除文件或空目录（非空目录删除失败）  |
| `getName()`           | 获取文件/目录名称                      |
| `getPath()`           | 获取路径字符串（构造时传入的路径）    |
| `getAbsolutePath()`   | 获取绝对路径                          |
| `length()`            | 获取文件大小（字节数，目录返回0）      |
| `listFiles()`         | 返回目录下所有文件/目录的File数组      |  


**示例：创建文件和目录**  
```java
import java.io.File;
import java.io.IOException;

public class FileDemo {
    public static void main(String[] args) {
        // 定义文件路径
        File file = new File("data/test.txt");

        // 若目录不存在，创建多级目录
        File dir = file.getParentFile(); // 获取父目录（data）
        if (!dir.exists()) {
            dir.mkdirs(); // 创建data目录（若不存在）
        }

        // 创建文件
        try {
            if (file.createNewFile()) {
                System.out.println("文件创建成功");
            } else {
                System.out.println("文件已存在");
            }
        } catch (IOException e) {
            e.printStackTrace();
        }

        // 打印文件信息
        System.out.println("是否为文件：" + file.isFile()); // true
        System.out.println("绝对路径：" + file.getAbsolutePath());
        System.out.println("文件大小：" + file.length() + "字节");
    }
}
```  


### 三、流（Stream）：数据读写的核心  
流按**处理数据的单位**分为**字节流**和**字符流**，按**功能**分为**节点流**（直接连接数据源）和**处理流**（包装节点流，增强功能）。  


#### 1. 字节流（Byte Stream）  
以**字节（8位）** 为单位处理数据，适用于所有类型的文件（如文本、图片、音频、视频等）。  
- 基类（抽象类）：  
  - 输入字节流：`InputStream`（核心方法：`int read()` 读取一个字节，返回字节值或-1表示结束）。  
  - 输出字节流：`OutputStream`（核心方法：`void write(int b)` 写入一个字节）。  


##### 常用字节流实现类  
| 流类型         | 作用                                  | 示例场景                          |
|----------------|---------------------------------------|-----------------------------------|
| `FileInputStream` / `FileOutputStream` | 节点流，直接读写文件（字节）          | 复制图片、视频等非文本文件        |
| `BufferedInputStream` / `BufferedOutputStream` | 处理流（缓冲流），包装节点流，提供缓冲区（减少IO次数，提高效率） | 增强文件读写效率                  |
| `ByteArrayInputStream` / `ByteArrayOutputStream` | 节点流，读写内存中的字节数组          | 在内存中临时存储数据              |  


**示例：用字节流复制文件（如图片）**  
```java
import java.io.*;

public class ByteStreamDemo {
    public static void main(String[] args) {
        // 源文件和目标文件路径
        File source = new File("source.jpg");
        File dest = new File("dest.jpg");

        // 声明流对象（try-with-resources语法：自动关闭流，无需手动close()）
        try (InputStream in = new FileInputStream(source);
             OutputStream out = new BufferedOutputStream(new FileOutputStream(dest))) {

            byte[] buffer = new byte[1024]; // 缓冲区（一次读1024字节）
            int len; // 记录每次读取的字节数

            // 循环读取：in.read(buffer)返回读取的字节数，-1表示结束
            while ((len = in.read(buffer)) != -1) {
                out.write(buffer, 0, len); // 写入读取的字节（从0到len）
            }
            System.out.println("文件复制完成");

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```  


#### 2. 字符流（Character Stream）  
以**字符（16位，Java char）** 为单位处理数据，适用于**文本文件**（如`.txt`、`.java`），会涉及字符编码（如UTF-8、GBK）。  
- 基类（抽象类）：  
  - 输入字符流：`Reader`（核心方法：`int read()` 读取一个字符，返回字符值或-1表示结束）。  
  - 输出字符流：`Writer`（核心方法：`void write(int c)` 写入一个字符）。  


##### 常用字符流实现类  
| 流类型         | 作用                                  | 示例场景                          |
|----------------|---------------------------------------|-----------------------------------|
| `FileReader` / `FileWriter` | 节点流，直接读写文本文件（默认编码）  | 读写本地文本文件（编码与系统一致）|
| `BufferedReader` / `BufferedWriter` | 处理流（缓冲流），包装字符流，提供缓冲区，支持`readLine()`（读一行）、`newLine()`（换行） | 高效读写文本，按行处理            |
| `InputStreamReader` / `OutputStreamWriter` | 处理流（转换流），将字节流转换为字符流，可指定编码（如UTF-8） | 处理非默认编码的文本文件          |  


**示例：用字符流读写文本文件（指定UTF-8编码）**  
```java
import java.io.*;

public class CharacterStreamDemo {
    public static void main(String[] args) {
        File file = new File("test.txt");

        // 写文本（使用转换流指定UTF-8编码，包装FileOutputStream）
        try (Writer writer = new OutputStreamWriter(
                new FileOutputStream(file), "UTF-8")) {
            writer.write("Hello, 世界！\n"); // 写入字符串
            writer.write("这是一行文本。");
        } catch (IOException e) {
            e.printStackTrace();
        }

        // 读文本（使用转换流指定UTF-8编码，包装FileInputStream）
        try (Reader reader = new InputStreamReader(
                new FileInputStream(file), "UTF-8");
             BufferedReader br = new BufferedReader(reader)) { // 缓冲流，支持readLine()

            String line;
            while ((line = br.readLine()) != null) { // 逐行读取
                System.out.println(line);
            }

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```  


#### 3. 节点流与处理流的区别  
- **节点流**：直接连接数据源（如文件、内存、网络），是IO操作的基础（如`FileInputStream`、`FileReader`）。  
- **处理流**：不直接连接数据源，而是包装节点流或其他处理流，提供额外功能（如缓冲、转换、对象序列化），增强读写效率或灵活性（如`BufferedInputStream`、`InputStreamReader`）。  

**关系**：处理流依赖节点流，形成“流链”（如`BufferedReader → InputStreamReader → FileInputStream`）。  


### 四、其他常用流  
#### 1. 对象流（Object Stream）  
用于**对象的序列化**（将对象转换为字节序列）和**反序列化**（将字节序列恢复为对象），需通过`ObjectInputStream`和`ObjectOutputStream`实现。  
- 注意：被序列化的类必须实现`Serializable`接口（标记接口，无方法），否则会抛出`NotSerializableException`。  

**示例：对象序列化与反序列化**  
```java
import java.io.*;

// 序列化的类必须实现Serializable
class User implements Serializable {
    private String name;
    private int age;

    public User(String name, int age) {
        this.name = name;
        this.age = age;
    }

    @Override
    public String toString() {
        return "User{name='" + name + "', age=" + age + "}";
    }
}

public class ObjectStreamDemo {
    public static void main(String[] args) {
        User user = new User("Alice", 20);
        File file = new File("user.obj");

        // 序列化：将对象写入文件
        try (ObjectOutputStream oos = new ObjectOutputStream(
                new FileOutputStream(file))) {
            oos.writeObject(user);
            System.out.println("对象序列化完成");
        } catch (IOException e) {
            e.printStackTrace();
        }

        // 反序列化：从文件恢复对象
        try (ObjectInputStream ois = new ObjectInputStream(
                new FileInputStream(file))) {
            User deserializedUser = (User) ois.readObject();
            System.out.println("反序列化结果：" + deserializedUser);
        } catch (IOException | ClassNotFoundException e) {
            e.printStackTrace();
        }
    }
}
```  


#### 2. 打印流（Print Stream/Writer）  
`PrintStream`和`PrintWriter`是处理流，提供方便的打印方法（如`print()`、`println()`），自动转换数据类型为字符串，且不会抛出`IOException`（需通过`checkError()`检查错误）。  
- `System.out`就是`PrintStream`的实例（输出到控制台）。  

**示例：使用PrintWriter写入文本**  
```java
try (PrintWriter pw = new PrintWriter("output.txt", "UTF-8")) {
    pw.println("Hello"); // 自动换行
    pw.print(123);      // 打印数字
    pw.printf("年龄：%d", 20); // 格式化输出
} catch (IOException e) {
    e.printStackTrace();
}
```  


### 五、IO流的关闭与try-with-resources  
流是**有限资源**，使用后必须关闭（释放文件句柄等系统资源），否则会导致资源泄漏。  
- Java 7+提供**try-with-resources语法**：将流对象声明在`try()`中，程序结束后会自动调用`close()`，无需手动关闭，推荐使用。  

```java
// 自动关闭流（无需finally块）
try (InputStream in = new FileInputStream("file.txt")) {
    // 读写操作
} catch (IOException e) {
    e.printStackTrace();
}
```  


### 六、字节流与字符流的选择  
- **字节流**：适用于所有文件类型（文本、图片、音频等），是IO的基础。  
- **字符流**：仅适用于文本文件，自动处理字符编码（需注意编码一致性，避免乱码）。  


### 总结  
Java IO通过`File`类操作文件/目录元数据，通过流（字节流/字符流）处理数据读写：  
- 字节流以字节为单位，适用于所有文件，核心类是`InputStream`/`OutputStream`及其实现类。  
- 字符流以字符为单位，适用于文本文件，核心类是`Reader`/`Writer`及其实现类。  
- 处理流（如缓冲流、转换流）包装节点流，增强功能和效率。  
- 推荐使用try-with-resources语法自动关闭流，避免资源泄漏。  

掌握IO流是实现文件操作、网络通信等功能的基础，需根据场景选择合适的流类型。
