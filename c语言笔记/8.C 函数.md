### 一、C语言中for、while、do-while循环的优缺点对比


| 循环类型       | 优点                                  | 缺点                                  | 适用场景                                  |
|----------------|---------------------------------------|---------------------------------------|-------------------------------------------|
| **for循环**    | 1. 结构紧凑：初始化、条件判断、更新操作集中在`for(...)`中，逻辑清晰，易于维护；<br>2. 循环变量作用域可控：若在初始化中定义循环变量（如`for(int i=0;...)`），则变量仅在循环内有效，减少命名冲突；<br>3. 适合固定次数循环：代码可读性高，一眼可看出循环范围（如`i从0到n-1`）。 | 1. 灵活性较低：若循环次数未知（依赖动态条件），强行使用for会使代码冗余（如省略初始化或更新）；<br>2. 格式严格：需保留两个分号，初学者易漏写导致编译错误。 | 循环次数已知的场景（如遍历数组、计算1到n的和）。 |
| **while循环**  | 1. 灵活性高：仅关注循环条件，初始化和更新操作可放在外部，适合条件动态变化的场景；<br>2. 结构简洁：语法简单，仅需`while(条件)`，适合表达“当满足条件时循环”的逻辑。 | 1. 逻辑分散：初始化和更新操作与循环条件分离，若代码较长，可能导致逻辑混乱（如忘记更新循环变量导致死循环）；<br>2. 循环变量作用域扩大：初始化通常在循环外，变量作用域超出循环体，可能引发命名冲突。 | 循环次数未知，依赖动态条件终止的场景（如等待用户输入特定值、处理文件直到末尾）。 |
| **do-while循环** | 1. 保证循环体至少执行一次：先执行后判断，适合“必须先执行一次”的场景（如输入验证、菜单选择）；<br>2. 避免冗余初始化：无需在循环外提前执行一次循环体代码（对比while循环的“先执行一次再循环”的冗余写法）。 | 1. 适用场景有限：仅当循环体必须至少执行一次时才有优势，否则会导致不必要的执行；<br>2. 条件判断在末尾：若循环体执行后条件仍为真，会继续循环，初学者可能忽略条件更新导致死循环；<br>3. 末尾必须加分号，易漏写。 | 循环体至少需要执行一次的场景（如密码验证、程序菜单选择）。 |


**总结**：三种循环的核心区别在于**执行顺序**和**结构紧凑性**。选择时需根据“循环次数是否已知”“循环体是否必须执行一次”来决定：已知次数用for，未知次数且不必先执行用while，必须先执行一次用do-while。


### 二、C 函数


在C语言中，**函数**是一段完成特定功能的独立代码块，可被重复调用，是模块化编程的核心。通过函数，可将复杂程序拆分为多个小模块，提高代码的可读性、复用性和可维护性。


#### 1. 函数的基本结构
C函数的定义格式如下：
```c
返回值类型 函数名(参数列表) {
    // 函数体（实现功能的代码）
    return 返回值;  // 若返回值类型为void，可省略return
}
```

- **返回值类型**：函数执行后返回的数据类型（如`int`、`float`、`char`等）；若无需返回值，用`void`。
- **函数名**：标识符，遵循C语言命名规则（字母、数字、下划线，首字符非数字），需见名知意（如`sum`、`printInfo`）。
- **参数列表**：函数接收的输入数据（可选），格式为“类型 参数名, 类型 参数名...”；若无需参数，可写`void`或空。
- **函数体**：用`{}`包裹的代码块，包含实现功能的语句。
- **return语句**：结束函数执行，并返回指定值（类型需与返回值类型一致）；若返回值类型为`void`，`return`可省略（或仅写`return;`）。


#### 2. 函数的分类
按来源可分为两类：
- **库函数**：C语言标准库提供的预定义函数（如输入输出、字符串处理等），使用前需包含对应头文件。  
  例：`printf()`（输出，需`#include <stdio.h>`）、`strlen()`（求字符串长度，需`#include <string.h>`）。
- **自定义函数**：用户根据需求自己定义的函数，用于实现特定功能。  


#### 3. 函数的参数与返回值
- **参数**：  
  - 形参（形式参数）：函数定义时参数列表中的参数，仅在函数体内有效，相当于局部变量，用于接收实参的值。  
  - 实参（实际参数）：函数调用时传递的参数，可以是常量、变量或表达式，需与形参的类型、顺序一致。  
  例：
  ```c
  int add(int a, int b) {  // a、b是形参
      return a + b;
  }
  int main() {
      int x = 3, y = 5;
      int res = add(x, y);  // x、y是实参
      return 0;
  }
  ```

- **返回值**：  
  函数执行的结果通过`return`语句返回给调用者，若返回值类型为`void`，则无返回值。  
  注意：`return`语句会立即终止函数，其后的代码不再执行。


#### 4. 函数的调用方式
- **传值调用**：实参将值传递给形参，形参的修改不影响实参（形参与实参是独立的内存空间）。  
  例：
  ```c
  void swap(int a, int b) {  // 形参a、b是实参的副本
      int temp = a;
      a = b;
      b = temp;  // 仅修改形参，实参不变
  }
  int main() {
      int x=1, y=2;
      swap(x, y);
      printf("x=%d, y=%d", x, y);  // 输出：x=1, y=2（实参未变）
      return 0;
  }
  ```

- **传址调用**：通过指针传递实参的地址，形参（指针）指向实参的内存空间，此时修改形参指向的内容会影响实参。  
  例（实现两数交换）：
  ```c
  void swap(int* a, int* b) {  // 形参是指针，接收实参的地址
      int temp = *a;
      *a = *b;  // 修改指针指向的内存（实参的值）
      *b = temp;
  }
  int main() {
      int x=1, y=2;
      swap(&x, &y);  // 传递x、y的地址
      printf("x=%d, y=%d", x, y);  // 输出：x=2, y=1（实参被修改）
      return 0;
  }
  ```


#### 5. 函数的声明与原型
若函数定义在调用之后（或其他文件中），需在调用前进行**函数声明**（告知编译器函数的存在及结构），格式为：  
`返回值类型 函数名(参数列表);`（参数名可省略，仅需类型）。  

例：
```c
// 函数声明（原型）：告知编译器存在一个add函数，接收两个int，返回int
int add(int, int);  

int main() {
    int res = add(3, 5);  // 调用前已声明，编译器可识别
    return 0;
}

// 函数定义（在调用之后，需提前声明）
int add(int a, int b) {
    return a + b;
}
```


#### 6. 递归函数
函数直接或间接调用自身的函数称为**递归函数**，适合解决具有“递归性质”的问题（如阶乘、斐波那契数列、二叉树遍历等）。  

例：计算n的阶乘（n! = n × (n-1) × ... × 1）：
```c
int factorial(int n) {
    if (n == 0 || n == 1) {  // 递归终止条件（必须有，否则无限递归）
        return 1;
    }
    return n * factorial(n - 1);  // 递归调用自身
}
```

**注意**：递归需有明确的终止条件，否则会导致栈溢出（栈内存不足）；递归次数过多时效率较低（相比循环）。


#### 函数的核心作用
- **模块化**：将大程序拆分为小函数，每个函数负责单一功能，便于开发和维护。  
- **代码复用**：同一功能的代码只需编写一次，可在多处调用，减少冗余。  
- **抽象逻辑**：隐藏实现细节，只需关注函数的输入输出（如调用`printf`无需知道其内部如何实现输出）。


通过合理使用函数，可显著提升C程序的质量和开发效率。
