在C语言中，**命令行参数**是程序运行时通过终端（命令行）传递给程序的输入数据。它允许程序在启动时接收外部参数，而无需硬编码输入，使程序更灵活（例如`ls -l`中的`-l`就是传递给`ls`程序的命令行参数）。


### 一、命令行参数的基本形式
C程序通过`main`函数的参数接收命令行参数，`main`函数有两种标准形式：  
```c
// 1. 无参数（不处理命令行参数）
int main() { ... }

// 2. 带参数（用于处理命令行参数）
int main(int argc, char *argv[]) { ... }
```  

其中，两个参数的含义是：  
- `argc`（argument count）：**参数个数**（整数），至少为1（包含程序名本身）。  
- `argv`（argument vector）：**参数数组**（字符串指针数组），每个元素`argv[i]`是一个字符串，对应一个命令行参数。  


### 二、核心概念解析
假设编译后的程序名为`myprog`，在终端执行：  
```bash
./myprog 100 "hello" -v
```  

此时，`main`函数的`argc`和`argv`取值如下：  

| 参数       | 含义                                                                 |
|------------|----------------------------------------------------------------------|
| `argc`     | 4（总共有4个参数：程序名 + 3个用户输入参数）                        |
| `argv[0]`  | `"./myprog"`（程序名本身，通常是执行路径）                           |
| `argv[1]`  | `"100"`（第一个用户参数，字符串类型）                                |
| `argv[2]`  | `"hello"`（第二个用户参数）                                          |
| `argv[3]`  | `"-v"`（第三个用户参数，常作为选项标志）                             |
| `argv[4]`  | `NULL`（数组的结束标志，`argv`的最后一个元素固定为`NULL`）           |  


### 三、访问命令行参数的示例
#### 示例1：打印所有命令行参数
编写程序，遍历并打印所有传递的命令行参数：  
```c
#include <stdio.h>

int main(int argc, char *argv[]) {
    // 打印参数个数
    printf("参数总数：%d\n", argc);

    // 遍历参数数组（argv[0]到argv[argc-1]）
    for (int i = 0; i < argc; i++) {
        printf("argv[%d] = %s\n", i, argv[i]);
    }

    return 0;
}
```  

编译并执行：  
```bash
gcc args.c -o args
./args first 200 "third arg"
```  

输出结果：  
```
参数总数：4
argv[0] = ./args
argv[1] = first
argv[2] = 200
argv[3] = third arg
```  


#### 示例2：根据参数执行不同逻辑
编写程序，接收两个整数参数，计算它们的和（需将字符串参数转换为整数）：  
```c
#include <stdio.h>
#include <stdlib.h>  // 包含atoi函数（字符串转整数）

int main(int argc, char *argv[]) {
    // 检查参数个数是否足够（需要2个用户参数，总argc应为3）
    if (argc != 3) {
        printf("用法：%s <整数1> <整数2>\n", argv[0]);  // 用argv[0]提示程序名
        return 1;  // 参数错误，非0退出
    }

    // 将字符串参数转换为整数（atoi：ASCII to integer）
    int a = atoi(argv[1]);
    int b = atoi(argv[2]);

    // 处理转换失败的情况（如参数不是数字）
    // 注意：atoi转换失败时返回0，需额外判断（更严谨可使用strtol）
    printf("%d + %d = %d\n", a, b, a + b);

    return 0;
}
```  

编译并执行：  
```bash
gcc sum.c -o sum
./sum 15 25   # 输出：15 + 25 = 40
./sum 100 abc # 输出：100 + 0 = 100（abc转换失败，atoi返回0）
./sum 5       # 输出：用法：./sum <整数1> <整数2>（参数个数不足）
```  


### 四、关键注意事项
1. **参数类型都是字符串**：命令行参数无论输入时是数字还是符号，传递给程序时都是`char*`类型的字符串，需手动转换为其他类型（如`int`用`atoi`，`float`用`atof`，更安全的转换用`strtol`/`strtod`）。  

2. **`argc`的最小值为1**：即使不传递任何参数，`argc`也为1（`argv[0]`是程序名）。因此，判断用户是否传递了`n`个参数时，需检查`argc >= n+1`。  

3. **`argv`的结束标志是`NULL`**：`argv`数组以`NULL`结尾（`argv[argc] == NULL`），因此也可通过`while (*argv != NULL)`遍历参数：  
   ```c
   char **p = argv;
   while (*p != NULL) {
       printf("%s\n", *p);
       p++;
   }
   ```  

4. **程序名的不确定性**：`argv[0]`的值由操作系统决定，可能是相对路径（`./myprog`）、绝对路径（`/home/user/myprog`），甚至在某些情况下可能为空（极少数），因此不应依赖`argv[0]`的具体值，仅用于提示用法。  

5. **带空格的参数需用引号包裹**：若参数包含空格（如`"hello world"`），必须用单引号或双引号包裹，否则会被解析为多个参数。例如：  
   ```bash
   ./args "hello world"  # 视为1个参数："hello world"
   ./args hello world    # 视为2个参数："hello"和"world"
   ```  


### 五、进阶：解析选项参数（如`-v`、`-h`）
命令行参数常包含“选项标志”（如`ls -l`中的`-l`，`gcc -O2`中的`-O2`），用于控制程序行为。可通过手动判断`argv[i]`的前缀（如`-`）来解析：  

#### 示例：支持`-v`（verbose）选项的程序
```c
#include <stdio.h>
#include <string.h>  // 包含strcmp函数

int main(int argc, char *argv[]) {
    int verbose = 0;  // 是否启用详细模式

    // 解析选项参数（遍历所有参数）
    for (int i = 1; i < argc; i++) {
        if (strcmp(argv[i], "-v") == 0) {  // 检查是否为-v选项
            verbose = 1;
        }
    }

    // 根据选项执行逻辑
    if (verbose) {
        printf("详细模式：程序启动成功\n");
    } else {
        printf("简洁模式：启动\n");
    }

    return 0;
}
```  

执行结果：  
```bash
./option          # 输出：简洁模式：启动
./option -v       # 输出：详细模式：程序启动成功
./option abc -v xyz  # 输出：详细模式：程序启动成功（只要包含-v就生效）
```  


### 六、更复杂的参数解析：`getopt`函数
对于包含大量选项（如`-a -b value -c`）的程序，手动解析繁琐，可使用标准库的`getopt`函数（`<unistd.h>`），它支持短选项（如`-v`）和长选项（如`--verbose`）的解析。  

#### 示例：用`getopt`解析短选项
```c
#include <stdio.h>
#include <unistd.h>  // 包含getopt

int main(int argc, char *argv[]) {
    int opt;
    int a_flag = 0, b_flag = 0;
    char *c_value = NULL;

    // getopt循环解析选项：选项字符串"ab:c:"表示
    // -a：无参数；-b：需参数；-c：需参数
    while ((opt = getopt(argc, argv, "ab:c:")) != -1) {
        switch (opt) {
            case 'a':
                a_flag = 1;
                break;
            case 'b':
                b_flag = 1;
                printf("b的参数：%s\n", optarg);  // optarg存储选项后的参数
                break;
            case 'c':
                c_value = optarg;
                break;
            case '?':  // 未知选项或缺少参数
                return 1;
            default:
                return 1;
        }
    }

    // 打印解析结果
    if (a_flag) printf("已启用-a选项\n");
    if (b_flag) printf("已启用-b选项\n");
    if (c_value) printf("c的参数：%s\n", c_value);

    return 0;
}
```  

执行结果：  
```bash
./getopt_demo -a -b 123 -c test
# 输出：
# b的参数：123
# 已启用-a选项
# 已启用-b选项
# c的参数：test
```  


### 总结
C语言通过`main`函数的`argc`和`argv`接收命令行参数，核心是：  
- `argc`记录参数总数（含程序名）；  
- `argv`是字符串数组，存储具体参数（以`NULL`结尾）。  

命令行参数使程序能在运行时动态接收输入，常用于工具类程序（如编译器、脚本解释器）。简单场景可手动遍历解析，复杂场景（多选项）可使用`getopt`函数。使用时需注意参数类型转换和个数检查，避免错误。
