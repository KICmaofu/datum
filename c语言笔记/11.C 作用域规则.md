在C语言中，**作用域规则**（Scope Rules）定义了变量、函数或其他标识符（如结构体标签、枚举常量等）在程序中**可被访问的有效范围**。简单来说，作用域决定了“在代码的哪个位置可以使用某个标识符”。


C语言的作用域主要分为以下几类，每种作用域有明确的边界和访问规则：


### 一、局部作用域（块作用域，Block Scope）
**定义**：在**代码块**（用 `{}` 括起来的区域，如函数体、循环体、分支体等）内定义的标识符，其作用域仅限于该代码块内部（包括嵌套的子块）。  

**特点**：  
- 变量在代码块内定义，进入块时创建，离开块时销毁（生命周期与块的执行同步）。  
- 子块（嵌套的 `{}`）可以访问父块的变量，但父块不能访问子块的变量（作用域嵌套，内层可访问外层，外层不可访问内层）。  
- 同一作用域内不能定义同名标识符（变量、函数等），但不同作用域可重名（内层会屏蔽外层，称为“名字隐藏”）。  


**示例1：函数体内的局部变量**  
```c
#include <stdio.h>

void func() {
    int a = 10;  // 作用域：整个func函数体
    printf("func内：a = %d\n", a);  // 合法：在作用域内

    if (1) {
        int b = 20;  // 作用域：仅当前if块内
        printf("if块内：a = %d, b = %d\n", a, b);  // 合法：a（父块）和b（本块）均在作用域内
    }

    // printf("func内访问b：%d\n", b);  // 错误：b的作用域已结束（离开if块）
}

int main() {
    // printf("main内访问a：%d\n", a);  // 错误：a的作用域仅限func函数体
    return 0;
}
```


**示例2：作用域嵌套与名字隐藏**  
```c
#include <stdio.h>

int main() {
    int x = 100;  // 外层作用域（main函数体）的x
    printf("外层x：%d\n", x);  // 输出：100

    if (1) {
        int x = 200;  // 内层作用域（if块）的x，与外层重名
        printf("内层x：%d\n", x);  // 输出：200（内层x屏蔽外层x）
    }

    printf("外层x：%d\n", x);  // 输出：100（内层x不影响外层）
    return 0;
}
```


### 二、全局作用域（文件作用域，File Scope）
**定义**：在**所有函数外部**定义的标识符（如全局变量、函数、全局结构体等），其作用域从定义位置开始，延伸到**整个源文件的末尾**。  

**特点**：  
- 全局变量的生命周期是**整个程序运行期间**（从程序启动到结束），在内存中固定存在。  
- 同一源文件中，全局标识符可被定义位置之后的所有函数访问（无需额外声明）。  
- 其他源文件可通过 `extern` 声明引用全局标识符（前提是该标识符未被 `static` 修饰，即具有外部链接属性）。  
- 全局变量若未显式初始化，默认值为0（数值型）或 `'\0'`（字符型）。  


**示例：全局变量的作用域**  
```c
#include <stdio.h>

int global = 50;  // 全局变量，作用域：从这里到文件结束

void func1() {
    printf("func1访问global：%d\n", global);  // 合法：在全局作用域内
}

void func2() {
    global = 100;  // 修改全局变量
    printf("func2修改后global：%d\n", global);  // 合法
}

int main() {
    printf("main访问global：%d\n", global);  // 合法：输出50
    func1();  // 输出：50
    func2();  // 输出：100
    printf("main再次访问global：%d\n", global);  // 输出：100（已被修改）
    return 0;
}
```


### 三、函数原型作用域（Function Prototype Scope）
**定义**：在**函数原型声明**中出现的参数名，其作用域仅限于该原型声明本身。  

**特点**：  
- 函数原型的参数名仅用于说明参数类型，不影响函数定义或调用（甚至可以省略参数名，只需保留类型）。  
- 原型中的参数名与函数定义中的参数名可以不同（作用域相互独立）。  


**示例：函数原型作用域**  
```c
#include <stdio.h>

// 函数原型：参数a、b的作用域仅限于此原型
int add(int a, int b);  // 等价于 int add(int, int);（可省略参数名）

// 函数定义：参数x、y的作用域是函数体，与原型中的a、b无关
int add(int x, int y) {
    return x + y;
}

int main() {
    printf("%d\n", add(3, 5));  // 输出：8
    return 0;
}
```


### 四、函数作用域（Function Scope）
**定义**：仅适用于**goto语句的标签**，标签的作用域是**整个函数体**（无论标签定义在函数的哪个代码块中，整个函数内都可通过 `goto` 跳转）。  

**特点**：  
- 同一函数内不能有同名标签（函数作用域内唯一）。  
- 标签仅能被本函数的 `goto` 语句使用，不能跨函数跳转。  


**示例：函数作用域（goto标签）**  
```c
#include <stdio.h>

void func() {
    int i = 0;
    if (i == 0) {
        goto label;  // 跳转到label标签（即使label在后面的块中）
    }

    {
        label:  // 标签的作用域是整个func函数
            printf("跳转到label\n");  // 会被执行
    }
}

int main() {
    func();  // 输出：跳转到label
    // goto label;  // 错误：label的作用域仅限func函数
    return 0;
}
```


### 五、作用域与存储类的关系
作用域（可见性）与存储类（生命周期、链接属性）密切相关，但并非同一概念：  
- 局部作用域的变量通常是 `auto` 或 `register` 存储类（生命周期与块同步），或 `static` 局部变量（生命周期是整个程序，但作用域仍在块内）。  
- 全局作用域的变量通常是普通全局变量（外部链接，可跨文件访问）或 `static` 全局变量（内部链接，仅本文件可见）。  


### 总结
C语言的作用域规则核心是“标识符的可见范围”，主要分为：  
- **局部作用域**：代码块内（函数体、循环/分支块等），内层可访问外层，外层不可访问内层。  
- **全局作用域**：从定义处到文件结束，可被同文件后续函数访问，或通过 `extern` 被其他文件访问。  
- **函数原型作用域**：仅函数原型内的参数名有效。  
- **函数作用域**：goto标签在整个函数内可见。  

理解作用域规则可避免“变量未定义”“重名冲突”等错误，使代码逻辑更清晰。
