在C语言中，没有像C++/Java那样的**异常处理机制**（try/catch），错误处理主要依赖**显式的返回值检查**、**全局错误码**和**信号机制**，需要开发者主动设计和处理错误场景。以下是C语言中常用的错误处理方式及实践。


### 一、返回值检查：最基础的错误处理
C语言中，函数通常通过**返回特定值**表示成功或失败，调用者需显式检查返回值以判断是否出错。这是最直接、最常用的错误处理方式。


#### 1. 标准库函数的返回值约定
许多C标准库函数通过返回“特殊值”标识错误，常见约定：  
- 返回指针的函数（如`fopen`、`malloc`）：失败时返回`NULL`；  
- 返回整数的函数（如`fread`、`write`）：失败时返回`-1`；  
- 其他特殊约定（如`strcmp`返回0表示相等，非0表示不等，但不代表错误）。  


**示例1：文件操作错误检查**  
`fopen`打开文件失败时返回`NULL`，需检查返回值：  
```c
#include <stdio.h>

int main() {
    FILE *file = fopen("test.txt", "r");  // 尝试打开文件
    if (file == NULL) {  // 检查返回值：若为NULL，说明打开失败
        printf("无法打开文件！\n");
        return 1;  // 非0退出表示程序异常
    }

    // 文件操作...
    fclose(file);
    return 0;  // 0表示程序正常结束
}
```


**示例2：内存分配错误检查**  
`malloc`分配内存失败时返回`NULL`（尤其在内存不足时）：  
```c
#include <stdlib.h>
#include <stdio.h>

int main() {
    int *arr = (int*)malloc(1000 * sizeof(int));  // 分配内存
    if (arr == NULL) {  // 检查是否分配成功
        printf("内存分配失败！\n");
        return 1;
    }

    // 使用内存...
    free(arr);
    return 0;
}
```


#### 2. 自定义函数的返回值设计
自定义函数时，需明确返回值的“成功/失败”语义，常见约定：  
- 返回`0`表示成功，非`0`表示错误码（不同非0值对应不同错误类型）；  
- 若函数需返回数据，可通过指针参数传出，返回值仅用于标识错误。  


**示例：自定义函数的错误返回**  
```c
#include <stdio.h>

// 功能：计算a除以b的结果，存入result
// 返回值：0=成功，1=除数为0（错误）
int divide(int a, int b, int *result) {
    if (b == 0) {
        return 1;  // 错误码1：除数为0
    }
    *result = a / b;
    return 0;  // 成功
}

int main() {
    int res;
    int err = divide(10, 0, &res);  // 调用函数，传入结果指针
    if (err != 0) {  // 检查错误码
        printf("错误：除数不能为0！\n");
        return 1;
    }
    printf("结果：%d\n", res);  // 仅在无错误时使用结果
    return 0;
}
```


### 二、`errno`：全局错误码变量
C标准库定义了一个全局变量`errno`（需包含`<errno.h>`），用于存储**系统调用或库函数的具体错误原因**。当函数返回错误标识（如`-1`或`NULL`）时，`errno`会被设置为对应的错误码（非0值），通过错误码可定位具体错误类型。


#### 1. `errno`的使用规则
- `errno`仅在函数返回“错误标识”时有效（如`fopen`返回`NULL`时），成功时`errno`的值不确定（可能保留上次错误）；  
- 不同错误对应不同的宏定义（如`ENOENT`表示“文件不存在”，`EIO`表示“I/O错误”），详见`<errno.h>`；  
- 可通过`perror`或`strerror`函数将`errno`转换为可读的错误信息。  


#### 2. `perror`与`strerror`：打印错误信息
- `perror(const char *msg)`：先打印`msg`，再打印`": "`和当前`errno`对应的错误描述（自动换行）；  
- `strerror(int errnum)`：返回`errnum`对应的错误描述字符串（需手动打印）。  


**示例：用`errno`和`perror`定位错误**  
```c
#include <stdio.h>
#include <errno.h>  // 包含errno定义
#include <string.h>  // 包含strerror

int main() {
    FILE *file = fopen("nonexistent.txt", "r");
    if (file == NULL) {
        // 方式1：用perror打印错误
        perror("fopen failed");  // 输出：fopen failed: No such file or directory

        // 方式2：用strerror打印错误（更灵活）
        printf("错误原因：%s\n", strerror(errno));  // 输出：错误原因：No such file or directory
        return 1;
    }

    fclose(file);
    return 0;
}
```


#### 3. 常见`errno`错误码
| 错误码宏   | 含义（示例）                  |
|------------|-------------------------------|
| `ENOENT`   | 文件或目录不存在              |
| `EIO`      | I/O操作失败                  |
| `ENOMEM`   | 内存不足（如malloc失败）      |
| `EACCES`   | 权限不足                      |
| `EBUSY`    | 资源正忙（如删除正在使用的文件） |


### 三、断言（`assert`）：调试阶段的错误检查
`assert`是C语言提供的**调试宏**（需包含`<assert.h>`），用于检查程序运行时的“逻辑正确性”（如变量取值范围、指针非空等）。若断言条件为假，程序会打印错误信息并终止（调用`abort()`）。


#### 1. `assert`的语法与作用
```c
assert(条件表达式);
```  
- 若“条件表达式”为`0`（假），则触发断言失败：打印包含文件名、行号、表达式的错误信息，然后终止程序；  
- 若为非0（真），则无操作，程序继续执行。  


**示例：用`assert`检查指针非空**  
```c
#include <stdio.h>
#include <assert.h>

void print_int(int *p) {
    // 调试阶段检查：指针p不能为NULL（否则触发断言）
    assert(p != NULL);  // 若p为NULL，程序终止并提示错误
    printf("%d\n", *p);
}

int main() {
    int x = 10;
    print_int(&x);  // 正常执行：输出10

    int *null_ptr = NULL;
    print_int(null_ptr);  // 断言失败：打印错误并终止
    return 0;
}
```


#### 2. `assert`与错误处理的区别
- `assert`主要用于**调试阶段**，检查“不应该发生的逻辑错误”（如内部参数异常），不能替代运行时错误处理（如用户输入错误、文件不存在）；  
- 发布版本中可通过定义`NDEBUG`宏禁用`assert`（避免性能损耗）：  
  ```c
  #define NDEBUG  // 放在#include <assert.h>之前，禁用所有assert
  #include <assert.h>
  ```  


### 四、信号（`signal`）：处理异步错误
信号是操作系统向进程发送的“事件通知”，用于处理**异步错误**（如除零、段错误、用户中断等）。C语言通过`<signal.h>`中的`signal`函数捕获并处理信号。


#### 1. 常见信号类型
| 信号   | 含义                          |
|--------|-------------------------------|
| `SIGSEGV` | 段错误（访问无效内存地址）    |
| `SIGFPE`  | 算术错误（如除零、浮点溢出）  |
| `SIGINT`  | 终端中断（用户按Ctrl+C）      |
| `SIGTERM` | 终止信号（请求进程退出）      |


#### 2. `signal`函数的使用
```c
#include <signal.h>
void (*signal(int signum, void (*handler)(int)))(int);
```  
- 作用：为`signum`指定的信号注册处理函数`handler`；  
- `handler`是一个函数指针，参数为信号值，返回值为`void`；  
- 特殊处理函数：`SIG_IGN`（忽略信号）、`SIG_DFL`（使用默认处理，通常是终止程序）。  


**示例：捕获`SIGINT`（Ctrl+C）信号**  
```c
#include <stdio.h>
#include <signal.h>
#include <stdlib.h>

// 信号处理函数：当收到SIGINT时调用
void handle_sigint(int signum) {
    printf("\n收到中断信号（%d），程序即将退出...\n", signum);
    exit(0);  // 正常退出
}

int main() {
    // 注册SIGINT信号的处理函数
    signal(SIGINT, handle_sigint);

    printf("程序运行中，按Ctrl+C尝试中断...\n");
    while (1) {
        // 无限循环，等待信号
    }
    return 0;
}
```


**注意**：信号处理函数应尽量简单（如只做清理和退出），避免复杂操作（如I/O、内存分配），否则可能导致程序不稳定。


### 五、错误处理的最佳实践
1. **永远检查返回值**：对库函数（如`fopen`、`malloc`）和自定义函数的返回值，必须显式检查，避免忽略错误；  
2. **使用`errno`细化错误信息**：当函数返回错误时，通过`perror`或`strerror`打印具体原因，方便调试；  
3. **区分调试与运行时错误**：`assert`用于调试阶段的逻辑检查，运行时错误（如用户输入无效）需用返回值+错误码处理；  
4. **及时清理资源**：出错时需释放已分配的资源（如关闭文件、释放内存），避免内存泄漏或资源占用；  
   ```c
   // 示例：出错时释放资源
   FILE *file = fopen("test.txt", "r");
   if (file == NULL) { /* 处理错误 */ }

   int *buf = malloc(1024);
   if (buf == NULL) {
       fclose(file);  // 先关闭已打开的文件
       perror("malloc failed");
       return 1;
   }
   ```  
5. **自定义错误码需文档化**：自定义函数的错误码（如返回1表示“参数无效”，2表示“资源不足”）需明确说明，方便调用者处理。  


### 总结
C语言的错误处理依赖**显式返回值检查**、**全局`errno`**、**断言**和**信号**，核心原则是“主动检查、明确标识、及时处理”。由于没有异常机制，开发者需在代码中处处留意可能的错误场景，才能保证程序的健壮性。
