在C语言中，**函数指针（Function Pointer）** 是一种特殊的指针，它指向函数的内存地址（函数在内存中占据一块连续的存储空间，其起始地址就是函数的入口地址）。通过函数指针，可以像调用函数一样调用它所指向的函数，这使得代码更加灵活（尤其是在回调函数、函数表等场景中）。


### 一、函数指针的定义与声明
函数指针的声明需要明确它所指向的函数的**返回类型**和**参数列表**（因为函数的类型由返回类型和参数列表共同决定）。  

**基本格式**：  
```c
返回类型 (*指针变量名)(参数类型1, 参数类型2, ...);
```

- `(*指针变量名)` 中的括号不可省略：若省略，`返回类型 *指针变量名(参数)` 会被解析为“返回指针的函数”（即指针函数），而非函数指针。  
- `返回类型`：函数指针所指向的函数的返回值类型。  
- `参数类型1, ...`：函数指针所指向的函数的参数类型列表。  


#### 示例：声明指向特定函数的指针
假设有一个加法函数：  
```c
int add(int a, int b) {
    return a + b;
}
```

指向该函数的函数指针声明如下：  
```c
int (*pAdd)(int, int);  // pAdd是函数指针，指向“返回int、接收两个int参数”的函数
```


### 二、函数指针的初始化与赋值
函数名本身就是函数的入口地址（类似数组名是数组首地址），因此可以直接将函数名赋值给函数指针，完成初始化或赋值。  


#### 示例：为函数指针赋值
```c
int add(int a, int b) { return a + b; }
int sub(int a, int b) { return a - b; }

int main() {
    int (*pFunc)(int, int);  // 声明函数指针

    pFunc = add;  // 赋值：pFunc指向add函数（函数名即地址）
    // 或 pFunc = &add;  // &可以省略，效果相同（函数名本身就是地址）

    pFunc = sub;  // 重新赋值：pFunc指向sub函数
    return 0;
}
```


### 三、通过函数指针调用函数
函数指针指向函数后，可以通过两种方式调用该函数：  
1. 直接用函数指针名调用（类似函数名）；  
2. 用解引用运算符`*`获取指针指向的函数后调用。  


#### 示例：通过函数指针调用函数
```c
#include <stdio.h>

int add(int a, int b) { return a + b; }
int sub(int a, int b) { return a - b; }

int main() {
    int (*pFunc)(int, int);

    pFunc = add;
    // 调用方式1：直接用指针名（等价于调用add(3,5)）
    printf("3 + 5 = %d\n", pFunc(3, 5));  // 输出：8

    pFunc = sub;
    // 调用方式2：解引用后调用（等价于调用sub(10,4)）
    printf("10 - 4 = %d\n", (*pFunc)(10, 4));  // 输出：6

    return 0;
}
```

**说明**：两种调用方式完全等价，实际中更常用第一种（简洁）。


### 四、函数指针的核心应用场景
函数指针的灵活性使其在很多场景中不可或缺，最典型的包括**回调函数**和**函数表**。


#### 1. 回调函数（Callback Function）
回调函数是指**被作为参数传递给另一个函数，并在该函数内部被调用**的函数。通过函数指针实现，允许“调用者”动态指定被调用的函数，实现“反向调用”。  

**示例：通用计算器函数（支持动态指定运算）**  
```c
#include <stdio.h>

// 定义加法、减法、乘法函数
int add(int a, int b) { return a + b; }
int sub(int a, int b) { return a - b; }
int mul(int a, int b) { return a * b; }

// 计算器函数：接收两个数和一个运算函数的指针（回调函数的指针）
int calculate(int a, int b, int (*op)(int, int)) {
    return op(a, b);  // 调用回调函数
}

int main() {
    int x = 10, y = 5;

    // 动态指定不同的运算（传递不同的函数指针）
    printf("%d + %d = %d\n", x, y, calculate(x, y, add));   // 15
    printf("%d - %d = %d\n", x, y, calculate(x, y, sub));   // 5
    printf("%d * %d = %d\n", x, y, calculate(x, y, mul));   // 50

    return 0;
}
```

**解析**：`calculate`函数不关心具体的运算逻辑，只负责调用传入的`op`函数（回调函数），因此可以通过传递不同的函数指针（`add`/`sub`/`mul`）实现不同运算，大幅提高了代码的通用性。


#### 2. 函数表（Function Table）
函数表是**存储多个同类型函数指针的数组**，通过索引访问不同的函数，实现多分支逻辑（比`switch`更灵活，尤其适合大量分支的场景）。  

**示例：用函数表实现菜单选择**  
```c
#include <stdio.h>

// 定义4个功能函数（返回类型和参数一致，可放入同一函数表）
void func1() { printf("执行功能1\n"); }
void func2() { printf("执行功能2\n"); }
void func3() { printf("执行功能3\n"); }
void func4() { printf("执行功能4\n"); }

int main() {
    // 函数表：存储4个void(*)()类型的函数指针
    void (*funcTable[])() = {func1, func2, func3, func4};
    int choice;

    printf("请选择功能（1-4）：");
    scanf("%d", &choice);

    if (choice >= 1 && choice <= 4) {
        funcTable[choice - 1]();  // 通过索引调用对应函数（choice-1是数组下标）
    } else {
        printf("无效选择\n");
    }

    return 0;
}
```

**解析**：若有更多功能，只需在函数表中添加新的函数指针，无需修改`switch`的分支逻辑，扩展性更强。


### 五、typedef简化函数指针声明
函数指针的声明格式较繁琐，尤其是复杂类型（如参数或返回值包含指针）。通过`typedef`为函数指针类型定义别名，可以简化声明。  

**示例：用typedef定义函数指针类型**  
```c
#include <stdio.h>

// 为“返回int、接收两个int参数”的函数指针定义别名FuncPtr
typedef int (*FuncPtr)(int, int);

// 加法函数
int add(int a, int b) { return a + b; }

int main() {
    FuncPtr p;  // 用别名声明函数指针（等价于int (*p)(int, int);）
    p = add;
    printf("%d\n", p(2, 3));  // 输出：5
    return 0;
}
```

**优势**：当函数指针类型复杂时（如`void (*)(int, char*)`），别名能显著提高代码可读性。


### 六、注意事项
1. **类型匹配**：函数指针的返回类型、参数类型和数量必须与它所指向的函数完全一致，否则会导致未定义行为（如程序崩溃）。  
   ```c
   int add(int a, int b);
   int (*p)(float, int);  // 错误：参数类型不匹配（int vs float），不能指向add
   ```  

2. **函数指针与指针函数的区别**：  
   - 函数指针：指向函数的指针（本质是指针），格式`返回类型 (*指针名)(参数)`。  
   - 指针函数：返回值为指针的函数（本质是函数），格式`返回类型* 函数名(参数)`。  
   例如：`int* func(int a)`是指针函数（返回int*），而`int (*p)(int)`是函数指针（指向返回int的函数）。  


### 总结
函数指针是C语言中实现“行为抽象”的重要工具，通过存储函数地址并动态调用，实现了代码的灵活性和通用性。核心应用包括回调函数（如排序算法中的比较函数、事件响应）和函数表（多分支逻辑）。使用时需注意类型匹配，并可通过`typedef`简化声明。掌握函数指针是理解C语言高级特性（如库函数设计、模块化编程）的关键。
