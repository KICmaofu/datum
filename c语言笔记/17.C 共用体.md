在C语言中，**共用体（Union，也称为联合体）** 是一种用户自定义的数据类型，它与结构体（Struct）类似，都可以包含多个不同类型的成员，但核心区别在于：**共用体的所有成员共享同一块内存空间**，而非像结构体那样每个成员占用独立的空间。这使得共用体适合在“不同时刻存储不同类型数据”的场景中节省内存。


### 一、共用体的定义
共用体通过 `union` 关键字定义，基本格式如下：  
```c
union 共用体名 {
    数据类型 成员1;  // 共用体的成员（不同类型）
    数据类型 成员2;
    // ... 更多成员
};
```

- **共用体名**：遵循标识符命名规则（如`Data`、`Value`），用于标识该共用体类型。  
- **成员**：共用体包含的不同类型数据项，但所有成员共享同一块内存，**同一时刻只能有效使用一个成员**（修改一个成员会覆盖其他成员的值）。  


#### 示例：定义一个简单的共用体
```c
// 定义共用体Data，包含int、float、char三种类型的成员
union Data {
    int i;      // 整数
    float f;    // 浮点数
    char c;     // 字符
};
```


### 二、共用体的核心特点
1. **内存共享**：所有成员从同一块内存的起始地址开始存储，共享相同的内存空间。  
   例如，上述`union Data`的`i`、`f`、`c`成员的起始地址完全相同。  

2. **大小由最大成员决定**：共用体的总大小等于其**最大成员的大小**（需能容纳最大的成员）。  
   例如，`int`占4字节，`float`占4字节，`char`占1字节，因此`union Data`的大小为4字节。  

3. **成员互斥**：同一时刻只能使用一个成员，修改一个成员会覆盖其他成员的值（因为内存共享）。  


### 三、共用体变量的声明与使用
共用体变量的声明、初始化和成员访问方式与结构体类似，但初始化和使用有特殊限制。


#### 1. 声明共用体变量
```c
// 方式1：先定义类型，再声明变量
union Data {
    int i;
    float f;
    char c;
};
union Data d1, d2;  // 声明d1、d2为union Data类型变量

// 方式2：定义类型的同时声明变量
union Value {
    short s;
    long l;
} v1, v2;
```


#### 2. 初始化与赋值
- **初始化**：只能在声明时为**一个成员**赋值（因为成员共享内存，无法同时初始化多个）。  
- **赋值**：声明后可给任意成员赋值，但赋值后其他成员的值会被覆盖（变为无效）。  


#### 示例：共用体的初始化与成员访问
```c
#include <stdio.h>

union Data {
    int i;
    float f;
    char c;
};

int main() {
    union Data d;

    // 初始化：只能给一个成员赋值（这里初始化i）
    union Data d_init = {10};  // d_init.i = 10，其他成员值不确定

    // 给i赋值
    d.i = 65;
    printf("d.i = %d\n", d.i);       // 输出：65
    printf("d.c = %c\n", d.c);       // 输出：A（65是'A'的ASCII码，c共享i的第一个字节）
    printf("d.f = %.1f\n", d.f);     // 输出：不确定（i的二进制被解析为float）

    // 给f赋值（覆盖i和c的内存）
    d.f = 3.14f;
    printf("d.f = %.2f\n", d.f);     // 输出：3.14
    printf("d.i = %d\n", d.i);       // 输出：不确定（f的二进制被解析为int）
    printf("d.c = %c\n", d.c);       // 输出：不确定（f的第一个字节被解析为char）

    // 查看成员的地址（全部相同）
    printf("&d.i = %p\n", &d.i);
    printf("&d.f = %p\n", &d.f);
    printf("&d.c = %p\n", &d.c);     // 输出的地址完全一致

    return 0;
}
```

**解析**：  
- 当`d.i = 65`时，`d.c`（占1字节）读取的是`i`的第一个字节（值65），对应字符`'A'`；但`d.f`会将`i`的4字节二进制解析为float，结果无意义。  
- 当`d.f = 3.14f`时，内存被float的二进制覆盖，此时`d.i`和`d.c`的取值都是对float二进制的“错误解析”，无实际意义。  


### 四、共用体与结构体的对比
| 特性         | 结构体（Struct）                     | 共用体（Union）                     |
|--------------|--------------------------------------|--------------------------------------|
| 内存分配     | 每个成员占用独立的内存空间，总大小为各成员大小之和（可能因对齐增加）。 | 所有成员共享同一块内存空间，总大小等于最大成员的大小。 |
| 成员关系     | 成员可同时有效（可同时访问多个成员）。 | 成员互斥（同一时刻只能有效使用一个成员，修改一个会覆盖其他）。 |
| 用途         | 组合不同类型数据，表示一个“完整对象”（如学生信息）。 | 在不同时刻存储不同类型数据，节省内存（如多类型变量复用）。 |


### 五、共用体的典型应用场景
共用体的核心价值是**节省内存**，适合以下场景：  


#### 1. 多类型数据的复用（同一内存区域在不同时刻存不同类型）
例如，一个变量有时需要存储int，有时需要存储float，且两种情况不同时发生，可用共用体节省内存（无需定义两个独立变量）。  


#### 2. 处理硬件或协议中的“变体数据”
在硬件寄存器或网络协议中，同一地址可能根据上下文表示不同类型的数据（如某寄存器低8位是char，高16位是int），共用体可直观映射这种结构。  


#### 3. 与结构体结合表示“标签化联合”（Tagged Union）
通过结构体包含一个“标签”（标识当前使用的共用体成员）和一个共用体，实现“安全访问”（先判断标签，再使用对应成员）。  

**示例：标签化联合（安全使用共用体）**  
```c
#include <stdio.h>

// 标签：标识当前共用体使用的成员
typedef enum { INT, FLOAT, CHAR } Type;

// 结构体：包含标签和共用体
typedef struct {
    Type type;  // 标签：当前有效成员的类型
    union {
        int i;
        float f;
        char c;
    } data;     // 共用体：存储实际数据
} TaggedData;

// 打印TaggedData的值（根据标签安全访问）
void printData(TaggedData td) {
    switch (td.type) {
        case INT:
            printf("整数：%d\n", td.data.i);
            break;
        case FLOAT:
            printf("浮点数：%.2f\n", td.data.f);
            break;
        case CHAR:
            printf("字符：%c\n", td.data.c);
            break;
    }
}

int main() {
    TaggedData td;

    td.type = INT;
    td.data.i = 100;
    printData(td);  // 输出：整数：100

    td.type = FLOAT;
    td.data.f = 3.14;
    printData(td);  // 输出：浮点数：3.14

    td.type = CHAR;
    td.data.c = 'A';
    printData(td);  // 输出：字符：A

    return 0;
}
```

**解析**：通过`type`标签明确当前共用体成员的类型，避免访问无效成员，是共用体的安全用法。  


### 六、注意事项
1. **成员互斥**：共用体的成员不能同时使用，修改一个成员后，其他成员的值会被覆盖（变为无效），访问时需确保当前成员是最近赋值的。  
2. **初始化限制**：只能在声明时初始化一个成员（用`{值}`的形式，默认初始化第一个成员，或指定成员初始化（C99及以上支持`{.i=10}`））。  
3. **内存对齐**：共用体的大小为最大成员的大小，但可能因内存对齐规则（如最大成员是4字节，共用体大小就是4字节）。  
4. **指针访问**：共用体指针访问成员的方式与结构体相同，用`->`运算符（如`union Data *p; p->i = 10;`）。  


### 总结
共用体是C语言中用于“内存共享”的特殊数据类型，所有成员共用同一块内存，大小等于最大成员的大小，适合在不同时刻存储不同类型数据以节省内存。其核心特点是成员互斥（同一时刻只能有效使用一个），常与结构体结合实现“标签化联合”以安全访问。与结构体相比，共用体更注重内存效率，而结构体更注重数据的组合完整性。
