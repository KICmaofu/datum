在C语言中，**递归（Recursion）** 是指函数直接或间接调用自身的编程技巧。它通过将复杂问题分解为**规模更小的同类子问题**，最终通过解决子问题来解决原问题。递归的核心是“自我调用”，但必须有明确的终止条件，否则会导致无限循环（栈溢出）。


### 一、递归的基本结构
递归函数的实现需要满足两个关键条件：  
1. **递归条件**：函数调用自身，处理规模更小的子问题；  
2. **基线条件（终止条件）**：当问题规模小到可直接解决时，返回结果，终止递归（避免无限调用）。  


**递归的通用逻辑**：  
```c
返回类型 递归函数(参数) {
    if (基线条件) {  // 终止条件：直接返回结果
        return 基础值;
    } else {  // 递归条件：调用自身解决子问题
        return 子问题结果与当前逻辑的组合;
    }
}
```  


### 二、经典递归示例
通过具体案例理解递归的工作原理。


#### 1. 阶乘计算（n!）
**阶乘定义**：  
- 0! = 1（基线条件）；  
- n! = n × (n-1)!（递归条件，n > 0）。  


**递归实现**：  
```c
#include <stdio.h>

// 计算n的阶乘
int factorial(int n) {
    // 基线条件：n=0时，直接返回1
    if (n == 0) {
        return 1;
    } else {
        // 递归条件：n! = n × (n-1)!，调用自身计算(n-1)!
        return n * factorial(n - 1);
    }
}

int main() {
    int num = 5;
    printf("%d! = %d\n", num, factorial(num));  // 输出：5! = 120
    return 0;
}
```


**执行过程解析**（以`factorial(5)`为例）：  
```
factorial(5) → 5 × factorial(4)  
factorial(4) → 4 × factorial(3)  
factorial(3) → 3 × factorial(2)  
factorial(2) → 2 × factorial(1)  
factorial(1) → 1 × factorial(0)  
factorial(0) → 1（基线条件，开始返回）  
→ 1 × 1 = 1  
→ 2 × 1 = 2  
→ 3 × 2 = 6  
→ 4 × 6 = 24  
→ 5 × 24 = 120（最终结果）  
```  


#### 2. 斐波那契数列（Fibonacci）
**斐波那契定义**：  
- F(0) = 0，F(1) = 1（基线条件）；  
- F(n) = F(n-1) + F(n-2)（递归条件，n ≥ 2）。  


**递归实现**：  
```c
#include <stdio.h>

// 计算第n个斐波那契数
int fibonacci(int n) {
    // 基线条件：n=0返回0，n=1返回1
    if (n == 0) {
        return 0;
    } else if (n == 1) {
        return 1;
    } else {
        // 递归条件：F(n) = F(n-1) + F(n-2)
        return fibonacci(n - 1) + fibonacci(n - 2);
    }
}

int main() {
    int num = 6;
    printf("F(%d) = %d\n", num, fibonacci(num));  // 输出：F(6) = 8（序列：0,1,1,2,3,5,8...）
    return 0;
}
```


**注意**：该实现存在**重复计算**问题（如`fibonacci(5)`需计算`fibonacci(4)`和`fibonacci(3)`，而`fibonacci(4)`又需`fibonacci(3)`，导致`fibonacci(3)`被计算多次），效率较低。实际开发中可通过“记忆化”（缓存已计算结果）优化，或改用迭代实现。


#### 3. 递归遍历字符串（反向打印）
通过递归逐步缩小字符串长度，直到空字符（`'\0'`）时终止，反向打印字符。  


**实现**：  
```c
#include <stdio.h>

// 反向打印字符串
void reverse_print(const char *str) {
    // 基线条件：若当前字符是结束符，直接返回
    if (*str == '\0') {
        return;
    }
    // 递归条件：先打印后面的字符，再打印当前字符
    reverse_print(str + 1);  // 处理下一个字符（子问题）
    printf("%c", *str);      // 打印当前字符
}

int main() {
    const char *s = "hello";
    reverse_print(s);  // 输出：olleh
    return 0;
}
```


**执行过程**：  
```
reverse_print("hello") → 先调用 reverse_print("ello")  
reverse_print("ello") → 先调用 reverse_print("llo")  
reverse_print("llo") → 先调用 reverse_print("lo")  
reverse_print("lo") → 先调用 reverse_print("o")  
reverse_print("o") → 先调用 reverse_print("")  
reverse_print("") → 基线条件，返回  
→ 打印 'o'  
→ 打印 'l'  
→ 打印 'l'  
→ 打印 'e'  
→ 打印 'h'  
最终输出：olleh  
```  


### 三、递归的优缺点
#### 优点：  
1. **代码简洁直观**：递归天然贴合某些问题的数学定义（如阶乘、斐波那契）或逻辑结构（如树、图的遍历），代码更易理解；  
2. **问题分解清晰**：将大问题拆解为同类小问题，符合“分治思想”。  


#### 缺点：  
1. **栈溢出风险**：每次递归调用会在**函数调用栈**上分配空间（存储参数、局部变量、返回地址等），若递归深度过大（如几万次），会导致栈空间耗尽，触发“栈溢出”错误（`Stack Overflow`）；  
2. **效率较低**：函数调用本身有开销（入栈、出栈操作），且可能存在重复计算（如斐波那契）；  
3. **调试难度高**：递归层次深时，调用栈复杂，难以跟踪中间状态。  


### 四、递归与迭代的对比
递归和迭代（循环）都可解决重复计算问题，选择需根据场景：  

| 场景                | 更适合的方式 | 原因                                                                 |
|---------------------|--------------|----------------------------------------------------------------------|
| 问题天然递归（树、分形） | 递归         | 代码更简洁，符合问题逻辑                                             |
| 深度小（如n<1000）   | 递归         | 可读性优先，无需担心栈溢出                                           |
| 深度大（如n>10000）  | 迭代         | 避免栈溢出，效率更高（循环开销低于函数调用）                         |
| 无重复计算           | 递归/迭代均可 | 若递归代码更简洁，可优先用递归；否则用迭代（如阶乘）                 |
| 有重复计算           | 迭代（或优化递归） | 迭代可通过变量缓存结果，避免重复计算（如斐波那契的迭代实现）         |  


**示例：阶乘的迭代实现**  
```c
int factorial_iter(int n) {
    int result = 1;
    for (int i = 1; i <= n; i++) {
        result *= i;
    }
    return result;
}
```  


### 五、递归的注意事项
1. **必须设置基线条件**：否则会导致无限递归，最终栈溢出；  
2. **控制递归深度**：避免深度过大（如超过1万次），必要时改用迭代；  
3. **减少重复计算**：对存在重复子问题的场景（如斐波那契），可通过“记忆化递归”（用数组缓存已计算结果）优化；  
   ```c
   // 记忆化优化斐波那契
   int memo[100];  // 缓存已计算的结果
   int fib_memo(int n) {
       if (memo[n] != 0) return memo[n];  // 若已计算，直接返回缓存
       if (n == 0) return 0;
       if (n == 1) return 1;
       memo[n] = fib_memo(n-1) + fib_memo(n-2);  // 缓存结果
       return memo[n];
   }
   ```  
4. **避免在递归中使用大量局部变量**：减少栈空间占用，降低溢出风险。  


### 总结
递归是一种强大的编程技巧，核心是“分解问题+自我调用+终止条件”。它适合解决具有递归结构的问题（如树遍历、分治算法），但需注意栈溢出和效率问题。实际开发中，应根据问题规模和特性，在递归与迭代之间选择更合适的方案。
