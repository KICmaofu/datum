C预处理器是C语言编译过程中的一个关键环节，它在编译器对代码进行实际编译之前，对源代码进行文本级别的处理，生成预处理后的代码供编译器使用。其核心作用是实现**文本替换、文件包含、条件编译**等功能，以提高代码的可维护性、可移植性和复用性。


### 一、预处理器的工作方式
预处理器不理解C语法，仅进行**文本处理**。它会扫描源代码，识别以`#`开头的预处理器指令（如`#include`、`#define`等），并按指令规则处理，最终输出“净化”后的代码（去除所有预处理器指令，完成替换/包含等操作），再交给编译器编译。


### 二、常用预处理器指令


#### 1. `#include`：文件包含
用于将其他文件的内容嵌入到当前文件中（通常是头文件`.h`），实现代码复用。  
- 格式1：`#include <文件名>`  
  用于包含系统标准头文件（如`stdio.h`），预处理器会到系统指定的头文件目录（如`/usr/include`）搜索文件。  

- 格式2：`#include "文件名"`  
  用于包含自定义头文件，预处理器先在当前源文件所在目录搜索，若未找到再去系统目录搜索。  

**示例**：  
```c
#include <stdio.h>  // 包含标准输入输出头文件
#include "myfunc.h" // 包含自定义头文件
```


#### 2. `#define`：宏定义
用于定义“宏”（文本替换规则），分为**对象宏**（无参数）和**函数宏**（带参数）。


##### （1）对象宏（无参数）  
格式：`#define 宏名 替换文本`  
作用：预处理器会将代码中所有`宏名`替换为`替换文本`（替换文本可省略，仅表示“宏已定义”）。  

**示例**：  
```c
#define PI 3.1415926  // 定义常量宏
#define MAX_SIZE 100  // 定义数组大小宏

// 用法：
float area = PI * r * r;  // 替换后：3.1415926 * r * r
int arr[MAX_SIZE];        // 替换后：int arr[100];
```


##### （2）函数宏（带参数）  
格式：`#define 宏名(参数列表) 替换文本`  
作用：类似函数调用，预处理器会将`宏名(参数)`替换为`替换文本`（参数会代入替换文本）。  

**注意**：  
- 替换文本和参数需加括号，避免因运算符优先级导致错误；  
- 宏无类型检查，且可能产生“副作用”（如参数被多次求值）。  

**示例**：  
```c
// 正确定义：参数和整体加括号
#define MAX(a, b) ((a) > (b) ? (a) : (b))

// 错误定义（缺少括号）：若a=1+2, b=3，会被替换为1+2>3?1+2:3，结果错误
// #define MAX(a,b) a > b ? a : b

// 副作用示例：参数x++被多次求值
int x=1, y=2;
int m = MAX(x++, y++);  // 替换后：((x++)>(y++)?(x++):(y++))，x和y可能被递增2次
```


#### 3. `#undef`：取消宏定义
用于取消已定义的宏，之后该宏不再生效。  

**示例**：  
```c
#define PI 3.14
#undef PI  // 取消PI的定义，后续代码中PI不再被替换
```


#### 4. 条件编译指令
用于根据条件决定是否编译某部分代码（实现“选择性编译”），常用于跨平台适配、调试代码开关等场景。


##### （1）`#ifdef` / `#ifndef` / `#endif`  
- `#ifdef 宏名`：若`宏名`已定义，则编译后续代码，直到`#endif`；  
- `#ifndef 宏名`：若`宏名`未定义，则编译后续代码，直到`#endif`（常用於頭文件保護）。  

**示例1：调试代码开关**  
```c
#define DEBUG  // 定义DEBUG宏，开启调试输出

#ifdef DEBUG
  #define LOG(msg) printf("Debug: %s\n", msg)  // 调试模式下启用LOG
#else
  #define LOG(msg)  // 非调试模式下，LOG为空（不执行）
#endif

// 用法：
LOG("Start processing...");  // 若DEBUG定义，则输出；否则无操作
```

**示例2：头文件保护（防止重复包含）**  
头文件（如`myhead.h`）中加入：  
```c
#ifndef MYHEAD_H  // 若MYHEAD_H未定义
#define MYHEAD_H  // 定义MYHEAD_H

// 头文件内容（函数声明、结构体定义等）
int add(int a, int b);

#endif  // 结束条件编译
```  
多次`#include "myhead.h"`时，第一次会定义`MYHEAD_H`，后续包含会因`MYHEAD_H`已定义而跳过内容，避免重复定义错误。


##### （2）`#if` / `#elif` / `#else` / `#endif`  
更灵活的条件编译，`#if`后接**常量表达式**（值为非0时条件成立），可配合`#elif`（else if）和`#else`使用。  

**示例：跨平台代码**  
```c
#define OS_WIN 1
#define OS_LINUX 2
#define CURRENT_OS OS_LINUX  // 当前系统为Linux

#if CURRENT_OS == OS_WIN
  #include <windows.h>  // Windows平台头文件
#elif CURRENT_OS == OS_LINUX
  #include <unistd.h>   // Linux平台头文件
#else
  #error "Unsupported OS"  // 未知系统时报错
#endif
```


#### 5. `#error`：预处理错误
在预处理阶段若满足条件，直接输出错误信息并终止预处理（编译失败）。  

**示例**：  
```c
#if VERSION < 1
  #error "VERSION must be at least 1"  // 若VERSION<1，预处理报错
#endif
```


#### 6. `#pragma`：编译器特定指令
用于向编译器传递特定选项（不同编译器支持的`#pragma`可能不同）。  

**常见用法**：  
- `#pragma once`：替代头文件保护（告诉编译器该文件仅包含一次，部分编译器支持）；  
- `#pragma pack(n)`：设置结构体对齐方式（n为对齐字节数）。  

**示例**：  
```c
#pragma once  // 替代#ifndef，防止头文件重复包含

#pragma pack(1)  // 结构体按1字节对齐
struct Data {
  char c;
  int i;  // 原本占4字节，对齐后总大小为1+4=5字节（默认对齐可能更大）
};
#pragma pack()  // 恢复默认对齐
```


### 三、预定义宏
C标准定义了一些预定义宏，可直接使用（通常用于调试或版本信息）：  

| 宏名       | 含义                                  |
|------------|---------------------------------------|
| `__LINE__` | 当前代码行号（整数）                  |
| `__FILE__` | 当前源文件名（字符串）                |
| `__DATE__` | 编译日期（格式："Mmm dd yyyy"）       |
| `__TIME__` | 编译时间（格式："hh:mm:ss"）          |
| `__STDC__` | 若定义为1，表示编译器遵循ANSI C标准   |

**示例**：  
```c
// 输出错误位置
printf("Error in file %s at line %d\n", __FILE__, __LINE__);
```


### 总结
C预处理器通过`#include`实现文件复用、`#define`实现文本替换、条件编译指令实现选择性编译，是C语言灵活性的重要支撑。合理使用预处理器可显著提升代码的可维护性和可移植性，但需注意宏的副作用（如参数多次求值）和跨编译器兼容性（如`#pragma`）。
