在C语言中，**可变参数**（Variable Arguments）指函数可以接受**数量不确定的参数**，典型例子是`printf`函数（如`printf("%d, %s", 10, "hello")`可接受2个、3个甚至更多参数）。C语言通过`<stdarg.h>`头文件提供的宏来实现对可变参数的处理，允许开发者自定义支持可变参数的函数。


### 一、核心概念与宏
处理可变参数依赖`<stdarg.h>`中的4个关键工具：  

| 宏/类型         | 作用                                                                 |
|-----------------|----------------------------------------------------------------------|
| `va_list`       | 一种类型，用于声明一个“可变参数列表指针”（用于访问可变参数）。         |
| `va_start`      | 初始化`va_list`变量，使其指向第一个可变参数。                         |
| `va_arg`        | 从`va_list`中获取下一个参数（需指定类型），并移动指针到下一个参数。   |
| `va_end`        | 结束对可变参数的处理，释放`va_list`相关资源（避免内存泄漏）。         |  


### 二、使用步骤（自定义可变参数函数）
实现一个支持可变参数的函数，需遵循以下步骤：  


#### 1. 函数声明：固定参数 + `...`
函数必须包含**至少一个固定参数**（用于确定可变参数的个数或类型），最后用`...`表示可变参数。格式如下：  
```c
返回类型 函数名(固定参数1, 固定参数2, ...) {
    // 函数体（用<stdarg.h>宏处理可变参数）
}
```  


#### 2. 用`va_list`声明参数指针
在函数内部，声明`va_list`类型的变量（如`ap`，即“argument pointer”），用于遍历可变参数：  
```c
va_list ap;  // 声明可变参数列表指针
```  


#### 3. 用`va_start`初始化指针
通过`va_start`初始化`ap`，使其指向**第一个可变参数**。语法：  
```c
va_start(ap, 最后一个固定参数);  // 以最后一个固定参数为基准，定位第一个可变参数
```  


#### 4. 用`va_arg`获取可变参数
通过`va_arg`依次读取可变参数，需显式指定参数类型（如`int`、`double`）。语法：  
```c
类型 值 = va_arg(ap, 类型);  // 读取当前参数，同时移动ap到下一个参数
```  


#### 5. 用`va_end`结束处理
处理完所有可变参数后，必须调用`va_end`释放资源（否则可能导致未定义行为）：  
```c
va_end(ap);  // 清理ap
```  


### 三、示例：自定义可变参数函数
#### 示例1：计算多个整数的和
实现`sum(int count, ...)`，其中`count`是可变参数的个数，后面跟`count`个`int`类型的参数，返回它们的和。  

```c
#include <stdio.h>
#include <stdarg.h>  // 必须包含该头文件

// 计算count个整数的和
int sum(int count, ...) {
    va_list ap;       // 1. 声明可变参数指针
    int total = 0;

    va_start(ap, count);  // 2. 初始化ap，指向第一个可变参数（以count为基准）

    // 3. 循环获取所有可变参数
    for (int i = 0; i < count; i++) {
        // 每次获取一个int类型的参数，累加
        total += va_arg(ap, int);
    }

    va_end(ap);  // 4. 结束处理
    return total;
}

int main() {
    printf("sum(2, 10, 20) = %d\n", sum(2, 10, 20));       // 30
    printf("sum(3, 5, 6, 7) = %d\n", sum(3, 5, 6, 7));    // 18
    printf("sum(1, 100) = %d\n", sum(1, 100));            // 100
    return 0;
}
```  


#### 示例2：简易日志函数（模仿printf）
实现`log_msg(const char* format, ...)`，支持格式化输出（如`%d`输出整数，`%s`输出字符串）。  

```c
#include <stdio.h>
#include <stdarg.h>

// 简易日志函数：支持%d和%s格式化
void log_msg(const char* format, ...) {
    va_list ap;
    va_start(ap, format);  // 以format为基准，定位可变参数

    // 遍历format字符串，解析格式符
    for (int i = 0; format[i] != '\0'; i++) {
        if (format[i] == '%' && format[i+1] != '\0') {
            i++;  // 跳过'%'，检查下一个字符
            switch (format[i]) {
                case 'd': {  // 处理整数
                    int num = va_arg(ap, int);
                    printf("%d", num);
                    break;
                }
                case 's': {  // 处理字符串
                    char* str = va_arg(ap, char*);
                    printf("%s", str);
                    break;
                }
                default:  // 未知格式符，直接输出
                    printf("%%%c", format[i]);
            }
        } else {
            // 非格式符，直接输出
            printf("%c", format[i]);
        }
    }

    va_end(ap);
    printf("\n");  // 每条日志换行
}

int main() {
    log_msg("用户%s，年龄%d，分数%d", "Alice", 18, 95);  // 输出：用户Alice，年龄18，分数95
    log_msg("数值：%d, 字符串：%s", 100, "test");        // 输出：数值：100, 字符串：test
    return 0;
}
```  


### 四、注意事项
1. **必须有固定参数**：可变参数函数至少需要一个固定参数（如`sum`中的`count`，`printf`中的`format`），用于`va_start`定位第一个可变参数。  
2. **参数类型必须明确**：`va_arg`需要显式指定参数类型（如`int`、`double`），若类型不匹配（如实际是`float`却按`int`读取），会导致**未定义行为**（数据错误或程序崩溃）。  
   - 注意：`float`会被隐式转换为`double`，因此读取浮点参数时需用`double`类型（如`va_arg(ap, double)`）。  
3. **参数个数需已知**：函数必须通过某种方式知道可变参数的个数（如`sum`中的`count`）或结束标志（如`printf`通过`format`中的`%`个数判断），否则可能读取到无效内存。  
4. **`va_end`必须调用**：即使函数提前返回，也需在返回前调用`va_end`，否则可能导致栈泄漏（不同编译器实现可能有差异，但遵循此规则可保证兼容性）。  
5. **无类型检查**：编译器不会检查可变参数的类型和个数是否与函数逻辑匹配，需开发者手动保证正确性（这也是`printf`格式错误可能导致崩溃的原因）。  


### 五、适用场景
可变参数函数适合需要处理**不确定数量参数**的场景，例如：  
- 格式化输出函数（`printf`、`fprintf`）；  
- 求和/求平均函数（支持任意个数字）；  
- 日志函数（支持不同类型的日志信息）；  
- 动态参数的工具函数（如动态创建数组、批量处理数据）。  


### 总结
C语言通过`<stdarg.h>`中的`va_list`、`va_start`、`va_arg`、`va_end`实现可变参数，核心是“声明指针→初始化→读取参数→清理”的流程。使用时需注意参数的类型、个数和`va_end`的调用，避免因类型不匹配或参数越界导致错误。可变参数为函数提供了灵活性，但也牺牲了部分类型安全性，需谨慎使用。
