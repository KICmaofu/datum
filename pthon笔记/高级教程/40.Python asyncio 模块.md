# Python asyncio 模块完全指南：异步 I/O 并发编程实战
asyncio 是 Python 3.4+ 内置的 **异步 I/O 框架**，基于「事件循环」和「协程」机制实现高并发，核心目标是高效处理 **I/O 密集型任务**（如高并发网络请求、数据库操作、文件读写）—— 无需多线程/多进程的资源开销，单线程内通过协程切换实现“并发”，能支持万级以上的并发连接，是 Python 处理高并发场景的核心工具。

与多线程（`threading`）相比，asyncio 避免了线程切换的内核开销和 GIL 限制；与多进程（`multiprocessing`）相比，它内存占用极低，无需进程间通信的复杂机制；与同步编程相比，它能在 I/O 阻塞时切换任务，充分利用 CPU 资源。


## 一、核心定位与优势
### 1. 核心特点
- **内置无依赖**：Python 3.4+ 标准库自带，3.5+ 支持 `async/await` 语法糖，导入即使用；
- **单线程异步**：基于事件循环调度协程，无线程切换开销，资源占用极低；
- **高并发支持**：单线程可处理万级以上并发连接（远超多线程的千级并发），适合高并发 I/O 场景；
- **原生协程支持**：通过 `async def` 定义协程，`await` 暂停/恢复任务，语法简洁直观；
- **丰富的异步工具**：支持任务管理、超时控制、并发调度、同步原语（锁、信号量）、异步 I/O 等；
- **跨平台兼容**：统一封装不同操作系统的异步 I/O 接口（如 epoll、kqueue、IOCP），行为一致；
- **生态完善**：配套大量异步第三方库（`aiohttp` 异步网络、`asyncmy` 异步 MySQL、`aiofiles` 异步文件）。

### 2. 适用场景
- **高并发网络请求**：爬虫（批量爬取网页/接口）、API 服务（高并发接口响应）、WebSocket 服务；
- **异步数据库操作**：批量数据库查询/写入（如 MySQL、Redis 的异步客户端）；
- **异步文件处理**：批量读取/写入大文件（避免同步 I/O 阻塞）；
- **消息队列消费**：高并发消费消息队列（如 Kafka、RabbitMQ 的异步客户端）；
- **延迟任务调度**：定时任务、延迟执行任务（无需额外进程/线程）；
- **微服务通信**：微服务间的高并发异步调用（如 gRPC 异步客户端）。

### 3. 与其他并发方案的对比
| 并发方案                | 核心优势                          | 缺点                                  | 适用场景                  |
|-------------------------|-----------------------------------|---------------------------------------|---------------------------|
| `asyncio` 模块（本文核心） | 单线程高并发、资源开销低、无 GIL 限制 | 需异步库支持，不能直接调用同步阻塞函数 | 高并发 I/O 场景（万级并发）、API 服务、爬虫 |
| `threading` 模块        | 轻量、内存共享、支持同步库        | GIL 限制、线程切换开销、并发量低（千级） | 普通 I/O 密集型任务（无异步库） |
| `multiprocessing` 模块  | 无 GIL 限制、支持 CPU 并行计算    | 资源开销大、进程间通信复杂            | CPU 密集型任务、大规模并行计算 |
| `concurrent.futures`    | 线程池/进程池封装，简化管理        | 并发量有限，仍受 GIL/进程开销限制     | 简单并发场景（无需高并发） |

**结论**：高并发 I/O 场景（如万级爬虫、高并发 API）优先用 `asyncio`；普通 I/O 场景用 `threading`；CPU 密集型用 `multiprocessing`；混合场景可结合 `asyncio` + `multiprocessing`（异步处理 I/O，多进程处理 CPU 密集任务）。

### 版本说明
- **关键版本节点**：
  - Python 3.4：首次引入 asyncio，需用 `@asyncio.coroutine` 装饰器定义协程；
  - Python 3.5：新增 `async/await` 语法糖（推荐），简化协程定义；
  - Python 3.7：新增 `asyncio.run()` 函数，简化事件循环启动（无需手动管理循环）；
  - Python 3.8：新增 `asyncio.to_thread()`，支持在异步中调用同步函数；

- **推荐版本**：Python 3.7+（语法简洁，功能稳定），优先 3.11+（支持 `TaskGroup`，并发控制更优雅）。

## 三、核心概念：理解异步编程模型
asyncio 的核心是「事件循环 + 协程 + 任务」，需先理清以下关键概念，否则容易陷入思维误区：

### 1. 核心概念解析
| 概念                | 中文名称          | 核心作用                                  | 通俗理解                          |
|---------------------|-------------------|-------------------------------------------|-----------------------------------|
| **协程（Coroutine）** | 可暂停的函数      | 异步任务的载体，通过 `async def` 定义，`await` 暂停执行 | “可暂停的任务”：执行到 `await` 时暂停，等待 I/O 完成后恢复 |
| **事件循环（Event Loop）** | 任务调度器      | 管理所有协程/任务，调度就绪任务执行，监听 I/O 事件 | “总指挥”：负责分配 CPU 时间，I/O 阻塞时切换任务 |
| **任务（Task）**     | 可调度的协程      | 对协程的封装，支持取消、超时、状态跟踪（运行/完成/失败） | “协程的管理者”：将协程包装成可被事件循环调度的单元 |
| **Future**          | 异步结果容器      | 存储异步任务的结果（成功/失败），协程通过 `await` 获取结果 | “结果快递盒”：任务完成后，结果会放入 Future 中 |
| **await**           | 暂停/恢复关键字   | 暂停当前协程，等待异步操作完成后恢复执行 | “等待快递”：当前任务暂停，先处理其他任务，直到等待的结果返回 |
| **异步 I/O**        | 非阻塞 I/O 操作   | I/O 操作（网络请求、文件读写）时不阻塞线程，事件循环可调度其他任务 | “非阻塞办事”：去快递站寄件，不用排队等，先去办其他事，快递员通知后再回来取 |

### 2. 异步编程核心流程
1. **定义协程**：用 `async def` 定义包含异步逻辑的协程（如异步网络请求、延迟）；
2. **创建任务**：将协程包装成 `Task`（通过 `asyncio.create_task()` 或 `asyncio.run()`）；
3. **启动事件循环**：事件循环开始调度任务，执行第一个就绪的协程；
4. **任务暂停与切换**：协程执行到 `await` 时（如等待 I/O 完成），暂停自身，事件循环切换到其他就绪任务；
5. **任务恢复与执行**：当 `await` 的异步操作完成（如 I/O 响应到达），事件循环唤醒暂停的协程，继续执行；
6. **任务完成**：所有任务执行完毕，事件循环退出。

### 3. 同步 vs 异步：代码执行流程对比
#### （1）同步代码（阻塞）
```python
import time

def sync_task(name, delay):
    print(f"同步任务 {name} 启动")
    time.sleep(delay)  # 阻塞线程，无法做其他事
    print(f"同步任务 {name} 完成")

# 执行两个同步任务（总耗时 = 2 + 1 = 3 秒）
start = time.time()
sync_task("A", 2)
sync_task("B", 1)
print(f"总耗时：{time.time() - start:.1f} 秒")
```

#### （2）异步代码（非阻塞）
```python
import asyncio
import time

async def async_task(name, delay):
    print(f"异步任务 {name} 启动")
    await asyncio.sleep(delay)  # 非阻塞，事件循环可切换其他任务
    print(f"异步任务 {name} 完成")

# 执行两个异步任务（总耗时 = max(2, 1) = 2 秒）
start = time.time()
asyncio.run(asyncio.gather(async_task("A", 2), async_task("B", 1)))
print(f"总耗时：{time.time() - start:.1f} 秒")
```

**核心差异**：同步代码阻塞时线程无法工作，异步代码阻塞时事件循环切换任务，充分利用时间。


## 四、核心功能：分场景详解（语法+示例）
### 1. 协程：异步任务的基础
协程是 asyncio 的核心，通过 `async def` 定义，必须在事件循环中运行（不能直接调用）。

#### （1）定义与运行协程
```python
import asyncio

# 定义协程（async def 关键字）
async def my_coroutine(x, y):
    print(f"协程开始：x={x}, y={y}")
    await asyncio.sleep(1)  # 异步延迟（必须 await 异步操作）
    return x + y  # 协程返回值

# 方式 1：用 asyncio.run() 运行（Python 3.7+，推荐）
# asyncio.run() 会自动创建事件循环，运行协程，最后关闭循环
result = asyncio.run(my_coroutine(3, 5))
print(f"协程返回结果：{result}")  # 输出：协程返回结果：8

# 方式 2：手动管理事件循环（Python 3.6 及以下）
# loop = asyncio.get_event_loop()
# result = loop.run_until_complete(my_coroutine(3, 5))
# loop.close()
```

#### （2）错误处理：捕获协程异常
```python
import asyncio

async def error_coroutine():
    print("协程开始，准备抛出异常")
    await asyncio.sleep(0.5)
    raise ValueError("协程执行出错！")

async def main():
    try:
        # 等待协程，捕获异常
        await error_coroutine()
    except ValueError as e:
        print(f"捕获到协程异常：{e}")

asyncio.run(main())

# 输出结果：
# 协程开始，准备抛出异常
# 捕获到协程异常：协程执行出错！
```

### 2. 任务：协程的调度与管理
任务（`Task`）是对协程的封装，支持并发执行、取消、超时、状态跟踪，是 asyncio 并发调度的核心。

#### （1）创建与运行多个任务
```python
import asyncio

async def task_func(name, delay):
    print(f"任务 {name} 启动，延迟 {delay} 秒")
    await asyncio.sleep(delay)
    return f"任务 {name} 完成"

async def main():
    # 1. 创建任务（asyncio.create_task()，Python 3.7+）
    # 任务创建后立即加入事件循环，开始执行
    task1 = asyncio.create_task(task_func("A", 2))
    task2 = asyncio.create_task(task_func("B", 1))

    # 2. 等待任务完成（两种方式）
    # 方式 a：逐个等待（获取单个结果）
    result1 = await task1
    result2 = await task2
    print(f"结果：{result1}, {result2}")

    # 方式 b：批量等待（asyncio.gather()，获取所有结果）
    # results = await asyncio.gather(task1, task2)
    # print(f"批量结果：{results}")  # 输出：批量结果：['任务 A 完成', '任务 B 完成']

asyncio.run(main())
```

#### （2）任务取消与超时控制
```python
import asyncio

async def long_running_task():
    print("长时间任务启动（预计运行 5 秒）")
    try:
        await asyncio.sleep(5)  # 模拟长时间运行
    except asyncio.CancelledError:
        # 捕获任务取消异常，释放资源
        print("长时间任务被取消，释放资源...")
        raise  # 可选：重新抛出异常，让任务标记为取消状态
    return "任务完成"

async def main():
    # 1. 创建并启动任务
    task = asyncio.create_task(long_running_task())

    # 2. 超时控制（asyncio.wait_for()，超过 2 秒取消任务）
    try:
        result = await asyncio.wait_for(task, timeout=2)
        print(f"任务结果：{result}")
    except asyncio.TimeoutError:
        print("任务超时，自动取消")
        # 手动取消任务（如果 wait_for 未自动取消）
        if not task.done():
            task.cancel()
            await task  # 等待任务取消完成（必须 await，否则会警告）

asyncio.run(main())

# 输出结果：
# 长时间任务启动（预计运行 5 秒）
# 任务超时，自动取消
# 长时间任务被取消，释放资源...
```

#### （3）任务状态查询
```python
import asyncio

async def simple_task():
    await asyncio.sleep(1)
    return "完成"

async def main():
    task = asyncio.create_task(simple_task())

    # 查询任务状态
    print(f"任务是否完成：{task.done()}")  # 输出：False
    print(f"任务状态：{task.state}")  # 输出：RUNNING

    # 等待任务完成
    await task

    print(f"任务是否完成：{task.done()}")  # 输出：True
    print(f"任务状态：{task.state}")  # 输出：FINISHED
    print(f"任务结果：{task.result()}")  # 输出：完成

asyncio.run(main())
```

### 3. 并发控制：批量任务调度
asyncio 提供多种批量任务调度方式，满足不同并发需求：

#### （1）`asyncio.gather()`：等待所有任务完成（推荐）
批量运行多个任务，等待所有任务完成后返回结果列表，支持异常捕获：
```python
import asyncio

async def task_a():
    await asyncio.sleep(2)
    return "A 完成"

async def task_b():
    await asyncio.sleep(1)
    return "B 完成"

async def task_c():
    await asyncio.sleep(3)
    raise ValueError("C 执行失败")

async def main():
    # 方式 1：正常运行（所有任务完成）
    results = await asyncio.gather(task_a(), task_b())
    print(f"正常结果：{results}")  # 输出：正常结果：['A 完成', 'B 完成']

    # 方式 2：捕获单个任务异常（其他任务继续执行）
    results = await asyncio.gather(
        task_a(),
        task_b(),
        task_c(),
        return_exceptions=True  # 异常会作为结果返回，不中断其他任务
    )
    print(f"含异常结果：{results}")
    # 输出：含异常结果：['A 完成', 'B 完成', ValueError('C 执行失败')]

asyncio.run(main())
```

#### （2）`asyncio.wait()`：灵活控制任务完成时机
等待多个任务，可设置等待策略（如“第一个完成”“所有完成”），返回完成和未完成的任务集：
```python
import asyncio

async def task(name, delay):
    await asyncio.sleep(delay)
    return f"任务 {name} 完成"

async def main():
    tasks = [
        asyncio.create_task(task("A", 2)),
        asyncio.create_task(task("B", 1)),
        asyncio.create_task(task("C", 3))
    ]

    # 等待第一个任务完成（return_when=asyncio.FIRST_COMPLETED）
    done, pending = await asyncio.wait(tasks, return_when=asyncio.FIRST_COMPLETED)
    print(f"已完成任务数：{len(done)}")  # 输出：1（任务 B 先完成）
    for t in done:
        print(f"完成任务结果：{t.result()}")

    # 取消未完成的任务
    for t in pending:
        t.cancel()
        await t

asyncio.run(main())
```

#### （3）`asyncio.TaskGroup()`：Python 3.11+ 新特性（推荐）
简化多任务管理，自动等待所有任务完成，支持异常聚合，语法更简洁：
```python
import asyncio

async def task(name, delay):
    print(f"任务 {name} 启动，延迟 {delay} 秒")
    await asyncio.sleep(delay)
    return f"任务 {name} 完成"

async def main():
    # 创建 TaskGroup（上下文管理器自动管理任务生命周期）
    async with asyncio.TaskGroup() as tg:
        # 添加任务（无需手动 create_task）
        task1 = tg.create_task(task("A", 2))
        task2 = tg.create_task(task("B", 1))
        task3 = tg.create_task(task("C", 3))

    # TaskGroup 退出时，所有任务已完成
    print(f"任务 1 结果：{task1.result()}")
    print(f"任务 2 结果：{task2.result()}")
    print(f"任务 3 结果：{task3.result()}")

asyncio.run(main())
```

### 4. 异步 I/O：核心应用场景
asyncio 最擅长处理异步 I/O 操作，需配合异步库（如 `aiohttp` 网络、`aiofiles` 文件、`asyncmy` 数据库）。

#### （1）异步网络请求（高并发爬虫示例）
需安装 `aiohttp`（异步 HTTP 库）：`pip install aiohttp`
```python
import asyncio
import aiohttp

# 待爬取的 URL 列表（万级 URL 也可高效处理）
URLS = [
    "https://www.baidu.com",
    "https://www.zhihu.com",
    "https://www.github.com",
    "https://www.python.org",
    "https://www.csdn.net"
]

async def fetch_url(session, url):
    """异步爬取单个 URL"""
    try:
        async with session.get(url, timeout=10) as response:
            # 读取响应内容（异步读取）
            html = await response.text()
            return {
                "url": url,
                "status": response.status,
                "length": len(html),
                "error": None
            }
    except Exception as e:
        return {
            "url": url,
            "status": -1,
            "length": 0,
            "error": str(e)
        }

async def main():
    # 创建异步 HTTP 会话（复用连接，提高效率）
    async with aiohttp.ClientSession() as session:
        # 批量创建爬取任务
        tasks = [fetch_url(session, url) for url in URLS]
        # 并发执行所有任务
        results = await asyncio.gather(*tasks)

    # 输出爬取结果
    for res in results:
        if res["error"]:
            print(f"爬取失败：{res['url']} → {res['error']}")
        else:
            print(f"爬取成功：{res['url']} → 状态码：{res['status']}，长度：{res['length']}")

asyncio.run(main())
```

#### （2）异步文件操作
需安装 `aiofiles`（异步文件库）：`pip install aiofiles`
```python
import asyncio
import aiofiles
import os

async def read_file(file_path):
    """异步读取文件"""
    try:
        async with aiofiles.open(file_path, "r", encoding="utf-8") as f:
            content = await f.read()  # 异步读取
            return {
                "file": os.path.basename(file_path),
                "lines": len(content.splitlines()),
                "error": None
            }
    except Exception as e:
        return {
            "file": os.path.basename(file_path),
            "lines": 0,
            "error": str(e)
        }

async def main():
    # 待读取的文件列表
    file_dir = "./text_files"
    file_paths = [
        os.path.join(file_dir, f) for f in os.listdir(file_dir)
        if f.endswith(".txt")
    ]

    # 并发读取所有文件
    tasks = [read_file(path) for path in file_paths]
    results = await asyncio.gather(*tasks)

    # 输出结果
    for res in results:
        if res["error"]:
            print(f"读取失败：{res['file']} → {res['error']}")
        else:
            print(f"读取成功：{res['file']} → 行数：{res['lines']}")

asyncio.run(main())
```

#### （3）异步数据库操作（MySQL）
需安装 `asyncmy`（异步 MySQL 库）：`pip install asyncmy`
```python
import asyncio
import asyncmy
```

### 5. 同步原语：异步中的线程安全
多协程并发访问共享资源时，需用 asyncio 提供的异步同步原语（而非 `threading` 的同步工具），避免数据错乱。

#### （1）异步锁（`asyncio.Lock`）
保证同一时间仅一个协程访问共享资源：
```python
import asyncio

# 共享资源
count = 0
# 异步锁（必须用 asyncio.Lock，不能用 threading.Lock）
lock = asyncio.Lock()

async def increment(name):
    global count
    for _ in range(100000):
        # 异步锁：with 语句自动获取/释放
        async with lock:
            count += 1  # 临界区：同一时间仅一个协程执行
    print(f"协程 {name} 完成，count = {count}")

async def main():
    # 并发执行 2 个协程修改共享资源
    await asyncio.gather(increment("A"), increment("B"))
    print(f"最终 count 值：{count}")  # 正确结果：200000

asyncio.run(main())
```

#### （2）异步信号量（`asyncio.Semaphore`）
限制同时访问共享资源的协程数量（如限制并发网络请求数）：
```python
import asyncio
import aiohttp

# 信号量：最多允许 5 个协程同时访问
semaphore = asyncio.Semaphore(5)
URLS = [f"https://httpbin.org/get?i={i}" for i in range(20)]  # 20 个 URL

async def fetch_url(url):
    # 异步信号量：with 语句自动获取/释放
    async with semaphore:
        print(f"正在爬取：{url}（当前并发数：{semaphore._value}）")
        async with aiohttp.ClientSession() as session:
            async with session.get(url, timeout=10) as response:
                await response.text()
                print(f"爬取完成：{url}")

async def main():
    tasks = [fetch_url(url) for url in URLS]
    await asyncio.gather(*tasks)

asyncio.run(main())
```

#### （3）异步事件（`asyncio.Event`）
协程间通信的简单机制，一个协程触发事件，其他协程等待事件：
```python
import asyncio

event = asyncio.Event()

async def waiter(name):
    print(f"协程 {name}：等待事件触发...")
    await event.wait()  # 阻塞，直到事件被 set()
    print(f"协程 {name}：事件触发，继续执行")

async def setter():
    print("协程 setter：准备触发事件（延迟 2 秒）")
    await asyncio.sleep(2)
    event.set()  # 触发事件，唤醒所有等待的协程
    print("协程 setter：事件已触发")

async def main():
    # 启动 3 个等待协程和 1 个触发协程
    await asyncio.gather(
        waiter("A"),
        waiter("B"),
        waiter("C"),
        setter()
    )

asyncio.run(main())
```

### 6. 其他常用功能
#### （1）异步定时器（`asyncio.sleep()` + 循环）
实现定时任务（如每隔 5 秒执行一次）：
```python
import asyncio
import time

async def periodic_task(interval):
    """定时任务：每隔 interval 秒执行一次"""
    while True:
        current_time = time.strftime("%Y-%m-%d %H:%M:%S", time.localtime())
        print(f"[{current_time}] 定时任务执行中...")
        await asyncio.sleep(interval)  # 异步延迟，不阻塞事件循环

async def main():
    # 启动定时任务（后台运行）
    task = asyncio.create_task(periodic_task(5))  # 每隔 5 秒执行
    await asyncio.sleep(20)  # 主程序运行 20 秒
    task.cancel()  # 取消定时任务
    await task  # 等待任务取消完成

asyncio.run(main())
```

#### （2）在异步中调用同步函数
同步函数会阻塞事件循环，需用 `asyncio.to_thread()`（Python 3.9+）或 `loop.run_in_executor()` 运行在线程池：
```python
import asyncio
import time

# 同步阻塞函数（如无异步版本的库）
def sync_blocking_func(name):
    print(f"同步函数 {name} 启动（阻塞 2 秒）")
    time.sleep(2)  # 同步阻塞
    return f"同步函数 {name} 完成"

async def main():
    # 方式 1：asyncio.to_thread()（Python 3.9+，推荐）
    # 自动将同步函数放入线程池执行，不阻塞事件循环
    result1 = await asyncio.to_thread(sync_blocking_func, "A")
    print(f"方式 1 结果：{result1}")

    # 方式 2：loop.run_in_executor()（Python 3.5+）
    loop = asyncio.get_running_loop()
    result2 = await loop.run_in_executor(None, sync_blocking_func, "B")
    print(f"方式 2 结果：{result2}")

    # 并发执行多个同步函数
    tasks = [
        asyncio.to_thread(sync_blocking_func, f"Task-{i}")
        for i in range(3)
    ]
    results = await asyncio.gather(*tasks)
    print(f"并发执行结果：{results}")

asyncio.run(main())
```

#### （3）异步上下文管理器
用 `async with` 管理异步资源（如数据库连接、网络会话），自动释放资源：
```python
import asyncio

# 自定义异步上下文管理器
class AsyncResource:
    async def __aenter__(self):
        print("获取异步资源...")
        await asyncio.sleep(0.5)  # 模拟异步初始化
        return self

    async def __aexit__(self, exc_type, exc_val, exc_tb):
        print("释放异步资源...")
        await asyncio.sleep(0.5)  # 模拟异步释放
        # 处理异常（可选）
        if exc_type:
            print(f"捕获异常：{exc_val}")
        return True  # 表示异常已处理，不向上传播

async def use_resource():
    async with AsyncResource() as resource:
        print("使用异步资源中...")
        # 模拟资源使用中的异常
        raise ValueError("资源使用出错")

asyncio.run(use_resource())
```


## 五、实战场景：asyncio 典型应用
### 场景 1：高并发异步爬虫（万级 URL 爬取）
```python
import asyncio
import aiohttp
from urllib.parse import urlparse

# 配置
MAX_CONCURRENT = 100  # 最大并发数（根据目标网站抗压能力调整）
URL_FILE = "urls.txt"  # 存储 URL 的文件（每行一个 URL）
RESULT_FILE = "crawl_results.csv"  # 爬取结果输出文件

# 异步信号量：限制并发数
semaphore = asyncio.Semaphore(MAX_CONCURRENT)

async def fetch_url(session, url):
    """异步爬取单个 URL，返回状态码和响应长度"""
    async with semaphore:
        try:
            async with session.get(url, timeout=15, allow_redirects=True) as response:
                content = await response.text()
                return {
                    "url": url,
                    "status": response.status,
                    "length": len(content),
                    "error": None
                }
        except Exception as e:
            return {
                "url": url,
                "status": -1,
                "length": 0,
                "error": str(e)[:100]  # 截取错误信息
            }

async def main():
    # 读取待爬取的 URL 列表
    with open(URL_FILE, "r", encoding="utf-8") as f:
        urls = [line.strip() for line in f if line.strip()]
    print(f"待爬取 URL 总数：{len(urls)}")

    # 异步爬取所有 URL
    async with aiohttp.ClientSession() as session:
        tasks = [fetch_url(session, url) for url in urls]

        batch_size = 1000
        for i in range(0, len(tasks), batch_size):
            batch_tasks = tasks[i:i+batch_size]
            results = await asyncio.gather(*batch_tasks)
            
            # 将结果写入 CSV 文件（异步写入）
            async with aiofiles.open(RESULT_FILE, "a", encoding="utf-8") as f:
                for res in results:
                    line = f"{res['url']},{res['status']},{res['length']},{res['error']}\n"
                    await f.write(line)
            print(f"已完成 {min(i+batch_size, len(urls))}/{len(urls)} 个 URL 爬取")

if __name__ == "__main__":
    # 初始化结果文件（写入表头）
    with open(RESULT_FILE, "w", encoding="utf-8") as f:
        f.write("url,status,length,error\n")
    # 启动爬取
    asyncio.run(main())

```

### 场景 2：异步 API 服务（基于 aiohttp.web）
需安装 `aiohttp`：`pip install aiohttp`
```python
from aiohttp import web
import asyncio
import asyncmy

# 数据库连接池（复用连接，提高效率）
DB_POOL = None

async def init_db(app):
    """应用启动时初始化数据库连接池"""
    global DB_POOL
    DB_POOL = await asyncmy.create_pool(
        host="localhost",
        user="root",
        password="123456",
        database="test_db",
        charset="utf8mb4",
        maxsize=20  # 连接池最大连接数
    )
    print("数据库连接池初始化完成")

async def close_db(app):
    """应用关闭时关闭数据库连接池"""
    if DB_POOL:
        DB_POOL.close()
        await DB_POOL.wait_closed()
    print("数据库连接池已关闭")

async def get_user_handler(request):
    """异步 API 接口：根据 ID 查询用户"""
    user_id = request.match_info.get("id")
    if not user_id.isdigit():
        return web.json_response({"error": "无效的用户 ID"}, status=400)

    # 从连接池获取连接，执行查询
    async with DB_POOL.acquire() as conn:
        async with conn.cursor(asyncmy.cursors.DictCursor) as cursor:
            await cursor.execute("SELECT id, name, age, city FROM users WHERE id = %s", (user_id,))
            user = await cursor.fetchone()

    if user:
        return web.json_response({"code": 200, "data": user})
    else:
        return web.json_response({"code": 404, "error": "用户不存在"})

async def batch_get_users_handler(request):
    """异步 API 接口：批量查询用户（高并发支持）"""
    user_ids = request.query.get("ids", "").split(",")
    user_ids = [id for id in user_ids if id.isdigit()]
    if not user_ids:
        return web.json_response({"error": "无效的用户 ID 列表"}, status=400)

    # 批量查询（异步并发执行多个 SQL）
    async def query_user(user_id):
        async with DB_POOL.acquire() as conn:
            async with conn.cursor(asyncmy.cursors.DictCursor) as cursor:
                await cursor.execute("SELECT id, name, age, city FROM users WHERE id = %s", (user_id,))
                return await cursor.fetchone()

    tasks = [query_user(id) for id in user_ids]
    users = await asyncio.gather(*tasks)
    users = [user for user in users if user]  # 过滤不存在的用户

    return web.json_response({"code": 200, "data": users, "count": len(users)})

# 创建应用并配置路由
app = web.Application()
# 注册路由
app.add_routes([
    web.get("/user/{id}", get_user_handler),
    web.get("/users", batch_get_users_handler)
])
# 注册应用生命周期回调（初始化/关闭数据库）
app.on_startup.append(init_db)
app.on_cleanup.append(close_db)

# 启动服务
if __name__ == "__main__":
    web.run_app(app, host="0.0.0.0", port=8080)
    print("异步 API 服务启动成功：http://0.0.0.0:8080")
```

### 场景 3：异步消息队列消费（基于 aioamqp）
需安装 `aioamqp`（异步 RabbitMQ 客户端）：`pip install aioamqp`
```python
import asyncio
import aioamqp

# RabbitMQ 配置
RABBITMQ_CONFIG = {
    "host": "localhost",
    "port": 5672,
    "login": "guest",
    "password": "guest",
    "virtualhost": "/"
}

async def process_message(message):
    """处理消息（模拟业务逻辑）"""
    print(f"收到消息：{message}")
    await asyncio.sleep(1)  # 模拟处理耗时
    print(f"消息处理完成：{message}")

async def consumer():
    """异步消费 RabbitMQ 消息"""
    try:
        # 连接 RabbitMQ
        transport, protocol = await aioamqp.connect(**RABBITMQ_CONFIG)
        channel = await protocol.channel()
        # 声明队列（不存在则创建）
        await channel.queue_declare(queue_name="task_queue", durable=True)
        # 公平分发（确保每个消费者处理完消息再分配新消息）
        await channel.basic_qos(prefetch_count=1)

        print("等待消息...（按 Ctrl+C 退出）")

        async def callback(channel, body, envelope, properties):
            """消息回调函数"""
            message = body.decode("utf-8")
            await process_message(message)
            # 确认消息已处理（RabbitMQ 移除消息）
            await channel.basic_ack(delivery_tag=envelope.delivery_tag)

        # 注册回调函数，开始消费
        await channel.basic_consume(callback, queue_name="task_queue")

        # 保持连接（直到手动退出）
        while True:
            await asyncio.sleep(1)
    except Exception as e:
        print(f"消费消息失败：{e}")
    finally:
        # 关闭连接
        if 'protocol' in locals() and not protocol.is_closed:
            await protocol.close()
        if 'transport' in locals() and transport.is_closing():
            transport.close()

async def main():
    # 启动多个消费者协程（提高并发消费能力）
    consumer_count = 5
    tasks = [consumer() for _ in range(consumer_count)]
    await asyncio.gather(*tasks)

if __name__ == "__main__":
    try:
        asyncio.run(main())
    except KeyboardInterrupt:
        print("消费者退出")
```


## 六、常见问题与避坑指南
### 1. 坑 1：直接调用协程而非 await（最常见错误）
- **问题**：直接调用 `coro()` 而非 `await coro()`，导致协程不执行，返回 `<coroutine object>`；
- **原因**：协程是“可暂停的函数”，必须通过 `await` 或 `asyncio.run()` 触发执行；
- **解决方案**：始终用 `await` 调用协程（在另一个协程中），或用 `asyncio.run()` 运行顶层协程：
  ```python
  # 错误：直接调用协程，不执行
  async def coro():
      print("协程执行")

  coro()  # 无输出，返回 <coroutine object coro at 0x...>

  # 正确：await 调用（在协程中）
  async def main():
      await coro()  # 输出：协程执行

  asyncio.run(main())
  ```

### 2. 坑 2：在异步中调用同步阻塞函数（阻塞事件循环）
- **问题**：在协程中直接调用 `time.sleep()`、`requests.get()` 等同步阻塞函数，导致整个事件循环卡住，所有协程无法执行；
- **原因**：同步阻塞函数会占用线程，事件循环无法切换到其他协程；
- **解决方案**：
  1. 优先使用异步替代库（`asyncio.sleep()` 替代 `time.sleep()`，`aiohttp` 替代 `requests`）；
  2. 无异步库时，用 `asyncio.to_thread()` 或 `loop.run_in_executor()` 将同步函数放入线程池执行；
  ```python
  # 错误：同步阻塞函数卡住事件循环
  async def bad_coro():
      time.sleep(5)  # 阻塞 5 秒，事件循环无法切换任务
      print("协程完成")

  # 正确：用 asyncio.to_thread() 运行同步函数
  async def good_coro():
      await asyncio.to_thread(time.sleep, 5)  # 非阻塞，事件循环可切换任务
      print("协程完成")
  ```

### 3. 坑 3：混用同步锁与异步代码（死锁）
- **问题**：在协程中使用 `threading.Lock` 等同步锁，导致死锁（同步锁会阻塞线程，事件循环无法切换）；
- **原因**：同步锁是线程级别的，会阻塞整个线程，而异步代码依赖事件循环切换协程；
- **解决方案**：始终使用 asyncio 提供的异步同步原语（`asyncio.Lock`、`asyncio.Semaphore`）；
  ```python
  # 错误：同步锁导致死锁
  from threading import Lock
  sync_lock = Lock()

  async def coro():
      with sync_lock:  # 阻塞线程，事件循环卡住
          await asyncio.sleep(1)

  # 正确：异步锁
  async_lock = asyncio.Lock()

  async def coro():
      async with async_lock:  # 非阻塞，事件循环可切换
          await asyncio.sleep(1)
  ```

### 4. 坑 4：任务取消后未释放资源
- **问题**：任务被取消（如超时、手动取消）时，协程中打开的资源（文件、数据库连接）未释放；
- **原因**：任务取消会抛出 `CancelledError`，中断协程执行，导致资源释放代码无法运行；
- **解决方案**：用 `try...finally` 或 `async with` 确保资源释放，捕获 `CancelledError` 处理收尾工作；
  ```python
  async def task_with_resource():
      conn = None
      try:
          conn = await asyncmy.connect(**DB_CONFIG)
          await asyncio.sleep(10)
      except asyncio.CancelledError:
          print("任务被取消，处理收尾工作...")
          raise  # 重新抛出，让任务标记为取消
      finally:
          if conn:
              await conn.close()  # 确保连接关闭
  ```

### 5. 坑 5：并发数失控（导致目标服务拒绝连接）
- **问题**：无限制创建异步任务（如万级网络请求），导致目标服务触发限流或拒绝连接；
- **原因**：asyncio 单线程支持高并发，但目标服务（如网站、数据库）的抗压能力有限；
- **解决方案**：用 `asyncio.Semaphore` 限制并发数，根据目标服务的抗压能力调整（如限制 100 并发）；
  ```python
  # 限制最大并发数为 50
  semaphore = asyncio.Semaphore(50)

  async def fetch_url(url):
      async with semaphore:  # 自动限制并发数
          async with aiohttp.ClientSession() as session:
              await session.get(url)
  ```

### 6. 坑 6：事件循环关闭后再次使用
- **问题**：`asyncio.run()` 会自动关闭事件循环，后续再调用 `asyncio.get_event_loop()` 会报错；
- **原因**：`asyncio.run()` 为每个调用创建独立的事件循环，执行完毕后关闭，不允许复用；
- **解决方案**：
  1. 顶层代码仅调用一次 `asyncio.run()`，所有异步逻辑放在 `main()` 协程中；
  2. 如需多次运行，在 `main()` 中重新创建任务，而非再次调用 `asyncio.run()`；
  ```python
  # 错误：多次调用 asyncio.run()
  async def coro():
      print("协程执行")

  asyncio.run(coro())
  asyncio.run(coro())  # 虽然不会报错，但不推荐（重复创建/关闭循环）

  # 正确：一次 asyncio.run()，多次运行任务
  async def main():
      await coro()
      await coro()

  asyncio.run(main())
  ```

### 7. 坑 7：忽略 `aiohttp.ClientSession` 的复用
- **问题**：每个网络请求创建一个 `aiohttp.ClientSession`，导致连接复用失效，性能下降；
- **原因**：`ClientSession` 会维护连接池，复用 TCP 连接，频繁创建会导致三次握手/四次挥手的额外开销；
- **解决方案**：全局复用一个 `ClientSession`，通过 `async with` 管理生命周期；
  ```python
  # 错误：每次请求创建 ClientSession
  async def fetch(url):
      async with aiohttp.ClientSession() as session:  # 频繁创建/关闭，性能差
          await session.get(url)

  # 正确：复用 ClientSession
  async def main():
      async with aiohttp.ClientSession() as session:  # 全局一个，复用连接
          tasks = [fetch(session, url) for url in URLS]
          await asyncio.gather(*tasks)

  async def fetch(session, url):
      await session.get(url)
  ```

### 8. 坑 8：Python 版本兼容问题
- **问题**：使用 `asyncio.create_task()`（Python 3.7+）或 `asyncio.TaskGroup()`（Python 3.11+）在低版本 Python 中报错；
- **解决方案**：
  1. 明确目标 Python 版本，优先使用 3.7+；
  2. 低版本兼容方案：用 `asyncio.ensure_future()` 替代 `create_task()`；
  ```python
  # Python 3.6 及以下兼容方案
  task = asyncio.ensure_future(coro())  # 替代 asyncio.create_task(coro())
  ```

## 八、总结
asyncio 是 Python 处理高并发 I/O 场景的“王牌工具”，核心价值在于 **单线程高并发、低资源开销、原生异步语法**，完美解决万级以上的并发 I/O 需求（如高并发爬虫、API 服务、消息队列消费）。

### 学习路径建议
1. 基础入门：理解协程、事件循环、任务的核心概念，掌握 `async/await` 语法，能运行简单异步任务；
2. 核心技能：熟练使用 `asyncio.gather()`/`create_task()` 进行并发调度，掌握异步 I/O 操作（网络、文件、数据库）；
3. 进阶应用：学会使用异步同步原语（Lock、Semaphore）解决线程安全问题，处理任务超时、取消、异常；
4. 实战优化：掌握连接池复用（HTTP、数据库）、并发数控制、资源释放等优化技巧，避开常见陷阱；
5. 生态扩展：学习主流异步库（`aiohttp`、`asyncmy`、`aiofiles`、`FastAPI`），结合实战场景开发项目；
6. 混合并发：理解 `asyncio` + `multiprocessing` 的混合方案（异步处理 I/O，多进程处理 CPU 密集任务）。

### 核心推荐
- 语法规范：优先使用 `async/await` 语法，避免 `@asyncio.coroutine` 装饰器；
- 事件循环：Python 3.7+ 用 `asyncio.run()` 管理循环，无需手动创建/关闭；
- 并发控制：简单并发用 `asyncio.gather()`，复杂场景用 `TaskGroup()`（Python 3.11+）；
- 资源复用：HTTP 请求复用 `aiohttp.ClientSession`，数据库复用连接池；
- 并发限制：高并发场景用 `Semaphore` 限制并发数，避免压垮目标服务；
- 同步函数：无异步库时用 `asyncio.to_thread()` 运行同步函数，不阻塞事件循环；
- 异常处理：必须捕获 `CancelledError` 和超时异常，确保资源正常释放。

通过熟练掌握 asyncio 模块，你可以轻松应对高并发 I/O 场景，开发出性能优异、资源占用低的异步应用，大幅提升系统的并发处理能力和响应速度！
