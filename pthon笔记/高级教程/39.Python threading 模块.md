# Python threading 模块完全指南：多线程并发编程实战
threading 是 Python 内置的 **多线程编程标准库**，基于操作系统原生线程（POSIX Threads 或 Windows Threads）实现，核心功能是提供轻量级并发执行能力——通过创建多个线程并行处理任务，充分利用 CPU 资源，尤其适合 I/O 密集型场景（如网络请求、文件读写、数据库操作）。

与多进程（`multiprocessing` 模块）相比，threading 模块创建的线程共享进程内存空间，线程间通信更高效，资源开销更低；但受限于 CPython 的 **GIL（全局解释器锁）**，同一时间仅能有一个线程执行 Python 字节码，因此不适合 CPU 密集型任务（需并行计算可选择 `multiprocessing` 或 `asyncio`）。


## 一、核心定位与优势
### 1. 核心特点
- **内置无依赖**：Python 标准库自带，无需额外安装，导入即使用；
- **轻量高效**：线程创建/销毁的资源开销远低于进程，启动速度快；
- **内存共享**：线程共享所属进程的内存空间（全局变量、堆内存），线程间通信无需额外机制；
- **丰富的同步原语**：支持锁（`Lock`）、递归锁（`RLock`）、信号量（`Semaphore`）、事件（`Event`）等同步工具，解决线程安全问题；
- **支持守护线程**：可设置后台线程（主线程退出时自动终止），适配任务后台执行场景；
- **跨平台兼容**：统一封装不同操作系统的原生线程接口，在 Windows、Linux、macOS 上行为一致。

### 2. 适用场景
- **I/O 密集型任务**：网络请求（爬虫、API 调用）、文件读写、数据库查询、等待用户输入等（线程阻塞时释放 GIL，其他线程可执行）；
- **并发任务调度**：同时处理多个独立任务（如同时监听多个端口、批量处理文件）；
- **后台任务处理**：日志收集、数据同步、定时任务等无需阻塞主程序的场景；
- **UI 程序响应**：GUI 程序中，用子线程处理耗时操作（如文件下载），避免主界面卡顿。

### 3. 与其他并发方案的对比
| 并发方案                | 核心优势                          | 缺点                                  | 适用场景                  |
|-------------------------|-----------------------------------|---------------------------------------|---------------------------|
| `threading` 模块（本文核心） | 轻量、内存共享、通信高效、内置无依赖 | 受 GIL 限制，CPU 密集型任务效率低      | I/O 密集型任务、并发调度、后台任务 |
| `multiprocessing` 模块  | 无 GIL 限制，支持 CPU 并行计算    | 资源开销大、进程间通信复杂（需用 Pipe/Queue） | CPU 密集型任务、大规模并行计算 |
| `asyncio` 模块          | 单线程异步，资源开销极低、并发量高 | 仅支持异步 I/O，需适配异步库（如 aiohttp） | 高并发 I/O 场景（如高并发爬虫、API 服务） |
| `concurrent.futures.ThreadPoolExecutor` | 线程池封装，简化线程管理          | 基于 `threading`，仍受 GIL 限制        | 简化版多线程开发（避免手动管理线程生命周期） |

**结论**：I/O 密集型任务优先用 `threading` 或 `ThreadPoolExecutor`；CPU 密集型任务用 `multiprocessing`；高并发 I/O 用 `asyncio`。


## 二、环境准备
threading 是 Python 内置模块，**无需额外安装**，直接导入即可使用：
```python
import threading
```

### 版本说明
- Python 2/3 均支持 threading 模块，核心功能（`Thread` 类、锁机制）完全一致；
- **关键差异**：
  - Python 3 中 `Thread` 类支持 `daemon` 参数直接设置守护线程（Python 2 需通过 `setDaemon(True)` 方法）；
  - Python 3 新增 `threading.Timer` 类（定时线程）、`threading.local` 优化（线程本地存储）；
- 推荐使用 Python 3.6+，支持更简洁的语法和更稳定的线程管理。

验证模块可用性：
```python
# 简单多线程示例：创建 2 个线程执行任务
def task(name):
    print(f"线程 {name} 启动")

# 创建线程
t1 = threading.Thread(target=task, args=("Thread-1",))
t2 = threading.Thread(target=task, args=("Thread-2",))

# 启动线程
t1.start()
t2.start()

# 等待线程结束
t1.join()
t2.join()

print("所有线程执行完成")

# 输出结果（线程执行顺序不确定，受操作系统调度影响）：
# 线程 Thread-1 启动
# 线程 Thread-2 启动
# 所有线程执行完成
```


## 三、核心概念：理解多线程与 threading 架构
### 1. 多线程基础概念
- **线程（Thread）**：进程内的最小执行单元，一个进程可包含多个线程，共享进程资源（内存、文件句柄等）；
- **主线程**：Python 程序启动时默认创建的线程，负责执行主程序逻辑，其他线程为子线程；
- **守护线程（Daemon Thread）**：后台线程，主程序结束时自动终止（无论是否执行完毕），非守护线程会阻塞主程序退出；
- **线程安全**：多线程并发访问共享资源（如全局变量、文件）时，避免数据错乱或逻辑异常的特性（需通过同步机制实现）；
- **GIL（全局解释器锁）**：CPython 的核心限制，确保同一时间仅一个线程执行 Python 字节码，导致多线程无法利用多核 CPU 进行并行计算（I/O 阻塞时 GIL 释放，其他线程可执行）。

### 2. threading 模块核心组件
threading 模块通过以下核心类/函数实现多线程管理与同步，各司其职：

| 组件                | 中文名称          | 核心作用                                  | 关键特点                          |
|---------------------|-------------------|-------------------------------------------|-----------------------------------|
| `threading.Thread`  | 线程类            | 创建并管理线程，定义线程执行逻辑          | 需指定 `target`（执行函数）或重写 `run()` 方法 |
| `threading.Lock`    | 互斥锁            | 保证同一时间仅一个线程访问共享资源        | 非递归锁，不可重复获取（避免死锁） |
| `threading.RLock`   | 递归锁            | 支持同一线程重复获取（解决嵌套锁场景）    | 自动计数，获取次数需与释放次数一致 |
| `threading.Semaphore` | 信号量           | 限制同时访问共享资源的线程数量            | 可设置最大并发数（如限制 5 个线程同时访问） |
| `threading.Event`   | 事件             | 线程间通信的简单机制（信号触发/等待）     | 支持 `set()`（触发）、`wait()`（等待）、`clear()`（重置） |
| `threading.Condition` | 条件变量         | 结合锁与事件，支持线程间复杂同步（如生产者-消费者） | 需配合锁使用，支持 `wait()`/`notify()` |
| `threading.Barrier` | 屏障             | 等待所有线程到达指定点后再继续执行        | 适合批量线程同步（如所有线程准备就绪后统一开始） |
| `threading.Timer`   | 定时线程          | 延迟指定时间后执行任务                    | 基于 `Thread`，支持 `cancel()` 取消任务 |
| `threading.local`   | 线程本地存储      | 为每个线程创建独立的变量实例（避免共享冲突） | 线程间数据隔离，互不干扰          |
| `threading.active_count()` | 活跃线程数统计 | 返回当前正在运行的线程数量                | 含主线程                          |
| `threading.current_thread()` | 当前线程对象 | 返回当前执行代码的线程实例                | 用于线程身份识别                  |

### 3. 线程生命周期
一个线程从创建到终止的完整流程：
1. **创建**：实例化 `threading.Thread` 对象，指定执行逻辑；
2. **就绪**：调用 `start()` 方法后，线程进入操作系统就绪队列，等待 CPU 调度；
3. **运行**：CPU 调度线程后，执行 `target` 函数或 `run()` 方法；
4. **阻塞**：线程执行 I/O 操作、等待锁/事件时，进入阻塞状态（释放 CPU 资源）；
5. **唤醒**：阻塞条件解除（如 I/O 完成、获取锁成功），线程重新进入就绪队列；
6. **终止**：`target` 函数执行完毕或调用 `exit()`/`raise SystemExit`，线程终止。


## 四、核心功能：分场景详解（语法+示例）
### 1. 创建与启动线程（核心用法）
threading 模块提供两种创建线程的方式：**传入目标函数**（推荐，简洁）和 **继承 Thread 类**（灵活，适合复杂逻辑）。

#### （1）方式 1：传入目标函数（`target` 参数）
最常用的创建方式，通过 `target` 指定线程执行的函数，`args`/`kwargs` 传递参数：
```python
import threading
import time

# 定义线程执行的函数
def worker(name, delay):
    """线程工作函数：每隔 delay 秒打印一次信息"""
    print(f"线程 {name} 启动（延迟 {delay} 秒）")
    time.sleep(delay)  # 模拟 I/O 阻塞（如网络请求、文件读写）
    print(f"线程 {name} 执行完成")

# 1. 创建线程对象（args 是元组，kwargs 是字典，传递函数参数）
t1 = threading.Thread(target=worker, args=("Thread-1", 2))  # args 按顺序传参
t2 = threading.Thread(target=worker, kwargs={"name": "Thread-2", "delay": 1})  # kwargs 关键字传参

# 2. 启动线程（必须调用 start()，而非直接调用 worker()）
t1.start()
t2.start()

# 3. 等待线程结束（join() 阻塞主线程，直到子线程完成）
t1.join()  # 可选：不调用则主线程直接退出，子线程继续执行
t2.join()

print("主线程执行完成")

# 输出结果（Thread-2 延迟 1 秒，先完成；Thread-1 延迟 2 秒，后完成）：
# 线程 Thread-1 启动（延迟 2 秒）
# 线程 Thread-2 启动（延迟 1 秒）
# 线程 Thread-2 执行完成
# 线程 Thread-1 执行完成
# 主线程执行完成
```

#### （2）方式 2：继承 Thread 类（重写 `run()` 方法）
适合线程逻辑复杂的场景，通过继承 `Thread` 类，重写 `run()` 方法定义执行逻辑：
```python
import threading
import time

# 继承 Thread 类，重写 run() 方法
class MyThread(threading.Thread):
    def __init__(self, name, delay):
        super().__init__()  # 调用父类构造函数
        self.name = name
        self.delay = delay

    def run(self):
        """线程执行逻辑（start() 会自动调用 run()）"""
        print(f"自定义线程 {self.name} 启动（延迟 {self.delay} 秒）")
        time.sleep(self.delay)
        print(f"自定义线程 {self.name} 执行完成")

# 创建并启动线程
t1 = MyThread("Thread-1", 2)
t2 = MyThread("Thread-2", 1)
t1.start()
t2.start()

# 等待线程结束
t1.join()
t2.join()

print("主线程执行完成")
```

#### （3）守护线程（后台线程）
通过 `daemon=True` 或 `setDaemon(True)` 设置守护线程，主线程退出时，守护线程会被强制终止（无论是否执行完毕），适用于后台任务（如日志收集）：
```python
import threading
import time

def daemon_worker():
    """守护线程工作函数"""
    while True:
        print("守护线程：正在后台运行...")
        time.sleep(1)

def non_daemon_worker():
    """非守护线程工作函数"""
    print("非守护线程：启动（会阻塞主线程）")
    time.sleep(3)
    print("非守护线程：执行完成")

# 创建守护线程（daemon=True）
daemon_thread = threading.Thread(target=daemon_worker, daemon=True)
# 创建非守护线程（默认 daemon=False）
non_daemon_thread = threading.Thread(target=non_daemon_worker)

# 启动线程
daemon_thread.start()
non_daemon_thread.start()

# 等待非守护线程完成（主线程会阻塞到 non_daemon_thread 结束）
non_daemon_thread.join()

print("主线程退出（守护线程会被强制终止）")

# 输出结果（守护线程在主线程退出后停止）：
# 守护线程：正在后台运行...
# 非守护线程：启动（会阻塞主线程）
# 守护线程：正在后台运行...
# 守护线程：正在后台运行...
# 非守护线程：执行完成
# 主线程退出（守护线程会被强制终止）
```

### 2. 线程同步：解决共享资源竞争（核心难点）
多线程共享进程内存，当多个线程同时修改共享资源（如全局变量、文件）时，会出现 **数据错乱**（竞态条件），需通过同步机制保证线程安全。

#### （1）互斥锁（`threading.Lock`）：最基础的同步工具
保证同一时间仅一个线程访问共享资源，需手动 `acquire()`（获取锁）和 `release()`（释放锁），或用 `with` 语句自动管理（推荐，避免忘记释放锁导致死锁）：
```python
import threading
import time

# 共享资源（全局变量）
count = 0
# 创建互斥锁
lock = threading.Lock()

def increment(name):
    """线程函数：修改共享变量 count"""
    global count
    for _ in range(100000):  # 循环次数足够大，触发竞态条件
        # 方式 1：手动获取/释放锁（需处理异常，避免死锁）
        # lock.acquire()
        # try:
        #     count += 1
        # finally:
        #     lock.release()

        # 方式 2：with 语句自动获取/释放锁（推荐）
        with lock:
            count += 1  # 临界区：同一时间仅一个线程执行
    print(f"线程 {name} 执行完成，count = {count}")

# 创建 2 个线程修改共享变量
t1 = threading.Thread(target=increment, args=("Thread-1",))
t2 = threading.Thread(target=increment, args=("Thread-2",))

t1.start()
t2.start()
t1.join()
t2.join()

print(f"最终 count 值：{count}")  # 正确结果应为 200000（无锁时可能小于 200000）
```

#### （2）递归锁（`threading.RLock`）：解决嵌套锁场景
`Lock` 是非递归锁，同一线程重复获取会导致死锁；`RLock`（递归锁）支持同一线程多次获取，自动计数，需释放相同次数：
```python
import threading

lock = threading.RLock()  # 递归锁
count = 0

def nested_func():
    """嵌套调用的函数，需重复获取锁"""
    global count
    with lock:  # 第二次获取锁（不会死锁）
        count += 1
        print(f"嵌套函数：count = {count}")

def worker(name):
    with lock:  # 第一次获取锁
        print(f"线程 {name}：第一次获取锁")
        nested_func()  # 调用嵌套函数，再次获取锁
    print(f"线程 {name}：执行完成")

t1 = threading.Thread(target=worker, args=("Thread-1",))
t2 = threading.Thread(target=worker, args=("Thread-2",))

t1.start()
t2.start()
t1.join()
t2.join()
```

#### （3）信号量（`threading.Semaphore`）：限制并发数
控制同时访问共享资源的线程数量（如限制 5 个线程同时访问数据库），本质是计数器：
```python
import threading
import time

# 信号量：最多允许 3 个线程同时访问
semaphore = threading.Semaphore(3)
resource = "数据库连接"

def access_resource(name):
    """线程函数：访问受限资源"""
    with semaphore:  # 获取信号量（计数器减 1，为 0 时阻塞）
        print(f"线程 {name}：获取 {resource}，开始访问...")
        time.sleep(2)  # 模拟资源访问耗时
        print(f"线程 {name}：释放 {resource}")

# 创建 10 个线程，同时竞争 3 个资源
threads = [threading.Thread(target=access_resource, args=(f"Thread-{i}",)) for i in range(10)]

for t in threads:
    t.start()
for t in threads:
    t.join()

print("所有线程访问完成")
```

#### （4）事件（`threading.Event`）：线程间信号通知
用于线程间简单通信，一个线程发送信号（`set()`），其他线程等待信号（`wait()`）：
```python
import threading
import time

# 创建事件对象
event = threading.Event()

def wait_for_signal(name):
    """等待信号的线程"""
    print(f"线程 {name}：等待信号触发...")
    event.wait()  # 阻塞，直到事件被 set()
    print(f"线程 {name}：收到信号，开始执行任务")
    time.sleep(1)
    print(f"线程 {name}：任务执行完成")

def send_signal():
    """发送信号的线程"""
    print("信号线程：准备发送信号（延迟 3 秒）...")
    time.sleep(3)
    event.set()  # 触发事件，唤醒所有等待的线程
    # event.clear()  # 可选：重置事件，允许再次等待

# 创建 3 个等待线程
wait_threads = [threading.Thread(target=wait_for_signal, args=(f"Wait-{i}",)) for i in range(3)]
# 创建信号线程
signal_thread = threading.Thread(target=send_signal)

# 启动线程
for t in wait_threads:
    t.start()
signal_thread.start()

# 等待所有线程完成
for t in wait_threads:
    t.join()
signal_thread.join()
```

#### （5）条件变量（`threading.Condition`）：复杂同步（生产者-消费者模式）
结合锁与事件，支持线程间“等待-通知”机制，适合生产者-消费者等需要动态同步的场景：
```python
import threading
import time
from collections import deque

# 缓冲区（共享队列）
buffer = deque(maxlen=5)  # 最大容量 5
# 创建条件变量（内置锁）
condition = threading.Condition()

class Producer(threading.Thread):
    """生产者线程：向缓冲区添加数据"""
    def run(self):
        for i in range(10):
            time.sleep(0.5)  # 模拟生产耗时
            data = f"产品-{i}"
            
            with condition:
                # 缓冲区满时，等待消费者消费（释放锁，进入阻塞）
                while len(buffer) >= buffer.maxlen:
                    print(f"生产者 {self.name}：缓冲区满，等待消费...")
                    condition.wait()  # 等待通知，被唤醒后重新获取锁
                
                # 生产数据，添加到缓冲区
                buffer.append(data)
                print(f"生产者 {self.name}：生产 {data}，缓冲区：{list(buffer)}")
                
                # 通知消费者：有新数据可消费
                condition.notify()  # 唤醒一个等待的消费者（notify_all() 唤醒所有）

class Consumer(threading.Thread):
    """消费者线程：从缓冲区获取数据"""
    def run(self):
        for _ in range(10):
            time.sleep(1)  # 模拟消费耗时
            
            with condition:
                # 缓冲区空时，等待生产者生产
                while len(buffer) == 0:
                    print(f"消费者 {self.name}：缓冲区空，等待生产...")
                    condition.wait()
                
                # 消费数据，从缓冲区取出
                data = buffer.popleft()
                print(f"消费者 {self.name}：消费 {data}，缓冲区：{list(buffer)}")
                
                # 通知生产者：有空闲位置可生产
                condition.notify()

# 创建 2 个生产者、3 个消费者
producers = [Producer(name=f"P{i}") for i in range(2)]
consumers = [Consumer(name=f"C{i}") for i in range(3)]

# 启动线程
for p in producers:
    p.start()
for c in consumers:
    c.start()

# 等待所有线程完成
for p in producers:
    p.join()
for c in consumers:
    c.join()

print("所有产品生产和消费完成")
```

### 3. 线程通信：安全的数据传递
线程共享内存，可通过全局变量通信，但需配合锁保证安全；更推荐使用 `queue.Queue`（线程安全的队列），无需手动加锁，支持生产者-消费者模式：

#### （1）`queue.Queue`：线程安全的队列（推荐）
`queue.Queue` 是 Python 内置的线程安全队列，`put()`（添加元素）和 `get()`（获取元素）方法自带同步机制，适合线程间数据传递：
```python
import threading
import queue
import time

# 创建线程安全队列（最大容量 3）
q = queue.Queue(maxsize=3)

def producer():
    """生产者：向队列添加数据"""
    for i in range(5):
        time.sleep(0.5)
        data = f"数据-{i}"
        q.put(data)  # 队列满时阻塞，直到有空闲位置
        print(f"生产者：放入 {data}，队列大小：{q.qsize()}")

def consumer():
    """消费者：从队列获取数据"""
    for _ in range(5):
        time.sleep(1)
        data = q.get()  # 队列空时阻塞，直到有数据
        print(f"消费者：取出 {data}，队列大小：{q.qsize()}")
        q.task_done()  # 标记任务完成（配合 join() 等待所有任务）

# 创建并启动线程
p = threading.Thread(target=producer)
c = threading.Thread(target=consumer)
p.start()
c.start()

q.join()  # 阻塞主线程，直到队列中所有任务被标记为 done
print("所有数据传递完成")
```

#### （2）线程本地存储（`threading.local`）：线程隔离的数据
`threading.local` 创建的对象，每个线程有独立的实例，线程间数据互不干扰，无需锁保护：
```python
import threading
import time

# 创建线程本地存储对象
local_data = threading.local()

def worker(name):
    """线程函数：操作线程本地数据"""
    # 每个线程的 local_data.name 是独立的
    local_data.name = name
    for i in range(3):
        time.sleep(0.5)
        print(f"线程 {threading.current_thread().name}：local_data.name = {local_data.name}")

# 创建 2 个线程
t1 = threading.Thread(target=worker, args=("张三",), name="Thread-A")
t2 = threading.Thread(target=worker, args=("李四",), name="Thread-B")

t1.start()
t2.start()
t1.join()
t2.join()

# 输出结果（线程间数据隔离）：
# 线程 Thread-A：local_data.name = 张三
# 线程 Thread-B：local_data.name = 李四
# 线程 Thread-A：local_data.name = 张三
# 线程 Thread-B：local_data.name = 李四
```

### 4. 线程池：简化线程管理（`concurrent.futures.ThreadPoolExecutor`）
手动创建线程需管理生命周期（启动、等待、销毁），适合少量线程；大量线程场景推荐使用 `concurrent.futures.ThreadPoolExecutor`（基于 threading 模块封装的线程池），自动管理线程复用，降低资源开销：
```python
import concurrent.futures
import time

def task(name, delay):
    """线程任务函数"""
    print(f"任务 {name} 启动（延迟 {delay} 秒）")
    time.sleep(delay)
    return f"任务 {name} 执行完成"

# 创建线程池（max_workers 指定最大线程数）
with concurrent.futures.ThreadPoolExecutor(max_workers=3) as executor:
    # 方式 1：提交单个任务（返回 Future 对象）
    future1 = executor.submit(task, "Task-1", 2)
    future2 = executor.submit(task, "Task-2", 1)
    
    # 获取任务结果（阻塞直到任务完成）
    print(f"结果：{future1.result()}")
    print(f"结果：{future2.result()}")

    # 方式 2：批量提交任务（map 函数，按顺序返回结果）
    tasks = [("Task-3", 1.5), ("Task-4", 0.8), ("Task-5", 2.2)]
    for result in executor.map(task, [t[0] for t in tasks], [t[1] for t in tasks]):
        print(f"批量任务结果：{result}")

print("所有任务执行完成")
```


## 五、实战场景：threading 模块典型应用
### 场景 1：多线程爬虫（I/O 密集型，核心场景）
爬取多个网页时，网络请求是 I/O 阻塞操作，多线程可并行发起请求，大幅提升效率：
```python
import threading
import requests
from bs4 import BeautifulSoup  # 需安装：pip install beautifulsoup4
from queue import Queue

# 爬取队列（存储待爬取的 URL）
url_queue = Queue()
# 结果队列（存储爬取结果）
result_queue = Queue()

def crawler():
    """爬虫线程：从队列获取 URL 并爬取"""
    while not url_queue.empty():
        url = url_queue.get()
        try:
            response = requests.get(url, headers={"User-Agent": "Mozilla/5.0"}, timeout=10)
            soup = BeautifulSoup(response.text, "html.parser")
            title = soup.find("title").get_text().strip() if soup.find("title") else "无标题"
            result = {"url": url, "title": title, "status": response.status_code}
            result_queue.put(result)
            print(f"爬取成功：{url} → 标题：{title}")
        except Exception as e:
            result_queue.put({"url": url, "error": str(e), "status": -1})
            print(f"爬取失败：{url} → 错误：{e}")
        finally:
            url_queue.task_done()

# 1. 初始化待爬取 URL
urls = [
    "https://www.baidu.com",
    "https://www.zhihu.com",
    "https://www.github.com",
    "https://www.python.org",
    "https://www.csdn.net"
]
for url in urls:
    url_queue.put(url)

# 2. 创建 3 个爬虫线程
thread_count = 3
threads = [threading.Thread(target=crawler) for _ in range(thread_count)]

# 3. 启动线程
for t in threads:
    t.start()

# 4. 等待所有 URL 爬取完成
url_queue.join()

# 5. 输出爬取结果
print("\n爬取结果汇总：")
while not result_queue.empty():
    print(result_queue.get())
```

### 场景 2：多线程文件处理（批量读取/写入文件）
批量处理多个文件时，文件 I/O 是阻塞操作，多线程可并行处理，提升效率：
```python
import threading
import os
from queue import Queue

# 文件队列（存储待处理的文件路径）
file_queue = Queue()

def process_file():
    """文件处理线程：读取文件内容，统计行数"""
    while not file_queue.empty():
        file_path = file_queue.get()
        try:
            with open(file_path, "r", encoding="utf-8") as f:
                lines = f.readlines()
                line_count = len(lines)
                print(f"文件 {os.path.basename(file_path)}：共 {line_count} 行")
        except Exception as e:
            print(f"处理文件 {file_path} 失败：{e}")
        finally:
            file_queue.task_done()

# 1. 初始化待处理文件（遍历指定目录下的所有 .txt 文件）
target_dir = "./text_files"
for filename in os.listdir(target_dir):
    if filename.endswith(".txt"):
        file_path = os.path.join(target_dir, filename)
        file_queue.put(file_path)

# 2. 创建 4 个文件处理线程
thread_count = 4
threads = [threading.Thread(target=process_file) for _ in range(thread_count)]

# 3. 启动线程
for t in threads:
    t.start()

# 4. 等待所有文件处理完成
file_queue.join()

print("所有文件处理完成")
```

### 场景 3：多线程数据库查询（I/O 密集型）
批量执行数据库查询时，网络延迟是阻塞点，多线程可并行发起查询，减少总耗时：
```python
import threading
import pymysql  # 需安装：pip install pymysql
from queue import Queue

# 数据库配置
DB_CONFIG = {
    "host": "localhost",
    "user": "root",
    "password": "123456",
    "database": "test_db",
    "charset": "utf8mb4"
}

# 查询队列（存储待查询的 SQL）
sql_queue = Queue()
# 结果队列（存储查询结果）
result_queue = Queue()

def db_query():
    """数据库查询线程：从队列获取 SQL 并执行"""
    # 每个线程创建独立的数据库连接（避免线程安全问题）
    conn = pymysql.connect(**DB_CONFIG)
    cursor = conn.cursor(pymysql.cursors.DictCursor)
    
    while not sql_queue.empty():
        sql = sql_queue.get()
        try:
            cursor.execute(sql)
            results = cursor.fetchall()
            result_queue.put({"sql": sql, "results": results, "error": None})
            print(f"执行 SQL 成功：{sql} → 结果数：{len(results)}")
        except Exception as e:
            result_queue.put({"sql": sql, "results": None, "error": str(e)})
            print(f"执行 SQL 失败：{sql} → 错误：{e}")
        finally:
            sql_queue.task_done()
    
    # 关闭连接
    cursor.close()
    conn.close()

# 1. 初始化待执行的 SQL
sqls = [
    "SELECT * FROM users WHERE age > 25",
    "SELECT * FROM orders WHERE create_time > '2025-01-01'",
    "SELECT COUNT(*) AS total FROM products",
    "SELECT category, COUNT(*) AS count FROM articles GROUP BY category"
]
for sql in sqls:
    sql_queue.put(sql)

# 2. 创建 2 个查询线程（避免过多连接导致数据库压力）
thread_count = 2
threads = [threading.Thread(target=db_query) for _ in range(thread_count)]

# 3. 启动线程
for t in threads:
    t.start()

# 4. 等待所有 SQL 执行完成
sql_queue.join()

# 5. 输出查询结果
print("\n查询结果汇总：")
while not result_queue.empty():
    print(result_queue.get())
```

### 场景 4：后台定时任务（守护线程+Timer）
用守护线程+`threading.Timer` 实现定时任务（如每隔 5 秒打印日志）：
```python
import threading
import time

def print_log():
    """定时任务：打印当前时间日志"""
    current_time = time.strftime("%Y-%m-%d %H:%M:%S", time.localtime())
    print(f"[{current_time}] 后台日志：系统正常运行中...")
    
    # 递归创建 Timer，实现循环执行（每隔 5 秒）
    timer = threading.Timer(5, print_log)
    timer.daemon = True  # 设置为守护线程
    timer.start()

# 启动定时任务（首次延迟 0 秒执行）
print_log()

# 主程序运行 20 秒后退出
print("主程序启动，将运行 20 秒...")
time.sleep(20)
print("主程序退出（守护线程定时任务停止）")
```


## 六、常见问题与避坑指南
### 1. 坑 1：混淆 `start()` 和 `run()` 方法（最常见错误）
- **问题**：直接调用 `thread.run()` 而非 `thread.start()`，导致线程未真正启动（同步执行而非并发）；
- **原因**：`start()` 方法会向操作系统注册线程，触发 `run()` 方法异步执行；直接调用 `run()` 只是普通函数调用，无并发效果；
- **解决方案**：始终用 `start()` 启动线程，切勿直接调用 `run()`：
  ```python
  # 错误：直接调用 run()，同步执行
  t = threading.Thread(target=worker, args=("错误线程", 1))
  t.run()  # 同步执行，阻塞主线程

  # 正确：调用 start()，异步执行
  t = threading.Thread(target=worker, args=("正确线程", 1))
  t.start()  # 并发执行，不阻塞主线程
  ```

### 2. 坑 2：GIL 导致 CPU 密集型任务效率低下
- **问题**：多线程处理 CPU 密集型任务（如数学计算、循环迭代）时，效率甚至低于单线程；
- **原因**：CPython 的 GIL 确保同一时间仅一个线程执行 Python 字节码，多线程在 CPU 密集型任务中会频繁切换线程，产生额外开销；
- **解决方案**：
  1. CPU 密集型任务用 `multiprocessing` 模块（多进程，无 GIL 限制）；
  2. 用 C 扩展（如 `numpy`）或 JIT 编译器（如 PyPy）绕过 GIL；
  ```python
  # 错误：多线程处理 CPU 密集型任务（效率低）
  def cpu_intensive_task():
      sum = 0
      for _ in range(10**8):
          sum += 1
      return sum

  # 正确：多进程处理 CPU 密集型任务
  from multiprocessing import Process
  p1 = Process(target=cpu_intensive_task)
  p2 = Process(target=cpu_intensive_task)
  p1.start()
  p2.start()
  ```

### 3. 坑 3：共享资源未加锁导致数据错乱
- **问题**：多线程同时修改全局变量、文件等共享资源，导致数据不一致（如计数错误、文件内容错乱）；
- **解决方案**：
  1. 共享资源访问时必须加锁（`Lock`/`RLock`）；
  2. 优先使用线程安全的数据结构（如 `queue.Queue`）；
  3. 避免共享资源，用线程本地存储（`threading.local`）或参数传递数据。

### 4. 坑 4：死锁（线程互相等待对方释放锁）
- **问题**：多个线程持有对方需要的锁，互相等待，导致程序卡死；
- **常见原因**：
  1. 线程获取锁的顺序不一致（如线程 A 先锁 1 再锁 2，线程 B 先锁 2 再锁 1）；
  2. 递归锁未正确释放（`RLock` 获取次数 != 释放次数）；
- **解决方案**：
  1. 统一所有线程的锁获取顺序；
  2. 用 `with` 语句自动释放锁，避免手动释放遗漏；
  3. 避免嵌套锁（如需嵌套，用 `RLock`）；
  4. 设置锁超时（如 `lock.acquire(timeout=5)`），避免无限阻塞；
  ```python
  # 死锁示例（错误）
  lock1 = threading.Lock()
  lock2 = threading.Lock()

  def thread_a():
      with lock1:
          time.sleep(1)
          with lock2:  # 等待 lock2，此时线程 B 已持有 lock2
              print("线程 A 获得所有锁")

  def thread_b():
      with lock2:
          time.sleep(1)
          with lock1:  # 等待 lock1，此时线程 A 已持有 lock1
              print("线程 B 获得所有锁")

  # 正确：统一锁获取顺序（先 lock1 再 lock2）
  def thread_b_fixed():
      with lock1:  # 先获取 lock1
          time.sleep(1)
          with lock2:
              print("线程 B 获得所有锁")
  ```

### 5. 坑 5：守护线程未处理资源释放
- **问题**：守护线程在主线程退出时被强制终止，可能导致资源未释放（如文件未关闭、数据库连接未断开）；
- **解决方案**：
  1. 守护线程中捕获 `SystemExit` 异常，在退出前释放资源；
  2. 非关键资源用守护线程，关键资源用非守护线程（配合 `join()` 等待完成）；
  ```python
  def daemon_worker():
      file = None
      try:
          file = open("daemon_log.txt", "a", encoding="utf-8")
          while True:
              file.write(f"[{time.time()}] 日志\n")
              file.flush()
              time.sleep(1)
      except SystemExit:
          # 主线程退出时，释放资源
          if file:
              file.close()
              print("守护线程：文件已关闭")
  ```

### 6. 坑 6：线程泄漏（线程未正常终止）
- **问题**：线程进入无限循环或阻塞在未设置超时的操作（如 `queue.get()` 无超时），导致线程无法终止，占用资源；
- **解决方案**：
  1. 线程任务设置超时机制（如 `queue.get(timeout=10)`）；
  2. 用 `Event` 作为退出信号，让线程优雅退出；
  ```python
  # 优雅退出的线程示例
  exit_event = threading.Event()

  def worker():
      while not exit_event.is_set():
          try:
              # 阻塞操作设置超时
              data = q.get(timeout=2)
              process_data(data)
          except queue.Empty:
              continue  # 超时后继续循环，检查退出信号
      print("线程优雅退出")

  # 主线程：触发退出信号
  exit_event.set()
  thread.join()
  ```

### 7. 坑 7：多线程操作非线程安全的库/对象
- **问题**：使用非线程安全的第三方库（如某些数据库驱动、网络库）或对象（如 `requests.Session` 未正确配置），导致程序崩溃或数据异常；
- **解决方案**：
  1. 每个线程创建独立的库实例（如每个线程一个 `requests.Session`、一个数据库连接）；
  2. 查阅库文档，确认是否线程安全，非线程安全则加锁保护；
  ```python
  # 正确：每个线程独立创建 requests.Session
  def worker():
      session = requests.Session()  # 线程独立的 Session
      response = session.get(url)
      session.close()
  ```

## 八、总结
threading 模块是 Python 处理 I/O 密集型并发任务的核心工具，核心价值在于 **轻量高效、内存共享、内置无依赖**，配合同步原语和线程安全队列，可安全解决多线程数据共享与通信问题。

### 学习路径建议
1. 基础入门：掌握线程创建（`Thread` 类）、启动（`start()`）、等待（`join()`）、守护线程等基础用法；
2. 进阶应用：理解线程安全问题，熟练使用锁（`Lock`/`RLock`）、信号量、事件等同步原语；
3. 实战优化：掌握 `queue.Queue` 线程通信、线程池（`ThreadPoolExecutor`）、线程本地存储，解决实际场景问题；
4. 避坑指南：重点关注死锁、线程泄漏、GIL 限制、非线程安全库等问题，培养线程安全编程思维；
5. 扩展提升：复杂并发场景可学习 `asyncio`（异步 I/O）、`multiprocessing`（多进程），根据任务类型选择合适的并发方案。

### 核心推荐
- 任务类型判断：I/O 密集型用 `threading` 或 `ThreadPoolExecutor`，CPU 密集型用 `multiprocessing`；
- 线程数量设置：I/O 密集型任务线程数可设为 CPU 核心数 * 5 ~ 10（根据 I/O 延迟调整），避免过多线程导致切换开销；
- 同步优先选择：简单共享资源用 `Lock`，嵌套锁用 `RLock`，限制并发用 `Semaphore`，线程通信用 `queue.Queue`；
- 优雅退出：用 `Event` 或超时机制让线程正常退出，避免强制终止导致资源泄漏；
- 库选择：多线程开发优先使用 `concurrent.futures.ThreadPoolExecutor`，简化线程管理，减少手动操作。

通过熟练掌握 threading 模块，你可以高效解决 I/O 密集型并发任务（如爬虫、文件处理、数据库查询），大幅提升程序执行效率，同时避开多线程常见陷阱，编写稳定可靠的并发代码！
