CGI（Common Gateway Interface，通用网关接口）是一种让Web服务器与外部程序（如Python脚本）交互的标准协议。通过CGI，Web服务器可以调用Python脚本处理用户请求（如表单提交），脚本生成动态内容（如HTML）并返回给服务器，最终由服务器发送给浏览器。

虽然现代Web开发中CGI已逐渐被WSGI、FastAPI等更高效的框架替代，但理解CGI有助于掌握Web交互的基本原理。


### 一、CGI工作流程
1. **用户请求**：用户通过浏览器访问特定URL（指向CGI脚本）。  
2. **服务器调用CGI**：Web服务器（如Apache、Nginx）识别出请求指向CGI脚本，启动脚本进程。  
3. **脚本处理**：Python脚本通过环境变量获取请求信息（如参数、请求方法），处理后生成响应内容（通常是HTML）。  
4. **返回响应**：脚本将响应内容输出到标准输出（stdout），服务器捕获后返回给浏览器。  


### 二、环境准备：配置Web服务器支持CGI
要运行Python CGI脚本，需先配置Web服务器（以**Apache**为例）：


#### 1. 安装Apache
- **Linux（Ubuntu/Debian）**：  
  ```bash
  sudo apt update && sudo apt install apache2
  ```
- **Windows**：下载 [Apache Lounge](https://www.apachelounge.com/)，解压后配置环境变量。


#### 2. 启用CGI模块
Apache默认可能未启用CGI模块，需手动启用：  
- **Linux**：  
  ```bash
  sudo a2enmod cgi  # 启用CGI模块
  sudo systemctl restart apache2  # 重启服务器
  ```
- **Windows**：编辑Apache配置文件 `conf/httpd.conf`，取消以下行的注释（删除`#`）：  
  ```apache
  LoadModule cgi_module modules/mod_cgi.so
  ```


#### 3. 配置CGI脚本目录
指定存放CGI脚本的目录（如 `/var/www/cgi-bin`），并允许服务器执行该目录下的脚本：  
编辑Apache配置文件（Linux：`/etc/apache2/sites-available/000-default.conf`；Windows：`conf/httpd.conf`），添加：  
```apache
<Directory "/var/www/cgi-bin">  # 脚本目录（Linux）
    Options +ExecCGI
    AddHandler cgi-script .py  # 允许执行.py后缀的CGI脚本
    Require all granted
</Directory>
ScriptAlias /cgi-bin/ "/var/www/cgi-bin/"  # 映射URL路径到实际目录
```

**Windows路径示例**：  
```apache
<Directory "C:/Apache24/cgi-bin">
    Options +ExecCGI
    AddHandler cgi-script .py
    Require all granted
</Directory>
ScriptAlias /cgi-bin/ "C:/Apache24/cgi-bin/"
```


#### 4. 重启服务器
- **Linux**：`sudo systemctl restart apache2`  
- **Windows**：在Apache安装目录的 `bin` 文件夹中，执行 `httpd -k restart`  


### 三、第一个Python CGI脚本
创建一个简单的CGI脚本（如 `hello_cgi.py`），放在配置的CGI目录（如 `/var/www/cgi-bin`）中：

```python
#!/usr/bin/env python3
# 上述行指定Python解释器路径（Linux必填，Windows可选）

# 必须先输出HTTP头（告诉服务器响应类型），以空行结束
print("Content-Type: text/html\n")

# 输出HTML内容
print("<html>")
print("<head><title>Python CGI</title></head>")
print("<body>")
print("<h1>Hello, CGI!</h1>")
print("</body>")
print("</html>")
```


#### 脚本注意事项：
1. **权限**（Linux）：确保脚本有执行权限：  
   ```bash
   chmod +x /var/www/cgi-bin/hello_cgi.py
   ```
2. **HTTP头**：必须以 `Content-Type` 开头，且头与内容之间需有一个空行（`\n\n`），否则服务器无法识别响应格式。  
3. **解释器路径**：第一行 `#!/usr/bin/env python3` 告诉Linux系统用Python3执行脚本（Windows可省略）。  


#### 访问脚本：
在浏览器中访问：  
```
http://localhost/cgi-bin/hello_cgi.py
```  
若配置正确，会显示 `Hello, CGI!`。


### 四、CGI环境变量：获取请求信息
Web服务器会通过**环境变量**将请求信息传递给CGI脚本，Python中可通过 `os.environ` 读取这些变量。常用环境变量：

| 环境变量         | 含义                                  |
|------------------|---------------------------------------|
| `REQUEST_METHOD` | 请求方法（`GET` 或 `POST`）           |
| `QUERY_STRING`   | GET请求的参数（如 `name=Alice&age=20`）|
| `CONTENT_LENGTH` | POST请求的参数长度（字节数）          |
| `REMOTE_ADDR`    | 客户端IP地址                          |
| `HTTP_USER_AGENT`| 客户端浏览器信息                      |


#### 示例：获取请求信息
```python
#!/usr/bin/env python3
import os

print("Content-Type: text/html\n")

print("<html><body>")
print("<h2>请求信息</h2>")
print(f"请求方法：{os.environ.get('REQUEST_METHOD', '未知')}<br>")
print(f"客户端IP：{os.environ.get('REMOTE_ADDR', '未知')}<br>")
print(f"浏览器信息：{os.environ.get('HTTP_USER_AGENT', '未知')}<br>")
print("</body></html>")
```


### 五、处理GET和POST请求
用户提交的数据（如表单）通过GET或POST方法传递，CGI脚本需分别处理。


#### 1. 处理GET请求
GET请求的参数附加在URL后（如 `http://...?name=Alice&age=20`），存储在 `QUERY_STRING` 环境变量中，可通过 `urllib.parse` 解析。

**示例：处理GET参数**
```python
#!/usr/bin/env python3
import os
from urllib.parse import parse_qs

print("Content-Type: text/html\n")

# 获取GET参数
query = os.environ.get('QUERY_STRING', '')
params = parse_qs(query)  # 解析为字典（值为列表，因可能有多个同名参数）

name = params.get('name', ['匿名'])[0]  # 取第一个值
age = params.get('age', ['未知'])[0]

print("<html><body>")
print(f"<h1>Hello, {name}!</h1>")
print(f"年龄：{age}")
print("</body></html>")
```

访问URL：`http://localhost/cgi-bin/get_demo.py?name=Bob&age=18`，会显示：  
`Hello, Bob! 年龄：18`


#### 2. 处理POST请求
POST请求的参数放在请求体中，需通过标准输入（`sys.stdin`）读取，长度由 `CONTENT_LENGTH` 环境变量指定。

**示例：处理POST参数**
```python
#!/usr/bin/env python3
import os
import sys
from urllib.parse import parse_qs

print("Content-Type: text/html\n")

# 获取POST参数
if os.environ.get('REQUEST_METHOD') == 'POST':
    content_length = int(os.environ.get('CONTENT_LENGTH', 0))
    post_data = sys.stdin.read(content_length)  # 读取请求体
    params = parse_qs(post_data)
else:
    params = {}

name = params.get('name', ['匿名'])[0]
message = params.get('message', ['无消息'])[0]

print("<html><body>")
print(f"<h2>收到{name}的消息：</h2>")
print(f"<p>{message}</p>")
print("</body></html>")
```

配合一个HTML表单（放在服务器的网页目录，如 `/var/www/html/form.html`）：  
```html
<form action="/cgi-bin/post_demo.py" method="post">
    姓名：<input type="text" name="name"><br>
    消息：<textarea name="message"></textarea><br>
    <input type="submit" value="提交">
</form>
```

访问 `http://localhost/form.html`，提交表单后，会显示提交的内容。


### 六、常见问题与调试
1. **500 Internal Server Error**：  
   - 检查脚本是否有执行权限（Linux：`chmod +x`）。  
   - 确保第一行 `Content-Type` 正确，且有 `\n\n` 分隔头和内容。  
   - 查看服务器错误日志（Linux：`/var/log/apache2/error.log`；Windows：`logs/error.log`）定位具体错误（如语法错误）。

2. **脚本不执行，直接下载**：  
   - 检查Apache配置中是否正确添加 `AddHandler cgi-script .py`，确保 `.py` 被识别为CGI脚本。

3. **中文乱码**：  
   - 在HTML中指定编码：`print("<meta charset='utf-8'>")`。  


### 七、CGI的局限性
- **性能低**：每个请求都会启动新的Python进程，不适合高并发场景。  
- **功能有限**：需手动处理参数解析、会话管理等，开发效率低。  
- **现代替代方案**：WSGI（Web Server Gateway Interface）协议及框架（如Flask、Django）已成为主流，性能更好且开发更便捷。  


### 总结
CGI是Web服务器与外部程序交互的基础协议，Python通过环境变量和标准输入输出实现CGI编程。核心步骤是：  
1. 配置Web服务器支持CGI；  
2. 脚本开头输出 `Content-Type` 头；  
3. 通过环境变量（GET）或标准输入（POST）获取请求数据；  
4. 生成HTML响应并输出。  

虽然CGI已不是现代Web开发的首选，但理解其原理有助于掌握Web交互的本质。
