# Python OpenAI 库完全使用指南
OpenAI 官方提供的 `openai` Python 库，是调用 GPT（3.5/4/4o）、DALL-E、Whisper 等 OpenAI 模型的核心工具。它封装了 OpenAI API 的所有功能，支持文本生成、图像生成、语音转文字、函数调用等场景，是 AI 应用开发的必备库。


## 一、环境准备
### 1. 安装 OpenAI 库
需 Python 3.8+ 版本，通过 pip 安装（推荐最新版本，兼容更多模型）：
```bash
pip3 install openai --upgrade
```

验证安装成功：
```python
import openai
print(openai.__version__)  # 输出版本号（如 1.40.0，正常则安装成功）
```

### 2. 获取 API 密钥
调用 OpenAI API 需先获取 API 密钥，步骤如下：
1. 访问 [OpenAI 官网](https://platform.openai.com/)，注册/登录账号；
2. 进入「API Keys」页面（https://platform.openai.com/api-keys）；
3. 点击「Create new secret key」，生成并复制密钥（仅显示一次，需妥善保存）。

### 3. 配置 API 密钥（核心步骤）
有 3 种配置方式，推荐「环境变量」（安全无硬编码）：

#### 方式 1：环境变量配置（推荐）
- **Windows**（命令提示符）：
  ```cmd
  set OPENAI_API_KEY="sk-你的密钥"
  ```
  （PowerShell 用：`$env:OPENAI_API_KEY="sk-你的密钥"`）

- **macOS/Linux**（终端）：
  ```bash
  export OPENAI_API_KEY="sk-你的密钥"
  ```
  （永久生效：编辑 `~/.bashrc` 或 `~/.zshrc`，添加上述命令，执行 `source ~/.bashrc` 生效）

#### 方式 2：代码内直接配置（测试用）
```python
import openai

# 初始化客户端（v1+ 版本推荐用法）
client = openai.OpenAI(
    api_key="sk-你的密钥",  # 替换为你的 API 密钥
    # 若使用代理（国内网络），添加 base_url
    # base_url="https://api.openai-proxy.com/v1"  # 示例代理，需替换为有效代理
)
```

#### 方式 3：配置文件（企业级）
创建 `~/.config/openai/api_key` 文件，写入密钥（仅当前用户生效）。


## 二、核心基础：文本生成（GPT 模型）
最常用场景，调用 GPT-3.5/4/4o 生成文本、对话、问答等，核心是 `ChatCompletion` 接口。

### 1. 简单文本生成（单轮对话）
```python
import openai

# 初始化客户端（自动读取环境变量 OPENAI_API_KEY）
client = openai.OpenAI()

# 发送聊天请求
response = client.chat.completions.create(
    model="gpt-3.5-turbo",  # 模型名称（gpt-4o 更优，gpt-3.5-turbo 更经济）
    messages=[
        # 角色：system（系统提示）、user（用户提问）、assistant（助手回复，多轮对话时需携带）
        {"role": "system", "content": "你是一个简洁的助手，只回答核心要点"},
        {"role": "user", "content": "Python 列表推导式的语法和优势"}
    ],
    temperature=0.7,  # 随机性（0=确定，1=随机，默认 0.7）
    max_tokens=500,    # 最大生成 tokens 数（含输入+输出，默认 1024）
)

# 提取响应结果
result = response.choices[0].message.content
print("GPT 回复：", result)
```

**输出示例**：
```
语法：[表达式 for 变量 in 可迭代对象 if 条件]
优势：1. 代码简洁，一行完成循环+过滤；2. 执行效率高于普通 for 循环；3. 可读性强（逻辑清晰）
```

### 2. 多轮对话（上下文关联）
多轮对话需在 `messages` 中携带历史对话记录（`user` 和 `assistant` 交替）：
```python
import openai

client = openai.OpenAI()

# 历史对话列表（初始含系统提示）
messages = [
    {"role": "system", "content": "你是一个数学老师，逐步讲解解题步骤"},
    {"role": "user", "content": "解方程：2x + 5 = 15"}
]

# 第一轮请求
response1 = client.chat.completions.create(model="gpt-3.5-turbo", messages=messages)
assistant_msg1 = response1.choices[0].message.content
print("助手：", assistant_msg1)

# 将助手回复加入历史，发起第二轮提问（上下文关联）
messages.append({"role": "assistant", "content": assistant_msg1})
messages.append({"role": "user", "content": "如果把等式改成 3x - 4 = 8，解法一样吗？"})

# 第二轮请求
response2 = client.chat.completions.create(model="gpt-3.5-turbo", messages=messages)
print("助手：", response2.choices[0].message.content)
```

**关键**：`messages` 列表会保留上下文，GPT 会基于所有历史信息回复。


## 三、进阶场景：图像生成（DALL-E）
调用 DALL-E 模型生成图像，核心是 `images.generate` 接口，支持自定义尺寸、数量。

### 1. 生成图像
```python
import openai

client = openai.OpenAI()

response = client.images.generate(
    model="dall-e-3",  # 模型（dall-e-3 效果更优，dall-e-2 更快）
    prompt="一只穿着宇航员服装的猫，在月球表面跳跃，背景是地球，卡通风格",  # 图像描述（越详细效果越好）
    n=1,  # 生成图像数量（1-4，dall-e-3 最多 1）
    size="1024x1024",  # 尺寸（dall-e-3 支持 1024x1024、1792x1024、1024x1792）
    quality="standard",  # 质量（standard/hd，hd 更清晰但耗时）
)

# 获取图像 URL（可直接下载或展示）
image_url = response.data[0].url
print("生成图像 URL：", image_url)

# 下载图像到本地
import requests
with open("cat_astronaut.jpg", "wb") as f:
    f.write(requests.get(image_url).content)
print("图像已保存为 cat_astronaut.jpg")
```

### 2. 图像编辑（基于现有图像修改）
需提供原始图像（Base64 编码）和蒙版（标注修改区域），示例：
```python
import openai
import base64

# 读取图像并转为 Base64
def image_to_base64(image_path):
    with open(image_path, "rb") as f:
        return base64.b64encode(f.read()).decode("utf-8")

response = client.images.edit(
    model="dall-e-2",
    image=image_to_base64("cat_astronaut.jpg"),  # 原始图像（Base64）
    mask=image_to_base64("mask.png"),  # 蒙版图像（白色区域为修改部分）
    prompt="将宇航员服装改为红色，背景添加星星",
    n=1,
    size="1024x1024"
)
```


## 四、进阶场景：语音转文字（Whisper）
Whisper 是 OpenAI 的语音识别模型，支持 99 种语言，可将音频转为文本，核心是 `audio.transcriptions.create` 接口。

### 1. 音频转文字（本地文件）
```python
import openai

client = openai.OpenAI()

# 支持格式：mp3、wav、m4a 等
response = client.audio.transcriptions.create(
    model="whisper-1",  # Whisper 仅需指定 whisper-1
    file=open("speech.mp3", "rb"),  # 本地音频文件
    language="zh",  # 语言（可选，自动检测时可不填，支持 "zh"、"en"、"ja" 等）
    response_format="text"  # 输出格式（text/json/vtt/srt，默认 json）
)

print("语音转文字结果：", response)
```

### 2. 音频翻译（外语转英文）
用 `audio.translations.create` 接口，自动将其他语言音频转为英文文本：
```python
response = client.audio.translations.create(
    model="whisper-1",
    file=open("chinese_speech.mp3", "rb")  # 中文音频
)
print("中文转英文结果：", response.text)
```


## 五、高级用法：函数调用（工具调用）
GPT 可通过函数调用连接外部工具（如查天气、算数据、调用 API），核心是让模型判断“是否需要调用函数”并生成符合格式的请求。

### 完整示例：调用天气工具（模拟）
```python
import openai
import json

# 1. 定义工具函数（模拟查询天气的 API）
def get_weather(city: str, date: str) -> str:
    """查询指定城市指定日期的天气"""
    # 实际场景中可替换为真实天气 API 调用
    weather_data = {
        "北京": {"2025-11-21": "晴，10-18℃", "2025-11-22": "多云，8-15℃"},
        "上海": {"2025-11-21": "阴，12-16℃", "2025-11-22": "小雨，10-14℃"}
    }
    return f"{city}{date}的天气：{weather_data.get(city, {}).get(date, '暂无数据')}"

# 2. 初始化客户端
client = openai.OpenAI()

# 3. 定义工具描述（告诉 GPT 有哪些工具可用）
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "查询指定城市和日期的天气信息",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {"type": "string", "description": "城市名称（如北京、上海）"},
                    "date": {"type": "string", "description": "日期（格式：YYYY-MM-DD，如 2025-11-21）"}
                },
                "required": ["city", "date"]  # 必填参数
            }
        }
    }
]

# 4. 发起对话（让 GPT 判断是否需要调用工具）
messages = [{"role": "user", "content": "查询北京2025年11月21日的天气"}]

response = client.chat.completions.create(
    model="gpt-3.5-turbo",
    messages=messages,
    tools=tools,
    tool_choice="auto"  # 自动判断是否调用工具（也可指定 "none" 不调用，"function:get_weather" 强制调用）
)

# 5. 解析响应：若需要调用工具，则执行工具并返回结果
response_message = response.choices[0].message
if response_message.tool_calls:
    # 提取工具调用参数
    tool_call = response_message.tool_calls[0]
    function_name = tool_call.function.name
    function_args = json.loads(tool_call.function.arguments)
    
    # 执行工具函数
    if function_name == "get_weather":
        weather_result = get_weather(**function_args)
    
    # 将工具结果加入历史对话，让 GPT 整理成自然语言回复
    messages.append(response_message)
    messages.append({
        "role": "tool",
        "tool_call_id": tool_call.id,
        "name": function_name,
        "content": weather_result
    })
    
    # 二次请求：让 GPT 基于工具结果生成回复
    final_response = client.chat.completions.create(
        model="gpt-3.5-turbo",
        messages=messages
    )
    print("最终回复：", final_response.choices[0].message.content)
```

**输出示例**：
```
最终回复：北京2025年11月21日的天气：晴，10-18℃
```


## 六、高级用法：流式响应（实时返回结果）
默认情况下，GPT 会生成完整结果后一次性返回，流式响应可实现“打字机”效果（逐字/逐句返回），提升用户体验。

### 流式文本生成示例
```python
import openai

client = openai.OpenAI()

print("GPT 流式回复：")
stream = client.chat.completions.create(
    model="gpt-3.5-turbo",
    messages=[{"role": "user", "content": "简述 Python 的优势，分点说明"}],
    stream=True,  # 开启流式响应
    temperature=0.6
)

# 迭代接收流式数据
for chunk in stream:
    # 提取当前片段的内容（忽略空片段）
    content = chunk.choices[0].delta.content
    if content:
        print(content, end="", flush=True)  # end="" 不换行，flush=True 强制刷新输出
```

**效果**：文字会逐字打印在终端，模拟实时对话体验。


## 七、常见问题与避坑指南
### 1. 网络问题（国内无法直接访问）
- 问题：国内网络直接调用 OpenAI API 会超时（`ConnectionError`）；
- 解决方案：
  1. 使用合规代理服务，在初始化客户端时指定 `base_url`：
     ```python
     client = openai.OpenAI(
         api_key="sk-你的密钥",
         base_url="https://合法代理地址/v1"  # 替换为有效代理（需支持 OpenAI API 转发）
     )
     ```
  2. 确保代理支持 HTTPS，且未被墙。

### 2. API 密钥错误/额度不足
- 错误提示：`InvalidApiKeyError`（密钥错误）、`InsufficientQuotaError`（额度不足）；
- 解决方案：
  1. 检查密钥是否正确（无多余空格、大小写一致）；
  2. 登录 OpenAI 官网，查看「Billing」页面，确认是否有可用额度（新账号通常有免费额度，过期后需充值）。

### 3. 模型不支持/参数错误
- 错误提示：`ModelNotFoundError`（模型不存在）、`InvalidRequestError`（参数错误）；
- 解决方案：
  1. 确认模型名称正确（如 `gpt-3.5-turbo`、`dall-e-3`、`whisper-1`）；
  2. 检查参数是否符合要求（如 `n` 最大值、`size` 支持的尺寸、`max_tokens` 不超过模型限制）。

### 4. 流式响应解析错误
- 问题：流式响应的 `chunk` 可能不含 `content`（初始片段为空）；
- 解决方案：添加判断 `if content:` 再处理，避免 `NoneType` 错误（见上文流式示例）。

### 5. 音频转文字时文件过大
- 问题：Whisper 支持的单个文件最大 25MB；
- 解决方案：分割音频文件（如用 `pydub` 库拆分），或使用更大容量的第三方 Whisper 部署版本。


## 八、核心参数速查表（文本生成）
| 参数名          | 功能描述                                  | 可选值/默认值                          |
|-----------------|-------------------------------------------|---------------------------------------|
| `model`         | 模型名称（核心参数）                      | gpt-3.5-turbo（经济）、gpt-4o（最优）、gpt-4-turbo |
| `messages`      | 对话历史（角色+内容）                     | 列表，元素为 `{"role": "", "content": ""}` |
| `temperature`   | 随机性（越低越确定）                      | 0-2，默认 0.7                         |
| `max_tokens`    | 最大生成 tokens 数（含输入+输出）         | 默认 1024，gpt-4o 最大支持 128000     |
| `top_p`         | 核采样（替代 temperature，0-1）            | 默认 1.0（不限制）                    |
| `n`             | 生成回复数量                              | 1-4，默认 1                           |
| `stream`        | 是否流式响应                              | True/False，默认 False                |
| `stop`          | 停止符（遇到该字符串停止生成）            | 字符串或列表，默认 None               |


## 九、总结
Python OpenAI 库的核心用法可总结为：
1. **基础流程**：安装库 → 配置 API 密钥 → 初始化客户端 → 调用对应接口（文本/图像/语音）；
2. **核心场景**：
   - 文本生成：`chat.completions.create`（GPT 模型）；
   - 图像生成：`images.generate`（DALL-E 模型）；
   - 语音转文字：`audio.transcriptions.create`（Whisper 模型）；
3. **高级功能**：函数调用（连接外部工具）、流式响应（实时交互）；
4. **避坑关键**：网络代理配置、密钥安全（不硬编码）、参数符合模型要求。

如需更复杂的场景（如批量处理、异步请求），可参考 [OpenAI 官方文档](https://platform.openai.com/docs/libraries/python-library)。
