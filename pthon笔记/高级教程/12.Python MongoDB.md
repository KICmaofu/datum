## 一、MongoDB 核心概念与应用场景
### 1. 什么是 MongoDB？
MongoDB 是一款 **文档型 NoSQL 数据库**（非关系型数据库），核心特点是存储“文档”（类似 JSON 格式的 BSON 数据），无需预先定义表结构，灵活性极高。与传统关系型数据库（如 MySQL）相比，更适合存储非结构化/半结构化数据。

### 2. MongoDB 与 MySQL 核心区别（英语学习场景适配）
| 特性         | MongoDB（文档型）                | MySQL（关系型）                  | 英语学习场景适配                  |
|--------------|----------------------------------|----------------------------------|-----------------------------------|
| 数据结构     | 文档（BSON，类似 JSON）          | 表+行+列（固定结构）             | 单词/学习记录字段灵活（可新增例句、词根） |
| Schema 约束  | 无（动态字段，无需预定义）        | 有（必须先建表结构）             | 快速调整数据结构（如新增“词根”字段）    |
| 查询方式     | 灵活的文档查询、聚合管道          | SQL 语句查询                     | 复杂条件筛选（如“难度 medium 且掌握度<0.6”） |
| 适用数据     | 非结构化/半结构化数据（如单词、日志） | 结构化数据（如用户信息、成绩）    | 存储单词、句子、学习记录、语料库        |

### 3. MongoDB 核心概念（对应 MySQL）
| MongoDB 概念 | MySQL 概念 | 说明（英语学习场景）                |
|--------------|------------|-------------------------------------|
| 数据库（Database） | 数据库     | 存储英语学习相关数据（如 `english_db`） |
| 集合（Collection） | 表         | 存储同一类数据（如 `vocabulary` 单词表、`study_records` 学习记录） |
| 文档（Document）  | 行         | 单条数据（如一个单词的完整信息、一条学习记录） |
| 字段（Field）     | 列         | 数据的属性（如单词的 `word`、`definition`、`difficulty`） |
| BSON          | JSON       | MongoDB 存储格式（支持更多数据类型，如日期、整数） |

### 4. 英语学习场景核心应用
- 存储单词库（支持灵活扩展字段：单词、释义、例句、词根、难度、掌握度、复习次数）。
- 记录学习日志（每日学习时长、单词记忆记录、听力练习结果）。
- 存储英语语料（新闻、短文、对话，支持嵌套结构如“文章→段落→句子”）。
- 统计学习数据（不同难度单词掌握情况、每周学习时长趋势）。
- 开发个性化学习工具（如单词复习系统、学习进度跟踪器）。

## 三、核心操作：Python 操作 MongoDB（CRUD）
### 基础流程：连接数据库 → 操作集合 → 处理文档
所有操作的前提是通过 `pymongo.MongoClient` 建立连接，后续 CRUD 操作围绕“集合”和“文档”展开。

### 1. 连接 MongoDB 数据库
```python
from pymongo import MongoClient
from pymongo.errors import ConnectionError

def connect_mongodb():
    """连接 MongoDB 数据库，返回数据库对象"""
    try:
        # 1. 连接本地 MongoDB（默认端口 27017，无密码）
        client = MongoClient("mongodb://localhost:27017/")
        
        # 2. 验证连接（可选）
        client.admin.command("ping")  # 成功则无异常
        print("✅ MongoDB 连接成功！")
        
        # 3. 选择/创建数据库（不存在则插入数据时自动创建）
        db = client["english_db"]  # 数据库名：english_db
        return db
    
    except ConnectionError as e:
        print(f"❌ MongoDB 连接失败：{e}")
        return None

# 初始化数据库连接
db = connect_mongodb()
if not db:
    exit()  # 连接失败则退出
```

### 2. 增（Create）：插入文档（单词/学习记录）
MongoDB 插入数据时无需预先创建集合，插入文档后集合自动生成。支持单文档插入和批量插入。

#### 示例 1：插入单个单词
```python
def insert_single_word(db):
    """插入单个单词到 vocabulary 集合"""
    # 集合对象（类似 MySQL 的表）
    vocab_collection = db["vocabulary"]
    
    # 单词文档（BSON 格式，类似字典）
    word_doc = {
        "word": "perseverance",
        "definition": "坚持不懈，毅力",
        "part_of_speech": "n.",
        "example": "Success requires perseverance and hard work.",
        "difficulty": "medium",  # easy/medium/hard
        "mastery": 0.7,          # 掌握度 0-1
        "review_count": 3,       # 复习次数
        "last_review": "2025-11-18"  # 最后复习日期
    }
    
    # 插入文档（返回 InsertOneResult 对象，包含插入 ID）
    result = vocab_collection.insert_one(word_doc)
    print(f"✅ 插入单个单词成功！文档 ID：{result.inserted_id}")

# 执行插入
insert_single_word(db)
```

### 3. 查（Read）：查询文档（灵活筛选单词/学习记录）
MongoDB 提供强大的查询语法，支持按条件筛选、模糊查询、分页、排序等，核心方法是 `find()`。

#### 示例 1：查询所有单词（无条件）
```python
def find_all_words(db):
    """查询 vocabulary 集合中所有单词"""
    vocab_collection = db["vocabulary"]
    
    # find() 无参数 → 查询所有文档，返回游标（迭代器）
    all_words = vocab_collection.find()
    
    print("📚 所有单词列表：")
    for idx, word in enumerate(all_words, 1):
        print(f"\n{idx}. 单词：{word['word']} ({word['part_of_speech']})")
        print(f"   释义：{word['definition']}")
        print(f"   难度：{word['difficulty']} | 掌握度：{word['mastery']}")
        print(f"   例句：{word['example']}")

# 执行查询
find_all_words(db)
```

#### 示例 2：按条件查询（筛选需复习的单词）
```python
def find_need_review_words(db):
    """查询：难度 medium + 掌握度 < 0.7 的单词（需复习）"""
    vocab_collection = db["vocabulary"]
    
    # 查询条件（类似字典，支持比较运算符 $lt/$gt/$eq 等）
    query = {
        "difficulty": "medium",  # 等于 medium
        "mastery": {"$lt": 0.7}  # 小于 0.7（$lt=less than）
    }
    
    # 执行查询（find(query)）
    need_review = vocab_collection.find(query)
    
    print("\n🔄 需复习的单词（难度 medium + 掌握度 < 0.7）：")
    for word in need_review:
        print(f"- {word['word']} | 掌握度：{word['mastery']} | 最后复习：{word['last_review']}")
```

#### 示例 3：模糊查询（单词包含指定字母）
```python
def find_word_by_fuzzy(db, keyword):
    """模糊查询：单词包含 keyword（不区分大小写）"""
    vocab_collection = db["vocabulary"]
    
    # 模糊查询语法：$regex（正则表达式），$options: "i" 表示不区分大小写
    query = {"word": {"$regex": keyword, "$options": "i"}}
    
    results = vocab_collection.find(query)
    
    print(f"\n🔤 包含字母「{keyword}」的单词：")
    for word in results:
        print(f"- {word['word']} → {word['definition']}")

# 查询包含 "flu" 的单词
find_word_by_fuzzy(db, "flu")
```

```python
def find_words_with_sort_and_limit(db):
    """查询：按掌握度升序排序，分页显示（第 1 页，每页 2 条）"""
    vocab_collection = db["vocabulary"]

        .limit(2)            # 每页显示 2 条
    
    print("\n📑 按掌握度升序分页查询（第 1 页）：")
    for word in results:
        print(f"- {word['word']} | 掌握度：{word['mastery']} | 难度：{word['difficulty']}")
```

### 4. 改（Update）：更新文档（修改单词掌握度/学习记录）
MongoDB 支持灵活的更新操作，可修改单个字段、多个字段，或批量更新符合条件的文档，核心方法是 `update_one()` 和 `update_many()`。

#### 示例 1：更新单个字段（单词掌握度）
```python
def update_word_mastery(db, word, new_mastery):
    """更新单个单词的掌握度"""
    vocab_collection = db["vocabulary"]
    
    # 查询条件：找到对应的单词
    query = {"word": word}
    # 更新操作：$set 表示设置字段值（仅修改指定字段，不影响其他字段）
    update = {"$set": {"mastery": new_mastery, "last_review": "2025-11-20"}}
    
    # update_one() → 仅更新第一个匹配的文档
    result = vocab_collection.update_one(query, update)
    
    if result.modified_count > 0:
        print(f"✅ 单词「{word}」的掌握度已更新为 {new_mastery}")
    else:
        print(f"❌ 未找到单词「{word}」或无需更新")

# 将 "proficiency" 的掌握度更新为 0.5
update_word_mastery(db, "proficiency", 0.5)
```

#### 示例 2：更新多个字段（单词难度+复习次数）
```python
def update_word_multiple_fields(db, word):
    """更新单词的多个字段：难度+复习次数+最后复习日期"""
    vocab_collection = db["vocabulary"]
    
    query = {"word": word}
    update = {
        "$set": {"difficulty": "advanced", "last_review": "2025-11-20"},
        "$inc": {"review_count": 1}  # $inc：字段值自增 1（复习次数+1）
    }
    
    result = vocab_collection.update_one(query, update)
    if result.modified_count > 0:
        print(f"✅ 单词「{word}」的多个字段已更新")
```

#### 示例 3：批量更新（所有 easy 难度单词的掌握度+0.1）

## 四、进阶操作：索引与聚合查询（统计学习数据）
### 1. 索引：提升查询效率
MongoDB 支持给字段建立索引，类似书籍目录，可大幅提升查询速度（尤其数据量较大时）。常用场景：给 `word`（单词名）、`difficulty`（难度）等高频查询字段建索引。

```python
def create_index(db):
    """给 vocabulary 集合的 word 字段建立唯一索引，difficulty 字段建立普通索引"""
    vocab_collection = db["vocabulary"]
    
    # 1. 给 word 建立唯一索引（避免插入重复单词）
    vocab_collection.create_index("word", unique=True)
    print("✅ 已给 word 字段建立唯一索引")
    
    # 2. 给 difficulty 建立普通索引（提升按难度查询的效率）
    vocab_collection.create_index("difficulty")
    print("✅ 已给 difficulty 字段建立普通索引")
    
    # 查看集合的所有索引
    indexes = vocab_collection.list_indexes()
    print("\n集合索引列表：")
    for idx in indexes:
        print(idx)

# 执行创建索引
create_index(db)
```

> 💡 注意：唯一索引会阻止插入重复值（如重复单词），若插入重复数据会报错，需捕获异常处理。

### 2. 聚合查询：统计学习数据（核心实用）
MongoDB 的 **聚合管道（Aggregation Pipeline）** 是强大的统计工具，支持分组、筛选、计算总和/平均值等，适合统计英语学习数据（如不同难度单词数量、平均掌握度、学习时长总和）。

#### 示例 1：统计不同难度的单词数量
```python
def aggregate_word_count_by_difficulty(db):
    """统计：不同难度的单词数量"""
    vocab_collection = db["vocabulary"]
    
    # 聚合管道：$group 分组统计
    pipeline = [
        {
            "$group": {
                "_id": "$difficulty",  # 按 difficulty 分组（_id 为分组字段）
                "count": {"$sum": 1}   # 统计每组数量（$sum:1 表示计数）
            }
        }
    ]
    
    results = vocab_collection.aggregate(pipeline)
    print("\n📊 不同难度的单词数量统计：")
    for res in results:
        print(f"难度 {res['_id']}：{res['count']} 个单词")
```

#### 示例 2：统计不同难度单词的平均掌握度
```python
def aggregate_avg_mastery_by_difficulty(db):
    """统计：不同难度单词的平均掌握度"""
    vocab_collection = db["vocabulary"]
    
    pipeline = [
        {
            "$group": {
                "_id": "$difficulty",
                "avg_mastery": {"$avg": "$mastery"}  # $avg 计算平均值
            }
        },
        {
            "$sort": {"avg_mastery": -1}  # 按平均掌握度降序排序
        }
    ]
    
    results = vocab_collection.aggregate(pipeline)
    print("\n📊 不同难度单词的平均掌握度：")
    for res in results:
        print(f"难度 {res['_id']}：{res['avg_mastery']:.2f}")
```

#### 示例 3：统计每周学习总时长
```python
def aggregate_weekly_study_duration(db, year=2025, week=47):
    """统计：指定年份和周数的学习总时长"""
    record_collection = db["study_records"]
    
    # 假设日期格式为 "YYYY-MM-DD"，需先转换为日期类型再分组（简化版：按日期前缀筛选）
    pipeline = [
        {
            "$match": {
                "date": {"$regex": f"^{year}-"}  # 筛选指定年份的记录
            }
        },
        {
            "$group": {
                "_id": {"$substr": ["$date", 0, 7]},  # 按月份分组（截取 YYYY-MM）
                "total_duration": {"$sum": "$total_duration"}  # 总时长求和
            }
        },
        {
            "$sort": {"_id": 1}  # 按月份升序
        }
    ]
    
    results = record_collection.aggregate(pipeline)
    print(f"\n📅 {year} 年各月学习总时长：")
    for res in results:
        print(f"{res['_id']}：{res['total_duration']:.1f} 小时")
```

```python
"""
功能：实现单词的添加、查询、更新、删除、导出复习列表
用到的 MongoDB 操作：insert_one、find、find_one、update_one、delete_one、aggregate
"""

class EnglishWordManager:
    def __init__(self, db):
        self.collection = db["vocabulary"]
    
    def add_word(self, word_doc):
        """添加单词（自动去重，基于 word 唯一索引）"""
        try:
            result = self.collection.insert_one(word_doc)
            print(f"✅ 添加单词「{word_doc['word']}」成功！ID：{result.inserted_id}")
        except Exception as e:
            if "duplicate key" in str(e).lower():
                print(f"❌ 单词「{word_doc['word']}」已存在，无需重复添加")
            else:
                print(f"❌ 添加失败：{e}")
    
    def search_word(self, keyword=None, difficulty=None, mastery_lt=None):
        """多条件查询单词"""
        query = {}
        if keyword:
            query["word"] = {"$regex": keyword, "$options": "i"}
        if difficulty:
            query["difficulty"] = difficulty
        if mastery_lt:
            query["mastery"] = {"$lt": mastery_lt}
        
        results = self.collection.find(query).sort("mastery", 1)
        return list(results)  # 转为列表返回
    
    def update_word(self, word, update_doc):
        """更新单词字段"""
        query = {"word": word}
        update = {"$set": update_doc}
        result = self.collection.update_one(query, update)
        if result.modified_count > 0:
            print(f"✅ 单词「{word}」更新成功")
        else:
            print(f"❌ 单词「{word}」未找到或无需更新")
    
    def delete_word(self, word):
        """删除单词"""
        query = {"word": word}
        result = self.collection.delete_one(query)
        if result.deleted_count > 0:
            print(f"✅ 单词「{word}」删除成功")
        else:
            print(f"❌ 单词「{word}」未找到")
    
    def export_review_list(self, filename="review_list.txt"):
        """导出需复习的单词（掌握度 < 0.7）到文本文件"""
        review_words = self.search_word(mastery_lt=0.7)
        with open(filename, "w", encoding="utf-8") as f:
            f.write("📝 英语单词复习列表（掌握度 < 0.7）\n")
            f.write("="*50 + "\n")
            for idx, word in enumerate(review_words, 1):
                f.write(f"\n{idx}. 单词：{word['word']} ({word['part_of_speech']})\n")
                f.write(f"   释义：{word['definition']}\n")
                f.write(f"   例句：{word['example']}\n")
                f.write(f"   掌握度：{word['mastery']} | 难度：{word['difficulty']}\n")
        print(f"✅ 复习列表已导出到 {filename}")

# 测试单词管理系统
if __name__ == "__main__":
    # 初始化管理器
    word_manager = EnglishWordManager(db)
    
    # 1. 添加单词
    new_word = {

        "definition": "勤奋的，勤勉的",
        "part_of_speech": "adj.",
        "example": "He is a diligent English learner.",
        "difficulty": "medium",
        "mastery": 0.5,
        "review_count": 1,
        "last_review": "2025-11-20"
    }
    word_manager.add_word(new_word)
    
    # 2. 多条件查询（难度 medium + 掌握度 < 0.7）
    print("\n🔍 多条件查询结果：")
    results = word_manager.search_word(difficulty="medium", mastery_lt=0.7)
```

```python
"""
功能：统计每日/每周/每月学习时长，生成学习报告
用到的 MongoDB 操作：insert_one、aggregate
"""

class StudyProgressAnalyzer:
    def __init__(self, db):
        self.collection = db["study_records"]
    
    def add_study_record(self, record_doc):
        """添加学习记录"""
        result = self.collection.insert_one(record_doc)
        print(f"✅ 学习记录添加成功！ID：{result.inserted_id}")
    
    def get_weekly_total_duration(self, user, year=2025, week=47):
        """统计用户指定周的学习总时长"""
        # 简化版：按日期前缀筛选（实际可通过日期函数精确计算周数）
        pipeline = [
            {"$match": {"user": user, "date": {"$regex": f"^{year}-11-"}}},  # 11月数据
            {"$group": {"_id": None, "total": {"$sum": "$total_duration"}}},
        ]
        result = list(self.collection.aggregate(pipeline))
        return result[0]["total"] if result else 0.0
    
    def generate_monthly_report(self, user, year=2025, month=11):
        """生成月度学习报告"""
        date_prefix = f"{year}-{month:02d}"
        pipeline = [
            {"$match": {"user": user, "date": {"$regex": f"^{date_prefix}"}}},
            {"$group": {
                "_id": "$date",
                "daily_duration": {"$sum": "$total_duration"},
                "task_count": {"$sum": {"$size": "$tasks"}}  # 统计每日任务数
            }},
            {"$sort": {"_id": 1}}
        ]
        
        daily_records = list(self.collection.aggregate(pipeline))
        total_duration = sum([r["daily_duration"] for r in daily_records])
        avg_duration = total_duration / len(daily_records) if daily_records else 0.0
        
        # 生成报告
        report = f"""
📊 {year}年{month}月学习报告（用户：{user}）
{'-'*50}
总学习时长：{total_duration:.1f} 小时
平均每日时长：{avg_duration:.1f} 小时
学习天数：{len(daily_records)} 天

每日学习详情：
"""
        for record in daily_records:
            report += f"  {record['_id']}：{record['daily_duration']:.1f} 小时 | {record['task_count']} 个任务\n"
        
        return report

# 测试学习进度统计系统
```

## 六、常见问题与解决方案
### 1. 连接 MongoDB 失败
- **问题**：`pymongo.errors.ConnectionError: Could not connect to server`。
- **原因**：
  - MongoDB 服务未启动（需手动启动服务）。
  - 端口被占用（默认端口 27017，可修改配置文件更换端口）。
  - 远程连接时未开放端口或权限不足。
- **解决方案**：
  - 启动 MongoDB 服务（`net start MongoDB` 或 `mongod --dbpath 路径`）。
  - 检查端口是否被占用（`netstat -ano | findstr 27017`），关闭占用进程。
  - 远程连接时添加认证（`MongoClient("mongodb://用户名:密码@IP:端口/数据库")`）。

### 2. 插入重复单词报错
- **问题**：`pymongo.errors.DuplicateKeyError: E11000 duplicate key error`。
- **原因**：给 `word` 字段建立了唯一索引，插入重复单词触发约束。
- **解决方案**：
  - 插入前先查询是否存在，避免重复插入。
  - 捕获 `DuplicateKeyError` 异常，提示用户单词已存在。

### 3. 中文乱码问题
- **问题**：查询结果中中文显示为乱码（如 `b'\xe5\x8a\xa9\xe6\x89\x8b'`）。
- **原因**：MongoDB 存储的 BSON 数据在 Python 中可能以字节形式返回，或编码不匹配。
- **解决方案**：
  - 确保 Python 代码文件编码为 UTF-8。
  - 读取数据时，对中文字段进行解码（如 `word['definition'].decode('utf-8')`，Python3 通常自动处理）。
  - 插入中文时，直接使用字符串（Python3 默认 UTF-8 编码）。

### 4. 索引失效
- **问题**：建立索引后，查询速度未提升。
- **原因**：
  - 查询条件未匹配索引字段（如给 `word` 建索引，却按 `example` 字段查询）。
  - 模糊查询使用 `$regex` 且匹配开头（如 `^flu`）才会走索引，中间/结尾匹配（如 `flu$`）可能失效。
- **解决方案**：
  - 给高频查询字段建立索引。
  - 优化查询条件，尽量匹配索引字段。

### 5. 聚合查询结果为空
- **问题**：聚合管道执行后无返回结果。
- **原因**：
  - `$match` 条件过于严格，无匹配数据。
  - 字段名拼写错误（如 `total_duration` 误写为 `total_time`）。
- **解决方案**：
  - 简化 `$match` 条件，逐步排查数据是否存在。
  - 检查字段名与文档中的实际字段一致（MongoDB 字段区分大小写）。
