# Python datetime 模块完全指南：日期时间处理实战
datetime 是 Python 内置的 **日期时间处理标准库**，核心功能是提供日期、时间的创建、转换、计算、格式化能力，支持精确到微秒的时间操作，涵盖时区处理、时间差计算、字符串与日期互转等核心场景，是处理时间相关逻辑的首选工具。

与 `time` 模块（偏底层，仅支持时间戳和基础时间操作）相比，datetime 模块提供更面向对象的 API，功能更全面（如日期时间组合、时区支持）；与第三方库（如 `arrow`、`pendulum`）相比，它无需额外安装，原生集成 Python 生态，稳定性强，适合大多数开发场景。


## 一、核心定位与优势
### 1. 核心特点
- **内置无依赖**：Python 标准库自带，无需额外安装，导入即使用；
- **面向对象设计**：通过 `date`、`time`、`datetime` 等类封装日期时间，API 直观易用；
- **功能全面**：支持日期时间创建、格式化、解析、比较、加减计算、时区转换；
- **高精度**：支持微秒级时间精度（`datetime` 类包含 `microsecond` 属性）；
- **时区兼容**：Python 3.9+ 内置 `zoneinfo` 模块，原生支持时区处理；旧版本可配合 `pytz` 库使用；
- **跨平台兼容**：自动适配 Windows、Linux、macOS 等系统的时间格式差异。

### 2. 适用场景
- **日期时间计算**：如计算两个日期的天数差、获取未来/过去的时间、统计程序运行耗时；
- **日志/数据格式化**：将日期时间转为指定字符串格式（如日志中的时间戳、数据导出的时间字段）；
- **字符串解析**：将用户输入、配置文件、数据库中的时间字符串转为日期时间对象；
- **时区转换**：处理跨时区数据（如 UTC 时间转北京时间、国际业务中的时区适配）；
- **定时任务**：判断当前时间是否满足任务执行条件、计算任务下次执行时间；
- **数据筛选**：按日期范围筛选数据（如查询近7天的日志、本月的订单）。

### 3. 与相似工具的对比
| 工具/模块                | 核心优势                          | 缺点                                  | 适用场景                  |
|---------------------------|-----------------------------------|---------------------------------------|---------------------------|
| `datetime` 模块（本文核心） | 内置无依赖、功能全面、面向对象、高精度 | 时区处理在 3.9 前需依赖第三方库        | 大多数日期时间处理场景    |
| `time` 模块               | 底层高效、支持时间戳、系统调用      | 功能单一（无日期时间组合、时区支持弱） | 简单时间戳操作、系统级时间控制 |
| `arrow` 库（第三方）      | API 简洁、原生时区支持、人性化设计  | 需额外安装（`pip install arrow`）      | 追求开发效率的复杂时间场景 |
| `pytz` 库（第三方）       | 完善的时区数据库、兼容旧 Python 版本 | 仅处理时区，需配合 datetime 使用       | Python 3.9 前的时区转换    |
| `zoneinfo` 模块（3.9+）   | 原生时区支持、无需第三方依赖        | 仅支持 Python 3.9+                    | Python 3.9+ 的时区处理    |

**结论**：优先使用 `datetime` 模块满足基础需求；需要简洁 API 或兼容旧版本时区时，再考虑 `arrow` 或 `pytz`。


## 二、环境准备
datetime 是 Python 内置模块，**无需额外安装**，直接导入核心类即可：
```python
# 导入核心类
from datetime import date, time, datetime, timedelta
# Python 3.9+ 导入时区支持（3.9 前需用 pytz）
from zoneinfo import ZoneInfo  # 原生时区
import pytz  # 兼容旧版本（需安装：pip install pytz）
```

### 版本说明
- Python 2/3 均支持 datetime 核心功能，但 Python 3 修复了部分兼容性问题（如时区处理）；
- Python 3.7+ 支持 `datetime.fromisoformat()`（解析 ISO 格式字符串，如 `2025-11-20`）；
- Python 3.9+ 新增 `zoneinfo` 模块，原生支持时区，无需依赖 `pytz`。

验证模块可用性：
```python
# 创建当前日期时间对象
now = datetime.now()
print("当前日期时间：", now)  # 输出：2025-11-20 16:45:30.123456
print("当前日期：", date.today())  # 输出：2025-11-20
print("当前时间：", now.time())  # 输出：16:45:30.123456
```


## 三、核心概念：理解 datetime 模块的核心组件
datetime 模块通过 5 个核心类封装日期时间相关功能，各司其职且可灵活组合：

### 1. 核心类说明
| 类名                  | 中文名称          | 核心作用                                  | 关键属性/特点                          |
|-----------------------|-------------------|-------------------------------------------|---------------------------------------|
| `datetime.date`       | 日期类            | 仅处理年、月、日（无时间信息）            | 属性：`year`、`month`、`day`；支持日期比较、加减 |
| `datetime.time`       | 时间类            | 仅处理时、分、秒、微秒（无日期信息）      | 属性：`hour`、`minute`、`second`、`microsecond`；支持时间比较 |
| `datetime.datetime`   | 日期时间类        | 组合日期和时间（最常用类）                | 继承 `date` 和 `time` 的属性；支持所有核心操作 |
| `datetime.timedelta`  | 时间差类          | 表示两个日期/时间的差值（如 2 天、3 小时） | 支持加减运算；仅支持天、时、分、秒、微秒（无月份/年份） |
| `datetime.tzinfo`     | 时区基类          | 时区处理的抽象基类（需自定义实现）        | Python 3.9+ 推荐用 `zoneinfo.ZoneInfo` |
| `zoneinfo.ZoneInfo`   | 时区实现类        | 原生时区支持（Python 3.9+）               | 支持 IANA 时区（如 `Asia/Shanghai`、`UTC`） |

### 2. 关键概念
- **Naive DateTime**：未指定时区的日期时间对象（默认创建的对象均为 naive），无法直接进行时区转换；
- **Aware DateTime**：包含时区信息的日期时间对象（需通过 `tzinfo` 或 `zoneinfo` 指定），支持跨时区计算；
- **时间戳（Timestamp）**：从 Unix 纪元（1970-01-01 00:00:00 UTC）到当前时间的秒数（支持小数，精确到微秒）；
- **格式化字符串**：通过 `strftime()` 将日期时间转为字符串（输出），通过 `strptime()` 将字符串解析为日期时间（输入），核心依赖格式化指令（如 `%Y` 表示4位年份）。


## 四、核心功能：分场景详解（语法+示例）
### 1. 创建日期时间对象
#### （1）创建当前日期/时间/日期时间
```python
from datetime import date, time, datetime

# 1. 创建当前日期（date 类）
current_date = date.today()
print("当前日期：", current_date)  # 输出：2025-11-20
print("年：", current_date.year, "月：", current_date.month, "日：", current_date.day)

# 2. 创建当前时间（datetime 类的 time() 方法）
current_time = datetime.now().time()
print("当前时间：", current_time)  # 输出：16:50:20.123456
print("时：", current_time.hour, "分：", current_time.minute, "秒：", current_time.second)

# 3. 创建当前日期时间（最常用）
# 方式 1：本地时间
local_now = datetime.now()
print("本地当前日期时间：", local_now)  # 输出：2025-11-20 16:50:20.123456

# 方式 2：UTC 时间（Python 3.3+）
utc_now = datetime.utcnow()
print("UTC 当前日期时间：", utc_now)  # 输出：2025-11-20 08:50:20.123456
```

#### （2）创建指定日期/时间/日期时间
```python
from datetime import date, time, datetime

# 1. 指定日期（年, 月, 日）
specific_date = date(2025, 12, 31)
print("指定日期：", specific_date)  # 输出：2025-12-31

# 2. 指定时间（时, 分, 秒, 微秒）
specific_time = time(23, 59, 59, 999999)
print("指定时间：", specific_time)  # 输出：23:59:59.999999

# 3. 指定日期时间（年, 月, 日, 时, 分, 秒, 微秒）
specific_datetime = datetime(2025, 12, 31, 23, 59, 59)
print("指定日期时间：", specific_datetime)  # 输出：2025-12-31 23:59:59
```

#### （3）从时间戳创建日期时间
```python
from datetime import datetime

# 时间戳（秒级，可含小数表示微秒）
timestamp = 1755676800  # 对应 2025-11-20 00:00:00 UTC
timestamp_ms = 1755676800.123456  # 含微秒

# 1. 时间戳转本地日期时间
local_dt = datetime.fromtimestamp(timestamp)
print("时间戳转本地时间：", local_dt)  # 输出：2025-11-20 08:00:00（取决于本地时区）

# 2. 时间戳转 UTC 日期时间
utc_dt = datetime.utcfromtimestamp(timestamp_ms)
print("时间戳转 UTC 时间：", utc_dt)  # 输出：2025-11-20 00:00:00.123456
```

### 2. 日期时间格式化（转为字符串）
使用 `strftime(format)` 方法，通过格式化指令将日期时间对象转为指定格式的字符串（常用指令见下表）：

| 格式化指令 | 描述                  | 示例          |
|------------|-----------------------|---------------|
| `%Y`       | 4位年份               | 2025          |
| `%y`       | 2位年份               | 25            |
| `%m`       | 2位月份（01-12）      | 11            |
| `%d`       | 2位日期（01-31）      | 20            |
| `%H`       | 24小时制（00-23）     | 16            |
| `%I`       | 12小时制（01-12）     | 04            |
| `%M`       | 2位分钟（00-59）      | 55            |
| `%S`       | 2位秒数（00-59）      | 30            |
| `%f`       | 6位微秒（000000-999999）| 123456      |
| `%w`       | 星期（0-6，0为周日）  | 4             |
| `%A`       | 完整星期名称          | Thursday      |
| `%a`       | 缩写星期名称          | Thu           |
| `%B`       | 完整月份名称          | November      |
| `%b`       | 缩写月份名称          | Nov           |
| `%x`       | 本地日期格式          | 11/20/25      |
| `%X`       | 本地时间格式          | 16:55:30      |

#### 示例：格式化日期时间
```python
from datetime import datetime

now = datetime.now()

# 1. 常用格式（日志时间戳）
log_format = now.strftime("%Y-%m-%d %H:%M:%S.%f")
print("日志格式：", log_format)  # 输出：2025-11-20 16:55:30.123456

# 2. 中文格式
chinese_format = now.strftime("%Y年%m月%d日 %H时%M分%S秒")
print("中文格式：", chinese_format)  # 输出：2025年11月20日 16时55分30秒

# 3. 缩写格式
short_format = now.strftime("%Y-%m-%d %a %H:%M")
print("缩写格式：", short_format)  # 输出：2025-11-20 Thu 16:55

# 4. ISO 8601 格式（国际标准）
iso_format = now.isoformat()  # 等价于 strftime("%Y-%m-%dT%H:%M:%S.%f")
print("ISO 格式：", iso_format)  # 输出：2025-11-20T16:55:30.123456
```

### 3. 字符串解析为日期时间
使用 `datetime.strptime(date_string, format)` 方法，按指定格式将字符串解析为 `datetime` 对象（格式指令与 `strftime` 一致）：

#### 示例：解析字符串
```python
from datetime import datetime

# 1. 解析标准格式字符串
date_str1 = "2025-11-20 17:00:00"
dt1 = datetime.strptime(date_str1, "%Y-%m-%d %H:%M:%S")
print("解析结果1：", dt1, type(dt1))  # 输出：2025-11-20 17:00:00 <class 'datetime.datetime'>

# 2. 解析中文格式字符串
date_str2 = "2025年12月31日 23:59"
dt2 = datetime.strptime(date_str2, "%Y年%m月%d日 %H:%M")
print("解析结果2：", dt2)  # 输出：2025-12-31 23:59:00

# 3. 解析 ISO 格式字符串（Python 3.7+ 支持）
date_str3 = "2025-11-20T17:00:00.123456"
dt3 = datetime.fromisoformat(date_str3)  # 无需指定格式，直接解析
print("解析结果3：", dt3)  # 输出：2025-11-20 17:00:00.123456

# 4. 解析带时区的字符串（Python 3.9+）
from zoneinfo import ZoneInfo
date_str4 = "2025-11-20 08:00:00 UTC"
dt4 = datetime.strptime(date_str4, "%Y-%m-%d %H:%M:%S %Z").replace(tzinfo=ZoneInfo("UTC"))
print("带时区解析结果：", dt4)  # 输出：2025-11-20 08:00:00+00:00
```

### 4. 时间差计算（`timedelta`）
`timedelta` 表示两个日期/时间的差值，支持加减运算，可用于计算未来/过去的时间、两个日期的间隔：

#### 语法
```python
from datetime import timedelta
# 创建时间差：days（天）、hours（时）、minutes（分）、seconds（秒）、microseconds（微秒）
delta = timedelta(days=2, hours=3, minutes=30)
```

#### 示例：时间差计算
```python
from datetime import datetime, timedelta

now = datetime.now()

# 1. 计算未来时间（当前时间 + 时间差）
future_1 = now + timedelta(days=7)  # 7天后
future_2 = now + timedelta(hours=3, minutes=30)  # 3小时30分钟后
print("7天后：", future_1.strftime("%Y-%m-%d %H:%M"))
print("3小时30分钟后：", future_2.strftime("%Y-%m-%d %H:%M"))

# 2. 计算过去时间（当前时间 - 时间差）
past_1 = now - timedelta(weeks=1)  # 1周前
past_2 = now - timedelta(seconds=120)  # 2分钟前
print("1周前：", past_1.strftime("%Y-%m-%d %H:%M"))
print("2分钟前：", past_2.strftime("%Y-%m-%d %H:%M"))

# 3. 计算两个日期的差值
date1 = datetime(2025, 1, 1)
date2 = datetime(2025, 12, 31)
delta = date2 - date1
print("2025年1月1日到12月31日的天数：", delta.days)  # 输出：364（2025年不是闰年）
print("总秒数：", delta.total_seconds())  # 输出：31449600.0（364天 * 24小时 * 3600秒）

# 4. 时间差的加减
delta1 = timedelta(days=2)
delta2 = timedelta(hours=12)
delta_total = delta1 + delta2
print("2天 + 12小时 =", delta_total.days, "天", delta_total.seconds // 3600, "小时")  # 输出：2 天 12 小时
```

### 5. 时区转换（Python 3.9+ `zoneinfo`）
Python 3.9+ 内置 `zoneinfo` 模块，支持 IANA 时区（如 `Asia/Shanghai`、`Europe/London`、`UTC`），可轻松实现时区转换：

#### 示例：时区转换
```python
from datetime import datetime
from zoneinfo import ZoneInfo  # Python 3.9+
import pytz  # 兼容 Python 3.9 前（需安装：pip install pytz）

# 1. 创建带时区的日期时间对象（Aware DateTime）
# 方式 1：Python 3.9+（zoneinfo）
utc_dt = datetime(2025, 11, 20, 8, 0, 0, tzinfo=ZoneInfo("UTC"))  # UTC 时间
shanghai_dt = datetime(2025, 11, 20, 16, 0, 0, tzinfo=ZoneInfo("Asia/Shanghai"))  # 北京时间

# 方式 2：Python 3.9 前（pytz）
# utc_dt = datetime(2025, 11, 20, 8, 0, 0, tzinfo=pytz.UTC)
# shanghai_tz = pytz.timezone("Asia/Shanghai")
# shanghai_dt = shanghai_tz.localize(datetime(2025, 11, 20, 16, 0, 0))

print("UTC 时间：", utc_dt)  # 输出：2025-11-20 08:00:00+00:00
print("北京时间：", shanghai_dt)  # 输出：2025-11-20 16:00:00+08:00

# 2. 时区转换（Aware DateTime 支持直接转换）
# UTC 转北京时间
utc_to_shanghai = utc_dt.astimezone(ZoneInfo("Asia/Shanghai"))
print("UTC 转北京时间：", utc_to_shanghai.strftime("%Y-%m-%d %H:%M:%S %Z%z"))  # 输出：2025-11-20 16:00:00 CST+0800

# 北京时间转纽约时间
shanghai_to_ny = shanghai_dt.astimezone(ZoneInfo("America/New_York"))
print("北京时间转纽约时间：", shanghai_to_ny.strftime("%Y-%m-%d %H:%M:%S %Z%z"))  # 输出：2025-11-20 03:00:00 EST-0500

# 3. 本地时间转 UTC
local_dt = datetime.now(ZoneInfo("Asia/Shanghai"))  # 当前北京时间（Aware）
local_to_utc = local_dt.astimezone(ZoneInfo("UTC"))
print("本地时间转 UTC：", local_to_utc)
```

### 6. 日期时间比较与属性操作
```python
from datetime import datetime, date

# 1. 日期时间比较（==、!=、>、>=、<、<=）
dt1 = datetime(2025, 11, 20)
dt2 = datetime(2025, 11, 21)
print("dt1 < dt2：", dt1 < dt2)  # 输出：True
print("dt1 == dt2：", dt1 == dt2)  # 输出：False

date1 = date(2025, 1, 1)
date2 = date(2025, 1, 1)
print("date1 == date2：", date1 == date2)  # 输出：True

# 2. 获取日期时间属性
now = datetime.now()

print("月份：", now.month)
print("日期：", now.day)
print("小时：", now.hour)
print("分钟：", now.minute)

new_dt = now.replace(year=2026, month=1)  # 修改年和月
print("修改后的日期时间：", new_dt)  # 输出：2026-01-20 17:30:00.123456
```


## 五、实战场景：datetime 典型应用
### 场景 1：计算两个日期的天数差（如用户注册天数）
```python
from datetime import datetime

def calculate_days_between(start_date_str, end_date_str, format="%Y-%m-%d"):
    """
    计算两个日期的天数差
    :param start_date_str: 开始日期字符串（如 "2025-01-01"）
    :param end_date_str: 结束日期字符串（默认当前日期）
    :param format: 日期格式
    :return: 天数差（正数：end > start；负数：end < start）
    """
    start_dt = datetime.strptime(start_date_str, format)
    end_dt = datetime.strptime(end_date_str, format) if end_date_str else datetime.now()
    delta = end_dt - start_dt
    return delta.days

# 示例：计算用户注册天数（2025-01-01 注册，当前日期 2025-11-20）
register_date = "2025-01-01"
days = calculate_days_between(register_date, "2025-11-20")
print(f"用户注册天数：{days} 天")  # 输出：324 天
```

### 场景 2：生成指定范围内的日期列表（如近7天日期）
```python
from datetime import datetime, timedelta

def generate_date_range(start_date, days, format="%Y-%m-%d"):
    """
    生成从 start_date 开始的连续日期列表
    :param start_date: 开始日期（datetime 对象或字符串）
    :param days: 生成的日期个数（含 start_date）
    :param format: 输出格式（None 则返回 datetime 对象）
    :return: 日期列表
    """
    if isinstance(start_date, str):
        start_dt = datetime.strptime(start_date, format)
    else:
        start_dt = start_date
    
    date_list = []
    for i in range(days):
        current_dt = start_dt + timedelta(days=i)
        if format:
            date_list.append(current_dt.strftime(format))
        else:
            date_list.append(current_dt)
    return date_list

# 示例 1：生成近7天日期（含今天）
today = datetime.now().strftime("%Y-%m-%d")
recent_7_days = generate_date_range(today, 7)
print("近7天日期：", recent_7_days)
# 输出：['2025-11-20', '2025-11-21', '2025-11-22', '2025-11-23', '2025-11-24', '2025-11-25', '2025-11-26']

# 示例 2：生成 2025年12月的所有日期
dec_start = datetime(2025, 12, 1)
dec_days = 31  # 12月有31天
dec_dates = generate_date_range(dec_start, dec_days)
print("2025年12月日期：", dec_dates)
```

### 场景 3：计算程序运行耗时
```python
from datetime import datetime
import time

def test_function():
    """模拟耗时操作"""
    time.sleep(2)  # 休眠 2 秒

# 记录开始时间
start_time = datetime.now()

# 执行程序
test_function()

# 记录结束时间
end_time = datetime.now()

# 计算耗时（秒）
elapsed_time = (end_time - start_time).total_seconds()
print(f"程序运行耗时：{elapsed_time:.2f} 秒")  # 输出：程序运行耗时：2.00 秒
```

### 场景 4：时区转换（UTC 日志转北京时间）
```python
from datetime import datetime
from zoneinfo import ZoneInfo

def utc_to_shanghai(utc_time_str, format="%Y-%m-%d %H:%M:%S"):
    """
    UTC 时间字符串转为北京时间字符串
    :param utc_time_str: UTC 时间字符串（如 "2025-11-20 08:00:00"）
    :param format: 时间格式
    :return: 北京时间字符串
    """
    # 解析 UTC 时间字符串为 Aware DateTime
    utc_dt = datetime.strptime(utc_time_str, format).replace(tzinfo=ZoneInfo("UTC"))
    # 转换为北京时间
    shanghai_dt = utc_dt.astimezone(ZoneInfo("Asia/Shanghai"))
    # 格式化为字符串
    return shanghai_dt.strftime(format)

# 示例：UTC 日志时间转北京时间
utc_log_time = "2025-11-20 03:15:30"
shanghai_log_time = utc_to_shanghai(utc_log_time)
print(f"UTC 时间：{utc_log_time} → 北京时间：{shanghai_log_time}")  # 输出：UTC 时间：2025-11-20 03:15:30 → 北京时间：2025-11-20 11:15:30
```

### 场景 5：判断日期是否为工作日（排除周末）
```python
from datetime import datetime

def is_workday(date_obj):
    """
    判断日期是否为工作日（周一到周五）
    :param date_obj: date 或 datetime 对象
    :return: True（工作日）/ False（周末）
    """
    weekday = date_obj.weekday()  # 返回 0-6（0=周一，6=周日）
    return 0 <= weekday <= 4

# 示例
test_date1 = datetime(2025, 11, 20)  # 周四（weekday=3）
test_date2 = datetime(2025, 11, 22)  # 周六（weekday=5）
print(f"{test_date1.strftime('%Y-%m-%d')} 是否为工作日：{is_workday(test_date1)}")  # 输出：True
print(f"{test_date2.strftime('%Y-%m-%d')} 是否为工作日：{is_workday(test_date2)}")  # 输出：False
```

### 场景 6：格式化 Excel/CSV 中的日期字段
```python
from datetime import datetime
import csv

# 模拟从 CSV 读取的日期字符串（格式不统一）
csv_data = [
    ["订单号", "下单时间", "金额"],
    ["001", "2025-11-18 09:30", 199],
    ["002", "2025/11/19 14:45", 299],
    ["003", "2025.11.20 10:15", 399]
]

# 定义需要解析的日期格式（支持多种格式）
date_formats = ["%Y-%m-%d %H:%M", "%Y/%m/%d %H:%M", "%Y.%m.%d %H:%M"]

def parse_date(date_str):
    """解析多种格式的日期字符串为 datetime 对象"""
    for fmt in date_formats:
        try:
            return datetime.strptime(date_str, fmt)
        except ValueError:
            continue
    raise ValueError(f"无法解析日期格式：{date_str}")

# 格式化日期并写入新 CSV
with open("formatted_orders.csv", "w", newline="", encoding="utf-8") as f:
    writer = csv.writer(f)
    writer.writerow(csv_data[0])  # 写入表头
    for row in csv_data[1:]:
        order_id, date_str, amount = row
        # 解析日期
        dt = parse_date(date_str)
        # 格式化为统一格式（如 "2025年11月18日 09时30分"）
        formatted_date = dt.strftime("%Y年%m月%d日 %H时%M分")
        # 写入新行
        writer.writerow([order_id, formatted_date, amount])

print("CSV 日期格式化完成！")
```


## 六、常见问题与避坑指南
### 1. 坑 1：Naive DateTime 与 Aware DateTime 混用
- **问题**：未指定时区的 Naive DateTime 与带时区的 Aware DateTime 比较/运算，抛出 `TypeError: can't compare offset-naive and offset-aware datetimes`；
- **原因**：Python 不允许直接混用两种类型的日期时间对象；
- **解决方案**：
  - 统一转为 Aware DateTime（推荐）：给 Naive 对象添加时区；
  - 统一转为 Naive DateTime（不推荐，可能丢失时区信息）：移除 Aware 对象的时区；
  ```python
  from datetime import datetime
  from zoneinfo import ZoneInfo

  naive_dt = datetime(2025, 11, 20, 16, 0)
  aware_dt = datetime(2025, 11, 20, 8, 0, tzinfo=ZoneInfo("UTC"))

  # 正确：Naive → Aware（添加本地时区）
  naive_to_aware = naive_dt.replace(tzinfo=ZoneInfo("Asia/Shanghai"))
  print(naive_to_aware == aware_dt.astimezone(ZoneInfo("Asia/Shanghai")))  # 输出：True

  # 不推荐：Aware → Naive（移除时区）
  aware_to_naive = aware_dt.replace(tzinfo=None)
  ```

### 2. 坑 2：`strftime`/`strptime` 格式化指令错误
- **问题**：使用错误的格式化指令（如用 `%m` 表示分钟，`%M` 表示月份），抛出 `ValueError: time data '2025-11-20' does not match format '%Y-%M-%d'`；
- **原因**：混淆了月份（`%m`）和分钟（`%M`）、小时（`%H`/`%I`）等指令；
- **解决方案**：牢记常用指令（参考核心功能中的指令表），确保指令与字符串格式完全匹配：
  ```python
  # 错误：%M 是分钟，不能用于月份
  # dt = datetime.strptime("2025-11-20", "%Y-%M-%d")
  # 正确：%m 表示月份
  dt = datetime.strptime("2025-11-20", "%Y-%m-%d")
  ```

### 3. 坑 3：`timedelta` 不支持月份/年份加减
- **问题**：试图用 `timedelta(months=1)` 计算1个月后的时间，抛出 `TypeError: 'months' is an invalid keyword argument for timedelta()`；
- **原因**：`timedelta` 仅支持天、时、分、秒、微秒（月份/年份的天数不固定，如2月28天/31天）；
- **解决方案**：
  - 简单场景：手动处理（如月份+1，超过12则年份+1）；
  - 复杂场景：使用第三方库 `dateutil.relativedelta`（支持月份/年份加减）；
  ```python
  from datetime import datetime
  from dateutil.relativedelta import relativedelta  # 需安装：pip install python-dateutil

  now = datetime(2025, 11, 20)
  next_month = now + relativedelta(months=1)  # 1个月后（2025-12-20）
  next_year = now + relativedelta(years=1)    # 1年后（2026-11-20）
  print("1个月后：", next_month)
  ```

### 4. 坑 4：时间戳单位混淆（秒 vs 毫秒）
- **问题**：将毫秒级时间戳（如 `1755676800123`）直接传入 `fromtimestamp()`，导致日期时间错误（年份显示为 57786）；
- **原因**：`fromtimestamp()` 接收的是**秒级**时间戳，毫秒级需先转为秒（除以1000）；
- **解决方案**：根据时间戳单位调整，毫秒→秒（/1000）、微秒→秒（/1000000）：
  ```python
  from datetime import datetime

  # 毫秒级时间戳（1755676800123 → 1755676800.123 秒）
  timestamp_ms = 1755676800123
  dt = datetime.fromtimestamp(timestamp_ms / 1000)
  print("毫秒级时间戳解析：", dt)  # 输出：2025-11-20 08:00:00.123
  ```

### 5. 坑 5：闰年/闰月处理不当
- **问题**：手动计算日期时未考虑闰年（如 2024年2月有29天），导致 `ValueError: day is out of range for month`；
- **原因**：2月天数随闰年变化（平年28天，闰年29天），1、3、5等月份有31天，4、6等月份有30天；
- **解决方案**：避免手动计算日期，使用 `datetime.replace()` 或 `timedelta` 自动处理：
  ```python
  from datetime import datetime, timedelta

  # 错误：手动设置 2024年2月30日（2024是闰年，2月只有29天）
  # dt = datetime(2024, 2, 30)  # 抛出 ValueError

  # 正确：用 timedelta 计算（自动处理闰年）
  dt = datetime(2024, 2, 28) + timedelta(days=2)
  print("2024年2月28日+2天：", dt)  # 输出：2024-03-01（自动跨月）
  ```

### 6. 坑 6：Python 版本兼容性问题
- **问题**：在 Python 3.8 及以下版本使用 `zoneinfo` 模块，抛出 `ModuleNotFoundError: No module named 'zoneinfo'`；
- **原因**：`zoneinfo` 是 Python 3.9+ 新增模块；
- **解决方案**：
  - 升级 Python 到 3.9+（推荐）；
  - 兼容旧版本：使用 `pytz` 库替代 `zoneinfo`；
  ```python
  # Python 3.9 前兼容方案（pytz）
  import pytz
  from datetime import datetime

  utc_dt = datetime(2025, 11, 20, 8, 0, tzinfo=pytz.UTC)
  shanghai_dt = utc_dt.astimezone(pytz.timezone("Asia/Shanghai"))
  ```

### 7. 坑 7：`datetime.utcnow()` 创建的是 Naive 对象
- **问题**：`datetime.utcnow()` 返回的是无时区信息的 Naive 对象，而非 Aware UTC 对象，可能导致时区转换错误；
- **解决方案**：Python 3.11+ 推荐使用 `datetime.now(tz=ZoneInfo("UTC"))` 创建 Aware UTC 对象：
  ```python
  from datetime import datetime
  from zoneinfo import ZoneInfo

  # 不推荐：utcnow() 返回 Naive 对象
  naive_utc = datetime.utcnow()
  print("naive_utc 时区：", naive_utc.tzinfo)  # 输出：None

  # 推荐：创建 Aware UTC 对象
  aware_utc = datetime.now(tz=ZoneInfo("UTC"))
  print("aware_utc 时区：", aware_utc.tzinfo)  # 输出：UTC
  ```

## 八、总结
datetime 模块是 Python 日期时间处理的“基石”，核心价值在于 **功能全面、易用性强、原生无依赖**，覆盖从简单日期格式化到复杂时区转换的所有场景。

### 学习路径建议
1. 基础入门：掌握日期时间对象的创建、格式化（`strftime`）与解析（`strptime`）；
2. 进阶应用：熟练使用 `timedelta` 进行时间差计算，理解 Naive/Aware 日期时间的区别；
3. 实战优化：掌握时区转换（`zoneinfo`）、日期范围生成、时间比较等实用场景；
4. 避坑指南：重点关注时区混用、格式化指令错误、时间戳单位等常见问题；
5. 扩展提升：复杂场景可结合 `python-dateutil` 库（支持月份/年份加减）、`pytz`（兼容旧版本时区）。

### 核心推荐
- 日常开发：优先使用 `datetime.datetime` 类（兼顾日期和时间），避免单独使用 `date`/`time`；
- 格式化：日志/数据输出优先使用 `%Y-%m-%d %H:%M:%S` 格式（统一标准，便于解析）；
- 时区处理：Python 3.9+ 用 `zoneinfo`，旧版本用 `pytz`，避免手动处理时区偏移；
- 时间计算：优先使用 `timedelta` 或 `relativedelta`，避免手动处理闰年/闰月/跨月问题。

通过熟练掌握 datetime 模块，你可以高效解决所有日期时间相关的开发需求，避免因时间处理不当导致的线上问题！
