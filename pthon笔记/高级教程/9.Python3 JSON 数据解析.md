## 一、JSON 基础与核心概念
### 1. 什么是 JSON？
JSON（JavaScript Object Notation，JavaScript 对象表示法）是一种轻量级的**数据交换格式**，基于键值对结构，采用人类易读的文本格式存储和传输数据。它独立于编程语言，是目前 Web 开发、API 交互、数据存储的主流格式。

### 2. JSON 与 XML 的核心区别（结合英语学习场景）
| 特性         | JSON                          | XML                          |
|--------------|-------------------------------|------------------------------|
| 语法复杂度   | 简洁（键值对+数组）           | 繁琐（标签嵌套）             |
| 可读性       | 高（类 Python 字典/列表）     | 中等（标签冗余）             |
| 解析速度     | 快（轻量无冗余）              | 较慢（标签解析开销）         |
| 适用场景     | API 数据交互、配置文件、轻量存储 | 复杂结构化文档、配置文件     |
| 英语学习示例 | 单词表、学习进度、API 翻译结果 | 语法教程、复杂语料库         |

### 3. JSON 支持的数据类型
| JSON 类型   | Python 对应类型                | 示例                          |
|-------------|--------------------------------|-------------------------------|
| 对象（Object） | 字典（dict）                   | `{"word": "perseverance", "definition": "毅力"}` |
| 数组（Array）  | 列表（list）                   | `["easy", "medium", "hard"]`  |
| 字符串（String）| 字符串（str）                  | `"Success requires hard work"`|
| 数字（Number） | 整数（int）/浮点数（float）    | `95`（正确率）、`2.5`（小时） |
| 布尔值（Boolean）| 布尔值（bool）                 | `true`（完成）、`false`（未完成） |
| 空值（Null）  | None                          | `null`（无例句）              |

### 4. Python 处理 JSON 的核心优势
- 内置 `json` 模块，无需第三方依赖
- 语法与 Python 字典/列表高度兼容，学习成本低
- 解析速度快，适合处理 API 返回的英语学习数据
- 支持格式化输出，便于调试和阅读
- 广泛用于英语学习类工具（翻译 API、单词查询、学习进度同步）

## 三、核心模块：`json` 模块详解
Python 内置 `json` 模块，提供 4 个核心函数实现 JSON 与 Python 数据的相互转换（序列化/反序列化），无需安装额外依赖。

### 1. 核心函数对比表
| 函数          | 功能                                  | 输入/输出                          | 适用场景                          |
|---------------|---------------------------------------|-----------------------------------|-----------------------------------|
| `json.dumps()` | Python 数据 → JSON 字符串             | 输入：dict/list/str/int 等        | 网络传输、打印调试                |
| `json.dump()`  | Python 数据 → JSON 文件               | 输入：数据 + 文件对象             | 数据持久化存储（保存为 JSON 文件） |
| `json.loads()` | JSON 字符串 → Python 数据             | 输入：JSON 格式字符串             | 解析 API 返回数据（字符串格式）   |
| `json.load()`  | JSON 文件 → Python 数据               | 输入：文件对象                    | 读取本地 JSON 配置/数据文件        |

### 2. 基础用法：序列化（Python → JSON）
将 Python 字典/列表转换为 JSON 格式（字符串或文件）。

#### 示例 1：`json.dumps()` 转为 JSON 字符串
```python
import json

# Python 字典（英语学习计划）
learning_plan = {
    "user": "小明",
    "week": 1,
    "tasks": [
        {"day": 1, "content": "记忆 20 个单词", "completed": True},
        {"day": 2, "content": "语法练习 30 分钟", "completed": False}
    ],
    "progress": 0.3
}

# 转换为 JSON 字符串（默认 ASCII 编码中文，会转义）
json_str_default = json.dumps(learning_plan)
print("默认输出（中文转义）：")
print(json_str_default)
print("\n" + "-"*50 + "\n")

# 优化参数：支持中文、格式化输出
json_str_pretty = json.dumps(
    learning_plan,
    ensure_ascii=False,  # 不转义中文（关键！）
    indent=2,            # 缩进 2 空格，格式化输出
    sort_keys=False      # 不按键名排序（保持原顺序）
)
print("优化输出（中文+格式化）：")
print(json_str_pretty)
```

#### 示例 2：`json.dump()` 保存为 JSON 文件
```python
import json

# Python 列表（新增单词）
new_words = [
    {"id": 5, "word": "diligent", "definition": "勤奋的", "difficulty": "medium"},

with open("new_words.json", "w", encoding="utf-8") as f:

        indent=2             # 格式化
    )

print("✅ 新增单词已保存为 new_words.json")
```

### 3. 基础用法：反序列化（JSON → Python）
将 JSON 格式（字符串或文件）转换为 Python 字典/列表，便于后续处理。

#### 示例 1：`json.load()` 读取 JSON 文件
```python
import json

def read_word_json(file_path):
    """读取单词表 JSON 文件，转换为 Python 字典"""
    with open(file_path, "r", encoding="utf-8") as f:
        # JSON 文件 → Python 字典
        data = json.load(f)
    
    # 提取单词列表
    vocabulary = data["vocabulary"]
    print("📚 读取单词表成功，共 {} 个单词：".format(len(vocabulary)))
    
    # 遍历单词信息
    for word in vocabulary[:2]:  # 只打印前 2 个
        print(f"\n单词：{word['word']} ({word['part_of_speech']})")
        print(f"释义：{word['definition']}")
        print(f"难度：{word['difficulty']} | 掌握度：{word['mastery']}")

if __name__ == "__main__":
    read_word_json("english_words.json")
```

import json

def filter_unmastered_words(file_path):
    """筛选掌握度低于 0.6 的单词，生成复习列表"""
    with open(file_path, "r", encoding="utf-8") as f:
        data = json.load(f)
    vocabulary = data["vocabulary"]

    unmastered = [
        word for word in vocabulary 
        if word["mastery"] < 0.6

    for word in unmastered:

    review_data = {"review_list": unmastered, "review_date": "2025-11-20"}
    with open("word_review_list.json", "w", encoding="utf-8") as f:

### 3. 修改 JSON 数据并保存
#### 示例：更新学习任务状态，标记未完成任务为已完成
```python
import json

def update_task_status(file_path, task_date):
    """更新指定日期的任务状态为已完成"""
    # 读取 JSON 文件
    with open(file_path, "r", encoding="utf-8") as f:
        data = json.load(f)
    
    # 查找并更新任务
    updated = False
    for task in data["recent_tasks"]:
        if task["date"] == task_date and not task["completed"]:
            task["completed"] = True
            updated = True
            print(f"✅ 已标记任务「{task['task']}」为已完成")
    
    if not updated:
        print(f"❌ 未找到 {task_date} 的未完成任务")
        return
    
    # 保存修改后的 JSON
    with open(file_path, "w", encoding="utf-8") as f:
        json.dump(data, f, ensure_ascii=False, indent=2)
    print(f"✅ 学习进度文件已更新：{file_path}")

if __name__ == "__main__":
    update_task_status("learning_progress.json", "2025-11-19")
```

## 五、实战案例：结合英语学习场景
### 案例 1：JSON 单词表导入 MySQL 数据库
结合之前的 MySQL 教程，将 JSON 中的单词批量导入数据库：
```python
import json
import pymysql
from pymysql import Error

def json_to_mysql(json_file, db_config):
    """将 JSON 单词表导入 MySQL 数据库"""
    # 1. 读取 JSON 数据
    with open(json_file, "r", encoding="utf-8") as f:
        data = json.load(f)
    words = data["vocabulary"]
    
    # 转换为数据库插入格式（元组列表）
    word_data = [
        (
            word["id"],
            word["word"],
            word["definition"],
            word["example"],
            word["difficulty"],
            word["mastery"]
        )
        for word in words
    ]
    
    # 2. 连接 MySQL 并批量插入
    try:
        with pymysql.connect(**db_config) as connection:
            with connection.cursor() as cursor:
                # 创建表（若不存在）
                create_table_sql = """
                CREATE TABLE IF NOT EXISTS vocabulary_json (
                    id INT PRIMARY KEY,
                    word VARCHAR(50) NOT NULL,
                    definition TEXT,
                    example VARCHAR(200),
                    difficulty ENUM('easy', 'medium', 'hard'),
                    mastery FLOAT COMMENT '掌握度（0-1）'
                ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
                """
                cursor.execute(create_table_sql)
                
                # 批量插入（存在则更新）
                insert_sql = """
                INSERT INTO vocabulary_json 
                (id, word, definition, example, difficulty, mastery)
                VALUES (%s, %s, %s, %s, %s, %s)
                ON DUPLICATE KEY UPDATE
                    definition = VALUES(definition),
                    example = VALUES(example),
                    mastery = VALUES(mastery);
                """
                cursor.executemany(insert_sql, word_data)
                connection.commit()
                print(f"✅ 成功导入 {cursor.rowcount} 条单词数据到 MySQL")
    
    except Error as e:
        print(f"❌ 导入失败：{e}")

if __name__ == "__main__":
    # 数据库配置（替换为自己的信息）
    db_config = {
        "host": "localhost",
        "user": "root",
        "password": "your_password",
        "database": "english_db",
        "charset": "utf8mb4"
    }
    json_to_mysql("english_words.json", db_config)
```

### 案例 2：调用翻译 API 解析 JSON 响应
模拟调用百度翻译 API（返回 JSON 格式），解析翻译结果（实际使用需替换为真实 API Key）：
```python
import json
import requests

def translate_word(word, api_key="your_api_key"):
    """调用百度翻译 API，解析 JSON 响应"""
    url = "https://fanyi-api.baidu.com/api/trans/vip/translate"
    params = {
        "q": word,
        "from": "en",
        "to": "zh",
        "appid": "your_appid",  # 需在百度翻译开放平台申请
        "salt": "123456",
        "sign": "your_sign"     # 按 API 要求生成签名
    }
    
    try:
        # 发送请求（返回 JSON 格式字符串）
        response = requests.get(url, params=params, timeout=10)
        if response.status_code != 200:
            print(f"❌ 请求失败：状态码 {response.status_code}")
            return
        
        # JSON 字符串 → Python 字典
        result = json.loads(response.text)
        
        # 解析翻译结果
        if "trans_result" in result:
            translation = result["trans_result"][0]["dst"]
            print(f"🔍 翻译结果：{word} → {translation}")
        else:
            print(f"❌ 翻译失败：{result.get('error_msg', '未知错误')}")
    
    except requests.exceptions.Timeout:
        print("❌ 请求超时")
    except json.JSONDecodeError:
        print("❌ 响应格式错误（非 JSON）")
    except Exception as e:
        print(f"❌ 异常：{e}")

if __name__ == "__main__":
    # 翻译英语单词
    translate_word("perseverance")
    translate_word("fluency")
```

### 案例 3：批量下载 JSON 格式英语新闻并解析
结合网络编程，下载 JSON 格式的英语新闻，筛选简单难度的文章并保存：
```python
import json
import requests
import os

def download_english_news(api_url, save_dir="english_news"):
    """下载 JSON 格式英语新闻，筛选简单难度文章"""
    # 创建保存目录
    if not os.path.exists(save_dir):
        os.makedirs(save_dir)
    
    try:
        # 发送请求获取 JSON 数据
        response = requests.get(api_url, timeout=10)
        response.encoding = "utf-8"
        news_data = json.loads(response.text)  # JSON 字符串 → 字典
        
        if news_data["status"] != "ok":
            print(f"❌ 新闻获取失败：{news_data.get('message', '未知错误')}")
            return
        
        # 筛选简单难度（easy）的新闻
        easy_news = [
            news for news in news_data["articles"]
            if news["difficulty"] == "easy"
        ]
        
        print(f"📰 共获取 {len(news_data['articles'])} 条新闻，其中简单难度 {len(easy_news)} 条")
        
        # 保存每条新闻为单独的 JSON 文件
        for idx, news in enumerate(easy_news, 1):
            news_filename = f"easy_news_{idx}.json"
            news_path = os.path.join(save_dir, news_filename)
            
            with open(news_path, "w", encoding="utf-8") as f:
                json.dump(news, f, ensure_ascii=False, indent=2)
            
            print(f"✅ 保存：{news['title']} → {news_filename}")
    
    except requests.exceptions.RequestException as e:
        print(f"❌ 网络请求异常：{e}")
    except json.JSONDecodeError:
        print(f"❌ 响应不是有效的 JSON 格式")
    except Exception as e:
        print(f"❌ 异常：{e}")

if __name__ == "__main__":
    # 模拟 API 地址（实际可替换为真实新闻 API）
    mock_api_url = "https://example.com/api/english-news"
    # 本地测试：使用之前创建的 english_news.json 文件路径

    download_english_news(local_json_path)
```

## 六、常见问题与解决方案
### 1. 中文乱码问题（最常见！）
- 问题：JSON 中的中文显示为 `\u7cbe\u9009` 等 Unicode 转义字符。
- 解决方案：
  - 序列化时添加 `ensure_ascii=False` 参数（关键）：
    ```python
    json.dump(data, f, ensure_ascii=False, encoding="utf-8")
    ```
  - 读取文件时指定 `encoding="utf-8"`：
    ```python
    with open(file_path, "r", encoding="utf-8") as f:
        data = json.load(f)
    ```

### 2. JSON 格式错误（`JSONDecodeError`）
- 问题：解析时提示 `Expecting value`、`Invalid syntax` 等错误。
- 常见原因：
  - JSON 字符串/文件中存在语法错误（如逗号遗漏、引号不匹配）。
  - 键名未用双引号（JSON 要求键名必须是双引号，单引号无效）。
  - 尾部有多余逗号（如 `{"word": "test",}`）。
- 解决方案：
  - 用在线 JSON 校验工具（如 [JSON.cn](https://www.json.cn/)）检查格式。
  - 确保键名、字符串值均使用双引号。
  - 避免尾部多余逗号。

### 3. 嵌套 JSON 解析困难
- 问题：JSON 层级过深（如 `data["progress"]["vocabulary"]["mastered"]`），容易报错。
- 解决方案：
  - 分步解析，添加异常处理：
    ```python
    # 安全获取嵌套数据
    try:
        vocab_mastered = data.get("progress", {}).get("vocabulary", {}).get("mastered", 0)
    except AttributeError:
        vocab_mastered = 0
    ```
  - 使用第三方库 `glom`（需安装：`pip install glom`）简化嵌套获取：
    ```python
    from glom import glom
    vocab_mastered = glom(data, "progress.vocabulary.mastered", default=0)
    ```

### 4. Python 数据类型不支持
- 问题：序列化时提示 `TypeError: Object of type 'datetime' is not JSON serializable`。
- 原因：JSON 不支持 Python 特有的数据类型（如 `datetime.datetime`、`set`、自定义对象）。
- 解决方案：
  - 手动转换不支持的类型（如 `datetime` 转为字符串）：
    ```python
    from datetime import datetime
    data["date"] = datetime.now().strftime("%Y-%m-%d")  # 转为字符串
    ```
  - 自定义序列化函数：
    ```python
    def serialize_datetime(obj):
        if isinstance(obj, datetime):
            return obj.strftime("%Y-%m-%d %H:%M:%S")
        raise TypeError(f"无法序列化类型：{type(obj)}")
    
    json_str = json.dumps(data, default=serialize_datetime, ensure_ascii=False)
    ```

### 5. 大型 JSON 文件解析内存溢出
- 问题：解析 GB 级 JSON 文件时，`json.load()` 会将整个文件加载到内存，导致溢出。
- 解决方案：
  - 使用流式 JSON 解析库（如 `ijson`，需安装：`pip install ijson`）：
    ```python
    import ijson

    # 流式解析大型 JSON 文件
    with open("large_english_corpus.json", "r", encoding="utf-8") as f:
        parser = ijson.parse(f)
        for prefix, event, value in parser:
            if prefix == "vocabulary.item.word":
                print(f"单词：{value}")
    ```

## 七、学习小结与进阶建议
### 核心知识点总结
1. **JSON 解析核心流程**：
   - 序列化：Python 数据（dict/list）→ `json.dump/dumps` → JSON 文件/字符串。
   - 反序列化：JSON 文件/字符串 → `json.load/loads` → Python 数据（dict/list）。
2. **关键参数**：`ensure_ascii=False`（支持中文）、`indent`（格式化输出）。
3. **核心技巧**：嵌套数据安全获取、条件筛选、批量处理与持久化。
