# Python requests 模块完全指南
requests 是 Python 最流行的 **HTTP/HTTPS 请求库**，核心优势是「简洁易用、功能强大」—— 相比 Python 内置的 `urllib`，它简化了请求构造、响应处理、Cookie 管理等操作，支持 RESTful API 调用、文件上传下载、会话保持等场景，是爬虫、接口测试、后端交互的必备工具。


## 一、安装 requests
requests 非 Python 内置模块，需通过 pip 安装：
```bash
# Python3 专用（推荐，避免与 Python2 冲突）
pip3 install requests

# 或通用写法（已配置 Python3 环境变量）
pip install requests
```

验证安装成功：
```python
import requests
print(requests.__version__)  # 输出版本号（如 2.32.3，正常则安装成功）
```


## 二、核心基础：HTTP 请求方法
HTTP 协议的常用请求方法（GET/POST/PUT/DELETE）在 requests 中对应同名函数，语法统一为 `requests.方法(url, 参数)`，以下是高频用法示例：

### 1. GET 请求（获取数据）
最常用场景：从服务器获取数据（如查询信息、爬取网页内容），参数通常拼在 URL 中。

#### （1）基础 GET 请求（无参数）
```python
import requests

# 目标 URL（获取 GitHub 公共 API 数据）
url = "https://api.github.com/users/octocat"

# 发送 GET 请求，获取响应对象
response = requests.get(url)

# 处理响应（核心属性）
print("状态码：", response.status_code)  # 200 表示成功（HTTP 状态码）
print("响应文本（字符串）：", response.text[:100])  # 截取前 100 字符（避免输出过长）
print("响应 JSON（字典）：", response.json()["name"])  # 自动解析 JSON（API 场景常用）
print("响应头：", response.headers["Content-Type"])  # 获取响应头信息
```

#### （2）带 URL 参数的 GET 请求
参数通过 `params` 参数传入（字典格式），requests 会自动拼接成 `url?key1=val1&key2=val2`：
```python
import requests

url = "https://httpbin.org/get"  # 测试用 API，返回请求信息
params = {
    "name": "张三",
    "age": 25,
    "city": "北京"
}

# 传入 params 参数，自动拼接 URL
response = requests.get(url, params=params)

print("最终请求 URL：", response.url)  # 输出：https://httpbin.org/get?name=%E5%BC%A0%E4%B8%89&age=25&city=%E5%8C%97%E4%BA%AC（自动 URL 编码）
print("响应 JSON：", response.json())
```


### 2. POST 请求（提交数据）
用于向服务器提交数据（如登录、表单提交、上传文件），数据通常放在请求体中，而非 URL。

#### （1）提交表单数据（`application/x-www-form-urlencoded`）
用 `data` 参数传入字典，适用于普通表单提交：
```python
import requests

url = "https://httpbin.org/post"  # 测试用 API，返回提交的表单数据
data = {
    "username": "test_user",
    "password": "123456"
}

# 发送 POST 请求，提交表单数据
response = requests.post(url, data=data)

print("响应 JSON：", response.json()["form"])  # 输出提交的表单数据
```

#### （2）提交 JSON 数据（`application/json`）
API 交互最常用，用 `json` 参数传入字典（requests 自动序列化 JSON 并设置请求头）：
```python
import requests

url = "https://httpbin.org/post"
json_data = {
    "user_id": 1001,
    "nickname": "Pythoner",
    "hobbies": ["coding", "reading"]
}

# 传入 json 参数，自动处理 JSON 序列化和请求头
response = requests.post(url, json=json_data)

print("响应 JSON：", response.json()["json"])  # 输出提交的 JSON 数据
```

#### （3）上传文件（`multipart/form-data`）
用 `files` 参数传入字典，适用于文件上传场景（如图片、文档）：
```python
import requests

url = "https://httpbin.org/post"
files = {
    # "file" 是接口要求的字段名，open() 打开文件（rb 二进制模式）
    "file": open("test.jpg", "rb")
}

# 发送 POST 请求，上传文件
response = requests.post(url, files=files)

print("响应 JSON：", response.json()["files"])  # 输出文件上传信息

# 注意：文件使用后需关闭，或用 with 语句更安全
with open("test.jpg", "rb") as f:
    requests.post(url, files={"file": f})
```


### 3. 其他请求方法（PUT/DELETE）
适用于 RESTful API 场景（更新/删除资源），用法与 GET/POST 一致：
```python
import requests

url = "https://httpbin.org/put/1001"  # 模拟更新 ID=1001 的资源
put_data = {"nickname": "NewName"}
response = requests.put(url, json=put_data)  # 更新资源

url = "https://httpbin.org/delete/1001"  # 模拟删除 ID=1001 的资源
response = requests.delete(url)  # 删除资源

print("PUT 状态码：", response.status_code)  # 200/204 通常表示成功
print("DELETE 状态码：", response.status_code)
```


## 三、进阶用法：解决实际场景问题
### 1. 自定义请求头（Headers）
模拟浏览器请求（避免被服务器识别为爬虫）、设置授权信息（Token）等场景：
```python
import requests

url = "https://www.baidu.com"
headers = {
    # 模拟 Chrome 浏览器（关键：User-Agent 字段）
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36",
    "Authorization": "Bearer your_token_here"  # API 授权 Token（如有）
}

# 传入 headers 参数
response = requests.get(url, headers=headers)

print("响应状态码：", response.status_code)  # 200 表示模拟成功
```


### 2. Cookie 管理
requests 自动处理 Cookie（会话保持），也可手动设置/获取 Cookie：
#### （1）自动保持 Cookie（登录后免重复验证）
```python
import requests

# 1. 登录接口（提交账号密码，获取 Cookie）
login_url = "https://httpbin.org/post"
login_data = {"username": "test", "password": "123"}
response = requests.post(login_url, data=login_data)

# 2. 访问需要登录的页面（自动携带上一步的 Cookie）
profile_url = "https://httpbin.org/cookies"
response = requests.get(profile_url)

print("当前 Cookie：", response.json()["cookies"])  # 输出登录后的 Cookie
```

#### （2）手动设置 Cookie
```python
import requests

url = "https://httpbin.org/cookies"
cookies = {"user_id": "1001", "is_login": "true"}

# 传入 cookies 参数
response = requests.get(url, cookies=cookies)

print("响应 Cookie：", response.json()["cookies"])  # 输出手动设置的 Cookie
```


### 3. 超时设置（避免无限等待）
网络不稳定时，设置 `timeout` 参数（单位：秒），防止程序卡死：
```python
import requests

url = "https://httpbin.org/get"

try:
    # timeout=5：连接+响应总超时 5 秒
    response = requests.get(url, timeout=5)
except requests.exceptions.Timeout:
    print("请求超时！")
except requests.exceptions.RequestException as e:
    print("请求失败：", e)  # 捕获其他所有请求异常
```


### 4. 代理设置（隐藏真实 IP）
爬虫、跨地区访问场景常用，支持 HTTP/HTTPS/SOCKS 代理：
```python
import requests

url = "https://httpbin.org/ip"  # 返回当前请求的 IP
proxies = {
    # HTTP 代理（替换为真实代理 IP 和端口）
    "http": "http://127.0.0.1:8888",
    "https": "https://127.0.0.1:8888"
}

# 传入 proxies 参数
response = requests.get(url, proxies=proxies)

print("当前请求 IP：", response.json()["origin"])  # 输出代理 IP
```

#### 带认证的代理：
```python
proxies = {
    "http": "http://用户名:密码@127.0.0.1:8888",
    "https": "https://用户名:密码@127.0.0.1:8888"
}
```


### 5. 会话保持（Session）
`requests.Session()` 用于维持一个会话（共享 Cookie、请求头），适用于多步交互（如登录→浏览→下单）：
```python
import requests

# 创建 Session 对象（自动保存 Cookie）
session = requests.Session()

# 1. 登录（Session 自动保存登录 Cookie）
login_url = "https://httpbin.org/post"
session.post(login_url, data={"username": "test", "password": "123"})

# 2. 访问需要登录的页面（Session 自动携带 Cookie）
profile_url = "https://httpbin.org/cookies"
response = session.get(profile_url)

print("会话 Cookie：", response.json()["cookies"])

# 3. 关闭 Session（可选）
session.close()
```


### 6. SSL 证书验证（HTTPS）
requests 默认验证 HTTPS 证书，若访问自签名证书的网站，需关闭验证（`verify=False`）：
```python
import requests

url = "https://自签名证书网站.com"

# verify=False：关闭 SSL 证书验证（不推荐生产环境，仅测试用）
response = requests.get(url, verify=False)

# 关闭警告（可选，避免控制台输出 SSL 警告）
requests.packages.urllib3.disable_warnings()
```


## 四、响应对象（Response）核心属性
发送请求后返回的 `response` 对象包含丰富信息，常用属性如下：
| 属性/方法          | 功能描述                                  | 示例                                  |
|---------------------|-------------------------------------------|---------------------------------------|
| `response.status_code` | HTTP 状态码（200=成功，404=未找到，500=服务器错误） | `200`                                 |
| `response.text`      | 响应内容（字符串，自动猜测编码）          | `response.text[:50]`                  |
| `response.content`   | 响应内容（二进制，适用于下载图片/文件）    | `with open("img.jpg", "wb") as f: f.write(response.content)` |
| `response.json()`    | 解析 JSON 响应（返回字典，仅适用于 JSON 数据） | `response.json()["key"]`              |
| `response.headers`   | 响应头（字典格式）                        | `response.headers["Content-Type"]`    |
| `response.cookies`   | 响应 Cookie（RequestsCookieJar 对象）     | `response.cookies["user_id"]`         |
| `response.url`       | 最终请求的 URL（可能被重定向后的值）      | `response.url`                        |
| `response.encoding`  | 响应编码（可手动设置，解决中文乱码）       | `response.encoding = "utf-8"`         |


## 五、常见问题排查
### 1. 中文乱码
原因：响应编码猜测错误，解决方案：
```python
# 方法1：手动设置编码
response.encoding = "utf-8"  # 或 "gbk"，根据网站编码调整

# 方法2：用 content 手动解码
response.content.decode("utf-8")
```

### 2. 请求被拒绝（403 Forbidden）
原因：服务器识别为爬虫，解决方案：
- 设置 `User-Agent` 模拟浏览器；
- 携带 Cookie 或 Referer 请求头；
- 使用代理 IP。

### 3. 超时/连接失败
- 检查网络是否正常；
- 增加超时时间（`timeout=10`）；
- 更换代理或目标服务器。

### 4. JSON 解析错误（JSONDecodeError）
原因：响应内容不是 JSON 格式，解决方案：
- 先打印 `response.text` 查看实际响应内容；
- 确认接口是否返回 JSON（检查 `response.headers["Content-Type"]` 是否为 `application/json`）。


## 六、总结
requests 模块的核心优势是「简洁高效」，核心用法可总结为：
1. 基础请求：`get()`/`post()` 覆盖 80% 场景，参数用 `params`（URL）、`data`（表单）、`json`（JSON）；
2. 进阶配置：`headers`（模拟浏览器）、`cookies`（身份验证）、`timeout`（超时）、`proxies`（代理）；
3. 会话管理：`Session()` 维持多步交互的 Cookie 共享；
4. 响应处理：`text`（字符串）、`content`（二进制）、`json()`（API 数据）。

它是 Python 网络请求的首选工具，无论是爬虫、接口测试还是后端服务交互，都能大幅简化代码。如需更复杂的功能（如异步请求），可结合 `aiohttp` 模块使用。
