# Python queue 模块完全指南：线程安全队列与多线程通信实战
queue 是 Python 内置的 **线程安全队列模块**，核心功能是提供线程间安全的数据交换容器，支持先进先出（FIFO）、后进先出（LIFO）、优先级排序等队列类型，内置锁机制避免多线程并发访问冲突。它是多线程编程中实现“生产者-消费者”模型、任务分发、线程间通信的核心工具，无需手动处理锁同步，简洁高效且安全。

与 `collections.deque`（非线程安全、仅支持基础队列操作）相比，queue 模块专为多线程设计，提供阻塞/超时控制、任务完成追踪等高级功能；与 `multiprocessing.Queue`（进程安全）相比，queue 模块仅适用于线程间通信，资源消耗更低、速度更快。


## 一、核心定位与优势
### 1. 核心特点
- **内置无依赖**：Python 标准库自带，无需额外安装，导入即使用；
- **线程安全**：内部通过 `threading.Lock` 机制实现同步，多线程并发读写不会出现数据错乱；
- **支持多种队列类型**：FIFO（默认）、LIFO（栈结构）、优先级队列，满足不同场景需求；
- **阻塞与超时控制**：入队（`put`）和出队（`get`）支持阻塞模式和超时设置，避免线程死等；
- **任务追踪机制**：提供 `task_done()` 和 `join()` 方法，支持主线程等待所有任务执行完成；
- **轻量高效**：基于内置数据结构实现，操作响应快，资源消耗低。

### 2. 适用场景
- **生产者-消费者模型**：多线程协作（如爬虫：生产者爬取数据，消费者解析数据；批量处理：生产者生成任务，消费者执行任务）；
- **多线程任务分发**：主线程分配任务到多个工作线程，实现并行处理；
- **线程间数据传递**：线程间安全交换数据（如日志收集、结果汇总）；
- **任务队列管理**：按优先级处理任务（如紧急任务优先执行）、控制并发任务数量；
- **缓冲机制**：平衡生产和消费速度差异（如生产者生成数据快，消费者处理慢，队列暂存数据避免阻塞）。

### 3. 与相似工具的对比
| 工具/模块                | 核心优势                          | 缺点                                  | 适用场景                  |
|---------------------------|-----------------------------------|---------------------------------------|---------------------------|
| `queue.Queue`（本文核心） | 线程安全、支持阻塞/超时、任务追踪 | 仅线程安全（不支持进程间通信）        | 多线程间通信、任务分发    |
| `collections.deque`       | 速度快、支持双向操作、轻量        | 非线程安全（多线程需手动加锁）        | 单线程队列/栈操作        |
| `multiprocessing.Queue`   | 进程安全、支持跨进程通信          | 资源消耗高、速度较慢                  | 多进程间通信              |
| `heapq` 模块              | 轻量、支持优先级排序              | 非线程安全、无阻塞/任务追踪功能       | 单线程优先级队列          |

**结论**：多线程场景优先使用 `queue` 模块，兼顾安全、功能与效率；单线程用 `deque`，多进程用 `multiprocessing.Queue`。


## 二、环境准备
queue 是 Python 内置模块，**无需额外安装**，直接导入即可使用：
```python

import threading  # 多线程配合使用（常用）
import time  # 用于示例中的延时控制
```

验证模块可用性（简单队列操作）：
```python
# 创建 FIFO 队列
q = queue.Queue(maxsize=3)  # maxsize=3：队列最大容量为 3

# 入队
q.put("任务1")
q.put("任务2")
q.put("任务3")

print("出队顺序（FIFO）：", q.get(), q.get(), q.get())  # 输出：任务1 任务2 任务3

```


## 三、核心概念：理解 queue 模块的核心组件
### 1. 队列类型（3种核心队列）
queue 模块提供 3 种常用队列类，覆盖不同业务场景：
| 队列类                | 中文名称          | 特点                                  | 适用场景                  |
|-----------------------|-------------------|---------------------------------------|---------------------------|
| `queue.Queue(maxsize=0)` | 先进先出队列（FIFO） | 先入队的元素先出队（类似排队），`maxsize=0` 表示无界队列（默认） | 任务按顺序执行、普通队列场景 |
| `queue.LifoQueue(maxsize=0)` | 后进先出队列（LIFO） | 后入队的元素先出队（类似栈）          | 最近任务优先处理（如撤销操作） |
| `queue.PriorityQueue(maxsize=0)` | 优先级队列        | 按元素优先级排序，优先级高的先出队（元素需为可比较的元组，如 `(优先级, 数据)`） | 紧急任务优先、按权重处理任务 |

### 2. 核心方法（所有队列类通用）
queue 模块的队列类提供统一的核心方法，操作简洁直观：
| 方法名                  | 功能描述                                  | 关键参数                                  | 注意事项                                  |
|-------------------------|-------------------------------------------|-------------------------------------------|-------------------------------------------|
| `put(item, block=True, timeout=None)` | 入队（添加元素）                          | `item`：入队元素；`block=True`：队列满时阻塞；`timeout`：阻塞超时时间（秒） | 队列满且 `block=False` 时抛出 `Full` 异常；`timeout` 超时后抛出 `Full` 异常 |
| `get(block=True, timeout=None)` | 出队（获取并移除元素）                    | `block=True`：队列为空时阻塞；`timeout`：阻塞超时时间（秒） | 队列为空且 `block=False` 时抛出 `Empty` 异常；`timeout` 超时后抛出 `Empty` 异常 |
| `qsize()`               | 返回队列当前元素个数                      | -                                        | 多线程环境下结果可能不准确（仅作参考）    |
| `empty()`               | 判断队列是否为空（返回 `True`/`False`）    | -                                        | 多线程环境下结果可能瞬时变化（仅作参考）  |
| `full()`                | 判断队列是否已满（返回 `True`/`False`）    | -                                        | 仅对有界队列（`maxsize>0`）有效；多线程环境下结果可能瞬时变化 |
| `task_done()`           | 标记任务完成（通知队列该元素已处理完毕）  | -                                        | 需与 `join()` 配合使用，每个 `get()` 后需调用一次 |
| `join()`                | 阻塞主线程，等待队列中所有任务完成        | -                                        | 需配合 `task_done()`，所有元素都调用 `task_done()` 后才解除阻塞 |
| `put_nowait(item)`      | 非阻塞入队（等价于 `put(item, block=False)`） | `item`：入队元素                          | 队列满时直接抛出 `Full` 异常              |
| `get_nowait()`          | 非阻塞出队（等价于 `get(block=False)`）    | -                                        | 队列为空时直接抛出 `Empty` 异常            |

### 3. 异常类型
queue 模块定义了 3 种常用异常，用于处理队列操作的边界情况：
| 异常类                  | 触发场景                                  |
|-------------------------|-------------------------------------------|
| `queue.Empty`           | 队列为空时，非阻塞 `get()` 或超时 `get()` |
| `queue.Full`            | 队列已满时，非阻塞 `put()` 或超时 `put()` |
| `queue.IllegalQueueOperation` | 非法队列操作（如关闭队列后继续入队，极少用到） |


## 四、核心功能：分场景详解（语法+示例+应用）
### 1. 基础用法：3种队列类型实战
#### （1）FIFO 队列（`queue.Queue`）
最常用队列类型，按“先进先出”顺序处理元素，适合普通任务排队场景：
```python
import queue

# 创建无界队列（maxsize=0，可无限添加元素）
fifo_q = queue.Queue()

# 入队（支持批量添加）
tasks = ["任务1", "任务2", "任务3", "任务4"]
for task in tasks:
    fifo_q.put(task)
    print(f"入队：{task}，当前队列大小：{fifo_q.qsize()}")

# 出队（按入队顺序）
print("\n出队顺序（FIFO）：")
while not fifo_q.empty():
    task = fifo_q.get()
    print(f"出队：{task}，剩余队列大小：{fifo_q.qsize()}")
    fifo_q.task_done()  # 标记任务完成（若后续用 join() 需调用）
```

输出结果：
```
入队：任务1，当前队列大小：1
入队：任务2，当前队列大小：2
入队：任务3，当前队列大小：3
入队：任务4，当前队列大小：4

出队顺序（FIFO）：
出队：任务1，剩余队列大小：3
出队：任务2，剩余队列大小：2
出队：任务3，剩余队列大小：1
出队：任务4，剩余队列大小：0
```

#### （2）LIFO 队列（`queue.LifoQueue`）
“后进先出”，类似栈结构，适合“最近任务优先处理”场景（如撤销操作、临时缓存）：
```python
import queue

# 创建 LIFO 队列
lifo_q = queue.LifoQueue(maxsize=3)  # 有界队列，最大容量 3

# 入队（队列满时阻塞，此处未满）
lifo_q.put("数据1")
lifo_q.put("数据2")
lifo_q.put("数据3")

# 尝试入队第4个元素（非阻塞模式，队列满会抛出 Full 异常）
try:
    lifo_q.put_nowait("数据4")
except queue.Full:
    print("队列已满，无法入队：数据4")

# 出队（后入先出）
print("\n出队顺序（LIFO）：")
while not lifo_q.empty():
    data = lifo_q.get()
    print(f"出队：{data}")
```

输出结果：
```
队列已满，无法入队：数据4

出队顺序（LIFO）：
出队：数据3
出队：数据2
出队：数据1
```

#### （3）优先级队列（`queue.PriorityQueue`）
按元素优先级排序，元素需为 **可比较的元组**（如 `(优先级数值, 数据)`），数值越小优先级越高（默认升序排序）：
```python
import queue

# 创建优先级队列
priority_q = queue.PriorityQueue()

# 入队（元素为 (优先级, 数据)，优先级数值越小越先出队）
priority_q.put((3, "普通任务"))
priority_q.put((1, "紧急任务"))
priority_q.put((2, "重要任务"))
priority_q.put((1, "另一个紧急任务"))  # 优先级相同，按入队顺序排列

# 出队（按优先级排序）
print("出队顺序（按优先级）：")
while not priority_q.empty():
    priority, task = priority_q.get()
    print(f"优先级：{priority} → 任务：{task}")
```

输出结果：
```
出队顺序（按优先级）：
优先级：1 → 任务：紧急任务
优先级：1 → 任务：另一个紧急任务
优先级：2 → 任务：重要任务
优先级：3 → 任务：普通任务
```

**自定义优先级规则**：若需“数值越大优先级越高”，可在入队时取负数（如 `(-3, "普通任务")`），或自定义可比较对象。

### 2. 阻塞与超时控制（避免线程死等）
`put()` 和 `get()` 方法的 `block` 和 `timeout` 参数是多线程场景的核心，用于避免队列满/空时线程无限阻塞：
```python
import queue
import time

# 创建有界队列（容量 2）
q = queue.Queue(maxsize=2)

# 测试入队阻塞与超时
def test_put_timeout():
    try:
        print("入队：任务1")
        q.put("任务1", block=True, timeout=1)  # 阻塞 1 秒（队列未满，立即入队）
        print("入队：任务2")
        q.put("任务2", block=True, timeout=1)  # 队列满，阻塞 1 秒
        print("入队：任务3（超时测试）")
        q.put("任务3", block=True, timeout=1)  # 队列满，阻塞 1 秒后抛出异常
    except queue.Full:
        print("入队失败：队列已满（超时 1 秒）")

# 测试出队阻塞与超时
def test_get_timeout():
    time.sleep(2)  # 等待入队操作完成
    try:
        print("\n出队：", q.get(block=True, timeout=1))
        print("出队：", q.get(block=True, timeout=1))
        print("出队：（超时测试）")
        print(q.get(block=True, timeout=1))  # 队列为空，阻塞 1 秒后抛出异常
    except queue.Empty:
        print("出队失败：队列为空（超时 1 秒）")

# 启动测试
test_put_timeout()
test_get_timeout()
```

输出结果：
```
入队：任务1
入队：任务2
入队：任务3（超时测试）
入队失败：队列已满（超时 1 秒）

出队： 任务1
出队： 任务2
出队：（超时测试）
出队失败：队列为空（超时 1 秒）
```

### 3. 任务追踪（`task_done()` + `join()`）
`join()` 方法会阻塞主线程，直到队列中所有元素都被 `task_done()` 标记完成，适用于“主线程等待所有子线程任务执行完毕”的场景：
```python
import queue
import threading
import time

# 创建队列
q = queue.Queue()

# 工作线程函数（消费者）
def worker():
    while True:
        task = q.get()  # 阻塞等待任务
        print(f"线程 {threading.current_thread().name} 处理任务：{task}")
        time.sleep(1)  # 模拟任务处理耗时
        q.task_done()  # 标记任务完成
        print(f"线程 {threading.current_thread().name} 完成任务：{task}")

# 创建 2 个工作线程（守护线程，主线程退出时自动终止）
for i in range(2):
    t = threading.Thread(target=worker, name=f"Worker-{i+1}", daemon=True)
    t.start()

# 主线程（生产者）添加任务
tasks = ["任务1", "任务2", "任务3", "任务4"]
for task in tasks:
    q.put(task)
    print(f"主线程添加任务：{task}")

# 阻塞主线程，等待所有任务完成（所有 task 都调用 task_done() 后解除阻塞）
q.join()
print("\n所有任务处理完成，主线程退出！")
```

输出结果（线程执行顺序可能略有差异）：
```
主线程添加任务：任务1
主线程添加任务：任务2
主线程添加任务：任务3
主线程添加任务：任务4
线程 Worker-1 处理任务：任务1
线程 Worker-2 处理任务：任务2
线程 Worker-1 完成任务：任务1
线程 Worker-1 处理任务：任务3
线程 Worker-2 完成任务：任务2
线程 Worker-2 处理任务：任务4
线程 Worker-1 完成任务：任务3
线程 Worker-2 完成任务：任务4

所有任务处理完成，主线程退出！
```

**关键说明**：
- 每个 `get()` 调用后必须对应一个 `task_done()`，否则 `join()` 会无限阻塞；
- 守护线程（`daemon=True`）确保主线程退出时子线程自动终止，避免孤儿线程。


## 五、实战场景：queue 模块典型应用
### 场景 1：多生产者-多消费者模型（爬虫示例）
经典多线程协作场景：多个生产者线程爬取网页链接，多个消费者线程解析网页内容并保存：
```python
import queue
import threading
import time
import random

# 全局队列（链接队列：生产者放入，消费者取出；结果队列：消费者放入解析结果）
url_queue = queue.Queue(maxsize=10)  # 待爬取链接队列（有界，避免内存溢出）
result_queue = queue.Queue()  # 解析结果队列

# 生产者线程（爬取链接）
def producer(name):
    urls = [f"https://example.com/page{i}" for i in range(10)]  # 模拟待爬链接
    for url in urls:
        try:
            url_queue.put(url, block=True, timeout=2)  # 队列满时阻塞 2 秒
            print(f"生产者 {name} 放入链接：{url}，当前待爬队列大小：{url_queue.qsize()}")
            time.sleep(random.uniform(0.1, 0.3))  # 模拟爬取间隔
        except queue.Full:
            print(f"生产者 {name} 放入失败：队列已满（{url}）")

# 消费者线程（解析链接）
def consumer(name):
    while True:
        try:
            url = url_queue.get(block=True, timeout=3)  # 队列为空时阻塞 3 秒
            print(f"消费者 {name} 取出链接：{url}")
            # 模拟解析过程
            result = f"[{name}] 解析 {url} 成功，内容长度：{random.randint(100, 500)}"
            result_queue.put(result)
            url_queue.task_done()  # 标记链接处理完成
            time.sleep(random.uniform(0.2, 0.5))  # 模拟解析耗时
        except queue.Empty:
            print(f"消费者 {name} 退出：无待解析链接（超时 3 秒）")
            break

# 启动生产者线程（2个）
producers = [
    threading.Thread(target=producer, name=f"Producer-{i+1}") for i in range(2)
]
for p in producers:
    p.start()

# 等待所有生产者完成任务
for p in producers:
    p.join()

print("\n所有生产者已完成链接爬取，等待消费者解析...\n")

# 启动消费者线程（3个）
consumers = [
    threading.Thread(target=consumer, name=f"Consumer-{i+1}") for i in range(3)
]
for c in consumers:
    c.start()

# 等待链接队列所有任务完成
url_queue.join()

# 输出解析结果
print("\n所有链接解析完成，结果如下：")
while not result_queue.empty():
    print(result_queue.get())
```

### 场景 2：多线程任务分发（批量文件处理）
主线程分发文件处理任务到多个工作线程，控制并发数，避免资源占用过高：
```python
import queue
import threading
import time
import os

# 任务队列（存储待处理文件路径）
task_queue = queue.Queue()

# 工作线程函数（处理文件）
def file_processor(thread_name):
    while True:
        try:
            file_path = task_queue.get(block=True, timeout=2)
            print(f"线程 {thread_name} 处理文件：{file_path}")
            # 模拟文件处理（如读取内容、转换格式等）
            time.sleep(0.5)
            task_queue.task_done()
            print(f"线程 {thread_name} 完成文件：{file_path}")
        except queue.Empty:
            print(f"线程 {thread_name} 退出：无待处理文件")
            break

# 主线程：收集待处理文件并分发任务
def main():
    # 模拟待处理文件（当前目录下的 .txt 文件）
    file_dir = "./"
    for file in os.listdir(file_dir):
        if file.endswith(".txt"):
            file_path = os.path.join(file_dir, file)
            task_queue.put(file_path)

    print(f"共收集到 {task_queue.qsize()} 个待处理文件")
    if task_queue.empty():
        print("无待处理文件，程序退出")
        return

    # 启动 4 个工作线程（并发数控制为 4）
    thread_count = 4
    threads = [
        threading.Thread(target=file_processor, name=f"Processor-{i+1}")
        for i in range(thread_count)
    ]
    for t in threads:
        t.start()

    # 等待所有任务完成
    task_queue.join()
    print("\n所有文件处理完成！")

if __name__ == "__main__":
    main()
```

### 场景 3：优先级任务调度（紧急任务优先处理）
使用 `PriorityQueue` 实现任务优先级调度，紧急任务插队执行：
```python
import queue
import threading
import time

# 优先级队列（(优先级, 任务内容)，1 最高，5 最低）
priority_queue = queue.PriorityQueue()

# 任务处理线程
def task_handler():
    while True:
        try:
            priority, task = priority_queue.get(block=True, timeout=5)
            print(f"\n处理任务：优先级 {priority} → {task}")
            time.sleep(2)  # 模拟任务处理耗时
            priority_queue.task_done()
            print(f"完成任务：优先级 {priority} → {task}")
        except queue.Empty:
            print("\n无更多任务，线程退出")
            break

# 启动任务处理线程
handler_thread = threading.Thread(target=task_handler, name="TaskHandler")
handler_thread.start()

# 模拟任务提交（不同优先级）
tasks = [
    (3, "日常报表生成"),
    (1, "服务器故障修复"),  # 最高优先级
    (2, "客户需求响应"),
    (4, "数据备份"),
    (1, "数据库连接恢复"),  # 最高优先级
    (5, "系统日志清理"),
]

for priority, task in tasks:
    priority_queue.put((priority, task))
    print(f"提交任务：优先级 {priority} → {task}")
    time.sleep(0.5)

# 等待所有任务完成
priority_queue.join()
handler_thread.join()
print("\n所有任务处理完毕！")
```

## 六、常见问题与避坑指南
### 1. 坑 1：混淆“线程安全”与“进程安全”
- **问题**：在多进程场景中误用 `queue.Queue`，导致数据错乱或进程挂起；
- **原因**：`queue.Queue` 仅支持线程安全（内部用线程锁），不支持进程间通信（进程资源隔离，线程锁无效）；
- **解决方案**：
  - 多进程通信使用 `multiprocessing.Queue`（进程安全，基于管道实现）；
  - 多线程通信使用 `queue.Queue`（线程安全，资源消耗更低）。

### 2. 坑 2：`join()` 无限阻塞（未调用 `task_done()`）
- **问题**：主线程调用 `queue.join()` 后一直阻塞，无法退出；
- **原因**：每个 `get()` 取出的元素都必须调用 `task_done()` 标记完成，否则队列认为还有未完成任务，`join()` 不会解除阻塞；
- **解决方案**：
  - 确保 `get()` 与 `task_done()` 一一对应（即使任务处理失败，也需调用 `task_done()` 避免阻塞）；
  - 异常场景中添加 `try-finally` 确保 `task_done()` 执行：
    ```python
    try:
        task = q.get()
        # 处理任务
    finally:
        q.task_done()  # 无论任务是否成功，都标记完成
    ```

### 3. 坑 3：队列满/空时无超时导致线程死锁
- **问题**：`put(block=True)` 或 `get(block=True)` 未设置 `timeout`，队列满/空时线程无限阻塞；
- **场景**：生产者线程异常退出，消费者线程一直 `get()` 阻塞；或消费者处理过慢，生产者 `put()` 阻塞；
- **解决方案**：
  - 始终设置合理的 `timeout` 参数，避免无限阻塞；
  - 结合 `qsize()`/`empty()`/`full()` 做预判（注意：多线程环境下这些方法结果仅作参考，不能作为唯一判断依据）；
  - 使用守护线程（`daemon=True`），确保主线程退出时子线程自动终止。

### 4. 坑 4：优先级队列元素不可比较
- **问题**：`PriorityQueue` 入队元素非可比较类型（如 `(优先级, 字典)`，字典不可比较），抛出 `TypeError: '<' not supported between instances of 'dict' and 'dict'`；
- **原因**：`PriorityQueue` 需按元素排序，要求元素必须支持比较操作（如数字、字符串、元组）；
- **解决方案**：
  - 入队元素统一使用“优先级+可比较数据”的元组（如 `(priority, id, data)`，id 确保优先级相同时可比较）；
  - 自定义数据类型需实现 `__lt__` 等比较方法（如自定义任务类，重写比较逻辑）。

### 5. 坑 5：无界队列导致内存溢出
- **问题**：`maxsize=0`（默认无界队列）时，生产者生成数据速度远快于消费者，队列无限增长，耗尽内存；
- **解决方案**：
  - 使用有界队列（`maxsize>0`），限制队列最大容量；
  - 结合 `put(timeout)` 控制生产者入队速度，避免队列溢出；
  - 监控队列大小（`qsize()`），动态调整生产者/消费者数量。

### 6. 坑 6：多线程中依赖 `empty()`/`full()` 做判断
- **问题**：多线程环境下，`empty()` 返回 `False` 后，另一线程可能已取走元素，导致当前线程 `get()` 阻塞；或 `full()` 返回 `False` 后，另一线程已放入元素，导致当前线程 `put()` 阻塞；
- **原因**：`empty()`/`full()` 是“瞬时快照”，多线程并发时结果可能瞬时变化，不具备原子性；
- **解决方案**：
  - 优先使用 `get(timeout)`/`put(timeout)`，通过超时机制处理边界情况，而非依赖 `empty()`/`full()`；
  - 若必须判断，需配合额外的锁（`threading.Lock`）确保判断和操作的原子性：
    ```python
    lock = threading.Lock()
    with lock:
        if not q.empty():
            task = q.get()
    ```


## 七、核心类/方法速查表
| 分类          | 核心接口                  | 功能描述                                  | 关键参数/注意事项                                  |
|---------------|---------------------------|-------------------------------------------|---------------------------------------------------|
| 队列类        | `queue.Queue(maxsize=0)`  | FIFO 队列（先进先出）                      | `maxsize=0` 无界，`>0` 有界                       |
|               | `queue.LifoQueue(maxsize=0)` | LIFO 队列（后进先出）                    | 类似栈结构，后入先出                               |
|               | `queue.PriorityQueue(maxsize=0)` | 优先级队列                              | 元素需为可比较元组（如 `(优先级, 数据)`）           |
| 核心方法      | `put(item, block=True, timeout=None)` | 入队                                    | `block=True` 阻塞，`timeout` 超时秒数，满时抛 `Full` |
|               | `get(block=True, timeout=None)` | 出队（移除并返回元素）                  | `block=True` 阻塞，`timeout` 超时秒数，空时抛 `Empty` |
|               | `put_nowait(item)`        | 非阻塞入队（等价于 `put(item, block=False)`） | 队列满时抛 `Full`                                 |
|               | `get_nowait()`            | 非阻塞出队（等价于 `get(block=False)`）    | 队列为空时抛 `Empty`                               |
| 状态查询      | `qsize()`                 | 返回队列元素个数                          | 多线程环境下结果不准确（参考用）                   |
|               | `empty()`                 | 判断队列是否为空                          | 多线程环境下结果可能瞬时变化                       |
|               | `full()`                  | 判断队列是否已满                          | 仅对有界队列有效，多线程环境下结果可能瞬时变化     |
| 任务追踪      | `task_done()`             | 标记任务完成                              | 每个 `get()` 后需调用，配合 `join()` 使用           |
|               | `join()`                  | 阻塞等待所有任务完成                      | 所有元素调用 `task_done()` 后解除阻塞               |
| 异常类        | `queue.Empty`             | 队列为空时非阻塞/超时 `get()` 触发         | -                                                 |
|               | `queue.Full`              | 队列已满时非阻塞/超时 `put()` 触发         | -                                                 |


## 八、总结
queue 模块是 Python 多线程编程的“瑞士军刀”，核心价值在于提供**线程安全的队列容器**和**简洁的任务协作接口**，无需手动处理锁同步，即可轻松实现生产者-消费者模型、任务分发、线程间通信等复杂场景。

### 学习路径建议
1. 掌握基础用法：熟悉 3 种队列类型（Queue/LifoQueue/PriorityQueue）的核心方法（put/get/qsize）；
2. 理解关键机制：掌握阻塞/超时控制、task_done() 与 join() 的配合使用（避免死锁和阻塞）；
3. 实战核心场景：重点练习“多生产者-多消费者模型”（多线程协作的核心）；
4. 规避常见坑点：区分线程安全/进程安全、避免依赖 empty()/full()、合理设置队列大小；
5. 灵活扩展：结合 threading 模块实现复杂线程协作，如动态调整线程数、任务优先级调度。

### 核心推荐
- 普通多线程任务队列：优先使用 `queue.Queue`（FIFO）；
- 最近任务优先：使用 `queue.LifoQueue`（LIFO）；
- 紧急任务插队：使用 `queue.PriorityQueue`；
- 多进程通信：改用 `multiprocessing.Queue`（切勿用 `queue.Queue`）；
- 高并发场景：结合线程池（`concurrent.futures.ThreadPoolExecutor`）与 queue，提升效率。

通过熟练掌握 queue 模块，你可以从容应对多线程编程中的数据交换和任务协作需求，开发出高效、安全、稳定的多线程应用！
