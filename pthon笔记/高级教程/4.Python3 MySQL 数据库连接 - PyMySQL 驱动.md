## 一、什么是 PyMySQL？
`PyMySQL` 是一款纯 Python 实现的 MySQL 数据库连接驱动，兼容 Python3.x 版本，遵循 Python DB API 2.0 规范。它无需依赖任何 C 扩展，跨平台兼容性强，且与经典的 `MySQLdb` 接口高度兼容，是 Python3 环境下替代 `MySQLdb`（仅支持 Python2）的首选方案，广泛用于 Web 开发、数据处理等场景。

### 核心优势
- 纯 Python 编写，安装简单，无需编译依赖
- 完全兼容 Python3.x，支持 MySQL 5.5+ 版本
- 接口简洁，与 `MySQLdb` 无缝切换（代码几乎无需修改）
- 支持事务、批量操作、参数化查询、SSL 连接等核心功能
- 社区活跃，bug 修复及时，文档完善

## 三、连接 MySQL 数据库
### 1. 基础连接模板（推荐写法）
```python
import pymysql
from pymysql import OperationalError, ProgrammingError

try:
    # 建立数据库连接
    connection = pymysql.connect(
        host='localhost',        # 数据库地址（本地默认 localhost）
        user='root',             # 数据库用户名
        password='your_password',# 数据库密码
        database='english_db',   # 要连接的数据库（需提前创建）
        port=3306,               # 端口（默认 3306，可选）
        charset='utf8mb4'        # 字符编码（支持中文/Emoji，推荐 utf8mb4）
    )

    if connection.open:
        print("MySQL 连接成功！")
        # 创建游标对象（用于执行 SQL 语句）
        cursor = connection.cursor()
        # 执行查询数据库信息的 SQL
        cursor.execute("SELECT DATABASE();")
        db_name = cursor.fetchone()[0]
        print(f"当前连接数据库：{db_name}")

except OperationalError as e:
    print(f"连接失败：{e}")  # 常见错误：密码错误、服务未启动、数据库不存在
except ProgrammingError as e:
    print(f"SQL 执行错误：{e}")
finally:
    # 关闭连接（必须执行！避免资源泄露）
    if 'connection' in locals() and connection.open:
        cursor.close()
        connection.close()
        print("MySQL 连接已关闭")
```

### 2. 关键配置参数详解
| 参数名       | 必选 | 说明                                  | 推荐值                  |
|--------------|------|---------------------------------------|-------------------------|
| `host`       | 是   | 数据库服务器地址                      | `localhost`/`127.0.0.1` |
| `user`       | 是   | 数据库登录用户名                      | `root`                  |
| `password`   | 是   | 数据库登录密码                        | 你的数据库密码          |
| `database`   | 否   | 初始连接的数据库（可后续用 `use` 切换）| `english_learning`      |
| `port`       | 否   | 数据库端口（MySQL 默认 3306）         | `3306`                  |
| `charset`    | 否   | 字符编码（避免中文乱码）              | `utf8mb4`               |
| `autocommit` | 否   | 是否自动提交事务（默认 False）        | `False`（手动控制更安全）|
| `cursorclass`| 否   | 游标类型（默认元组，可选字典格式）    | `pymysql.cursors.DictCursor` |

## 四、核心操作：CRUD（增删改查）
延续上一篇教程的「英语学习平台」场景，以 `vocabulary`（生词表）为例，演示完整 CRUD 操作。  
### 先创建测试表（SQL 语句）
若未创建表，先执行以下 SQL（可在 Navicat/MySQL 终端执行）：
```sql
CREATE TABLE vocabulary (
    id INT AUTO_INCREMENT PRIMARY KEY,
    word VARCHAR(50) NOT NULL COMMENT '英文单词',
    definition TEXT COMMENT '中文释义',
    example VARCHAR(200) COMMENT '英文例句',
    difficulty ENUM('easy', 'medium', 'hard') DEFAULT 'medium' COMMENT '难度等级',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '添加时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 1. 插入数据（Create）
#### 单条插入（参数化查询，避免 SQL 注入）
```python
import pymysql
from pymysql import Error

try:
    connection = pymysql.connect(
        host='localhost',
        user='root',
        password='your_password',
        database='english_db',
        charset='utf8mb4'
    )
    cursor = connection.cursor()

    # SQL 插入语句（%s 是占位符，不是字符串格式化！）
    insert_sql = """
    INSERT INTO vocabulary (word, definition, example, difficulty)
    VALUES (%s, %s, %s, %s)
    """
    # 插入的数据（元组格式，顺序与占位符对应）
    word_data = (
        "perseverance",
        "坚持不懈，毅力",
        "Success requires perseverance and hard work.",
        "hard"
    )

    cursor.execute(insert_sql, word_data)  # 执行插入
    connection.commit()  # 插入/更新/删除必须提交事务！
    print(f"插入成功！新增记录 ID：{cursor.lastrowid}")

except Error as e:
    print(f"插入失败：{e}")
    connection.rollback()  # 出错时回滚事务，避免数据不一致
finally:
    if connection.open:
        cursor.close()
        connection.close()
```

#### 批量插入（高效批量操作）
当需要插入多条数据时，使用 `executemany()` 效率远高于循环单条插入：
```python
import pymysql

    connection = pymysql.connect(
        host='localhost',
        user='root',
        password='your_password',
        database='english_db',
        charset='utf8mb4'
    )

    insert_sql = """
    INSERT INTO vocabulary (word, definition, example, difficulty)
    VALUES (%s, %s, %s, %s)
    """
    # 批量数据（列表嵌套元组）
    batch_data = [
        ("consistent", "一致的，持续的", "Be consistent in your English practice.", "medium"),
        ("proficiency", "熟练，精通", "She has a high level of English proficiency.", "hard"),
        ("vocabulary", "词汇，词汇量", "Reading widely helps expand your vocabulary.", "easy"),
        ("fluency", "流利度", "Practice speaking to improve your fluency.", "medium")
    ]

    cursor.executemany(insert_sql, batch_data)  # 批量执行
    connection.commit()
    print(f"批量插入成功！共插入 {cursor.rowcount} 条记录")

except Error as e:
    print(f"批量插入失败：{e}")
    connection.rollback()
finally:
    if connection.open:
        cursor.close()
        connection.close()
```

## 五、高级特性
### 1. 事务处理（ACID 特性）
PyMySQL 默认关闭自动提交（`autocommit=False`），需通过 `commit()` 提交事务，`rollback()` 回滚事务，确保数据一致性：
```python
import pymysql
from pymysql import Error

try:
    connection = pymysql.connect(
        host='localhost',
        user='root',
        password='your_password',
        database='english_db',
        charset='utf8mb4'
    )
    cursor = connection.cursor()

    # 事务：插入 + 更新 原子操作（要么都成功，要么都失败）
    # 步骤1：插入新单词
    insert_sql = "INSERT INTO vocabulary (word, definition) VALUES (%s, %s)"
    cursor.execute(insert_sql, ('diligent', '勤奋的，勤勉的'))
    
    # 步骤2：更新关联单词难度
    update_sql = "UPDATE vocabulary SET difficulty = 'medium' WHERE word = %s"
    cursor.execute(update_sql, ('diligent',))

    # 所有操作成功，提交事务
    connection.commit()
    print("事务执行成功！")

except Error as e:
    print(f"事务执行失败：{e}")
    # 出错时回滚所有操作，恢复到事务前状态
    connection.rollback()
finally:
    if connection.open:
        cursor.close()
        connection.close()
```

### 2. 上下文管理器（自动关闭连接）
使用 `with` 语句，无需手动关闭 `connection` 和 `cursor`，代码更简洁安全：
```python
import pymysql
from pymysql import Error

try:
    # 连接上下文：自动关闭连接
    with pymysql.connect(
        host='localhost',
        user='root',
        password='your_password',
        database='english_db',
        charset='utf8mb4',
        cursorclass=pymysql.cursors.DictCursor
    ) as connection:
        # 游标上下文：自动关闭游标
        with connection.cursor() as cursor:
            query_sql = "SELECT word FROM vocabulary WHERE difficulty = 'hard'"
            cursor.execute(query_sql)
            hard_words = cursor.fetchall()
            print("难词列表：", [word['word'] for word in hard_words])

except Error as e:
    print(f"操作失败：{e}")
```

### 3. 分页查询（处理大数据集）
避免一次性加载大量数据，使用 `LIMIT + OFFSET` 实现分页，减轻数据库压力：
```python
import pymysql
from pymysql import Error

def get_words_by_page(page=1, page_size=5):
    """
    分页查询单词
    :param page: 页码（默认第1页）
    :param page_size: 每页条数（默认5条）
    :return: 分页数据列表
    """
    offset = (page - 1) * page_size  # 计算偏移量
    try:
        with pymysql.connect(
            host='localhost',
            user='root',
            password='your_password',
            database='english_db',
            charset='utf8mb4',
            cursorclass=pymysql.cursors.DictCursor
        ) as connection:
            with connection.cursor() as cursor:
                query_sql = """
                SELECT id, word, definition, difficulty 
                FROM vocabulary 
                ORDER BY create_time DESC 
                LIMIT %s OFFSET %s
                """
                cursor.execute(query_sql, (page_size, offset))
                return cursor.fetchall()
    except Error as e:
        print(f"分页查询失败：{e}")
        return []

# 测试：获取第2页数据（每页5条）
page_2_data = get_words_by_page(page=2, page_size=5)
print(f"第2页单词（共 {len(page_2_data)} 条）：")
for item in page_2_data:
    print(f"ID: {item['id']}, 单词: {item['word']}, 难度: {item['difficulty']}")
```

### 4.  SSL 加密连接（安全场景）
若数据库部署在远程服务器，可通过 SSL 加密连接保障数据传输安全：
```python
import pymysql

# 需提前准备 SSL 证书文件（ca.pem、cert.pem、key.pem）
connection = pymysql.connect(
    host='remote-mysql-server.com',
    user='remote_user',
    password='your_password',
    database='english_db',
    charset='utf8mb4',
    ssl={
        'ca': 'ca.pem',    # CA 证书路径
        'cert': 'cert.pem',# 客户端证书路径
        'key': 'key.pem'   # 客户端密钥路径
    }
)
```

## 六、常见问题与解决方案
### 1. 连接失败：`Access denied for user 'root'@'localhost'`
- 检查用户名/密码是否正确（注意大小写敏感）
- 确认 MySQL 服务已启动（Windows：服务面板；Linux：`systemctl status mysql`）
- 若远程连接，需授权用户访问：
  ```sql
  GRANT ALL PRIVILEGES ON english_db.* TO 'root'@'%' IDENTIFIED BY 'your_password';
  FLUSH PRIVILEGES;  # 刷新权限
  ```

### 2. 中文乱码问题
- 连接时必须指定 `charset='utf8mb4'`（而非 `utf8`，MySQL 中 `utf8` 仅支持 3 字节字符）
- 数据库表字符集设置为 `utf8mb4`（创建表时指定 `DEFAULT CHARSET=utf8mb4`）
- 确保 Python 代码文件编码为 UTF-8（一般 IDE 默认配置）

### 3. SQL 注入风险（重点！）
- ❌ 危险写法：直接拼接字符串
  ```python
  word = "test'; DROP TABLE vocabulary; --"  # 恶意输入
  sql = f"SELECT * FROM vocabulary WHERE word = '{word}'"  # 注入风险！
  ```
- ✅ 安全写法：使用参数化查询（`%s` 占位符）
  ```python
  sql = "SELECT * FROM vocabulary WHERE word = %s"
  cursor.execute(sql, (word,))  # 自动过滤特殊字符
  ```

### 4. `fetchall()` 内存溢出
- 原因：查询结果集过大（如 10 万+ 条数据），一次性加载到内存
- 解决方案：使用分页查询（`LIMIT + OFFSET`）或迭代读取：
  ```python
  cursor.execute("SELECT word FROM vocabulary")
  while True:
      row = cursor.fetchone()
      if not row:
          break
      print(row[0])  # 逐行处理，占用内存小
  ```

### 5. 版本兼容性问题
- PyMySQL 1.0+ 仅支持 Python3.6+，若使用 Python3.5 及以下，需安装旧版本：
  ```bash
  pip install pymysql==0.10.1
  ```
- 支持 MySQL 5.5+、MariaDB 5.5+，若使用更旧版本，需升级数据库或降低 PyMySQL 版本

## 八、学习小结
PyMySQL 是 Python3 环境下操作 MySQL 的轻量高效工具，核心流程可总结为：  
**安装驱动 → 建立连接 → 创建游标 → 执行 SQL（参数化查询）→ 处理结果 → 提交/回滚事务 → 关闭连接**  

### 适用场景
- Python3 开发的 Web 应用（Django/Flask 项目常用）
- 数据采集与存储、批量数据处理
- 英语学习平台、博客系统等中小型项目
- 需兼容 MySQLdb 旧代码的迁移项目

### 进阶建议
- 大型项目建议使用 ORM 框架（如 SQLAlchemy、Django ORM），简化 SQL 编写
- 数据库配置信息（用户名、密码）不要硬编码，使用 `configparser` 或环境变量存储
- 学习 MySQL 索引优化（如给 `word` 字段加索引），提升查询效率
- 生产环境建议使用连接池（如 `DBUtils.PooledDB`），减少连接开销
