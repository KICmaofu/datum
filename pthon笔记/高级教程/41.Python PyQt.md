# Python PyQt 完全指南：跨平台 GUI 开发实战
PyQt 是 Qt 框架的 Python 绑定库（分为 PyQt5 和 PyQt6 两个主流版本），是 Python 中最强大、最成熟的 **跨平台 GUI 开发工具** 之一。它继承了 Qt 框架的全部功能：丰富的原生控件、强大的布局管理、灵活的信号与槽机制、支持 2D/3D 绘图、多媒体、网络编程等，同时保持 Python 简洁易用的特性，可快速开发出美观、高效、跨 Windows/macOS/Linux 的桌面应用。

与 Tkinter（Python 内置 GUI 库）相比，PyQt 控件更丰富、界面更美观、功能更强大；与 wxPython 相比，PyQt 的文档更完善、生态更活跃、跨平台兼容性更优；与 Electron（前端 GUI 框架）相比，PyQt 内存占用更低、运行速度更快，且原生支持系统级功能。


## 一、核心定位与优势
### 1. 核心特点
- **跨平台无缝兼容**：一次编写，多平台运行（Windows 7+、macOS 10.12+、Linux），无需修改代码，原生适配各系统外观（如 Windows 11 圆角、macOS 菜单栏）；
- **丰富的原生控件**：提供 100+ 内置控件（按钮、文本框、表格、树状图、滑块、日历等），支持自定义控件扩展；
- **信号与槽机制**：Qt 核心通信机制，解耦控件交互逻辑，无需回调函数嵌套，代码更清晰；
- **强大的布局管理**：支持弹性布局（QVBoxLayout、QHBoxLayout）、网格布局（QGridLayout）等，自动适配窗口缩放，避免固定坐标的混乱；
- **高可扩展性**：支持 Qt 设计师（Qt Designer）可视化开发，可与 C++ 混合编程，集成 OpenCV、Matplotlib 等库；
- **功能全面**：内置网络编程（HTTP、TCP/UDP）、数据库支持（SQLite、MySQL）、多媒体处理、2D/3D 绘图、打印、国际化等；
- **活跃的生态**：大量第三方组件（如 PyQtGraph 数据可视化、QWebEngineView 浏览器内核），社区文档丰富。

### 2. 适用场景
- **桌面应用开发**：办公软件（文本编辑器、表格工具）、工具类软件（批量处理工具、日志分析器）；
- **数据可视化工具**：科学计算结果展示、实时数据监控（如传感器数据面板）；
- **工业控制软件**：设备监控、参数配置、数据采集（结合串口/网络通信）；
- **教育类软件**：交互式教学工具、编程编辑器、模拟仿真软件；
- **多媒体应用**：视频播放器、音频编辑器、图片查看器；
- **嵌入式 GUI**：配合 Raspberry Pi 等开发板，开发嵌入式设备的控制界面。

### 3. 与其他 Python GUI 库的对比
| GUI 库         | 核心优势                          | 缺点                                  | 适用场景                  |
|----------------|-----------------------------------|---------------------------------------|---------------------------|
| PyQt（本文核心） | 跨平台、控件丰富、功能强大、信号槽机制 | 需安装（非内置）、商业使用需授权（PyQt6 商业授权费用较高） | 复杂桌面应用、工业软件、数据可视化工具 |
| Tkinter        | 内置无依赖、轻量、入门简单        | 控件简陋、界面美观度低、功能有限      | 简单工具、小型脚本、入门学习 |
| wxPython       | 原生系统外观、开源免费（无商业限制） | 文档较少、布局管理不够灵活            | 中型桌面应用、开源项目    |
| Kivy           | 支持移动平台（iOS/Android）        | 界面风格非原生、性能略低              | 跨移动+桌面的简易应用     |
| Electron（前端） | 前端技术栈（HTML/CSS/JS）、生态庞大 | 内存占用高、启动慢、打包体积大        | 网页化桌面应用、快速原型开发 |

**结论**：追求功能强大、界面美观、跨平台兼容性的桌面应用，优先选 PyQt；简单工具或入门学习用 Tkinter；需支持移动平台用 Kivy；熟悉前端技术栈可选 Electron。


## 二、环境准备

PyQt 分为两个主流版本，核心功能一致，但 API 有部分差异，推荐根据需求选择：
| 特性                | PyQt5                          | PyQt6                          |
|---------------------|--------------------------------|--------------------------------|
| 稳定性              | 非常稳定，生态成熟             | 稳定，较新（2020 年发布）      |

| 推荐场景            | 企业级应用、需兼容旧系统       | 新开发项目、追求现代 API 设计   |

### 3. 验证安装
编写一个简单的 Hello World 程序，验证 PyQt 是否正常工作：
```python
# PyQt6 示例（PyQt5 只需将 import 中的 6 改为 5）
from PyQt6.QtWidgets import QApplication, QWidget, QLabel, QVBoxLayout
import sys

# 1. 创建应用程序对象（每个 PyQt 程序必须有且仅有一个 QApplication）

# 2. 创建主窗口
window = QWidget()
window.setWindowTitle("PyQt 入门示例")

# 4. 设置布局（垂直布局）
layout = QVBoxLayout()
layout.addWidget(label)

# 6. 运行应用程序主循环（监听事件，如点击、关闭）

```

运行后会弹出一个窗口，显示 "Hello, PyQt!"，说明安装成功。


## 三、核心概念：理解 PyQt 开发模型
PyQt 开发的核心是「应用程序对象 + 控件 + 布局 + 信号与槽」，需先掌握以下关键概念：

### 1. 核心概念解析
| 概念                | 中文名称          | 核心作用                                  | 通俗理解                          |
|---------------------|-------------------|-------------------------------------------|-----------------------------------|
| **QApplication**    | 应用程序对象      | 管理应用程序的事件循环、初始化资源、处理系统消息 | “应用程序的大脑”：负责接收用户操作（如点击、键盘输入）并分发到对应控件 |
| **QWidget**         | 基础控件类        | 所有可视化控件的父类（窗口、按钮、文本框等都继承自 QWidget） | “控件的祖先”：提供控件的基础属性（大小、位置、显示/隐藏） |
| **信号（Signal）**  | 事件触发信号      | 控件触发特定事件时发出的通知（如按钮点击、文本框内容变化） | “警报”：按钮被点击时发出“点击信号”，文本框内容变时发出“文本变化信号” |
| **槽（Slot）**      | 信号响应函数      | 接收信号并执行相应逻辑的函数（如点击按钮后执行的函数） | “响应动作”：收到“点击信号”后，执行“打开文件”的槽函数 |
| **布局管理器**      | 控件排列工具      | 自动管理控件在窗口中的位置和大小，支持窗口缩放 | “自动排版工具”：无需手动设置控件坐标，自动调整控件位置以适应窗口大小 |
| **窗口生命周期**    | 窗口从创建到关闭的流程 | 初始化（__init__）→ 显示（show()）→ 事件循环（exec()）→ 关闭（close()） | “窗口的一生”：创建后显示，等待用户操作，最后关闭释放资源 |

### 2. 开发流程三步曲
1. **创建应用程序**：实例化 `QApplication`，管理全局资源和事件循环；
2. **构建 UI 界面**：创建窗口（`QWidget`/`QMainWindow`）、控件（按钮、文本框等），通过布局管理器排列控件；
3. **绑定交互逻辑**：将控件的信号（如按钮点击）与槽函数（如处理点击的函数）绑定，实现用户操作响应；
4. **启动事件循环**：调用 `app.exec()`，让应用程序进入循环，监听并处理用户事件（直到窗口关闭）。


## 四、核心功能：分场景详解（语法+示例）
### 1. 基础控件与布局管理（UI 构建核心）
PyQt 提供丰富的基础控件，配合布局管理器可快速构建美观、自适应的界面。

#### （1）常用基础控件
| 控件类名          | 功能描述                  | 关键方法/属性                          |
|-------------------|---------------------------|---------------------------------------|
| `QLabel`          | 显示文本/图片             | `setText(text)`、`setPixmap(pixmap)`  |
| `QPushButton`     | 按钮（支持点击事件）      | `setText(text)`、`clicked` 信号        |
| `QLineEdit`       | 单行文本输入框            | `setText(text)`、`text()`、`textChanged` 信号 |
| `QTextEdit`       | 多行文本输入/显示框       | `setPlainText(text)`、`toPlainText()`  |
| `QCheckBox`       | 复选框                    | `setText(text)`、`isChecked()`、`stateChanged` 信号 |
| `QRadioButton`    | 单选按钮（互斥）          | 需配合 `QButtonGroup` 使用，`toggled` 信号 |
| `QComboBox`       | 下拉选择框                | `addItem(text)`、`currentText()`  |
| `QSlider`         | 滑动条（调节数值）        | `setRange(min, max)`、`valueChanged` 信号 |

#### （2）布局管理器（推荐优先使用）
避免手动设置控件坐标（绝对布局），推荐使用布局管理器实现自适应：
- `QVBoxLayout`：垂直布局（控件从上到下排列）；
- `QHBoxLayout`：水平布局（控件从左到右排列）；
- `QGridLayout`：网格布局（控件按行/列排列，支持跨行列）；
- `QFormLayout`：表单布局（标签+输入框成对排列）。

**示例：用布局管理器构建表单界面**
```python
from PyQt6.QtWidgets import (QApplication, QWidget, QLabel, QLineEdit,
                             QPushButton, QVBoxLayout, QHBoxLayout, QFormLayout)
import sys

class FormWindow(QWidget):
    def __init__(self):
        super().__init__()
        self.init_ui()

    def init_ui(self):
        # 设置窗口属性
        self.setWindowTitle("表单布局示例")
        self.resize(400, 300)

        # 1. 创建表单布局（标签+输入框成对排列）
        form_layout = QFormLayout()

        # 添加表单项（标签文本，输入控件）
        self.name_edit = QLineEdit()
        self.age_edit = QLineEdit()
        self.email_edit = QLineEdit()

        form_layout.addRow("姓名：", self.name_edit)
        form_layout.addRow("年龄：", self.age_edit)
        form_layout.addRow("邮箱：", self.email_edit)

        # 2. 创建按钮布局（水平排列两个按钮）
        btn_layout = QHBoxLayout()
        self.submit_btn = QPushButton("提交")
        self.clear_btn = QPushButton("清空")
        btn_layout.addWidget(self.submit_btn)
        btn_layout.addWidget(self.clear_btn)

        # 3. 主布局（垂直排列表单布局和按钮布局）
        main_layout = QVBoxLayout()
        main_layout.addLayout(form_layout)
        main_layout.addLayout(btn_layout)

        # 设置主布局
        self.setLayout(main_layout)

        # 绑定信号与槽（后续讲解）
        self.submit_btn.clicked.connect(self.on_submit)
        self.clear_btn.clicked.connect(self.on_clear)

    def on_submit(self):
        # 获取输入内容
        name = self.name_edit.text()
        age = self.age_edit.text()
        email = self.email_edit.text()
        # 弹出消息框（后续讲解）
        from PyQt6.QtWidgets import QMessageBox
        QMessageBox.information(self, "提交成功", f"姓名：{name}\n年龄：{age}\n邮箱：{email}")

    def on_clear(self):
        # 清空输入框
        self.name_edit.clear()
        self.age_edit.clear()
        self.email_edit.clear()

if __name__ == "__main__":
    app = QApplication(sys.argv)
    window = FormWindow()
    window.show()
    sys.exit(app.exec())
```

### 2. 信号与槽：控件交互核心机制
信号与槽是 PyQt 最核心的特性，用于实现控件间的通信，无需回调函数嵌套，逻辑更清晰。

#### （1）信号与槽的绑定方式
##### 方式 1：直接绑定内置信号与自定义槽函数（最常用）
```python
from PyQt6.QtWidgets import QApplication, QWidget, QPushButton, QVBoxLayout
import sys

class SignalSlotDemo(QWidget):
    def __init__(self):
        super().__init__()
        self.init_ui()

    def init_ui(self):
        self.setWindowTitle("信号与槽示例")
        self.resize(300, 200)

        # 创建按钮
        self.btn1 = QPushButton("点击触发槽函数")
        self.btn2 = QPushButton("禁用按钮")
        self.btn3 = QPushButton("退出应用")

        # 布局
        layout = QVBoxLayout()
        layout.addWidget(self.btn1)
        layout.addWidget(self.btn2)
        layout.addWidget(self.btn3)
        self.setLayout(layout)

        # 绑定信号与槽
        self.btn1.clicked.connect(self.on_btn1_click)  # 按钮点击信号 → 自定义槽函数
        self.btn2.clicked.connect(self.btn1.setEnabled)  # 按钮点击信号 → 内置槽函数（设置启用状态）
        self.btn3.clicked.connect(QApplication.instance().quit)  # 按钮点击信号 → 应用退出

    def on_btn1_click(self):
        """自定义槽函数：按钮 1 点击后执行"""
        print("按钮 1 被点击了！")

if __name__ == "__main__":
    app = QApplication(sys.argv)
    window = SignalSlotDemo()
    window.show()
    sys.exit(app.exec())
```

##### 方式 2：用 `@pyqtSlot` 装饰器（明确指定槽函数参数类型）
```python
from PyQt6.QtCore import pyqtSlot
from PyQt6.QtWidgets import QApplication, QWidget, QSlider, QLabel, QVBoxLayout
import sys

class SliderDemo(QWidget):
    def __init__(self):
        super().__init__()
        self.init_ui()

    def init_ui(self):
        self.setWindowTitle("Slider 信号与槽")
        self.resize(300, 150)

        # 滑动条（0-100）
        self.slider = QSlider()
        self.slider.setRange(0, 100)
        self.slider.setValue(50)

        # 标签（显示滑动条数值）
        self.label = QLabel(f"当前值：50")
        self.label.setStyleSheet("font-size: 18px; text-align: center;")

        # 布局
        layout = QVBoxLayout()
        layout.addWidget(self.slider)
        layout.addWidget(self.label)
        self.setLayout(layout)

        # 绑定滑动条数值变化信号（带 int 类型参数）
        self.slider.valueChanged.connect(self.on_slider_change)

    @pyqtSlot(int)  # 明确槽函数接收 int 类型参数
    def on_slider_change(self, value):
        """滑动条数值变化时更新标签"""
        self.label.setText(f"当前值：{value}")

if __name__ == "__main__":
    app = QApplication(sys.argv)
    window = SliderDemo()
    window.show()
    sys.exit(app.exec())
```

##### 方式 3：自定义信号（`pyqtSignal`）
当内置信号满足不了需求时，可自定义信号，实现控件间的自定义通信：
```python
from PyQt6.QtCore import pyqtSignal, QObject
from PyQt6.QtWidgets import QApplication, QWidget, QPushButton, QVBoxLayout, QLabel
import sys

# 自定义信号类（需继承 QObject）
class MySignal(QObject):
    # 定义自定义信号（可指定参数类型）
    custom_signal = pyqtSignal(str)  # 带字符串参数的信号
    num_signal = pyqtSignal(int, int)  # 带两个 int 参数的信号

class CustomSignalDemo(QWidget):
    def __init__(self):
        super().__init__()
        self.init_ui()

    def init_ui(self):
        self.setWindowTitle("自定义信号示例")
        self.resize(300, 200)

        # 创建自定义信号实例
        self.my_signal = MySignal()

        # 绑定自定义信号与槽函数
        self.my_signal.custom_signal.connect(self.on_custom_signal)
        self.my_signal.num_signal.connect(self.on_num_signal)

        # 按钮
        self.btn = QPushButton("触发自定义信号")
        self.label = QLabel("等待信号触发...")

        # 布局
        layout = QVBoxLayout()
        layout.addWidget(self.btn)
        layout.addWidget(self.label)
        self.setLayout(layout)

        # 绑定按钮点击信号，触发自定义信号
        self.btn.clicked.connect(self.trigger_custom_signal)

    def trigger_custom_signal(self):
        """触发自定义信号"""
        self.my_signal.custom_signal.emit("自定义信号触发成功！")  # 发送字符串参数
        self.my_signal.num_signal.emit(10, 20)  # 发送两个 int 参数

    def on_custom_signal(self, msg):
        """处理自定义信号（字符串参数）"""
        self.label.setText(msg)

    def on_num_signal(self, a, b):
        """处理自定义信号（两个 int 参数）"""
        print(f"自定义数字信号：a={a}, b={b}，和为 {a+b}")

if __name__ == "__main__":
    app = QApplication(sys.argv)
    window = CustomSignalDemo()
    window.show()
    sys.exit(app.exec())
```

### 3. 对话框（`QDialog`）：交互弹窗
对话框用于临时交互（如提示信息、确认操作、输入数据、选择文件等），PyQt 提供多种内置对话框：

#### （1）消息对话框（`QMessageBox`）
用于显示提示、警告、错误、确认等信息：
```python
from PyQt6.QtWidgets import (QApplication, QWidget, QPushButton, QVBoxLayout,
                             QMessageBox)
import sys

class MessageBoxDemo(QWidget):
    def __init__(self):
        super().__init__()
        self.init_ui()

    def init_ui(self):
        self.setWindowTitle("消息对话框示例")
        self.resize(300, 200)

        # 按钮
        self.info_btn = QPushButton("信息提示")
        self.warning_btn = QPushButton("警告提示")
        self.question_btn = QPushButton("确认对话框")
        self.error_btn = QPushButton("错误提示")

        # 布局
        layout = QVBoxLayout()
        layout.addWidget(self.info_btn)
        layout.addWidget(self.warning_btn)
        layout.addWidget(self.question_btn)
        layout.addWidget(self.error_btn)
        self.setLayout(layout)

        # 绑定信号
        self.info_btn.clicked.connect(self.show_info)
        self.warning_btn.clicked.connect(self.show_warning)
        self.question_btn.clicked.connect(self.show_question)
        self.error_btn.clicked.connect(self.show_error)

    def show_info(self):
        QMessageBox.information(self, "信息", "操作成功！", QMessageBox.StandardButton.Ok)

    def show_warning(self):
        QMessageBox.warning(self, "警告", "数据可能丢失！", QMessageBox.StandardButton.Cancel)

    def show_question(self):
        # 确认对话框（返回用户选择的按钮）
        reply = QMessageBox.question(
            self, "确认", "是否删除该文件？",
            QMessageBox.StandardButton.Yes | QMessageBox.StandardButton.No
        )
        if reply == QMessageBox.StandardButton.Yes:
            print("用户选择：删除")
        else:
            print("用户选择：取消")

    def show_error(self):
        QMessageBox.critical(self, "错误", "文件读取失败！", QMessageBox.StandardButton.Close)
```

### 7. 多线程：避免 UI 卡顿（核心避坑点）
PyQt 中，UI 操作必须在 **主线程（UI 线程）** 中执行，若在主线程中执行耗时操作（如文件下载、网络请求、数据计算），会导致 UI 卡顿（无法响应用户操作）。因此，耗时操作需放在子线程中，通过信号槽机制与主线程通信（更新 UI）。

#### 示例：子线程执行耗时操作，主线程更新进度条
```python
from PyQt6.QtCore import QThread, pyqtSignal
from PyQt6.QtWidgets import (QApplication, QWidget, QProgressBar, QPushButton,
                             QVBoxLayout, QLabel)
import sys
import time

# 子线程类（继承 QThread）
class WorkerThread(QThread):
    # 自定义信号：更新进度（int 类型参数）
    progress_signal = pyqtSignal(int)
    # 自定义信号：任务完成
    finish_signal = pyqtSignal()

    def run(self):
        """子线程执行的耗时操作（重写 run 方法）"""
        for i in range(1, 101):
            time.sleep(0.05)  # 模拟耗时操作
            self.progress_signal.emit(i)  # 发送进度信号
        self.finish_signal.emit()  # 发送完成信号

class ThreadDemo(QWidget):
    def __init__(self):
        super().__init__()
        self.init_ui()

    def init_ui(self):
        self.setWindowTitle("多线程进度条示例")
        self.resize(400, 200)

        # 控件
        self.label = QLabel("进度：0%")
        self.progress_bar = QProgressBar()
        self.progress_bar.setRange(0, 100)
        self.start_btn = QPushButton("开始任务")
        self.start_btn.clicked.connect(self.start_task)

        # 布局
        layout = QVBoxLayout()
        layout.addWidget(self.label)
        layout.addWidget(self.progress_bar)
        layout.addWidget(self.start_btn)
        self.setLayout(layout)

        # 禁用按钮重复点击
        self.start_btn.setEnabled(True)

    def start_task(self):
        """启动子线程执行任务"""
        self.start_btn.setEnabled(False)
        self.label.setText("进度：0%")

        # 创建子线程实例
        self.worker_thread = WorkerThread()
        # 绑定子线程信号与主线程槽函数
        self.worker_thread.progress_signal.connect(self.update_progress)
        self.worker_thread.finish_signal.connect(self.task_finish)

        # 启动子线程（调用 run 方法）
        self.worker_thread.start()

    def update_progress(self, value):
        """更新进度条和标签（主线程执行）"""
        self.progress_bar.setValue(value)
        self.label.setText(f"进度：{value}%")

    def task_finish(self):
        """任务完成后启用按钮（主线程执行）"""
        self.label.setText("任务完成！")
        self.start_btn.setEnabled(True)

if __name__ == "__main__":
    app = QApplication(sys.argv)
    window = ThreadDemo()
    window.show()
    sys.exit(app.exec())
```

### 8. 结合 asyncio：异步操作 UI
PyQt 可与 `asyncio` 结合，实现异步 I/O 操作（如网络请求），避免 UI 卡顿，无需手动管理线程：
```python
from PyQt6.QtWidgets import (QApplication, QWidget, QTextEdit, QPushButton,
                             QVBoxLayout)
from PyQt6.QtCore import QObject, pyqtSlot
import asyncio
import aiohttp
import sys

# 异步任务管理器（桥接 asyncio 和 PyQt）
class AsyncManager(QObject):
    def __init__(self, parent=None):
        super().__init__(parent)
        self.loop = asyncio.get_event_loop()

    async def fetch_url(self, url, text_edit):
        """异步爬取 URL 内容"""
        try:
            async with aiohttp.ClientSession() as session:
                async with session.get(url, timeout=10) as response:
                    content = await response.text()[:500]  # 只取前 500 字符
                    text_edit.append(f"\n=== 爬取 {url} 成功 ===")
                    text_edit.append(content)
        except Exception as e:
            text_edit.append(f"\n=== 爬取 {url} 失败 ===")
            text_edit.append(f"错误：{str(e)}")

    @pyqtSlot(str, QTextEdit)
    def run_async_task(self, url, text_edit):
        """槽函数：启动异步任务（主线程调用）"""
        self.loop.create_task(self.fetch_url(url, text_edit))

class AsyncQtDemo(QWidget):
    def __init__(self):
        super().__init__()
        self.init_ui()

    def init_ui(self):
        self.setWindowTitle("PyQt + asyncio 异步示例")
        self.resize(600, 400)

        # 控件
        self.text_edit = QTextEdit()
        self.text_edit.setPlaceholderText("爬取结果将显示在这里...")
        self.btn1 = QPushButton("爬取百度")
        self.btn2 = QPushButton("爬取 Python 官网")

        # 布局
        layout = QVBoxLayout()
        layout.addWidget(self.text_edit)
        layout.addWidget(self.btn1)
        layout.addWidget(self.btn2)
        self.setLayout(layout)

        # 创建异步管理器
        self.async_manager = AsyncManager()

        # 绑定信号
        self.btn1.clicked.connect(
            lambda: self.async_manager.run_async_task("https://www.baidu.com", self.text_edit)
        )
        self.btn2.clicked.connect(
            lambda: self.async_manager.run_async_task("https://www.python.org", self.text_edit)
        )

if __name__ == "__main__":
    app = QApplication(sys.argv)
    window = AsyncQtDemo()
    window.show()
    # 启动 asyncio 事件循环和 PyQt 事件循环
    loop = asyncio.get_event_loop()
    sys.exit(loop.run_until_complete(app.exec()))
```

```python
            content = self.centralWidget().toPlainText()
            with open(self.current_file, "w", encoding="utf-8") as f:
                f.write(content)
            self.statusBar().showMessage(f"已保存：{self.current_file}")
        else:
            self.save_as_file()

    def save_as_file(self):
        """另存为文件"""
        file_path, _ = QFileDialog.getSaveFileName(self, "另存为", "", "文本文件 (*.txt)")
        if file_path:
            self.current_file = file_path
            self.save_file()
            self.setWindowTitle(f"{os.path.basename(file_path)} - 文本编辑器")

        """选择字体"""
        current_font = self.centralWidget().font()
        font, ok = QFontDialog.getFont(current_font, self, "选择字体")

            self.centralWidget().setFont(font)

        if color.isValid():

            )
            if reply == QMessageBox.StandardButton.Yes:
                self.save_file()
                event.accept()
            elif reply == QMessageBox.StandardButton.No:
                event.accept()
            else:
                event.ignore()
        else:
            event.accept()

if __name__ == "__main__":
    app = QApplication(sys.argv)
```

### 场景 3：串口通信工具（工业控制场景）
结合 `pyserial` 实现串口通信，用于设备控制、数据采集：
```python
from PyQt6.QtWidgets import (QApplication, QMainWindow, QWidget, QVBoxLayout, QHBoxLayout,
                             QTextEdit, QPushButton, QComboBox, QLineEdit, QLabel, QMessageBox)
from PyQt6.QtCore import QThread, pyqtSignal, Qt
import serial
import serial.tools.list_ports
import sys
import time

# 串口通信子线程
class SerialThread(QThread):
    # 信号：接收数据
    recv_signal = pyqtSignal(str)
    # 信号：状态变化（连接/断开）
    status_signal = pyqtSignal(str)

    def __init__(self, port, baudrate=9600, timeout=0.1):
        super().__init__()
        self.port = port
        self.baudrate = baudrate
        self.timeout = timeout
        self.ser = None
        self.is_running = False

    def run(self):
        """子线程：监听串口数据"""
        try:
            self.ser = serial.Serial(
                port=self.port,
                baudrate=self.baudrate,
                timeout=self.timeout,
                parity=serial.PARITY_NONE,
                stopbits=serial.STOPBITS_ONE,
                bytesize=serial.EIGHTBITS
            )
            if self.ser.isOpen():
                self.status_signal.emit(f"已连接串口：{self.port}")
                self.is_running = True
                while self.is_running:
                    if self.ser.in_waiting > 0:
                        # 读取串口数据
                        data = self.ser.read_all().decode("utf-8", errors="ignore")
                        self.recv_signal.emit(f"[{time.strftime('%H:%M:%S')}] 接收：{data}")
                    time.sleep(0.05)
        except Exception as e:
            self.status_signal.emit(f"连接失败：{str(e)}")
        finally:
            if self.ser and self.ser.isOpen():
                self.ser.close()
            self.status_signal.emit("已断开连接")

    def stop(self):
        """停止串口通信"""
        self.is_running = False

    def send_data(self, data):
        """发送数据到串口"""
        if self.ser and self.ser.isOpen():
            try:
                self.ser.write(data.encode("utf-8"))
                return True
            except Exception as e:
                self.status_signal.emit(f"发送失败：{str(e)}")
                return False
        return False

class SerialTool(QMainWindow):
    def __init__(self):
        super().__init__()
        self.serial_thread = None
        self.init_ui()
        self.scan_ports()  # 扫描可用串口

    def init_ui(self):

        self.scan_btn.clicked.connect(self.scan_ports)

        # 波特率选择
        self.baud_label = QLabel("波特率：")
        self.baud_combo = QComboBox()
        self.baud_combo.addItems(["9600", "19200", "38400", "57600", "115200"])
        self.baud_combo.setCurrentText("9600")

        # 连接/断开按钮

        config_layout.addWidget(self.port_label)
        config_layout.addWidget(self.port_combo)
        config_layout.addWidget(self.scan_btn)
        config_layout.addSpacing(20)
        config_layout.addWidget(self.baud_label)
        config_layout.addWidget(self.baud_combo)
        config_layout.addSpacing(20)
        config_layout.addWidget(self.connect_btn)

        # 2. 数据收发区域
        self.recv_edit = QTextEdit()
        self.recv_edit.setReadOnly(True)
        self.recv_edit.setPlaceholderText("接收数据将显示在这里...")

        """扫描可用串口"""
        self.port_combo.clear()
        ports = serial.tools.list_ports.comports()
        if ports:
            for port in ports:
                self.port_combo.addItem(port.device)
        else:
            self.port_combo.addItem("无可用串口")

    def toggle_connect(self):

            self.serial_thread = SerialThread(port, baudrate)

            if self.serial_thread:

                self.serial_thread.wait()

            self.connect_btn.setText("连接")

        self.recv_edit.append(data)

        """更新状态栏"""
        self.statusBar().showMessage(status)

    def send_data(self):
        """发送数据"""
        data = self.send_edit.text().strip()
        if not data:
            QMessageBox.warning(self, "警告", "请输入要发送的数据！")
            return
        if self.serial_thread and self.serial_thread.is_running:
            if self.serial_thread.send_data(data):
                self.recv_edit.append(f"[{time.strftime('%H:%M:%S')}] 发送：{data}")
                self.send_edit.clear()
        else:
            QMessageBox.warning(self, "警告", "请先连接串口！")

    def closeEvent(self, event):
        """关闭窗口时停止串口线程"""
        if self.serial_thread:
            self.serial_thread.stop()
            self.serial_thread.wait()
        event.accept()

    tool.show()
    sys.exit(app.exec())
```

## 六、常见问题与避坑指南
### 1. 坑 1：在子线程中直接操作 UI（最致命错误）
- **问题**：在 `QThread` 或 `threading.Thread` 中直接修改控件属性（如 `label.setText()`、`progressBar.setValue()`），导致程序崩溃或 UI 异常；
- **原因**：PyQt 的 UI 控件不是线程安全的，仅允许主线程（UI 线程）操作；
- **解决方案**：子线程通过 **信号槽机制** 发送数据，主线程的槽函数更新 UI（示例见“多线程”部分）；

### 2. 坑 2：布局管理混乱（窗口缩放时控件错位）
- **问题**：使用绝对布局（手动设置 `widget.move(x, y)`、`widget.resize(w, h)`），窗口缩放时控件位置不变，导致界面错乱；
- **解决方案**：优先使用布局管理器（`QVBoxLayout`/`QHBoxLayout`/`QGridLayout`），通过 `setContentsMargins()`、`setSpacing()` 调整间距，控件会自动适应窗口大小；

### 3. 坑 3：信号槽绑定错误（无响应或报错）
- **问题**：信号与槽绑定后，触发信号时槽函数不执行，或报错 `TypeError: argument mismatch`；
- **常见原因**：
  1. 信号与槽的参数数量/类型不匹配（如带参数的信号绑定无参数的槽函数）；
  2. 槽函数未定义或名称错误；
  3. 控件未正确初始化就绑定信号；
- **解决方案**：
  1. 用 `@pyqtSlot` 装饰器明确槽函数参数类型；
  2. 检查信号与槽的参数数量一致（如 `valueChanged(int)` 信号绑定 `def on_value_change(self, value)` 槽函数）；
  3. 确保控件初始化后再绑定信号；

### 4. 坑 4：应用程序退出后进程未终止
- **问题**：关闭窗口后，Python 进程仍在后台运行（尤其使用多线程时）；
- **原因**：子线程未正确停止（如 `QThread` 未调用 `wait()`，或子线程进入无限循环）；
- **解决方案**：
  1. 重写 `closeEvent()` 方法，在窗口关闭时停止子线程并等待其结束；
  2. 子线程中通过 `is_running` 标志控制循环，避免无限循环；

### 5. 坑 5：PyQt5 与 PyQt6 API 混淆
- **问题**：将 PyQt5 的代码直接运行在 PyQt6 环境中，报错（如 `QtCore.Qt.AlignCenter` 变为 `QtCore.Qt.AlignmentFlag.AlignCenter`）；
- **主要 API 差异**：
  | PyQt5                          | PyQt6                          |
  |--------------------------------|--------------------------------|
  | `Qt.AlignCenter`               | `Qt.AlignmentFlag.AlignCenter` |
  | `QMessageBox.Ok`               | `QMessageBox.StandardButton.Ok` |
  | `QFileDialog.getOpenFileName()` 返回 (path, filter) | 同 PyQt5（兼容） |
  | `signal.connect(slot)`         | 同 PyQt5（兼容） |
- **解决方案**：明确版本，根据版本调整 API，或使用兼容性代码（如 `from PyQt6.QtCore import Qt; AlignCenter = Qt.AlignmentFlag.AlignCenter`）；

### 6. 坑 6：QSS 样式不生效
- **问题**：设置 QSS 后，控件样式无变化；
- **常见原因**：
  1. QSS 语法错误（如括号不匹配、属性名错误）；
  2. 控件被父布局覆盖，样式无法生效；
  3. 样式优先级问题（后续设置的样式覆盖前面的）；
- **解决方案**：
  1. 检查 QSS 语法（可分段注释排查错误）；
  2. 确保控件添加到布局后再设置样式；
  3. 给控件设置 `objectName`，通过 `#objectName` 精准匹配样式（如 `#login_btn { background-color: red; }`）；

### 7. 坑 7：打包后程序无法运行（PyInstaller）
- **问题**：用 PyInstaller 打包后，运行程序报错（如“缺少 Qt 库”“无法找到插件”）；
- **解决方案**：
  1. 打包命令添加 `--hidden-import PyQt6.QtWidgets`（确保导入所有必要模块）；
  2. 对于 PyQt6，需指定 Qt 插件路径（如 `--paths C:\Python\Lib\site-packages\PyQt6\Qt6\plugins`）；
  3. 打包时添加 `--windowed`（无控制台）或 `--console`（有控制台，便于调试）；
  4. 避免使用相对路径，用 `os.path.abspath()` 处理文件路径；

### 8. 坑 8：耗时操作导致 UI 卡顿
- **问题**：在主线程中执行耗时操作（如文件读写、网络请求、循环计算），导致 UI 无响应
