# Python re 模块完全指南：正则表达式实战应用
re 是 Python 内置的 **正则表达式处理标准库**，核心功能是提供字符串的高效匹配、查找、替换、分割与提取能力——通过定义“模式字符串”描述目标字符串的结构，实现对复杂文本的精准处理，支持贪婪/非贪婪匹配、分组捕获、零宽断言等高级特性，是文本处理（如数据提取、格式验证、日志分析）的核心工具。

与字符串内置方法（如 `str.find()`、`str.replace()`）相比，re 模块支持复杂模式匹配（如模糊匹配、多规则匹配），灵活性更强；与第三方正则库（如 `regex`）相比，它无需额外安装，原生集成 Python 生态，性能稳定，能满足绝大多数开发场景的需求。


## 一、核心定位与优势
### 1. 核心特点
- **内置无依赖**：Python 标准库自带，无需额外安装，导入即使用；
- **功能强大**：支持正则表达式全特性（元字符、量词、分组、断言、转义等）；
- **高效灵活**：基于底层优化，匹配速度快，可处理复杂文本规则（如邮箱、手机号、URL 验证）；
- **多场景适配**：支持匹配、查找、替换、分割、提取等全流程文本处理；
- **支持 Unicode**：兼容多语言字符（如中文、特殊符号），Python 3 原生支持 Unicode 编码；
- **可复用性**：通过 `re.compile()` 编译正则表达式，支持重复调用，提升性能。

### 2. 适用场景
- **格式验证**：验证手机号、邮箱、身份证号、密码强度、日期格式等；
- **数据提取**：从日志、HTML/XML、JSON、CSV 中提取目标信息（如 IP 地址、URL、关键词）；
- **字符串替换**：批量替换敏感词、格式化文本（如手机号脱敏、日期格式统一）；
- **文本分割**：按复杂分隔符（如多个空格、逗号+空格、换行符）分割字符串；
- **日志分析**：从海量日志中筛选特定模式的记录（如错误日志、用户操作日志）；
- **爬虫数据解析**：提取 HTML 页面中的链接、图片地址、文本内容（辅助爬虫开发）。

### 3. 与字符串内置方法的对比
| 特性                | 字符串内置方法（`str.find()`/`str.replace()`） | re 模块                          |
|---------------------|-----------------------------------------------|---------------------------------|
| 匹配能力            | 仅支持精确匹配或简单子串查找                  | 支持复杂模式匹配（模糊、多规则） |
| 灵活性              | 低（需手动处理多规则、模糊场景）              | 高（通过正则模式描述任意规则）  |
| 性能                | 简单场景下高效                                | 复杂场景下更高效（编译后复用）  |
| 功能覆盖            | 仅支持基础查找、替换、分割                    | 支持匹配、查找、替换、分割、提取、验证全流程 |

**结论**：简单子串操作（如精确查找固定字符串）用字符串内置方法；复杂模式处理（如模糊匹配、多规则验证）用 re 模块。

## 三、核心概念：正则表达式基础与 re 模块架构
### 1. 正则表达式基础语法
正则表达式（Regular Expression）是一套描述字符串结构的“模式语言”，核心通过 **元字符** 组合定义匹配规则，常用元字符如下：

| 类别          | 元字符/语法                | 描述                                  | 示例                                  |
|---------------|---------------------------|---------------------------------------|---------------------------------------|
| **单个字符匹配** | `.`                       | 匹配任意单个字符（除换行符 `\n`）      | `a.b` 匹配 `acb`、`a1b`、`a#b`        |
|               | `[abc]`                   | 匹配括号内任意一个字符（字符类）      | `[0-9]` 匹配任意数字，`[a-zA-Z]` 匹配任意字母 |
|               | `[^abc]`                  | 匹配括号外任意一个字符（否定字符类）  | `[^0-9]` 匹配非数字字符               |
|               | `\d`                      | 匹配数字（等价于 `[0-9]`）             | `\d{3}` 匹配 3 位数字                 |
|               | `\D`                      | 匹配非数字（等价于 `[^0-9]`）          | `\D+` 匹配1个及以上非数字字符         |
|               | `\w`                      | 匹配字母、数字、下划线（等价于 `[a-zA-Z0-9_]`） | `\w+` 匹配单词（如 `user_name123`）   |
|               | `\W`                      | 匹配非字母、数字、下划线              | `\W` 匹配 `@`、`#`、` `（空格）等     |
|               | `\s`                      | 匹配空白字符（空格、制表符 `\t`、换行符 `\n` 等） | `a\sb` 匹配 `a b`、`a\tb`             |
|               | `\S`                      | 匹配非空白字符                        | `\S+` 匹配非空白字符串                |
| **量词（重复次数）** | `*`                       | 匹配前面的字符 0 次或多次（贪婪）      | `ab*` 匹配 `a`、`ab`、`abb`、`abbb`   |
|               | `+`                       | 匹配前面的字符 1 次或多次（贪婪）      | `ab+` 匹配 `ab`、`abb`、`abbb`        |
|               | `?`                       | 匹配前面的字符 0 次或 1 次（贪婪）     | `ab?` 匹配 `a`、`ab`                  |
|               | `{n}`                     | 匹配前面的字符恰好 n 次                | `a{3}` 匹配 `aaa`                     |
|               | `{n,}`                    | 匹配前面的字符至少 n 次（贪婪）        | `a{2,}` 匹配 `aa`、`aaa`、`aaaa`      |
|               | `{n,m}`                   | 匹配前面的字符 n~m 次（贪婪）          | `a{2,3}` 匹配 `aa`、`aaa`             |
|               | `*?`/`+?`/`??`/`{n,m}?`   | 非贪婪匹配（尽可能少匹配）             | `a.*?b` 匹配 `aab` 中的 `aab`（而非 `ab`） |
| **位置匹配**   | `^`                       | 匹配字符串开头（多行模式下匹配行开头） | `^abc` 匹配 `abc123`，不匹配 `xabc`   |
|               | `$`                       | 匹配字符串结尾（多行模式下匹配行结尾） | `abc$` 匹配 `123abc`，不匹配 `abcx`   |
|               | `\b`                      | 单词边界（匹配单词开头/结尾的空白）    | `\bhello\b` 匹配 `hello`，不匹配 `helloworld` |
|               | `\B`                      | 非单词边界                            | `\Bhello\B` 匹配 `xhelloy` 中的 `hello` |
| **分组与引用** | `(abc)`                   | 捕获分组（将匹配结果存入组，可通过索引引用） | `(a)(b)` 匹配 `ab`，组 1 为 `a`，组 2 为 `b` |
|               | `(?:abc)`                 | 非捕获分组（仅匹配，不存储结果）        | `(?:ab)+` 匹配 `abab`，不单独捕获 `ab` |
|               | `(?P<name>abc)`           | 命名分组（通过名称引用分组结果）        | `(?P<year>\d{4})` 匹配年份，可通过 `group("year")` 获取 |
|               | `\1`/`\2`                 | 反向引用（引用前面的分组结果）          | `(ab)\1` 匹配 `abab`（引用组 1 的 `ab`） |
| **逻辑与断言** | `|`                       | 逻辑或（匹配任意一个模式）             | `abc|def` 匹配 `abc` 或 `def`         |
|               | `(?=abc)`                 | 正向先行断言（后面必须是 abc，不占字符） | `a(?=b)` 匹配 `ab` 中的 `a`，不匹配 `ac` 中的 `a` |
|               | `(?!abc)`                 | 反向先行断言（后面不能是 abc，不占字符） | `a(?!b)` 匹配 `ac` 中的 `a`，不匹配 `ab` 中的 `a` |

### 2. re 模块核心函数分类
re 模块提供两类接口：**直接调用函数**（适用于单次匹配）和 **编译后调用**（适用于多次重复匹配，性能更优），核心函数如下：

| 功能类别      | 核心函数                          | 描述                                  |
|---------------|-----------------------------------|---------------------------------------|
| **匹配验证**   | `re.match(pattern, string)`       | 从字符串 **开头** 匹配，返回匹配对象或 None |
|               | `re.fullmatch(pattern, string)`   | 匹配 **整个字符串**（完全符合模式），返回匹配对象或 None |
| **查找提取**   | `re.search(pattern, string)`      | 在字符串 **任意位置** 查找第一个匹配，返回匹配对象或 None |
|               | `re.findall(pattern, string)`     | 查找所有匹配结果，返回列表（无分组则返回字符串列表，有分组则返回元组列表） |
|               | `re.finditer(pattern, string)`    | 查找所有匹配结果，返回迭代器（每个元素是匹配对象，支持分组提取） |
| **替换修改**   | `re.sub(pattern, repl, string)`   | 替换所有匹配的子串，返回修改后的字符串 |
|               | `re.subn(pattern, repl, string)`  | 替换所有匹配的子串，返回元组 `(修改后字符串, 替换次数)` |
| **分割字符串** | `re.split(pattern, string)`       | 按正则模式分割字符串，返回列表        |
| **编译正则**   | `re.compile(pattern)`             | 编译正则模式为正则对象，支持重复调用（提升性能） |
| **辅助功能**   | `re.escape(string)`               | 转义字符串中的所有元字符（如 `.`、`*`），用于精确匹配 |
|               | `re.purge()`                      | 清除正则缓存，释放内存                |

### 3. 匹配对象（Match Object）
`re.match()`、`re.search()`、`re.finditer()` 成功匹配后会返回 **匹配对象**，包含匹配的详细信息，核心方法/属性如下：
- `group(num=0)`：返回指定分组的匹配结果（`num=0` 表示整个匹配结果，`num=1` 表示第 1 个分组，命名分组可传名称）；
- `groups()`：返回所有捕获分组的匹配结果，以元组形式；
- `groupdict()`：返回所有命名分组的匹配结果，以字典形式（键为分组名，值为匹配结果）；
- `start()`/`end()`：返回匹配结果在原字符串中的起始/结束索引；
- `span()`：返回元组 `(start, end)`，表示匹配结果的位置。


## 四、核心功能：分场景详解（语法+示例）
### 1. 编译正则表达式（`re.compile()`）
对于需要重复使用的正则模式，建议先用 `re.compile()` 编译为正则对象，避免重复解析模式，提升性能：

```python
import re

# 编译正则模式（匹配手机号：11位数字，以13/14/15/17/18/19开头）
phone_pattern = re.compile(r"^1[345789]\d{9}$")

# 重复使用正则对象（无需再次解析模式）
phones = ["13812345678", "12345678901", "19987654321"]
for phone in phones:
    if phone_pattern.fullmatch(phone):
        print(f"{phone} 是合法手机号")
    else:
        print(f"{phone} 是非法手机号")

# 输出结果：
# 13812345678 是合法手机号
# 12345678901 是非法手机号
# 19987654321 是合法手机号
```

### 2. 匹配与验证（`match()`/`fullmatch()`）
- `match()`：从字符串 **开头** 匹配，即使只匹配部分字符串也返回成功（需结合 `$` 确保完全匹配）；
- `fullmatch()`：匹配 **整个字符串**，仅当字符串完全符合模式时返回成功（推荐用于格式验证）。

#### 示例 1：验证邮箱格式
```python
import re

# 邮箱正则模式（简化版：支持字母、数字、下划线，@后为域名）
email_pattern = re.compile(r"^[a-zA-Z0-9_.-]+@[a-zA-Z0-9-]+\.[a-zA-Z]{2,4}$")

emails = [
    "user@example.com",    # 合法
    "user.name+tag@example.co.cn",  # 复杂合法（简化模式不支持，需更复杂正则）
    "user@.com",           # 非法（域名开头为.）
    "user@example"         # 非法（无后缀）
]

for email in emails:
    if email_pattern.fullmatch(email):
        print(f"{email} → 合法邮箱")
    else:
        print(f"{email} → 非法邮箱")
```

#### 示例 2：匹配字符串开头（`match()`）
```python
import re

# 匹配以 "Hello" 开头的字符串
pattern = r"Hello"
text1 = "Hello World!"
text2 = "World Hello!"

# match() 仅匹配开头
result1 = re.match(pattern, text1)
result2 = re.match(pattern, text2)

print(result1.group() if result1 else "未匹配")  # 输出：Hello
print(result2.group() if result2 else "未匹配")  # 输出：未匹配
```

### 3. 查找与提取（`search()`/`findall()`/`finditer()`）
- `search()`：在字符串任意位置查找 **第一个** 匹配结果；
- `findall()`：查找 **所有** 匹配结果，返回列表（有分组则返回分组结果列表）；
- `finditer()`：查找 **所有** 匹配结果，返回迭代器（支持分组提取，内存更高效）。

#### 示例 1：提取日志中的 IP 地址
```python
import re

# 日志文本
log_text = """
2025-11-20 10:00:00 192.168.1.1 访问 /index
2025-11-20 10:01:00 10.0.0.1 访问 /login
2025-11-20 10:02:00 172.16.0.1 访问 /api
"""

# IP 正则模式（IPv4，简化版）
ip_pattern = re.compile(r"\b(?:\d{1,3}\.){3}\d{1,3}\b")

# 方法 1：findall() 提取所有 IP（返回列表）
ip_list = ip_pattern.findall(log_text)
print("提取的 IP 列表：", ip_list)  # 输出：['192.168.1.1', '10.0.0.1', '172.16.0.1']

# 方法 2：finditer() 提取（支持位置信息）
for match in ip_pattern.finditer(log_text):
    ip = match.group()
    position = match.span()
    print(f"IP: {ip}, 位置: {position}")
```

#### 示例 2：分组提取日期中的年、月、日
```python
import re

# 日期文本
date_text = "今天是 2025年11月20日，明天是 2025年11月21日"

# 正则模式（命名分组：year, month, day）
date_pattern = re.compile(r"(?P<year>\d{4})年(?P<month>\d{2})月(?P<day>\d{2})日")

# 方法 1：findall() 提取分组（返回元组列表）
date_groups = date_pattern.findall(date_text)
print("分组提取（元组）：", date_groups)  # 输出：[('2025', '11', '20'), ('2025', '11', '21')]

# 方法 2：finditer() 提取命名分组（返回字典）
for match in date_pattern.finditer(date_text):
    date_dict = match.groupdict()
    print("命名分组提取（字典）：", date_dict)
    # 输出：
    # {'year': '2025', 'month': '11', 'day': '20'}
    # {'year': '2025', 'month': '11', 'day': '21'}
```

### 4. 替换字符串（`re.sub()`/`re.subn()`）
- `re.sub(pattern, repl, string, count=0)`：`count=0` 表示替换所有匹配（默认），`count=N` 表示仅替换前 N 个；
- `repl` 可为字符串（支持分组引用，如 `\1` 引用第 1 个分组）或函数（动态处理替换内容）。

#### 示例 1：手机号脱敏（中间 4 位替换为 *）
```python
import re

# 正则模式（匹配手机号，分组捕获前3位、中间4位、后4位）
phone_pattern = re.compile(r"(1[345789])(\d{4})(\d{4})")

# 替换：中间4位改为 ****（引用分组 1 和 3）
phone = "13812345678"
desensitized_phone = re.sub(phone_pattern, r"\1****\3", phone)
print("脱敏后手机号：", desensitized_phone)  # 输出：138****5678

# 用 subn() 获取替换次数
phone_list = ["13812345678", "19987654321", "123456"]
for phone in phone_list:
    new_phone, count = re.subn(phone_pattern, r"\1****\3", phone)
    print(f"{phone} → {new_phone}（替换次数：{count}）")
```

#### 示例 2：动态替换（repl 为函数）
```python
import re

# 文本：将数字乘以 2
text = "苹果价格：10元，香蕉价格：20元，橙子价格：30元"

# 正则模式：匹配数字
num_pattern = re.compile(r"\d+")

# 替换函数：将匹配的数字转为整数后乘以 2
def double_num(match):
    num = int(match.group())
    return str(num * 2)

# 动态替换
new_text = re.sub(num_pattern, double_num, text)
print("替换后文本：", new_text)  # 输出：苹果价格：20元，香蕉价格：40元，橙子价格：60元
```

### 5. 分割字符串（`re.split()`）
按正则模式分割字符串，支持复杂分隔符（如多个空格、逗号+空格、换行符等）：

```python
import re

# 示例 1：按多个空格分割（忽略连续空格）
text1 = "Hello   World  Python"  # 多个连续空格
split1 = re.split(r"\s+", text1)
print("按多个空格分割：", split1)  # 输出：['Hello', 'World', 'Python']

# 示例 2：按逗号或分号分割（支持带空格）
text2 = "苹果, 香蕉; 橙子,葡萄"
split2 = re.split(r",\s*|\s*;", text2)
print("按逗号或分号分割：", split2)  # 输出：['苹果', '香蕉', '橙子', '葡萄']

# 示例 3：保留分割符（用分组）
text3 = "2025-11-20"
split3 = re.split(r"(-)", text3)  # 分组捕获分隔符 '-'
print("保留分割符：", split3)  # 输出：['2025', '-', '11', '-', '20']
```

### 6. 贪婪与非贪婪匹配
正则默认是 **贪婪匹配**（尽可能多匹配），在量词后加 `?` 可转为 **非贪婪匹配**（尽可能少匹配）：

```python
import re

text = "<div>标题1</div><div>标题2</div>"

# 示例 1：贪婪匹配（默认）
greedy_pattern = r"<div>.*</div>"  # .* 贪婪匹配所有字符
greedy_match = re.search(greedy_pattern, text)
print("贪婪匹配结果：", greedy_match.group())  # 输出：<div>标题1</div><div>标题2</div>（匹配整个字符串）

# 示例 2：非贪婪匹配（加 ?）
non_greedy_pattern = r"<div>.*?</div>"  # .*? 非贪婪匹配到第一个 </div>
non_greedy_match = re.search(non_greedy_pattern, text)
print("非贪婪匹配结果：", non_greedy_match.group())  # 输出：<div>标题1</div>（匹配第一个 div）

# 示例 3：findall() 非贪婪匹配所有结果
all_non_greedy = re.findall(non_greedy_pattern, text)
print("非贪婪匹配所有结果：", all_non_greedy)  # 输出：['<div>标题1</div>', '<div>标题2</div>']
```

### 7. 零宽断言（高级匹配）
零宽断言用于匹配“位置”而非字符（不占用匹配结果），常用于复杂场景（如“前面是 A 后面是 B 的字符”）：

#### 示例 1：正向先行断言（`(?=abc)`）—— 后面必须是 abc
```python
import re

# 匹配后面是 "元" 的数字（提取价格）
text = "苹果10元，香蕉20斤，橙子30元"
```

```python
    验证密码强度：

    - 包含小写字母（a-z）
```

```python
import re
from collections import Counter

def top_access_ips(log_file, top_n=3):
    """分析 Nginx 日志，统计访问量最高的前 N 个 IP"""
    # Nginx 日志格式示例：192.168.1.1 - - [20/Nov/2025:10:00:00 +0800] "GET / HTTP/1.1" 200 1234
    ip_pattern = re.compile(r"^\d+\.\d+\.\d+\.\d+")  # 匹配日志开头的 IP
    ip_counts = Counter()

    with open(log_file, "r", encoding="utf-8") as f:
        for line in f:
            match = ip_pattern.match(line)
            if match:
                ip = match.group()
                ip_counts[ip] += 1

    # 返回前 N 个访问量最高的 IP
    return ip_counts.most_common(top_n)
```

## 六、常见问题与避坑指南
### 1. 坑 1：贪婪匹配导致结果不符合预期
- **问题**：使用 `.*` 等贪婪量词时，匹配结果超出预期（如匹配整个字符串而非目标片段）；
- **解决方案**：将贪婪量词改为非贪婪量词（加 `?`），或用更精确的模式限制匹配范围：
  ```python
  # 错误：贪婪匹配导致匹配整个字符串
  text = "aabbaacc"
  pattern = r"a.*c"  # 匹配从第一个 a 到最后一个 c
  print(re.findall(pattern, text))  # 输出：['aabbaacc']

  # 正确：非贪婪匹配（a.*?c），匹配到第一个 c
  pattern = r"a.*?c"
  print(re.findall(pattern, text))  # 输出：['aabbaac']
  ```

### 2. 坑 2：元字符未转义，导致匹配失败
- **问题**：匹配包含元字符（如 `.`、`*`、`?`、`(`）的字符串时，未转义元字符，导致匹配失败；
- **原因**：元字符在正则中有特殊含义，需用 `\` 转义（或用 `re.escape()` 自动转义）；
- **解决方案**：
  ```python
  import re

  # 需求：匹配字符串 "www.example.com" 中的 ".com"
  text = "www.example.com"
  # 错误：. 是元字符，匹配任意字符（而非字面意义的 .）
  pattern1 = r".com"
  print(re.findall(pattern1, text))  # 输出：['xample.com']（匹配最后一个字符 + com）

  # 正确：转义 .（用 \.）
  pattern2 = r"\.com"
  print(re.findall(pattern2, text))  # 输出：['.com']

  # 推荐：用 re.escape() 自动转义所有元字符
  target = ".com"
  pattern3 = re.escape(target)
  print(re.findall(pattern3, text))  # 输出：['.com']
  ```

### 3. 坑 3：`match()` 与 `search()` 混淆
- **问题**：用 `match()` 查找字符串中间的子串，返回 `None`（误以为匹配失败）；
- **原因**：`match()` 仅从字符串 **开头** 匹配，`search()` 从任意位置匹配；
- **解决方案**：根据需求选择函数——验证字符串开头用 `match()`，查找任意位置用 `search()`：
  ```python
  import re

  text = "abc123def"
  pattern = r"123"

  # 错误：match() 从开头匹配，找不到 123
  print(re.match(pattern, text))  # 输出：None

  # 正确：search() 从任意位置匹配
  print(re.search(pattern, text).group())  # 输出：123
  ```

### 4. 坑 4：分组索引错误或命名分组引用失败
- **问题**：用 `group(1)` 引用分组时，抛出 `IndexError: no such group`，或命名分组引用返回 `None`；
- **原因**：分组索引从 1 开始（`group(0)` 是整个匹配结果），命名分组需确保分组名正确；
- **解决方案**：
  ```python
  import re

  text = "2025年11月20日"
  # 正确：2 个捕获分组（year, month-day）
  pattern = r"(?P<year>\d{4})年(\d{2}月\d{2}日)"
  match = re.search(pattern, text)

  print(match.group(0))  # 输出：2025年11月20日（整个匹配结果）
  print(match.group(1))  # 输出：2025（第 1 个分组）
  print(match.group("year"))  # 输出：2025（命名分组）
  print(match.group(2))  # 输出：11月20日（第 2 个分组）
  # print(match.group(3))  # 错误：无第 3 个分组，抛出 IndexError
  ```

### 5. 坑 5：忽略大小写/多行模式等标志位
- **问题**：匹配大小写不敏感的字符串（如 `HTML`/`html`）或多行文本时，匹配失败；
- **解决方案**：使用 re 模块的标志位（如 `re.IGNORECASE`、`re.MULTILINE`）：
  ```python
  import re

  # 示例 1：忽略大小写（re.IGNORECASE 或 re.I）
  text = "HTML html Html"
  pattern = r"html"
  print(re.findall(pattern, text, re.IGNORECASE))  # 输出：['HTML', 'html', 'Html']

  # 示例 2：多行模式（re.MULTILINE 或 re.M）—— ^ 匹配每行开头，$ 匹配每行结尾
  text = """line1: 123
  line2: 456
  line3: 789"""
  pattern = r"^line\d+: (\d+)"  # 匹配每行开头的 lineX: 数字
  print(re.findall(pattern, text, re.MULTILINE))  # 输出：['123', '456', '789']
  ```

### 6. 坑 6：正则表达式性能问题（复杂模式匹配缓慢）
- **问题**：复杂正则模式（如嵌套分组、大量量词）在处理长文本时，匹配速度极慢；
- **原因**：正则引擎的“回溯”机制导致冗余计算；
- **解决方案**：
  1. 简化正则模式（避免不必要的分组和量词）；
  2. 用 `re.compile()` 编译正则，复用对象；
  3. 避免嵌套量词（如 `(a+)*`）；
  4. 用“占有量词”（Python 不直接支持，可通过断言模拟）或优化模式逻辑。

### 7. 坑 7：`findall()` 分组导致结果格式异常
- **问题**：正则模式包含分组时，`findall()` 返回分组结果列表而非整个匹配结果；
- **原因**：`findall()` 优先返回分组结果（多个分组则返回元组列表）；
- **解决方案**：
  1. 无需分组时，移除多余分组；
  2. 需保留分组但要整个匹配结果时，用非捕获分组 `(?:...)`；
  ```python
  import re

  text = "a1b a2b a3b"
  # 情况 1：无分组 → 返回整个匹配结果
  pattern1 = r"a\d+b"
  print(re.findall(pattern1, text))  # 输出：['a1b', 'a2b', 'a3b']

  # 情况 2：有分组 → 返回分组结果
  pattern2 = r"a(\d+)b"
  print(re.findall(pattern2, text))  # 输出：['1', '2', '3']（仅返回分组 \d+ 的结果）

  # 情况 3：非捕获分组 → 返回整个匹配结果
  pattern3 = r"a(?:\d+)b"
  print(re.findall(pattern3, text))  # 输出：['a1b', 'a2b', 'a3b']
  ```

## 八、总结
re 模块是 Python 文本处理的“瑞士军刀”，核心价值在于 **复杂模式匹配能力强、灵活高效、原生无依赖**，覆盖格式验证、数据提取、文本修改等全场景需求。

### 学习路径建议
1. 基础入门：掌握正则表达式核心语法（元字符、量词、分组），理解 `match()`/`search()`/`findall()` 的区别；
2. 进阶应用：熟练使用 `sub()` 替换、`split()` 分割、分组捕获与引用、贪婪/非贪婪匹配；
3. 实战优化：掌握零宽断言、标志位（忽略大小写、多行模式）、正则编译优化；
4. 避坑指南：重点关注元字符转义、贪婪匹配、分组索引、`match()` 与 `search()` 混淆等问题；
5. 扩展提升：复杂场景可学习正则引擎原理（NFA/DFA），或使用第三方库 `regex`（支持更多特性）。

### 核心推荐
- 正则编写：优先使用“原始字符串”（`r""`），避免转义字符冲突（如 `\d` 无需写成 `\\d`）；
- 性能优化：重复使用的正则模式必须用 `re.compile()` 编译；
- 模式调试：使用正则测试工具（如 regex101.com）验证模式正确性，提高编写效率；
- 格式验证：优先用 `fullmatch()` 确保字符串完全符合模式，避免部分匹配导致的错误；
- 复杂匹配：合理使用分组和断言，避免过度复杂的模式（可读性差、性能低）。

通过熟练掌握 re 模块，你可以高效解决各类文本处理问题，大幅提升数据处理、日志分析、爬虫开发等场景的开发效率！
