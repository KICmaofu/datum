Python3 提供了丰富的内置数据结构，这些结构是处理数据的基础，每种结构都有其独特的特性和适用场景。以下是常用数据结构的详细介绍：


### 一、序列类型（有序，可通过索引访问）
序列是 Python 中最基础的数据结构类型，特点是**元素有序排列**，可通过索引（`0` 开始）访问，支持切片、长度计算等通用操作。


#### 1. 列表（List）
**定义**：用 `[]` 表示的可变序列，元素可以是任意类型（整数、字符串、列表等），且允许重复。  
**特点**：可变（可修改、添加、删除元素）、有序、元素可重复。  

**示例**：
```python
# 创建列表
nums = [1, 2, 3, 4]
mixed = [1, "hello", True, [5, 6]]  # 元素类型可混合

# 常用操作
nums.append(5)  # 末尾添加元素 → [1,2,3,4,5]
nums.insert(0, 0)  # 指定位置插入 → [0,1,2,3,4,5]
nums.pop(2)  # 删除指定索引元素 → [0,1,3,4,5]
nums[1] = 100  # 修改元素 → [0,100,3,4,5]
print(nums[1:3])  # 切片（左闭右开）→ [100, 3]
print(len(nums))  # 长度 → 5
```

**适用场景**：需要频繁修改元素（如动态增减数据）、存储有序且可能重复的数据（如待办事项列表、日志记录）。  


#### 2. 元组（Tuple）
**定义**：用 `()` 表示的不可变序列，元素类型可混合，允许重复。  
**特点**：不可变（创建后无法修改、添加、删除元素）、有序、元素可重复。  

**示例**：
```python
# 创建元组（小括号可省略，但建议加上）
point = (3, 4)
info = ("Alice", 25, True)

# 单元素元组需加逗号（避免被解析为表达式）
single = (5,)

# 常用操作（仅查询类操作，无修改操作）
print(point[0])  # 索引访问 → 3
print(info[1:3])  # 切片 → (25, True)
print(len(info))  # 长度 → 3
```

**适用场景**：存储不可变的数据（如坐标、配置信息）、作为字典的键（因不可变）、函数多返回值（本质是元组）。  


#### 3. 字符串（String）
**定义**：用 `''` 或 `""` 表示的不可变字符序列。  
**特点**：不可变（无法修改单个字符）、有序、元素为字符。  

**示例**：
```python
s = "hello world"

# 常用操作
print(s[6])  # 索引访问 → 'w'
print(s[:5])  # 切片 → 'hello'
print(s.split())  # 分割 → ['hello', 'world']
print(s.replace("world", "python"))  # 替换（返回新字符串）→ 'hello python'
```

**适用场景**：存储文本信息（如用户名、描述）、字符串处理（拼接、分割、格式化）。  


### 二、映射类型（键值对存储）
#### 字典（Dictionary）
**定义**：用 `{key: value}` 表示的键值对集合，Python 3.7+ 后**保持插入顺序**。  
**特点**：键（key）唯一且不可变（如字符串、数字、元组），值（value）可任意类型；支持通过键快速访问值。  

**示例**：
```python
# 创建字典
person = {
    "name": "Bob",
    "age": 30,
    "is_student": False
}

# 常用操作
print(person["name"])  # 通过键访问值 → 'Bob'
person["age"] = 31  # 修改值
person["gender"] = "male"  # 添加新键值对
del person["is_student"]  # 删除键值对
print(person.keys())  # 获取所有键 → dict_keys(['name', 'age', 'gender'])
print(person.values())  # 获取所有值 → dict_values(['Bob', 31, 'male'])
```

**适用场景**：存储关联数据（如用户信息、配置项）、需要通过唯一标识（键）快速查询数据（时间复杂度 O(1)）。  


### 三、集合类型（无序，无重复元素）
#### 1. 集合（Set）
**定义**：用 `{}` 或 `set()` 表示的无序集合，元素不可重复且必须可哈希（如数字、字符串、元组）。  
**特点**：可变（可添加、删除元素）、无序（无索引）、元素唯一。  

**示例**：
```python
# 创建集合（注意：空集合需用 set()，{} 表示空字典）
nums = {1, 2, 3, 3, 4}  # 自动去重 → {1,2,3,4}
words = set(["apple", "banana", "apple"])  # → {'apple', 'banana'}

# 常用操作
nums.add(5)  # 添加元素 → {1,2,3,4,5}
nums.remove(2)  # 删除元素 → {1,3,4,5}
print(1 in nums)  # 成员检查 → True

# 集合运算
a = {1, 2, 3}
b = {3, 4, 5}
print(a & b)  # 交集 → {3}
print(a | b)  # 并集 → {1,2,3,4,5}
print(a - b)  # 差集 → {1,2}
```

**适用场景**：去重（如过滤重复数据）、集合运算（如求共同元素、差异元素）。  


#### 2. 冻结集合（Frozenset）
**定义**：用 `frozenset()` 表示的不可变集合。  
**特点**：不可变（创建后无法修改）、无序、元素唯一，可作为字典的键或其他集合的元素。  

**示例**：
```python
fs = frozenset([1, 2, 3])
# fs.add(4)  # 报错：'frozenset' object has no attribute 'add'
```

**适用场景**：需要将集合作为字典的键，或需要不可变的集合（如作为其他集合的元素）。  


### 四、collections 模块（扩展数据结构）
Python 的 `collections` 模块提供了更多高效实用的数据结构，补充了内置结构的不足。


#### 1. defaultdict（带默认值的字典）
解决普通字典访问不存在的键时抛出 `KeyError` 的问题，自动为缺失的键设置默认值（如列表、整数等）。  

**示例**：
```python
from collections import defaultdict

# 默认值为列表（用于分组）
groups = defaultdict(list)
groups["math"].append("Alice")
groups["math"].append("Bob")
print(groups)  # defaultdict(list, {'math': ['Alice', 'Bob']})

# 默认值为整数（用于计数）
counts = defaultdict(int)
counts["apple"] += 1
print(counts["apple"])  # 1
```


#### 2. deque（双端队列）
高效的双端操作队列，相比列表，`appendleft()`（左侧添加）和 `popleft()`（左侧删除）的时间复杂度为 O(1)（列表为 O(n)）。  

**示例**：
```python
from collections import deque

q = deque([1, 2, 3])
q.append(4)  # 右侧添加 → deque([1,2,3,4])
q.appendleft(0)  # 左侧添加 → deque([0,1,2,3,4])
q.popleft()  # 左侧删除 → 0，剩余 deque([1,2,3,4])
```

**适用场景**：队列（FIFO）、栈（LIFO）、滑动窗口等需要频繁首尾操作的场景。  


#### 3. Counter（计数器）
用于统计可迭代对象中元素的出现次数，返回类似字典的结构。  

**示例**：
```python
from collections import Counter

words = ["apple", "banana", "apple", "orange", "apple"]
count = Counter(words)
print(count)  # Counter({'apple': 3, 'banana': 1, 'orange': 1})
print(count.most_common(2))  # 前2个最常见元素 → [('apple', 3), ('banana', 1)]
```


#### 4. namedtuple（命名元组）
为元组的元素命名，提高代码可读性（避免通过索引访问的混乱）。  

**示例**：
```python
from collections import namedtuple

# 定义命名元组（类名，字段名）
Point = namedtuple("Point", ["x", "y"])
p = Point(3, 4)
print(p.x)  # 3（通过名称访问，比 p[0] 更清晰）
print(p.y)  # 4
```


### 五、数据结构选择指南
| 需求场景                | 推荐数据结构               | 核心原因                          |
|-------------------------|----------------------------|-----------------------------------|
| 有序+可变+可重复        | 列表（list）               | 支持动态修改，索引访问方便        |
| 有序+不可变+可重复      | 元组（tuple）              | 数据安全，适合作为字典键          |
| 键值对存储+快速查询     | 字典（dict）               | 键值映射，查询时间复杂度 O(1)     |
| 去重+集合运算           | 集合（set）                | 自动去重，支持交/并/差集          |
| 频繁首尾操作            | deque                      | 高效的 appendleft/popleft 操作    |
| 元素计数                | Counter                    | 简化计数逻辑，直接返回统计结果    |
| 提高元组可读性          | namedtuple                 | 通过名称访问元素，代码更清晰      |


### 总结
Python 数据结构的核心是根据场景选择合适的工具：有序且可变用列表，不可变用元组；键值对用字典；去重或集合运算用集合；复杂场景可借助 `collections` 模块的扩展结构。掌握这些结构的特性和操作，能大幅提升代码效率和可读性。
