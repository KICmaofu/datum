Python 3.5+ 引入了**类型注解（Type Hints）** 特性，允许开发者为变量、函数参数和返回值指定**预期的数据类型**。它的核心作用是**提供静态类型提示**（不影响Python的动态类型特性），帮助IDE、静态检查工具（如`mypy`）和其他开发者理解代码的类型逻辑，减少类型相关的错误。


### 一、为什么需要类型注解？
- **提高可读性**：明确的类型标注让其他开发者（或未来的自己）快速理解变量/函数的用途（如“这个参数应该传入整数”）。  
- **IDE支持**：主流IDE（如PyCharm、VS Code）可根据类型注解提供更精准的自动补全、语法提示和错误预警。  
- **静态检查**：通过工具（如`mypy`）在运行前检测类型不匹配的问题（如“给需要整数的参数传入了字符串”）。  


### 二、基本语法：变量与函数的类型注解
类型注解使用**冒号 `:`** 指定类型，语法简洁，且**不影响Python的动态执行**（即使注解与实际类型不符，Python解释器也不会报错，仅工具会提示）。


#### 1. 变量的类型注解
对变量标注类型的语法：  
```python
变量名: 类型 = 初始值
```

**示例**：
```python
# 基本类型注解
age: int = 20  # age预期是整数
name: str = "Alice"  # name预期是字符串
is_student: bool = True  # 布尔值

# 未初始化的变量也可注解
score: float  # 后续需赋值为浮点数
score = 95.5  # 符合注解
```


#### 2. 函数的类型注解
函数的类型注解包括**参数类型**和**返回值类型**，返回值类型用**箭头 `->`** 标注：  
```python
def 函数名(参数1: 类型1, 参数2: 类型2) -> 返回值类型:
    函数体
```

**示例**：
```python
# 函数参数和返回值的类型注解
def add(a: int, b: int) -> int:
    return a + b  # 正确：返回int，符合注解

def greet(name: str) -> str:
    return f"Hello, {name}"  # 正确：返回str

# 无返回值的函数（返回None），可用None或省略返回值注解
def print_info(age: int) -> None:
    print(f"Age: {age}")
```


### 三、复杂类型的注解（`typing` 模块）
对于复合类型（如列表、字典、元组等），需要使用标准库 `typing` 模块中的工具（Python 3.9+ 支持直接用内置类型如 `list[int]`，无需 `typing`）。


#### 1. 容器类型（列表、字典、元组等）
| 类型场景                | Python 3.9+ 语法       | Python 3.8及以下（需`typing`） |
|-------------------------|-------------------------|--------------------------------|
| 元素为int的列表         | `list[int]`             | `List[int]`（from typing import List） |
| 键为str、值为int的字典  | `dict[str, int]`        | `Dict[str, int]`               |
| 元素为float的集合       | `set[float]`            | `Set[float]`                   |
| 固定长度和类型的元组    | `tuple[int, str, bool]` | `Tuple[int, str, bool]`        |


**示例**：
```python
# Python 3.9+ 直接用内置类型
from typing import List, Dict  # Python 3.8及以下需导入

# 列表：元素为int
nums: list[int] = [1, 2, 3]  # 3.9+
# nums: List[int] = [1,2,3]  # 3.8及以下

# 字典：键str，值int
scores: dict[str, int] = {"Alice": 90, "Bob": 85}  # 3.9+
# scores: Dict[str, int] = {"Alice": 90}  # 3.8及以下

# 元组：(int, str, bool) 固定顺序和类型
person: tuple[int, str, bool] = (20, "Alice", True)
```


#### 2. 可选类型（`Optional`）
表示变量/参数可以是指定类型**或 `None`**，Python 3.10+ 可用 `|` 简化。  

**示例**：
```python
from typing import Optional  # 3.10- 需导入

# 可选类型：int 或 None
age: Optional[int] = None  # 3.10-
age: int | None = None  # 3.10+（更简洁）

# 函数参数可选
def get_name(id: int | None) -> str:
    if id is None:
        return "Unknown"
    return f"User_{id}"
```


#### 3. 联合类型（`Union`）
表示变量/参数可以是多种类型中的一种，Python 3.10+ 可用 `|` 简化。  

**示例**：
```python
from typing import Union  # 3.10- 需导入

# 联合类型：int 或 str
data: Union[int, str] = 100  # 3.10-
data: int | str = "100"  # 3.10+

# 函数参数接受int或float
def square(x: int | float) -> int | float:
    return x **2
```


#### 4. 函数类型（`Callable`）
标注“函数作为参数”或“返回值是函数”的场景，用 `Callable[[参数类型...], 返回值类型]` 表示。  

**示例**：
```python
from typing import Callable

# 函数参数是另一个函数（接受int，返回int）
def apply(func: Callable[[int], int], x: int) -> int:
    return func(x)

def double(n: int) -> int:
    return n * 2

print(apply(double, 5))  # 输出10（符合类型注解）
```


#### 5. 泛型（`TypeVar`）
用于定义“通用类型”，让函数/类支持多种类型但保持一致性（如“列表中的元素类型与返回值类型相同”）。  

**示例**：
```python
from typing import TypeVar

# 定义泛型变量T（可代表任何类型）
T = TypeVar('T')

# 函数返回列表的第一个元素，类型与列表元素一致
def get_first(lst: list[T]) -> T:
    return lst[0]

# 正确：列表元素是int，返回int
print(get_first([1, 2, 3]))  # 1

# 正确：列表元素是str，返回str
print(get_first(["a", "b"]))  # "a"
```


### 四、类型注解的“非强制性”
重要提醒：**Python不会在运行时检查类型注解**，即使实际类型与注解不符，程序也能正常运行（仅静态工具会提示错误）。  

例如：
```python
def add(a: int, b: int) -> int:
    return a + b

# 实际传入字符串，Python运行时不报错，但类型注解提示错误
result = add("1", "2")  # 运行结果："12"（不符合预期，但不报错）
```

这是因为Python本质上仍是**动态类型语言**，类型注解仅用于“提示”，而非“强制约束”。


### 五、静态类型检查工具：`mypy`
要利用类型注解检测错误，需使用静态检查工具，最常用的是 `mypy`。

#### 1. 安装 `mypy`
```bash
pip install mypy
```

#### 2. 使用 `mypy` 检查代码
创建一个 `demo.py` 文件：
```python
def add(a: int, b: int) -> int:
    return a + b

result = add("1", "2")  # 类型不匹配（参数应为int，实际是str）
```

执行检查：
```bash
mypy demo.py
```

输出错误提示：
```
error: Argument 1 to "add" has incompatible type "str"; expected "int"  [arg-type]
error: Argument 2 to "add" has incompatible type "str"; expected "int"  [arg-type]
Found 2 errors in 1 file (checked 1 source file)
```

`mypy` 会根据类型注解找出类型不匹配的问题，帮助在运行前发现错误。


### 六、类型注解的最佳实践
1.** 公共API优先 **：在库、框架的公共函数/类中使用，方便使用者理解参数和返回值类型。  
2.** 复杂逻辑必加 **：在逻辑复杂的函数（如多参数、多返回值）中添加，减少理解成本。  
3.** 配合IDE使用 **：依赖IDE（如PyCharm）的实时提示，提升开发效率。  
4.** 避免过度注解 **：简单变量（如`x = 5`）无需注解，避免代码冗余。  


### 总结
- 类型注解是**静态类型提示**，不影响Python动态执行，核心作用是提高可读性和辅助工具检查。  
- 基础语法：变量用 `x: int`，函数用 `def f(a: int) -> str`。  
- 复杂类型需用 `typing` 模块（Python 3.9+ 可直接用内置类型如 `list[int]`）。  
- 用 `mypy` 工具进行静态类型检查，提前发现类型错误。  

合理使用类型注解能让代码更健壮、更易维护，尤其适合大型项目和团队协作。
