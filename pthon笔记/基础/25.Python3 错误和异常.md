在Python中，程序运行时可能出现两种问题：**语法错误**和**异常**。语法错误是代码不符合Python语法规则导致的，而异常是代码语法正确，但运行时出现的错误（如除以零、访问不存在的文件等）。异常可以通过代码捕获并处理，避免程序直接崩溃。


### 一、语法错误（Syntax Error）
语法错误是最基础的错误，发生在Python解析代码时，因代码不符合语法规则导致。例如：

```python
# 缺少冒号（if语句后必须有冒号）
if 5 > 3
    print("5大于3")  # 报错：SyntaxError: invalid syntax
```

**解决**：修正语法错误（如上例需在`if 5 > 3`后加冒号）。


### 二、异常（Exception）
即使代码语法正确，运行时也可能因逻辑或环境问题出错，这类错误称为**异常**。例如：

```python
# 除以零（逻辑错误）
10 / 0  # 报错：ZeroDivisionError: division by zero

# 访问不存在的变量
print(x)  # 报错：NameError: name 'x' is not defined

# 打开不存在的文件
open("nonexistent.txt")  # 报错：FileNotFoundError: [Errno 2] No such file or directory: 'nonexistent.txt'
```

Python内置了多种异常类型，常见的有：

| 异常类型           | 触发场景                                  |
|--------------------|-------------------------------------------|
| `NameError`        | 访问未定义的变量                          |
| `TypeError`        | 数据类型错误（如字符串和数字相加）        |
| `ValueError`       | 数据值无效（如`int("abc")`）              |
| `ZeroDivisionError`| 除以零                                    |
| `IndexError`       | 列表/元组索引越界（如`[1,2][3]`）         |
| `KeyError`         | 访问字典中不存在的键                      |
| `FileNotFoundError`| 打开不存在的文件                          |
| `PermissionError`  | 无权限操作文件/目录                       |


### 三、异常处理：`try-except` 语句
异常处理的核心是通过 `try-except` 捕获并处理异常，让程序在出错时不会直接崩溃，而是执行预设的处理逻辑。


#### 1. 基本语法
```python
try:
    # 可能发生异常的代码块
    risky_code()
except 异常类型1:
    # 当发生异常类型1时执行的处理逻辑
    handle_error1()
except 异常类型2:
    # 当发生异常类型2时执行的处理逻辑
    handle_error2()
```

**示例**：处理除以零的异常
```python
try:
    a = 10 / 0  # 这里会触发ZeroDivisionError
    print(a)
except ZeroDivisionError:
    print("错误：除数不能为零！")  # 捕获异常后执行

# 输出：错误：除数不能为零！
```


#### 2. 捕获多个异常
可以在一个 `except` 中捕获多个异常（用元组包裹），或用多个 `except` 分别处理：

```python
try:
    num = int(input("请输入一个数字："))
    print(10 / num)
except (ValueError, ZeroDivisionError) as e:  # 捕获两种异常，用e接收异常信息
    print(f"输入错误：{e}")  # 打印异常详情

# 测试1：输入"abc" → 触发ValueError → 输出：输入错误：invalid literal for int() with base 10: 'abc'
# 测试2：输入"0" → 触发ZeroDivisionError → 输出：输入错误：division by zero
```


#### 3. 捕获所有异常（不推荐）
用 `except Exception` 可以捕获所有**非系统退出类异常**（不包括`KeyboardInterrupt`等），但可能隐藏未知错误，不推荐滥用：

```python
try:
    # 可能出错的代码
    x = 1 / 0
except Exception as e:
    print(f"发生未知错误：{e}")  # 不推荐：无法区分具体异常类型
```


#### 4. `else` 子句
`else` 可选，当 `try` 块**没有发生异常**时执行（用于区分“正常逻辑”和“异常处理”）：

```python
try:
    num = int(input("请输入数字："))
except ValueError:
    print("输入不是有效数字")
else:
    # 只有无异常时才执行
    print(f"你输入的数字是：{num}")

# 测试：输入"5" → 输出：你输入的数字是：5；输入"abc" → 输出：输入不是有效数字
```


#### 5. `finally` 子句
`finally` 可选，**无论是否发生异常，都会执行**（常用于释放资源，如关闭文件、网络连接等）：

```python
file = None
try:
    file = open("test.txt", "r")
    content = file.read()
except FileNotFoundError:
    print("文件不存在")
finally:
    # 无论是否出错，都确保文件关闭
    if file:
        file.close()
        print("文件已关闭")

# 测试：文件不存在 → 输出：文件不存在 → 文件已关闭
# 测试：文件存在 → 读取内容 → 文件已关闭
```

**注意**：`with` 语句（文件操作常用）可自动释放资源，替代 `finally` 关闭文件的操作。


### 四、主动抛出异常：`raise` 语句
除了被动捕获异常，还可以用 `raise` 主动抛出指定异常（用于验证输入、终止不合理流程等）。

**语法**：
```python
raise 异常类型(异常信息)
```

**示例**：验证年龄必须为正数
```python
def check_age(age):
    if age <= 0:
        # 主动抛出ValueError
        raise ValueError("年龄必须是正数！")
    print(f"年龄是：{age}")

try:
    check_age(-5)  # 调用时触发异常
except ValueError as e:
    print(f"检查失败：{e}")  # 输出：检查失败：年龄必须是正数！
```


### 五、自定义异常类
当内置异常类型不足以描述特定场景的错误时，可以自定义异常类（需继承 `Exception` 类，而非 `BaseException`）。

**示例**：自定义“分数超出范围”异常
```python
# 自定义异常类（继承Exception）
class ScoreOutOfRangeError(Exception):
    """分数必须在0-100之间"""
    pass  # 可自定义初始化方法或其他逻辑

def check_score(score):
    if not (0 <= score <= 100):
        # 抛出自定义异常
        raise ScoreOutOfRangeError(f"分数 {score} 无效，必须在0-100之间")

try:
    check_score(150)
except ScoreOutOfRangeError as e:
    print(f"错误：{e}")  # 输出：错误：分数 150 无效，必须在0-100之间
```


### 六、异常的传递性
当函数中发生异常且未处理时，异常会**向上传递**到调用该函数的地方，直到被捕获或导致程序终止。

```python
def func1():
    func2()  # 调用func2，未处理异常

def func2():
    func3()  # 调用func3，未处理异常

def func3():
    1 / 0  # 触发ZeroDivisionError

try:
    func1()  # 最终在顶层捕获异常
except ZeroDivisionError:
    print("捕获到func3中抛出的异常")  # 输出：捕获到func3中抛出的异常
```


### 七、异常处理最佳实践
1. **捕获具体异常**：避免用 `except Exception` 捕获所有异常，应针对性捕获已知异常（如 `FileNotFoundError`），防止隐藏未知错误。  
2. **明确异常信息**：抛出或处理异常时，提供清晰的错误信息（如 `raise ValueError("年龄必须为正数")`），方便调试。  
3. **释放资源优先用 `finally` 或 `with`**：文件、网络连接等资源需确保关闭，`with` 语句是更简洁的选择。  
4. **不滥用异常**：异常用于处理“意外错误”，而非正常流程控制（如不应通过捕获 `IndexError` 判断列表是否为空）。  


### 总结
- **语法错误**：代码不符合Python语法，必须修正才能运行。  
- **异常**：语法正确但运行时出错，可通过 `try-except` 捕获处理。  
- 核心结构：`try`（可能出错的代码）+ `except`（捕获异常）+ `else`（无异常时执行）+ `finally`（必执行，清理资源）。  
- 可通过 `raise` 主动抛异常，或继承 `Exception` 自定义异常类。  

合理的异常处理能让程序更健壮，减少崩溃，同时便于定位问题。
