在Python中，装饰器（Decorator）是一种特殊的语法结构，用于**在不修改原函数/类源代码的前提下，增强其功能**。它本质上是一个函数或类，通过包装被装饰的对象（函数或类），实现功能的扩展。


### 为什么需要装饰器？
装饰器的核心价值是**“解耦”**：将与函数核心逻辑无关的辅助功能（如日志记录、性能统计、权限校验等）抽离出来，单独维护，避免代码冗余。


### 装饰器的基础：函数的特性
装饰器依赖Python中函数的两个重要特性：
1. 函数可以作为参数传递
2. 函数可以嵌套定义，且可以返回另一个函数


### 一、函数装饰器（最常用）
函数装饰器是一个函数，它接收被装饰的函数作为参数，并返回一个新的函数（包装后的函数）。


#### 1. 无参数的函数装饰器
**示例：给函数添加“执行前后打印日志”的功能**

```python
# 定义装饰器：接收被装饰函数作为参数
def log_decorator(func):
    # 嵌套定义一个包装函数（增强原函数功能）
    def wrapper(*args, **kwargs):  # *args和**kwargs接收原函数的所有参数
        print(f"[日志] 函数 {func.__name__} 开始执行...")
        result = func(*args, **kwargs)  # 调用原函数
        print(f"[日志] 函数 {func.__name__} 执行结束...")
        return result  # 返回原函数的执行结果
    return wrapper  # 返回包装函数

# 使用装饰器：用@符号放在函数定义上方
@log_decorator
def add(a, b):
    return a + b

# 调用被装饰的函数
print(add(2, 3))
```

**执行结果**：
```
[日志] 函数 add 开始执行...
[日志] 函数 add 执行结束...
5
```

这里的`@log_decorator`等价于：`add = log_decorator(add)`，即通过装饰器重新定义了`add`函数。


#### 2. 带参数的函数装饰器
如果需要给装饰器传递参数（如日志级别、缓存过期时间等），需要在原装饰器外层再套一层“参数接收函数”。

**示例：带日志级别的装饰器**

```python
# 外层函数：接收装饰器的参数
def log_decorator_with_level(level):
    # 中层函数：接收被装饰的函数
    def decorator(func):
        # 内层包装函数：实现增强功能
        def wrapper(*args, **kwargs):
            print(f"[{level}] 函数 {func.__name__} 开始执行...")
            result = func(*args, **kwargs)
            print(f"[{level}] 函数 {func.__name__} 执行结束...")
            return result
        return wrapper
    return decorator

# 使用带参数的装饰器
@log_decorator_with_level("INFO")
def multiply(a, b):
    return a * b

print(multiply(2, 3))
```

**执行结果**：
```
[INFO] 函数 multiply 开始执行...
[INFO] 函数 multiply 执行结束...
6
```

这里的`@log_decorator_with_level("INFO")`等价于：`multiply = log_decorator_with_level("INFO")(multiply)`。


#### 3. 保留原函数的元信息
装饰器会默认覆盖被装饰函数的元信息（如`__name__`、`__doc__`），可以用`functools.wraps`解决：

```python
import functools

def log_decorator(func):
    @functools.wraps(func)  # 保留原函数元信息
    def wrapper(*args, **kwargs):
        print(f"[日志] 函数 {func.__name__} 执行...")
        return func(*args, **kwargs)
    return wrapper

@log_decorator
def add(a, b):
    """返回a和b的和"""
    return a + b

print(add.__name__)  # 若不加wraps，会输出"wrapper"；加了后输出"add"
print(add.__doc__)   # 加了wraps后输出"返回a和b的和"
```


### 二、类装饰器
类也可以作为装饰器，需实现`__call__`方法（使类的实例可被调用）。

**示例：用类装饰器统计函数调用次数**

```python
class CountCalls:
    def __init__(self, func):  # 接收被装饰的函数
        self.func = func
        self.count = 0  # 记录调用次数

    def __call__(self, *args, **kwargs):  # 实现调用逻辑
        self.count += 1
        print(f"函数 {self.func.__name__} 已被调用 {self.count} 次")
        return self.func(*args, **kwargs)

@CountCalls  # 等价于：add = CountCalls(add)
def add(a, b):
    return a + b

add(1, 2)  # 函数 add 已被调用 1 次
add(3, 4)  # 函数 add 已被调用 2 次
```


### 三、装饰器的嵌套
多个装饰器可以同时应用于一个函数，执行顺序是**从下到上**（靠近函数的装饰器先执行）。

```python
def decorator1(func):
    def wrapper(*args, **kwargs):
        print("decorator1 开始")
        result = func(*args, **kwargs)
        print("decorator1 结束")
        return result
    return wrapper

def decorator2(func):
    def wrapper(*args, **kwargs):
        print("decorator2 开始")
        result = func(*args, **kwargs)
        print("decorator2 结束")
        return result
    return wrapper

@decorator1
@decorator2  # 靠近函数，先执行
def test():
    print("test 执行")

test()
```

**执行结果**：
```
decorator1 开始
decorator2 开始
test 执行
decorator2 结束
decorator1 结束
```


### 四、装饰器的常见应用场景
1. **日志记录**：记录函数调用参数、返回值、时间等。
2. **性能分析**：统计函数执行耗时。
3. **权限校验**：函数执行前检查用户权限。
4. **缓存**：缓存函数返回结果，避免重复计算（如`functools.lru_cache`）。
5. **输入验证**：检查函数参数是否符合要求。


### 总结
装饰器是Python中“增强代码复用性”的重要工具，核心逻辑是“包装与扩展”。通过函数装饰器（无参数/带参数）或类装饰器，可以在不修改原代码的前提下，灵活地为函数或类添加功能。
