在Python中，**命名空间（Namespace）** 和**作用域（Scope）** 是理解变量访问规则的核心概念。它们共同决定了变量的可见性和生命周期，避免了命名冲突，让代码逻辑更清晰。


### 一、命名空间（Namespace）：名字的“仓库”
命名空间是一个**从变量名到对象的映射表**，用于存储变量、函数、类等名字，确保不同位置的同名名字不会冲突（因为它们属于不同的命名空间）。

#### 1. 命名空间的类型
Python中有三种主要的命名空间，按创建顺序和生命周期划分：

| 命名空间类型       | 包含的名字                                  | 创建时机                  | 销毁时机                  |
|--------------------|---------------------------------------------|---------------------------|---------------------------|
| **内置命名空间**   | Python自带的内置函数、类型（如`print`、`int`、`len`） | Python解释器启动时        | Python解释器退出时        |
| **全局命名空间**   | 模块级别的变量、函数、类（在函数/类外定义的名字） | 模块被导入时              | 模块被卸载时（如程序结束）|
| **局部命名空间**   | 函数/类方法内部定义的变量、参数；函数内的嵌套函数/类 | 函数/方法被调用时         | 函数/方法执行结束时（返回或报错）|


#### 2. 命名空间的特点
- **独立性**：不同命名空间中的同名变量互不干扰（例如，全局的`x`和函数内的`x`是两个不同的变量）。  
- **层级性**：局部命名空间嵌套在全局命名空间中，全局命名空间嵌套在内置命名空间中（查找变量时按层级顺序）。  


**示例：不同命名空间的名字**
```python
# 全局命名空间：模块级别的变量a、函数foo
a = 100

def foo(b):
    # 局部命名空间：参数b、变量c、嵌套函数bar
    c = 200
    def bar():
        d = 300  # bar的局部命名空间
        print(d)
    bar()

foo(50)

# 内置命名空间：print、int等（无需定义，直接可用）
print(int("123"))  # int属于内置命名空间
```


### 二、作用域（Scope）：名字的“可见范围”
作用域是**命名空间的可见范围**，即一个名字在程序的哪些部分可以被访问。作用域决定了“在当前位置，哪些命名空间中的名字是可访问的”。

Python中按作用域从内到外分为4种，遵循**LEGB规则**（查找变量时的优先级顺序）：

| 作用域类型       | 英文全称          | 对应命名空间          | 说明                                  |
|------------------|-------------------|-----------------------|---------------------------------------|
| **局部作用域**   | Local             | 函数/方法内部的局部命名空间 | 仅在当前函数/方法内部可见              |
| **嵌套作用域**   | Enclosing         | 外层嵌套函数的局部命名空间 | 适用于嵌套函数（内层函数访问外层函数的变量） |
| **全局作用域**   | Global            | 模块的全局命名空间     | 整个模块内可见（函数/类外定义的名字）  |
| **内置作用域**   | Built-in          | 内置命名空间           | 全程序可见（Python自带的名字）        |


#### 变量查找规则：LEGB
当访问一个变量时，Python会按以下顺序查找，找到第一个匹配的名字后停止：  
1. **局部作用域（L）**：当前函数/方法内部。  
2. **嵌套作用域（E）**：外层嵌套函数（如果有）。  
3. **全局作用域（G）**：当前模块的全局命名空间。  
4. **内置作用域（B）**：Python内置命名空间。  

如果所有作用域都找不到，会抛出 `NameError`。


**示例：LEGB规则查找变量**
```python
# 全局作用域：变量x
x = "全局"

def outer():
    # 嵌套作用域：变量x（对外层是局部，对内层是嵌套）
    x = "嵌套"
    def inner():
        # 局部作用域：变量x
        x = "局部"
        print(x)  # 查找顺序：局部→嵌套→全局→内置 → 输出"局部"
    inner()
    print(x)  # 内层的x不影响外层 → 输出"嵌套"

outer()
print(x)  # 函数内的x不影响全局 → 输出"全局"
```


### 三、全局变量与局部变量
- **全局变量**：定义在全局作用域（函数/类外）的变量，在整个模块内可见（需注意函数内修改全局变量的规则）。  
- **局部变量**：定义在局部作用域（函数/类方法内）的变量，仅在当前函数内可见，函数执行结束后销毁。  


#### 1. 函数内访问全局变量
函数内可以直接访问全局变量（无需特殊声明）：
```python
a = 100  # 全局变量

def func():
    print(a)  # 访问全局变量 → 输出100

func()
```


#### 2. 函数内修改全局变量：`global` 关键字
如果在函数内**修改全局变量**，必须用 `global` 声明该变量是全局的，否则Python会将其视为新的局部变量：
```python
a = 100  # 全局变量

def func():
    global a  # 声明a是全局变量
    a = 200  # 修改全局变量
    print("函数内的a：", a)  # 输出200

func()
print("全局的a：", a)  # 输出200（全局变量已被修改）
```

**错误示例**（未用`global`）：
```python
a = 100

def func():
    a = 200  # 这里的a是局部变量，不影响全局
    print("函数内的a：", a)  # 输出200

func()
print("全局的a：", a)  # 输出100（全局变量未变）
```


#### 3. 嵌套函数中修改外层变量：`nonlocal` 关键字
在嵌套函数中，内层函数若要**修改外层函数的局部变量**（嵌套作用域的变量），需用 `nonlocal` 声明：
```python
def outer():
    x = 10  # 外层函数的局部变量（嵌套作用域）
    def inner():
        nonlocal x  # 声明x是外层的变量
        x = 20  # 修改外层变量
        print("inner的x：", x)  # 输出20
    inner()
    print("outer的x：", x)  # 输出20（被inner修改）

outer()
```

**错误示例**（未用`nonlocal`）：
```python
def outer():
    x = 10
    def inner():
        x = 20  # 内层的局部变量，不影响外层
        print("inner的x：", x)  # 输出20
    inner()
    print("outer的x：", x)  # 输出10（未被修改）

outer()
```


### 四、命名空间与作用域的关系
- **命名空间是“存储名字的容器”**，作用域是“这些名字的可见范围”。  
- 一个作用域通常对应一个命名空间（如局部作用域对应局部命名空间），但作用域更强调“可访问性”，而命名空间强调“存储结构”。  
- 作用域是静态的（定义时确定），命名空间是动态的（运行时创建/销毁）。  


### 五、常见问题与注意事项
1. **“先使用后定义”的错误**：在函数内，若对变量有赋值操作，Python会将其视为局部变量。如果先使用再赋值，会报错：
   ```python
   a = 100

   def func():
       print(a)  # 错误：局部变量a在赋值前被引用
       a = 200  # 这里的a被视为局部变量

   func()  # 报错：UnboundLocalError: cannot access local variable 'a' where it is not associated with a value
   ```

2. **避免滥用全局变量**：全局变量在整个模块可见，过多使用会导致代码逻辑混乱，且函数的可复用性降低（依赖外部变量）。

3. **`global` 与 `nonlocal` 的区别**：  
   - `global` 用于声明变量是全局作用域的（模块级别）。  
   - `nonlocal` 用于声明变量是嵌套作用域的（外层函数的局部变量），不能用于全局变量。  


### 总结
- **命名空间**：存储名字的映射表，分为内置、全局、局部三类，避免命名冲突。  
- **作用域**：名字的可见范围，按LEGB规则（局部→嵌套→全局→内置）查找变量。  
- 函数内修改全局变量需用 `global`，嵌套函数内修改外层变量需用 `nonlocal`。  

理解命名空间和作用域，能帮助你更清晰地控制变量的访问和修改，避免常见的变量引用错误。
