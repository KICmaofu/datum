在Python中，`__name__` 是每个模块（包括脚本文件）都自带的**内置属性**，而 `__main__` 是一个特殊的字符串，用于标识“当前执行的顶层模块”。理解这两者的关系，能帮助你更灵活地控制代码的执行逻辑。


### 一、`__name__` 属性的含义
`__name__` 用于表示当前模块的“名称”，其值取决于模块的**运行方式**：

- **当模块被直接运行时**（作为脚本执行），`__name__` 的值为 `'__main__'`。
- **当模块被导入到其他模块时**，`__name__` 的值为模块本身的文件名（不含 `.py` 后缀）。


### 二、`__main__` 的含义
`__main__` 可以理解为“当前执行的主模块”的标识。当你直接运行一个 `.py` 文件时，这个文件就成为了“主模块”，其 `__name__` 被自动设置为 `'__main__'`。

简单说：`__main__` 是 Python 给“直接运行的脚本”分配的“模块名”。


### 三、核心用法：`if __name__ == '__main__'`
这个判断是 Python 中最常见的用法之一，其作用是：**区分模块是“直接运行”还是“被导入”**，从而让代码在不同场景下执行不同逻辑。

#### 场景：在模块中同时包含“功能代码”和“测试代码”
假设我们有一个 `calculator.py` 模块，定义了一个加法函数，并希望测试它：

```python
# calculator.py

def add(a, b):
    return a + b

# 测试代码：希望直接运行时执行，被导入时不执行
if __name__ == '__main__':
    print("测试加法函数：")
    print(add(2, 3))  # 输出 5
    print(add(10, 20))  # 输出 30
```


#### 两种运行方式的区别：
1. **直接运行 `calculator.py`**（作为脚本）：  
   此时 `calculator.py` 是主模块，`__name__` 的值为 `'__main__'`，因此 `if` 条件成立，测试代码会执行。  
   终端执行：`python calculator.py`，输出：
   ```
   测试加法函数：
   5
   30
   ```

2. **在其他模块中导入 `calculator`**：  
   例如创建 `use_calculator.py`，导入 `calculator`：
   ```python
   # use_calculator.py
   import calculator

   print("使用加法函数：", calculator.add(5, 5))  # 输出 10
   ```
   此时 `calculator.py` 是被导入的模块，其 `__name__` 的值为 `'calculator'`（模块名），因此 `if` 条件不成立，测试代码**不会执行**。  
   终端执行：`python use_calculator.py`，输出：
   ```
   使用加法函数：10
   ```


### 四、为什么需要这个机制？
核心目的是**“代码复用与测试分离”**：  
- 当模块作为“工具”被其他代码导入时，我们只需要它的功能（如 `add` 函数），不需要执行测试代码。  
- 当模块被直接运行时（通常是开发者自测），才需要执行测试代码验证功能。  

这个机制让一个模块既能作为“库”被复用，又能作为“脚本”独立运行，非常灵活。


### 五、扩展：`__main__` 模块的特性
- 命令行参数 `sys.argv` 只在 `__main__` 模块中有效（因为它是当前执行的入口）。  
- 交互式环境（如 Python 解释器）中，`__name__` 的值也是 `'__main__'`：
  ```python
  >>> print(__name__)
  __main__
  ```


### 总结
- `__name__` 是模块的内置属性，值随模块运行方式变化：直接运行时为 `'__main__'`，被导入时为模块名。  
- `if __name__ == '__main__'` 是区分模块“直接运行”和“被导入”的关键，用于隔离测试代码和功能代码。  
- 这个机制是 Python 模块化设计的重要组成部分，让代码更灵活、可复用。
