在Python3中，**元组（Tuple）** 是一种有序、不可变的序列类型，与列表类似，但核心区别在于**元组创建后无法修改元素**。元组常用于存储不可变的数据集，如坐标、配置信息等。


### 一、元组的定义
元组用**圆括号 `()`** 表示（也可省略括号），元素之间用逗号 `,` 分隔。元组可包含任意类型的元素（包括混合类型），且支持嵌套。


#### 1. 基本定义方式
```python
# 空元组
empty_tuple = ()
print(empty_tuple, type(empty_tuple))  # () <class 'tuple'>

# 包含多个元素的元组
nums = (1, 2, 3, 4)  # 整数元组
mixed = (10, "hello", 3.14, [1, 2])  # 混合类型（包含列表）
print(mixed)  # (10, 'hello', 3.14, [1, 2])

# 省略括号（Python允许）
fruits = "apple", "banana", "orange"
print(fruits, type(fruits))  # ('apple', 'banana', 'orange') <class 'tuple'>

# 单元素元组（必须加逗号，否则会被当作其他类型）
single_tuple = (5,)  # 正确：单元素元组
not_tuple = (5)      # 错误：这是整数5，不是元组
print(type(single_tuple))  # <class 'tuple'>
print(type(not_tuple))     # <class 'int'>
```


#### 2. 元组的核心特性
- **有序性**：元素按插入顺序排列，支持索引和切片（与列表、字符串一致）；  
- **不可变性**：创建后**不能修改、添加、删除元素**（但元组中若包含可变元素如列表，可修改列表内部的值）；  
- **可嵌套**：元组中可包含其他元组或序列（如列表、字符串）；  
- **元素可重复**：允许包含相同的值。  


### 二、元组的索引与切片
元组作为序列类型，支持与列表、字符串相同的**索引（访问单个元素）** 和**切片（获取子元组）** 操作，但因不可变，不能通过索引/切片修改元素。


#### 1. 索引（访问单个元素）
```python
t = ("a", "b", "c", "d")

# 正向索引
print(t[0])  # 'a'（第1个元素）
print(t[2])  # 'c'（第3个元素）

# 反向索引
print(t[-1])  # 'd'（最后1个元素）
print(t[-3])  # 'b'（倒数第3个元素）

# 注意：元组不可修改，以下操作报错
# t[1] = "B"  # 报错：TypeError: 'tuple' object does not support item assignment
```


#### 2. 切片（获取子元组）
语法：`t[start:end:step]`，返回一个**新元组**（原元组不变）。

```python
t = (0, 1, 2, 3, 4, 5)

# 基本切片
print(t[1:4])  # (1, 2, 3)（索引1到3的元素）
print(t[:3])   # (0, 1, 2)（从开头到索引2）
print(t[3:])   # (3, 4, 5)（从索引3到结尾）

# 步长为2
print(t[::2])  # (0, 2, 4)（间隔1个元素）

# 反向切片（倒序）
print(t[::-1])  # (5, 4, 3, 2, 1, 0)（整个元组倒序）
```


### 三、元组的常用操作
#### 1. 拼接与重复
- 拼接：用 `+` 连接两个元组（返回新元组，原元组不变）；  
- 重复：用 `*` 重复元组元素多次（返回新元组）。  

```python
a = (1, 2)
b = (3, 4)

# 拼接
print(a + b)  # (1, 2, 3, 4)（新元组）
print(a)      # (1, 2)（原元组不变）

# 重复
print(a * 3)  # (1, 2, 1, 2, 1, 2)（重复3次）
```


#### 2. 成员运算
用 `in` 或 `not in` 判断元素是否在元组中（返回布尔值）。

```python
t = ("apple", "banana", "orange")
print("banana" in t)     # True（存在）
print("grape" not in t)  # True（不存在）
```


#### 3. 长度、最大/最小值
- `len(t)`：返回元组长度（元素个数）；  
- `max(t)`/`min(t)`：返回元组中最大/最小元素（要求元素可比较）。  

```python
nums = (5, 2, 9, 1)
print(len(nums))  # 4（长度为4）
print(max(nums))  # 9（最大值）
print(min(nums))  # 1（最小值）
```


### 四、元组的内置方法
由于元组不可变，其内置方法较少，主要用于查询元素：

- `count(x)`：返回元素 `x` 在元组中出现的**次数**；  
- `index(x, start, end)`：返回元素 `x` 在元组中**第一次出现的索引**（`start` 和 `end` 可选，指定查找范围；若 `x` 不存在则报错）。  


#### 示例
```python
t = (1, 2, 3, 2, 4, 2)

# count：统计2出现的次数
print(t.count(2))  # 3

# index：查找第一个2的索引
print(t.index(2))  # 1

# 指定查找范围（索引2到5之间）
print(t.index(2, 2, 5))  # 3（在索引2-4范围内，第一个2的索引是3）
```


### 五、元组的特殊用法：解包（Unpacking）
元组支持**解包**操作，即把元组的元素依次赋值给多个变量，简化赋值过程。


#### 1. 基本解包
```python
t = (10, 20, 30)
a, b, c = t  # 解包：a=10, b=20, c=30
print(a, b, c)  # 10 20 30
```


#### 2. 部分解包（用 `*` 接收剩余元素）
若变量数少于元组长度，可用 `*` 定义一个变量接收剩余元素（返回列表）：

```python
t = (1, 2, 3, 4, 5)
first, *middle, last = t  # first=1，middle=[2,3,4]，last=5
print(first, middle, last)  # 1 [2, 3, 4] 5
```


#### 3. 嵌套元组解包
```python
t = (1, (2, 3), 4)
a, (b, c), d = t  # a=1, b=2, c=3, d=4
print(a, b, c, d)  # 1 2 3 4
```


### 六、元组与列表的区别
| 特性         | 元组（Tuple）                  | 列表（List）                    |
|--------------|--------------------------------|---------------------------------|
| 语法         | 圆括号 `()`（可省略）          | 方括号 `[]`                     |
| 可变性       | 不可变（元素不能修改/添加/删除）| 可变（支持修改/添加/删除元素）  |
| 内置方法     | 较少（仅 `count()`、`index()`） | 丰富（`append()`、`sort()` 等） |
| 内存占用     | 较小（因不可变，存储更紧凑）   | 较大（需预留修改空间）          |
| 适用场景     | 存储固定不变的数据（如坐标、配置） | 存储需动态修改的数据（如动态列表） |
| 可哈希性     | 可哈希（可作为字典的键）       | 不可哈希（不能作为字典的键）    |


### 七、注意事项
1. **元组的不可变性是“表层”的**：若元组中包含可变元素（如列表），可修改该元素的内部值（但不能替换该元素本身）。  
   ```python
   t = (1, [2, 3], 4)
   t[1].append(5)  # 允许：修改元组中的列表元素
   print(t)  # (1, [2, 3, 5], 4)

   # 但不能替换列表本身
   # t[1] = [6,7]  # 报错：TypeError（元组元素不可修改）
   ```

2. **元组的不可变性带来的优势**：  
   - 安全性：确保数据不被意外修改；  
   - 可哈希性：可作为字典的键（列表不可）；  
   - 性能：遍历和访问速度略快于列表，内存占用更少。  


### 总结
元组是有序、不可变的序列，语法上用圆括号表示（可省略），核心特性是不可变性。它支持索引、切片、拼接等操作，内置方法较少（主要用于查询），且支持便捷的解包操作。与列表相比，元组更适合存储固定不变的数据，在安全性和性能上有一定优势。掌握元组与列表的区别，能帮助你在不同场景下选择更合适的数据结构。
