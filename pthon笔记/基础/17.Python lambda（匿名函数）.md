在Python中，**lambda表达式**（也称为“匿名函数”）是一种简洁的创建函数的方式，用于定义简单的、单行的函数逻辑。与 `def` 定义的“具名函数”不同，lambda函数没有正式的名称，通常用于临时需要一个简单函数的场景（如作为其他函数的参数）。


### 一、lambda的基本语法
lambda表达式的语法非常简洁，格式如下：  
```python
lambda 参数列表: 表达式
```

- **`lambda`**：关键字，标记这是一个lambda表达式；  
- **参数列表**：与普通函数的参数规则一致，可包含0个或多个参数（如 `a, b`、`x, y=1` 等）；  
- **表达式**：lambda函数的核心逻辑，必须是**单个表达式**（不能包含循环、多条件分支等复杂语句）；  
- **返回值**：表达式的计算结果会自动作为lambda函数的返回值（无需显式写 `return`）。  


### 二、lambda与普通函数（`def`）的对比
lambda本质是函数的简化形式，与 `def` 定义的函数功能类似，但更简洁（适合单行逻辑）。

| 特性         | lambda表达式                          | def定义的函数                          |
|--------------|---------------------------------------|---------------------------------------|
| 名称         | 匿名（无名称）                        | 有名称（通过函数名调用）              |
| 语法         | 单行表达式，无代码块                  | 可包含多行代码块（缩进）              |
| 逻辑复杂度   | 仅支持单个表达式（简单逻辑）          | 支持任意复杂逻辑（循环、分支、嵌套等）|
| 返回值       | 自动返回表达式结果                    | 需显式用 `return` 返回（无则返回None）|
| 适用场景     | 临时、简单的逻辑（如作为函数参数）    | 复杂、可重用的逻辑                    |  


#### 示例：实现相同功能的lambda与def函数
```python
# 1. 计算两数之和
# def定义的函数
def add(a, b):
    return a + b

# lambda表达式（赋值给变量，通过变量调用）
add_lambda = lambda a, b: a + b

print(add(3, 5))        # 输出：8
print(add_lambda(3, 5)) # 输出：8


# 2. 计算数字的平方
square_def = def square(x):
    return x **2

square_lambda = lambda x: x** 2

print(square_def(4))     # 输出：16
print(square_lambda(4))  # 输出：16
```


### 三、lambda的参数规则
lambda的参数规则与普通函数一致，支持多种参数形式：


#### 1. 无参数
```python
# 无参数的lambda，返回固定值
hello = lambda: "Hello, lambda!"
print(hello())  # 输出：Hello, lambda!
```


#### 2. 单个参数
```python
# 单个参数：返回参数的大写形式
to_upper = lambda s: s.upper()
print(to_upper("hello"))  # 输出：HELLO
```


#### 3. 多个参数
```python
# 多个参数：计算a*b + c
calc = lambda a, b, c: a * b + c
print(calc(2, 3, 4))  # 输出：2*3 +4 = 10
```


#### 4. 默认参数
```python
# 默认参数：b的默认值为10
multiply = lambda a, b=10: a * b
print(multiply(5))    # 输出：5*10=50（使用默认b）
print(multiply(5, 3)) # 输出：5*3=15（覆盖默认b）
```


#### 5. 不定长参数（`*args`、`**kwargs`）
```python
# *args：接收任意数量的位置参数（打包为元组）
sum_args = lambda *args: sum(args)
print(sum_args(1, 2, 3))  # 输出：6

# **kwargs：接收任意数量的关键字参数（打包为字典）
print_kwargs = lambda** kwargs: kwargs
print(print_kwargs(name="Alice", age=18))  # 输出：{'name': 'Alice', 'age': 18}
```


### 四、lambda的典型使用场景
lambda的核心价值在于**简洁性**，尤其适合作为“一次性”函数参数（如传递给高阶函数）。以下是常见场景：


#### 1. 作为高阶函数的参数（如 `sorted()`、`map()`、`filter()`）
高阶函数是指接收函数作为参数的函数，lambda可简化这类场景的代码。

- **`sorted()` 排序：自定义排序规则**  
  `sorted(iterable, key=函数)` 中，`key` 函数用于指定排序依据，lambda可快速定义该函数。  
  ```python
  # 对列表中的元组，按第二个元素排序
  students = [("Alice", 20), ("Bob", 18), ("Charlie", 22)]
  # 用lambda指定key为元组的第二个元素（x[1]）
  sorted_students = sorted(students, key=lambda x: x[1])
  print(sorted_students)  # 输出：[('Bob', 18), ('Alice', 20), ('Charlie', 22)]
  ```

- **`map()`：对可迭代对象的每个元素应用函数**  
  `map(函数, 可迭代对象)` 用函数处理每个元素，返回迭代器。  
  ```python
  # 将列表中的每个数字转为其平方
  nums = [1, 2, 3, 4]
  squared = map(lambda x: x**2, nums)
  print(list(squared))  # 输出：[1, 4, 9, 16]
  ```

- **`filter()`：筛选可迭代对象中符合条件的元素**  
  `filter(函数, 可迭代对象)` 保留函数返回 `True` 的元素。  
  ```python
  # 筛选列表中的偶数
  nums = [1, 2, 3, 4, 5, 6]
  evens = filter(lambda x: x % 2 == 0, nums)
  print(list(evens))  # 输出：[2, 4, 6]
  ```


#### 2. 简化条件判断（配合三元表达式）
lambda支持三元表达式（`表达式1 if 条件 else 表达式2`），可实现简单的条件逻辑。  
```python
# 判断数字正负（正数返回1，负数返回-1，0返回0）
sign = lambda x: 1 if x > 0 else (-1 if x < 0 else 0)
print(sign(5))   # 输出：1
print(sign(-3))  # 输出：-1
print(sign(0))   # 输出：0
```


#### 3. 作为字典的值（实现简单的映射逻辑）
```python
# 用字典存储不同的计算逻辑（lambda作为值）
operations = {
    "add": lambda a, b: a + b,
    "subtract": lambda a, b: a - b,
    "multiply": lambda a, b: a * b
}

print(operations["add"](3, 5))      # 输出：8
print(operations["subtract"](10, 4)) # 输出：6
```


### 五、lambda的局限性
lambda虽然简洁，但也有明显的限制，不能替代所有函数：  
1. **只能有一个表达式**：无法包含循环（`for`/`while`）、分支（`if-elif-else` 需简化为三元表达式）、异常处理（`try-except`）等复杂逻辑；  
2. **无文档字符串**：无法像 `def` 函数那样用文档字符串（`docstring`）说明功能，降低了代码可读性；  
3. **匿名性**：没有函数名，不利于调试（错误信息中不会显示函数名），且无法直接重用（需赋值给变量才能多次调用）。  


### 总结
lambda表达式是Python中创建简单匿名函数的快捷方式，语法为 `lambda 参数: 表达式`，核心特点是简洁、单行、自动返回结果。它最适合作为高阶函数的参数（如 `sorted()`、`map()`）或实现简单的临时逻辑，但其局限性（单表达式、无文档、匿名）使其无法替代 `def` 定义的复杂函数。

使用原则：**简单逻辑用lambda，复杂逻辑用def**，平衡代码简洁性和可读性。
