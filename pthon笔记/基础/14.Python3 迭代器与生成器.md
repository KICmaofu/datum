在Python中，**迭代器（Iterator）** 和**生成器（Generator）** 是处理可迭代对象的核心机制，它们支持按需生成数据（而非一次性加载全部数据），尤其适合处理大量数据或无限序列，能显著节省内存。


### 一、迭代器（Iterator）
迭代器是一个**可以记住遍历位置的对象**，它通过实现迭代协议（`__iter__()` 和 `__next__()` 方法）来支持迭代操作。


#### 1. 迭代器的核心特性
- **可迭代（Iterable）与迭代器（Iterator）的区别**：  
  - 可迭代对象（如列表、字符串、字典等）：实现了 `__iter__()` 方法，可通过 `iter()` 函数转为迭代器；  
  - 迭代器：同时实现了 `__iter__()`（返回自身）和 `__next__()`（返回下一个元素）方法，支持通过 `next()` 函数获取下一个元素。  

- **一次性遍历**：迭代器只能向前遍历，不能后退，且元素被访问后不会重复出现。  
- **惰性计算**：迭代器不会一次性生成所有元素，而是按需返回下一个元素，节省内存。  


#### 2. 迭代器的使用流程
1. 用 `iter(可迭代对象)` 将可迭代对象转为迭代器；  
2. 用 `next(迭代器)` 逐个获取元素，直到抛出 `StopIteration` 异常（表示没有更多元素）。  


#### 3. 示例：基本迭代器操作
```python
# 可迭代对象（列表）
lst = [1, 2, 3]

# 用iter()转为迭代器
it = iter(lst)

# 用next()获取元素
print(next(it))  # 1（第一个元素）
print(next(it))  # 2（第二个元素）
print(next(it))  # 3（第三个元素）
print(next(it))  # 抛出StopIteration（无更多元素）
```


#### 4. 示例：自定义迭代器（实现迭代协议）
通过类实现 `__iter__()` 和 `__next__()` 方法，可自定义迭代器。  
**场景**：生成1到指定最大值的平方数迭代器。

```python
class SquareIterator:
    def __init__(self, max_num):
        self.max_num = max_num  # 最大值
        self.current = 1        # 当前值（从1开始）

    # 实现__iter__()：返回迭代器自身
    def __iter__(self):
        return self

    # 实现__next__()：返回下一个平方数，无元素时抛StopIteration
    def __next__(self):
        if self.current > self.max_num:
            raise StopIteration  # 终止迭代
        result = self.current **2
        self.current += 1
        return result

# 使用自定义迭代器
square_it = SquareIterator(3)
for num in square_it:  # 迭代器可直接用于for循环（自动调用next()）
    print(num)  # 1、4、9
```


### 二、生成器（Generator）
生成器是**一种特殊的迭代器**，它无需手动实现 `__iter__()` 和 `__next__()` 方法，而是通过**生成器函数**或**生成器表达式**创建，语法更简洁。


#### 1. 生成器函数（核心：`yield` 关键字）
生成器函数是包含 `yield` 关键字的函数，调用时返回一个生成器对象（而非执行函数体）。  
- 每次调用 `next()` 或遍历生成器时，函数执行到 `yield` 处暂停，返回 `yield` 后的值；  
- 再次调用时，从上次暂停的位置继续执行，直到函数结束（自动抛出 `StopIteration`）。  


#### 2. 示例：生成器函数
**场景**：生成斐波那契数列（前n项）。

```python
def fibonacci(n):
    a, b = 0, 1  # 初始值
    count = 0
    while count < n:
        yield a  # 暂停并返回a，下次从这里继续
        a, b = b, a + b  # 更新下一项
        count += 1

# 调用生成器函数：返回生成器对象（不执行函数体）
fib_gen = fibonacci(5)

# 遍历生成器
for num in fib_gen:
    print(num)  # 0、1、1、2、3
```


#### 3. 生成器表达式（简洁的生成器）
生成器表达式的语法与列表推导式类似，但用**圆括号 `()`** 表示，返回生成器对象（而非列表）。  

**语法**：  
```python
(表达式 for 变量 in 可迭代对象 if 条件)
```


#### 4. 示例：生成器表达式
```python
# 生成器表达式：生成1-5的平方
gen = (x**2 for x in range(1, 6))

# 遍历生成器
print(next(gen))  # 1
print(next(gen))  # 4
for num in gen:
    print(num)  # 9、16、25
```


#### 5. 生成器的优势
- **简洁性**：无需手动实现迭代协议，比自定义迭代器更简单；  
- **内存高效**：惰性生成元素，不会一次性占用大量内存（适合处理大文件、无限序列等）；  
- **可暂停/恢复**：通过 `yield` 实现函数的暂停和恢复，适合复杂的状态管理。  


### 三、迭代器与生成器的关系
- 生成器**本质是一种迭代器**：生成器自动实现了 `__iter__()` 和 `__next__()` 方法，可直接用于 `for` 循环和 `next()` 函数；  
- 迭代器是更通用的概念：生成器是迭代器的“简化版”，而迭代器还可以通过类手动实现；  
- 共同特性：都支持惰性计算、一次性遍历，都可用于 `for` 循环。  


### 四、典型应用场景
1. **处理大量数据**：如读取10GB的日志文件，生成器可逐行读取（而非一次性加载到内存）：  
   ```python
   def read_large_file(file_path):
       with open(file_path, 'r') as f:
           for line in f:  # 文件对象本身是迭代器，逐行返回
               yield line.strip()  # 逐行返回，节省内存

   # 遍历大文件
   for line in read_large_file("large_log.txt"):
       process(line)  # 处理每行数据
   ```

2. **生成无限序列**：如生成所有正整数（迭代器/生成器可无限生成，按需获取）：  
   ```python
   def infinite_positives():
       num = 1
       while True:
           yield num
           num += 1

   # 取前5个正整数
   pos_gen = infinite_positives()
   for _ in range(5):
       print(next(pos_gen))  # 1、2、3、4、5
   ```

3. **优化循环性能**：对于不需要存储所有结果的场景（如中间计算），生成器比列表更高效。  


### 五、总结
- **迭代器**：实现 `__iter__()` 和 `__next__()` 方法的对象，支持惰性遍历，可通过 `iter()` 从可迭代对象转换而来，或自定义类实现；  
- **生成器**：特殊的迭代器，通过 `yield` 函数或生成器表达式创建，语法简洁，无需手动管理迭代状态；  
- **核心价值**：惰性计算，节省内存，适合处理大量数据、无限序列或需要暂停/恢复的场景。  

掌握迭代器和生成器是Python高效处理数据的关键，尤其在大数据处理和性能优化中不可或缺。
